# Command - Windows - cmd - arp - IPアドレスとMACアドレスの対応情報の表示・管理

## 概要

`arp` (Address Resolution Protocol) コマンドは、IP アドレス（L3）を物理的な MAC アドレス（L2）に解決するために使用される ARP キャッシュの表示および変更を行います。

同一セグメント内の通信相手がどのハードウェア（NIC）であるかを OS がどのように認識しているかを確認できます。

(出自: `Windows 標準搭載 - TCP/IP ユーティリティ`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `getmac` | 自身の MAC アドレスを表示する。 | 自端末確認 |
| `ipconfig /all` | 自身のネットワーク設定（IP/MAC/DNS）を網羅的に表示。 | - |
| `Get-NetNeighbor` (PS) | PowerShell 版。IPv4/IPv6 両方の隣接情報をオブジェクトとして取得。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: 同一サブネット内のアクティブホストの特定 (偵察・調査)

* **目的**: ネットワークをスキャンし、生存している端末の MAC アドレスを一覧化する。
* **連携コマンド**: `for`, `ping`, `arp`
* **解説**: `ping` を打つことで相手の ARP 応答を強制し、その結果を `arp -a` で確認します。
* **コマンド例**:
	```cmd
    REM 192.168.1.1から254までpingを打ち、ARPキャッシュを更新してから表示
    for /L %i in (1,1,254) do @ping -n 1 -w 100 192.168.1.%i > nul && arp -a 192.168.1.%i
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `-a` | 現在の ARP キャッシュをすべて表示する。特定の IP を指定して表示も可能。 |
| `-g` | `-a` と同じ動作。 |
| `-d` | 特定の IP（または `*` で全て）の ARP エントリを削除する。 |
| `-s` | 静的な (Static) ARP エントリを追加する。再起動後も有効な場合が多い。 |
| `-v` | 詳細モード。無効なエントリやループバックインターフェイスも含めて表示する。 |
| `eth_addr` | MAC アドレスを指定（例: `00-aa-00-62-c6-09`）。 |
| `if_addr` | 特定のインターフェイスの IP アドレスを指定。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: IP アドレスの重複が発生していないか確認する。
* **解説**: 特定の IP に対して複数の MAC アドレスが交互に現れる場合や、意図しない MAC アドレスが紐付いている場合に、物理的な接続先を特定します。
* **例**:
	```cmd
    REM 特定のサーバーIPのMACアドレスを確認
    arp -a 192.168.1.10
    ```

### 2. ブルーチーム視点

* **タスク**: ARP スプーフィング（中間者攻撃）の検知。
* **解説**: ルーター（デフォルトゲートウェイ）の MAC アドレスが、他の一般端末の MAC アドレスと重複していないか監視します。
* **例**:
	```cmd
    REM ゲートウェイ(192.168.1.1)のMACアドレスを監視
    arp -a | findstr "192.168.1.1 "
    ```

### 3. レッドチーム視点

* **タスク**: ターゲットセグメント内のホスト列挙と横移動 (Lateral Movement) の準備。
* **解説**: `arp -a` は非常にノイズが少ない（パケットを投げない）偵察手法です。既にキャッシュにある情報は、過去に通信が発生した「生きているホスト」であることを示します。
* **例**:
	```cmd
    REM 現在のキャッシュから、自分以外の他端末を素早くリストアップ
    arp -a
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `The ARP entry addition failed: Access is denied.`
		* **考えられる原因**: 管理者権限がない。
		* **解決策**: 静的エントリの追加 (`-s`) や削除 (`-d`) には、管理者として実行したコマンドプロンプトが必要です。

2.  **エラーメッセージ例 2**: `No ARP Entires Found.`
		* **考えられる原因**: キャッシュが空。または通信が一度も発生していない。
		* **解決策**: ゲートウェイや対象 IP に一度 `ping` を打ってから再度確認してください。

## 環境変数と設定ファイル

`arp` コマンドの結果は、OS カーネル内のメモリ上のテーブル（ARP テーブル）から読み取られます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **ARP Spoofing / ARP Poisoning**。ARP プロトコルには認証がないため、偽の ARP 応答をネットワークに流すことで、通信を攻撃者の端末経由に誘導（MITM）することが可能です。
* **悪用シナリオ**: 攻撃者が「自分こそがルーターである」という偽の ARP を被害者に送り、被害者のトラフィックを盗聴・改ざんします。

### 対応策・緩和策 (ブルーチーム視点)

* **Static ARP**: 重要なサーバーやゲートウェイの MAC アドレスを `arp -s` で固定する。
* **DAI (Dynamic ARP Inspection)**: L2 スイッチの機能を使用して、不正な ARP パケットをドロップする。
* **監視**: 同一の MAC アドレスが複数の IP に紐付く（またはその逆）異常な状態を EDR や IDS で検知する。

## 注意点・補足

* **IPv6**: `arp` コマンドは IPv4 専用です。IPv6 の近隣探索情報を確認する場合は、PowerShell の `Get-NetNeighbor` または `netsh interface ipv6 show neighbors` を使用します。
* **エントリの寿命**: 動的なエントリは、一定時間（通常 2分〜10分程度）通信がないと自動的に削除されます。
