# Command - Linux - groupdel - 既存のグループを削除する

## 概要

`groupdel` は、システムから特定のグループを削除します。この操作により、`/etc/group` および `/etc/gshadow` ファイルから該当するエントリが抹消されます。ただし、安全のために「誰かのメイングループ（プライマリグループ）」として設定されているグループは削除できない仕組みになっています。

(出自: `shadow-utils` パッケージに含まれる)

## 類似・関連コマンド

| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `groupadd` | 新しいグループを作成する。 | グループ作成 |
| `userdel` | ユーザーを削除する。オプションでそのユーザー専用のグループも消せる。 | ユーザー削除 |
| `groupmod` | グループ名やGIDを変更する。 | グループの修正 |
| `groupdel` | グループそのものを完全に抹消する。 | **不要グループの整理** |

## ⚠️ 実行前の注意点

**「孤立したファイル」の発生**:

グループを削除しても、そのグループが所有者（Group ID）として設定されていたファイルやディレクトリは消えません。削除後は、所有グループが数値（旧GID）のまま残るため、後から同じGIDで別のグループを作ると、その新しいグループに古いファイルへの権限が意図せず引き継がれるリスクがあります。

## よく連携されるコマンド

### シナリオ例: プロジェクト終了に伴うグループの完全撤去 (運用視点)

* **目的**: `old_project` グループを削除し、関連するファイルの所有権を `root` に戻す。
* **手順**:
    1. グループ所有のファイルを探す (`find`)
    2. 所有権を変更する (`chown`)
    3. グループを削除する (`groupdel`)
* **コマンド例**:
    ```bash
    # 1. 'old_project' グループのファイルをリストアップ
    sudo find / -group old_project
    # 2. 安全のために所有グループを書き換えておく
    sudo find / -group old_project -exec chown :root {} +
    # 3. グループを削除
    sudo groupdel old_project
    ```

## オプション説明

※実行には **root 権限** が必須です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-f`, `--force` | 他のユーザーのプライマリグループであっても強制的に削除する（非常に危険、非推奨）。 |
| `-R <prefix>`, `--root` | 指定したディレクトリを `/` と見なして処理する（chroot 環境用）。 |
| `-h`, `--help` | ヘルプを表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーの初期セットアップにおける不要なプリセットグループの削除。
* **解説**: 特定のディストリビューションやパッケージが勝手に作成した、使用予定のないグループを削除してアタックサーフェスを最小化します。
* **例**:
    ```bash
    # 使用していないゲーム用などのグループを削除
    sudo groupdel games
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害後のクリーンアップ。
* **解説**: 攻撃者が権限維持（パーシスタンス）のために作成した「偽の管理者グループ」や「バックドア用グループ」を特定し、証拠保全の後に削除します。
* **例**:
    ```bash
    # 攻撃者が作成した不審なグループを抹消
    sudo groupdel backup_service_admin
    ```

### 3. レッドチーム視点

* **タスク**: 活動痕跡の消去（Anti-Forensics）。
* **解説**: 侵入中に利便性のために作成したグループを、撤退前に削除して調査官の目をくらまします。ただし、`groupdel` の実行自体がログに残る可能性があるため、慎重に行われます。

## エラーメッセージとトラブルシューティング

1.  **「groupdel: cannot remove the primary group of user 'alice'」**:
    * **原因**: ユーザー alice がそのグループをメインのグループとして使っている。
    * **解決策**: `usermod -g` で alice のメイングループを変更してから削除するか、alice 自体を削除する。

2.  **「groupdel: group 'xxx' does not exist」**:
    * **原因**: 既に削除されているか、名前のタイポ。
    * **解決策**: `grep xxx /etc/group` で存在を確認する。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **GIDの再利用攻撃**: 削除したグループのGIDが `1005` だったとします。その後、全く別の用途で新しいグループを作り、システムが偶然 `1005` を割り当てると、古い「消し忘れたファイル」へのアクセス権が新しいグループのメンバーに漏洩します。

### 対応策・緩和策 (ブルーチーム視点)

* **定期監査**: `find / -nogroup` を定期的に実行し、どのグループにも属していない（かつての削除の残骸）ファイルがないかチェックします。
* **ログ監視**: `/etc/group` への変更を `auditd` で監視し、誰がいつグループを消したかを記録します。

## 注意点・補足

* **NIS/LDAP環境**: ネットワーク経由でグループを管理している環境では、`groupdel` はローカルのファイルにしか作用しません。
