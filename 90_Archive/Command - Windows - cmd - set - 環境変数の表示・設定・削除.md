# Command - Windows - cmd - set - 環境変数の表示・設定・削除

## 概要

`set` コマンドは、現在のコマンド プロンプト セッションにおける環境変数を表示、設定、または削除するために使用されます。

変数はメモリ上に一時的に保持され、そのウィンドウを閉じると破棄されます。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `setx` | `set` は現在のセッションのみ。`setx` はレジストリを書き換え、永続的に変数を設定する。 | 永続化なら `setx` |
| `setlocal` | バッチファイル内で変数のスコープを限定するために使用。`endlocal` までが有効範囲。 | バッチ作成時は必須 |
| `$env:VAR` | PowerShellでの環境変数操作。オブジェクトとして扱える。 | PS環境ならこちら |

## よく連携されるコマンド

`set` は他のコマンドの出力を変数に格納したり、ループ処理の中での計算に使用されます。

### シナリオ例: コマンド結果の変数格納 (自動化)

* **目的**: 特定のファイルのフルパスを変数に格納し、後続の処理で利用する。
* **連携コマンド**: `for /f`
* **解説**: `for` 文の実行結果を `set` で変数に代入します。
* **コマンド例**:
    ```cmd
    REM カレントディレクトリの target.txt のフルパスを変数に格納
    for /f "delims=" %i in ('where target.txt') do set TARGET_PATH=%i
    echo %TARGET_PATH%
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| `set` | 引数なしで実行すると、現在のすべての環境変数を一覧表示する。 |
| `set VAR=` | 変数 `VAR` を削除（未定義状態に）する。 |
| **拡張機能** | |
| ⭐ `/a` | 数値計算（算術演算）を行う。四則演算、論理演算、ビットシフトが可能。 |
| ⭐ `/p` | ユーザーからの入力を受け取り、変数に代入する（プロンプト表示）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 動的なディレクトリ作成とログ管理
* **組み合わせ**: `%DATE%`, `%TIME%` (動的変数)
* **解説**: 現在時刻を整形し、一意のバックアップフォルダ名を作成します。
* **例**:
    ```cmd
    REM 時刻から ":" を除外して変数に格納し、フォルダを作成
    set DTS=%DATE:/=%%TIME::=%
    set DTS=%DTS: =0%
    mkdir C:\Backup\%DTS%
    ```

### 2. ブルーチーム視点

* **タスク**: 不審な環境変数の調査
* **組み合わせ**: `findstr`
* **解説**: 現在のシェルに設定されている変数を一覧し、攻撃者が設定した可能性のあるパスや、ハードコードされたパスワードがないか確認します。
* **例**:
    ```cmd
    REM 「PASS」や「SECRET」が含まれる環境変数を抽出
    set | findstr /i "PASS SECRET KEY TOKEN"
    ```

### 3. レッドチーム視点

* **タスク**: 認証情報の収集と環境認識 (Enumeration)
* **組み合わせ**: `set` (引数なし)
* **解説**: シェルを取得後、まず `set` を実行して、ドメイン名、ログオンサーバー、ユーザー名、プロセッサアーキテクチャなどの情報を一気に収集します。
* **例**:
    ```cmd
    REM 環境情報の全取得（偵察の基本）
    set
    
    REM 特定の情報のみ抽出
    echo %USERDOMAIN%
    echo %LOGONSERVER%
    echo %PROCESSOR_ARCHITECTURE%
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: (メッセージは出ないが、変数が空のまま)
    * **考えられる原因**: `set VAR = value` のように `=` の前後にスペースを入れている。
    * **解決策**: `set VAR=value` と記述する（スペースも変数名や値の一部として認識されるため）。

2.  **エラーメッセージ例 2**: `演算子が欠落しています。`
    * **考えられる原因**: `/a` スイッチでの計算式が不正、または数値以外の文字列を計算させようとした。
    * **解決策**: 計算式を見直す。変数を参照する場合は `%VAR%` ではなく、`/a` 内であれば `VAR` と直接書くことも可能。

## 環境変数と設定ファイル

### 影響を受ける環境変数

* **`%PATH%`**: `set PATH=...` とすることで、そのセッション中のみ実行ファイルの検索パスを変更できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **Credential Dumping**: 開発者が利便性のために設定した `DB_PASSWORD` や `AWS_SECRET_ACCESS_KEY` などの変数を、`set` コマンド一つで窃取される。
    * **Path Hijacking**: `set PATH=C:\Temp;%PATH%` のように、攻撃者の制御下にあるディレクトリを優先させることで、正規コマンドの実行時に悪意あるバイナリを身代わりに実行させる。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `set` は内部コマンドですが、攻撃者が環境を把握するための「標準的な偵察ツール」として必ず利用されます。
* **テクニック**: ペイロードの一部を環境変数に分割して格納し、実行時に結合させることで、シグネチャベースの検知を回避する手法があります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * 秘密情報（パスワードやAPIキー）を環境変数に直接保持せず、Secret Managerなどのセキュアなストレージを利用する。
* **Detection (検知)**:
    * 踏み台サーバーなどで `set` コマンドが頻繁に実行されたり、`PATH` 変数が不自然に書き換えられたりするログを監視する。

## 注意点・補足

* **遅延環境変数の展開**: `if` や `for` の中で `set` した値を即座に参照したい場合、`setlocal enabledelayedexpansion` を有効にし、`%VAR%` の代わりに `!VAR!` を使う必要があります。これはバッチ作成時の最大の落とし穴です。
* **永続性**: 前述の通り、`set` での設定は一時的です。再起動後も有効にしたい場合は `setx` を使用してください。
