# Command - Windows - cmd - find - 文字列の検索

## 概要

`find` コマンドは、1 つまたは複数のファイルから特定のテキスト文字列を検索します。

指定した文字列が見つかった行をすべて表示するほか、出現回数のカウントや、指定した文字列を「含まない」行の抽出も可能です。パイプ (`|`) を介して他のコマンドの出力をフィルタリングする用途で最も頻繁に利用されます。

(出自: `Windows 標準搭載 - 外部コマンド (find.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ `findstr` | **上位互換**。正規表現が使え、複数の検索条件を指定可能。 | 複雑な検索 |
| `grep` | Linux の標準。正規表現が強力。 | (Linux環境) |
| `Select-String` | PowerShell 版。出力をオブジェクトとして扱い、より精密な抽出が可能。 | PS環境 |

## よく連携されるコマンド：深掘りシナリオ

### 1. レッドチーム視点：ポストエクスプロイトにおける情報収集

* **シナリオ A: クリアテキストパスワードの探索 (T1552.001)**
    * **目的**: 無人インストールファイルや、設定ファイル内に残された平文のパスワードを迅速に特定する。
    * **連携**: `dir`, `find`
    * **コマンド例**:
        ```cmd
        REM システム内の unattend.xml を探し、その中から password という単語を含む行を抽出
        dir /s /b C:\unattend.xml | find /i "password"
        ```

* **シナリオ B: 権限昇格の足がかり（書き込み可能なサービスの探索）**
    * **目的**: サービスの設定情報を列挙し、実行パスに不備（引用符がない、特定のディレクトリ配下にある等）がないか確認する。
    * **連携**: `wmic`, `find`
    * **コマンド例**:
        ```cmd
        REM サービス一覧から、引用符で囲まれていない可能性のあるパスを簡易的に抽出
        wmic service get name,pathname | find /v """" | find /i "C:\Program Files"
        ```

* **シナリオ C: ネットワーク内の生存確認 (Ping Sweep)**
    * **目的**: 同一サブネット内のホストに Ping を飛ばし、応答があった（"Reply from"）ホストだけを表示させる。
    * **連携**: `for`, `ping`, `find`
    * **コマンド例**:
        ```cmd
        REM 192.168.1.1-254 までの生存ホストを列挙
        for /L %i in (1,1,254) do @ping -n 1 -w 100 192.168.1.%i | find "Reply from"
        ```

### 2. ブルーチーム視点：迅速な異常検知とログ解析

* **シナリオ A: 不審なネットワーク接続の監視 (Egress Filtering Check)**
    * **目的**: 外部の不審な IP やポート（例：4444, 8080）に対して「ESTABLISHED（接続確立）」状態になっている通信をリアルタイムに近い形で抽出する。
    * **連携**: `netstat`, `find`
    * **コマンド例**:
        ```cmd
        REM 接続確立済みの通信の中から、攻撃によく使われるポートを監視
        netstat -an | find "ESTABLISHED" | find "4444"
        ```

* **シナリオ B: ログオン失敗の統計把握 (Internal Reconnaissance Detection)**
    * **目的**: `wevtutil` で出力したログから、短時間での「Failure（失敗）」の発生回数をカウントし、ブルートフォース攻撃の予兆を掴む。
    * **連携**: `wevtutil`, `find`
    * **コマンド例**:
        ```cmd
        REM セキュリティログから失敗イベントの数だけをカウント
        wevtutil qe Security /q:"*[System[(EventID=4625)]]" /f:text | find /c "Failure"
        ```

* **シナリオ C: 永続化（Persistence）のチェック**
    * **目的**: 実行中のプロセスや自動起動設定の中から、本来あるべきではない場所（`Temp` や `Downloads`）から動いているものを炙り出す。
    * **連携**: `tasklist`, `find`
    * **コマンド例**:
        ```cmd
        REM 実行中プロセスの中から Temp フォルダが含まれる怪しいパスを表示
        wmic process get name,executablepath | find /i "Temp"
        ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/v` | 指定した文字列を**含まない**すべての行を表示する。 |
| ⭐ `/c` | 指定した文字列が含まれている行の**数だけ**を表示する。 |
| ⭐ `/n` | 行番号を付けて表示する。 |
| ⭐ `/i` | 文字の**大文字と小文字を区別せず**に検索する。 |
| `/off[line]` | オフライン属性が設定されているファイルをスキップしない。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サービス一覧の中から、現在「停止 (Stopped)」しているものだけをカウントして報告する。
* **例**:
    ```cmd
    REM 停止中のサービス数を取得
    sc query type= service | find /i /c "STOPPED"
    ```

### 2. ブルーチーム視点 (インシデント調査)

* **タスク**: 膨大なシステムログや hosts ファイルの中から、特定の不審なドメインや IP アドレスが含まれている箇所を特定する。
* **例**:
    ```cmd
    REM hosts ファイルに不審なリダイレクト設定がないか確認
    find /i "google.com" C:\Windows\System32\drivers\etc\hosts
    ```

### 3. レッドチーム視点 (偵察と列挙 - T1083)

* **タスク 1: Hunting for Credentials (認証情報の探索)**
    * **目的**: 設定ファイルやバッチファイルの中に "password" という単語が含まれている行を、行番号付きで特定する。
    * **コマンド例**:
        ```cmd
        REM .ini ファイルの中からパスワードを検索
        find /i /n "password" C:\inetpub\wwwroot\*.config
        ```

* **タスク 2: Environment Enumeration (OS 環境の特定)**
    * **目的**: `systeminfo` の結果を絞り込み、パッチの適用状況（KB番号）や OS 名だけを素早く抜き出す。
    * **コマンド例**:
        ```cmd
        systeminfo | find "OS Name"
        systeminfo | find "KB"
        ```

## エラーメッセージとトラブルシューティング

1.  **File not found - [filename]**
    * **原因**: 指定したファイルが存在しない。またはワイルドカードが `find` コマンド自体では直接サポートされていない（`for` 文等と組み合わせる必要があります）。
    * **解決策**: ファイルパスを確認する。

2.  **スイッチの順序**
    * **現象**: 文字列を引数の最後ではなく途中に置くと、正しく検索されない場合がある。
    * **解決策**: `find /i "string" filename` の順序を厳守してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**:
    * 攻撃者が `find` をパイプで繋いで「特定の文字列」を自動検索するスクリプトを走らせることで、短時間に大量の機密情報（トークン、APIキーなど）が持ち出されるリスクがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: Windows 標準ツールであるため、検索行動自体がセキュリティソフトに「攻撃」として検知されにくい傾向があります。

## 注意点・補足

* **正規表現不可**: `find` は単純な文字列検索のみをサポートします。ワイルドカードや正規表現を使いたい場合は `findstr` を使用してください。
* **空行の扱い**: `find /v ""` というコマンドを実行すると、空行（改行のみ）を除去してテキストを表示するテクニックとして使えます。
