# Command - Linux - nproc - 利用可能なプロセッシングユニット数を表示する

## 概要

`nproc` は、現在のプロセスが利用可能なプロセッサ（コア）の数を出力します。システムの物理的なコア数だけでなく、スケジューリングの制限（`taskset` 等）によって制限されている場合は、その「実際に使える数」を教えてくれます。

(出典: `GNU coreutils`)

## 🛠 関連コマンドとの使い分け

| コマンド | 得られる情報 | 用途 |
| :--- | :--- | :--- |
| **`nproc`** | **純粋な「数字」のみ** | スクリプト内での変数代入、並列数の決定。 |
| `lscpu` | アーキテクチャ、キャッシュ、ソケット数等。 | CPU の詳細スペック確認。 |
| `top` / `htop` | 各コアの使用率、プロセスごとの負荷。 | リアルタイムのパフォーマンス監視。 |
| `cat /proc/cpuinfo` | プロセッサごとの生データ。 | 詳細なハードウェア情報のパース。 |

## 主要なオプション

オプションは非常に少なく、シンプルです。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `--all` | OS が認識している**全ての**コア数を表示する（制限を無視）。 |
| ⭐️ `--ignore=N` | 可能であれば、`N` 個のコアを差し引いた数を出力する。 |

## コマンド例

```bash
# 現在の環境で利用可能なコア数を表示
nproc

# OS が持っている全コア数を表示（制限下でも全数を出す）
nproc --all

# スクリプト内で、コア数に合わせた並列ビルドを行う例
make -j$(nproc)
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: コンパイルやバッチ処理の最適化。
- **解説**: ソースコードのビルド（`make`）や動画のエンコードなど、マルチスレッド対応の処理を行う際、`$(nproc)` を引数に渡すことで、マシンスペックを最大限に活かした最速の処理が可能になります。

### 2. ブルーチーム視点

- **タスク**: コンテナや仮想マシンのリソース制限の確認。
- **解説**: Docker 等のコンテナ環境において、CPU クォータ制限が正しく適用されているかを確認します。`lscpu` ではホスト全数が見えてしまうことがありますが、`nproc` はそのプロセスに割り当てられた現実的な数を示します。

### 3. レッドチーム視点

- **タスク**: 効率的な攻撃（ブルートフォース等）の実行計画。
- **解説**: 侵入した環境でパスワードクラックやマイニング（※許可されたペネトレーションテストの範囲内）を行う際、コア数を確認して最適なスレッド数を割り当てることで、効率を上げつつ、CPU 使用率をあえて抑えて検知を免れるための計算材料にします。

## 注意点・補足

- **ハイパースレッディング**: `nproc` が返す数字は「論理コア数」です。物理 4 コア / 8 スレッドのマシンの場合、結果は `8` となります。
- **ポータビリティ**: `nproc` は GNU ツールキットの一部です。古い BSD 系や商用 UNIX では存在しない場合があるため、汎用スクリプトでは `getconf _NPROCESSORS_ONLN` を代わりに使用することがあります。
