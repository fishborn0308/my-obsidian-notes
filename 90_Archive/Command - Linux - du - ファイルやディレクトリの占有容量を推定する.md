# Command - Linux - du - ファイルやディレクトリの占有容量を推定する

## 概要

`du` は、指定されたファイルやディレクトリが、ディスク上のブロックをどれだけ占有しているかを再帰的に計算します。ファイルシステム上の実際のデータ量を確認するのに最適です。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `df` | ファイルシステム全体の空き状況を表示。計算が瞬時。 | 全体の空き確認 |
| `ls -lh` | 個々のファイルのサイズを表示。ディレクトリの合計は出せない。 | 単体ファイルの確認 |
| `ncdu` | `du` をインタラクティブにした TUI ツール。視覚的に分かりやすい。 | 手動での大掃除 |
| `du` | 標準的なツールで、スクリプトやパイプ処理との相性が抜群。 | **詳細な容量分析・自動化** |

## よく連携されるコマンド

### シナリオ例: 容量を食っている上位10ディレクトリの特定 (運用視点)

* **目的**: どのディレクトリが原因でディスクが圧迫されているかを迅速に特定する。
* **連携コマンド**: `sort`, `head`
* **解説**: `du` の結果を数値としてソートし、上位のみを表示します。
* **コマンド例**:
    ```bash
    # 現在のディレクトリ下で容量の大きい順にトップ10を表示
    sudo du -sh * | sort -rh | head -n 10
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐️ `-h`, `--human-readable` | 1K, 234M, 2G のように読みやすい単位で表示する。 |
| ⭐️ `-s`, `--summarize` | 引数で指定した対象の合計値のみを表示する（再帰的な中身は出さない）。 |
| `-a`, `--all` | ディレクトリだけでなく、個々のファイルについても表示する。 |
| **深度・制御** | |
| ⭐️ `-d <N>`, `--max-depth=<N>` | 指定した階層までの合計を表示する（例: `-d 1` で直下の階層のみ）。 |
| `-c`, `--total` | 最後に全体の合計値を表示する。 |
| `-x`, `--one-file-system` | 異なるファイルシステム（マウントポイント）はスキップする。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: `/var/log` 内で肥大化しているログの調査。
* **組み合わせ**: `du -m --max-depth=1`
* **解説**: MB単位で各ディレクトリのサイズを出し、調査対象を絞り込みます。
* **例**:
    ```bash
    # /var 内の各ディレクトリサイズを1階層目まで確認
    sudo du -sh /var/* --max-depth=0
    ```

### 2. ブルーチーム視点

* **タスク**: 攻撃者によって作成された「隠し大容量データ（圧縮された流出待ちデータ等）」の探索。
* **組み合わせ**: `du -a`
* **解説**: 通常は意識しない隠しディレクトリ（`.cache` や `/tmp` の奥底）に、数GBの不審なファイルがないかスキャンします。
* **例**:
    ```bash
    # /tmp 配下で 100MB 以上のファイルをリストアップ
    sudo du -am /tmp | awk '$1 > 100 {print $0}'
    ```

### 3. レッドチーム視点

* **タスク**: 持ち出し価値のあるディレクトリの特定。
* **組み合わせ**: `du -sh`
* **解説**: `/etc`, `/home`, `/var/www` などを調査し、転送時間がかかりすぎない範囲で、かつ重要なデータが詰まっていそうな場所を特定します。
* **例**:
    ```bash
    # ユーザーのホームディレクトリの容量を確認
    du -sh /home/user_name 2>/dev/null
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `du: cannot read directory '...': Permission denied`
    * **原因**: 読み取り権限がないディレクトリを計算しようとした。
    * **解決策**: `sudo` を使用するか、`2>/dev/null` を付けてエラーを非表示にする。

2.  **「ls で見たサイズと du のサイズが違う」**:
    * **原因**: **スパースファイル（スカスカなファイル）**の影響。`ls` は論理的なサイズを表示し、`du` は実際にディスク上で占有している物理的なサイズを表示します。
    * **解決策**: 意図通り。物理的な消費量を知りたいなら `du` が正解。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **情報漏洩**: `du` の実行結果から、ユーザーの活動量（どのディレクトリを頻繁に使っているか）や、インストールされているソフトウェアの規模を推測される可能性があります。

### GTFOBins における利用例

`du` が `sudo` で実行可能な場合、本来アクセスできないディレクトリ内のファイル名やその存在（およびサイズ）を確認できます。また、特定のパラメータを介してファイルを読み取る動作を悪用できる場合があります。

* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo du を使って、本来見えないディレクトリの構造を把握
    sudo du -ah /root
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/du/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 一般ユーザーに `sudo du` を無制限に許可しない。
* **Detection (検知)**: `/root` や `/etc` などに対して、一般ユーザーが `du` を繰り返し実行して構造を偵察している動きを監視する。

## 注意点・補足

* **計算コスト**: 非常に多くのファイルがあるディレクトリ（数百万ファイルなど）で `du` を実行すると、ディスク I/O が激しくなりシステム負荷が上がることがあります。
* **マウントポイント**: `-x` オプションを付けないと、ネットワークマウント（NFS）先まで計算しにいってしまい、ネットワーク帯域を消費したり、フリーズしたりすることがあります。
