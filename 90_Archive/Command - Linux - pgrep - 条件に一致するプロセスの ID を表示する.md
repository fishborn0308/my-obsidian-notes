# Command - Linux - pgrep - 条件に一致するプロセスの ID を表示する

## 概要

`pgrep` は、実行中のプロセスの名前や属性を検索し、一致したプロセスの PID を標準出力に表示します。最大の利点は、検索結果から自分自身の `pgrep` プロセスを自動的に除外してくれる点と、他のコマンド（`kill` や `renice` 等）へ PID を渡すのが非常に容易になる点です。

## 🛠 プロセス検索・操作ツールの比較

| コマンド | 役割 | 出力形式 |
| :--- | :--- | :--- |
| `ps | grep` | 検索と詳細表示 | 人間が読むための詳細なリスト |
| **`pgrep`** | **検索（PID特定）** | **PID のみ（または名前付き）** |
| `pkill` | 検索とシグナル送信 | なし（直接信号を送る） |
| `pidof` | 完全一致でのPID特定 | PID のみ |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-l`, `--list-name` | PID と一緒に**プロセス名**も表示する（確認に便利）。 |
| ⭐️ `-a`, `--list-full` | PID と一緒に**コマンドライン全体（引数含む）**を表示する。 |
| ⭐️ `-u <user>` | 特定のユーザーが所有するプロセスのみを検索する。 |
| ⭐️ `-n`, `--newest` | 条件に一致するもののうち、**最も新しく起動した**ものだけを表示。 |
| ⭐️ `-o`, `--oldest` | 条件に一致するもののうち、**最も古くから動いている**ものだけを表示。 |
| ⭐️ `-c`, `--count` | PID ではなく、一致したプロセスの**個数**を表示する。 |

## コマンド例

```bash
# 「ssh」に関連するプロセスの PID と名前を表示
pgrep -l ssh

# ユーザー「alice」が動かしている「python」プロセスのフルパスを表示
pgrep -au alice python

# 暴走した最新のプロセスを特定して nice 値を変更する（例）
renice +10 $(pgrep -n heavy_process)
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 二重起動の防止スクリプト。
- **解説**: シェルスクリプトの冒頭で `pgrep` を使い、既に同じスクリプトが動いていないかを確認します。
- **例**:

    Bash

    ```
    if pgrep -x "my_backup_script" > /dev/null; then
      echo "Already running."
      exit 1
    fi
    ```

### 2. ブルーチーム視点

- **タスク**: 隠れたバックドアプロセスの監視。
- **解説**: 定期的に `pgrep` で不審な名前（`nc`, `ncat`, `socat` 等）や、本来 root で動くべきではないプロセスの PID を監視し、異常があればアラートを飛ばします。

### 3. レッドチーム視点

- **タスク**: プロセスインジェクションのターゲット選定。
- **解説**: 権限昇格後、より安定した（あるいはより高い権限を持つ）プロセスに寄生するために、`pgrep -o` でシステムの始動時から動いている安定した親プロセスを特定します。

## 💡 プロの知恵：pkill の前の pgrep

`pkill` でプロセスを一掃する前に、必ず **`pgrep -a <パターン>`** を実行して、自分が何を殺そうとしているのかを指差し確認するのが「プロの作法」です。これにより、誤爆によるシステムダウンを未然に防ぐことができます。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: `pgrep` は `/proc` 以下の情報を読み取ります。デフォルト設定では、一般ユーザーが他人のプロセス名や起動引数を覗き見ることができてしまい、そこに含まれるパスワードなどの機密情報が漏洩するリスクがあります。

### 対応策・緩和策

- **hidepid**: 前述の `ps` 同様、`/proc` マウント時に `hidepid=2` を設定することで、他人のプロセスを `pgrep` できないように制限することが有効です。
