# Command - git - git reflog - Git操作の履歴を追跡・復元する

## 概要

`git reflog`コマンドは、ローカルリポジトリにおける**HEAD**（現在のブランチが指すコミット）やその他の参照（ブランチ、タグなど）が移動した履歴を記録するコマンドです。Gitのコミット履歴(`git log`)がコミット間の親子関係を示すのに対し、`reflog`はユーザーが行ったGit操作による参照の移動を記録します。

これにより、誤って削除してしまったコミットや、`git reset`、`git rebase`などで失われたと見えるコミットを**復元する**ための強力なツールとなります。

(出自: `git` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`git log`** | `git log`はコミット間の親子関係に基づいた履歴（グラフ構造）を表示します。`reflog`は、ローカルで行われたHEADの移動履歴（線形な操作履歴）を表示します。`log`では見えなくなったコミットも、`reflog`なら見つけられる可能性があります。 | `reflog` (操作履歴) |
| **`git fsck`** | Gitリポジトリの内部整合性をチェックし、到達不能なオブジェクトなどを検出します。`reflog`はHEADの移動履歴に特化しており、よりユーザーフレンドリーに失われたコミットを探すのに役立ちます。 | `reflog` (失われたコミット) |

## よく連携されるコマンド

`git reflog`は、主に「やり直し」や「復元」のシナリオで、他のGitコマンドと組み合わせて使用されます。

### シナリオ例: 誤ってコミットを削除してしまった場合の復元 (共通)

* **目的**: `git reset --hard`などで誤ってコミットを削除してしまった場合に、そのコミットを見つけて元の状態に復元する。
* **連携コマンド**: `git reset`
* **解説**: `git reset --hard`はローカルのコミット履歴を巻き戻す強力なコマンドですが、これにより**ブランチの履歴からは見えなくなったコミット**が生まれます。`git reflog`を使ってそのコミットのハッシュ値を見つけ出し、`git reset`で安全に復元します。
* **コマンド例**:

		```bash
    # (誤って最新の2つのコミットを削除)

    git reset --hard HEAD~2

    # git reflog で過去の操作履歴を確認し、削除前のコミットIDを探す

    git reflog

    # 出力例:
    # a1b2c3d HEAD@{0}: reset: moving to HEAD~2
    # e4f5g6h HEAD@{1}: commit: feat: Add new feature

    # 削除前の状態 (e4f5g6h) に復元

    git reset --hard e4f5g6h

    ```




## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `show [<ref>]` | (デフォルト) 指定した参照（デフォルトはHEAD）のreflogを表示します。 |
| `--all` | すべての参照（ブランチ、タグなど）のreflogを表示します。 |
| `--relative-date` | 日付を「2 days ago」のように相対的な形式で表示します。 |
| **その他** | |
| `expire` | reflogのエントリを削除します。通常、手動で実行することは稀です。 |
| `delete <entry>`| 指定したreflogエントリを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複雑な`rebase`操作に失敗し、元の状態に戻したい。
* **組み合わせ**: `git reflog`, `git reset --hard <reflog_entry>`
* **解説**: `git rebase`はコミット履歴を書き換えるため、失敗すると履歴が混乱することがあります。`reflog`はリベース前の状態を記録しているので、問題が発生した場合でも簡単に元の地点に戻り、リベースを最初からやり直すことができます。
* **例**:

		```bash
    # (rebase中に問題が発生)

    # git reflog でリベース前のブランチの状態 (例: HEAD@{2}) を確認

    git reflog

    # リベース開始前の状態に戻す

    git reset --hard HEAD@{2}

    ```




## 2. ブルーチーム視点

* **タスク**: インシデント調査で、攻撃者によって隠蔽された可能性のあるコミット履歴を追跡する。
* **組み合わせ**: `git reflog show --date=iso <branch_name>`
* **解説**: `git reflog`は、通常の`git log`では見えなくなったコミット（悪意のあるコミットや削除されたバックドアなど）の痕跡を追跡するのに役立ちます。`--date=iso`で正確なタイムスタンプを取得し、いつどのような操作が行われたかを詳細に分析できます。
* **例**:

		```bash
    # 調査対象ブランチのreflogをISO形式の正確なタイムスタンプで表示

    git reflog show --date=iso main

    ```




## 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)**。自身の活動履歴をreflogから削除する。
* **組み合わせ**: `git reflog expire --all --expire=now`
* **解説**: 侵害したシステム上でGitリポジトリを操作した場合、自身の痕跡を隠蔽するために、`reflog`を意図的にクリーンアップすることが考えられます。`expire`コマンドを使うことで、HEADの移動履歴を削除し、後のフォレンジック調査を困難にします。
* **例**:

		```bash
    # 全てのreflogエントリを即座に期限切れとしてマークし、削除する

    git reflog expire --all --expire=now --rewrite

    git gc --prune=now

    ```




## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](../linux/troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `fatal: your current branch 'main' does not have any commits yet`
		* **考えられる原因**: `git init`したばかりで、まだ一度もコミットが行われていません。
		* **解決策**: 少なくとも1つコミットを作成してから`git reflog`を実行します。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](../linux/environment_and_config.md) を参照。

### 関連する設定ファイル

* **`.git/config`**: `gc.reflogexpire` でreflogのエントリが自動的に削除されるまでの期間（デフォルトは90日）を設定できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `git reflog` 自体ではなく、**reflogの削除による痕跡消去**がリスクとなります。
* **悪用シナリオ**: 攻撃者が`git reset --hard`などで不正なコミットを隠蔽した後、`git reflog expire`で操作履歴を削除し、フォレンジック調査を妨害します。

### GTFOBins / LOLBAS における利用例

`git reflog` は直接的なコード実行機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: Gitリポジトリの定期的なバックアップとスナップショットを取得する。
* **Detection (検知)**: **Gitサーバーサイドの監査ログ**を監視する。ローカルのreflogが削除されても、サーバー側のプッシュ履歴などは残ります。

## 注意点・補足

* **ローカルのみの履歴**: `git reflog`は**ローカルリポジトリ**に固有の履歴です。リモートリポジトリには反映されませんし、他のユーザーのリポジトリにも影響を与えません。
* **失われたコミットの救世主**: `git reflog`は、Gitの操作ミスによって**履歴から見えなくなったコミット**を復元するための最後の砦です。
