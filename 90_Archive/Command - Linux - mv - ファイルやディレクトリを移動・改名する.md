# Command - Linux - mv - ファイルやディレクトリを移動・改名する

## 概要

`mv` は、1つまたは複数のファイルやディレクトリを別の場所に移動したり、名前を変更したりします。同一ファイルシステム内の移動であれば、データの実体を動かさずインデックスを書き換えるだけなので、数GBのファイルでも瞬時に完了します。

(出自: `coreutils` パッケージに含まれる)

## 🛠 類似・関連コマンド

| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `cp` | ファイルを複製する（元データが残る）。 | バックアップ作成 |
| `rename` | 複数のファイルをパターン（正規表現等）で一括改名する。 | 大量のファイル名変更 |
| `rsync` | 移動ではなく同期。ネットワーク越しや、大量のデータ転送に。 | 大規模なデータ移行 |
| `mv` | **移動、または単純なリネーム。** | **日常的な整理・改名** |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-i`, `--interactive` | 移動先に同名ファイルがある場合、上書きしていいか確認する。 |
| ⭐️ `-f`, `--force` | 確認なしで強制的に上書きする。 |
| ⭐️ `-n`, `--no-clobber` | 既存のファイルを上書きしない（既にある場合は移動をスキップ）。 |
| ⭐️ `-u`, `--update` | 移動先より新しいファイル、または存在しない場合のみ移動する。 |
| ⭐️ `-b` | 上書きされるファイルのバックアップを自動的に作成する。 |
| `-v`, `--verbose` | どのファイルがどこへ移動したか、実行経過を表示する。 |

## よく使われる構文

```bash
# 名前を変える
mv old_name.txt new_name.txt

# ファイルをディレクトリへ移動する
mv file.txt /path/to/destination/

# 複数のファイルをまとめて移動する
mv file1.txt file2.txt /path/to/destination/
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 設定ファイルの更新と世代管理。
- **解説**: 新しい設定ファイルを適用する前に、古いファイルを `.bak` や `.old` として退避させるのは鉄則です。
- **例**:

    Bash

    ```
    # 現在の config を日付付きでバックアップし、新版を配置
    sudo mv config.conf config.conf.20260223
    sudo mv config.conf.new config.conf
    ```

### 2. ブルーチーム視点

- **タスク**: 侵害調査中の証拠保全（隔離）。
- **解説**: 不審なスクリプトを発見した際、安易に削除（`rm`）せず、解析のために実行権限のないディレクトリや隔離用のパスへ `mv` します。

### 3. レッドチーム視点

- **タスク**: 痕跡の隠蔽と「紛れ込み」。
- **解説**: 侵入後に設置したツールを、システム標準のバイナリ名に似せた名前に `mv` して偽装します（例: `nc` を `syslogd` に変える等）。また、`/tmp` などの一時領域に盗み出したデータを集約する際にも多用します。

## エラーメッセージとトラブルシューティング

1. **「mv: cannot move '...' to '...': Permission denied」**:
    
    - **原因**: 移動元または移動先のディレクトリに対する書き込み権限がない。
    - **解決策**: `sudo` を使うか、ディレクトリの権限を確認（`ls -ld`）してください。
        
2. **「mv: inter-device move failed: ...; unable to remove target: Directory not empty」**:
    
    - **原因**: 異なるディスク（パーティション）を跨いでディレクトリを移動しようとし、かつ移動先に同名の空でないディレクトリがある。
    - **解説**: パーティションを跨ぐ移動は「コピー ＋ 削除」として動作するため、同一パーティション内よりも時間がかかる場合があります。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **Accidental Overwrite**: `root` 権限で `-f` を付けて `mv` を実行すると、重要なシステムファイルを一瞬で上書きし、システムを破壊する恐れがあります。
- **TOCTOU (Time of Check to Time of Use)**: 共有ディレクトリで `mv` を行う際、チェックしてから移動するまでの極わずかな隙を狙ってシンボリックリンクを差し込まれる攻撃（Race Condition）に注意が必要です。

### 対応策・緩和策 (ブルーチーム視点)

- **エイリアスの活用**: `alias mv='mv -i'` を設定し、意図しない上書きを常に防止するように徹底します。
- **監査ログ**: `/etc` などの重要ディレクトリでの `mv` 操作を `auditd` で記録し、不審な設定変更がないか監視します。
