# Command - Linux - lsusb - USB デバイスの一覧を表示する

## 概要

`lsusb` は、システム内の USB バスと、そこに接続されているすべてのデバイス情報を表示します。デバイスのベンダー名、プロダクト名、および ID（16進数）を確認でき、ドライバが正しく当たっているか、デバイスが給電されているかなどを診断する際に使用します。

(出自: `usbutils` パッケージに含まれる)

## 🛠 ハードウェア調査ツールの最終比較

| コマンド | 主な対象 | シナリオ |
| :--- | :--- | :--- |
| **`lsusb`** | **USB 接続機器** | 「キーボードが効かない」「USBメモリが認識しない」。 |
| `lspci` | PCI 接続機器 | 「GPU が認識しない」「NIC のドライバを確認したい」。 |
| `lshw` | 全ハードウェア | 「サーバー全体のスペックを棚卸ししたい」。 |
| `lsof` | 開いているファイル | 「どのプロセスがこの USB メモリ内のファイルを使っているか」。 |

## 主要なオプション

※詳細なディスクリプタ情報を表示するには **root 権限 (sudo)** が必要です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-v`, `--verbose` | **推奨。** 非常に詳細な情報を表示（エンドポイント、最大電流消費など）。 |
| ⭐️ `-t`, `--tree` | 物理的な接続関係をツリー形式で表示。どのハブに何が刺さっているか一目でわかる。 |
| ⭐️ `-d [vendor]:[product]` | 特定の ID を持つデバイスのみを表示。 |
| ⭐️ `-s [bus]:[devnum]` | バス番号とデバイス番号を指定して表示。 |
| `-D <device>` | デバイスファイルを直接指定して詳細を表示。 |

## 💡 出力項目の読み解き

`Bus 002 Device 003: ID 046d:c52b Logitech, Inc. Unifying Receiver`

* **Bus 002**: 接続されている USB バスの番号。
* **Device 003**: そのバス内でのデバイス番号（抜き差しすると変わる）。
* **ID 046d:c52b**: ⭐️ **重要。** 前半がベンダー ID、後半がプロダクト ID。
* **Logitech, Inc...**: デバイスの名称。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: 消費電力オーバーの調査。
* **解説**: USB ハブに多くのデバイスを繋いで動作が不安定な際、`lsusb -v` で各デバイスの `MaxPower` を確認し、給電不足が起きていないか診断します。
* **例**:
    ```bash
    # 消費電力を確認する（膨大な出力から MaxPower を抽出）
    sudo lsusb -v | grep -E "Bus|Device|MaxPower"
    ```

### 2. ブルーチーム視点

* **タスク**: BadUSB や不正デバイスの検知。
* **解説**: 物理的なセキュリティ侵害を調査します。正規のキーボードに見えて、実は `lsusb` 上では「ハブ + キーボード + ストレージ」として認識されているような、不自然な複合デバイス（Rubber Ducky 等）を特定します。

### 3. レッドチーム視点

* **タスク**: 物理アクセスによるペイロード実行準備。
* **解説**: ターゲットマシンに物理的に接触できる場合、自分の持ち込んだツールが正しく認識されているか、またアンチウイルス（AV）や EDR が特定のベンダー ID をブロックしていないかを確認します。
* **例**:
    ```bash
    # ツリー形式で見やすく表示し、自分のデバイスがどこにぶら下がっているか確認
    lsusb -t
    ```

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **BadUSB 攻撃**: USB のファームウェアを書き換え、キーボードとして振る舞わせることで、OS に任意のコマンドを打ち込ませる攻撃です。`lsusb` で見ると「身に覚えのない HID デバイス」として現れます。
* **情報漏洩**: 接続されているデバイス名から、その端末のユーザーの私物（スマホの機種名など）が判明し、プライバシーの侵害や、さらなる標的型攻撃のヒントになるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)

* **USB ポートの物理的な封印**: 未使用のポートを物理的に塞ぐ、またはマザーボードの設定（BIOS/UEFI）で無効化します。
* **USBGuard の導入**: `lsusb` で得られる ID に基づき、許可されたデバイス以外が接続された瞬間にブロックするソフトウェア（USBGuard 等）を運用します。

## 注意点・補足

* **名前の出ないデバイス**: `ID 046d:c52b` の後に名前が表示されない場合、`/var/lib/usbutils/usb.ids` が古いです。`sudo update-usbids` で最新のデータベースに更新しましょう。
