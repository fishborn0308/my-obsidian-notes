# Command  - Linux - cp - ファイルやディレクトリをコピーする

## 概要

`cp` (copy) コマンドは、ファイルやディレクトリを複製するための基本的なコマンドです。主に2つの使い方があります。

1. **ファイルを別のファイルとしてコピーする**: `cp <コピー元ファイル> <コピー先ファイル名>`
2. **1つまたは複数のファイルやディレクトリを、指定したディレクトリ配下にコピーする**: `cp <ファイル1> <ファイル2> ... <コピー先ディレクトリ>`

日々のバックアップ作成から、設定ファイルの編集前の一時保存、アプリケーションのデプロイまで、幅広く利用されます。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`mv`** | `mv` はファイルを**移動**または**名前変更**します。元のファイルは残りません。`cp` はファイルを**複製**し、元のファイルはそのまま残ります。 | - |
| **`rsync`** | より高機能なファイル同期ツール。ネットワーク越しのコピーや、差分転送により2回目以降のコピーが非常に高速です。ディレクトリのバックアップには `rsync` が適しています。 | `rsync` (ディレクトリ同期) |
| **`scp`** | SSHプロトコルを使い、ネットワーク上の異なるホスト間で**安全に**ファイルをコピーします。`cp` はローカルホスト内でのコピーに限定されます。 | - |

## よく連携されるコマンド

`cp` は他のコマンドと組み合わせることで、より柔軟なファイル操作を実現します。

### シナリオ例: タイムスタンプ付きで設定ファイルをバックアップ (インフラ構築・運用視点)

* **目的**: 編集前に、設定ファイル `nginx.conf` のバックアップを、現在の日時を付けたファイル名で作成する。
* **連携コマンド**: `date`, コマンド置換 (`$()`)
* **解説**: コマンド置換 `$(date +%Y%m%d-%H%M%S)` を使って、ファイル名に年月日-時分秒のタイムスタンプを動的に埋め込みます。これにより、いつの時点のバックアップかが一目瞭然になります。
* **コマンド例**:

```bash
# nginx.conf をタイムスタンプ付きのファイル名でコピーする
cp -v /etc/nginx/nginx.conf /etc/nginx/nginx.conf.$(date +%Y%m%d-%H%M%S)
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ファイル/ディレクトリ操作** | |
| ⭐ `-r`, `-R`, `--recursive` | ディレクトリを再帰的にコピーします。ディレクトリをコピーする際の必須オプションです。 |
| ⭐ `-a`, `--archive` | **アーカイブモード**。`-dR --preserve=all` とほぼ同等で、再帰的なコピーに加え、シンボリックリンク、パーミッション、タイムスタンプなど、可能な限り全ての属性を保持します。**ディレクトリの完全なバックアップに最適です。** |
| `-p`, `--preserve` | ファイルのパーミッション、所有者・グループ、更新日時などの属性情報を保持したままコピーします。 |
| **動作制御** | |
| ⭐ `-i`, `--interactive` | コピー先に同名ファイルが存在する場合、上書きしてよいか確認を求めます。誤操作防止に有効です。 |
| `-f`, `--force` | `-i` の逆。確認プロンプトを表示せず、強制的に上書きします。 |
| `-u`, `--update` | コピー元のファイルがコピー先のファイルより新しい場合、またはコピー先にファイルが存在しない場合にのみコピーを実行します。 |
| **出力制御** | |
| `-v`, `--verbose` | コピーされたファイル名などを画面に表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Webサーバのドキュメントルートを属性情報ごと完全にバックアップする。
* **組み合わせ**: `cp -a <source_dir>/ <backup_dir>/`
* **解説**: `-a` (アーカイブ)オプションは、ディレクトリを属性情報ごと完全に複製するための最も信頼性の高い方法です。
* **例**:

```bash
# Webサーバのドキュメントルートを丸ごとバックアップ
sudo cp -a /var/www/html/ /mnt/backups/www-backup-$(date +%F)/
```

### 2. ブルーチーム視点

* **タスク**: 証拠保全のため、ファイルのメタデータを維持したまま収集する。
* **組み合わせ**: `sudo cp -p /path/to/evidence /forensic/images/`
* **解説**: `-p` (`preserve`) オプションは、ファイルのメタデータ（特に最終更新日時 `mtime`、最終アクセス日時 `atime`）を保持します。タイムスタンプは「いつ、何が起きたか」を示す重要な情報源であるため、これを変更せずにコピーすることが極めて重要です。
* **例**:

```bash
# 侵害されたWebサーバから、改ざんされた可能性のあるファイルをタイムスタンプを維持したまま収集する
sudo cp -p /var/www/html/uploads/shell.php /mnt/evidence/case-001/
```

### 3. レッドチーム視点

* **タスク**: 権限昇格攻撃の準備段階として、システムの正規コマンドを書き込み可能なディレクトリにコピーする。
* **組み合わせ**: `cp /bin/bash /tmp/mybash`
* **解説**: システムの `/bin/bash` を直接変更せずに、コピーしたファイルを悪用して権限昇格を試みます。これは検知回避（Defense Evasion）の一環でもあります。
* **例**:

```bash
# bashをコピーし、後でSUID化するなどの準備をする
cp /bin/bash /dev/shm/shell
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `cp: -r not specified; omitting directory 'some_directory'`
		* **考えられる原因**: ディレクトリをコピーしようとしましたが、再帰的コピーを指定する `-r` または `-a` オプションを付けていません。
		* **解決策**: `cp -r <source_dir> <dest_dir>` のように、`-r` または `-a` オプションを追加します。

2. **現象**: **隠しファイル（ドットファイル）がコピーされない**
		* **考えられる原因**: `cp -r sourcedir destdir` のようにコマンドを実行すると、`sourcedir` 直下の `.bashrc` のような隠しファイルはコピーの対象になりません。
		* **解決策**: `cp -r sourcedir/. destdir/` のように、コピー元の末尾に `/.` を付けます。これにより、隠しファイルを含む全てのファイルが対象になります。

* `cp` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `cp` 自体は安定したコアユーティリティであり、脆弱性を持つ可能性は低いです。
* **悪用シナリオ**:
	* **データ窃取**: `cp ~/.ssh/id_rsa /tmp/key` のように、機密情報を `/tmp` やWebサーバの公開ディレクトリなどの外部からアクセスしやすい場所にコピーし、情報窃取の準備をします。
	* **永続化**: 既存の正規のプログラムを悪意のあるものに置き換えるために使われます。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格), **File Write** (ファイル書き込み)
* **解説**: もしユーザーが `sudo` で `cp` を実行できる権限を持っている場合、システム上の**あらゆるファイルを上書き**できます。例えば、`sudo cp known_shadow /etc/shadow` のようにしてパスワードハッシュを既知のものに置き換えたり、`sudoers` ファイルを自分に都合の良い内容に書き換えたりすることで、root権限を奪取できます。
* **参照**: `https://gtfobins.github.io/gtfobins/cp/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、ユーザーに `cp` のような汎用性の高いコマンドの `sudo` 権限を安易に与えない。
* **Detection (検知)**: `auditd` を用いて、`/etc/shadow`, `/etc/sudoers` などの重要ファイルへの書き込みイベントを監視する。

## 注意点・補足

* **デフォルトでの上書き**: `cp` は、コピー先に同名ファイルが存在すると**警告なしに上書きします**。多くのディストリビューションでは、`alias cp='cp -i'` がデフォルトで設定されていますが、スクリプト内などではこのエイリアスは有効にならないため注意が必要です。
