# Command - Windows - cmd - time - システム時刻の表示・設定

## 概要

`time` コマンドは、現在のシステム時刻を表示、または新しい時刻を設定するために使用されます。

Windows の Active Directory 環境では、サーバーとクライアントの時刻が 5 分以上ずれると Kerberos 認証に失敗するため、このコマンドでの確認は基本中の基本となります。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `date` | システムの「日付」を表示・設定する。 | セットで使用 |
| `w32tm` | Windows タイム サービスの構成・診断ツール。NTP 同期状況の確認に。 | `w32tm` (詳細調査) |
| `Get-Date` | PowerShell 版。日付と時刻をオブジェクトとして取得。書式変更が容易。 | `Get-Date` (PS) |

## よく連携されるコマンド

### シナリオ例: スクリプトの実行時間の計測 (デバッグ)

* **目的**: 簡易的にバッチファイルの処理時間をログに記録する。
* **連携コマンド**: `echo`, リダイレクト (`>>`)
* **解説**: `/t` スイッチを使用して、余計なプロンプトを出さずに現在の時刻のみを出力します。
* **コマンド例**:
    ```cmd
    echo 処理開始: >> log.txt
    time /t >> log.txt
    rem [メイン処理]
    echo 処理終了: >> log.txt
    time /t >> log.txt
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| (なし) | 現在の時刻を表示し、新しい時刻の入力を求める。 |
| ⭐ `/t` | **重要。** 現在の時刻のみを表示し、新しい時刻の入力を求めない（スクリプト向き）。 |
| `HH:mm:ss` | コマンドの後に直接時刻を入力することで、対話なしで時刻を設定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Kerberos 認証エラー（時刻同期ずれ）の確認
* **解説**: ドメイン参加に失敗する際、DC（ドメインコントローラー）とクライアントの時刻を手動で突き合わせます。
* **例**:
    ```cmd
    REM ドメインコントローラー(DC01)の時刻を表示
    net time \\DC01
    REM 自分のローカル時刻を表示
    time /t
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害発生時刻の特定（タイムライン分析）
* **解説**: 攻撃者が残したファイルのタイムスタンプやイベントログの時刻を、システム設定のタイムゾーンと照らし合わせて分析します。
* **例**: `time /t` で現在のシステム時刻を確認し、攻撃者が「どの時点」で活動していたかの基準にします。

### 3. レッドチーム視点 (Anti-Forensics)

* **タスク**: タイムスタンプの操作（Timestomping）
* **解説**: 自分の活動をログに残さないため、あるいは「過去に実行されたように見せる」ために、一時的にシステム時刻を変更してファイルを作成・編集します（※管理者権限が必要）。
* **例**:
    ```cmd
    REM 一時的に時刻を1時間前に戻す
    time 14:00:00
    REM [痕跡を残す操作]
    REM 時刻を元に戻す
    ```

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `クライアントは必要な特権を保有していません。`
    * **原因**: システム時刻の設定には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」する。

2.  **現象**: 時刻を変更してもすぐに元に戻ってしまう。
    * **原因**: Windows Time サービス（NTP 同期）が有効で、自動的に同期されている。
    * **解決策**: `sc stop w32time` で一時的に同期を止める。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Authentication Bypass**: 時刻を大幅にずらすことで、有効期限切れの証明書を一時的に有効に見せかけたり、ワンタイムパスワード（TOTP）の生成タイミングを操作したりする。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    - **イベントID 4616**: システム時刻が変更されたことを記録します。通常、時刻同期サービス以外による頻繁な変更は異常とみなすべきです。

## 注意点・補足

* **書式**: 時刻の区切り文字や 12/24 時間制の表示は、コントロールパネルの「地域」の設定に依存します。
* **精度**: `time` コマンドで表示されるのは秒単位までですが、内部的にはより細かい精度で保持されています。
