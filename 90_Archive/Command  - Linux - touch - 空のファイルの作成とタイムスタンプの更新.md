# Command  - Linux - touch - 空のファイルの作成とタイムスタンプの更新

## 概要

`touch` は、主に2つの機能を持つ基本的なコマンドです。

1. **ファイルの作成**: 指定したファイルが存在しない場合、サイズが0の**空のファイルを作成**します。
2. **タイムスタンプの更新**: 指定したファイルが既に存在する場合、そのファイルの**アクセス時刻**と**変更時刻**を、現在の時刻に更新します。

プログラムが特定のファイルの存在を前提とする場合のプレースホルダー作成や、`make` のようなビルドツールへの通知、フォレンジック調査におけるタイムスタンプ操作などに利用されます。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`mkdir`** | **ディレクトリ**を作成します。`touch` は**ファイル**を作成します。 | - |
| **`> filename`** | (リダイレクション) これも空のファイルを作成できますが、主な目的はコマンドの標準出力をファイルに書き込むことです。`touch` はタイムスタンプの更新に特化しています。 | `touch` |
| **`stat`** | ファイルのタイムスタンプ（アクセス、変更、ステータス変更）を**表示**します。`touch` はそれらを**更新**します。 | - |

## よく連携されるコマンド

`touch` は、`chown` と連携して、特定の所有者で空のファイルを作成する際によく使われます。

### シナリオ例: アプリケーション用のログファイルを事前に作成する (インフラ構築・運用視点)

* **目的**: デーモンが起動時に必要とするログファイルを、適切な所有者とパーミッションで事前に作成しておく。
* **連携コマンド**: `chown`, `chmod`
* **解説**: `touch` で空のファイルを作成した後、`chown` と `chmod` で、アプリケーションの実行ユーザーがそのファイルに書き込めるように権限を整えます。
* **コマンド例**:

		```bash
		# 1. 空のログファイルを作成

		sudo touch /var/log/my-app.log

		# 2. 所有者をアプリケーションユーザーに変更

		sudo chown my-app-user:my-app-group /var/log/my-app.log

		# 3. 適切なパーミッションを設定

		sudo chmod 644 /var/log/my-app.log

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ファイル作成/非作成** | |
| ⭐ `<filename>` | ファイル名。存在しない場合は空のファイルを作成。存在する場合はタイムスタンプを更新。 |
| `-c`, `--no-create`| ファイルが存在しない場合、**新しいファイルを作成しません**。 |
| **タイムスタンプ制御** | |
| ⭐ `-r <ref_file>` | 現在時刻の代わりに、**参照ファイル (`ref_file`) と同じタイムスタンプ**を設定します。 |
| `-a` | **アクセス時刻 (access time)** のみ更新します。 |
| `-m` | **変更時刻 (modification time)** のみ更新します。 |
| `-d <string>`| `"2 days ago"` や `"2025-01-01 10:30"` のような指定した日時の文字列をタイムスタンプとして使用します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションが必要とする空のログファイルやロックファイルを事前に作成する。
* **組み合わせ**: `touch <filename>`
* **解説**: デーモンが起動時に特定のファイルを要求する場合、`touch` で先に作成しておくことで、起動エラーを防ぎます。
* **例**:

		```bash
		# アプリケーション用のロックファイルを作成

		touch /var/run/my-app.pid

		```

## 2. ブルーチーム視点

* **タスク**: **ハニーポット（おとり）ファイル**を作成し、不正アクセスを検知する。
* **組み合わせ**: `touch /path/to/honeypot-file.txt`
* **解説**: 攻撃者が侵入後に探索しそうな場所に、`passwords.txt` のような魅力的な名前の空ファイルを設置します。そして `auditd` などで、そのファイルへの全てのアクセスを監視します。もしこのファイルへのアクセスがあれば、それは不審な偵察活動の強力な兆候となります。
* **例**:

		```bash
		# 秘密鍵に見せかけたおとりファイルを作成

		touch /home/admin/.ssh/id_rsa.bak

		```

## 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)** のため、ファイルのタイムスタンプを偽装する。
* **組み合わせ**: `touch -r <original_file> <modified_file>`
* **解説**: 攻撃者は正規のスクリプトなどを改ざんした後、`-r` オプションを使って、改ざんしたファイルのタイムスタンプを**元のファイルと全く同じ**に戻します。これにより、`ls -lt` などによる単純な変更検知を回避しようとします。
* **例**:

		```bash
		# 1. 元のファイルのタイムスタンプを一時ファイルにコピー

		touch -r /etc/important_script.sh /tmp/timestamp_ref

		# 2. (important_script.sh を改ざん)

		echo "evil_command" >> /etc/important_script.sh

		# 3. タイムスタンプを元に戻す

		touch -r /tmp/timestamp_ref /etc/important_script.sh

		```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **エラーメッセージ例 1**: `touch: cannot touch '<filename>': Permission denied`
		* **考えられる原因**: ファイルを作成しようとしているディレクトリへの書き込み権限がないか、既存のファイルのタイムスタンプを変更しようとしたが、そのファイルの所有者ではありません。
		* **解決策**: ディレクトリやファイルのパーミッションを確認し、必要であれば `sudo` を付けて実行してください。

## 環境変数と設定ファイル

* `touch` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `touch` 自体ではなく、それが持つ**タイムスタンプの偽装 (Timestomping)** 機能が悪用されます。
* **悪用シナリオ**: 攻撃者は `touch -r` を使い、バックドアを仕込んだファイルの最終変更日時を偽装し、管理者による発見を遅らせようとします。

### GTFOBins / LOLBAS における利用例

* **機能**: **File Write**, **Privilege Escalation**
* **解説**: もしユーザーが `sudo touch` を実行できる権限を持っている場合、システム上の**任意の場所に空のファイルを作成**できます。これは、特定の場所にファイルが存在すること自体がトリガーとなるような設定を悪用するのに使えます。
* **参照**: `https://gtfobins.github.io/gtfobins/touch/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: ファイルのタイムスタンプだけでなく、ファイルの**ハッシュ値の変更**をファイル整合性監視(FIM)ツールで監視することが不可欠です。`auditd` で `touch` コマンドの実行や、`utime` システムコールの呼び出しを監視します。

## 注意点・補足

* **複数ファイルの作成**: `touch file1 file2 dir/file3` のように、一度に複数のファイルを指定して作成できます。
* **ブレース展開の活用**: `touch image_{001..100}.jpg` のように、シェルのブレース展開と組み合わせることで、連番のファイルを一括で作成できます。
