# Command - Linux - pkill - 名前や属性に基づいてプロセスに信号を送る

## 概要

`pkill` は、指定したパターン（正規表現）に一致する名前を持つプロセスに対して、シグナル（デフォルトは `SIGTERM`）を送信します。`kill` コマンドが PID を必要とするのに対し、`pkill` はプロセス名で直接指定できるため、複数のプロセスをまとめて終了させる際に非常に強力です。

## 🛠 関連コマンドの使い分け

| コマンド | ターゲットの指定方法 | 特徴 |
| :--- | :--- | :--- |
| `kill` | **PID (数字)** | 最も基本的。誤爆が少ない。 |
| **`pkill`** | **プロセス名 / パターン** | 複数一括操作に最適。オプションが豊富。 |
| `killall` | **プロセス名 (完全一致)** | 指定した名前の全プロセスを殺す。 |
| `pgrep` | 名前から PID を探す | 殺さずに「生存確認」だけする場合に使用。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-f`, `--full` | 引数を含む**コマンドライン全体**をマッチ対象にする。 |
| ⭐️ `-u <user>` | 特定のユーザーが実行しているプロセスのみを対象にする。 |
| ⭐️ `-t <tty>` | 特定のターミナル（端末）に関連付けられたプロセスを対象にする。 |
| ⭐️ `-9`, `--signal KILL` | 強制終了シグナル (`SIGKILL`) を送る。 |
| ⭐️ `-v`, `--inverse` | 指定した条件に**一致しない**プロセスを対象にする。 |
| `-n`, `--newest` | 一致するプロセスのうち、最も新しいものだけを対象にする。 |

## コマンド例

```bash
# 「firefox」という名前を含むプロセスをすべて終了
pkill firefox

# 特定のユーザー (alice) のプロセスをすべて強制終了
sudo pkill -u alice -9

# 設定ファイルを再読み込みさせる (HUPシグナル)
sudo pkill -HUP nginx
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: ゾンビ化、またはハングアップした古いセッションの掃除。
- **解説**: 特定の端末 (`pts/1` など) で固まってしまったシェルやエディタを、端末番号指定で一掃します。
- **例**:

    Bash

    ```
    # pts/1 に紐づく全プロセスを終了
    pkill -t pts/1
    ```

### 2. ブルーチーム視点

- **タスク**: 不審なシェルの即時遮断。
- **解説**: 侵入者が `nc` (Netcat) でリバースシェルを張っている疑いがある場合、名前ベースで一気にプロセスを止め、通信を遮断します。
- **例**:

    Bash

    ```
    # netcat 関連のプロセスをすべて殺す
    sudo pkill -f nc
    ```

### 3. レッドチーム視点

- **タスク**: セキュリティ製品や監視ツールの無効化。
- **解説**: 侵入後、自分の行動をログに記録したり、検知したりするエージェント（例: `auditd`, `syslog`, EDR関連）を `pkill` で密かに停止させます。
- **例**:

    Bash

    ```
    # 監視デーモンを停止 (権限が必要)
    sudo pkill auditd
    ```

## 💡 プロのテクニック：予行演習

`pkill` は強力すぎて、間違ったプロセスを巻き込む「誤爆」が怖いです。実行前に必ず **`pgrep`** を同じ引数で実行し、どの PID が対象になるか確認する癖をつけましょう。

Bash

```
# 1. まず確認
pgrep -fl my_script

# 2. 対象が正しいことを確認してから仕留める
pkill -f my_script
```

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **権限昇格の副作用**: root で `pkill` を実行する際、パターンの指定が甘いと（例: `pkill -f .`）、システムに必要な重要プロセスをすべて殺してしまい、カーネルパニックやシステムダウンを引き起こす「セルフ DoS」のリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)

- **sudo の制限**: `/etc/sudoers` で、一般ユーザーが `pkill` を使える範囲（特に `-9` オプションや root プロセスへの実行）を制限することを検討します。
