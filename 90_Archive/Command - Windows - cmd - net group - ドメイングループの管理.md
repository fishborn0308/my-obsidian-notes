# Command - Windows - cmd - net group - ドメイングループの管理

## 概要

`net group` コマンドは、Active Directory ドメイン内のグローバルグループを追加、表示、および変更するために使用されます。

ドメイン全体のユーザーを論理的なグループ（部署別、権限別など）にまとめ、リソースへのアクセス権を一括管理する際に使用されます。実行にはドメイン環境への接続（ドメイン参加済み端末またはDCへの照会）が必要です。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `net localgroup` | **ローカル**コンピュータ上のグループを管理する。 | ローカル権限管理 |
| `Get-ADGroup` | PowerShell ADモジュール。より詳細な属性（SID、作成日等）を取得可能。 | AD管理ツール導入環境 |
| `ADUC` (GUI) | 「Active Directory ユーザーとコンピューター」。視覚的な管理。 | - |

## よく連携されるコマンド

### シナリオ例: ドメイン管理者リストの特定 (レッドチーム視点)

* **目的**: 標的ドメインにおいて、最も高い権限を持つユーザー（Domain Admins）を特定する。
* **連携コマンド**: `net group`, `net user`
* **解説**: `/domain` スイッチを使用（またはドメイン参加端末から実行）して、特権グループのメンバーを列挙します。
* **コマンド例**:
    ```cmd
    REM ドメイン内の全グループを一覧表示
    net group /domain

    REM "Domain Admins" グループのメンバーを確認
    net group "Domain Admins" /domain
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | ドメイン内の全てのグローバルグループを一覧表示する。 |
| `groupname` | 指定したドメイングループのメンバーを一覧表示する。 |
| ⭐ `/domain` | **【重要】** 操作を現在のドメインのドメインコントローラに対して行う。 |
| **グループ・メンバー操作** | |
| `/add` | 新しいグローバルグループを作成する、または既存グループにユーザーを追加する。 |
| `/delete` | グローバルグループを削除する、または既存グループからユーザーを削除する。 |
| `username [ ...]` | グループに追加または削除する 1 つ以上のドメインユーザー名を指定。 |
| `/comment:"text"` | グループに対する説明（コメント）を追加または変更する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新プロジェクト用のセキュリティグループを作成し、担当メンバーを一括登録する。
* **例**:
    ```cmd
    REM "Project_Alpha" グループを作成し、ユーザーを追加
    net group Project_Alpha /add /domain
    net group Project_Alpha user01 user02 /add /domain
    ```

### 2. ブルーチーム視点

* **タスク**: 特権グループ（Enterprise Admins 等）に、承認されていないユーザーが追加されていないか定期監査する。
* **例**:
    ```cmd
    REM Enterprise Admins のメンバーを抽出し、ログに記録
    net group "Enterprise Admins" /domain >> privileged_audit.log
    ```

### 3. レッドチーム視点 (偵察と権限昇格)

* **タスク 1: Domain Group Enumeration (ドメイングループの列挙)**
    * **目的**: 組織の構造を把握し、攻撃に有利なグループ（IT部門、経理、役員など）を探す。
    * **コマンド例**:
        ```cmd
        net group /domain
        ```
    * **実務ログ例**:
        ```text
        \\DC01.EXAMPLE.COM のグループ アカウント
        -------------------------------------------------------------------------------
        *Domain Admins           *Domain Guests           *Domain Users
        *Enterprise Admins       *Schema Admins           *Finance_Dept
        *IT_Support              *HR_Users                *VPN_Users
        ```

* **タスク 2: High-Value Target (HVT) の特定**
    * **目的**: 誰を侵害すればドメイン全体の支配権を得られるかを特定する。
    * **コマンド例**:
        ```cmd
        net group "Domain Admins" /domain
        ```
    * **実務ログ例**:
        ```text
        グループ名                           Domain Admins
        コメント                             指定されたドメインのすべての管理権限
        メンバー
        -------------------------------------------------------------------------------
        Administrator            svc_backup               t.tanaka
        コマンドは正常に終了しました。
        REM 考察: t.tanaka という個人アカウントがドメイン管理者であるため、このユーザーを標的にする。
        ```

* **タスク 3: 情報収集（コメント欄の探索）**
    * **目的**: 管理者が不用意に残したメモ（パスワード、サーバー名、用途）を奪取する。
    * **解説**: 特定のグループを詳細表示すると、コメント欄に重要なヒントが隠されていることがあります。
    * **コマンド例**:
        ```cmd
        net group "IT_Support" /domain
        ```

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: ドメイン管理者権限を持たないユーザーがグループの内容を変更（`/add` や `/delete`）しようとした。
    * **解決策**: 表示のみであれば一般ユーザーでも可能ですが、変更には適切な特権が必要です。

2.  **The group name could not be found.**
    * **原因**: スペルミス、またはグループ名にスペースがあるのに引用符で囲んでいない。
    * **解決策**: `net group "Domain Admins" /domain` のように `"` で囲んでください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **ドメイン情報の過剰な露出 (Domain Enumeration - T1087.002):**
    * **列挙による標的選定**: Active Directory のデフォルト設定では、**一般のドメインユーザー権限**であっても `net group /domain` を実行して、特権グループのメンバーをすべて閲覧可能です。攻撃者はこの情報を元に、「誰の認証情報を盗めばドメイン全体の支配権（Domain Admin）を得られるか」という最短ルートを容易に特定できます。
    * **組織構造の可視化**: グループ名（例: `Finance_Admin`, `C-Suite_Users`, `IT_Security`）から、組織内の重要部署や高権限者の所在をマッピングされ、標的型攻撃（フィッシング等）に悪用されます。
    * **不適切な情報保持**: グループの「コメント」欄に、暫定的なパスワードや、システムへのアクセス手順、サーバー名などが管理者によってメモされていることがあり、これが重要な情報漏洩源となります。
* **アカウント操作と永続化 (Account Manipulation - T1098):**
    * **バックドアとしての特権付与**: 攻撃者が一時的にドメイン管理権限を得た際、自身の作成した目立たないユーザーを `Domain Admins` や `Enterprise Admins` に追加して永続化を図ります。
    * **ネストされたグループの悪用**: 監視の目を逃れるため、一見無害な一般グループを特権グループのメンバーに追加（ネスト）し、間接的に管理者権限を維持する手法が取られます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察), `Persistence` (永続化)
* **特性**: `net.exe` は Microsoft によって署名された信頼できる管理ツールであるため、セキュリティソフトによる実行のブロッキングが困難です。攻撃者は、外部ツール（BloodHound等）が検知されるリスクを避けるため、まずこの標準コマンドを用いて「静かに」偵察を行います。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **AD ACL の見直し**: 重要なセキュリティグループ（特権グループ）のプロパティに対する読み取り権限を、管理者のみに制限することを検討します。
    * **特権管理の厳格化**: 特権ユーザーの管理には PAW (Privileged Access Workstation) を使用し、ドメイン管理者権限を持つアカウントが一般のワークステーションにログオン（＝ハッシュをメモリに残す）しないように徹底します。
* **Detection (検知)**:
    * **イベントIDの重点監視**: SIEM 等で以下のイベント、特に「メンバーシップの変更」をリアルタイムで追跡します。
        * **4727**: セキュリティ グローバル グループが作成されました。
        * **4728**: セキュリティ グローバル グループにメンバーが追加されました（**最優先アラート対象**）。
        * **4729**: セキュリティ グローバル グループからメンバーが削除されました。
    * **ハニーグループの設置**: 魅力的な名前のダミーグループ（例: `Global_Admin_Credentials`）を作成し、そのグループに対して `net group` が実行された際に即座に検知・通報する仕組みを導入します。

## 注意点・補足

* **グローバルグループとユニバーサルグループ**: `net group` は主にグローバルグループを対象とします。ユニバーサルグループやドメインローカルグループの操作には、PowerShell の `Set-ADGroup` 等の方が適している場合があります。
* **ADの複製ラグ**: 複数のドメインコントローラがある環境では、変更が全体に反映されるまで数分〜数十分のラグが発生することがあります。
