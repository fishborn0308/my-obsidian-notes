# Command - Linux - gunzip - gzip 形式（.gz）のファイルを伸張（解凍）する

## 概要

`gunzip` は、`gzip` コマンドで圧縮されたファイルを元の状態に復元します。基本的には元の `.gz` ファイルを削除し、圧縮前のファイルに置き換える動作をします。

(出自: GNU プロジェクト)

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `gzip -d` | `gunzip` と完全に同一の機能。好みの問題。 | - |
| `zcat` | 伸張した内容を標準出力に流す。元のファイルはそのまま。 | **中身のクイック確認** |
| `tar` | 複数のファイルをまとめる「アーカイブ」ツール。`-z` オプションで gzip と連携する。 | 複数ファイル・フォルダの展開 |
| `bzip2 / bunzip2` | より高い圧縮率を誇るが、処理が低速。`.bz2` 用。 | 高圧縮が必要な場合 |
| `xz / unxz` | 現代の主流。非常に高い圧縮率。`.xz` 用。 | 最新の配布パッケージ等 |

## よく連携されるコマンド

### シナリオ例: 圧縮された古いログファイルの調査 (運用・ブルーチーム視点)

* **目的**: `logrotate` によって圧縮された `/var/log/syslog.2.gz` の中身を一時的に確認する。
* **連携**: `grep`
* **解説**: ファイルを解凍せずに中身だけをパイプで渡します（`zcat` 的な使い方）。
* **コマンド例**:
    ```bash
    # 圧縮されたログから "error" を検索
    gunzip -c /var/log/syslog.2.gz | grep "error"
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-c`, `--stdout` | 元のファイルを削除せず、伸張した内容を標準出力に書き出す。 |
| ⭐️ `-k`, `--keep` | 伸張後も元の圧縮ファイル（.gz）を削除せずに残す（新しいバージョンの gunzip のみ）。 |
| ⭐️ `-f`, `--force` | 出力ファイルが既に存在する場合、強制的に上書きする。 |
| ⭐️ `-r`, `--recursive` | ディレクトリ内の全ての `.gz` ファイルを再帰的に伸張する。 |
| `-t`, `--test` | 圧縮ファイルの整合性をテストし、壊れていないか確認する。 |
| `-v`, `--verbose` | 処理したファイル名や圧縮率などの詳細を表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 配布された設定テンプレートやスクリプトの展開。
* **解説**: 単一のファイルとして圧縮されて配布されているツールを元に戻します。
* **例**:
    ```bash
    # ダウンロードしたスクリプトを解凍
    gunzip install_script.sh.gz
    # (実行権限を与えて実行)
    chmod +x install_script.sh
    ```

### 2. ブルーチーム視点

* **タスク**: フォレンジック調査における証拠ログの展開。
* **解説**: 長期間保存のために圧縮されていたログを、解析ツールにかけるために一括で解凍します。
* **例**:
    ```bash
    # アーカイブディレクトリ内の全ログを解凍
    sudo gunzip -r /var/log/old_archives/
    ```

### 3. レッドチーム視点

* **タスク**: 持ち出したデータの復元。
* **解説**: ターゲットから `gzip` で固めて転送したデータを、手元の解析端末で元に戻します。また、`gunzip` を利用した「Zip Bomb（解凍すると巨大なデータになりディスクをパンクさせる攻撃）」への耐性を確認することもあります。

## エラーメッセージとトラブルシューティング

1.  **「gunzip: filename.gz: unknown suffix -- ignored」**:
    * **原因**: 拡張子が `.gz` ではないファイルを処理しようとした。
    * **解決策**: 拡張子を適切に変更するか、`-S` オプションでサフィックスを指定する。

2.  **「gunzip: filename.gz: unexpected end of file」**:
    * **原因**: ダウンロードの失敗やディスクエラーにより、圧縮ファイルが壊れている。
    * **解決策**: 再度ダウンロードするか、バックアップから取得し直す。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **Resource Exhaustion (Zip Bomb)**: 数 KB の `.gz` ファイルを解凍したら数百 GB になり、サーバーのディスク容量を使い果たさせる攻撃。信頼できないソースからのファイルを `gunzip` する際は、クォータ（制限）のある環境で行うのが安全です。
* **ファイル権限の継承**: `gunzip` は元の `.gz` ファイルの権限を可能な限り引き継ごうとしますが、伸張プロセスで一時的に作成されるファイルのパーミッションに注意が必要です。

### 対応策・緩和策 (ブルーチーム視点)

* **Monitoring**: 短時間に大量の `gunzip` が実行され、ディスク I/O が異常に高まっているプロセスを監視し、DoS 攻撃の可能性を排除します。

## 注意点・補足

* **上書き動作**: デフォルトでは、解凍後のファイル名が既に存在する場合、上書きするかどうかの確認プロンプトが出ます。自動化スクリプトでは `-f` の使用を検討してください。
* **`.tgz` ファイル**: これは `tar` + `gzip` の略です。`gunzip` だけで解凍すると `tar` ファイルが出てくるので、通常は `tar -zxvf` を使って一気に展開します。
