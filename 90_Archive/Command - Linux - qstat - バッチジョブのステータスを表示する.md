# Command - Linux - qstat - バッチジョブのステータスを表示する

## 概要

`qstat` は、PBS (Portable Batch System) や Sun Grid Engine (SGE) などのバッチスケジューラにおいて、投入されたジョブの実行状況を確認するためのコマンドです。共有のリソース（CPU、メモリ、GPU）を効率よく使うため、ジョブが「実行中 (Running)」なのか「待機中 (Queued)」なのかをリアルタイムで把握できます。

## 🛠 スケジューラ別・確認コマンド

環境によって採用されているスケジューラが異なり、コマンドも変わります。

| スケジューラ | 確認コマンド | 特徴 |
| :--- | :--- | :--- |
| **PBS / SGE** | **`qstat`** | 伝統的なデファクトスタンダード。 |
| Slurm | `squeue` | 現代の主流。`qstat` 互換コマンドが用意されていることも多い。 |
| LSF | `bjobs` | 大規模な商用環境でよく使われる。 |
| Local PC | `top` / `ps` | 単体マシンのプロセス確認。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-u <user>` | 特定のユーザーのジョブのみを表示する。 |
| ⭐️ `-f` | 特定のジョブに関する全ての情報を詳細に表示する（Full）。 |
| ⭐️ `-q` | キュー（ジョブの並び列）ごとの要約を表示する。 |
| ⭐️ `-n` | どのノード（計算機）でジョブが動いているかを表示する。 |
| ⭐️ `-a` | 全てのジョブを表示する。 |

## コマンド例

```bash
# 自分のジョブの状況を確認
qstat -u $(whoami)

# 特定のジョブ (ID: 12345) の詳細情報を表示
qstat -f 12345

# 現在のキューごとの負荷状況を確認
qstat -q
````

---

## 📊 ジョブステータスの読み方

- **Q (Queued)**: 待機中。リソースが空くのを待っています。
- **R (Running)**: 実行中。
- **H (Held)**: 保留中。何らかの理由で実行が止められています。
- **E (Exiting)**: 終了処理中。
- **C (Completed)**: 完了。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: クラスタ全体の負荷調整。
- **解説**: 特定のユーザーがリソースを独占していないか、あるいは特定の計算ノードが故障してジョブが滞っていないかを `qstat` で監視し、スケジューラの設定を最適化します。

### 2. ブルーチーム視点

- **タスク**: リソースの不正利用（クリプトマイニング等）の検知。
- **解説**: スパコンや計算クラスタは非常に高い計算能力を持つため、乗っ取られた際に仮想通貨のマイニングに悪用されるケースがあります。不審な名前のジョブや、24時間 365 日動き続けている異常なジョブを `qstat` で特定します。

### 3. レッドチーム視点

- **タスク**: ターゲット環境の偵察（Internal Reconnaissance）。
- **解説**: 侵入後、そのクラスタで「誰が」「どんな研究・開発をしているか」を `qstat` から読み取ります。ジョブ名やスクリプト名から、機密性の高いプロジェクト名や、利用されているソフトウェアの脆弱性を推測するヒントを得ます。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出 (Information Disclosure)**:

    `qstat -f` の出力には、実行スクリプトのパス、環境変数、コマンドライン引数などが含まれることがあります。これらにパスワードや API キーが直接書かれている場合、同じクラスタを使う他のユーザーにそれらが漏洩するリスクがあります。

### 対応策・緩和策

- **機密情報の秘匿**: ジョブスクリプト内でパスワードを扱う場合は、直接記述せず、環境ファイルや秘密情報管理ツールから読み込むようにします。
- **閲覧制限**: 管理者設定で、他人のジョブの詳細情報を一般ユーザーが見られないように制限することを検討します。
