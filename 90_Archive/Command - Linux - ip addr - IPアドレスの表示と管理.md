# Command - Linux - ip addr - IPアドレスの表示と管理

## 概要

`ip addr` オブジェクトは、ネットワークインターフェースに割り当てられたプロトコル（主に IPv4/IPv6）アドレスを管理します。アドレスの追加、削除、および表示を行い、それぞれの「有効範囲（Scope）」や「属性（Flags）」を細かく制御できます。

## 主要サブコマンド

| サブコマンド | 略称 | 説明 |
| :--- | :--- | :--- |
| **`show`** | `sh`, `s` | インターフェースのアドレスを表示する（デフォルト）。 |
| **`add`** | `a` | 新しいアドレスをインターフェースに割り当てる。 |
| **`del`** | `d` | 割り当てられているアドレスを削除する。 |
| **`flush`** | `f` | 特定の条件（インターフェース名など）に一致する全アドレスを一括削除する。 |

## コマンド例と構文

### 1. アドレスの確認 (show)

最も基本の操作です。特定のインターフェースだけに絞り込むことも可能です。

```bash
# 全てのインターフェースを表示
ip addr show

# eth0 の情報をカラーで表示（見やすさ重視）
ip -c addr show dev eth0

# IPv4 のアドレスのみを表示
ip -4 addr show
````

### 2. アドレスの割り当て (add)

サブネットマスクは CIDR 形式 (`/24` など) で指定するのが一般的です。

Bash

```
# eth0 に 192.168.10.55/24 を追加
sudo ip addr add 192.168.10.55/24 dev eth0

# ブロードキャストアドレスを明示的に指定して追加
sudo ip addr add 10.0.0.1/8 brd + dev eth0
```

### 3. アドレスの削除 (del / flush)

`flush` は、リモート接続している場合には**「自分を締め出す」リスク**があるため注意が必要です。

Bash

```
# 特定の IP を削除
sudo ip addr del 192.168.10.55/24 dev eth0

# eth0 に設定されている全ての IP アドレスを消去する (危険！)
sudo ip addr flush dev eth0
```

## 出力項目の読み解き

- **`flags <UP,BROADCAST,RUNNING,MULTICAST>`**: インターフェースの状態。
- **`inet / inet6`**: 割り当てられている IP アドレスとマスク。
- **`brd`**: ブロードキャストアドレス。
- **`scope`**: アドレスの有効範囲。
    - `global`: どこからでも有効（通常の IP）。
    - `link`: このリンク（セグメント）内でのみ有効。
    - `host`: このマシン内でのみ有効（ループバック等）。
- **`valid_lft / preferred_lft`**: アドレスの有効期間（動的割当の場合）。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: VIP（バーチャル IP）の追加。
- **解説**: ロードバランサーや高可用性構成（Keepalived 等）において、既存の IP を残したまま予備の IP を追加する際に、`ip addr add` を使用します。

### 2. ブルーチーム視点

- **タスク**: 未知の IP アドレスの監視。
- **解説**: 攻撃者がバックドア通信のために勝手にセカンダリ IP を設定していないか、定期的に `ip addr` の結果をスナップショットと比較します。

### 3. レッドチーム視点

- **タスク**: 隠れみのとしてのマルチ IP 利用。
- **解説**: 攻撃者は複数の IP を一つの NIC に設定し、スキャンごとに送信元 IP を切り替えることで、監視システムの検知（しきい値制限など）を回避しようとします。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: 接続されている全てのセグメントが判明するため、攻撃者にとっては「次にどのネットワークへ横展開（Lateral Movement）できるか」のヒントになります。

### 対応策・緩和策 (ブルーチーム視点)

- **不可視化**: `ip addr` の実行自体を制限することは難しいですが、`/etc/network/interfaces` や `Netplan` の設定ファイルへのアクセスを厳格に管理することで、意図しない IP 設定を防止します。
