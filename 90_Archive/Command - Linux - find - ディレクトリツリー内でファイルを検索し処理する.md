# Command - Linux - find - ディレクトリツリー内でファイルを検索し処理する

## 概要

`find` は、指定された開始地点からディレクトリ構造を再帰的に辿り、ユーザーが指定した条件（述語）に一致するファイルを検索します。一致したファイルに対して、パスの表示だけでなく、コマンドの実行や削除などの「アクション」を直接指示できるのが最大の特徴です。

(出自: `findutils` パッケージに含まれる)

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `locate` | 事前に作成されたデータベースを検索するため爆速。ただし、最新のファイルは見つからない。 | 既知のファイルの高速検索 |
| `which / whence` | 環境変数 `$PATH` から実行ファイルの場所を探す。 | コマンドのパス確認 |
| `fd` | Rust 製のモダンな代替。デフォルトで隠しファイルを無視し、構文が簡潔で非常に高速。 | インタラクティブな日常使い |
| `find` | 権限、時間、サイズなど、OS レベルの詳細な条件指定が可能。標準搭載で移植性が高い。 | **複雑な自動化・セキュリティ監査** |

## よく連携されるコマンド

### シナリオ例: 巨大なログファイルの一括削除 (運用・クリーンアップ視点)

* **目的**: 30日以上更新がなく、かつサイズが 100MB 以上の `.log` ファイルを探して削除する。
* **連携**: `-exec` または `xargs`
* **解説**: 条件に合うものだけを抽出し、安全に処理します。
* **コマンド例**:
    ```bash
    # 条件に合うファイルを確認してから削除する (安全策)
    find /var/log -name "*.log" -mtime +30 -size +100M -delete
    ```

## 主要な検索条件 (Expressions)

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-name <pattern>` | ファイル名で検索（ワイルドカード使用可）。`-iname` は大文字小文字無視。 |
| ⭐️ `-type <t>` | 種類で検索（`f`: ファイル, `d`: ディレクトリ, `l`: シンボリックリンク）。 |
| ⭐️ `-mtime <n>` | 更新日時で検索（`-7` は7日以内, `+7` は7日より前）。 |
| ⭐️ `-size <n>` | サイズで検索（`k`, `M`, `G` 単位指定可）。 |
| ⭐️ `-perm <mode>` | 権限で検索（例: `-perm 4000` は SUID が付いたものを検索）。 |
| `-user / -group` | 所有ユーザーやグループで検索。 |

## 主要なアクション (Actions)

| アクション | 説明 |
| :--- | :--- |
| ⭐️ `-print` | マッチしたファイルのパスを表示する（デフォルト）。 |
| ⭐️ `-exec <cmd> {} \;` | マッチしたファイルに対してコマンドを実行する（`{}` がパスに置換される）。 |
| ⭐️ `-delete` | マッチしたファイルを直接削除する。 |
| `-ls` | `ls -dils` 形式で詳細を表示する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 権限が不適切なディレクトリの一括修正。
* **解説**: ディレクトリだけを `755` に、ファイルだけを `644` に揃えます。
* **例**:
    ```bash
    # ディレクトリの権限を修正
    find /var/www/html -type d -exec chmod 755 {} +
    # ファイルの権限を修正
    find /var/www/html -type f -exec chmod 644 {} +
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査（Web シェルやバックドアの探索）。
* **解説**: 最近数時間以内に作成・変更された PHP ファイルや、SUID が設定された不審なバイナリを特定します。
* **例**:
    ```bash
    # 過去24時間以内に変更された .php ファイルをリストアップ
    sudo find /var/www/html -name "*.php" -mtime -1 -ls
    # 所有者が root で SUID が付いているファイル（権限昇格の標的）を検索
    find / -user root -perm -4000 -print 2>/dev/null
    ```

### 3. レッドチーム視点

* **タスク**: 設定ファイル内の機密情報の探索。
* **解説**: 書き込み可能なディレクトリを探したり、名前に "config" や "secret" を含むファイルを全検索します。
* **例**:
    ```bash
    # 全ユーザーから書き込み可能なディレクトリを検索（永続化の足場探し）
    find / -type d -perm -o+w 2>/dev/null
    # 設定ファイルから "password" という文字列を含むものを検索
    find /etc -name "*.conf" -exec grep -l "password" {} + 2>/dev/null
    ```

## エラーメッセージとトラブルシューティング

1.  **「Permission denied」が大量に出る**:
    * **原因**: 権限のないディレクトリ（`/root` や `/proc` 等）をスキャンしようとしている。
    * **解決策**: `sudo` で実行するか、エラー出力を捨てる (`2>/dev/null`)。

2.  **「find: paths must precede expression」**:
    * **原因**: 検索パス（`.` や `/`）より前にオプション（`-name` 等）を書いてしまった。
    * **解決策**: `find [パス] [オプション]` の順序を守る。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Command Injection**: `find` の結果を適切に処理しないスクリプトにおいて、ファイル名に `; rm -rf /` のような細工がされていると、意図しないコマンドが実行されるリスクがあります（`-print0` と `xargs -0` の組み合わせで回避可能）。

### GTFOBins における利用例

`find` に `sudo` 権限がある場合、`-exec` を介して root シェルを即座に取得できます。

* **機能**: `Sudo`, `Interactive Shell`
* **コマンド例**:
    ```bash
    # sudo find を使って root シェルを取得
    sudo find . -exec /bin/sh \; -quit
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/find/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 一般ユーザーに `sudo find` を許可しない。許可が必要な場合は、パスを特定のデータディレクトリに限定する。
* **Detection (検知)**: `/etc` や重要システムパスに対して `find` が `-exec` を伴って実行されているログを監視する。

## 注意点・補足

* **`+` vs `\;`**:
    * `-exec cmd {} \;`: ファイルごとにコマンドを 1回ずつ起動する。
    * `-exec cmd {} +`: 可能な限り多くのファイルを引数にまとめて、コマンドを 1回だけ起動する（高速）。
* **`-delete` の位置**: 必ず条件の**最後**に置いてください。最初に置くと、そのディレクトリ内の全ファイルがマッチする前に消される危険があります。
