# Command - Windows - cmd - systeminfo - システムの詳細構成情報の表示

## 概要

`systeminfo` コマンドは、オペレーティング システムの構成に関する詳細な情報（サービス パックのレベル、BIOS のバージョン、メモリ構成、ネットワーク アダプターの情報、および適用済みの修正プログラム（Hotfix）など）を表示します。

ローカルマシンだけでなく、リモートマシンの情報取得も可能です。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `hostname` | コンピューター名のみを表示する。 | `hostname` |
| `ver` | OSのバージョン（ビルド番号）のみを簡潔に表示する。 | `ver` |
| `msinfo32` | GUIツール。より詳細なハードウェアリソースやコンポーネント情報を表示できる。 | GUIなら `msinfo32` |
| `wmic os get /all` | WMI経由でOS情報を取得。構造化データとして扱いやすいが、将来的に廃止予定。 | `Get-CimInstance` (PS) |

## よく連携されるコマンド

`systeminfo` は出力が非常に長いため、特定の情報を抽出するために検索コマンドと併用されるのが一般的です。

### シナリオ例: OS情報の特定 (偵察/調査)

* **目的**: OSの名称、バージョン、ビルド番号、アーキテクチャを一目で確認する。
* **連携コマンド**: `findstr`
* **解説**: 複数のキーワードを同時に検索し、必要な行だけをフィルタリングします。
* **コマンド例**:
    ```cmd
    REM OS名、バージョン、システムモデルのみを抽出
    systeminfo | findstr /B /C:"OS 名" /C:"OS バージョン" /C:"システム モデル"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **出力フォーマット** | |
| `/fo <Format>` | 出力形式を指定。`TABLE` (表), `LIST` (リスト:デフォルト), `CSV` (カンマ区切り) が選択可能。 |
| `/nh` | 出力に列ヘッダーを表示しない。`/fo TABLE` または `CSV` と併用。 |
| **リモート操作** | |
| ⭐ `/s <Computer>` | リモートコンピューターの名前またはIPアドレスを指定して実行。 |
| `/u [<Domain>\]<User>` | リモート実行時のユーザーコンテキストを指定。 |
| `/p [<Password>]` | 指定したユーザーのパスワードを入力。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーのパッチ適用状況（Hotfix）のリスト出力
* **組み合わせ**: `/fo CSV`
* **解説**: 適用済みのKB番号をCSV形式で保存し、Excelなどで管理台帳と比較します。
* **例**:
    ```cmd
    REM 適用済みパッチ一覧をCSVで保存
    systeminfo /fo CSV > %COMPUTERNAME%_patches.csv
    ```

### 2. ブルーチーム視点

* **タスク**: システムの起動時間の確認（不審な再起動の調査）
* **組み合わせ**: `findstr`
* **解説**: 「システム起動時間」を確認することで、パッチ適用後や攻撃による強制終了後に再起動されたタイミングを把握します。
* **例**:
    ```cmd
    REM 最終起動時間を確認
    systeminfo | findstr /C:"システム起動時間"
    ```

### 3. レッドチーム視点

* **タスク**: 権限昇格（Privilege Escalation）のための脆弱性調査
* **組み合わせ**: `wesng` (Windows Exploit Suggester - Next Generation) などの外部ツール
* **解説**: ターゲットで `systeminfo` を実行し、その結果を攻撃端末に持ち帰って解析ツールに読み込ませることで、未適用のパッチから利用可能なカーネルエクスプロイトを特定します。
* **例**:
    ```cmd
    REM ターゲットマシンで実行し、結果を保存
    systeminfo > sysinfo.txt
    
    REM (攻撃端末側で解析)
    REM python wes.py sysinfo.txt
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `エラー: ネットワーク パスが見つかりません。`
    * **考えられる原因**: リモートマシン（`/s`）に接続できない。ファイル共有（SMB）やRPCサービスが停止している、またはファイアウォールで遮断されている。
    * **解決策**: ターゲットの疎通確認と、必要なサービスの有効化を確認する。

2.  **エラーメッセージ例 2**: `エラー: アクセスが拒否されました。`
    * **考えられる原因**: 実行ユーザーにリモートマシンの情報を取得する権限がない。
    * **解決策**: `/u` スイッチを使用して、適切な管理者権限を持つアカウントで実行する。

## 環境変数と設定ファイル

`systeminfo` 自体に影響を与える固有の変数は少ないですが、表示される情報の多くはレジストリから取得されています。

* **レジストリ位置**: `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion`

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **Enumeration**: 侵入後、攻撃者が最初に実行するコマンドの一つ。インストールされているパッチ、OSのビルド、ドメイン参加状況を把握されることで、攻撃のネクストステップ（権限昇格やラテラルムーブメント）の戦略を立てられてしまいます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Information Requirements`
* **解説**: 標準機能であるため、実行しても不審がられにくい。ただし、実行時に大量の情報を取得するため、EDRなどの挙動検知に引っかかる可能性があります。
* **参照**: [LOLBAS - systeminfo.exe](https://lolbas-project.github.io/lolbas/Binaries/Systeminfo/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * 攻撃者が情報を得られないよう、一般ユーザーによる `systeminfo` 実行を制限することは困難（多くの正当な用途があるため）ですが、重要なサーバーへのアクセス制限を徹底します。
* **Detection (検知)**:
    * **コマンドライン監査**: 短時間に `systeminfo` を実行し、その後すぐにエクスプロイトコードが実行されるようなパターンを監視します。

## 注意点・補足

* **実行時間**: ネットワークアダプターやパッチ情報の収集に時間がかかる場合があります。スクリプトに組み込む際は、タイムアウトに注意してください。
* **日本語環境**: 「OS Name」が「OS 名」となるなど、OSの言語設定によって出力ラベルが変わるため、`findstr` で検索する際は注意が必要です。
