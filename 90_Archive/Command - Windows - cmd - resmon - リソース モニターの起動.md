# Command - Windows - cmd - resmon - リソース モニターの起動

## 概要

`resmon` は、システムのハードウェアおよびソフトウェア リソースの使用状況をリアルタイムで監視するためのツールです。

CPU、メモリ、ディスク、ネットワークの 4 つのカテゴリについて、どのプロセスがどれだけリソースを消費しているか、どのファイルにアクセスしているか、どの IP アドレスと通信しているかを詳細に追跡できます。

(出自: Windows Vista 以降標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `taskmgr` | タスク マネージャー。概要把握には良いが、ネットワークの接続先やディスクのI/O詳細は `resmon` が勝る。 | `taskmgr` (簡易確認) |
| `perfmon` | パフォーマンス モニター。長期間のログ収集やカウンタの分析に適している。 | `perfmon` (長期分析) |
| `netstat` | ネットワーク接続のみを CLI で表示。`resmon` はこれを GUI でリアルタイム更新し、プロセスと紐付ける。 | `netstat` (CLI調査) |

## よく連携されるコマンド

`resmon` は GUI ツールですが、CLI で異常を察知した後に、その詳細を「目視」で追いかける際によく呼び出されます。

### シナリオ例: 異常なディスク I/O を発生させているプロセスの特定 (インフラ運用)

* **目的**: サーバーのレスポンスが極端に遅い際、どのプロセスがどのファイルに対して大量の書き込みを行っているかを特定する。
* **連携コマンド**: `tasklist`, `fsutil`
* **解説**: CLI で負荷を確認後、`resmon` を起動し「ディスク」タブの「ディスク活動のプロセス」と「ディスク活動」を照合します。
* **コマンド例**:
    ```cmd
    REM リソースモニターを起動して視覚的に調査を開始
    resmon
    ```

## スイッチ/オプション説明

`resmon` 自体は引数なしで起動するのが一般的ですが、以下のコマンドでも起動可能です。

| スイッチ | 説明 |
| :--- | :--- |
| (なし) | 通常のリソース モニターを起動する。 |
| `resmon /res` | リソース モニターを起動する（標準と同じ）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: メモリリークの調査
* **解説**: 「メモリ」タブの「コミット」や「ワーキング セット」の推移を監視し、特定のアプリケーションが物理メモリを食い潰していないか確認します。
* **例**: `resmon` > 「メモリ」タブ > 列ヘッダーの「コミット」でソートして監視。

### 2. ブルーチーム視点

* **タスク**: 不審な C2 通信の特定
* **解説**: 「ネットワーク」タブの「ネットワーク活動のプロセス」と「ネットワーク接続」を同時に見ることで、特定のプロセス（例: `powershell.exe`）が外部の未知の IP アドレスと通信している状況をリアルタイムで捕捉します。
* **例**: `resmon` > 「ネットワーク」タブ > 「TCP 接続」で不審なポート（4444, 8080等）がないかチェック。

### 3. レッドチーム視点 (偵察)

* **タスク**: アンチウイルスや EDR プロセスの挙動監視
* **解説**: 自分が送り込んだペイロードが実行された際、どのセキュリティプロセス（例: `MsMpEng.exe`）が反応し、どのファイルやレジストリをスキャンし始めたかを観察し、検知回避のヒントにします。
* **例**: `resmon` > 「CPU」タブ > 「関連付けられたハンドル」で、自分のツール名を検索して、他プロセスからの干渉を確認。

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `resmon` を起動しようとすると、何も起きない、またはエラーが出る。
    * **考えられる原因**: ユーザーに管理者権限がない、またはグループポリシー（GPO）でシステムツールの実行が制限されている。
    * **解決策**: 「管理者として実行」を試みる。制限されている場合は、`tasklist` や `netstat` などの CLI 代替案を使用する。

2.  **現象**: 情報が更新されない。
    * **原因**: 「モニター」メニューで「監視の停止」が選択されている可能性がある。
    * **解決策**: 「監視の開始」をクリックする。

## 環境変数と設定ファイル

`resmon` は設定ファイルを持ちませんが、前回終了時のウィンドウサイズやタブの状態はレジストリ等にユーザープロファイルごとに保持されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Enumeration**: 攻撃者が GUI 環境（RDP など）を得た場合、`resmon` を使ってシステムの弱点（空きメモリ容量、書き込み可能なディスク、通信中のサーバー等）を効率よく把握されます。
    - **Side-channel analysis**: リソースの消費パターンから、裏で動いているバックアップ処理やセキュリティスキャンのタイミングを計り、その隙を突いて攻撃（大容量データの窃取など）を行う。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - 不要なサーバーでの RDP 接続を制限し、GUI ツールの使用機会を減らす。
* **Detection (検知)**:
    - 一般ユーザー権限で `resmon.exe` が起動されたログ（イベントID 4688）を監視し、特に管理業務時間外であれば詳細を調査する。

## 注意点・補足

* **「待機チェーンの分析」**: CPU タブでプロセスを右クリックして実行できるこの機能は、アプリケーションがなぜフリーズしているのか（どのリソースを待っているのか）を特定するのに非常に強力です。
* **パフォーマンスへの影響**: `resmon` 自体もリソースを消費するため、極限まで負荷が高いサーバーで実行すると、さらに状況を悪化させる可能性がある点に注意してください。
