# Command  - Linux - sort - テキストの行を並べ替える

## 概要

`sort` は、ファイルや標準入力から受け取ったテキストデータの**行を、指定した順序で並べ替える（ソートする）**ためのコマンドです。デフォルトでは、アルファベット順（辞書順）で昇順（A→Z）に並べ替えます。ログファイルの分析、コマンド出力の整理、データの集計など、テキスト処理パイプラインにおいて中心的な役割を果たします。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`uniq`** | **隣接する重複行**を削除または集計します。`uniq` が正しく機能するためには、**入力が事前に `sort` されている必要があります**。`sort -u` は `sort \| uniq` とほぼ同じ動作をします。 | `sort -u` (重複削除) |
| **`shuf`** | 行を**ランダムに**並べ替えます（シャッフルします）。 | - |
| **`ls -l`** | ファイルの一覧を表示しますが、`ls` 自体が持つソート機能（名前順、時間順 `-t`など）を使います。`sort` は、より汎用的なテキストデータを、より柔軟なキーで並べ替えることができます。 | - |

## よく連携されるコマンド

`sort` は `uniq -c` と連携して、項目ごとの出現回数を集計する際に不可欠な役割を果たします。

### シナリオ例: アクセスログから攻撃元IPをランキングする (ブルーチーム視点)

* **目的**: Webサーバーのアクセスログから、最もアクセス回数が多いIPアドレス（DDoS攻撃の兆候など）を特定する。
* **連携コマンド**: `awk`, `uniq`
* **解説**: `awk` でIPアドレスを抽出し、`sort` でIPアドレスをまとめ（`uniq` の前処理）、`uniq -c` で出現回数を集計します。最後に再度 `sort -nr` で結果を降順にソートするのが黄金パターンです。
* **コマンド例**:

		```bash
		# このパイプラインで、アクセスログから攻撃元IPトップ10を抽出する

		awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -n 10

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ソート順序** | |
| ⭐ `-r`, `--reverse` | **逆順**（降順）でソートします。 |
| ⭐ `-n`, `--numeric-sort` | 文字列としてではなく、**数値**としてソートします。 |
| `-h`, `--human-numeric-sort`| `du` コマンドなどの出力に含まれる `K`, `M`, `G` といった**人間が読みやすい単位**を解釈してソートします。 |
| `-f`, `--ignore-case` | 大文字と小文字を区別せずにソートします。 |
| **キー指定** | |
| ⭐ `-k`, `--key=<POS>` | **キー**を指定してソートします。`<POS>` には `1`, `2` のように、フィールド（列）番号を指定します。 |
| `-t <char>`, `--field-separator=<char>`| `-k` と併用し、フィールドを区切る文字を指定します。デフォルトは空白です。（例: `-t ':'`） |
| **その他** | |
| ⭐ `-u`, `--unique` | 重複する行を削除し、一意な行のみを出力します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディスク容量を消費しているディレクトリを大きい順にリストアップする。
* **組み合わせ**: `du -h --max-depth=1 /var | sort -rh`
* **解説**: `du` の出力を、人間が読みやすい単位 (`-h`) で、逆順 (`-r`) にソートする定番の使い方です。
* **例**:

		```bash
		# /etc/passwd ファイルを、ユーザーID (3番目のフィールド) の数値順でソート

		sort -t ':' -k 3 -n /etc/passwd

		```

## 2. ブルーチーム視点

* **タスク**: ログから特定のイベントを抽出し、発生回数が多い順にランキングする。
* **組み合わせ**: `... | sort | uniq -c | sort -nr`
* **解説**: 上記「よく連携されるコマンド」シナリオの通り、ログ分析における頻出パターンです。
* **例**:

		```bash
		# 認証ログからログインに失敗しているユーザー名を多い順にリストアップ

		grep "Failed password for" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr

		```

## 3. レッドチーム視点

* **タスク**: パスワード攻撃用のワードリストから重複を削除し、整理する。
* **組み合わせ**: `sort -u wordlist.txt -o wordlist_unique.txt`
* **解説**: ブルートフォース攻撃やディレクトリ・ファジングに使用するワードリストから重複を削除し、攻撃の効率を高めます。`-o` オプションで、結果を直接新しいファイルに書き出します。
* **例**:

		```bash
		# 複数のワードリストを結合し、重複を排除してクリーンなリストを作成

		cat list1.txt list2.txt | sort -u > clean_wordlist.txt

		```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **現象**: **数字をソートしたが、`1, 10, 2, 20` のような順番になってしまう。**
		* **考えられる原因**: デフォルトの辞書順ソートが適用されており、文字列として比較されています。
		* **解決策**: `-n` (`--numeric-sort`) オプションを追加して、数値としてソートしてください。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 影響を与える環境変数

* **`LC_ALL`**: `sort` の動作は、システムのロケール（言語設定）に影響を受けます。厳密なバイト順でソートしたい場合は、`LC_ALL=C sort` のように、ロケールをCに設定して実行するのが定石です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `sort` 自体は読み取り専用ツールですが、**リソースの枯渇**に繋がる可能性があります。
* **悪用シナリオ**: 非常に巨大なファイル（数GB以上）を `sort` させると、大量のメモリとCPUリソースを消費し、一時ファイルを `/tmp` に作成します。これにより、システムのパフォーマンスが低下し、簡易的なサービス妨害 (DoS) 状態を引き起こす可能性があります。

### GTFOBins / LOLBAS における利用例

`sort` はGTFOBinsにリストされています。

* **機能**: **File Read**
* **解説**: もしユーザーが `sudo sort` を実行できる権限を持っている場合、これを利用して本来読み取れないはずのファイル（例: `/etc/shadow`）の内容を読み取り、ソートされた結果として表示させることが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/sort/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `ulimit` や `cgroups` を使って、ユーザーが使用できるリソース（メモリ、ファイルサイズ）に制限をかける。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `auditd` で `sort` コマンドの実行を監視し、特に機密ファイルを入力とするものを検知する。

## 注意点・補足

* **安定ソート**: `sort` は安定ソートです（`-u` を使わない場合）。これは、キーが同じである行の元の順序が、ソート後も保持されることを意味します。
