# Command - Windows - cmd - net share - ローカル共有リソースの管理

## 概要

`net share` コマンドは、ローカルコンピュータの共有リソース（フォルダ、プリンタ）を作成、削除、または表示するために使用されます。

ネットワーク上の他のユーザーが自分のコンピュータ上のファイルにアクセスできるように設定するための中心的なツールです。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `net view` | **リモート**サーバーの共有リソースを列挙する。 | 相手側の調査 |
| `net use` | リモートの共有リソースに**接続（マウント）**する。 | 共有へのアクセス |
| `Get-SmbShare` | PowerShell版。アクセス権（ACL）や詳細な設定をオブジェクトとして取得可能。 | PS環境 |
| `fsmgmt.msc` | GUIによる「共有フォルダ」管理ツール。視覚的な操作が可能。 | - |

## よく連携されるコマンド

### シナリオ例: データの持ち出し用共有フォルダの作成 (レッドチーム/運用共通)

* **目的**: 一時的にファイルを転送するための受け皿となる共有フォルダを即座に作成する。
* **連携コマンド**: `mkdir`, `net share`, `icacls`
* **解説**: フォルダを作成し、特定のユーザーに対して書き込み権限を付与した共有を設定します。
* **コマンド例**:
    ```cmd
    REM フォルダを作成し "DropZone" という名前でフルコントロール共有を設定
    mkdir C:\Temp\DropZone
    net share DropZone=C:\Temp\DropZone /grant:everyone,FULL
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | 現在コンピュータで共有されているすべてのリソースを一覧表示する。 |
| `sharename` | 特定の共有リソースの詳細情報（パス、最大ユーザー数など）を表示する。 |
| **共有の作成・設定** | |
| `sharename=drive:path` | 指定したパスを新しい共有名で公開する。 |
| ⭐ `/grant:user,perm` | 指定したユーザーに権限（READ, CHANGE, FULL）を付与する。 |
| `/users:number` | 共有に同時に接続できる最大ユーザー数を制限する。 |
| `/unlimited` | 同時接続ユーザー数を無制限に設定する。 |
| `/remark:"text"` | 共有に関する説明（注釈）を追加する。 |
| ⭐ `/delete` | **【重要】** 指定した共有を停止（削除）する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー上の特定のフォルダを、社内ユーザーが読み取り専用でアクセスできるように公開する。
* **例**:
    ```cmd
    REM "Manuals" フォルダを読み取り専用で共有
    net share Manuals=D:\Data\Manuals /grant:"Authenticated Users",READ
    ```

### 2. ブルーチーム視点

* **タスク**: 意図しない「隠し共有」や、全ユーザーにフルコントロールが与えられている危険な共有がないか監査する。
* **例**:
    ```cmd
    REM 共有一覧を表示し、各共有のアクセス権をチェック
    net share
    ```

### 3. レッドチーム視点 (偵察と横展開)

* **タスク 1: Admin Share Reconnaissance (管理共有の確認)**
    * **目的**: `C$` や `ADMIN$` などの管理共有が有効かどうかを確認し、横展開の経路を特定する。
    * **解説**: デフォルトの管理共有（末尾に `$` が付くもの）が表示されない設定になっていないか、あるいはアクセス可能かを確認します。
    * **コマンド例**:
        ```cmd
        net share
        ```
    * **実務ログ例**:
        ```text
        共有名       リソース                            注釈
        -------------------------------------------------------------------------------
        C$           C:\                                 Default share
        IPC$                                             Remote IPC
        ADMIN$       C:\Windows                          Remote Admin
        Users        C:\Users
        コマンドは正常に終了しました。
        ```

* **タスク 2: Data Exfiltration Point (持ち出しポイントの作成)**
    * **目的**: ネットワーク内の他の端末から盗み出したファイルを、自分が支配している端末へ集約するために共有を作成する。
    * **解説**: 攻撃者は `net share` を使って、目立たない名前（例: `Default_Log`）の共有を作成し、そこにデータを集めます。
* **タスク 3: 共有パスワードの探索 (Looting)**
    * **目的**: 共有の設定情報から、機密データが保存されていそうな物理パスを特定する。
    * **解説**: 共有名が `Secret_Docs` で、物理パスが `D:\Confidential\ProjectX` であれば、そのディレクトリを重点的に探索します。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 管理者権限がない。共有の作成・削除には昇格したプロンプトが必要です。
    * **解決策**: 「管理者として実行」で再度実行してください。

2.  **The name has already been shared.**
    * **原因**: 指定した共有名が既に使用されている。
    * **解決策**: 別の共有名を使うか、既存の共有を一度 `/delete` してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **不適切な ACL 設定 (Excessive Permissions - T1082):**
    * **悪用シナリオ**: 共有作成時に `/grant:everyone,FULL` を使用すると、組織内の誰でも（時にはゲストでも）そのフォルダ内のファイルを読み書き・削除できてしまいます。攻撃者はこのような「ゆるい」共有を真っ先に探し、機密情報を盗んだり、マルウェアを設置したりします。
* **管理共有の悪用 (Lateral Movement - T1021.002):**
    * 攻撃者が管理者権限を奪取すると、`net use \\Target\C$` を通じてシステム全体にアクセスできるようになります。`net share` はこの管理共有の状態を確認するために悪用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察), `Lateral Movement` (横展開)
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **管理共有の制限**: 必要に応じてレジストリ (`AutoShareServer`, `AutoShareWks`) で管理共有を無効化する。
    * **共有の最小化**: 不要な共有は `net share <name> /delete` で即座に削除する。
* **Detection (検知)**:
    * **イベントID 5142**: 新しい共有リソースが作成されました。
    * **イベントID 5143**: 共有リソースが変更されました。
    * **イベントID 5144**: 共有リソースが削除されました。
    * 特にサーバーではなくクライアント端末で共有が作成された場合は、侵害の兆候としてアラートを発報すべきです。

## 注意点・補足

* **非表示共有**: 共有名の末尾に `$` を付けると、`net view` などのブラウジング一覧に表示されない「隠し共有」になります。
* **物理パスの引用符**: 物理パスにスペースが含まれる場合は、`net share "My Data"=C:\My Data` のように記述してください。
