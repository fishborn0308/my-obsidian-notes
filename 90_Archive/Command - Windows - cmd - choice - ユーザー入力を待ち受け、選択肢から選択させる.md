# Command - Windows - cmd - choice - ユーザー入力を待ち受け、選択肢から選択させる

## 概要

`choice` コマンドは、ユーザーに選択肢（1文字のリスト）を提示し、入力を待ち受けます。ユーザーが選択した結果は、選択肢のインデックス番号として環境変数 `%ERRORLEVEL%` に返されます。

自由入力を許さず、特定のキー入力のみを強制したい場合に非常に有用です。

(出自: Windows標準搭載 / 旧リソースキット由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `set /p` | 任意の文字列を入力可能。エラーチェックを自前で書く必要がある。 | 自由入力なら `set /p` |
| `pause` | 「続行するには何かキーを押してください...」と表示するだけで、どのキーか判別できない。 | 単なる一時停止なら `pause` |
| `timeout` | 指定秒数待機する。キー入力で中断できるが、選択肢は持てない。 | 待機が主目的なら `timeout` |

## よく連携されるコマンド

`choice` は、入力直後にその結果（ERRORLEVEL）を判定する `if errorlevel` と組み合わせて使用されます。

### シナリオ例: メンテナンス・メニューの作成 (運用自動化)

* **目的**: 複数の運用タスクから1つを選択して実行する。
* **連携コマンド**: `if errorlevel`, `goto`
* **解説**: `ERRORLEVEL` は「数値以上」を判定するため、**大きな数値から順に判定する**のがバッチファイルの定石です。
* **コマンド例**:
    ```cmd
    @echo off
    echo 1: ログのクリーニング
    echo 2: サービスの再起動
    echo 3: 終了
    choice /c 123 /m "実行する番号を選んでください:"

    if errorlevel 3 goto END
    if errorlevel 2 goto RESTART
    if errorlevel 1 goto CLEAN

    :CLEAN
    echo ログを清掃中...
    goto END
    :RESTART
    echo サービスを再起動中...
    goto END
    :END
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本設定** | |
| ⭐ `/C <choices>` | 選択肢となる文字列を指定（例: `/C YNC`）。デフォルトは `YN`。 |
| ⭐ `/M "<message>"` | 選択前に表示するメッセージ（プロンプト）を指定する。 |
| `/N` | プロンプト内に選択肢の一覧（例: `[Y,N]?`）を表示しない。 |
| `/CS` | 選択肢の大文字と小文字を区別する。 |
| **タイムアウト設定** | |
| `/T <seconds>` | 入力を待機する秒数を指定（0～9999秒）。 |
| `/D <choice>` | タイムアウト時に自動的に選択されるデフォルトの文字を指定。`/T` と併用必須。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー再起動前の最終確認（タイムアウト付き）
* **組み合わせ**: `/T`, `/D`
* **解説**: 30秒間入力がなければ、安全のために「No（再起動しない）」を選択させてスクリプトを終了させます。
* **例**:
    ```cmd
    choice /c YN /t 30 /d N /m "サーバーを再起動しますか？"
    if errorlevel 2 exit /b
    shutdown /r /t 0
    ```

### 2. ブルーチーム視点

* **タスク**: インシデント発生時のネットワーク隔離判断
* **組み合わせ**: `if errorlevel`
* **解説**: 調査用端末で、特定の感染端末を隔離するかどうかをアナリストに即座に判断させ、結果をログに記録します。
* **例**:
    ```cmd
    choice /c YN /m "端末 [PC-001] をネットワークから切断しますか？"
    if errorlevel 2 (
        echo [%DATE% %TIME%] アナリストが切断を拒否しました >> audit.log
    ) else (
        echo [%DATE% %TIME%] 切断を実行しました >> audit.log
        REM 切断コマンド実行
    )
    ```

### 3. レッドチーム視点

* **タスク**: アンチ・サンドボックス（遅延実行）
* **組み合わせ**: `/T`, `/D`
* **解説**: サンドボックス（自動解析環境）は短時間で処理を終えようとします。`choice` で長い待機時間を設け、かつ「特定のキーが押された時のみ」ペイロードを展開することで、自動解析を回避します。
* **例**:
    ```cmd
    REM 60秒待機し、かつ 'G' を押さない限り自爆（終了）する
    choice /c G /t 60 /d G /n /m "Loading..."
    if errorlevel 1 start payload.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `ERROR: Invalid syntax. /D must be used with /T.`
    * **考えられる原因**: デフォルトの選択肢 `/D` を指定したが、タイムアウト秒数 `/T` が指定されていない。
    * **解決策**: 両方のオプションをセットで指定する。

2.  **エラーメッセージ例 2**: (Ctrl+C を押した場合)
    * **挙動**: ユーザーが `Ctrl+C` や `Ctrl+Break` で中断しようとすると、`ERRORLEVEL` 0 が返されます。
    * **解決策**: スクリプト側で 0 が返った場合の処理を考慮するか、中断を禁止する設定（`break` 等）を検討する。

## 環境変数と設定ファイル

* **`%ERRORLEVEL%`**: `choice` の実行結果が格納される最も重要な変数です。
    * 最初(左端)の文字 = 1
    * 2番目の文字 = 2 ... と続きます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **Social Engineering**: 「Press 'Y' to install updates」といった偽のプロンプトを出し、ユーザーに悪意ある操作を承認させる。
    * **Anti-Sandbox**: 解析環境での実行を遅延させる、あるいは人間の介在を要求させることで、動的解析を困難にする。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `choice.exe` 自体が直接攻撃（ファイルのダウンロード等）を行うことはありませんが、**実行フローの制御（Delaying）**において活用されます。
* **参照**: [LOLBAS - choice.exe](https://lolbas-project.github.io/lolbas/Binaries/Choice/) (※現在は主に `timeout` や `ping` が遅延に使われますが、`choice` も有効です)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * バッチファイル内で `/T` による長時間の待機が含まれている場合、解析回避の意図がないか調査する。
    * 不審なバイナリと一緒に `choice` がループ内で回されているような挙動を監視する。

## 注意点・補足

* **戻り値の順番**: 前述の通り、`if errorlevel 2` は「2以上」を意味します。1と2を区別したい場合は、必ず 2, 1 の順で判定するか、`if %ERRORLEVEL% == 1` のように等号を使用してください。
* **OSバージョン**: 非常に古い Windows (XPなど) では標準搭載されていない場合がありましたが、Vista 以降は標準コマンドとして復活しています。
