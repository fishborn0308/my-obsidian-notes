# Command  - Linux - fail2ban - ログ監視によるブルートフォース攻撃防御

## 概要

`Fail2ban` は、ログファイルを監視し、設定されたパターン（例: 短時間での大量のログイン失敗）に一致する不審なアクセス元IPアドレスを、一定期間自動的にファイアウォール（`iptables`, `firewalld`など）でブロックするための侵入防止ソフトウェア（IPS）です。SSH、FTP、Webアプリケーションのログインフォームなど、様々なサービスをブルートフォース攻撃やパスワードスプレー攻撃から保護するための、シンプルかつ効果的なツールです。

`Fail2ban` はファイアウォールそのものではなく、ログを監視して**ファイアウォールを動的に操作する自動化ツール**です。

(出自: `fail2ban` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`iptables` / `firewalld`**| これらは`Fail2ban`が**操作する対象**のファイアウォールです。`Fail2ban` はログ監視の結果に基づき、これらのツールを使ってIPアドレスをブロックするルールを自動的に追加・削除します。 | - |
| **IDS/IPS (Snortなど)**| より高度な侵入検知・防御システム。パケットの内容を詳細に分析し、広範なネットワーク攻撃を検知します。`Fail2ban` はログベースの認証攻撃対策に特化しています。 | - |
| **WAF** | Webアプリケーションの保護に特化し、SQLインジェクションやXSSなどを防ぎます。`Fail2ban` はIPアドレスベースでブロックする、より低レイヤーの防御です。 | - |

## よく連携されるコマンド

`Fail2ban` は設定ファイルを編集してサービスを再起動する、という運用が基本のため、直接的なコマンド連携は少ないですが、その動作結果は他のコマンドで確認します。

### シナリオ例: ブロックされたIPアドレスを確認する (ブルーチーム視点)

* **目的**: Fail2banによって現在ブロックされているIPアドレスの一覧を確認する。
* **連携コマンド**: `fail2ban-client`
* **解説**: `fail2ban-client status <jail>` を実行することで、特定のjail（例: `sshd`）でブロックされているIPアドレスのリストや、検知回数などの統計情報を確認できます。
* **コマンド例**:

```bash
# sshd jailで現在ブロックされているIPアドレスを確認
sudo fail2ban-client status sshd
```

## オプション説明 (`fail2ban-client`)

| オプション | 説明 |
| :--- | :--- |
| **状態確認** | |
| ⭐ `status` | Fail2banの全体ステータスと、有効になっているJailの一覧を表示します。 |
| ⭐ `status <jail>` | 特定のJail (`sshd`など) の詳細ステータス（ブロック中のIPアドレス一覧など）を表示します。 |
| **手動操作** | |
| `set <jail> banip <ip>`| 手動で特定のIPアドレスをブロックします。 |
| `set <jail> unbanip <ip>`| ブロック中のIPアドレスを手動で解除します。 |
| **設定管理** | |
| `reload` | 設定ファイルを再読み込みします。`jail.local` を変更した後に実行します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: SSHサーバーをブルートフォース攻撃から保護する。
* **組み合わせ**: `/etc/fail2ban/jail.local` ファイルの編集
* **解説**: `jail.local` ファイルに保護したいサービスのJail設定を記述します。`enabled = true` にし、必要に応じて `maxretry` (試行回数) や `bantime` (ブロック期間) を調整するのが最も基本的な使い方です。
* **例 (`/etc/fail2ban/jail.local`)**:

```ini
[sshd]
enabled = true
port    = ssh
maxretry = 3
bantime = 1h
```

### 2. ブルーチーム視点

* **タスク**: どのようなIPアドレスから攻撃を受けているかをリアルタイムで把握する。
* **組み合わせ**: `fail2ban-client status <jail>`
* **解説**: インシデント調査中に、ブロックされているIPアドレスの一覧を取得します。このIPリストは、他の脅威インテリジェンスと照合したり、より上位のファイアウォールで恒久的にブロックしたりするためのIOC (Indicator of Compromise) となります。
* **例**:

```bash
# sshd jailでブロック中のIPアドレスを確認
sudo fail2ban-client status sshd
```

### 3. レッドチーム視点

* **タスク**: 標的サーバーのFail2ban設定を偵察し、検知を回避する。
* **組み合わせ**: (ログイン試行による偵察)
* **解説**: 意図的に数回SSHログインを失敗させてみて、接続が拒否されるようになるかを確認し、`maxretry` や `bantime` を推測します。
* **回避策**:
	* **Low and Slow攻撃**: `maxretry` に達しないように、非常にゆっくりとしたペースでパスワード試行を行う。
	* **分散ブルートフォース攻撃**: 大量のIPアドレスから、各IPが数回ずつしか試行しないように分散させて攻撃する。

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **Jailを有効化したのに、IPがブロックされない。**
		* **考えられる原因**:
				* Filterの正規表現が、ログファイルの実際のエラーメッセージと一致していない。
				* `jail.local` の `logpath` で指定したログファイルのパスが間違っている。
		* **解決策**: `fail2ban-regex` というテストツールが非常に便利です。`sudo fail2ban-regex /var/log/auth.log /etc/fail2ban/filter.d/sshd.conf` のように実行し、フィルターがログに正しくマッチするかを確認できます。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](OS%20%20-%20Linux%20-%20environment_and_config%20-%20環境変数と設定ファイル.md) を参照。

### 関連する設定ファイル

* **`/etc/fail2ban/fail2ban.conf`**: デーモン自体のグローバル設定。
* **`/etc/fail2ban/jail.conf`**: デフォルト設定集。**編集しないこと**。
* ⭐ **`/etc/fail2ban/jail.local`**: **ユーザーがカスタマイズを行うためのファイル**。`jail.conf` の設定を上書きします。
* **`/var/log/fail2ban.log`**: `Fail2ban` 自体の動作ログ。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: Fail2ban自体よりも、それが依存するログやファイアウォールの設定不備がリスクとなります。
* **悪用シナリオ**: 攻撃者がログを改ざん・削除して検知を逃れたり、分散ブルートフォース攻撃でBANを回避したりします。また、リバースプロキシ環境下での設定ミスは、正規のプロキシサーバーをBANしてしまい、全ユーザーがアクセスできなくなる自己DoSを引き起こす可能性があります。

### GTFOBins / LOLBAS における利用例

Fail2banは防御側のツールであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `ignoreip` 設定で信頼できるIP（自社のプロキシなど）をBAN対象から除外する。パスワード認証を無効化し、SSH公開鍵認証のみを許可することが、より根本的な対策となる。
* **Detection (検知)**: Fail2banのログをSIEMに転送し、BANイベントを集中監視する。

## 注意点・補足

* **リアクティブな防御**: Fail2banは、数回の不正な試行が**行われた後**にブロックする、事後対応的なツールです。攻撃の最初の数回は防げません。
