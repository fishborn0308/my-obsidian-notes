# Command - Linux - pstree - プロセスの親子関係を樹形図で表示する

## 概要

`pstree` は、実行中のプロセスをツリー形式で表示します。すべてのプロセスの始祖である `systemd` (または `init`) を根（ルート）として、どのプロセスがどのプロセスを生成（fork）したのかを一目で把握できます。

## 🛠 `ps` vs `pstree`

| 項目 | `ps` | `pstree` |
| :--- | :--- | :--- |
| **表示形式** | リスト（一覧）形式 | **ツリー（樹形図）形式** |
| **得意分野** | 特定のプロセスの詳細（CPU/メモリ） | **プロセスの階層・依存関係の把握** |
| **視認性** | PID を追う必要がある | 親子が線で結ばれており直感的 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-p` | **最頻用。** プロセス ID (PID) も一緒に表示する。 |
| ⭐️ `-a` | **最頻用。** コマンドライン引数（実行時のオプション等）も表示する。 |
| ⭐️ `-u` | プロセスの所有ユーザーを表示する（所有者が変わる箇所が明確になる）。 |
| ⭐️ `-h` | 現在のプロセスとその親プロセスをハイライト表示する。 |
| ⭐️ `-n` | 名前のアルファベット順ではなく、PID 順で並べ替える。 |
| `-c` | 同名のプロセスをまとめず、すべて個別に表示する（デフォルトは `2*[nginx]` のようにまとめられる）。 |

## コマンド例

```bash
# 最も情報量の多い「全部入り」表示
pstree -pau

# 特定のユーザーのプロセスツリーだけを見る
pstree alice -p
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 多重起動しているデーモン（Apache, Nginx, PHP-FPM 等）の構造把握。
- **解説**: 親プロセスがどれで、子がいくつぶら下がっているかを確認します。設定変更時のリロード（HUP信号）がどの子まで波及しているかの確認に役立ちます。

### 2. ブルーチーム視点

- **タスク**: 不自然な親子関係の検知（バックドア調査）。
- **解説**: ⭐️ **重要。** 例えば `www-data` ユーザーが動かす `apache2` や `nginx` の下に、本来あるはずのない `sh` や `nc` (Netcat) がぶら下がっていたら、それはウェブ脆弱性を突かれたリバースシェルの決定的な証拠です。
- **例**:

    Bash

    ```
    # ウェブサーバー周辺の怪しい子プロセスを視覚的に探す
    pstree -pau | grep -A 5 httpd
    ```

### 3. レッドチーム視点

- **タスク**: 寄生先プロセスの選定（Process Injection）。
- **解説**: 権限昇格後、どのプロセスに自分のツールを紛れ込ませるのが最も自然かを判断します。非常に長い期間動き続けている親プロセスを見つけ、その子プロセスとして振る舞うことで、監視の目を欺きます。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: プロセスの親子関係がわかることで、システムの内部構造（どのミドルウェアがどのスクリプトを呼び出しているか等）が攻撃者に筒抜けになります。

### 対応策・緩和策 (ブルーチーム視点)

- **hidepid の導入**: `ps` と同様、`/proc` マウント時の `hidepid` オプションを有効にすることで、一般ユーザーが他人のプロセスツリーを覗き見るのを防ぐことができます。

## 知っておくと便利な豆知識

- `pstree` で `---` と繋がっているのは「子プロセス」ですが、`{...}` と表示されているのは「スレッド（LWP: Light Weight Process）」です。これを知っていると、マルチスレッドアプリの挙動がより深く理解できます。
