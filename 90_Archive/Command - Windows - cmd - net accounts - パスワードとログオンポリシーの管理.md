# Command - Windows - cmd - net accounts - パスワードとログオンポリシーの管理

## 概要

`net accounts` コマンドは、ユーザーアカウントのパスワード要件（最小文字数、有効期限など）およびログオン制限（アカウントロックアウトの閾値など）を表示または変更するために使用されます。

ローカルコンピュータのポリシーだけでなく、`/domain` スイッチを使用することでドメイン全体のパスワードポリシーを確認することも可能です。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `secpol.msc` | GUIツール（ローカルセキュリティポリシー）。視覚的に設定可能。 | 初心者・単体管理 |
| `gpresult` | 実際に適用されているグループポリシー（GPO）の結果を確認する。 | GPOトラブルシューティング |
| `Get-ADDefaultDomainPasswordPolicy` | PowerShell版。ADドメインのパスワードポリシーを詳細なオブジェクトで取得。 | AD管理 (PS環境) |

## よく連携されるコマンド

### シナリオ例: パスワード攻撃前のポリシー調査 (レッドチーム視点)

* **目的**: アカウントロックアウトを避けつつ、パスワードスプレー攻撃が可能か判断する。
* **連携コマンド**: `net accounts`, `net user`
* **解説**: 総当たり攻撃を開始する前に、何回失敗したらロックされるか（閾値）を調査します。
* **コマンド例**:
    ```cmd
    REM ドメイン全体のパスワードポリシーとロックアウト閾値を確認
    net accounts /domain
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | 現在のローカルパスワードポリシー設定を表示する。 |
| ⭐ `/domain` | **【最重要】** ローカルではなく Active Directory ドメインのポリシーを表示・変更する。 |
| **ポリシー設定** | |
| `/minpwlen:length` | パスワードの最小文字数を設定（0〜20文字）。 |
| `/maxpwage:days` | パスワードの最大有効期間を設定。 `unlimited` で無期限。 |
| `/minpwage:days` | パスワード変更後の最小使用期間を設定。 |
| `/uniquepw:number` | パスワード履歴を保持する世代数を指定。過去のパスワードの再利用を制限。 |
| `/lockoutthreshold:n` | アカウントがロックされるまでの不当なログオン試行回数（1〜999）。 |
| `/lockoutduration:m` | ロックされたアカウントが自動的に解除されるまでの時間（分）。 |
| `/lockoutwindow:m` | ログオン試行回数がリセットされるまでの時間（分）。 |
| `/sync` | ドメインコントローラと強制的に同期させる。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: セキュリティ要件に基づき、ローカルアカウントの最低文字数を12文字以上に設定する。
* **例**:
    ```cmd
    REM ローカルの最小パスワード長を12文字に設定
    net accounts /minpwlen:12
    ```

### 2. ブルーチーム視点

* **タスク**: パスワードの有効期限が適切に設定されており、無期限（unlimited）になっていないか監査する。
* **例**:
    ```cmd
    REM ドメインポリシーを表示し、"パスワードの最長期間" を確認
    net accounts /domain | findstr "最長期間"
    ```

### 3. レッドチーム視点 (偵察と偵察)

* **タスク 1: アカウントロックアウト情報の窃取**
    * **目的**: 「何回までならパスワードを間違えてもバレないか」を知る。
    * **解説**: `Lockout threshold` が `Never` であればロックされませんが、数値が入っている場合はその回数未満で攻撃を止める必要があります。
    * **コマンド例**:
        ```cmd
        net accounts /domain
        ```
    * **実務ログ例**:
        ```text
        強制ログオフまでの時間:                    なし
        パスワードの最短期間 (日):                 1
        パスワードの最長期間 (日):                 90
        パスワードの最小の長さ:                    8
        保持されるパスワード履歴の長さ:            5
        ロックアウトしきい値:                      5     <-- 5回失敗でロックされる！
        ロックアウト期間 (分):                     30
        ロックアウト観測ウィンドウ (分):           30
        コマンドは正常に終了しました。
        ```

* **タスク 2: パスワードの複雑性要件の推測**
    * **目的**: 推測するパスワードの候補を絞り込む。
    * **解説**: 最小文字数が `12` であれば、8文字の単語リストを使う必要がないことが分かり、攻撃効率を高められます。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 設定の変更には管理者権限が必要です。
    * **解決策**: 「管理者として実行」したプロンプトを使用してください（表示だけであれば一般ユーザーでも可能な場合があります）。

2.  **The /DOMAIN option is only available on Windows domain members.**
    * **原因**: ワークグループ環境の端末で `/domain` スイッチを使用した。
    * **解決策**: ドメイン参加済み端末から実行してください。

## 環境変数と設定ファイル

* **Registry**: `HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account` 以下にポリシートデータが格納されていますが、直接編集は推奨されません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **パスワードスプレー攻撃の準備 (T1110.003):**
    * 攻撃者は `net accounts /domain` を実行して、ターゲット組織のロックアウトポリシーを事前に把握します。例えば閾値が「5回」であれば、各ユーザーに対して「4回」までの試行に抑えることで、SOC（セキュリティ監視チーム）に検知されずに攻撃を継続します。
* **弱いポリシーの特定**:
    * `minpwlen` が `0` や低い値になっているサーバーを特定し、ブルートフォース攻撃の容易なターゲットとしてマークします。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: 攻撃者は Mimikatz や Nmap などのツールを使う前に、この標準コマンドで「防御側の設定（ルール）」を読み取ります。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **強力なパスワードポリシーの適用**: NISTなどのガイドラインに則り、長さだけでなく、ロックアウト閾値と解除時間を適切に設定します。
    * **FGPP (Fine-Grained Password Policies)**: 特定の特権ユーザー（管理者）に対して、一般ユーザーよりも厳しいポリシーを個別に適用します。
* **Detection (検知)**:
    * **イベントID 4767**: ユーザー アカウントがアンロックされました。
    * **イベントID 4740**: ユーザー アカウントがロックアウトされました（これが大量に発生している場合は、攻撃者がポリシーを読み違えたか、強行突破を試みています）。

## 注意点・補足

* **日本語環境の表示**: 出力結果の項目名は OS の言語に依存します。スクリプトで解析（パース）する場合は `findstr` のキーワードに注意してください。
* **GPOとの競合**: `net accounts` でローカルに設定を変更しても、上位のグループポリシー (GPO) で設定が上書きされる場合があります。
