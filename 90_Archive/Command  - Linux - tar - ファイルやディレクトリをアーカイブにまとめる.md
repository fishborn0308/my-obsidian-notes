# Command  - Linux - tar - ファイルやディレクトリをアーカイブにまとめる

## 概要

`tar` (tape archive) は、多数のファイルやディレクトリを、**単一のアーカイブファイル** (`.tar` ファイル) にまとめるためのコマンドです。元々は磁気テープにバックアップを作成するために開発されました。

`tar` 自体には圧縮機能はありませんが、`gzip` (`-z`) や `bzip2` (`-j`) といった圧縮ツールとシームレスに連携できるため、ファイルのバックアップ、配布、転送など、あらゆる場面で「**アーカイブと圧縮**」を一度に行うための標準的なツールとして利用されています。

(出自: `tar` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`zip`** | **単一のコマンド**でアーカイブと圧縮を行います。Windowsとの互換性が高いです。`tar` はLinux/Unix環境の標準であり、パーミッションやシンボリックリンクの保持に優れています。 | `tar` (Linux環境) |
| **`gzip` / `bzip2`** | これらは**単一のファイル**を圧縮するためのツールです。ディレクトリを扱うには、まず `tar` で一つにまとめてから圧縮する必要があります。 | - |
| **`cp -a`** | ディレクトリを属性を保持したまま**コピー**します。`tar` はディレクトリを**単一のファイルに**まとめます。 | - |

## よく連携されるコマンド

`tar` はパイプと `ssh` を連携させることで、中間ファイルを作成せずにリモートホストへのバックアップやリストアを行うことができます。

### シナリオ例: SSH経由でリモートにディレクトリを転送する (インフラ構築・運用視点)

* **目的**: ローカルのディレクトリを、ディスクに一時的なアーカイブファイルを作成することなく、直接リモートサーバーに転送して展開する。
* **連携コマンド**: `ssh`, パイプ (`|`)
* **解説**: `tar czf - ...` のように、アーカイブファイル名として `-` を指定すると、結果が標準出力に送られます。これをパイプで `ssh` に渡し、リモート側で `tar xzf -` を実行して標準入力からデータを受け取り展開します。
* **コマンド例**:

		```bash
		# ローカルの current-dir をリモートホストの /backup/ に転送

		tar czf - ./current-dir | ssh user@remote-host "cd /backup/ && tar xzf -"

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **操作モード** | |
| ⭐ `-c`, `--create` | 新しいアーカイブを**作成**します。 |
| ⭐ `-x`, `--extract`| アーカイブからファイルを**展開（取り出し）**します。 |
| ⭐ `-t`, `--list` | アーカイブの**中身を一覧表示**します。展開はしません。 |
| **共通オプション** | |
| ⭐ `-f <file>`| 操作対象のアーカイブファイル名を指定します。ほぼ常に必須です。 |
| ⭐ `-v`, `--verbose` | 処理中のファイル名を詳細に表示します。 |
| **圧縮形式** | |
| ⭐ `-z`, `--gzip` | **gzip** を使って圧縮/展開します。拡張子は `.tar.gz` または `.tgz`。 |
| `-j`, `--bzip2`| **bzip2** を使って圧縮/展開します。拡張子は `.tar.bz2` または `.tbz2`。 |
| `-J`, `--xz` | **xz** を使って圧縮/展開します。拡張子は `.tar.xz`。bzip2より高圧縮。 |
| **その他** | |
| `-p` | ファイルのパーミッション情報を保持したままアーカイブ/展開します。 |
| `-C <dir>` | 指定したディレクトリに移動してから操作を開始します。展開先の指定に便利です。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディレクトリを `gzip` で圧縮しながら、新しいアーカイブファイルとして作成する。
* **組み合わせ**: `tar czf <archive_name>.tar.gz <directory_to_archive>`
* **解説**: **C**reate **Z**ip **F**ile のように覚えると便利な、最も代表的な使い方です。Webサーバーのドキュメントルートのバックアップなどに使用します。
* **例**:

		```bash
		# /home/user ディレクトリを user-backup.tar.gz という名前で圧縮・アーカイブ

		tar czf user-backup-$(date +%F).tar.gz /home/user

		```

## 2. ブルーチーム視点

* **タスク**: 不審なアーカイブファイルを、安全に内容物を確認する。
* **組み合わせ**: `tar tf <archive_file>.tar.gz`
* **解説**: `-t` (`--list`) を使い、アーカイブを実際にディスクに展開することなく、中にどのようなファイルが含まれているかを確認します。これにより、マルウェアなどを安全に調査できます。
* **例**:

		```bash
		# 侵害されたサーバーから収集した不審な .tar.gz ファイルの中身をリストアップ

		tar tf suspicious_archive.tar.gz

		```

## 3. レッドチーム視点

* **タスク**: **データ窃取 (Data Exfiltration)**。窃取した複数のファイルやディレクトリを、転送しやすいように1つの圧縮ファイルにまとめる。
* **組み合わせ**: `tar czf loot.tar.gz /path/to/stolen/data`
* **解説**: 攻撃者はシステムから `/etc` ディレクトリ、ユーザーのホームディレクトリなどを盗み出した後、`tar` でそれらを1つのファイルにまとめ、自身のサーバーに転送します。
* **例**:

		```bash
		# 窃取した /etc と /home/user をまとめて圧縮

		tar czf loot.tar.gz /etc /home/user

		```

		* **考えられる原因**: アーカイブファイルが破損しているか、ダウンロードが不完全です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **Tar Slip / Zip Slip 脆弱性**。
* **悪用シナリオ**: アーカイブ内に `../../../../etc/passwd` のような、ディレクトリトラバーサルを狙った悪意のあるパス名を持つファイルを含める攻撃です。もし、特権ユーザーが安全でないスクリプトでこのようなアーカイブを展開してしまうと、システム上の重要なファイルが上書きされ、システムの乗っ取りに繋がる可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo tar` を実行できる権限を持っている場合、`--checkpoint` や `--checkpoint-action` といったあまり知られていないオプションを悪用して、任意のコマンドをroot権限で実行できます。
* **参照**: `https://gtfobins.github.io/gtfobins/tar/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 信頼できないソースからのアーカイブファイルは、専用のディレクトリ内で、かつ権限の低いユーザーで展開する。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `auditd` を使い、`tar` コマンドによる予期せぬ場所へのファイル書き込みを監視する。

## 注意点・補足

* **`-xzf` の覚え方**: **X**tract **Z**ip **F**ile のように覚えると便利です。（厳密にはZipではありませんが）
