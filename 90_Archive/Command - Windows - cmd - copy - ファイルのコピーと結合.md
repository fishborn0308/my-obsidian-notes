# Command - Windows - cmd - copy - ファイルのコピーと結合

## 概要

`copy` は、1つまたは複数のファイルを別の場所にコピーするための最も基本的な内部コマンドです。ファイルの複製だけでなく、複数のテキストファイルを1つに結合したり、特殊なデバイス（プリンタ等）にファイルを送る際にも使用されます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `xcopy` | ディレクトリ構造（サブフォルダ）を再帰的にコピーできる。 | フォルダ単位 |
| `robocopy` | ミラーリングや再試行が可能。大量のファイル転送に最適。 | 大規模移行 |

## よく連携されるコマンド

### シナリオ例: 代替データストリーム (ADS) を利用した隠蔽 (レッドチーム視点)

* **目的**: 通常のエクスプローラーや `dir` コマンドでは見えない場所にファイルを隠す。
* **連携コマンド**: `copy`, `type`
* **解説**: `copy` を使用して、既存のファイル（例: `text.txt`）の裏側（ストリーム）に別のファイル（例: `malicious.exe`）を書き込みます。
* **コマンド例**:
    ```cmd
    REM 実行ファイルをテキストファイルの代替データストリームとしてコピー
    copy malicious.exe text.txt:malicious.exe
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| `/V` | 新しいファイルが正しく書き込まれたか検証する。 |
| `/Y` | 既存のファイルを上書きする際、確認メッセージを表示しない。 |
| `/-Y` | 上書き時に確認メッセージを表示する。 |
| `/B` | バイナリファイルを指定する。 |
| `/A` | ASCII テキストファイルを指定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルのバックアップを、タイムスタンプを付けて作成する。
* **例**:
    ```cmd
    REM 設定ファイルを .bak としてコピー
    copy config.xml config.xml.bak
    ```

### 2. レッドチーム視点

* **タスク**: 複数の断片化されたペイロードを 1 つの実行ファイルに結合する。
* **例**:
    ```cmd
    REM 2つのファイルをバイナリモードで結合して1つのexeにする
    copy /B part1.dat+part2.dat complete_payload.exe
    ```

## セキュリティに関する考慮事項

* **ADSの悪用**: 上述の通り、`copy` はファイルを隠すために使われることがあります。
* **LOLBAS**: `copy` 自体はファイルを複製するだけですが、スクリプト実行ツールなどを「信頼された場所」へコピーして防御を回避する前段として使われます。
    * 参照: [LOLBAS - copy.exe](https://lolbas-project.github.io/lolbas/Binaries/Copy/)
