# Command - Windows - cmd - dir - ディレクトリの内容表示と情報の列挙

## 概要

`dir` コマンドは、ディレクトリに含まれるファイルとサブディレクトリのリストを表示します。

ファイルサイズ、最終更新日時、属性（隠しファイル等）を確認できるほか、特定の条件（拡張子、作成日時など）でフィルタリングして検索する機能も備えています。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ls` | Linux 標準。出力形式が異なる（Windowsの `dir` はデフォルトで詳細情報を出す）。 | (Linux環境) |
| `Get-ChildItem` | PowerShell 版。出力をオブジェクトとして扱い、複雑なソートや加工が可能。 | PS 環境 |
| `tree` | ディレクトリ構造をツリー形式で視覚的に表示する。 | 構造の全体把握 |

## よく連携されるコマンド

### シナリオ例: 特定の拡張子を持つ機密ファイルの全検索 (レッドチーム視点)

* **目的**: ターゲット端末内の全ドライブから、パスワードが含まれていそうな設定ファイルやデータベースファイルを一括で探し出す。
* **連携コマンド**: `dir`, `findstr`
* **解説**: `/s`（サブディレクトリ）とワイルドカードを使用し、結果を `findstr` で絞り込みます。
* **コマンド例**:
    ```cmd
    REM Cドライブ内のすべての .config または .xml ファイルを再帰的に検索
    dir /s /b C:\*.config C:\*.xml | findstr /i "pass config secret"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `/b` | 属性やサイズなどを省き、**ファイル名（またはパス）のみ**をベタ書きで表示する。スクリプト処理に最適。 |
| `/w` | ワイド一覧形式で表示。1行に複数のファイル名を並べる。 |
| `/l` | ファイル名とディレクトリ名を小文字で表示する。 |
| **検索・抽出** | |
| ⭐ `/s` | 指定したディレクトリおよび**すべてのサブディレクトリ**のファイルを表示する。 |
| ⭐ `/a[:attr]` | 指定した属性のファイルを表示する（D:ディレクトリ, H:隠し, S:システム, R:読み取り専用）。 |
| **ソート・日時** | |
| `/o[:sort]` | ソート順を指定（N:名前, S:サイズ, E:拡張子, D:日時）。 |
| ⭐ `/t[:time]` | 表示またはソートに使用するタイムスタンプを指定（C:作成, A:最終アクセス, W:最終書き込み）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディスク容量を圧迫している巨大なログファイルを特定する。
* **例**:
    ```cmd
    REM サイズの大きい順（-S）にソートして表示
    dir /o-s /a-d C:\Logs
    ```

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: 特定の期間内に作成または変更されたファイルを探し、侵害のタイミングを特定する。
* **例**:
    ```cmd
    REM 作成日時（/tc）を表示し、不審なバイナリがいつ置かれたかを確認
    dir /s /tc C:\Windows\System32\*.exe
    ```

### 3. レッドチーム視点 (偵察と列挙)

* **タスク 1: Hidden File Discovery (隠しファイルの探索)**
    * **目的**: 管理者が隠しているメモ書きや、他の攻撃者が設置したツールを見つける。
    * **コマンド例**:
        ```cmd
        REM カレントディレクトリ内の隠しファイルのみを表示
        dir /ah
        ```

* **タスク 2: Interesting File Hunting (ターゲット選定)**
    * **目的**: `Unattend.xml`（自動セットアップファイル）など、平文パスワードが含まれる可能性が高いファイルを狙い撃ちする。
    * **コマンド例**:
        ```cmd
        dir /s /b C:\unattend.xml C:\sysprep.inf C:\web.config
        ```

* **タスク 3: Persistence Check (永続化の痕跡調査)**
    * **目的**: スタートアップフォルダに不審なショートカットやスクリプトが追加されていないか確認する。
    * **コマンド例**:
        ```cmd
        dir "C:\Users\%USERNAME%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup"
        ```

## エラーメッセージとトラブルシューティング

1.  **File Not Found (ファイルが見つかりません)**
    * **原因**: 指定したパスにファイルが存在しない、または隠しファイル属性が付いていて `/a` を指定していない。
    * **解決策**: パスを再確認するか、`dir /a` で属性に関わらず表示を試みる。

2.  **Access is denied. (アクセスが拒否されました)**
    * **原因**: 権限のないディレクトリ（例: `System Volume Information`）をスキャンしようとした。
    * **解決策**: 管理者権限で実行するか、エラーを無視して続行する。

## 環境変数と設定ファイル

* **`%DIRCMD%`**: この環境変数にオプションを設定しておくと、`dir` 実行時のデフォルト動作を変更できます。
    * 例: `set DIRCMD=/o:n /p` (常に名前順で、画面ごとに停止する)

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **情報の露出 (Information Disclosure - T1083):**
    * 攻撃者は `dir /s` を多用してネットワーク共有やローカルドライブを網羅的にスキャンし、バックアップ、ソースコード、証明書ファイルなどを特定します。
* **Timestomping の検知回避**:
    * 攻撃者はファイルの「最終書き込み日時」を偽装することがありますが、`dir /tc` で「作成日時」を確認することで、その矛盾を突いて侵害を発見できる場合があります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **参照**: [LOLBAS - dir](https://lolbas-project.github.io/lolbas/Binaries/Dir/) (※内部コマンドのため公式リストには掲載されないことが多いが、役割は同様)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection**:
    * 短時間に大量のディレクトリを再帰的にスキャン（`dir /s`）するプロセスを監視する。
    * 一般ユーザーが本来アクセスすべきでない領域に対して `dir` を試行しているログを SIEM で検知する。

## 注意点・補足

* **ジャンクションとシンボリックリンク**: `dir` の出力には `<JUNCTION>` や `<SYMLINK>` と表示されます。これらは別の場所へのショートカットであり、再帰スキャン時にループが発生しないよう注意が必要です。
* **パスの引用符**: スペースを含むパスを指定する場合は、必ず `"C:\Program Files"` のようにダブルクォーテーションで囲んでください。
