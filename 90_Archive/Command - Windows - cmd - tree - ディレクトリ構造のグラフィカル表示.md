# Command - Windows - cmd - tree - ディレクトリ構造のグラフィカル表示

## 概要

`tree` コマンドは、指定したドライブまたはパスのディレクトリ構造を、ツリー形式（樹形図）でグラフィカルに表示します。

`dir` コマンドよりも視覚的に親子関係が理解しやすいため、プロジェクトの構成確認や、大量のデータが格納されたサーバー内の整理状況を把握するのに最適です。

(出自: MS-DOS 以降標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `dir /s` | ファイルとディレクトリを再帰的にリスト表示するが、階層構造はテキストの羅列になる。 | `dir /s` (詳細情報) |
| `ls -R` | Linux 版の再帰表示。標準の `ls` にはツリー表示機能はない（別途 `tree` のインストールが必要）。 | `tree` (構造把握) |
| `Explorer` | GUI のエクスプローラー。直感的な操作は可能だが、構成をテキストとして書き出すには不向き。 | `tree` (記録用) |

## よく連携されるコマンド

`tree` の出力は非常に長くなることが多いため、ページ制御やファイル出力とセットで使われます。

### シナリオ例: プロジェクト構成のドキュメント化 (インフラ構築)

* **目的**: サーバー内のディレクトリ構成をテキストファイルに保存し、設計書や報告書に添付する。
* **連携コマンド**: リダイレクト (`>`)
* **解説**: `/f` でファイル名も含め、`/a` でテキスト形式（ASCII）にすることで、どのエディタでも崩れず閲覧できるようにします。
* **コマンド例**:
    ```cmd
    REM 構成をテキストファイルに書き出し
    tree C:\Projects\MyApps /f /a > project_structure.txt
    ```

## スイッチ/オプション説明

Windows 版の `tree` は非常にシンプルで、主要なオプションは 2 つだけです。

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/f` | **Files.** 各ディレクトリに含まれる **ファイル名** も含めて表示する。 |
| ⭐ `/a` | **ASCII.** 階層の区切り線にグラフィック文字（└─ など）ではなく、**テキスト文字** (+, -, | など) を使用する。テキストエディタに貼り付ける際に便利。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: バックアップ対象フォルダの事前確認
* **解説**: バックアップを回す前に、どの階層にどれくらいのサブフォルダがあるのかを視覚的にプレビューします。
* **例**: `tree D:\Data /a | more` (画面に収まらない場合に 1 ページずつ表示)

### 2. ブルーチーム視点

* **タスク**: インシデント発生時のファイル配置調査
* **解説**: マルウェアが自身のコピーをどのサブフォルダにばらまいたか、あるいはどのフォルダに機密データがステージング（集約）されているかを一俯瞰で調査します。
* **例**: `tree C:\Users\Public /f`

### 3. レッドチーム視点 (OSCP / Reconnaissance)

* **タスク**: ターゲット端末の内部情報の列挙
* **解説**: 侵入後、その端末にどのようなデータがあるか、開発環境なのか、機密文書があるのかを迅速に判断します。
* **例**:
    ```cmd
    REM ユーザーのドキュメントフォルダを高速にスキャン
    tree %USERPROFILE%\Documents /f /a
    ```

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: 出力が止まらず、画面が流れてしまう。
    * **原因**: 対象のディレクトリが深すぎる、またはファイル数が膨大すぎる。
    * **解決策**: `Ctrl + C` で強制終了するか、特定のサブディレクトリに絞って実行する。

2.  **現象**: テキストファイルに保存して開くと、線が文字化けしている。
    * **原因**: エディタの文字コード設定やフォントがグラフィック文字に対応していない。
    * **解決策**: `/a` オプションを付けて ASCII 文字で出力し直す。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Information Disclosure**: 攻撃者が `tree` を実行することで、通常の `dir` では見逃しがちな深い階層にあるバックアップファイルや設定ファイルを効率的に発見されるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - 重要なデータが含まれるディレクトリへのアクセス権（ACL）を適切に設定し、一般ユーザーが構造を走査（トラバース）できないようにする。
* **Detection (検知)**:
    - **イベントID 4688**: `tree.com` の実行を監視。特に Web サーバーの一時フォルダや、公開ディレクトリからこのコマンドが呼ばれた場合は、Web シェルによる偵察を疑います。

## 注意点・補足

* **実行ファイル名**: Windows の `tree` は実は `.com` 拡張子のファイル（`tree.com`）ですが、コマンドプロンプトからは単に `tree` で実行可能です。
* **Linux との最大の違い**: Linux の `tree` は表示する階層の深さを指定する `-L` オプションなど非常に多機能ですが、Windows 版にはそれがありません。
