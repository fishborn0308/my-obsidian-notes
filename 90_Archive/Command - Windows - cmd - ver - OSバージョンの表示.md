# Command - Windows - cmd - ver - OSバージョンの表示

## 概要

`ver` は、現在実行しているオペレーティング システムの名称とバージョン番号（ビルド番号を含む）を表示するコマンドです。

GUIを開く余裕がないシェル環境や、OSのバージョンによって処理を分岐させたいバッチスクリプト内で多用されます。

(出自: MS-DOS 2.0 以降、すべてのWindowsに標準搭載)

]

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `winver` | GUI版。ポップアップでライセンス情報やエディションも確認できる。 | `winver` (目視確認) |
| `systeminfo` | インストール日、ホットフィックス、メモリ量など「すべて」を表示。重い。 | `systeminfo` (詳細調査) |
| `ver` | **最速。** バージョン文字列だけを一行で返す。 | `ver` (スクリプト/初期調査) |

## よく連携されるコマンド

### シナリオ例: OSバージョンによる処理の自動分岐 (スクリプト)

* **目的**: Windows 10以前か、Windows 11（Build 22000以上）かによって実行するコマンドを変える。
* **連携コマンド**: `findstr`, `if`
* **解説**: `ver` の出力から特定のビルド番号を探し、ERRORLEVEL で判定します。
* **コマンド例**:
    ```cmd
    REM Windows 11 (28000系 = 26H1など) かどうかを判定
    ver | findstr "10.0.28000" > nul
    if %ERRORLEVEL% equ 0 (
        echo 最新の Windows 11 環境を検知しました。
    )
    ```

## スイッチ/オプション説明

Windows標準の `cmd.exe` における `ver` には、実質的なオプションは存在しません。

| スイッチ | 説明 |
| :--- | :--- |
| (なし) | バージョン情報を表示する。 |
| `/?` | ヘルプを表示する（が、表示される内容は極めてシンプル）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートキッティング時のOS確認
* **解説**: 大量のPCをセットアップする際、意図したビルド（例: 24H2, 26H1）が適用されているかを一瞬でログに残します。
* **例**: `ver >> setup_log.txt`

### 2. ブルーチーム視点

* **タスク**: サポート終了(EOL)バージョンの洗い出し
* **解説**: 資産管理ツールが使えない環境で、古いビルド（例: Windows 10の特定バージョン）が残っていないか調査スクリプトで収集します。
* **例**: `wmic os get caption, version` (verと併用して詳細を取得)

### 3. レッドチーム視点 (OSCP / Reconnaissance)

* **タスク**: カーネルエクスプロイトの適合性判断
* **解説**: 特定のビルド番号にのみ存在する脆弱性を突く際、`ver` で表示される末尾の数字まで細かくチェックします。
* **例**:
    - `10.0.19045` -> Windows 10 22H2
    - `10.0.22631` -> Windows 11 23H2
    - `10.0.28000` -> Windows 11 26H1 (2026年時点の最新候補)

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `ver` と打っても `Microsoft Windows [Version 10.0.xxxx]` としか出ず、Windows 11 なのか 10 なのか一見わからない。
    * **原因**: Windows 11 も内部的なメジャーバージョンは `10.0` のまま維持されているため。
    * **解決策**: ビルド番号（末尾の数字）で判断する。
        - 22000以上 = Windows 11
        - 19045以下 = Windows 10

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Reconnaissance**: 攻撃者が最初に実行する「三種の神器（whoami, ipconfig, ver）」の一つ。OSの正確なビルドを知ることで、パッチが当たっていない既知の脆弱性を特定されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    - `ver` 自体を止めることは難しいですが、`systeminfo` や `wmic` とセットで短時間に実行されている場合は、侵害後の初期偵察を疑うべきです。

## 注意点・補足

* **PowerShellでの挙動**: PowerShell上で `ver` と打つと、内部的に `cmd /c ver` が呼ばれます。PowerShell独自のOS情報が欲しい場合は `$PSVersionTable.OS` や `[System.Environment]::OSVersion` を使用します。
