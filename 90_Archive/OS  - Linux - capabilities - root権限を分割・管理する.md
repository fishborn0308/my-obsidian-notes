# OS  - Linux - capabilities - root権限を分割・管理する

## 概要

Linuxケーパビリティは、伝統的に`root`ユーザーに集約されていた特権を、より小さな独立した単位（ケーパビリティ）に分割するためのセキュリティ機構です。これにより、プロセスは全ての`root`権限を持つ代わりに、そのタスク遂行に**必要最小限の特権だけ**を付与されることができます。例えば、「特権ポート（1024番未満）にバインドする権限」だけをWebサーバに与え、ファイルシステムへの完全なアクセス権は与えない、といった運用が可能になります。これは、**最小権限の原則 (Principle of Least Privilege)** を実現するための重要な仕組みです。

(出自: `libcap` パッケージ (`getcap`, `setcap`) に含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **SUID (Set User ID)** | SUIDはファイルの所有者（通常はroot）の**全ての権限**でプロセスを動作させる「オール・オア・ナッシング」な仕組みです。ケーパビリティは、必要な権限だけを細かく付与できる点で優れています。 | Capabilities |
| **`sudo`** | `sudo` は**ユーザー**に基づいて、一時的にコマンド単位で権限を昇格させます。ケーパビリティは、**ファイル（実行バイナリ）**に永続的に権限を付与する点が異なります。 | - |
| **SELinux / AppArmor**| これらは**強制アクセス制御 (MAC)** と呼ばれる、より包括的なセキュリティシステムです。ケーパビリティは、特権操作に焦点を当てた、よりシンプルな権限昇格の制御機構です。 | - |

## よく連携されるコマンド

ケーパビリティは `find` と連携して、権限が付与されたファイルをシステム全体から検索する際によく使われます。

### シナリオ例: 権限昇格の足がかりを探す (レッドチーム視点)

* **目的**: システムに侵入後、ケーパビリティが不適切に付与されたファイルを探し出し、権限昇格の足がかりとする。
* **連携コマンド**: `find`, `getcap`
* **解説**: `find` でシステム全体をスキャンし、`getcap` を `-exec` で実行することで、ケーパビリティを持つファイルをリストアップします。特に、スクリプトインタプリタなどに強力なケーパビリティが付与されている場合、それは攻撃の起点となり得ます。
* **コマンド例**:

```bash
# システム全体でケーパビリティが設定されたファイルを探す
find / -type f -exec getcap {} + 2>/dev/null
```

## オプション説明 (`getcap`/`setcap`)

| オプション | 説明 |
| :--- | :--- |
| **`getcap` (表示)** | |
| ⭐ `-r` | ディレクトリを再帰的に検索します。 |
| **`setcap` (設定)** | |
| ⭐ `<capability>+ep <file>` | `<file>` に、指定したケーパビリティを**有効(effective)**かつ**許可(permitted)**セットとして付与します。 |
| `-r <file>` | 指定したファイルから全てのケーパビリティを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Webサーバをroot以外のユーザーで実行しつつ、ポート80番をリッスンさせる。
* **組み合わせ**: `sudo setcap 'cap_net_bind_service=+ep' /path/to/binary`
* **解説**: 実行ファイルに `CAP_NET_BIND_SERVICE` を付与し、root以外のユーザーで実行しても特権ポート（1024番未満）をリッスンできるようにします。これはSUIDを使わずに特権操作を許可する、モダンで安全な方法です。
* **例**:

```bash
# Webサーババイナリにポート80/443へのバインド権限を付与
sudo setcap 'cap_net_bind_service=+ep' /opt/my-app/my-web-server
```

### 2. ブルーチーム視点

* **タスク**: システムに不審なケーパビリティ設定がないか監査する。
* **組み合わせ**: `getcap -r / 2>/dev/null`
* **解説**: ファイルシステムのルート (`/`) から再帰的に (`-r`) 検索し、ケーパビリティが設定されたファイルを全てリストアップします。ベースラインと比較し、意図しないファイルに危険なケーパビリティが付与されていないかを確認します。
* **例**:

```bash
# システム全体をスキャンしてケーパビリティを持つファイルを探す
getcap -r / 2>/dev/null
```

### 3. レッドチーム視点

* **タスク**: 権限昇格の足がかりを探す。
* **組み合わせ**: `getcap -r / 2>/dev/null`
* **解説**: 侵入後、このコマンドでケーパビリティを持つファイルを探します。もし、`python` や `perl`, `tar` のような、他のファイルを実行したり書き換えたりできるプログラムに `CAP_SETUID` や `CAP_DAC_READ_SEARCH` のような強力なケーパビリティが付与されているのを発見した場合、それを悪用してroot権限を奪取しようと試みます。
* **例**:

```bash
# 権限昇格に悪用できる可能性のあるケーパビリティを持つファイルを探す
getcap -r / 2>/dev/null | grep -E "cap_setuid|cap_dac_read_search"
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `Failed to set capabilities on file '<file>' (Operation not supported)`
		* **考えられる原因**: ファイルシステムが拡張属性 (xattr) をサポートしていないか、マウントオプションで無効化されています。ケーパビリティはxattrを使ってファイルに保存されます。
		* **解決策**: `mount` コマンドでファイルシステムのマウントオプションを確認します。必要であれば、`user_xattr` オプションを付けて再マウントします。

## 環境変数と設定ファイル

* `getcap`/`setcap` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: ケーパビリティは、設定を誤ると**直接的な権限昇格（Privilege Escalation）の脆弱性**となります。
* **悪用シナリオ**: `CAP_SYS_ADMIN` のように過度に強力なケーパビリティを、本来不要なプログラムに付与することは、SUIDビットを付与するのと同様に危険です。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: 多くのGTFOBinsのエントリには、「Capabilities」というセクションがあり、権限昇格の手法が記載されています。例えば、もし `python` のバイナリに `CAP_SETUID` が付与されている場合、Pythonスクリプトを使って自身のプロセスのUIDを0 (root) に変更でき、結果としてrootシェルを起動できます。
* **参照**: `https://gtfobins.github.io/gtfobins/python/#capabilities`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、アプリケーションには本当に必要なケーパビリティのみを付与する。
* **Detection (検知)**: `getcap -r /` で定期的にシステムをスキャンし、意図しないケーパビリティ設定がないか監視する。特にスクリプトインタプリタや汎用ツールに強力なケーパビリティが設定されている場合は、即座に見直す必要があります。

## 注意点・補足

* **スクリプトへの設定不可**: ケーパビリティは、シェルスクリプトのようなテキストファイルに直接設定することはできません。ケーパビリティは、**実行バイナリ（インタプリタ自身）**に設定する必要があります。
* **コンテナセキュリティ**: ケーパビリティはコンテナセキュリティの根幹をなす技術です。Dockerはデフォルトでコンテナから多くの危険なケーパビリティを剥奪 (`drop`) します。`docker run --cap-add=NET_ADMIN` のようにして、コンテナに特定のケーパビリティを明示的に追加できます。
