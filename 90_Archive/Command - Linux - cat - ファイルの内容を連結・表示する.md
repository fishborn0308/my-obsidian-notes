# Command - Linux - cat - ファイルの内容を連結・表示する

## 概要

`cat` (con**cat**enate) コマンドは、指定されたファイルの内容を順番に読み込み、標準出力（通常は画面）に出力するための基本的なユーティリティです。主な用途は3つあります。

1. ファイルの内容を画面に表示する。
2. 複数のファイルを1つに連結する。
3. リダイレクションと組み合わせて新しいファイルを簡易的に作成する。

そのシンプルさから、多くのコマンドライン操作の基礎となります。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`less`, `more`** | これらは「ページャ」と呼ばれ、大きなファイルの内容を1画面ずつ対話的に閲覧するために使います。`cat` はファイル全体を一気に出力するため、画面が流れてしまいます。 | `less` |
| **`head`, `tail`** | `head` はファイルの**先頭部分**、`tail` はファイルの**末尾部分**を表示します。`cat` は**ファイル全体**を表示します。 | - |
| **`tac`** | `cat` を逆から書いた名前の通り、ファイルの内容を**最終行から逆順に**表示します。 | - |

## よく連携されるコマンド

`cat` の真価は、シェルのリダイレクションやパイプと組み合わせることで発揮されます。

### シナリオ例: ヒアドキュメントで設定ファイルを生成する (インフラ構築視点)

* **目的**: シェルスクリプト内で、複数行の設定ファイル (`config.json`) を対話なしで作成する。
* **連携コマンド**: ヒアドキュメント (`<<EOF`), リダイレクション (`>`)
* **解説**: ヒアドキュメントは、指定した終端文字列 (`EOF`など) までの複数行の入力を、コマンドの標準入力として渡す機能です。`cat` と組み合わせることで、スクリプト内に記述した内容をそのままファイルに出力できます。
* **コマンド例**:

```bash
# config.json というファイルにJSONデータを書き込む
cat <<EOF > config.json
{
	"hostname": "db-server-01",
	"port": 5432,
	"user": "admin"
}
EOF
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐️ `-n`, `--number` | 全ての行に行番号を付けて出力します。 |
| `-b`, `--number-nonblank` | 空行を除いた行にのみ行番号を付けて出力します。 |
| `-s`, `--squeeze-blank` | 2行以上連続する空行を、1行の空行にまとめて出力します。 |
| **特殊文字の表示** | |
| ⭐️ `-A`, `--show-all` | `-vET` と同等。タブ、行末、非表示文字を全て表示します。 |
| `-E`, `--show-ends` | 各行の末尾に `$` 記号を表示します。 |
| `-T`, `--show-tabs` | タブ文字を `^I` として表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システム情報を格納したファイルの内容を素早く確認する。
* **組み合わせ**: `cat <file>`
* **解説**: オプションなしで、カーネルバージョンやディストリビューション情報などを格納したファイルを直接表示します。
* **例**:

```bash
# OSのディストリビューション情報を確認
cat /etc/os-release
```

### 2. ブルーチーム視点

* **タスク**: 不審なスクリプトに隠された文字がないか調査する。
* **組み合わせ**: `cat -A <filename>`
* **解説**: `-A` オプションで、ファイル内の全ての非表示文字（タブ、改行、制御文字など）を可視化します。これにより、難読化や偽装のために埋め込まれた文字を発見できます。
* **例**:

```bash
# 不審なシェルスクリプトに隠された文字がないか調査
cat -A suspicious.sh
```

### 3. レッドチーム視点

* **タスク**: システム上の重要なファイルを読み取り、情報を窃取する。
* **組み合わせ**: `cat <sensitive_file>`
* **解説**: 権限昇格に成功した後、パスワードハッシュが保存されている `/etc/shadow` ファイルや、SSHの秘密鍵、設定ファイル内のパスワード、コマンド履歴などを読み取り、さらなる侵入拡大や情報収集の足がかりとします。
* **例**:

```bash
# ユーザーのコマンド履歴を読み取り、パスワードや接続先IPなどの情報を探す
cat /home/user/.bash_history
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **バイナリファイルに `cat` を実行すると、ターミナル画面が文字化けする**
		* **考えられる原因**: `cat` はテキストファイル用のコマンドであり、実行ファイルや画像などのバイナリデータを表示しようとすると、ターミナルが解釈できない制御文字が出力され、表示が乱れます。
		* **解決策**: `reset` コマンドを実行すると、ターミナルの状態をリセットできます。バイナリファイルの内容を確認したい場合は `xxd` や `hexdump` といったコマンドを使用します。

## 環境変数と設定ファイル

* `cat` は非常にシンプルなコアユーティリティであり、その動作に影響を与える特定の環境変数や設定ファイルはありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `cat` 自体は長年使われている安定したコマンドであり、脆弱性を持つ可能性は極めて低いです。
* **悪用シナリオ**: `cat` は、攻撃者がシステム侵入後に行う**情報収集 (Information Discovery)** のために必ずと言っていいほど使用されます。`/etc/passwd`, `/etc/shadow`, 設定ファイル、 SSHキー, コマンド履歴などを読み取り、機密情報を窃取します。

### GTFOBins / LOLBAS における利用例

* **機能**: **File Read** (ファイル読み取り), **File Write** (ファイル書き込み)
* **解説**: 主な用途はファイルの読み取りです。もし `sudoers` の設定で特定のユーザーが `cat` のみを `sudo` で実行できる場合、そのユーザーはシステム上の**あらゆるファイル**を読み取ることが可能になります (`sudo cat /etc/shadow` など）。また、リダイレクションを組み合わせることで、root権限でのファイル書き込みも可能です (`sudo cat > /etc/cron.d/backdoor`)。
* **参照**: `https://gtfobins.github.io/gtfobins/cat/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、ファイルパーミッションを適切に設定する。`sudoers` の設定を厳格にし、`cat` のようなコマンドに安易に権限を与えない。
* **Detection (検知)**: `auditd` などの監査ツールを用いて、重要なファイルへの読み取りアクセスを監視し、不審な挙動を検知する。

## 注意点・補足

* **Useless Use of Cat (UUOC)**: `cat file | grep 'pattern'` のように、パイプの先頭で `cat` を使うのは多くの場合冗長です。「Useless Use of Cat (ムダなcatの使い方)」として知られており、`grep 'pattern' file` と書くのがより効率的です。
