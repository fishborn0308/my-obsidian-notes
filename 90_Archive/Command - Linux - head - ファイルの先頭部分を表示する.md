# Command - Linux - head - ファイルの先頭部分を表示する

## 概要

`head` は、指定されたファイル（または標準入力）の最初の数行（デフォルトは10行）を表示します。バイナリファイルかテキストファイルかを判断する際の初動確認や、パイプラインの途中で出力を制限する際によく使われます。

(出自: `coreutils` パッケージに含まれる)

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `tail` | ファイルの「末尾」を表示する。 | ログの最新情報確認 |
| `cat` | ファイルの「全て」を表示する。 | 小さいファイルの全確認 |
| `less / more` | スクロール可能なページャー。 | じっくり中身を読む |
| `sed` | 指定した行番号（例: 5〜10行目）だけを抽出できる。 | 特定の中間行の抽出 |

## よく連携されるコマンド

### シナリオ例: パイプラインの出力を上位10個に絞る (運用・解析視点)

* **目的**: ファイル数が多いディレクトリで、最新のファイルいくつかだけを確認したい。
* **連携**: `ls`, `sort`
* **解説**: 他のコマンドの膨大な結果を、必要な分量だけにカットします。
* **コマンド例**:
    ```bash
    # 更新日時順に並べて、最新の5ファイルだけを表示
    ls -t | head -n 5
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-n <N>`, `--lines=<N>` | 先頭から `N` 行を表示する（数値に `-` を付けると、末尾から N 行を除いた全てを表示）。 |
| ⭐️ `-c <N>`, `--bytes=<N>` | 先頭から `N` バイトを表示する。 |
| `-q`, `--quiet` | 複数ファイルを指定した際、ファイル名のヘッダーを表示しない。 |
| `-v`, `--verbose` | 常にファイル名のヘッダーを表示する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルのヘッダー（コメント）部分の確認。
* **解説**: `/etc` 配下のファイルがどのような役割か、先頭のコメントを数行読んで把握します。
* **例**:
    ```bash
    # sshd_config の先頭 20 行を確認
    head -n 20 /etc/ssh/sshd_config
    ```

### 2. ブルーチーム視点

* **タスク**: ログローテーション直後の不審なアクティビティ確認。
* **解説**: ログファイルの先頭（そのファイルの開始時点）に、異常なサービス起動メッセージやエラーがないか確認します。
* **例**:
    ```bash
    # syslog の開始 50 行をチェック
    sudo head -n 50 /var/log/syslog
    ```

### 3. レッドチーム視点

* **タスク**: データベース出力やパスワードリストのサンプル取得。
* **解説**: 巨大なユーザーリストや設定ファイルを盗み出す前に、フォーマットが正しいか数行だけ抽出して確認（フィンガープリンティング）します。
* **例**:
    ```bash
    # 盗み出した DB ダンプの構造を最初の 100 バイトで確認
    head -c 100 database_dump.sql
    ```

## エラーメッセージとトラブルシューティング

1.  **「行数が指定したより少ない」**:
    * **原因**: ファイル自体の行数が、指定した `-n` の値より少ない。
    * **解決策**: 異常ではありません。ある分だけが表示されます。

2.  **「バイナリが表示されて画面が崩れた」**:
    * **原因**: `head` はバイナリファイルもそのまま出力しようとします。
    * **解決策**: `reset` コマンドでターミナルを復旧し、次は `strings` や `od` を併用してください。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **情報漏洩**: `head` を使って、本来権限のないユーザーが機密ファイルの一部を覗き見ることができてしまう設定（不適切な SUID や sudo 権限）に注意が必要です。

### GTFOBins における利用例

`head` に `sudo` 権限がある場合、本来読み取れないファイル（`/etc/shadow` 等）の先頭を読み取ることが可能です。

* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo head を使って特権ファイルの先頭を読み取る
    sudo head -n 1 /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/head/`

## 注意点・補足

* **負の数指定**: `head -n -5` とすると、「最後の5行を除いた、最初からそこまでの全て」という特殊な抽出が可能です。
* **デフォルト値**: 引数なしの `head` は 10 行表示することを覚えておくと、タイピングがさらに速くなります。
