# Command - Linux - ip netns - ネットワーク名前空間の管理

## 概要

`ip netns` オブジェクトは、ネットワーク名前空間（Network Namespace）の作成、削除、および操作を行います。名前空間を使用すると、OS 内に仮想的な「隔離されたネットワーク環境」を構築できます。例えば、ある名前空間では eth0 を使い、別の名前空間では全く異なるルーティング設定で仮想的な eth1 を使う、といったことが可能です。

## 主要サブコマンド

| サブコマンド | 略称 | 説明 |
| :--- | :--- | :--- |
| **`list`** | `ls`, `l` | 存在する全てのネットワーク名前空間を表示する。 |
| **`add <NAME>`** | `a` | 新しい名前空間 `<NAME>` を作成する。 |
| **`del <NAME>`** | `d` | 指定した名前空間を削除する。 |
| **`exec <NAME> <CMD>`** | `e` | 指定した名前空間の内側でコマンド `<CMD>` を実行する。 |
| **`identify <PID>`** | | 指定したプロセス ID が属している名前空間を表示する。 |
| **`set <NAME> <ID>`** | | 名前空間に ID を割り当てる（主に管理用）。 |

## コマンド例と構文

### 1. 名前空間の作成と確認

まずは「箱」を作る作業です。

```bash
# 'vpn-net' という名前の名前空間を作成
sudo ip netns add vpn-net

# 存在する名前空間を一覧表示
ip netns list
````

### 2. 名前空間内での操作 (exec) ⭐️重要

これが `ip netns` の真骨頂です。名前空間の中に入ってコマンドを打つのと同じ効果があります。

Bash

```
# 'vpn-net' の中でのネットワークインターフェースの状態を確認
sudo ip netns exec vpn-net ip addr

# 'vpn-net' の中で bash を起動し、その空間に居座る
sudo ip netns exec vpn-net bash
# (この中の操作はすべて vpn-net の設定に縛られる)
```

### 3. 名前空間の接続 (veth ペアの利用)

名前空間同士、あるいはホストと名前空間を繋ぐには「仮想的な LAN ケーブル」である `veth` ペアを使います。

Bash

```
# veth ペア (veth0, veth1) を作成
sudo ip link add veth0 type veth peer name veth1

# 片方の端 (veth1) を 'vpn-net' 名前空間に放り込む
sudo ip link set veth1 netns vpn-net

# 名前空間側でインターフェースを有効化し IP を振る
sudo ip netns exec vpn-net ip addr add 10.0.0.1/24 dev veth1
sudo ip netns exec vpn-net ip link set veth1 up
```

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 複雑なルーティングのシミュレーション。
- **解説**: 本番環境に手を加えることなく、複数の名前空間を繋いで仮想ルーターを構築し、BGP や OSPF などの動的ルーティングが意図通り動くかテストできます。

### 2. ブルーチーム視点

- **タスク**: プロセスのネットワーク隔離。
- **解説**: 脆弱な可能性がある古いアプリケーションや、不審なバイナリを実行する際、専用の名前空間に閉じ込めることで、ホスト側のネットワークや他のインターフェースへのアクセスを遮断（サンドボックス化）します。

### 3. レッドチーム視点

- **タスク**: 検知回避のための通信経路秘匿。
- **解説**: 特定の C2 通信（制御用通信）だけを専用の名前空間 + VPN インターフェース経由で流すように設定し、ホスト側のメインログや EDR に通信が残りにくいように細工します。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **Namespace Escape**: 名前空間は隔離されていますが、ルート権限があれば容易に行き来が可能です。カーネルの脆弱性を突かれると、名前空間を越えてホスト側のネットワークを操作（エスケープ）されるリスクがあります。
- **リソースの枯渇**: 無制限に名前空間を作成すると、OS の管理リソース（PID やメモリ）を圧迫し、DoS 状態に陥ることがあります。

### 対応策・緩和策 (ブルーチーム視点)

- **権限管理**: `ip netns` の操作には root 権限が必要です。一般ユーザーへの `sudo` 許可は最小限にします。
- **可視性の確保**: `lsns` コマンドなどを使用して、意図しない名前空間が作成されていないか、不審なプロセスが隔離環境で動いていないかを監視します。

## 注意点・補足

- **ファイルの場所**: 作成された名前空間は `/var/run/netns/` 配下にファイルとして存在します。
- **ループバック**: 新しく作った名前空間の `lo` インターフェースはデフォルトで `DOWN` 状態です。内部で通信を行う場合は `ip link set lo up` を忘れないようにしましょう。
