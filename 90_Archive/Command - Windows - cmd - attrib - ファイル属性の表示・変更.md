# Command - Windows - cmd - attrib - ファイル属性の表示・変更

## 概要

`attrib` (Attribute) コマンドは、ファイルやディレクトリに付与されている属性（読み取り専用、アーカイブ、システムファイル、隠しファイル）を表示または変更するためのコマンドです。

GUIの「プロパティ」から変更できるものと同等ですが、コマンドラインからは「システム属性 (S)」などのより強力な制限を操作できます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `Get-ItemProperty` (PS) | PowerShellでファイルオブジェクトの属性を確認する。 | オブジェクト操作ならPS |
| `Set-ItemProperty` (PS) | PowerShellで属性を変更する。`-Attributes` パラメータを使用する。 | - |

## よく連携されるコマンド

### シナリオ例: 隠し属性・システム属性を持つファイルの全検索 (ブルーチーム視点)

* **目的**: 攻撃者が `attrib +h +s` で隠したバックドアやツールをディレクトリ内から見つけ出す。
* **連携コマンド**: `attrib`, `findstr`
* **解説**: `attrib` は隠しファイルもデフォルトで表示するため、通常の `dir` コマンド（オプションなし）で見えないファイルを列挙するのに適しています。
* **コマンド例**:
    ```cmd
    REM カレントディレクトリ以下の全ファイル属性を表示し、隠しファイル(H)またはシステムファイル(S)を抽出
    attrib /s | findstr "H S"
    ```

## スイッチ/オプション説明

属性を設定するには `+` を、解除するには `-` を頭に付けます。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **属性指定** | |
| `+R` / `-R` | 読み取り専用 (Read-only) 属性を設定/解除。 |
| `+A` / `-A` | アーカイブ (Archive) 属性を設定/解除。 |
| `+S` / `-S` | システムファイル (System) 属性を設定/解除。 |
| `+H` / `-H` | 隠しファイル (Hidden) 属性を設定/解除。 |
| `+O` / `-O` | オフライン (Offline) 属性を設定/解除。 |
| `+I` / `-I` | コンテンツ インデックス無効 (Not content indexed) 属性を設定/解除。 |
| `+X` / `-X` | スクラビングなし (No scrub) 属性を設定/解除。 |
| `+V` / `-V` | 整合性 (Integrity) 属性を設定/解除。 |
| **動作制御** | |
| `/S` | 現在のフォルダーとすべてのサブフォルダーにある一致するファイルを処理する。 |
| `/D` | フォルダーも処理対象に含める。 |
| `/L` | シンボリック リンク自体に属性を適用する（リンク先ではなく）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 誤操作による削除を防ぐため、重要な設定ファイルを読み取り専用にする。
* **組み合わせ**: `attrib +R config.ini`
* **解説**: 管理者であっても、誤って上書き保存や削除をすることを防ぐためのガードレールとして機能します。
* **例**:
    ```cmd
    REM 重要な構成ファイルを読み取り専用に設定
    attrib +R C:\App\config\server.conf
    ```

### 2. ブルーチーム視点

* **タスク**: 感染疑いのある端末で、不自然に隠された実行ファイルがないか調査。
* **組み合わせ**: `attrib /S /D`
* **解説**: マルウェアは自身を `Hidden` だけでなく `System` 属性に設定することで、エクスプローラーのデフォルト設定（保護されたオペレーティングシステムファイルを表示しない）を掻い潜ろうとします。
* **例**:
    ```cmd
    REM 公開ディレクトリ配下で隠されているものをすべてリストアップ
    attrib C:\Users\Public\*.* /S
    ```

### 3. レッドチーム視点

* **タスク**: アップロードしたツールやバックドアを、ユーザーや管理者の目から隠す（Defense Evasion）。
* **組み合わせ**: `attrib +H +S +R filename.exe`
* **解説**: 隠し属性(`H`)とシステム属性(`S`)を組み合わせることで、通常の「隠しファイルを表示」設定だけでは見えないようにします。
* **例**:
    ```cmd
    REM バックドアをシステムファイルとして隠蔽し、変更も禁止する
    attrib +h +s +r C:\Windows\Temp\debug.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `Access denied - C:\path\to\file`
    * **考えられる原因**: ファイルが他のプロセスで使用中であるか、現在のユーザーに変更権限がない。または、システム属性が付与されているファイルを変更しようとしている。
    * **解決策**: 管理者権限で実行する。システム属性を変更する場合は、`-s` を含めて実行する。

2.  **エラーメッセージ例 2**: `Not resetting system file - C:\path\to\file`
    * **考えられる原因**: システムファイル属性(`S`)がついているファイルに対して、他の属性（`H`など）だけを解除しようとした。
    * **解決策**: システム属性を持つファイルは、システム属性そのものを同時に解除（`-s`）しないと他の属性も変更できない仕様があります。 `attrib -h -s filename` のように同時に指定してください。

## 環境変数と設定ファイル

`attrib` 自体に特有の環境変数はありませんが、操作対象となるファイルの属性情報は NTFS ファイルシステムのメタデータとして保存されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: **Defense Evasion (防御回避)**。攻撃者が C:\Users\Public や C:\Windows\Temp に置いたペイロードを隠すために必ずと言っていいほど使用されます。また、ランサムウェアがバックアップファイル（VSS等）を削除する前に、属性を操作してアクセス可能にする前段階として使われることもあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: なし（`attrib.exe` 自体は実行コードのダウンロードやプロセスの生成を行わないため、厳密にはバイナリ単体での悪用機能は限定的です。しかし、攻撃チェーンの中での「隠蔽」フェーズにおいて標準ツールとして利用されます。）
* **参照**: [LOLBAS - attrib.exe](https://lolbas-project.github.io/lolbas/Binaries/Attrib/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイルシステムの整合性監視 (FIM) を導入し、重要なディレクトリでの属性変更を検知する。
* **Detection (検知)**: EDRやコマンドラインロギング（Event ID 4688）で、`+h +s` のような特定の組み合わせが使われていないか、特に `Temp` フォルダ等の書き込み可能ディレクトリで監視する。

## 注意点・補足

* **隠しファイルの限界**: `attrib` で隠したとしても、`dir /a` コマンド（全属性表示）を使えば簡単に見つけられてしまいます。あくまで「カジュアルな調査」や「エクスプローラーのデフォルト表示」を欺くための手法であることを理解しておく必要があります。
