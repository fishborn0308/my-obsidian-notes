# Command - Linux - pip - Python パッケージのインストールと管理

## 概要

`pip` は、Python の標準的なパッケージ管理システムです。Python Package Index (PyPI) に登録されているソフトウェアパッケージをインストール・管理するために使用されます。依存関係（そのソフトを動かすために必要な他のソフト）を自動的に解決してインストールしてくれるのが最大の特徴です。

## 🛠 システム管理ツールとの違い

| コマンド | 管理対象 | 範囲 |
| :--- | :--- | :--- |
| `apt` / `pacman` | OS 全体のソフト・ライブラリ | システム全体 (`/usr/bin` 等) |
| **`pip`** | Python 専用のライブラリ | Python の環境内 (`site-packages`) |

## 主要なオプション（基本呪文）

現在では `python3 -m pip` の形式で実行することが推奨されています（環境の混同を防ぐため）。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `install <pkg>` | 新しいパッケージをインストールする。 |
| ⭐️ `uninstall <pkg>` | パッケージを削除する。 |
| ⭐️ `list` | インストール済みのパッケージを一覧表示する。 |
| ⭐️ `show <pkg>` | 特定のパッケージのバージョンや場所を表示する。 |
| ⭐️ `freeze` | インストール済みパッケージを `requirements.txt` 形式で出力する。 |
| ⭐️ `-r <file>` | 設定ファイル（requirements.txt等）から一括インストールする。 |

---

## 💡 モダンな pip 運用の三則

1.  **システム領域を汚さない**: `sudo pip` は避けるべきです。OS 自身が使っている Python ライブラリと衝突し、最悪の場合システムが壊れます。
2.  **仮想環境 (`venv`) の活用**: プロジェクトごとに隔離された箱（環境）を作り、その中で `pip` を使います。
3.  **信頼できるソースのみ**: PyPI 以外の野良リポジトリからのインストールは慎重に行います。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: プロジェクトの環境再現。
* **解説**: 開発環境と同じライブラリ構成をサーバー上に構築します。
* **例**:
    ```bash
    # 別の場所で作成した requirements.txt に基づいて環境を完全再現
    python3 -m pip install -r requirements.txt
    ```

### 2. ブルーチーム視点

* **タスク**: 脆弱なライブラリの特定と監査。
* **解説**: 利用しているライブラリに既知の脆弱性（CVE）が含まれていないかチェックします。
* **ツール**: `pip-audit`（既知の脆弱性をスキャンするツール）。
* **例**:
    ```bash
    # プロジェクト内の脆弱性をスキャン
    pip-audit
    ```

### 3. レッドチーム視点

* **タスク**: 難読化ツールや偵察ツールの導入。
* **解説**: 侵入後、データの解析や内部ネットワークの探索に必要な高度なライブラリを即座に展開します。また、悪意あるパッケージを PyPI にアップロードし、名前の打ち間違い（タイポスクワッティング）を狙って開発者に踏ませる攻撃手法の調査も行います。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **依存性の混乱 (Dependency Confusion)**: 社内用パッケージと同じ名前を PyPI に公開することで、誤って外部の悪意あるコードをダウンロードさせる攻撃。
* **タイポスクワッティング**: `requests` ではなく `requesst` のような似た名前の偽パッケージをインストールしてしまうミス。

### 対応策・緩和策 (ブルーチーム視点)

* **ハッシュ値の確認**: `requirements.txt` に各パッケージのハッシュ値を記載し、改ざんされたファイルがダウンロードされないよう制限します。
* **プライベートリポジトリ**: 重要なプロジェクトでは、自社専用のパッケージリポジトリ（Artifactory 等）を介して安全なものだけを配布します。
