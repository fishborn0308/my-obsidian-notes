# Command - Linux - rmdir - 空のディレクトリを削除する

## 概要

`rmdir` (remove directory) は、**中身が空のディレクトリ**のみを削除するためのコマンドです。ディレクトリ内にファイルやサブディレクトリが1つでも存在する場合、`rmdir` は安全のためにエラーを出して失敗します。

この挙動のため、`rm -r` のように意図せず重要なファイルを削除してしまう危険性がなく、ディレクトリを安全に削除したい場合に利用されます。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`rm -r`** | ディレクトリとその**中身を全て再帰的に**削除します。非常に強力ですが危険も伴います。`rmdir` は**空のディレクトリしか削除できない**ため、より安全です。 | `rm -r` (中身ごと削除) |
| **`mkdir`** | `rmdir` の逆の操作。新しいディレクトリを**作成**します。 | - |

## よく連携されるコマンド

`rmdir` は、クリーンアップスクリプトなどで、他のコマンドでファイルを削除した後に残った空のディレクトリを片付ける際に使われます。

### シナリオ例: ネストした空ディレクトリの一括削除 (インフラ構築・運用視点)

* **目的**: 中身を削除した結果、`a/b/c` のように複数階層にわたって空になったディレクトリを一括で削除する。
* **連携コマンド**: `rmdir` (`-p` オプション)
* **解説**: `-p` (`--parents`) オプションを使うと、指定したディレクトリを削除した後、その親ディレクトリも空であれば、それも再帰的に削除します。
* **コマンド例**:

		```bash
		# a/b/c ディレクトリを削除し、もしbとaも空になればそれらも削除する

		rmdir -p a/b/c

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ディレクトリ操作** | |
| ⭐ `<directory_name>` | 削除する空のディレクトリの名前を指定します。 |
| `-p`, `--parents` | 指定したディレクトリを削除した後、その親ディレクトリも空であれば、それも再帰的に削除します。 |
| **出力制御** | |
| `-v`, `--verbose` | 処理した各ディレクトリに関するメッセージを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 中身が空であることを確認した上で、不要になったディレクトリを安全に削除する。
* **組み合わせ**: `rmdir <directory_name>`
* **解説**: プロジェクトのファイルを別の場所に移した後、残った空のディレクトリを片付ける際に使用します。
* **例**:

		```bash
		# 'old_project' ディレクトリが空であることを確認してから削除

		ls -A old_project/  # -Aで隠しファイルもないことを確認

		rmdir old_project/

		```

## 2. ブルーチーム視点

* **タスク**: 攻撃者の残した空のディレクトリをクリーンアップする。
* **組み合わせ**: `rmdir <attacker_dir>`
* **解説**: インシデント対応のクリーンアップフェーズで使用します。攻撃者が作成したディレクトリから、悪意のあるファイルを `rm` ですべて削除した後、残った空のディレクトリを `rmdir` で安全に消去します。
* **例**:

		```bash
		# 攻撃者が作成した /tmp/.x ディレクトリからペイロードを削除後、ディレクトリ自体を削除

		rmdir /tmp/.x

		```

## 3. レッドチーム視点

* **タスク**: -
* **組み合わせ**: -
* **解説**: 攻撃者が `rmdir` を積極的に使用することは稀です。攻撃者は通常、痕跡を消すために `rm -rf` のような、より強力で確実なコマンドを好むためです。

# エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `rmdir: failed to remove '<dir_name>': Directory not empty`
		* **考えられる原因**: **最もよくあるエラー**。削除しようとしたディレクトリが空ではありません。`.` で始まる隠しファイル（例: `.gitkeep`）が存在する場合もこのエラーになります。
		* **解決策**: `ls -A <dir_name>` を実行して、隠しファイルを含めてディレクトリの中身を確認し、先にファイルを全て削除してください。中身ごと全て削除して良いことが確実な場合は、`rm -r <dir_name>` を使います。

# 環境変数と設定ファイル

* `rmdir` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

# セキュリティに関する考慮事項

## 脆弱性と悪用事例

* **脆弱性**: `rmdir` は、空でないディレクトリを削除できない仕様上、Linuxのファイル操作コマンドの中では非常に安全な部類に入り、直接的な脆弱性はほとんどありません。
* **悪用シナリオ**: スクリプトが `rmdir` の成功を前提としている場合、攻撃者がそのディレクトリにファイルを1つ置くだけで `rmdir` は失敗するため、スクリプトの動作を妨害する簡易的なサービス妨害（DoS）に利用される可能性は理論的には考えられます。

# 注意点・補足

* **安全な削除の選択肢**: ディレクトリを削除する際、その中身が本当に不要かどうかが100%確信できない場合は、まず `mv old_dir old_dir.bak` のようにリネームし、しばらく様子を見てから削除するという手順を踏むとより安全です。
