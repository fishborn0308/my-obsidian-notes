# Command - git -git branch - ブランチの作成、一覧表示、削除

## 概要

`git branch` コマンドは、Gitの**ブランチ**を管理するための中心的なコマンドです。ブランチは、プロジェクトのメイン開発ライン(通常は`main`または`master`ブランチ)から分岐し、個別の機能開発やバグ修正などを隔離された環境で行うことを可能にします。

`git branch`は、ブランチの作成、既存のブランチの一覧表示、ブランチの削除といった基本的な操作を提供します。

(出自: `git` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`git switch`** | **ブランチの切り替えに特化**したモダンなコマンド。`git branch <name>` はブランチを**作成**するだけですが、`git switch -c <name>` は**作成と切り替え**を同時に行います。 | `git switch` (切り替え) |
| **`git checkout`** | 旧来のコマンドで、ブランチの切り替えとファイルの復元の両方の機能を持っていました。ブランチ切り替えには `git switch` の利用が推奨されます。 | `git switch` |
| **`git merge`**| 異なるブランチで行われた変更を1つのブランチに統合します。`git branch`はブランチ自体を管理するのに対し、`git merge`はブランチ間の変更を結合します。 | - |

## よく連携されるコマンド

`git branch` は、ブランチを作成した後に、その新しいブランチに移動する `git switch` と連携するのが最も基本的なワークフローです。

### シナリオ例: 新しい機能ブランチの作成と作業開始 (インフラ構築・運用視点)

* **目的**: 新しい機能開発のためにブランチを作成し、そのブランチに切り替えて作業を開始する。
* **連携コマンド**: `git switch`
* **解説**: まず`git branch`で新しいブランチを作成し、次に`git switch`でそのブランチに切り替えます。
* **コマンド例**:

		```bash
    # feature/new-dashboard という新しいブランチを作成

    git branch feature/new-dashboard

    # 作成したブランチに切り替える

    git switch feature/new-dashboard

    ```



## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ブランチ操作** | |
| ⭐ `[<branch-name>]`| 新しくブランチを作成する場合、そのブランチ名を指定します。引数なしの場合は、ローカルブランチを一覧表示します。 |
| ⭐ `-d`, `--delete`| 指定したブランチを**安全に削除**します。マージされていないコミットを含んでいる場合はエラーになります。 |
| `-D`| 指定したブランチを**強制的に削除**します。マージされていない変更も破棄されるため注意が必要です。 |
| `-m`, `--move`| ブランチ名を変更します。`git branch -m <old> <new>` |
| **表示制御** | |
| ⭐ `-a`, `--all`| 全てのローカルブランチとリモート追跡ブランチを一覧表示します。 |
| `-r`, `--remotes`| リモート追跡ブランチのみを一覧表示します。 |
| `--merged` / `--no-merged` | 現在のブランチにマージ済み / 未マージのブランチを一覧表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: マージ済みで不要になったローカルブランチをクリーンアップする。
* **組み合わせ**: `git branch --merged | grep -v "\*" | xargs -n 1 git branch -d`
* **解説**: マージ済みのブランチをリストアップし、現在のブランチを除外して、`xargs` を通じて一括で安全に削除します。
* **例**:

		```bash
    # マージ済みの feature/old-feature ブランチを安全に削除

    git branch -d feature/old-feature

    ```



## 2. ブルーチーム視点

* **タスク**: リポジトリの健全性を確認するため、全てのブランチの状況を詳細に表示する。
* **組み合わせ**: `git branch -a -v`
* **解説**: 全てのブランチの最新コミットのハッシュ値とメッセージを表示し、不審なブランチや長期未マージのブランチがないかを確認します。
* **例**:

		```bash
    # 全てのローカル・リモートブランチの情報を詳細に表示

    git branch -a -v

    ```



## 3. レッドチーム視点

* **タスク**: 偵察段階で、過去の特定のコミットの状態を確認するための調査用ブランチを作成する。
* **組み合わせ**: `git branch <new-branch-name> <commit-id>`
* **解説**: ターゲットリポジトリの特定の過去のコミットに機密情報が誤って含まれていた可能性がある場合、そのコミットから新しい調査用ブランチを作成し、元のブランチに影響を与えずに分析します。
* **例**:

		```bash
    # 特定のコミットID (例: a1b2c3d4) から historical-review という新しいブランチを作成

    git branch historical-review a1b2c3d4

    ```



## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](../linux/troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `fatal: A branch named 'existing-branch' already exists.`
		* **考えられる原因**: 指定した名前のブランチがすでに存在します。
		* **解決策**: 別のブランチ名を指定してください。

2. **エラーメッセージ例 2**: `error: The branch 'feature/my-feature' is not fully merged.`
		* **考えられる原因**: `git branch -d`でブランチを削除しようとしましたが、そのブランチにまだマージされていないコミットが含まれています。
		* **解決策**: 変更が不要で、データ損失を許容する場合は、`git branch -D feature/my-feature`を使用して強制的に削除します。

## 環境変数と設定ファイル

* `git branch`に直接影響を与える独自の環境変数はありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](../linux/environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `git branch` 自体ではなく、**不適切なブランチ管理**がリスクとなります。
* **悪用シナリオ**: 攻撃者がリポジトリへの書き込み権限を得た場合、正規のブランチから派生したブランチに悪意のあるコードをコミットし、そのブランチを巧妙に隠蔽（例: 似たような名前）することで、検出を遅らせようとする可能性があります。

### GTFOBins / LOLBAS における利用例

`git branch` は直接的なコード実行機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `main` などの主要なブランチに**ブランチ保護ルール**を設定し、レビューなしでのマージや強制プッシュを禁止する。
* **Detection (検知)**: Gitホスティングサービスの監査ログを監視し、不審なブランチの作成や削除がないかを確認する。

## 注意点・補足

* **リモートブランチとの同期**: ローカルで作成したブランチは、`git push -u origin <branch-name>` を実行するまでリモートリポジトリには存在しません。
