# Command  - Linux - head - ファイルの先頭部分を表示する

## 概要

`head` は、ファイルや標準入力の**先頭部分**を表示するためのコマンドです。デフォルトでは、先頭の10行を表示します。巨大なログファイルやCSVファイルの内容を、全体を読み込むことなく、ヘッダー部分や最初の数行だけを素早く確認したい場合に非常に便利です。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`tail`** | `head` の**対になるコマンド**で、ファイルの**末尾部分**を表示します。 | - |
| **`cat`** | ファイルの**全体**を一度に表示します。大きなファイルには `head` や `less` を使うのが適切です。 | - |
| **`sed`** | `sed -n '1,5p' <file>` のように、行番号を指定してファイルの特定の部分を抜き出すことができます。`head` は先頭部分の表示に特化した、よりシンプルなコマンドです。 | `head` (先頭表示) |

## よく連携されるコマンド

`head` は、他のコマンドの大量の出力をフィルタリングするために、パイプと連携して使われることが非常に多いです。

### シナリオ例: コマンド出力の絞り込み (インフラ構築・運用視点)

* **目的**: `ps aux` のように大量の行を出力するコマンドの結果を、上位の数行だけ見て概要を把握する。
* **連携コマンド**: `ps`, `sort`, パイプ (`|`)
* **解説**: 他のコマンドの出力をパイプで `head` に渡すことで、画面が大量のテキストで埋め尽くされるのを防ぎ、必要な部分だけを効率的に確認できます。
* **コマンド例**:

```bash
# システムで最もCPUを消費しているプロセスの上位5件を表示する
ps aux --sort=-%cpu | head -n 6
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `-n`, `--lines=<N>` | 表示する行数を `N` に指定します。`head -20` のように `-` の後に直接数字を指定するショートカットも一般的です。 |
| `-c`, `--bytes=<N>` | 行単位ではなく、先頭から `N` **バイト**分を表示します。 |
| **ヘッダー表示** | |
| `-q`, `--quiet` | 複数のファイルを指定した場合に、ファイル名のヘッダーを表示しません。 |
| `-v`, `--verbose` | 常にファイル名のヘッダーを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: CSVファイルのヘッダー行を確認して、カラム構成を把握する。
* **組み合わせ**: `head -n 1 <file.csv>`
* **解説**: 巨大なCSVファイルを扱う際に、まずヘッダー行だけを確認し、どのようなデータが含まれているかを把握する基本的な操作です。
* **例**:

```bash
# access.log.csv のヘッダー行（1行目）のみを表示
head -n 1 access.log.csv
```

### 2. ブルーチーム視点

* **タスク**: 不審なバイナリファイルのファイルヘッダー（マジックナンバー）を確認する。
* **組み合わせ**: `head -c <bytes> <suspicious_binary> | xxd`
* **解説**: `-c` オプションでファイルの先頭数バイトを抽出し、`xxd` で16進数表示します。これにより、ファイルの `file` コマンドの結果を偽装しようとするマルウェアなどに対して、実際のファイルシグネチャを確認できます。（例: `7f 45 4c 46` はELF実行ファイル）
* **例**:

```bash
# invoice.pdf というファイルの先頭4バイトを確認
head -c 4 invoice.pdf | xxd
```

### 3. レッドチーム視点

* **タスク**: 偵察の一環として、重要な設定ファイルの冒頭部分を素早く確認する。
* **組み合わせ**: `head -n 5 /etc/passwd`
* **解説**: 攻撃者はシステムに侵入後、`cat` でファイル全体を表示してログに大きな痕跡を残すのを避けつつ、`head` を使ってファイルの内容を断片的に調査します。`/etc/passwd` の先頭を見ることで `root` やシステムアカウントの存在を確認したり、ソースコードファイルの先頭を見てコメントから情報を得たりします。
* **例**:

```bash
# /etc/passwd の最初の5行を表示
head -n 5 /etc/passwd
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。
* `head` は非常にシンプルなコマンドであり、`No such file or directory` 以外の特有のエラーはほとんど発生しません。

## 環境変数と設定ファイル

* `head` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `head` 自体は読み取り専用のツールであり、脆弱性を持つ可能性は低いです。
* **悪用シナリオ**: **情報漏洩**。設定ファイルやスクリプトのヘッダーコメント部分に、誤ってパスワードやAPIキーなどの機密情報が記述されている場合があります。`head` を実行することで、これらの情報が意図せず画面に表示されてしまう可能性があります。

### GTFOBins / LOLBAS における利用例

`head` はGTFOBinsにリストされています。

* **機能**: **File Read**
* **解説**: もし `head` にSUIDビットが設定されている（重大な設定ミス）、またはユーザーが `sudo head` を実行できる権限を持っている場合、これを利用して本来読み取れないはずのファイル（例: `/etc/shadow`）の先頭部分を閲覧できます。
* **参照**: `https://gtfobins.github.io/gtfobins/head/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイルの先頭部分に機密情報を記述しないようにする。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `auditd` を使い、機密ファイルへの `head` によるアクセスを監視する。

## 注意点・補足

* **`tail`との組み合わせ**: `head` と `tail` をパイプで組み合わせることで、ファイルの特定の中間部分を抜き出すことができます。

```bash
# ファイルの11行目から20行目までを抽出する
head -n 20 some_file.txt | tail -n 10
```
