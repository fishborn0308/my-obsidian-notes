# Command - Linux - fg - バックグラウンドジョブをフォアグラウンド（最前面）に移動する

## 概要

`fg` は、現在バックグラウンドで実行中、または一時停止中（Suspended）のジョブを、ユーザーが直接操作できるフォアグラウンドに呼び戻します。複数のタスクを 1つのターミナルで切り替えながら作業する「ジョブコントロール」機能の核心です。

(出自: シェルの組み込みコマンド（Built-in）)

## 類似・関連コマンド

| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `bg` | 停止中のジョブを**バックグラウンドのまま**実行再開させる。 | 裏で動かし続けたい時 |
| `jobs` | 現在のシェルセッションで管理されているジョブの一覧と番号を表示する。 | ジョブ番号の確認 |
| `Ctrl+Z` | フォアグラウンドのジョブを一時停止し、バックグラウンドへ送る（シグナル TSTP）。 | 作業の一時中断 |
| `screen / tmux` | セッション全体を維持。ターミナルを閉じてもプロセスを維持できる。 | 長時間の作業維持 |

## よく連携されるコマンド

### シナリオ例: エディタとコマンド実行の高速切り替え (運用・開発視点)

* **目的**: `vim` で設定ファイルを編集しながら、その場で反映テストを行い、また編集に戻る。
* **手順**:
    1. `vim config.conf` を開く。
    2. `Ctrl+Z` を押して一時停止（シェルに戻る）。
    3. `./test_config.sh` を実行して確認。
    4. `fg` を打って `vim` に戻る。
* **コマンド例**:
    ```bash
    # 中断した vim を最前面に戻す
    fg
    ```

## 引数と指定方法

`fg` は通常、`jobs` コマンドで確認できる「ジョブ番号」を引数に取ります。

| 指定形式 | 説明 |
| :--- | :--- |
| ⭐️ `fg` | 直近に操作（停止またはバックグラウンド化）したジョブ（`+` が付いているもの）を戻す。 |
| ⭐️ `fg %<n>` | 指定したジョブ番号 `n` のジョブを戻す（例: `fg %1`）。 |
| `fg %<string>` | コマンド名が `string` で始まるジョブを戻す。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 誤って中断してしまった重要プロセスの復帰。
* **解説**: 大規模なファイルコピーやバックアップの最中に、無意識に `Ctrl+Z` を押してしまった場合、`fg` で即座に再開させないと、プロセスが停止したまま時間が経過してしまいます。
* **例**:
    ```bash
    # 停止中のコピー処理をフォアグラウンドに戻して続行
    fg %1
    ```

### 2. ブルーチーム視点

* **タスク**: ユーザーの操作履歴からの不審なジョブの特定。
* **解説**: 攻撃者がフォアグラウンドでリバースシェルなどを維持している場合、`jobs` でそれを確認し、`fg` で呼び出すことで、今まさに何が実行されているかを直接確認できる場合があります。

### 3. レッドチーム視点

* **タスク**: インタラクティブなツールの隠蔽と再開。
* **解説**: 侵入したサーバー上でツール（例: `nmap`, `python3` 等）を動かす際、管理者がログインしてきたら `Ctrl+Z` で一旦隠し、去った後に `fg` で作業を再開するといった「隠密操作」に使われます。

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `bash: fg: no such job`
    * **原因**: 指定した番号のジョブが存在しない、あるいは既に終了している。
    * **解決策**: `jobs` コマンドを打って、現在生きているジョブのリストを再確認する。

2.  **エラーメッセージ例 2**: `bash: fg: job has terminated`
    * **原因**: 呼び戻そうとした瞬間にプロセスが終了した。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **情報の露出**: `fg` でジョブを戻すと、エディタに表示されていた機密情報や、パスワード入力プロンプトが画面に現れます。共有端末で不用意に `fg` を行うと、第三者に情報を覗き見られるリスクがあります。

### 緩和策 (ブルーチーム視点)

* **Detection (検知)**: 長時間「停止状態（Stopped）」のまま放置されている不審なプロセスがないか、`ps aux` 等で定期的に確認する（STAT 列が `T` のもの）。

## 注意点・補足

* **シェルの子プロセス**: `fg` は現在のシェルセッションに紐付いています。別のターミナルタブや、一度ログアウトした後の再ログイン環境では、元のジョブを `fg` で呼び出すことはできません（その場合は `screen` や `tmux` が必要です）。
* **標準入出力**: `fg` で戻したジョブは再び標準入力を奪うため、キーボード操作がそのプロセスに送られるようになります。
