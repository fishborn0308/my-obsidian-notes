# Command  - Linux - brctl - Linuxイーサネットブリッジを管理する (旧)

## 概要

`brctl` (bridge control) は、Linuxカーネル内でイーサネットブリッジを作成、管理、監視するためのコマンドラインユーティリティです。ソフトウェアブリッジは、物理的なネットワークスイッチと同様に動作する仮想的なネットワークデバイスであり、複数のネットワークインターフェイス（物理NIC、TAPデバイスなど）を1つの論理的なネットワークセグメントに接続します。

**注**: `brctl` は `bridge-utils` パッケージに含まれるレガシーなコマンドです。現在では `iproute2` パッケージに含まれる `ip link add type bridge` などのコマンドを使うことが推奨されます。

(出自: `bridge-utils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`ip link` (iproute2)** | `ip` コマンドは、ブリッジを含む全てのネットワークインターフェイスを管理するためのモダンな標準ツールです。`brctl` の全ての機能は `ip link` や `bridge` サブコマンドで実現可能です。 | `ip link` |
| **`ovs-vsctl`** | Open vSwitchという、VLANやVXLANなど、より高機能な仮想スイッチを管理します。単純なVM接続はLinuxブリッジ、複雑な仮想ネットワークはOpen vSwitchと使い分けます。 | - |

## よく連携されるコマンド

ブリッジの作成から有効化までは、複数のコマンドを連携させる一連のフローになります。

### シナリオ例: 仮想マシン接続用のブリッジ `br0` を作成する (インフラ構築・運用視点)

* **目的**: 物理NIC `eth0` と仮想マシン用のTAPインターフェイス `tap0` を接続するブリッジ `br0` を作成し、ネットワーク通信を可能にする。
* **連携コマンド**: `ip`
* **解説**: `brctl addbr` でブリッジデバイスを作成し、`brctl addif` でインターフェイスを接続します。その後、`ip` コマンドを使ってIPアドレスの割り当てや、インターフェイスの有効化を行います。
* **コマンド例**:

```bash
# このコマンドシーケンスで基本的なブリッジを構築する

sudo brctl addbr br0

sudo ip addr flush dev eth0

sudo brctl addif br0 eth0

sudo ip link set br0 up

```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ブリッジ操作** | |
| ⭐ `addbr <bridge>` | `<bridge>` という名前のブリッジを新しく作成します。 |
| `delbr <bridge>` | 指定したブリッジを削除します。 |
| **ポート操作** | |
| ⭐ `addif <bridge> <dev>` | `<bridge>` にネットワークデバイス `<dev>` をポートとして追加（接続）します。 |
| `delif <bridge> <dev>` | `<bridge>` からデバイス `<dev>` を切り離します。 |
| **情報表示** | |
| ⭐ `show` | システム上の全てのブリッジとその接続ポートの一覧を表示します。 |
| `showmacs <bridge>` | `<bridge>` が学習したMACアドレスのテーブルを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: KVMで仮想マシンを作成し、物理ネットワーク上の他のマシンと同じように通信させる。
* **組み合わせ**: `brctl addbr br0`, `brctl addif br0 eth0`
* **解説**: 物理NIC `eth0` を含むブリッジ `br0` を作成します。その後、KVMの起動設定で、仮想マシンのネットワークインターフェイスが `br0` に接続されるように指定します。これにより、仮想マシンはブリッジを通じて物理ネットワークに直接アクセスできるようになります。
* **例**:

```bash
# ブリッジbr0を作成し、eth0を追加する

sudo brctl addbr br0

sudo brctl addif br0 eth0

```

### 2. ブルーチーム視点

* **タスク**: 仮想化基盤上で、不審な通信を行っている仮想マシンを特定する。
* **組み合わせ**: `brctl showmacs <bridge>`
* **解説**: 物理スイッチのポートから不審なMACアドレスを持つ通信が検出された場合、`brctl showmacs` でブリッジのMACアドレステーブルを確認します。`port no` 列から、そのMACアドレスがどの仮想インターフェイス（例: `vnet0`）に紐付いているかを特定し、問題の仮想マシンを割り出すことができます。
* **例**:

```bash
# br0ブリッジのMACアドレステーブルを確認

sudo brctl showmacs br0

```

### 3. レッドチーム視点

* **タスク**: 侵入したサーバが仮想化ホストであった場合に、ネットワーク構成を把握する。
* **組み合わせ**: `brctl show`
* **解説**: 侵入後の偵察の一環として `brctl show` を実行し、ブリッジの構成や接続インターフェイス名を把握します。これにより、ホストの役割や、同じネットワークセグメントに存在する他の仮想マシンを推測し、ラテラルムーブメント（横展開）の足がかりを探します。
* **例**:

```bash
# 存在するブリッジと接続インターフェイスをリストアップ

brctl show

```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `device eth0 is already a member of a bridge; can't enslave it to bridge br0`
		* **考えられる原因**: インターフェイス `eth0` が、すでに別のブリッジに接続されています。
		* **解決策**: `brctl show` で現在のブリッジ構成を確認し、もし意図しないブリッジに接続されている場合は `brctl delif <old_bridge> eth0` で切り離してから、新しいブリッジに追加します。

## 環境変数と設定ファイル

* `brctl` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **Debian/Ubuntu**: `/etc/network/interfaces`
* **RHEL/CentOS**: `/etc/sysconfig/network-scripts/ifcfg-<bridge>`
* **解説**: `brctl` コマンドで設定した内容は再起動で失われます。設定を永続化させるには、これらのディストリビューション固有のネットワーク設定ファイルにブリッジの定義を記述する必要があります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `brctl` 自体ではなく、ブリッジの仕組みが悪用される可能性があります。
* **悪用シナリオ**: 攻撃者がroot権限を奪取した場合、`brctl` を使って既存のブリッジに悪意のあるインターフェイスを追加し、ブリッジを通過するVM間通信などを盗聴（スニッフィング）する仮想的な中間者攻撃を仕掛けることが可能です。

### GTFOBins / LOLBAS における利用例

`brctl` はネットワーク構成ツールであり、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。しかし、侵入後の**情報収集**や、**通信盗聴**のための環境構築に悪用される可能性があります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ホストOSへのrootアクセスを厳格に管理する。
* **Detection (検知)**: `auditd` を使って、`ioctl` システムコールによるブリッジへのインターフェイス追加・削除などのネットワーク構成変更を監視する。`brctl show` の結果を定期的に取得し、ベースラインとの差分を確認する。

## 注意点・補足

* **IPアドレスの割り当て先**: ブリッジに物理インターフェイスを追加した場合、IPアドレスは物理インターフェイス (`eth0`) から削除し、ブリッジインターフェイス (`br0`) に割り当てるのが基本です。
