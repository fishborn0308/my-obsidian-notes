# Command  - Linux - useradd - 新しいユーザーアカウントを作成する

## 概要

`useradd` は、システムに新しいユーザーアカウントを作成するための基本的なコマンドです。このコマンドを実行すると、`/etc/passwd`, `/etc/shadow`, `/etc/group` といったユーザー管理ファイルに必要なエントリが追加され、オプションに応じてユーザーのホームディレクトリの作成などが行われます。

新しいユーザーや、サービスを実行するための専用のシステムアカウントを作成する際に使用されます。このコマンドの実行には `root` 権限が必要です。

(出自: `shadow-utils` または `passwd` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`adduser`** | Debian/Ubuntu系で利用できる、`useradd` の**対話的なフロントエンド**。パスワードやフルネームなどを対話形式で尋ねてきます。手動でユーザーを作成する場合はこちらが推奨されます。 | `adduser` (対話的作成) |
| **`usermod`** | **既存の**ユーザーアカウントの属性を変更します。 | - |
| **`userdel`** | ユーザーアカウントを**削除**します。 | - |
| **`passwd`** | ユーザーの**パスワードを設定・変更**します。`useradd` でアカウントを作成した後、最初にパスワードを設定するために使います。 | - |

## よく連携されるコマンド

`useradd` は、アカウント作成後に `passwd` コマンドと連携して、新しいユーザーのパスワードを設定するのが一連の必須ワークフローです。

### シナリオ例: 新規ユーザーの作成とパスワード設定 (インフラ構築・運用視点)

* **目的**: 新しい開発者 `newdev` のアカウントを作成し、初期パスワードを設定する。
* **連携コマンド**: `passwd`
* **解説**: `useradd` でアカウントの骨子を作成しただけでは、パスワードが設定されておらずログインできません。必ず `passwd` を実行して、パスワードを設定する必要があります。
* **コマンド例**:

		```bash
		# このコマンドシーケンスで新しいユーザーアカウントを払い出す
		# 1. 'newdev' ユーザーを作成し、ホームディレクトリも作成、sudoグループに追加

		sudo useradd -m -s /bin/bash -G sudo newdev

		# 2. 作成したユーザーのパスワードを設定 (対話形式)

		sudo passwd newdev

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本設定** | |
| ⭐ `-m`, `--create-home` | ユーザーの**ホームディレクトリを作成**します。 |
| ⭐ `-s <SHELL>` | ユーザーのログインシェルを指定します。対話ログインをさせないサービスアカウントには **`/sbin/nologin`** を指定します。 |
| `-d <DIR>` | ホームディレクトリのパスを明示的に指定します。 |
| **グループ所属** | |
| ⭐ `-g <GROUP>` | ユーザーの**プライマリグループ**を指定します。 |
| ⭐ `-G <GROUPS>`| ユーザーが所属する**セカンダリ（追加）グループ**をカンマ区切りで指定します。（例: `-G wheel,docker`） |
| **その他** | |
| `-c <COMMENT>` | ユーザーに関するコメント（フルネームなど）を指定します。 |
| `-r`, `--system` | **システムアカウント**を作成します。通常、UIDが低く、ホームディレクトリは作成されません。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しい一般ユーザーを作成し、ホームディレクトリとログインシェル、追加のグループを設定する。
* **組み合わせ**: `sudo useradd -m -s /bin/bash -G <groups> <username>`
* **解説**: 新しい開発者のためにアカウントを作成し、`sudo` や `docker` を実行できる権限を持つグループに追加する、最も基本的な使い方です。
* **例**:

		```bash
		# ユーザー newdev を作成し、sudoグループに追加

		sudo useradd -m -s /bin/bash -G sudo newdev

		sudo passwd newdev

		```

## 2. ブルーチーム視点

* **タスク**: システムのアカウントを監査し、不審なアカウントがないか確認する。
* **組み合わせ**: `cat /etc/passwd`
* **解説**: `useradd` は直接使いませんが、`/etc/passwd` ファイルを監査し、意図しないユーザーアカウントが作成されていないかを確認します。特に、UIDが `0` のアカウントや、特権グループ (`root`, `wheel`, `sudo`) に所属している不審なアカウントがないかを重点的にチェックします。
* **例**:

		```bash
		# UIDが0の特権アカウントを全てリストアップする

		awk -F: '($3 == 0) { print $1 }' /etc/passwd

		```

## 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のために、隠れた特権アカウントを作成する。
* **組み合わせ**: `sudo useradd -o -u 0 -g 0 ... attacker`
* **解説**: 攻撃者がroot権限を奪取した後、システムに永続的なバックドアを設置するためにこのコマンドを実行します。`-u 0 -g 0` で `root` と同じ権限を持つユーザーを作成し、`-o` でUIDの重複を許可します。ユーザー名を `root` ではない目立たない名前にすることで、特権アカウントだと気づかれにくくします。
* **例**:

		```bash
		# 'system-svc' という名前で、UID 0 の隠しアカウントを作成

		sudo useradd -o -u 0 -g 0 -m -d /home/system-svc -s /bin/bash system-svc

		```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **エラーメッセージ例 1**: `useradd: user '<username>' already exists`
		* **考えられる原因**: 指定したユーザー名は既にシステムに存在します。
		* **解決策**: 別のユーザー名を使用してください。

2. **エラーメッセージ例 2**: `useradd: group '<groupname>' does not exist`
		* **考えられる原因**: `-g` または `-G` で指定したグループが存在しません。
		* **解決策**: 先に `groupadd <groupname>` でグループを作成してください。

## 環境変数と設定ファイル

* `useradd` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/default/useradd`**: `useradd` のデフォルトの挙動（デフォルトのシェル、ホームディレクトリのスケルトンパスなど）を定義します。
* **`/etc/passwd`, `/etc/shadow`, `/etc/group`**: `useradd` はこれらのファイルを更新します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `useradd` 自体ではなく、**不適切なアカウントの作成・管理**がセキュリティリスクとなります。
* **悪用シナリオ**: 攻撃者がroot権限を奪取した後、このコマンドを使ってバックドアとなる特権アカウントを作成し、永続化を図ります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: もしユーザーが `sudo useradd` を実行できる権限を持っている場合、新しいユーザーを作成し、そのユーザーを `sudo` グループに追加 (`-G sudo`) し、パスワードを設定 (`passwd`) してからその新しいユーザーに `su` することで間接的に権限昇格が可能です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、サービスアカウントには `-s /sbin/nologin` を設定し、対話的ログインを無効化する。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `auditd` やファイル整合性監視(FIM)ツールで、`/etc/passwd` や `/etc/shadow` ファイルへの変更を監視する。

## 注意点・補足

* **パスワードの設定**: `useradd` でユーザーを作成しただけでは、そのアカウントはパスワードが設定されておらずロックされた状態です。`sudo passwd <username>` を実行してパスワードを設定するまで、そのユーザーはログインできません。
