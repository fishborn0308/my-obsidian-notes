# Command - Windows - cmd - cmdkey - ユーザー名とパスワードの保存・表示・削除

## 概要

`cmdkey` は、Windows の「資格情報マネージャー（Windows Vault）」に格納されているユーザー名とパスワード、またはスマート カードの資格情報を作成、一覧表示、および削除するために使用されます。

一度保存された資格情報は、対象のリソース（共有フォルダやリモートデスクトップなど）にアクセスする際に自動的に使用されます。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `net use` | 特定の共有フォルダへの接続と認証を「一時的」に行う。再起動で消えることが多い。 | 一時接続なら `net use` |
| 資格情報マネージャー | `cmdkey` の GUI 版。コントロール パネルから視覚的に管理できる。 | 一般ユーザーなら GUI |
| `vaultcmd` | 内部的なツール。より詳細な「保管庫」の操作が可能だが一般的ではない。 | 通常は `cmdkey` |

## よく連携されるコマンド

### シナリオ例: バックアップ用 NAS への自動認証設定 (インフラ構築)

* **目的**: バッチファイルで NAS に接続する際、スクリプト内にパスワードを平文で書かずに認証を済ませる。
* **連携コマンド**: `net use`
* **解説**: 先に `cmdkey` で認証情報を保存しておけば、`net use` 実行時にパスワードを求められません。
* **コマンド例**:
    ```cmd
    REM NAS の認証情報を保存
    cmdkey /add:NAS01 /user:BackupUser /pass:P@ssw0rd123
    
    REM パスワード指定なしでドライブをマウント
    net use Z: \\NAS01\Backup
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/list` | 保存されているすべての資格情報のターゲット名とユーザー名を表示する。**（パスワード自体は表示されない）** |
| ⭐ `/add:<Target>` | 指定したターゲット（ホスト名やドメイン）の資格情報を新規作成または更新する。 |
| ⭐ `/delete:<Target>` | 指定したターゲットの資格情報を削除する。 |
| `/generic:<Target>` | 汎用的な資格情報として保存する。 |
| `/pass:<Password>` | `/add` と併用してパスワードを指定する。指定しない場合はプロンプトが表示される。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 共有 PC での特定リソースへのアクセス権リセット
* **解説**: 以前のユーザーの認証情報が残っていて別のアカウントでログインできない場合、一度キャッシュをクリアします。
* **例**:
    ```cmd
    REM 特定サーバーへの保存済み認証情報を削除
    cmdkey /delete:DomainController01
    ```

### 2. ブルーチーム視点

* **タスク**: 不適切な資格情報保存の監査
* **解説**: サーバー上で `runas /savecred` などによって管理者の資格情報が不用意に残されていないか定期的にチェックします。
* **例**:
    ```cmd
    REM 保存されている資格情報を一覧化して調査
    cmdkey /list
    ```

### 3. レッドチーム視点 (OSCP / Credential Access)

* **タスク**: 保存された資格情報の悪用による横展開
* **解説**: 侵入した端末で `cmdkey /list` を実行し、他のサーバーやドメイン管理者へのパスワードが保存されているか確認します。
* **例**:
    ```cmd
    REM ステップ1: 保存された資格情報の確認
    cmdkey /list
    
    REM ステップ2: もし管理者(Administrator)の資格情報が保存されていれば
    REM パスワードを知らなくても、それを利用して別サーバーの情報を閲覧できる
    runas /savecred /user:Administrator "cmd.exe /c dir \\TargetServer\C$"
    ```

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `cmdkey /list` で何も表示されない。
    * **原因**: 現在のユーザーコンテキストで保存された資格情報がない、またはシステムアカウント（SYSTEM）で実行している。
    * **解決策**: 資格情報はユーザープロファイルごとに保存されるため、調査したいユーザーとして実行する必要がある。

2.  **現象**: `cmdkey /add` で追加したのに `net use` で弾かれる。
    * **原因**: ターゲット名の指定（IP アドレスかホスト名か）が、アクセス時の指定と一致していない。
    * **解決策**: 接続時に使う名前（例: `\\192.168.1.1`）と `cmdkey` に登録する名前を一致させる。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **`runas /savecred` の罠**: 管理者が一度でも `/savecred` を使ってログインすると、そのパスワード（のハッシュ/トークン）が Windows Vault に保存されます。攻撃者は **`cmdkey` でその存在を確認し、パスワードを知らなくてもその権限でプログラムを動かせてしまいます。**

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Credential Access`
* **解説**: システムに保存されている認証情報を列挙し、その後の攻撃（横展開）の足がかりにする。
* **参照**: [LOLBAS - cmdkey.exe](https://lolbas-project.github.io/lolbas/Binaries/Cmdkey/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - GPO で「ネットワーク パスワードおよび資格情報の保存を許可しない」を有効にする。
    - 管理業務に `/savecred` を絶対に使用しないよう徹底する。
* **Detection (検知)**:
    - **イベントID 4648**: 明示的な資格情報を使用したログオン（`runas` と `cmdkey` の組み合わせなど）を監視する。
    - **イベントID 4688**: `cmdkey.exe /list` の実行を監視。一般ユーザーがこれを叩く理由は通常ありません。

## 注意点・補足

* **パスワードの秘匿性**: `cmdkey` 自体は保存されたパスワードを画面に表示しません。しかし、サードパーティ製ツール（Mimi-katz 等）や `runas /savecred` を使えば、その保存された「状態」を悪用できてしまうのが Windows の仕様です。
