# Command - Linux - less - ファイルの内容を 1 画面ずつ表示する

## 概要

`less` は、テキストファイルの内容を表示するためのプログラム（ページャー）です。最大の特徴は、**「ファイル全体を読み込む前に表示を開始する」** ことにあります。これにより、数 GB あるような巨大なログファイルでも、待たされることなく瞬時に開き、前後自由にスクロールしたり検索したりすることが可能です。

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `more` | 古典的。基本的に前方にしか進めない。ファイル全体を読み込む必要がある。 | (互換用) |
| `cat` | 全内容を一気にターミナルに出力する。 | 小さいファイル用 |
| `most` | 画面分割が可能で、カラー表示に強い。 | 複数ファイル同時閲覧 |
| `view` | Vim の読み取り専用モード。高度な編集機能の感覚で読める。 | コードリーディング |
| `zless` | 圧縮された（.gz）ファイルを解凍せずに直接 `less` で開く。 | **圧縮ログの調査** |

## 基本的な操作（キーバインド）

`less` を起動している最中に使える主なキーです。

| キー | アクション |
| :--- | :--- |
| ⭐️ `Space` / `f` | 1 画面分、先に進む（Forward）。 |
| ⭐️ `b` | 1 画面分、後ろに戻る（Backward）。 |
| ⭐️ `j` / `k` | 1 行ずつ進む / 戻る（Vim 互換）。 |
| ⭐️ `g` / `G` | ファイルの先頭へ移動 / 末尾へ移動。 |
| ⭐️ `/` / `?` | 前方検索 / 後方検索（キーワード入力後に `Enter`）。 |
| ⭐️ `n` / `N` | 次の検索結果へ / 前の検索結果へ。 |
| ⭐️ `-N` | 行番号を表示 / 非表示に切り替える。 |
| `F` | 追記を待ち受ける（`tail -f` 相当）。`Ctrl+C` で通常の less に戻る。 |
| `q` | `less` を終了する（Quit）。 |

## よく使われるオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-N`, `--line-numbers` | 起動時から行番号を表示する。 |
| ⭐️ `-S`, `--chop-long-lines` | 長い行を折り返さず、横スクロールにする（ログが読みやすくなる）。 |
| ⭐️ `-i`, `--ignore-case` | 検索時に大文字・小文字を区別しない。 |
| ⭐️ `-R`, `--RAW-CONTROL-CHARS` | ANSI カラーエスケープシーケンスを解釈して色付きで表示する。 |
| `-X`, `--no-init` | 終了時に画面内容を消さず、ターミナルに残す。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: 数 GB の Apache/Nginx アクセスログの調査。
* **解説**: 巨大なログを `cat` してターミナルをフリーズさせたり、エディタで開いてメモリを使い果たしたりすることなく、安全に特定のエラーコードを検索します。
* **例**:
    ```bash
    # 行を折り返さず、行番号付きでログを開く
    less -SN /var/log/nginx/access.log
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査（フォレンジック）中のファイル閲覧。
* **解説**: `less` はデフォルトでファイルを変更しません（書き込み権限があっても）。エディタのように「うっかり書き換えて証拠を汚染する」リスクが低いため、安全な閲覧ツールとして重宝されます。

### 3. レッドチーム視点

* **タスク**: シェルエスケープによる権限昇格。
* **解説**: `less` の中から他のコマンドを実行できる機能（シェルエスケープ）を悪用します。もし `less` が `sudo` 権限で実行可能な設定になっていれば、そこから root シェルを奪取できます。
* **GTFOBins 手順**:
    1. `sudo less /etc/passwd`
    2. 閲覧中に `!/bin/sh` と入力して `Enter`
    3. root 権限のシェルが起動する。

## エラーメッセージとトラブルシューティング

1.  **「Missing filename ("less --help" for help)」**:
    * **原因**: ファイル名を指定し忘れている。
    * **解決策**: `less filename` とするか、パイプから流し込む（`ls | less`）。

2.  **「Binary file (standard input) (y or n)?」**:
    * **原因**: テキストではなくバイナリファイルを開こうとしている。
    * **解決策**: 無理に開くと画面が崩れるため、`n` で止めて `hexdump` や `strings` を使う。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **Information Disclosure**: `-o` オプション（出力をファイルに記録）が許可されている場合、閲覧内容を意図しない場所に保存されるリスクがあります。
* **Shell Escape**: 前述の通り、`sudo` 権限下の `less` は非常に危険です。

### 対応策・緩和策 (ブルーチーム視点)

* **Environment**: `LESSSECURE=1` という環境変数を設定しておくと、`less` の「外部コマンド実行」や「ファイル保存」機能を無効化でき、セキュリティを向上させることができます。
