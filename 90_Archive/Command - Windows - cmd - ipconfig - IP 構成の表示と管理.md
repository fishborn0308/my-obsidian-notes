# Command - Windows - cmd - ipconfig - IP 構成の表示と管理

## 概要

`ipconfig` (Internet Protocol Configuration) は、Windows コンピューターの現在のすべての TCP/IP ネットワーク構成値を表示します。また、動的ホスト構成プロトコル (DHCP) およびドメイン ネーム システム (DNS) の設定を更新したり、キャッシュを消去したりするためにも使用されます。

(出自: `Windows 標準搭載 - 外部コマンド (ipconfig.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ifconfig` | Linux/Unix 版。インターフェースの有効化/無効化も行う。 | (Linux環境) |
| `netsh interface ip` | IP アドレスの静的設定や詳細なプロトコル操作が可能。 | 高度な構成変更 |
| `Get-NetIPAddress` | PowerShell 版。オブジェクト形式で情報を取得でき、抽出に強い。 | PS 環境 |
| `nmcli` | 近年の Linux (NetworkManager) での標準的な管理ツール。 | (Linux環境) |

## よく連携されるコマンド

### シナリオ例: ネットワーク通信不可時の切り分け (運用・トラブルシューティング)

* **目的**: 自分の端末に正しく IP アドレスが割り当てられているか、ゲートウェイまで届いているかを確認する。
* **連携コマンド**: `ipconfig`, `ping`, `tracert`
* **解説**: 情報を確認 (`ipconfig`) し、近隣 (`ping`) から遠隔 (`tracert`) へ疎通を確認します。
* **コマンド例**:
    ```cmd
    REM 自分の IP とデフォルト ゲートウェイを確認
    ipconfig
    REM ゲートウェイに対して疎通確認
    ping [Gateway_IP]
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| (なし) | 各アダプターの IP アドレス、サブネット マスク、ゲートウェイを表示。 |
| ⭐ `/all` | **【重要】** MAC アドレス、DNS サーバー、DHCP の有無を含む全情報を表示。 |
| ⭐ `/release` | 現在の DHCP リースを解除し、IP アドレスを返却する。 |
| ⭐ `/renew` | DHCP サーバーから新しい IP アドレスを取得（更新）する。 |
| ⭐ `/flushdns` | DNS リゾルバー キャッシュを消去する（名前解決トラブル時に多用）。 |
| `/displaydns` | 現在キャッシュされている DNS レコードを表示する。 |
| `/registerdns` | DHCP リースを更新し、DNS 名を再登録する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: クライアント PC が社内 LAN に接続できなくなった際、DHCP からの取得をやり直させる。
* **例**:
    ```cmd
    REM 一度 IP を手放し、再度取得し直す
    ipconfig /release & ipconfig /renew
    ```

### 2. ブルーチーム視点 (インシデント調査)

* **タスク**: 端末が外部の不審な DNS サーバーを参照するように書き換えられていないか（DNS ポイズニング/ハイジャック）を確認する。
* **例**:
    ```cmd
    REM 設定されている DNS サーバーのアドレスを精査
    ipconfig /all | findstr /i "DNS Server"
    ```

### 3. レッドチーム視点 (偵察と横展開 - T1016)

* **タスク 1: Internal Network Enumeration (内部ネットワークの列挙)**
    * **目的**: 侵入した端末がマルチホーム（複数のネットワークに接続）されていないか、接続先セグメントがどこかを確認する。
    * **解説**: `/all` を使用して、物理 NIC だけでなく、VPN、仮想マシンのブリッジ、トンネルインターフェースの存在を暴きます。
* **タスク 2: Finding High-Value Targets (HVT の特定)**
    * **目的**: デフォルト ゲートウェイや DNS サーバーの IP を特定し、それらを次の攻撃対象（スニフィングや中間者攻撃のポイント）にする。
* **タスク 3: Anti-Forensics (キャッシュ消去)**
    * **目的**: 調査の過程で発生した不審なドメインへの名前解決記録を、`/flushdns` で消去する（ただし、実行自体が目立つ可能性もある）。

## エラーメッセージとトラブルシューティング

1.  **メディアの状態 . . . . . . . . . . . : メディアは接続されていません**
    * **原因**: LAN ケーブルが抜けている、Wi-Fi がオフ、または物理的なリンクアップができていない。
    * **解決策**: ハードウェア接続を確認するか、ドライバの状態を `driverquery` 等で確認。

2.  **DHCP サーバーに到達できません**
    * **原因**: ネットワーク内に DHCP サーバーがない、またはセグメントが分離されている。
    * **解決策**: 固定 IP を設定してテストするか、ネットワーク構成を再確認。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Information Disclosure (情報漏洩)**:
    * 攻撃者が `ipconfig /all` を実行することで、ホスト名、OS の詳細バージョン（一部）、内部ドメイン名、プライベート IP 体系、MAC アドレス（ハードウェアベンダー特定）といった情報がすべて露呈します。

### LOLBAS における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: どのような権限（標準ユーザー）でも実行可能であり、ネットワーク調査の初手としてほぼ確実に実行されます。

## 注意点・補足

* **IPv6**: 近年の環境では IPv6 アドレスも大量に表示されます。IPv4 のみを見たい場合は、`findstr` で絞り込むと見やすくなります。
* **権限**: `/flushdns` や `/renew` などの「変更」を伴う操作には、通常、**管理者権限**が必要です。
