# Procedure - VS Code Git Workflow - VS CodeによるGit運用

## 概要

Visual Studio Code (VS Code) は、Gitの機能をコアに深く統合しており、コマンドラインを使わなくても、直感的なGUIでほとんどのバージョン管理操作を実行できます。変更の追跡、ステージング、コミット、ブランチの切り替え、リモートリポジトリとの同期といった一連のワークフローをシームレスに行えるため、開発効率が飛躍的に向上します。

このチートシートでは、VS CodeのGUIを使った基本的なGitのワークフローと、各視点での活用シナリオを解説します。

(出自: VS Codeの組み込み機能およびGit拡張機能)

## 類似コマンドと差異

| 操作方法 | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **コマンドライン (`git ...`)** | 全てのGitの機能にアクセスでき、スクリプトによる自動化にも不可欠です。`rebase -i` のような高度な操作はCLIの方が柔軟です。 | CLI (自動化・高度な操作) |
| ⭐ **VS Code GUI** | 日常的な操作（ステージング、コミット、プッシュ、プル、ブランチ切り替え、マージ）を視覚的かつ直感的に行えます。学習コストが低く、ヒューマンエラーを減らせます。 | VS Code GUI (日常的な操作) |

## 基本的なワークフローとUI要素

VS CodeでのGit操作は、主に左側のサイドバーにある**ソース管理ビュー**と、左下の**ステータスバー**で行います。

### 1. 変更のステージングとコミット

1. ファイルを変更・保存すると、ソース管理ビューの「変更」セクションにリストアップされます。
2. ファイルにカーソルを合わせ、「**+**」（変更をステージ）アイコンをクリックしてステージングします。
3. 上部のメッセージボックスにコミットメッセージを入力します。
4. `Ctrl+Enter` またはチェックマークアイコンをクリックしてコミットします。

### 2. ブランチの作成と切り替え

1. 左下のステータスバーに表示されている現在のブランチ名をクリックします。
2. 表示されたリストから既存のブランチを選択して切り替えるか、「`+ 新しいブランチを作成...`」を選択して新しいブランチを作成します。

### 3. リモートとの同期

* ステータスバーの同期アイコン（循環する矢印）をクリックすると、`git pull` と `git push` を実行します。

### 4. 競合の解決

* マージやプルで競合が発生すると、VS Codeは競合ファイルをハイライトします。エディタ内で「現在の変更を取り込む」「入力側の変更を取り込む」などのボタンが表示され、視覚的に競合を解決できます。

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](../linux/troubleshooting_common_errors.md) を参照。

1. **現象**: **GitHubへのプッシュ/プル時に認証エラーが表示される。**
    * **考えられる原因**: 認証情報が古いか、Personal Access Token (PAT) の権限が不足しています。
    * **解決策**:
        1. Windowsの場合は資格情報マネージャー、macOSの場合はキーチェーンアクセスから、古いGitHubの資格情報を削除します。
        2. VS Codeで再度同期を試みると、ブラウザが開きGitHubの認証を求められるので、再認証します。
        3. PATを使用している場合は、GitHubの設定ページで `repo`, `workflow` などのスコープが正しく設定されているか確認します。

## 環境変数と設定ファイル

* `settings.json` でGitの挙動をカスタマイズできます。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](../linux/environment_and_config.md) を参照。

### 関連する設定ファイル (`settings.json`)

```json
{
  "git.autofetch": true,
  "git.confirmSync": false,
  "git.enableSmartCommit": true,
  "gitlens.currentLine.enabled": true
}
```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **拡張機能の脆弱性**。VS Code自体は安全性が高いですが、インストールしたサードパーティ製の拡張機能に悪意のあるコードが含まれている可能性があります。
* **悪用シナリオ**: 攻撃者が作成した人気の拡張機能に似せた偽の拡張機能（タイポスクワッティング）をインストールしてしまい、コードの編集中に機密情報（APIキーなど）が外部に送信される。

### GTFOBins / LOLBAS における利用例

VS Code自体はGTFOBinsにはリストされていませんが、それが呼び出す `git` コマンドやターミナルは悪用される可能性があります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **拡張機能は公式マーケットプレイスからのみインストール**し、発行元が信頼できることを確認する。`GitGuardian` や `TruffleHog` のようなシークレットスキャン系の拡張機能を導入し、機密情報がコミットされるのを防ぐ。
* **Detection (検知)**: ネットワークトラフィックを監視し、VS Codeやその関連プロセスから予期せぬ外部への通信がないかを確認する。

## 注意点・補足

* **拡張機能の活用**: `GitLens` や `Git Graph` といった拡張機能を導入することで、コミット履歴の可視化や、行ごとのコミット情報表示などが可能になり、Git操作の理解と効率が飛躍的に向上します。
