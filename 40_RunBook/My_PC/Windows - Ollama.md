# Ollama 導入・運用ガイド (Windows / Minisforum M1 Pro-285H 向け)

## 1. インストール手順

### 1. インストール

管理者権限で PowerShell を開き、以下のコマンドを貼り付けて Enter を押します。

```powershell
irm https://ollama.com/install.ps1 | iex
```

> **解説:** > * `irm` (Invoke-RestMethod): インターネットからインストールスクリプトを読み込みます。
> * `iex` (Invoke-Expression): 読み込んだスクリプトをその場で実行します。
> 
> 

### 2. インストール後の確認

インストールが完了したら、以下のコマンドで Ollama サーバーが正常に反応するか確認します。

```powershell
ollama --version
```

## 2. モデルのダウンロードと管理

ターミナル（PowerShell または コマンドプロンプト）を開き、以下のコマンドを使用します。

### 主要なモデルの取得

* **汎用（軽量・高速）:** `ollama run llama3.1` (8Bモデル)
* **日本語特化:** `ollama run gemma2` (9Bモデル)
* **最高性能（低速）:** `ollama run llama3.3` (70Bモデル)

### 管理コマンド一覧

| コマンド | 内容 |
| --- | --- |
| `ollama list` | インストール済みモデルの一覧表示 |
| `ollama rm [モデル名]` | 不要なモデルの削除 |
| `ollama pull [モデル名]` | 実行せずにダウンロードだけ行う |

```
# 精度重視（まずはこれ！）
ollama run llama3.1:70b

# 速度と整形のバランス重視
ollama run qwen2.5:32b

# 思考型（分析させたい時）
ollama run deepseek-r1:32b

# 重厚・高知能モデル（高性能だが非常に重い）
ollama run llama3.3
```

## 3. 実践的なプロンプトの投げ方

ターミナルでの対話（`>>>` 表示時）における操作方法です。

### 基本操作

* **一行入力:** そのまま文章を打って Enter。
* **複数行入力:** `"""` で囲む。
```text
>>> """
ここに長い文章や
ソースコードを貼り付ける
"""

```


* **終了:** `/bye` と入力。

### システムプロンプト（役割設定）の例

モデルの振る舞いを固定したい場合、最初にこう伝えます。

> `あなたはプロのITコンサルタントです。専門用語を避け、中学生にもわかるように解説してください。`

## 4. パフォーマンス最適化（Minisforum特有のポイント）

Core Ultra 9 285Hの性能を引き出すためのチェックリストです。

* **GPUアクセラレーション:** タスクマネージャーの「パフォーマンス」タブで、推論中に **Intel Arc Graphics (GPU)** が稼働しているか確認してください。
* **メモリ割り当て:** 64GBあるため、70Bモデルを動かす際は他の重いアプリ（ブラウザのタブ大量開きなど）を閉じると、スワップが発生せず安定します。
* **速度の目安:**
* **8B/9Bモデル:** リアルタイム（人間が読むより速い）
* **70Bモデル:** 1文字ずつゆっくり（熟考型）

## 5. 推奨エコシステム（GUI化）

黒い画面を卒業し、快適に使うためのツールです。

* **Page Assist (ブラウザ拡張):** ブラウザのサイドバーでChatGPTのようにOllamaを使えます。
* **Chatbox AI:** Windowsアプリとして動作。UIが綺麗で、プロンプトのテンプレート保存が可能です。

## 💡 知っておくと便利な PowerShell 操作術

### 1. 起動しているモデルを「即座に」終了させる

70Bモデルが重すぎて PC の動作が怪しくなったときは、別の PowerShell ウィンドウで以下を叩けば強制終了できます。

```powershell
stop-process -name "ollama*" -force

```

### 2. 自分がダウンロードしたモデルのサイズを確認する

どれだけディスク（SSD）を圧迫しているか一目でわかります。

```powershell
ollama list

```

### 3. テキストファイルを読み込ませる (RAGの第一歩)

例えば、`memo.txt` の内容を AI に要約させたい場合：

```powershell
Get-Content .\memo.txt | ollama run gemma2 "この内容を3行で要約して"

```

### 4. 使用していないモデルの削除（ディスク容量の確保）

Llama 3.3 70B などの大型モデルは、1つで **40GB〜50GB** もの容量を占有します。試してみて「重すぎる」と感じたモデルや、使わなくなった古いモデルを削除する手順です。

#### インストール済みモデルの確認

まず、現在どのモデルがどれだけの容量を使っているかリストアップします。

```powershell
ollama list

```

#### 特定のモデルを削除する

不要なモデル名を指定して削除します。

```powershell
# 例：Llama 3.3 70Bを削除する場合
ollama rm llama3.3

```

#### まとめて削除する（上級編）

もし複数のテスト用モデルを一気に消したい場合は、PowerShellのフィルタリング機能が使えます。

```powershell
# 名前に "test" が含まれるモデルをすべて削除する例
ollama list | Select-String "test" | ForEach-Object { ollama rm $_.ToString().Split(' ')[0] }

```
