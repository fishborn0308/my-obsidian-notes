# Cheatsheet - VSCode Regular Expression

## Visual Studio Code(VS Code)で正規表現を使いこなすためのチートシート

Visual Studio Code(VS Code)で、正規表現を使ってテキストを効率的に検索・置換するための便利な方法を解説します。特定の文字列を含む行だけを残す（あるいは削除する）といった基本的な操作から、より高度な編集テクニックまでを網羅します。

状況に応じて使い分けられるように、いくつかの方法をご紹介します。

1. **正規表現を使った検索と置換(最も強力)**
2. **「Search Editor」機能を使う(正規表現が苦手な方向け)**
3. **拡張機能を使う(特定の作業を頻繁に行う方向け)**

-----

## 方法1：正規表現を使った検索と置換

VS Codeの標準機能だけで完結する、最も強力で応用範囲の広い方法です。

### 基本操作：特定の文字列を含む行だけを残す

ここでは例として「`ERROR`」という文字列を含む行だけを残す（＝`ERROR`を含まない行を削除する）手順を説明します。

#### ステップ1：不要な行を検索する

1. `Command` + `F`(Mac)または `Ctrl` + `F`(Windows/Linux)で検索ウィンドウを開きます。
2. 検索ウィンドウ右側の `正規表現を使用` アイコン(`.*`)をオンにします。
3. 検索ボックスに以下の正規表現を入力します。`ERROR` の部分を残したい文字列に書き換えてください。

		```regexp

		^(?!.*ERROR).*$\n?

		```

		**【正規表現の解説】**

			* `^` : 行の先頭を表します。
			* `(?!.*ERROR)` : **否定先読み**という機能です。「この位置の直後に `.*ERROR` というパターンが続かない」という条件を指定します。これが「`ERROR`を含まない」を実現する重要な部分です。
			* `.*` : 任意の文字が0回以上続くことを意味し、行全体の内容にマッチします。
			* `$` : 行の末尾を表します。
			* `\n?` : 行末の改行文字にマッチします。`?` を付けることで、ファイルの最終行に改行がなくてもマッチするようにしています。

#### ステップ2：見つかった行を削除(置換)する

1. 検索ボックスの左にある `>` をクリックして、置換ボックスを表示します。
2. **置換ボックスは空のまま**にしておきます。
3. `全て置換` アイコンをクリックします。

これで、「`ERROR`」を含まない全ての行が削除され、目的の行だけが残ります。

### (補足)残った空行を削除する

上記の手順を実行すると、削除された行が空行として残ってしまう場合があります。その場合は、以下の手順で空行を一括削除できます。

1. 検索ボックスに、空行にマッチする以下の正規表現を入力します。

		```regexp

		^\s*$\n

		```

2. 置換ボックスは空のまま、「全て置換」をクリックします。

## 【追加】覚えておくと便利な正規表現 活用例

検索と置換を応用した、よく使う便利な編集テクニックです。

### 1\. 特定の文字列を含む行を**削除**する

上記とは逆に、「`DEBUG`」という文字列を含む行**だけ**を全て削除したい場合の正規表現です。

* 検索ボックス: `^.*DEBUG.*\n?`
* 置換ボックス: (空)

### 2\. 行頭/行末に文字列を追記する

全ての行の先頭に ` // ` を追記してコメントアウトする場合。

* 検索ボックス: `^`
* 置換ボックス: ` // `

全ての行の末尾に `;` を追記する場合。

* 検索ボックス: `$`
* 置換ボックス: `;`

### 3\. 行全体を引用符などで囲む

行全体をダブルクォーテーション(`"`)で囲む場合。

* 検索ボックス: `(.+)`
* 置換ボックス: `"$1"`

**【解説】**

`()`で囲んだ部分は「キャプチャグループ」となり、置換ボックス内で `$1`（1番目のグループ）、`$2`（2番目のグループ）のように参照して再利用できます。ここでは行全体 `(.+)` をキャプチャし、置換時にその前後を `"` で囲んでいます。

`$0` を使って `"$0"` とすると、マッチした文字列全体を参照できるため、同様の結果が得られます。

### 4\. キャプチャグループを利用した置換（順番の入れ替え）

`姓 名` のフォーマットを `名, 姓` に入れ替える例です。

* 検索ボックス: `(\w+)\s(\w+)`
* 置換ボックス: `$2, $1`

**【解説】**

`(\w+)`は1つ以上の英数字（単語）にマッチします。1つ目の `(\w+)` が `$1` に、2つ目の `(\w+)` が `$2` に対応するため、置換ボックスで順番を入れ替えることができます。

### 5\. 複数行にまたがる文字列を置換

`<p>` タグで囲まれた複数行のコンテンツを削除する例です。

* 検索ボックス: `<p>[\s\S\n]*?</p>`
* 置換ボックス: (空)

**【解説】**

`[\s\S\n]` は、空白文字(`\s`)、空白以外の文字(`\S`)、改行(`\n`)のいずれかにマッチするため、あらゆる文字（改行含む）にマッチします。 `*?` は「最短一致」を意味し、最初に出現する `</p>` までを範囲とします。

## 【追加】正規表現チートシート

VS Codeの検索でよく使う基本的なメタ文字やパターンです。

| 記号/パターン | 説明 | 使用例 |
| :--- | :--- | :--- |
| **アンカー** | | |
| `^` | 行の先頭 | `^abc` は "abc" で始まる行にマッチ |
| `$` | 行の末尾 | `xyz$` は "xyz" で終わる行にマッチ |
| **量指定子** | | |
| `*` | 直前の文字の0回以上の繰り返し | `a*` は ""(空文字), "a", "aa" にマッチ |
| `+` | 直前の文字の1回以上の繰り返し | `a+` は "a", "aa" にマッチ |
| `?` | 直前の文字が0回または1回 | `jpe?g` は "jpg", "jpeg" にマッチ |
| `{n}` | 直前の文字のn回の繰り返し | `\d{3}` は3桁の数字にマッチ |
| `{n,}` | 直前の文字のn回以上の繰り返し | `\d{2,}` は2桁以上の数字にマッチ |
| `{n,m}` | 直前の文字のn回以上m回以下の繰り返し | `\d{2,4}` は2〜4桁の数字にマッチ |
| **グループ化と参照** | | |
| `( )` | パターンをグループ化し、キャプチャする | `(abc)+` は "abc", "abcabc" にマッチ |
| `\|` | 「または」(OR) | `cat\|dog` は "cat" または "dog" にマッチ |
| `$1`, `$2`... | n番目のキャプチャグループを参照(置換時に使用) | |
| **文字クラス** | | |
| `.` | 任意の1文字（改行を除く） | `a.c` は "abc", "axc" にマッチ |
| `[ ]` | 角括弧内のいずれか1文字 | `[abc]` は "a", "b", "c" のいずれかにマッチ |
| `[^ ]` | 角括弧内を除くいずれか1文字 | `[^0-9]` は数字以外の1文字にマッチ |
| `\d` | 半角数字1文字 (`[0-9]`) | |
| `\w` | 半角英数字とアンダースコア1文字 (`[a-zA-Z0-9_]`) | |
| `\s` | スペース、タブなどの空白文字1文字 | |
| `\n` | 改行文字 | |
| `\t` | タブ文字 | |

## 方法2：「Search Editor」機能を使う

「正規表現は難しくてよくわからない」という方にオススメの、直感的な方法です。

1. `Command` + `Shift` + `F`(Mac)または `Ctrl` + `Shift` + `F`(Windows/Linux)でサイドバーに検索パネルを開きます。
2. 検索ボックスに残したい文字列(例: `ERROR`)を入力します。
3. 検索結果が表示されたら、検索ボックスの右(または検索結果の右上)にある `Open in editor`(エディタで開く)アイコンをクリックします。
4. すると、検索文字列を含む行だけが抽出された新しいタブ(`search-editor`)が開きます。
5. そのタブの内容を全て選択(`Command` + `A` または `Ctrl` + `A`)し、コピー(`Command` + `C` または `Ctrl` + `C`)して、元のファイルに貼り付けるか、新しいファイルとして保存します。

## 方法3：拡張機能を使う

この種のテキストフィルタリングを頻繁に行う場合は、専用の拡張機能を入れるのが最も効率的です。

1. VS Codeのサイドバーにある拡張機能アイコンをクリックします。
2. 検索ボックスに `Filter Lines` と入力し、同名の拡張機能などをインストールします。
3. ファイルを編集中の画面で、コマンドパレットを開きます(`Command` + `Shift` + `P` または `Ctrl` + `Shift` + `P`)。
4. `Filter` と入力すると、拡張機能のコマンドが表示されます。
5. `Filter: Keep lines containing`(含む行を残す)などを選択します。
6. 入力ボックスが表示されるので、残したい文字列(例: `ERROR`)を入力して `Enter` を押します。

すぐにファイルがフィルタリングされ、目的の行だけが残ります。

## まとめ

| 方法 | メリット | デメリット | オススメのケース |
| :--- | :--- | :--- | :--- |
| **正規表現** | VS Code標準機能のみ。強力で柔軟な条件指定が可能。 | 正規表現を覚える必要がある。 | 複雑な条件でフィルタリングしたい場合や、応用を効かせたい場合。 |
| **Search Editor** | 直感的で分かりやすい。正規表現が不要。 | 結果をコピー＆ペーストする手間がかかる。 | 手軽に特定の行を抽出したい場合。 |
| **拡張機能** | 最も簡単で速い。コマンドを選ぶだけ。 | 拡張機能のインストールが必要。 | 同じ作業を何度も繰り返す場合。 |
