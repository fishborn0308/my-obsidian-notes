以下の条件で「検知回避」の章を作成してください。

【対象技術】 （例：Apache HTTP Server / Nginx / MySQL などをここに記載）

Nginx

【想定読者】

SEC504受講レベルのペンテスト要員およびSOC要員

【目的】

・攻撃者がどのように検知を回避するか理解させる

・SOCが依存している前提条件を明確にする

・回避を前提にした検知設計を考えられるようにする

【分量】

約10ページ相当の詳細な解説

【重要制約】

・具体的な攻撃実行手順は記載しない（概念レベル）

・検知の前提モデルを最初に明示する

・回避手法は分類して説明する

・ログへの影響を具体的に示す

・実際のログ例（テキスト形式）を含める

・SOCが見逃す理由を明示する

・対抗策は「回避を前提」とした設計とする

・誤検知とのトレードオフを説明する

・第0章の基礎説明は繰り返さない

【構成】

1. 本章の到達目標
2. 検知の前提モデル
3. 主な検知回避手法（分類）
4. ログへの影響（ログ例を含む）
5. 検知困難化のメカニズム
6. SOC視点の落とし穴
7. 回避を前提とした検知設計
8. 誤検知とのトレードオフ
9. 演習（ログ分析と設計）
10. まとめと次章への接続

【必須要素】

・最低3種類以上の回避手法を解説する

・ログ例を最低2種類含める

・ネットワークとエンドポイント両方の視点を含める

・次章（ネットワーク検知／エンドポイント検知）との関係を明示する

出力は教科書形式で体系的に記述してください。
