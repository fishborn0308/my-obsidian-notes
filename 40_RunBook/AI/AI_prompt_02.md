あなたはSEC504レベルのペンテスター兼SOCアナリスト向け教材を作成する専門家です。

以下の条件で「技術モジュール章」を作成してください。

【対象技術】 （例：Apache HTTP Server / Nginx / MySQL などをここに記載）

Nginx

【想定読者】

SEC504受講レベルのペンテスト要員およびSOC要員

【目的】

・実務能力の強化

・資格対策

・攻撃／検知／防御を統合的に理解させる

【分量】

約10ページ相当の詳細な解説

【重要制約】

・基礎的なTCP/IPやHTTPの説明は第0章にある前提で繰り返さない

・攻撃観点、検知観点、防御（ハードニング）を必ず含める

・実際のログ例（テキスト形式）を含める

・ネットワーク視点とエンドポイント視点の両方を含める

・ATT&CKとの対応を明示する

・実務で使えるレベルの具体性を持たせる

・図は文章で説明する

・他製品との比較は簡潔に記載（横展開を意識）

【構成】

1. 本章の到達目標
	 - 読了後に何が説明・分析できるようになるか

2. 技術概要
	 - 役割
	 - 想定構成
	 - 使用レイヤ

3. アーキテクチャ詳細
	 - プロセス構造
	 - 通信フロー（文章による図解）
	 - 設定ファイル構造
	 - データ保存場所

4. ログ構造
	 - デフォルトログ
	 - 主要ログ項目の説明
	 - ログ出力タイミング
	 - 実際のログ例（テキスト形式）

5. 攻撃観点
	 - よく狙われるポイント
	 - 設定ミス
	 - 認証突破
	 - 典型的な脆弱性パターン
	 - 攻撃フローの説明

6. 検知観点
	 - どのログを確認するか
	 - 正常と異常の比較
	 - ネットワーク視点での検知
	 - エンドポイント視点での検知
	 - SOC分析観点

7. ハードニング
	 - 推奨設定
	 - 最小権限
	 - ログ保全
	 - 監査設定
	 - 攻撃との対応関係を明示

8. 演習
	 - 簡易攻撃再現手順（概念レベル）
	 - ログ確認手順
	 - 検知確認ポイント

9. まとめ
	 - 本章の要点
	 - 他章（攻撃フェーズ章）との接続

10. ATT&CK対応
	 - 本章に関連する代表的なテクニックID
	 - どのフェーズで悪用されるか

【必須要素】

・攻撃者がなぜこの技術を狙うのかを明確にする

・ログ例は最低2種類以上含める

・検知と防御は攻撃と対応関係を持たせる

・実務での誤検知例または見逃し例を1つ含める

・説明は抽象論ではなく実務シナリオを前提にすること

・箇条書きと文章をバランスよく使用すること

・不要な歴史的背景は省略すること

出力は教科書形式で体系的に記述してください。
