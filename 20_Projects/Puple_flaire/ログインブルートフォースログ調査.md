社内セキュリティチームから、管理しているサーバで不審な通信が発生していると連絡がありました。調査したところ、該当するサーバは昔の検証業務で利用していたUbuntuサーバでした。関係者によると、インターネットからSSHログインする目的で使用していたため、SSHサービス以外は動作させていないとのことでした。

あなたはUbuntuサーバにログインし、SSHログを調査することにしました。不審なSSHログインが最初に成功した日時を、ログに記載されている Jan 1 00:00:00 形式で解答してください。
開始手順

    調査対象サーバにSSHログイン

調査対象サーバにSSHログインします。なお、認証情報は、以下を利用してください。

    ユーザ名: root
    パスワード: ChT$7fIE
		
		ssh -p 8002 root@10.234.132.68

    ログ調査を開始

サーバ内のログ調査を開始して、問題に解答してください。

## 解説

この問題は、攻撃を受けて不審な通信が発生したことをきっかけにインシデント対応を行う場面を想定した問題です。 今回の場合、管理が手薄な検証用サーバでSSHパスワード認証が有効となっており、インターネットからの攻撃によってサーバが乗っ取られています。

インシデント対応を行う場合、インシデントの発生日時、つまり最初に不正ログインが成立した日時が重要です。 サーバの認証ログを調査することで、これを確認できる場合があります。

Linux OSでは、OpenSSHサーバの認証ログはデフォルトで以下のファイルに記録されています。

    RedHat系（CentOSなど）: /var/log/secure
    Debian系（Ubuntuなど）: /var/log/auth.log

今回の場合は問題文にある通りUbuntuのため、/var/log/auth.log が存在しています。

認証ログのフォーマットは以下のようになっています。

日時 ホスト名 プロセス名[プロセスID]: メッセージ

    日時: ログが記録された日時
    ホスト名: ログが記録されたホスト名
    プロセス名[プロセスID]: ログを記録したプロセス名とプロセスID
    メッセージ: ログの内容

メッセージの中でも特に重要なのが、SSHの認証成功/失敗を表す以下の2つです。

    認証失敗: Failed password for [ユーザ名] from [認証元のIPアドレス] port [認証元のポート番号] ssh2
    認証成功: Accepted password for [ユーザ名] from [認証元のIPアドレス] port [認証元のポート番号] ssh2

つまり、認証が成功した日時を見つけるには、メッセージに Accepted が含まれる行を検索すればよいことになります。
このとき、正常にログインが行われたか確認するため、前後数行も確認したほうが良いでしょう。

この結果から、攻撃者172.16.1.223 がユーザ ubuntu で複数回ログインを試みている事がわかります。さらに、Apr 10 05:47:13 にログインに成功していることがわかります。
また、Apr 10 05:47:16 にも再度ログインに成功していることがわかります。

この問題では「不審なSSHログインが最初に発生した日時」とあるため、解答は Apr 10 05:47:13 となります。

検証用サーバでは、一時的な用途だから、利用者には公開していないからという油断から、認証設定が甘くなることがあります。 この問題のように、時間が経過した結果、管理の手薄なサーバが動き続けていることもあります。 このようなサーバがインターネットから接続可能な場合、容易に攻撃を受けてしまうため、パスワード認証を無効にして公開鍵認証を使用する、サーバに接続可能なIPアドレスを制限するといった対策が重要です。

公開鍵認証で用いるSSH鍵は、ssh-keygenコマンドで生成できます。 SSH鍵の種類としてはRSAが主流ですが、近年ではより安全かつ効率的とされるEd25519も使用されるようになっています。 そのため、利用者の立場ではEd25519でのSSH鍵を生成する方法も知っておくとよいでしょう。

```
$ ssh-keygen -t ed25519
Generating public/private ed25519 key pair.
Enter file in which to save the key (/root/.ssh/id_ed25519): 
Created directory '/root/.ssh'.
Enter passphrase (empty for no passphrase): 
Enter same passphrase again: 
Your identification has been saved in /root/.ssh/id_ed25519
Your public key has been saved in /root/.ssh/id_ed25519.pub
```


```
$ ls ~/.ssh/
id_ed25519  id_ed25519.pub

$ cat ~/.ssh/id_ed25519.pub 
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIIud3SsuhCEQYd5vxJuJKOj4ZPT8stqKKrzT8zSkcC6G root@1695dfff97fe
```

id_ed25519 は秘密鍵、id_ed25519.pub は公開鍵です。 接続先の ~/.ssh/authorized_keys に id_ed25519.pub の内容を追記することで、公開鍵認証でSSHログインできるようになります。
