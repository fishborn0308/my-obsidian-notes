# [Pentest-Walkthrough] Web - Project: ログインブルートフォース攻撃検証

同僚が構築したUbuntuサーバに問題がないかの確認作業を依頼されました。SSHでアクセスしてみると、認証失敗時に次のようなレスポンスが返ってきます。

```
ubuntu@<target1_ip>: Permission denied (publickey,password).
```

これを見たあなたは、パスワード認証が無効になっていないことに気づきました。そこで同僚に相談したところ、実際にパスワード認証で不正ログインができるか検証することになりました。

SSHブルートフォース攻撃を試行し、ログイン可能なパスワードを解答してください。

開始手順

以下の情報をもとに、攻略対象サーバにSSHブルートフォース攻撃を開始し、問題に解答してください。

    IPアドレス: 10.234.132.139
    ポート番号: 22/tcp
    ユーザ名: ubuntu

なお、攻撃用マシンには、パスワードクラックツールであるHydraがインストールされており、 /usr/share/wordlists/にパスワードリスト rockyou.txt が配置されています。

パスワードクラックには1~3分程度かかります。3分以上かかる場合には、コマンドが間違っている可能性があるので、入力したコマンドを見直してください。

解説

この問題は、他者が構築したLinuxサーバに問題がないか検証する状況を想定した問題です。 実際の構成情報を使って検証することが多いですが、今回はパスワードクラックを試行することで、攻撃者から見た状態を知ることを目的としています。

SSHサーバの認証として何が有効になっているかは、認証失敗時のレスポンスから知ることができます。 問題文にあるように、認証失敗時に表示される Permission denied (publickey,password). から、公開鍵認証とパスワード認証が有効であることがわかります。

今回は、SSHのパスワードクラックを行うツールとしてHydraを使います。

/usr/share/wordlists/に用意されているパスワードリスト rockyou.txtを使って、ユーザ名 ubuntu のパスワードをクラックする場合のコマンドは以下のようになります。

```
hydra -l ubuntu -P /usr/share/wordlists/rockyou.txt ssh://<target1_ip>
```

    -l : ユーザ名
    -P : パスワードの候補が記載されたテキストファイル
    ssh://<target1_ip>: 攻撃対象のプロトコルとIPアドレス

コマンドを実行すると、1分程度でパスワードクラックが成功し、ユーザ名 ubuntu のパスワード princess1 が得られます。

今回のUbuntuサーバでsshdの設定ファイルを確認すると、以下のようになっています。

```
ubuntu@adb9bc28334d:~$ cat /etc/ssh/sshd_config

# Authentication:

#LoginGraceTime 2m
#PermitRootLogin prohibit-password
#StrictModes yes
#MaxAuthTries 6
#MaxSessions 10

#PubkeyAuthentication yes

# To disable tunneled clear text passwords, change to no here!
#PasswordAuthentication yes
#PermitEmptyPasswords no
```

インストール直後の状態では、コメントアウトに書かれている設定がデフォルト設定を表しています。 ここでは、PermitRootLogin （rootユーザのログイン方式） でパスワードログインが禁止され、PubkeyAuthentication （公開鍵認証）は有効となっています。 しかし、PasswordAuthentication （パスワード認証）が有効となっており、rootユーザ以外はパスワードでのログインが可能な状態です。

パスワード認証を無効にするには、まず PasswordAuthentication （古いバージョンでは ChallengeResponseAuthentication も）の行をコメントアウトし、no を指定します。 その後、sshdを再起動することで設定の変更が反映されます。 参考に、パスワード認証が無効の場合の認証失敗時レスポンスを以下に示します。

```
ubuntu@<target1_ip>: Permission denied (publickey).
```
