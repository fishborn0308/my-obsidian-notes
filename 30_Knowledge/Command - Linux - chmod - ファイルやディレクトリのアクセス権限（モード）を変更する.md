# Command - Linux - chmod - ファイルやディレクトリのアクセス権限（モード）を変更する

## 概要

`chmod` は、ファイルやディレクトリの読み取り (r)、書き込み (w)、実行 (x) 権限を変更するためのコマンドです。数値（8進数）による指定と、記号による指定の両方をサポートしています。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `chown` | 所有者やグループを変更する。権限の内容自体は変更しない。 | 所有権の変更 |
| `umask` | ファイル作成時の「デフォルト権限」を制御する。 | 初期権限の設定 |
| `setfacl` | 標準の権限（rwx）よりも複雑な、ユーザー毎の詳細な制御を行う。 | 高度なアクセス制御 |

## よく連携されるコマンド

### シナリオ例: 特定の拡張子を持つファイル権限の一括変更 (運用視点)

* **目的**: ディレクトリ内にある全ての `.sh` ファイルに一括で実行権限を付与する。
* **連携コマンド**: `find`, `xargs`
* **解説**: `find` で対象を抽出し、`chmod` で権限を変更します。
* **コマンド例**:
    ```bash
    # 現在のディレクトリ以下の全シェルスクリプトを実行可能にする
    find . -name "*.sh" -type f -print0 | xargs -0 chmod +x
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐️ `-R`, `--recursive` | ディレクトリ内のファイルやサブディレクトリを再帰的に変更する。 |
| `-v`, `--verbose` | 変更した全てのファイルについて診断メッセージを表示する。 |
| `-c`, `--changes` | 実際に変更があった場合のみ報告する。 |
| **特殊設定 (OSCP等で重要)** | |
| `u+s` (4000) | **SUID (Set User ID)** を設定。実行時に所有者権限で動作する。 |
| `g+s` (2000) | **SGID (Set Group ID)** を設定。 |
| `+t` (1000) | **Sticky Bit** を設定。所有者以外はファイルの削除ができなくなる。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Webサーバ（Nginx/Apache）の公開ディレクトリの適切な権限設定。
* **組み合わせ**: `644` (ファイル), `755` (ディレクトリ)
* **解説**: セキュリティの基本として「ディレクトリは実行(x)が必要、ファイルは不要」という原則を適用します。
* **例**:
    ```bash
    # ディレクトリは 755 (drwxr-xr-x)
    find /var/www/html -type d -exec chmod 755 {} +
    # ファイルは 644 (-rw-r--r--)
    find /var/www/html -type f -exec chmod 644 {} +
    ```

### 2. ブルーチーム視点

* **タスク**: システム内の「誰でも書き込み可能 (World-Writable)」な不適切なファイルの探索と修正。
* **組み合わせ**: `find` との連携による調査
* **解説**: 攻撃者が永続化（Persistence）のために利用しそうな、権限の緩い重要ファイルを見つけ出します。
* **例**:
    ```bash
    # システム全体から誰でも書き込めるファイルを探す
    find / -perm -0002 -type f -ls 2>/dev/null
    ```

### 3. レッドチーム視点

* **タスク**: 権限昇格（Privilege Escalation）のための SUID バイナリの探索。
* **組み合わせ**: `-perm -4000`
* **解説**: 脆弱な SUID バイナリ（例: 意図せず root 権限で動く `awk` や `python`）を探し、root 奪取の糸口にします。
* **例**:
    ```bash
    # SUIDビットが立っているファイルを列挙
    find / -perm -4000 -type f 2>/dev/null
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `chmod: changing permissions of '...': Operation not permitted`
    * **考えられる原因**: ファイルの所有者ではないか、または root 権限が必要なファイルを操作しようとしている。
    * **解決策**: `sudo` を使用する。または `chattr` による不変属性 (`+i`) が付いていないか確認する。

2.  **実行権限を与えたのに実行できない**:
    * **考えられる原因**: マウントオプションで `noexec` が指定されているパーティション（`/tmp` 等によくある）にファイルがある。
    * **解決策**: `mount` コマンドでパーティションの設定を確認する。

## 環境変数と設定ファイル

* `chmod` 自体に設定ファイルはありませんが、新しいファイルが作成される際の権限は環境変数に近い役割を持つ `umask` の値によって決定されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: 不適切な SUID/SGID 設定。
* **悪用シナリオ**: 攻撃者が `sudo` 権限の一部（例: `chmod` のみ）を奪取した場合、`/etc/shadow` を `777` に変更することで、パスワードハッシュを直接読み書きし、パスワードを上書きすることが可能です。

### GTFOBins における利用例

`chmod` が `sudo` で実行可能、あるいは SUID が設定されている場合、特権ファイルを誰でも読めるように変更することで実質的に特権を奪取できます。

* **機能**: `Sudo`, `SUID`
* **コマンド例**:
    ```bash
    # sudo chmod を使って /etc/shadow を全公開にする
    sudo chmod 0777 /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/chmod/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `chmod` などのシステム管理コマンドを一般ユーザーに `sudo` で許可しない。
* **Detection (検知)**: `/etc/`, `/bin/`, `/usr/bin/` などのディレクトリ内での権限変更を `auditd` で常時監視する。

## 注意点・補足

* **数値指定と記号指定**:
    * 数値: `chmod 755 file` (確実かつ高速)
    * 記号: `chmod u+x,g-w file` (既存の権限を維持しつつ一部だけ変更したい場合に便利)
* **OSによる違い**: 一部の商用 UNIX (Solaris等) や特殊なファイルシステムでは、挙動や特殊ビットの扱いが異なる場合があります。
