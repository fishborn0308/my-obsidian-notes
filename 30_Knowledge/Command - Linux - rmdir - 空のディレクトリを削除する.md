# Command - Linux - rmdir - 空のディレクトリを削除する

## 概要

`rmdir` は、指定されたディレクトリを削除します。ただし、そのディレクトリが**完全に空（ファイルも隠しファイルもサブディレクトリもない状態）**である場合にのみ成功します。この制約があるため、重要なデータが含まれている場所を誤って消去してしまう事故を防ぐことができます。

[Image comparing rmdir failure on non-empty directory vs rm -r success]

## 🛠 `rmdir` vs `rm -r`

| 項目 | `rmdir` | `rm -r` |
| :--- | :--- | :--- |
| **安全性** | **非常に高い。** 中身があれば拒否される。 | 低い。中身ごと強制削除する。 |
| **主な用途** | 不要な空フォルダの整理。 | プロジェクトやデータの丸ごと削除。 |
| **エラー時の挙動** | 「Directory not empty」と表示され止まる。 | 止まらずにすべて消し去る。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-p`, `--parents` | **最頻用。** 子ディレクトリを消した後、その親ディレクトリも空になれば一緒に消す。 |
| ⭐️ `-v`, `--verbose` | 削除したディレクトリごとにメッセージを表示する。 |
| `--ignore-fail-on-non-empty` | 中身があって削除できなかった場合でも、エラーコード（終了ステータス）を返さない。 |

## コマンド例

```bash
# 単一の空ディレクトリを削除
rmdir empty_folder

# 階層構造を一気に、かつ安全に削除
# (a/b/c を消し、b も a も空になればそれらも消す)
rmdir -p a/b/c
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: スクリプトによる一時ディレクトリのクリーンアップ。
- **解説**: 処理の途中で作成した作業用フォルダを消す際、`rm -r` ではなく `rmdir` を使うことで、「予期せぬログや成果物が残っている場合は消さずに残す」という安全装置として機能させます。

### 2. ブルーチーム視点

- **タスク**: システム内の不自然な「空のディレクトリ」の調査。
- **解説**: 攻撃者がツールを置こうとして失敗した跡や、情報の隠し場所として作成したディレクトリを特定します。`rmdir` で消えるかどうかを試すことで、その場所に現在進行形でファイルが隠されていないかを（破壊せずに）確認する一つの指標になります。

### 3. レッドチーム視点

- **タスク**: ステルス性の維持と痕跡消去。
- **解説**: 侵入時に作成した一時的なディレクトリ構造を消去します。`rm -r` で誤って重要なシステムファイルを巻き込んでエラーログを残すリスクを避け、自分が作った「空のフォルダ」だけをピンポイントで、かつ静かに消し去るために使用します。

## 💡 プロの知恵：`-p` オプションの賢い使い方

深い階層 `dir1/dir2/dir3` を作った後、`rmdir -p dir1/dir2/dir3` を実行するのは、`mkdir -p` の対になる動作です。自分で作った構造を、周囲に影響を与えず「来た時よりも美しく」片付けるための紳士的な作法です。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **ディレクトリ・パーミッション**: `rmdir` を実行するには、そのディレクトリ自体に対する書き込み権限ではなく、**その親ディレクトリに対する書き込み権限**が必要です。

### 対応策・緩和策

- **Sticky Bit**: `/tmp` などの共有ディレクトリでは Sticky Bit が設定されており、自分に所有権がないディレクトリは（たとえ空であっても）他人が `rmdir` で消せないよう保護されています。
