# Command - Linux - od - ファイルを様々な形式（8進数等）でダンプする

## 概要

`od` は、指定されたファイルの内容を 8 進数（デフォルト）やその他の形式で出力します。主にバイナリファイルの内容を確認したり、テキストファイル内の不可視文字（改行コードやタブ）を特定したりするために使用されます。

## 🛠 類似・関連コマンド

| コマンド | 特徴 | 推奨 |
| :--- | :--- | :--- |
| **`od`** | POSIX 標準。8進数が基本だが多機能。 | 伝統的な環境での調査 |
| `hexdump` | 16進数での表示に特化。見やすい。 | クイックな確認 |
| `xxd` | 16進数表示に加え、バイナリへの逆変換も可能。 | **バイナリ編集・修正** |
| `strings` | バイナリから「読める文字」だけを抜き出す。 | 文字列の抽出 |

## 主要なオプション

`od` の真価は `-t` (タイプ指定) オプションの組み合わせにあります。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-c` | **最頻用。** 文字として表示（バックスラッシュ記法）。改行コードの確認に最適。 |
| ⭐️ `-t x1` | 1バイトずつ16進数で表示。 |
| ⭐️ `-t u4` | 4バイトずつ符号なし10進数で表示。 |
| ⭐️ `-A <base>` | アドレス表示の基数（d:10進, o:8進, x:16進, n:なし）を指定。 |
| `-N <bytes>` | 最初の指定バイト数だけをダンプする。 |
| `-j <bytes>` | 指定バイト数だけスキップしてから開始する。 |

## よく使われるコンボ

```bash
# 16進数と文字を並べて表示（最も「中身」がわかりやすい）
od -t x1z -A x <file>
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 改行コード（LF vs CRLF）や BOM の混入確認。
- **解説**: スクリプトが動かない原因が「Windows で編集した際の改行コード」である場合、`od -c` で見れば一発で `\r` の存在に気づけます。
- **例**:

    Bash

    ```
    # ファイルの末尾に不審な文字がないか確認
    od -c script.sh | tail
    ```

### 2. ブルーチーム視点

- **タスク**: マジックナンバーによるファイルタイプの特定。
- **解説**: 拡張子が偽装されたマルウェア（例: `.jpg` を装った `ELF` 実行ファイル）を、先頭数バイトのヘッダー情報から正しく識別します。
- **例**:

    Bash

    ```
    # 先頭4バイトを16進数で確認
    od -N 4 -t x1 image.jpg
    # 7f 45 4c 46 と出れば、それは ELF バイナリ（Linux実行ファイル）です
    ```

### 3. レッドチーム視点

- **タスク**: バイナリ内のハードコードされたデータの探索。
- **解説**: 実行ファイルの中に隠された暗号化キーや、難読化された文字列のパターンを 16 進数レベルで解析し、リバースエンジニアリングの足がかりにします。

## 注意点・補足

- **エンディアンの問題**: 2バイトや4バイト単位でダンプする際（`-t x2` や `-t x4`）、CPU のアーキテクチャによってバイト順が逆転して見えることがあります。生データを確認したい場合は **`-t x1`** (1バイト単位) を使うのが最も安全です。
- **重複行の省略**: `od` は同じデータが続く場合、出力を `*` で省略します。すべて見たい場合は `-v` オプションを付けます。

## セキュリティに関する考慮事項

- **機密情報の露出**: `od` はファイルパーミッションさえあれば、中身を一切の解釈なしに表示します。メモリダンプファイルやスワップファイルを `od` で覗くと、平文のパスワードなどが露出していることがあります。
