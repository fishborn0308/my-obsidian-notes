# `diff` - 2つのファイルやディレクトリの差分を表示する

## 概要

`diff` (difference) は、2つのファイルの内容を一行ずつ比較し、その差分を出力するコマンドです。また、2つのディレクトリを再帰的に比較し、どのファイルが異なるか、あるいは一方にしか存在しないかをリストアップすることもできます。`diff` の出力は、単に変更箇所を確認するだけでなく、`patch` コマンドへの入力として使い、あるファイルへの変更を別のファイルに適用するためにも利用されます。

(出自: `diffutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`cmp`** | ファイルを**バイト単位**で比較し、最初の差異だけを報告するため高速です。ファイルが同一かどうかの確認に適しています。 | - |
| **`git diff`** | `Git` に統合された差分表示ツール。コミット間、ブランチ間など、Gitリポジトリの文脈に沿った差分を表示します。 | `git diff` (Git管理下) |
| **`vimdiff`** | 2つのファイルを**左右に並べて**差分をインタラクティブに表示・編集できます。 | `vimdiff` (対話的編集) |

## よく連携されるコマンド

`diff` の出力は長くなることが多いため、`less` などのページャにパイプして閲覧するのが一般的です。

### シナリオ例: 設定変更の確認 (インフラ構築・運用視点)

* **目的**: 編集した設定ファイルが、バックアップと比べてどこが違うのかを分かりやすく確認する。
* **連携コマンド**: `less`
* **解説**: `diff -u` でユニファイド形式の差分を生成し、その出力を `less` に渡すことで、長い差分でもスクロールしながら快適に確認できます。
* **コマンド例**:

    ```bash
    # Nginxの設定変更箇所を確認する
    diff -u /etc/nginx/nginx.conf.bak /etc/nginx/nginx.conf | less
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **出力形式** | |
| ⭐ `-u`, `-U <N>` | **ユニファイド形式**で差分を出力します。`<N>` でコンテキスト（変更前後の行）の表示行数を指定できます。 |
| `-c` | コンテキスト形式で差分を出力します。 |
| **動作制御** | |
| ⭐ `-r`, `--recursive` | ディレクトリを**再帰的に**比較します。 |
| `-q`, `--brief` | 差分の詳細を表示せず、ファイルが異なるかどうかだけを報告します。 |
| `-i`, `--ignore-case` | 大文字と小文字の違いを無視します。 |
| `-w`, `--ignore-all-space` | 全ての空白文字を無視して比較します。 |
| `-b` | 空白の数の違いを無視します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルのバックアップと現在のファイルとの差分を確認する。
* **組み合わせ**: `diff -u <file.conf.bak> <file.conf>`
* **解説**: ユニファイド形式 (`-u`) で差分を確認し、どの変更が問題を引き起こしたのかを特定する、基本的なトラブルシューティング手法です。
* **例**:

    ```bash
    # Apacheの設定を変更後、動作がおかしくなった場合に差分を確認
    diff -u /etc/httpd/conf/httpd.conf.old /etc/httpd/conf/httpd.conf
    ```

### 2. ブルーチーム視点

* **タスク**: Webサイトの改ざん箇所を特定する。
* **組み合わせ**: `diff -qr <known_good_dir> <suspected_dir>`
* **解説**: `-q` (差分サマリー) と `-r` (再帰) を使い、正常なバックアップと現在稼働中のWebサーバーのドキュメントルートを比較します。これにより、攻撃者によって追加・改ざんされたファイルを素早く特定できます。これは**ファイル整合性監視 (FIM)** の手動での実践です。
* **例**:

    ```bash
    # 正常なWordPressのソースと、改ざんが疑われるディレクトリの差分ファイルのみリストアップ
    diff -qr /path/to/wordpress_clean/ /var/www/html/
    ```

### 3. レッドチーム視点

* **タスク**: 自身が仕込んだバックドアが、元のスクリプトとどう違うかを確認する。
* **組み合わせ**: `diff -u <original_script> <modified_script_with_backdoor>`
* **解説**: 侵入したシステムに存在する正規のスクリプトに、リバースシェルを起動するコードなどを一行追加します。その後、`diff` を使って元のスクリプトとの差分を確認し、自分の変更が最小限で検知されにくいものであるかを確認します。
* **例**:

    ```bash
    # バックアップスクリプトに悪意のあるコードを仕込んだ後、差分を確認
    diff -u backup.sh.orig backup.sh
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `diff: <directory_name>: Is a directory`
    * **考えられる原因**: ディレクトリを比較しようとしましたが、`-r` (recursive) オプションを指定していません。
    * **解決策**: `diff -r dir1 dir2` のように `-r` オプションを追加してください。

2. **エラーメッセージ例 2**: `Binary files <file1> and <file2> differ`
    * **考えられる原因**: テキストファイルではなく、バイナリファイル（実行ファイル、画像など）を比較しようとしました。
    * **解決策**: ファイルが同一かどうかだけを知りたい場合は `cmp` コマンドを使用します。

## 環境変数と設定ファイル

* `diff` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `diff` 自体は読み取り専用のツールであり、脆弱性を持つ可能性は低いです。
* **悪用シナリオ**: **パッチファイル経由の情報漏洩**。`/etc/shadow` のような機密情報を含むファイルと、別のファイルとの差分を取ったパッチファイルを作成した場合、そのパッチファイル内には機密情報が平文で含まれることになります。このパッチファイルを誤って公開すると、情報漏洩に繋がります。

### GTFOBins / LOLBAS における利用例

`diff` はファイルの比較と表示を行うツールであり、コード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 機密情報を含むファイルの差分をパッチとして共有しない。コードレビューの際には `diff` やパッチファイルに意図せず埋め込まれた機密情報がないかを確認する。
* **Detection (検知)**: `diff` は、**ファイル整合性監視 (FIM)** のコンセプトを理解し、手動で実践するための強力なツールです。`AIDE` や `Tripwire` といったFIMツールは、定期的にファイルのハッシュ値を計算し、以前の状態との「差分」を検知することでシステムの変更を警告します。

## 注意点・補足

* **引数の順序が重要**: `diff old_file new_file` は、**`old_file` を `new_file` に変更するための手順**を出力します。`-` (マイナス) は `old_file` から削除すべき行、`+` (プラス) は `old_file` に追加すべき行を意味します。
