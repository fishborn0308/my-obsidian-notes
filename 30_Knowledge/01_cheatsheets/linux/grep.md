# `grep` - テキストデータから正規表現に一致する行を検索する

## 概要

`grep` (Global Regular Expression Print) は、ファイルや標準入力から、指定した**パターン**（通常は正規表現）に一致する行を検索し、表示するための非常に強力なコマンドラインユーティリティです。ログファイルの分析、設定ファイル内の特定の設定値の検索、コマンド出力のフィルタリングなど、テキストデータを扱うあらゆる場面で不可欠なツールです。

(出自: `grep` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`find`** | **ファイルそのもの**を名前、サイズなどの**属性**で検索します。`grep` は**ファイルの中身**を**内容**で検索します。 | - |
| **`sed`, `awk`** | `grep` は一致する行を**見つけるだけ**ですが、`sed` は**置換**、`awk` は**フィールド単位での加工や集計**といった、より高度な**編集・処理**が可能です。 | - |
| **`ack`, `ag`, `rg`** | `grep` のモダンな代替ツール。デフォルトで再帰的に検索し、`.gitignore` を尊重するなど、特にソースコードの検索に最適化されており高速です。 | `rg` (ripgrep) |

## よく連携されるコマンド

`grep` の真価は、他のコマンドの出力をパイプ (`|`) で受け取り、リアルタイムでフィルタリングする際に最も発揮されます。

### シナリオ例: 実行中のプロセスから特定のものを絞り込む (インフラ構築・運用視点)

* **目的**: システムで実行中の全プロセスの中から、`nginx` という名前のプロセスだけを表示する。
* **連携コマンド**: `ps`, `grep`
* **解説**: `ps aux` で全プロセスのリストを出力し、その結果をパイプで `grep` に渡して `nginx` という文字列を含む行だけをフィルタリングします。
* **コマンド例**:

    ```bash
    # psコマンドの出力からnginxプロセスだけをフィルタリング
    ps aux | grep 'nginx'
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **検索動作** | |
| ⭐ `-i`, `--ignore-case` | 大文字と小文字を区別せずに検索します。 |
| ⭐ `-v`, `--invert-match` | パターンに**一致しない**行を表示します。 |
| ⭐ `-r`, `-R`, `--recursive` | ディレクトリ内を再帰的に検索します。 |
| `-E`, `--extended-regexp` | 拡張正規表現 (`+`, `?`, `\|`,`()`など) を使用できます (`egrep` と同義)。 |
| `-F`, `--fixed-strings` | パターンを正規表現ではなく、ただの固定文字列として扱います (`fgrep` と同義)。 |
| **出力制御** | |
| ⭐ `-l`, `--files-with-matches` | パターンに一致した**ファイル名のみ**を表示します。 |
| `-c`, `--count` | パターンに一致した**行数**のみを表示します。 |
| `-n`, `--line-number` | 一致した行の行番号を付けて表示します。 |
| `-o`, `--only-matching` | 行全体ではなく、**パターンに一致した部分だけ**を抜き出して表示します。 |
| ⭐ `-A <N>`, `-B <N>`, `-C <N>` | 一致した行の**後 (After)**、**前 (Before)**、**前後 (Context)** のN行も併せて表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログファイルから特定のエラーメッセージを含む行を抽出する。
* **組み合わせ**: `grep <pattern> <logfile>`
* **解説**: アプリケーションの動作がおかしい際に、ログファイルから "ERROR" や "FATAL" といった文字列を検索し、問題の原因を特定します。
* **例**:

    ```bash
    # Nginxのエラーログから "crit" (Critical) レベルのエラーを検索
    grep "crit" /var/log/nginx/error.log
    ```

### 2. ブルーチーム視点

* **タスク**: 全てのログファイルから攻撃者のIPアドレス (IOC) を横断的に検索する。
* **組み合わせ**: `grep -r <IOC> /var/log/`
* **解説**: 攻撃元IPアドレスが `198.51.100.10` であると特定された場合、このIPがWebサーバーのアクセスログ、SSHの認証ログなど、システムのどこに記録されているかを再帰的に調査し、攻撃の全容を解明します。
* **例**:

    ```bash
    # /var/log ディレクトリ配下から、特定のIPアドレスを再帰的に検索
    sudo grep -r "198.51.100.10" /var/log/
    ```

### 3. レッドチーム視点

* **タスク**: システム内のファイルから、ハードコードされたパスワードなどの機密情報を探し出す。
* **組み合わせ**: `grep -ri 'password' <directories> 2>/dev/null`
* **解説**: 侵入したサーバー上で、設定ファイル、ソースコード、ユーザーのメモなどにハードコードされたパスワードやAPIキー、接続文字列などを、再帰的かつ大文字小文字を無視して検索します。
* **例**:

    ```bash
    # 主要なディレクトリから "password" という文字列を含む箇所を探す
    grep -ri 'password' /etc /var/www /home 2>/dev/null
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`grep: (standard input): Binary file matches`**
    * **考えられる原因**: 検索対象のファイルがテキストファイルではなく、バイナリファイルです。`grep` は画面が文字化けするのを防ぐため、バイナリファイルの中身の表示をデフォルトで停止します。
    * **解決策**: バイナリファイルの中身も強制的にテキストとして扱いたい場合は `-a` または `--text` オプションを使用します。`strings <binary> | grep <pattern>` と組み合わせるのも有効です。

2. **現象**: **`*` や `.` などの記号を検索しようとすると、意図しない結果になる。**
    * **考えられる原因**: `grep` はデフォルトでパターンを正規表現として解釈するため、メタ文字はそのままでは検索できません。
    * **解決策**: `-F` オプションを使うか、バックスラッシュ `\` でエスケープします (`grep '\*\.log'`)。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 影響を与える環境変数

* **`GREP_OPTIONS`**: `grep` のデフォルトオプションを指定できますが、現在は非推奨です。エイリアス (`alias`) の使用が推奨されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `grep` 自体は読み取り専用ツールですが、非常に稀なケースとして、SUIDビットが設定されている場合、一般ユーザーがrootしか読めないファイル (`/etc/shadow`など) の内容を読み取ることが可能になります。
* **悪用シナリオ**: **情報漏洩**。`grep` は、システム内に平文で不用意に保存されたパスワードやAPIキーなどの機密情報を発見するための強力なツールです。

### GTFOBins / LOLBAS における利用例

* **機能**: **File Read**
* **解説**: `sudo` 権限で `grep` を実行できる場合、システム上のあらゆるファイルを読み取ることができます。
* **参照**: `https://gtfobins.github.io/gtfobins/grep/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードやAPIキーを設定ファイルやソースコードにハードコードしない。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `auditd` を使い、機密ファイルへの `grep` によるアクセスを監視する。

## 注意点・補足

* **正規表現**: `grep` の真価は正規表現と組み合わせることで発揮されます。`-E` オプションで拡張正規表現を使うと、より複雑なパターンを簡潔に記述できます。
