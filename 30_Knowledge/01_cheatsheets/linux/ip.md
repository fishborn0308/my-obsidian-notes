# `ip` - Linuxネットワーク設定の管理 (iproute2)

## 概要

`ip` コマンドは、**iproute2** パッケージに含まれる、モダンなLinuxシステムにおけるネットワーク設定を管理するための中心的なコマンドスイートです。IPアドレス、ルーティングテーブル、ネットワークインターフェイスの状態など、ネットワークに関するほぼ全ての設定をこのコマンド1つで行うことができます。

`ip` コマンドは、旧来の `net-tools` パッケージに含まれていた `ifconfig`, `route`, `arp`といった複数のコマンドの機能を、より高機能で統一的なインターフェイスで置き換えるものです。

(出自: `iproute2` パッケージに含まれる)

## 類似コマンド（旧世代）との対応

`ip` コマンドは、オブジェクト（何 `what`）とアクション（どうする `how`）を組み合わせて指定します。

| 新コマンド (`iproute2`) | 旧コマンド (`net-tools`) | 目的 |
| :--- | :--- | :--- |
| ⭐ **`ip addr`** | `ifconfig` | IPアドレスとネットワークインターフェイスの表示・管理 |
| ⭐ **`ip link`** | `ifconfig` | ネットワークインターフェイス自体の状態（UP/DOWNなど）の管理 |
| ⭐ **`ip route`** | `route` | ルーティングテーブルの表示・管理 |
| **`ip neigh`** | `arp` | ARPテーブル（近隣キャッシュ）の表示・管理 |
| **`ss -tulpn`** | `netstat -tulpn` | ネットワークソケットの状態表示 (`ss`もiproute2の一部) |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーの基本的なネットワーク情報を確認する。
* **組み合わせ**: `ip addr show`, `ip route show`
* **解説**: これらのコマンドは、サーバーにログインした際に「自分はどのIPアドレスを持ち、どこに繋がり、どうやって外に出るのか」を確認するための最も基本的な操作です。
* **例**:

    ```bash
    # 全てのインターフェースのIPアドレスを確認 (省略形: ip a)
    ip addr show

    # デフォルトゲートウェイを含むルーティングテーブルを確認 (省略形: ip r)
    ip route show
    ```

### 2. ブルーチーム視点

* **タスク**: 不審な通信の調査中に、ARPスプーフィングの兆候がないかを確認する。
* **組み合わせ**: `ip neigh show`
* **解説**: ARPテーブル（IPアドレスとMACアドレスの対応表）を表示します。もし、ゲートウェイのIPアドレスに対して、本来とは異なる不審なMACアドレスが登録されている場合、ARPスプーフィングによる中間者攻撃を受けている可能性があります。
* **例**:

    ```bash
    # ARPテーブルを監視 (省略形: ip n)
    watch ip neigh show
    ```

### 3. レッドチーム視点

* **タスク**: 侵入したホストのネットワーク構成を偵察し、次の攻撃対象を探す。
* **組み合わせ**: `ip addr`, `ip route`
* **解説**: 攻撃者は侵入後、まず `ip addr` を実行して、ホストがどのようなネットワークセグメントに接続しているかを把握します。次に `ip route` でゲートウェイと他の内部ネットワークへの経路を確認し、ラテラルムーブメント（横展開）の標的を探します。
* **例**:

    ```bash
    # 接続されているネットワークセグメントをリストアップ
    ip addr | grep "inet "
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `RTNETLINK answers: Operation not permitted`
    * **考えられる原因**: ネットワーク設定の変更（`add`, `del`, `set`など）を一般ユーザー権限で実行しようとしました。
    * **解決策**: コマンドの前に `sudo` を付けて実行してください。

## 環境変数と設定ファイル

* `ip` コマンドの動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ip` コマンド自体ではなく、それが管理するネットワークスタックの設定不備が悪用されます。
* **悪用シナリオ**: 攻撃者がroot権限を奪取した場合、`ip` コマンドを使ってルーティングテーブルを改ざんし通信を盗聴したり、MACアドレスを偽装したり、新しい仮想インターフェイスを作成して活動を隠蔽したりします。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**, **File Read/Write**
* **解説**: もしユーザーが `sudo ip` を実行できる権限を持っている場合、`-f` オプションでスクリプトファイルを読み込ませる機能などを悪用して、任意のコマンドを実行したり、ファイルを読み書きしたりすることが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/ip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、一般ユーザーに `ip` コマンドの `sudo` 権限を安易に与えない。
* **Detection (検知)**: `auditd` を使って、`netlink` 経由でのネットワーク構成の変更を監視する。定期的に `ip addr` や `ip route` の出力を取得し、ベースラインと比較して意図しない変更がないかを確認する。

## 注意点・補足

* **設定の永続化**: `ip` コマンドで行った設定変更は、デフォルトでは**OSを再起動すると失われます**。設定を永続化させるには、各ディストリビューションのネットワーク管理ツール（`systemd-networkd`, `NetworkManager`など）と、その設定ファイルを編集する必要があります。
* **省略形**: `ip` コマンドの多くのオブジェクトやサブコマンドは、一意に定まる範囲で省略できます。（例: `ip a`, `ip r`, `ip n`, `ip l`）
