# `nano` - シンプルで直感的なテキストエディタ

## 概要

`nano` は、使いやすさを重視したシンプルなコマンドラインのテキストエディタです。`vi` のような「モード」の概念がなく、起動するとすぐに文字を入力できます。画面下部に主要な操作コマンドが常に表示されているため、初心者でも直感的に扱うことができます。簡単な設定ファイルの編集やメモ書きなど、手早くテキストを編集したい場面で非常に便利です。多くのLinuxディストリビューションにデフォルトでインストールされています。

(出自: `nano` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| ⭐ **`vi` / `vim`** | `vi` は「モード」を切り替えて操作する高機能エディタで、習熟すれば非常に高速な編集が可能です。`nano` はモードレスで学習コストが低いです。 | `vim` (高機能・高速) |
| **`pico`** | `nano` はもともと、`pico` というエディタのクローンとして開発されました。`nano` の方が高機能で、現在では一般的です。 | `nano` |

## よく連携されるコマンド

`nano` は、環境変数 `EDITOR` に設定することで、`crontab -e` のような他のコマンドから呼び出されるエディタとして連携します。

### シナリオ例: cronジョブを安全に編集する (インフラ構築・運用視点)

* **目的**: `vi` に不慣れなユーザーが、cronジョブの登録や編集を、使い慣れた `nano` で安全かつ確実に行う。
* **連携コマンド**: `crontab`, `export`
* **解説**: 環境変数 `EDITOR` を `nano` に設定しておくことで、`crontab -e` を実行した際に `nano` が起動するようになります。
* **コマンド例**:

    ```bash
    # .bashrc などに追記しておくと永続化される
    export EDITOR=nano

    # crontab -e を実行するとnanoが起動する
    crontab -e
    ```

## オプション説明 (`nano` 起動時)

| オプション | 説明 |
| :--- | :--- |
| **ファイル操作** | |
| ⭐ `<filename>` | 編集したいファイル名を指定して開きます。 |
| **表示制御** | |
| ⭐ `-l`, `--linenumbers` | 画面の左端に行番号を表示します。 |
| `-v`, `--view` | ファイルを読み取り専用モードで開きます。誤編集を防ぎます。 |

### エディタ内の主要なコマンド

`nano` の操作は `Ctrl` キー（表示上は `^`）を使ったショートカットで行います。

| ショートカット | 機能 |
| :--- | :--- |
| ⭐ `Ctrl + X` | **終了 (Exit)** |
| ⭐ `Ctrl + O` | **保存 (Write Out)** |
| ⭐ `Ctrl + W` | **検索 (Where is)** |
| `Ctrl + G` | ヘルプ |
| `Ctrl + K` | 1行切り取り (Cut) |
| `Ctrl + U` | 貼り付け (Uncut) |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: シェルスクリプトのエラー箇所を特定し、修正する。
* **組み合わせ**: `nano -l <script.sh>`
* **解説**: `-l` オプションで行番号を表示させ、「`script.sh: line 25: ...`」のようなエラーメッセージに対応する行を素早く見つけます。
* **例**:

    ```bash
    # 行番号付きでスクリプトを開く
    nano -l /usr/local/bin/backup.sh
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠ファイルを誤って変更するリスクなしに調査する。
* **組み合わせ**: `nano -v <suspicious_file>`
* **解説**: `-v` (view) オプションを使い、ファイルを読み取り専用モードで開きます。これにより、攻撃者が作成したファイルやログファイルを調査する際に、証拠としての完全性を損なうリスクを防ぎます。
* **例**:

    ```bash
    # 不審なPHPファイルを安全に閲覧する
    nano -v /var/www/html/uploads/shell.php
    ```

### 3. レッドチーム視点

* **タスク**: DNSクエリを偽装するために `hosts` ファイルを編集する。
* **組み合わせ**: `nano /etc/hosts`
* **解説**: 侵入したサーバーで、ラテラルムーブメントを行う際に、標的のドメイン名を攻撃者が用意したサーバーのIPアドレスに向ける（名前解決を汚染する）ために使用します。
* **例**:

    ```bash
    # 権限昇格後、hostsファイルを編集
    sudo nano /etc/hosts

    # ファイルに追記:
    # 10.0.0.99 internal-portal.example.com
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `[ Error writing /etc/nginx/nginx.conf: Permission denied ]`
    * **考えられる原因**: ファイルを保存しようとしましたが、そのファイルまたはディレクトリへの書き込み権限がありません。
    * **解決策**: 一度 `Ctrl+X` で終了を選択し、保存せずに (`N`) エディタを抜けます。その後、`sudo nano <filename>` のように `sudo` を付けて再度コマンドを実行します。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 影響を与える環境変数

* ⭐ **`EDITOR`**, `VISUAL`: `crontab -e` や `git commit` など、他のプログラムがエディタを呼び出す際に参照されます。

### 関連する設定ファイル

* **`/etc/nanorc`**: システム全体に適用される `nano` のグローバルな設定ファイル。
* **`~/.nanorc`**: ユーザーごとの個人設定ファイル。`set linenumbers` や `include "/usr/share/nano/*.nanorc"` などを記述してカスタマイズします。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `nano` 自体にも過去に脆弱性（バッファオーバーフローなど）が報告されたことがあります。
* **悪用シナリオ**: 攻撃者は `vi` と同様に、侵入後のサーバー上で設定ファイルを改ざんしたり、攻撃用のスクリプトを作成したりするために `nano` を利用します。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo nano` を実行できる権限を持っている場合、`nano` のファイル読み込み機能やコマンド実行機能を悪用して、任意のファイルを読み書きしたり、root権限を奪取したりできます。
* **参照**: `https://gtfobins.github.io/gtfobins/nano/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定で、ユーザーに `nano` のようなエディタの `sudo` 権限を安易に与えない。特定のファイルのみを編集させたい場合は、`sudoedit` (または `sudo -e`) を使うように設定する方が安全です。
* **Detection (検知)**: `auditd` で `nano` からの不審なファイル書き込みやプロセス生成がないか監視する。

## 注意点・補足

* **設定のカスタマイズ**: `nano` の使いやすさは `~/.nanorc` ファイルのカスタマイズに大きく依存します。特にシンタックスハイライト (`include ...`) と行番号表示 (`set linenumbers`) は、作業効率を大幅に向上させるため、設定しておくことを強く推奨します。
