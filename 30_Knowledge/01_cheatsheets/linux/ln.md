# `ln` - ファイルへのリンクを作成する

## 概要

`ln` (link) は、ファイルやディレクトリへの**リンク**を作成するためのコマンドです。リンクは、既存のファイルへの別名やショートカットのように機能し、主に2つの種類があります。

* **ハードリンク (Hard Link)**: (デフォルト) ファイルの実体であるiノードへの直接的な参照。
* **シンボリックリンク (Symbolic Link / Soft Link)**: ファイルの**パス名**を指し示す特殊なファイル。

シンボリックリンクの方が、ディレクトリや異なるファイルシステムをまたいで作成できるため、柔軟性が高く一般的に利用されます。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **Windows ショートカット** | シンボリックリンクはWindowsのショートカットに似ていますが、より低レベルで動作し、多くのアプリケーションから通常のファイル/ディレクトリとして透過的に扱われます。 | - |
| **`cp`** | `cp` はファイルの**実体を複製**します。`ln` は既存のファイルへの**参照（ポインタ）**を作成するだけであり、ディスク容量をほとんど消費しません。 | - |

## よく連携されるコマンド

`ln` は、設定ファイルを特定の場所から参照させる際に、`rm` や `mv` と連携して使われることがあります。

### シナリオ例: アプリケーションのバージョンを切り替える (インフラ構築・運用視点)

* **目的**: 複数のバージョンのアプリケーションをサーバーに配置し、ダウンタイムなしで（または最小限で）使用するバージョンを切り替える。
* **連携コマンド**: `rm`, `ln`
* **解説**: `/opt/app/v1.0` と `/opt/app/v1.1` のようにバージョンごとにディレクトリを配置し、実際に参照されるパス (`/opt/app/current`) をシンボリックリンクで作成します。アップデート時は、リンクを一度削除して新しいバージョンに張り替えるだけで済みます。
* **コマンド例**:

    ```bash
    # このコマンドシーケンスで、アプリケーションの参照バージョンをv1.1に切り替える
    # 1. 古いシンボリックリンクを削除
    sudo rm /opt/app/current

    # 2. 新しいバージョンへのシンボリックリンクを作成
    sudo ln -s /opt/app/v1.1 /opt/app/current
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **リンクタイプ** | |
| ⭐ `-s`, `--symbolic` | **シンボリックリンク**を作成します。これを付けないとハードリンクが作成されます。 |
| **動作制御** | |
| ⭐ `-f`, `--force` | リンク作成先に同名のファイルやリンクが既に存在する場合、それを強制的に上書きします。 |
| `-v`, `--verbose` | リンクが作成される際に、その詳細を表示します。 |
| `-n`, `--no-dereference` | リンク作成先が「ディレクトリへのシンボリックリンク」である場合に、そのディレクトリの中にリンクを作成するのではなく、リンク自体を上書きします。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Nginxなどで、利用可能な設定 (`sites-available`) から、有効にしたい設定へシンボリックリンクを作成する。
* **組み合わせ**: `sudo ln -s <target_path> <link_path>`
* **解説**: 実際に有効化したい設定だけを `sites-enabled` ディレクトリにリンクとして配置するのは、設定管理における一般的なプラクティスです。
* **例**:

    ```bash
    # /etc/nginx/sites-availableにある設定ファイルを有効化する
    sudo ln -s /etc/nginx/sites-available/my-app.conf /etc/nginx/sites-enabled/my-app.conf
    ```

### 2. ブルーチーム視点

* **タスク**: システム上の全てのシンボリックリンクをリストアップし、不審なものがないか監査する。
* **組み合わせ**: `find / -type l -ls 2>/dev/null`
* **解説**: `/tmp` 配下にある攻撃者のスクリプトへ、正規のパス (`/usr/local/bin`など) からシンボリックリンクが張られている場合、それは永続化やバックドアの兆候である可能性があります。
* **例**:

    ```bash
    # システム上の全シンボリックリンクを調査
    sudo find / -type l -ls 2>/dev/null | less
    ```

### 3. レッドチーム視点

* **タスク**: **情報窃取**のため、機密ファイルへのシンボリックリンクをWebルートに作成する。
* **組み合わせ**: `ln -s <sensitive_file> <web_root_path>`
* **解説**: 攻撃者がWebサーバーのドキュメントルートへの書き込み権限を得た場合、`/etc/shadow` などへのシンボリックリンクを作成します。Webサーバーの設定によっては、これにより外部から機密ファイルをダウンロードできてしまう可能性があります。
* **例**:

    ```bash
    # Webルートに /etc/shadow へのシンボリックリンクを作成
    ln -s /etc/shadow /var/www/html/shadow.txt
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `ln: failed to create hard link './link' => './target': Invalid cross-device link`
    * **考えられる原因**: 異なるファイルシステム（パーティションやディスク）をまたいで**ハードリンク**を作成しようとしました。
    * **解決策**: ハードリンクは同一ファイルシステム内でしか作成できません。**シンボリックリンク (`ln -s`)** を使用してください。

## 環境変数と設定ファイル

* `ln` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **シンボリックリンク競合 (Symlink Race Condition)**。
* **悪用シナリオ**: 攻撃者は `/tmp` のような誰でも書き込める場所に、機密ファイル (`/etc/passwd`など) を指すシンボリックリンクを作成します。もし、`root` 権限で動作する不安全なプログラムが、そのシンボリックリンクを一時ファイルだと思い込んで書き込みを行うと、実際には `/etc/passwd` が書き換えられシステムが乗っ取られる可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **File Write**, **Privilege Escalation**
* **解説**: `sudo` 権限で `ln` を実行できる場合、機密ファイルへのハードリンクを作成したり、`sudo` が他のSUIDバイナリを実行する際のパスをハイジャックしたりすることで、権限昇格に繋がる可能性があります。
* **参照**: `https://gtfobins.github.io/gtfobins/ln/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 一時ファイルを作成する際は、`mktemp` コマンドを使用し、予測可能なファイル名を使わない。`/tmp` パーティションに `nosymfollow` オプションを設定する（ただし互換性に注意）。
* **Detection (検知)**: `auditd` を使って、重要なファイルに対するシンボリックリンクの作成や、`link` システムコールの実行を監視する。

## 注意点・補足

* **パスの指定**: シンボリックリンクを作成する際、リンク先のパスは**絶対パスで指定する**のが一般的です。相対パスで指定すると、リンクを置く場所によってリンク先が変わってしまい、意図しない挙動やリンク切れの原因となります。
