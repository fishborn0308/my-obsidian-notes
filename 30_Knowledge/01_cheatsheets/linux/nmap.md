# `nmap` - ネットワーク探索とセキュリティ監査ツール

## 概要

`nmap` (Network Mapper) は、ネットワーク上のホストを発見し、そのホストが提供しているサービス（オープンしているポート）、稼働しているOS、利用しているファイアウォールの種類などを特定するための、非常に強力なオープンソースツールです。元々はポートスキャナとして開発されましたが、現在では **Nmap Scripting Engine (NSE)** による拡張で、脆弱性診断や高度なネットワーク探索も可能になっています。インフラ管理者からセキュリティ専門家まで、ネットワークを調査する上で不可欠なツールです。

(出自: `nmap` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`netcat` (`nc`)** | `nc -zv` で単一ポートの接続テストは可能ですが、`nmap` は多数のポートを高速にスキャンし、サービスやOSを特定するなど、ネットワークスキャンに特化した専門的な機能を持ちます。 | `nmap` (スキャン) |
| **`masscan`** | インターネット全体のような、非常に大規模なネットワーク範囲を**極めて高速に**スキャンすることに特化しています。`nmap` は特定のターゲットに対し、より**詳細で正確な**情報を得ることに重点を置いています。 | `masscan` (大規模) |
| **`zenmap`** | `nmap` の公式**グラフィカルユーザインタフェース (GUI)**版。コマンド操作に不慣れなユーザーでも、スキャン結果を視覚的に分かりやすく表示・管理できます。 | - |

## よく連携されるコマンド

`nmap` の `-oG` (Grepable) 形式の出力は、`grep` や `awk` と連携して、スキャン結果から特定の情報を抽出するのに非常に便利です。

### シナリオ例: スキャン結果からWebサーバーだけをリストアップする (レッドチーム視点)

* **目的**: ネットワーク全体をスキャンした後、ポート80が開いているホストのIPアドレスだけを抽出する。
* **連携コマンド**: `grep`
* **解説**: `-oG -` でGrep可能な形式の結果を標準出力に送り、それをパイプで `grep` に渡して "80/open/http" を含む行だけをフィルタリングします。
* **コマンド例**:

    ```bash
    # Grep可能な形式でスキャンし、ポート80が開いているホストを抽出
    sudo nmap -p 80 -oG - 192.168.1.0/24 | grep "80/open/http"
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **スキャンタイプ** | |
| ⭐ `-sS` | **TCP SYNスキャン** (root権限要)。高速で隠密性の高いスキャン。デフォルトです。 |
| `-sT` | **TCP Connectスキャン**。完全なTCP接続を試みます。root権限がない場合のデフォルト。 |
| `-sU` | **UDPスキャン**。低速ですが、DNS(53)やSNMP(161)などのUDPサービスを発見します。 |
| `-sn` | **Pingスキャン**。ポートスキャンを行わず、ホストがオンラインか（生存確認）のみを行います。 |
| **ターゲット/ポート指定** | |
| ⭐ `-p <ports>` | スキャン対象のポートを指定します。（例: `-p 22,80,443`, `-p 1-1024`, `-p-` (全ポート)） |
| `-iL <file>` | ターゲットのリストをファイルから読み込みます。 |
| **サービス/OS検知** | |
| ⭐ `-sV` | オープンしているポートで稼働しているサービスのソフトウェア名とバージョンを特定しようと試みます。 |
| ⭐ `-O` | ターゲットOSを推測します。 |
| ⭐ `-A` | **Aggressiveスキャン**。`-O`, `-sV`, `-sC` (デフォルトスクリプト), `traceroute` を全て有効にする包括的なオプション。 |
| **スクリプト (NSE)** | |
| `-sC` | デフォルトで安全とされるNSEスクリプトを実行します。 |
| `--script=<script>` | 特定のNSEスクリプトまたはカテゴリ（`vuln`など）を実行します。 |
| **パフォーマンスと回避** | |
| ⭐ `-T<0-5>` | スキャンの速度を調整します。`-T4` (Aggressive) が一般的。 |
| `-n` | DNS解決を行わず、スキャンを高速化します。 |
| ⭐ `-Pn` | Pingをスキップします。ICMPをブロックするホストに有効。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー構築後、意図しないポートが開いていないか素早く確認する。
* **組み合わせ**: `nmap -F <server_ip>`
* **解説**: `-F` (Fastスキャン) で、一般的な100ポートのみを対象に高速なスキャンを実行し、ファイアウォールの設定ミスがないかを確認します。
* **例**:

    ```bash
    # my-web-server.com の一般的なポートの状態を確認
    nmap -F my-web-server.com
    ```

### 2. ブルーチーム視点

* **タスク**: 管理下のサーバーに既知の脆弱性がないか定期的に監査する。
* **組み合わせ**: `sudo nmap -sV --script=vuln <target_ip>`
* **解説**: サービスのバージョンを特定し、それに関連する既知の脆弱性をNSEスクリプトで検査します。古いバージョンのApacheやOpenSSHが稼働しており、既知のCVEに該当する、といった結果が出れば、パッチ適用の対象とします。
* **例**:

    ```bash
    # 192.168.1.50 に対して脆弱性スキャンを実行
    sudo nmap -sV --script=vuln 192.168.1.50
    ```

### 3. レッドチーム視点

* **タスク**: ペネトレーションテストの初期偵察として、ターゲットネットワークの全体像を把握する。
* **組み合わせ**: `sudo nmap -T4 -A -v <target_range>`
* **解説**: ターゲットネットワークに対して、OS・バージョン検知を含む包括的で高速なスキャンを実行します。開いているポート、稼働しているサービスのバージョン、OSの種類を特定し、攻撃の足がかりとなる脆弱なサービスを探し出します。
* **例**:

    ```bash
    # 192.168.1.0/24 ネットワーク全体に対して、高速かつ詳細なスキャンを実行し、結果を保存
    sudo nmap -T4 -A -v 192.168.1.0/24 -oA initial_scan
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`nmap` を実行したが、`Host seems down.` と表示される。**
    * **考えられる原因**: ターゲットホストがICMPエコー（Ping）に応答しないように設定されています。`nmap` はスキャン前にこの生存確認を行うため、ここで失敗するとポートスキャンに進みません。
    * **解決策**: `-Pn` オプションを付けて、Pingによる生存確認をスキップしてください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **許可なきスキャン**。**許可なく他者のネットワークやサーバーに `nmap` を実行することは、攻撃行為と見なされ、法に触れる可能性があります**。
* **悪用シナリオ**: 攻撃者は偵察段階で `nmap` を使い、標的の弱点を探します。また、`--script` オプションで脆弱性スキャンや、時には直接的な攻撃を行うことも可能です。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: 古いバージョンの `nmap` にはインタラクティブモードが存在しました。もしユーザーが `sudo nmap --interactive` を実行できる権限を持っている場合、そのモード内で `!sh` を実行することでrootシェルを奪取できます。
* **参照**: `https://gtfobins.github.io/gtfobins/nmap/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 不要なポートはファイアウォールで閉じる。ソフトウェアを常に最新の状態に保つ。`sudoers` の設定を厳格にする。
* **Detection (検知)**: `nmap` によるスキャンは、**IDS/IPSやファイアウォールに容易に検知・記録されます**。ネットワークの監視は、不審なスキャン活動を発見するための基本です。

## 注意点・補足

* **root権限**: `-sS` (SYNスキャン) や `-O` (OS検知) のような、低レベルのパケットを生成するスキャンタイプは、`root` 権限が必要です。`sudo nmap ...` のように実行するのが一般的です。
* **NSEの威力**: Nmapの真の力は **Nmap Scripting Engine (NSE)** にあります。`/usr/share/nmap/scripts/` 配下にある多数のスクリプトを活用することで、単純なポートスキャン以上の高度な調査が可能です。
