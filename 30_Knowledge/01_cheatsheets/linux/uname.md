# `uname` - システムの基本情報を表示する

## 概要

`uname` (Unix name) は、現在実行中の**カーネル**に関する基本的な情報を表示するコマンドです。カーネル名、ホスト名、カーネルのリリースバージョン、CPUアーキテクチャなどを確認できます。

特に `uname -a` は、システムを特定するための最も基本的な情報（OSの種類、バージョン、アーキテクチャ）を一行で表示するため、トラブルシューティングのレポートや、脆弱性情報を調査する際の最初のステップとして頻繁に利用されます。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| ⭐ **`lsb_release -a`** | **ディストリビューション**固有の情報（例: `Ubuntu 22.04.3 LTS`）を表示します。`uname` は、そのディストリビューションが使用している**Linuxカーネル**の情報を表示します。 | - |
| **`hostname`** | システムの**ホスト名**の表示・設定に特化しています。`uname -n` はホスト名を表示するだけで、設定はできません。 | `hostnamectl` |
| **`cat /proc/version`** | `uname` が表示する情報の大元となる、より詳細なカーネルバージョン文字列（コンパイル情報など）を確認できます。 | - |

## よく連携されるコマンド

`uname -r` は、現在稼働中のカーネルに対応するモジュールやヘッダーを探す際に、他のコマンドと連携して使われます。

### シナリオ例: カーネルモジュールのディレクトリを確認する (インフラ構築・運用視点)

* **目的**: 現在のカーネルバージョンに対応する、ロード可能なカーネルモジュールのディレクトリパスを確認する。
* **連携コマンド**: `ls`, コマンド置換 (`$()`)
* **解説**: `uname -r` の実行結果をコマンド置換で `ls` コマンドの引数に渡し、動的にパスを生成します。
* **コマンド例**:

    ```bash
    # 現在のカーネルに対応するモジュールディレクトリの内容をリスト表示
    ls /lib/modules/$(uname -r)/
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `-a`, `--all` | **最もよく使うオプション**。全ての情報を一行で表示します。 |
| ⭐ `-r`, `--kernel-release` | カーネルのリリースバージョン（例: `5.15.0-101-generic`）を表示します。 |
| `-s`, `--kernel-name` | (デフォルト) カーネル名（例: `Linux`）を表示します。 |
| `-n`, `--nodename` | ホスト名を表示します。 |
| `-m`, `--machine` | ハードウェアのアーキテクチャ（例: `x86_64`, `aarch64`）を表示します。 |
| `-o`, `--operating-system` | オペレーティングシステム名（例: `GNU/Linux`）を表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 現在稼働しているカーネルの正確なリリースバージョンを取得する。
* **組み合わせ**: `uname -r`
* **解説**: カーネルモジュールをビルド・インストールする際に、正しいバージョンのヘッダーファイルを指定するために使用します。
* **例**:

    ```bash
    # DKMSなどでカーネルモジュールをビルドする際にバージョンを指定する
    sudo apt install linux-headers-$(uname -r)
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害されたホストのOSとカーネルのバージョンを正確に記録する。
* **組み合わせ**: `uname -a`
* **解説**: **インシデントレスポンスの初期調査**。`uname -a` で得られたカーネルバージョンをCVEデータベースと照合し、そのカーネルに影響する既知の権限昇格の脆弱性がないかを判断します。
* **例**:

    ```bash
    # 調査対象ホストの情報を取得
    uname -a
    ```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、標的のOSとカーネルを特定（フィンガープリント）する。
* **組み合わせ**: `uname -a`
* **解説**: 攻撃者は侵入後、ほぼ確実に `uname -a` を実行します。この出力から得られるカーネルの正確なバージョンとアーキテクチャを元に、**権限昇格 (Privilege Escalation)** に利用可能な公開エクスプロイトコードを検索します。
* **例**:

    ```bash
    # 侵入したホストのカーネルバージョンを特定
    uname -a
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。
* `uname` はシステムの基本的な情報を表示するだけの非常にシンプルなコマンドであり、エラーが発生することは稀です。

## 環境変数と設定ファイル

* `uname` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `uname` 自体ではなく、その出力が**深刻な情報漏洩**に繋がります。
* **悪用シナリオ**: `uname -a` の出力は、システムの**正確なカーネルバージョン**を攻撃者に教えてしまいます。これは、攻撃者が膨大な数の脆弱性の中から、標的に対して有効なエクスプロイトをピンポイントで選択するための最も重要な情報源です。

### GTFOBins / LOLBAS における利用例

`uname` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **カーネルを常に最新のセキュリティパッチが適用された状態に保つ**ことが、この情報漏洩リスクに対する最も効果的な対策です。
* **Detection (検知)**: `uname` は基本的なコマンドですが、Webサーバーのプロセス (`www-data`) から `uname` が実行されるといった、通常ではありえないコンテキストでの実行をEDRや`auditd`で監視することで、攻撃の兆候を捉えることができます。

## 注意点・補足

* **コンテナ内での実行**: Dockerコンテナ内で `uname -a` を実行すると、コンテナOSの情報ではなく、**ホストOSのカーネル情報**が表示されます。これは、全てのコンテナがホストのカーネルを共有しているためです。
