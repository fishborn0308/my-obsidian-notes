# `lspci` - PCIデバイスを一覧表示する

## 概要

`lspci` (list PCI) は、システムに接続されている全ての **PCIバス** と、そこに接続された**デバイス**（ネットワークカード、グラフィックカード、ストレージコントローラなど）に関する情報を一覧表示するコマンドです。「このサーバーに搭載されているネットワークカードの正確なモデルは何か？」といった、ハードウェアに関する問いに答えるための基本的なユーティリティです。

(出自: `pciutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`lshw`** | `lspci` の情報を含み、さらにCPU、メモリなど、より**広範なハードウェア情報**を網羅的に表示します。 | `lshw` (全体像) |
| **`lsusb`** | **USBデバイス**の情報表示に特化しています。`lspci` は**PCI(e)デバイス**に特化しています。 | - |
| **`dmidecode`** | **BIOS/UEFIのDMIテーブル**からハードウェア情報を読み取ります。`lspci` は、OSカーネルが現在アクティブに認識しているPCIバスの情報を直接表示します。 | - |

## よく連携されるコマンド

`lspci` の出力は、`grep` と連携して特定の種類のデバイス情報だけをフィルタリングしたり、`-k` オプションで表示されたドライバ名を `modinfo` に渡して詳細を調べたりする際に使われます。

### シナリオ例: ネットワークカードのドライバを確認する (インフラ構築・運用視点)

* **目的**: システムに搭載されているネットワークカードのモデルを特定し、どのカーネルドライバが使用されているかを確認する。
* **連携コマンド**: `grep`
* **解説**: `lspci -k` でデバイスとドライバの情報を表示し、その中から `grep` で 'Ethernet' というキーワードを含む行とその後の数行 (`-A 2`) を抽出します。
* **コマンド例**:

    ```bash
    # ネットワークカードとそのカーネルドライバの情報を表示
    lspci -k | grep -i ethernet -A 2
    # 01:00.0 Ethernet controller: Realtek ... (rev 15)
    #         Subsystem: Dell ...
    #         Kernel driver in use: r8169
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `-v`, `-vv`, `-vvv` | **詳細表示**。`-v` を増やすほど、より詳細な情報（I/Oポート、メモリ領域、ケーパビリティなど）を表示します。 |
| ⭐ `-k` | そのデバイスを制御している**カーネルドライバ（モジュール）**を表示します。**非常に便利です**。 |
| `-n`, `-nn` | `-n` はベンダーとデバイスIDを**数値**で表示します。`-nn` は数値と**名前**の両方を表示します。 |
| `-t` | PCIデバイスの接続関係を**ツリー形式**で表示します。 |
| **フィルタリング** | |
| `-s <slot>` | 指定したバスアドレス (`[domain]:[bus]:[device].[function]`) のデバイス情報のみを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ネットワークカードがどのドライバを使っているか確認する。
* **組み合わせ**: `lspci -k`
* **解説**: ハードウェアが正常に動作しない場合、このコマンドで `Kernel driver in use:` の行を確認します。ドライバがロードされていない、あるいは意図しないドライバが使われている場合、それが問題の原因である可能性が高いです。
* **例**:

    ```bash
    # 全てのPCIデバイスについて、使用中のカーネルドライバを表示
    lspci -k
    ```

### 2. ブルーチーム視点

* **タスク**: システムのハードウェア構成を監査し、ベースラインと比較する。
* **組み合わせ**: `lspci -tv`
* **解説**: システムのハードウェア構成をツリー形式で詳細に把握し、ベースラインとして記録します。物理的なセキュリティが重要なサーバーで、意図しないPCIデバイス（不正なNICなど）が追加されていないかを確認します。
* **例**:

    ```bash
    # PCIバスのツリー構造とデバイスを詳細に表示
    lspci -tv
    ```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、標的システムのハードウェア構成を把握する。
* **組み合わせ**: `lspci`
* **解説**: 攻撃者は `lspci` の出力から、仮想化環境 (`VMware SVGA Adapter`, `Virtio network device`など) の特定、ネットワークカードやストレージコントローラの正確なモデルの把握を行い、そのデバイスドライバに存在する脆弱性（ローカルでの権限昇格など）を探します。
* **例**:

    ```bash
    # ネットワークコントローラとVGAコントローラの情報を確認
    lspci | grep -i -E "ethernet|vga"
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **デバイス名は表示されるが、ベンダー名などが `Unknown device` と表示される。**
    * **考えられる原因**: `lspci` がベンダーIDとデバイスIDを名前に変換するためのローカルデータベースが古い可能性があります。
    * **解決策**: `sudo update-pciids` コマンドを実行して、PCI IDデータベースを最新の状態に更新してください。

## 環境変数と設定ファイル

* `lspci` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `lspci` 自体は読み取り専用ツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: `lspci` は、システムのハードウェア構成に関する詳細な情報を出力します。これは、攻撃者がハードウェアやドライバ固有の脆弱性を探し出し、標的を絞った攻撃を計画するための重要な偵察情報となります。

### GTFOBins / LOLBAS における利用例

`lspci` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視-点)

* **Prevention (予防)**: 最小権限の原則に従い、不要なユーザーに `lspci` の `sudo` 実行権限を与えない。
* **Detection (検知)**: `auditd` などで `lspci` コマンドの実行を監視し、不審な偵察活動がないかを確認する。

## 注意点・補足

* **`update-pciids`**: `lspci` が表示するベンダー名やデバイス名は、`/usr/share/hwdata/pci.ids` というテキストファイルに基づいています。このファイルを更新するために、`update-pciids` というコマンドが提供されています。新しいハードウェアが正しく表示されない場合は、このコマンドの実行を試みてください。
