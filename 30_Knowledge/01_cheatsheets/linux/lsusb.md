# `lsusb` - USBデバイスを一覧表示する

## 概要

`lsusb` (list USB) は、システムに接続されている全ての **USBバス** と、そこに接続された**デバイス**（キーボード、マウス、USBメモリ、ウェブカメラなど）に関する情報を一覧表示するコマンドです。「接続したUSBデバイスがOSに認識されているか？」を確認するための基本的なユーティリティです。

(出自: `usbutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`lspci`** | **PCIデバイス**（ネットワークカード、グラフィックカードなど）の情報表示に特化しています。`lsusb` は**USBデバイス**に特化しています。 | - |
| **`lshw`** | `lsusb` の情報を含み、さらにCPU、メモリなど、より**広範なハードウェア情報**を網羅的に表示します。 | `lshw` (全体像) |
| **`dmesg`** | USBデバイスを接続・切断した際にカーネルが出力する**メッセージ**を確認できます。`lsusb` が現在の状態を表示するのに対し、`dmesg` は接続時のイベントログ（ドライバのロード状況など）を確認するのに役立ちます。 | - |

## よく連携されるコマンド

`lsusb` は、デバイス接続時のカーネルメッセージを表示する `dmesg` や、デバイスのリアルタイムな変化を監視する `watch` とよく連携されます。

### シナリオ例: USBデバイスの接続をリアルタイムで確認する (インフラ構築・運用視点)

* **目的**: USBデバイスを抜き差しした際に、OSがそれを正しく認識・切断しているかをリアルタイムで確認する。
* **連携コマンド**: `watch`
* **解説**: `watch -n 1 lsusb` を実行すると、1秒ごとに `lsusb` の結果が更新されます。この状態でUSBデバイスを抜き差しすると、デバイスリストが変動するのが分かり、OSレベルでの認識状況を簡単に確認できます。
* **コマンド例**:

    ```bash
    # 1秒ごとにUSBデバイスのリストを監視
    watch -n 1 lsusb
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `-v`, `-vv` | **詳細表示**。デバイスのディスクリプタ情報を全て表示します。`-v` を増やすほど、より詳細になります。 |
| ⭐ `-t` | USBデバイスの接続関係を**ツリー形式**で表示します。どのデバイスがどのハブに接続されているかが分かります。 |
| **フィルタリング** | |
| `-s [[bus]:][devnum]` | 指定したバス番号やデバイス番号のデバイス情報のみを表示します。 |
| `-d [vendor]:[product]` | 指定したベンダーIDやプロダクトIDを持つデバイスのみを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーに接続したUSBデバイスがシステムに正しく認識されているかを確認する。
* **組み合わせ**: `lsusb`
* **解説**: データセンターでサーバーにUSBデバイス（KVMスイッチ、USB-シリアル変換アダプタ、ハードウェアセキュリティキーなど）を接続したが、うまく動作しない場合に、まず `lsusb` を実行してOSレベルでデバイスの存在が認識されているかを確認します。
* **例**:

    ```bash
    # デバイスの階層構造を確認
    lsusb -t
    ```

### 2. ブルーチーム視点

* **タスク**: 意図しないUSBデバイスが接続されていないかを確認する。
* **組み合わせ**: `lsusb`
* **解説**: **物理的セキュリティインシデントの調査**。重要なサーバーへの不正な物理アクセスが疑われる場合に `lsusb` を実行します。もし、見慣れないデバイスや、**`Keyboard`** としか表示されない不審なデバイスがあれば、それは **BadUSB**（キーボードになりすまして悪意のあるコマンドを自動入力する攻撃ツール）である可能性を疑います。
* **例**:

    ```bash
    # 5秒ごとにUSBデバイスのリストを監視し、不正なデバイスの接続を検知
    watch -n 5 lsusb
    ```

### 3. レッドチーム視点

* **タスク**: **BadUSB攻撃**。キーボードになりすますUSBデバイスを物理的に接続し、ペイロードを送り込む。
* **組み合わせ**: (物理アクセス)
* **解説**: 攻撃者は、ターゲットの従業員が離席した隙に、PCのUSBポートにBadUSBを差し込みます。デバイスはOSにキーボードとして認識され、事前にプログラムされたキーストロークを高速で自動入力し、リバースシェルを起動させたり、マルウェアをダウンロードさせたりします。
* **偵察**: リモートで侵入した後、`lsusb` を実行して、接続されているUSBデバイスを確認します。もし、ハードウェアセキュリティモジュール (HSM) や暗号鍵を格納したUSBトークンなどが接続されていれば、それが次の攻撃の価値ある標的となります。

    * **考えられる原因**: `usbutils` パッケージがインストールされていません。
    * **解決策**: `sudo apt install usbutils` や `sudo yum install usbutils` などでパッケージをインストールしてください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `lsusb` 自体ではなく、**USBインターフェイス**そのものが攻撃ベクトルとなり得ます。
* **悪用シナリオ**: BadUSB攻撃は、キーボードやネットワークカードなど、信頼されたクラスのデバイスになりすますことで、多くの防御機構をバイパスします。

### GTFOBins / LOLBAS における利用例

`lsusb` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: サーバーラックを施錠する。データセンターへの物理アクセスを厳格に管理する。USBポートを物理的に塞ぐか、BIOS/UEFIレベルまたはOSレベル (`usbguard`など) で不要なUSBデバイスクラスをブロックする。
* **Detection (検知)**: `auditd` や `usbguard` を使って、新しいUSBデバイスの接続イベントを監視し、意図しないデバイスが接続された場合にアラートを発報する。

## 注意点・補足

* **`update-usbids`**: `lsusb` が表示するベンダー名やデバイス名は、`/var/lib/usbutils/usb.ids` というテキストファイルに基づいています。このファイルを更新するために、`update-usbids` というコマンドが提供されています。新しいハードウェアが正しく表示されない場合は、このコマンドの実行を試みてください。
