# `pip` - Pythonのパッケージ管理ツール

## 概要

`pip` (Pip Installs Packages) は、**Python**で書かれたソフトウェアパッケージをインストール・管理するための標準的なパッケージマネージャです。**PyPI (Python Package Index)** と呼ばれる公式の公開リポジトリから、膨大な数のライブラリやフレームワークを簡単に導入できます。アプリケーション開発において、必要な依存関係を管理するための中心的な役割を担います。

(出自: Pythonに標準でバンドルされていることが多い)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| ⭐ **`apt` / `yum`** | これらは**OSレベル**のパッケージマネージャです。`pip` は**Python言語**のパッケージに特化しています。`pip` を使う方が最新のバージョンを、プロジェクトごとに独立して管理できます。 | `pip` (Pythonパッケージ) |
| **`venv`** | Pythonの**仮想環境**を作成するツール。`pip` は、`venv` で作成された**仮想環境の中で使われる**のが一般的です。 | - |
| **`pipenv` / `Poetry`** | `pip` と `venv` の機能を組み合わせ、より高レベルな依存関係管理を自動化するツールです。 | `pipenv` / `Poetry` (プロジェクト管理) |

## よく連携されるコマンド

`pip` は、`requirements.txt` というファイルと連携して、プロジェクトの依存関係をまとめて管理するのが最も一般的な使い方です。

### シナリオ例: プロジェクトの依存関係を再現する (インフラ構築・運用視点)

* **目的**: 開発環境で使われていたPythonライブラリと全く同じバージョンを、本番環境に一括でインストールする。
* **連携コマンド**: `pip`, `requirements.txt`
* **解説**: まず開発環境で `pip freeze` を実行して、インストールされているパッケージのリストを `requirements.txt` に保存します。次に、そのファイルを本番環境に持っていき、`pip install -r` でリストの内容を一度にインストールします。
* **コマンド例**:

    ```bash
    # (開発環境で実行)
    # 現在の環境のパッケージを requirements.txt に書き出す
    pip freeze > requirements.txt

    # (本番環境で実行)
    # requirements.txt に基づいてパッケージを一括インストール
    pip install -r requirements.txt
    ```

## オプション説明 (`pip` のサブコマンド)

| オプション | 説明 |
| :--- | :--- |
| **パッケージ管理** | |
| ⭐ `install` | パッケージをインストールします。 |
| ⭐ `uninstall` | パッケージをアンインストールします。 |
| ⭐ `freeze` | インストール済みパッケージを `requirements.txt` 形式で出力します。 |
| **情報表示** | |
| ⭐ `list` | インストール済みのパッケージを一覧表示します。 |
| `show <pkg>` | 指定したパッケージの詳細情報（バージョン、依存関係など）を表示します。 |
| `check` | インストール済みパッケージ間の依存関係に問題がないかを確認します。 |
| **その他** | |
| `-r <file>` (`install`時) | `requirements.txt` ファイルからパッケージをまとめてインストールします。 |
| `-U`, `--upgrade` | パッケージを最新バージョンにアップグレードします。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのデプロイ時に、必要なライブラリを仮想環境に導入する。
* **組み合わせ**: `pip install -r requirements.txt`
* **解説**: `python3 -m venv venv` で仮想環境を作成・有効化した後、このコマンドでアプリケーションの実行に必要なライブラリをクリーンな環境に導入します。
* **例**:

    ```bash
    # 仮想環境を有効化した後、依存関係をインストール
    source venv/bin/activate
    pip install -r requirements.txt
    ```

### 2. ブルーチーム視点

* **タスク**: システムにインストールされているPythonパッケージに既知の脆弱性がないか監査する。
* **組み合わせ**: `pip list --outdated`
* **解説**: インストール済みのパッケージの中で、新しいバージョンがリリースされているものを一覧表示します。古いパッケージには既知の脆弱性が含まれている可能性が高いため、リストアップされたパッケージを `-U` オプションでアップグレードし、システムのセキュリティを維持します。
* **例**:

    ```bash
    # 古いパッケージがないか確認
    pip list --outdated
    ```

### 3. レッドチーム視点

* **タスク**: **サプライチェーン攻撃**として、タイポスクワッティングを利用して悪意のあるパッケージをインストールさせる。
* **組み合わせ**: (悪意のあるパッケージのアップロード)
* **解説**: 攻撃者は、`requests` を `reqeusts` のように、有名で人気のあるパッケージとよく似た名前の悪意のあるパッケージをPyPIにアップロードします。開発者がタイプミス (`pip install reqeusts`) をすると、マルウェアが仕込まれた偽のパッケージがインストールされてしまいます。
* **偵察**: `pip freeze` や `pip list` を実行し、インストールされているライブラリのバージョンを特定して、既知の脆弱性を探します。

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `error: externally-managed-environment`
    * **考えられる原因**: OSのパッケージマネージャ (`apt`など) によって管理されているシステムPython環境に、`pip` で直接パッケージをインストールしようとしました。近年のディストリビューションでは、システムの破損を防ぐためにこの操作がデフォルトで禁止されています。
    * **解決策**: **必ず `python3 -m venv <dir>` で仮想環境を作成し、それを有効化してから `pip` を使用してください**。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

* **`~/.config/pip/pip.conf`** (Linux): `pip` のデフォルトの挙動をカスタマイズできます。例えば、PyPIのミラーサーバーを指定してダウンロードを高速化できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **サプライチェーン攻撃**。PyPIは誰でも比較的容易にパッケージを公開できるため、全てのパッケージが安全であるとは限りません。
* **悪用シナリオ**: タイポスクワッティングや、人気パッケージが乗っ取られて悪意のあるコードが注入されるといった攻撃が常に発生しています。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo pip install` を実行できる権限を持っている場合、簡単にroot権限を奪取できます。攻撃者は、`setup.py` ファイルにリバースシェルを起動するような悪意のあるコードを仕込んだ簡単なパッケージを作成し、そのディレクトリで `sudo pip install .` を実行させます。`setup.py` が**root権限で実行され**、結果としてrootシェルが得られます。
* **参照**: `https://gtfobins.github.io/gtfobins/pip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **仮想環境を常に使う**。信頼できないパッケージはインストールしない。プライベートPyPIリポジトリを利用して、許可したパッケージのみを使えるようにする。
* **Detection (検知)**: `pip audit` などのツールで、インストール済みのパッケージに既知の脆弱性がないかスキャンする。`auditd` で `pip` による不審なファイル書き込みやプロセス生成を監視する。

## 注意点・補足

* **`pip` vs `pip3`**: システムによっては `python2` の `pip` と `python3` の `pip3` が共存している場合があります。`python3 -m pip ...` のように `-m` を使って実行すると、意図したPythonバージョンの `pip` を確実に呼び出せるため安全です。
