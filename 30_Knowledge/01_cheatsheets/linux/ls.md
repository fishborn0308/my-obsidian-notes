# `ls` - ファイルやディレクトリを一覧表示する

## 概要

`ls` (list) は、指定したディレクトリの内容（ファイルやサブディレクトリ）を一覧表示するための、最も基本的なコマンドです。オプションを組み合わせることで、詳細な情報（パーミッション、所有者、サイズ、更新日時など）を表示したり、表示順序を変更したりと、非常に柔軟な出力が可能です。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`find`** | 指定したディレクトリ以下を**再帰的に**検索し、複雑な条件に一致するファイルを見つけ出します。`ls` は基本的に指定したディレクトリの**一層のみ**を表示します。 | - |
| **`tree`** | ディレクトリの内容を再帰的に、かつ階層構造を**ツリー形式**で視覚的に分かりやすく表示します。 | `tree` (階層表示) |
| **`stat`** | **単一の**ファイルやディレクトリに関する、より詳細なメタデータ（iノード番号、全タイムスタンプなど）を表示します。 | - |

## よく連携されるコマンド

`ls` の出力は、`grep` でフィルタリングしたり、`wc -l` で項目数を数えたりと、他の多くのコマンドとパイプで連携されます。

### シナリオ例: 特定の種類のファイルだけをリストアップする (インフラ構築・運用視点)

* **目的**: カレントディレクトリにある `.conf` で終わる設定ファイルだけを詳細表示する。
* **連携コマンド**: ワイルドカード (`*`)
* **解説**: `ls` コマンドの引数としてワイルドカードを使うことで、パターンに一致するファイルのみを効率的にリストアップできます。
* **コマンド例**:

    ```bash
    # .conf 拡張子を持つファイルの詳細をリスト表示
    ls -l *.conf
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `-l` | **詳細表示形式** (long listing format) で表示します。 |
| ⭐ `-h`, `--human-readable` | `-l` と併用し、サイズを人間が読みやすい単位 (K, M, G) で表示します。 |
| `-F`, `--classify` | ファイルの種類に応じて末尾に記号を付与します (`/`:ディレクトリ, `*`:実行ファイル, `@`:シンボリックリンク)。 |
| **対象** | |
| ⭐ `-a`, `--all` | `.` や `..` を含む、**全てのファイル**（隠しファイル）を表示します。 |
| `-A`, `--almost-all` | `.` と `..` を除く、全てのファイルを表示します。 |
| `-R`, `--recursive` | サブディレクトリの内容も**再帰的に**表示します。 |
| `-d`, `--directory` | ディレクトリ自体の情報を表示し、その中身は表示しません。 |
| **ソート** | |
| ⭐ `-t` | **更新日時順**（新しいものが先）にソートします。 |
| `-r`, `--reverse` | ソート順を**逆順**にします。 |
| `-S` | **ファイルサイズ順**（大きいものが先）にソートします。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログディレクトリの内容を、最も古いものから順に時系列で確認する。
* **組み合わせ**: `ls -lhtr`
* **解説**: 詳細表示(`l`)、読みやすいサイズ(`h`)、時間順ソート(`t`)、逆順(`r`)を組み合わせることで、「**最も古く更新されたファイルが一番上に表示される**」リストが得られます。
* **例**:

    ```bash
    # /var/log ディレクトリの内容を古い順に表示
    ls -lhtr /var/log/
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害されたユーザーのホームディレクトリで、隠された攻撃ツールや設定ファイルがないか調査する。
* **組み合わせ**: `ls -la <directory>`
* **解説**: **フォレンジック調査の基本**。`-a` オプションで隠しファイルや隠しディレクトリを表示します。攻撃者は自身のツールや設定ファイルを `.ssh` や `.config` のような正規の隠しディレクトリに偽装したり、`.backdoor` のような新しい隠しファイルを作成したりして活動を隠蔽します。
* **例**:

    ```bash
    # 侵害されたユーザーのホームディレクトリを調査
    ls -la /home/compromised-user
    ```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、システム全体のファイルとディレクトリを調査する。
* **組み合わせ**: `ls -laR /`
* **解説**: システム全体のファイルとディレクトリを再帰的にリストアップし、ファイルシステムの全体像を把握します。パーミッションが緩い設定ファイル、ユーザーの秘密鍵、Webアプリケーションの認証情報ファイルなどを探します。
* **例**:

    ```bash
    # /etc 配下を再帰的にリストし、パーミッションが緩いファイルがないか探す
    ls -laR /etc | less
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`ls -l` を実行すると、一部のファイルの所有者やグループが数字で表示される。**
    * **考えられる原因**: そのファイルが所有するユーザーID (UID) またはグループID (GID) が、システムのユーザー/グループデータベース (`/etc/passwd`, `/etc/group`) に存在しません。
    * **解決策**: 別のシステムからファイルをコピーしてきた場合や、ユーザー/グループを削除した場合に発生します。これは必ずしもエラーではありませんが、意図しない状態である場合は `chown` で所有者を修正する必要があります。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 影響を与える環境変数

* **`LS_COLORS`**: `ls --color=auto` が有効な場合に、ファイルの種類ごとの色を定義します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ls` 自体ではなく、その**出力内容**が**情報漏洩**に繋がります。
* **悪用シナリオ**: `ls -l` の出力は、システムのファイル構成、所有者、パーミッションに関する詳細な情報を含んでおり、攻撃者にとって非常に価値の高い偵察情報となります。

### GTFOBins / LOLBAS における利用例

`ls` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、ユーザーがアクセスできるディレクトリやファイルを適切に制限する。
* **Detection (検知)**: `ls` は攻撃の偵察段階でほぼ必ず実行されるため、`auditd` で `ls` の実行（特に重要なディレクトリに対して）を監視することで、不審な活動の初期段階を捉えることができる可能性があります。

## 注意点・補足

* **エイリアスの活用**: 多くのディストリビューションでは `ll` が `ls -l` のエイリアスとして設定されています。`alias ll='ls -lhaF'` のように `~/.bashrc` でカスタマイズすると、日常的な操作が大幅に効率化できます。
