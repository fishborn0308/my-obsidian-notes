# `uniq` - 重複する隣接行を報告・削除する

## 概要

`uniq` は、ソート済みのテキストデータから、**隣接する重複行**を削除、または集計するためのコマンドです。

**⚠️ 最重要**: `uniq` が正しく機能するための大前提は、**入力データが事前に `sort` コマンドで並べ替えられている**ことです。`uniq` は隣り合った行しか比較しないため、ソートされていないデータに対して実行すると、意図しない結果になります。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| ⭐ **`sort -u`** | ソートと同時に重複行の削除も行います。`sort \| uniq` とほぼ同じ結果を、より効率的に一度のコマンドで得られます。 | `sort -u` (単純な重複削除) |
| **`awk '!seen[$0]++'`** | `awk` を使った、より高機能な重複削除テクニック。**元の行の順序を保持したまま**、最初に出現した行だけを残すことができます。 | `awk` (順序維持) |

## よく連携されるコマンド

`uniq` は `sort` と連携して、項目ごとの出現回数を集計する際に不可欠な役割を果たします。

### シナリオ例: ログから攻撃元IPをランキングする (ブルーチーム視点)

* **目的**: Webサーバーのアクセスログから、最もリクエスト回数が多いIPアドレス（DDoS攻撃やブルートフォース攻撃の兆候）を特定する。
* **連携コマンド**: `awk`, `sort`, `uniq`
* **解説**: `awk` でIPアドレスを抽出し、`sort` でIPアドレスをまとめ（`uniq` の前処理）、`uniq -c` で出現回数を集計します。最後に再度 `sort -nr` で結果を降順にソートするのが黄金パターンです。
* **コマンド例**:

    ```bash
    # このパイプラインで、アクセスログから攻撃元IPトップ10を抽出する
    awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -n 10
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **集計とフィルタ** | |
| ⭐ `-c`, `--count` | 各行の先頭に、その行が何回連続して出現したかの**出現回数**を付けて表示します。 |
| ⭐ `-d`, `--repeated` | **重複している行のみ**（2回以上出現する行）を表示します。 |
| ⭐ `-u`, `--unique` | **重複していない行のみ**（1回しか出現しない行）を表示します。 |
| **比較制御** | |
| `-i`, `--ignore-case` | 大文字と小文字を区別せずに比較します。 |
| `-f <N>`, `--skip-fields=<N>` | 先頭からN個のフィールド（列）を無視して比較します。 |
| `-s <N>`, `--skip-chars=<N>` | 各行の先頭からN文字を無視して比較します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: コマンドの出力やファイルの内容から、重複するエントリを削除して一意なリストを作成する。
* **組み合わせ**: `... | sort | uniq`
* **解説**: 設定ファイルに誤って同じエントリを複数記述してしまった場合に、重複を排除したクリーンな設定ファイルを作成します。
* **例**:

    ```bash
    # myconfig.txt の重複行を削除して myconfig.clean.txt に保存
    sort myconfig.txt | uniq > myconfig.clean.txt
    ```

### 2. ブルーチーム視点

* **タスク**: ログから特定のイベントを抽出し、発生回数が多い順にランキングする。
* **組み合わせ**: `... | sort | uniq -c | sort -nr`
* **解説**: 上記「よく連携されるコマンド」シナリオの通り、ログ分析における頻出パターンです。
* **例**:

    ```bash
    # 認証ログからログインに失敗しているユーザー名を多い順にリストアップ
    grep "Failed password for" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr
    ```

### 3. レッドチーム視点

* **タスク**: パスワード攻撃用のワードリストから重複を削除し、整理する。
* **組み合わせ**: `sort -u wordlist.txt -o wordlist_unique.txt`
* **解説**: 複数のソースから収集したワードリストを1つにまとめた後、このコマンドで重複エントリを削除し、無駄な試行を減らしたクリーンな辞書ファイルを作成します。(`uniq` よりも `sort -u` の方がこの用途には効率的です)
* **例**:

    ```bash
    # 複数のワードリストを結合し、重複を排除してクリーンなリストを作成
    cat list1.txt list2.txt | sort -u > clean_wordlist.txt
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`uniq` を実行したが、重複しているはずの行が消えない。**
    * **考えられる原因**: **最もよくある間違い**。`uniq` を実行する前に、**入力データを `sort` していません**。`uniq` は隣接する行しか比較しないため、ファイル内で離れた場所にある同じ行は重複と見なされません。
    * **解決策**: 必ず `sort filename | uniq` のように、`uniq` の前段で `sort` を実行してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `uniq` はテキストデータを処理するだけのコマンドであり、直接的なセキュリティリスクは低いです。
* **悪用シナリオ**: 攻撃者は、窃取したデータ（ユーザーリストなど）を `sort | uniq` で整理し、攻撃用のクリーンなリストを作成するためにこのコマンドを利用します。

### GTFOBins / LOLBAS における利用例

`uniq` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `uniq -c` は、ブルートフォース攻撃やDDoS攻撃のように、同じイベントが短時間に大量に発生するタイプの攻撃を検知するための非常に強力なツールです。

## 注意点・補足

* **`sort -u` vs `sort | uniq`**: 単純に重複行を削除したい場合は、`sort -u` の方が1つのプロセスで処理が完結するため効率的です。出現回数を数えたい (`-c`)、重複した行/しない行だけを抜き出したい (`-d`/`-u`) といった、`uniq` 独自の機能を使いたい場合に `sort | uniq` の組み合わせを使います。
