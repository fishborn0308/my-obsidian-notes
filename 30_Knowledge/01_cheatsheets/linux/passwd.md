# `passwd` - ユーザーのパスワードを変更する

## 概要

`passwd` は、ユーザーアカウントのパスワードを設定または変更するためのコマンドです。一般ユーザーが実行すると自身のパスワードを変更でき、`root`ユーザーが実行すると任意のユーザーのパスワードを変更できます。このコマンドは、入力された新しいパスワードを暗号化（ハッシュ化）し、`/etc/shadow` ファイルに安全に保存します。

(出自: `shadow-utils` または `passwd` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`chage`** | ユーザーパスワードの**有効期限ポリシー**（有効期限、最終変更日など）を詳細に管理します。`passwd` はパスワードそのものを変更します。 | - |
| **`useradd`** | 新しいユーザーを**作成**します。`passwd` は**既存のユーザー**のパスワードを設定・変更します。 | - |
| **`chpasswd`** | **スクリプトやバッチ処理**で、複数のユーザーのパスワードを一度に更新するために設計されています。対話的な操作には `passwd` を使います。 | `chpasswd` (バッチ処理) |

## よく連携されるコマンド

`passwd` は、新しいユーザーを作成した直後に、そのアカウントの初期パスワードを設定するために `useradd` と連携するのが最も一般的なワークフローです。

### シナリオ例: 新規ユーザーの作成とパスワード設定 (インフラ構築・運用視点)

* **目的**: 新しい開発者 `newdev` のアカウントを作成し、初期パスワードを設定して、次回ログイン時にパスワード変更を強制する。
* **連携コマンド**: `useradd`, `passwd`
* **解説**: `useradd` でアカウントの骨子を作成した後、`passwd` でパスワードを設定し、`-e` オプションで初回ログイン時のパスワード変更を強制します。
* **コマンド例**:

    ```bash
    # 1. 'newdev' ユーザーを作成
    sudo useradd -m newdev

    # 2. 'newdev' のパスワードを設定 (対話形式)
    sudo passwd newdev

    # 3. 'newdev' に次回ログイン時のパスワード変更を強制
    sudo passwd -e newdev
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ユーザー指定** | |
| ⭐ `(引数なし)` | 現在ログインしているユーザー自身のパスワードを変更します。 |
| ⭐ `<username>` | (`root`権限要) 指定したユーザーのパスワードを変更します。 |
| **アカウント状態** | |
| ⭐ `-l`, `--lock` | (`root`権限要) 指定したユーザーアカウントを**ロック**し、パスワードでのログインを無効にします。 |
| ⭐ `-u`, `--unlock` | (`root`権限要) ロックされているアカウントを**アンロック**します。 |
| ⭐ `-S`, `--status` | (`root`権限要) ユーザーのパスワードの状態（ロックされているか、最終変更日など）を表示します。 |
| `-d`, `--delete` | (`root`権限要) ユーザーのパスワードを削除します。**パスワードなしログインを許可する可能性があり危険**です。 |
| `-e`, `--expire` | (`root`権限要) ユーザーのパスワードを**期限切れ**にし、次回のログイン時にパスワードの変更を強制します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ユーザーがパスワードを忘れた場合にリセットする。
* **組み合わせ**: `sudo passwd <username>`
* **解説**: 管理者がユーザーのパスワードをリセットする際の基本的な操作です。
* **例**:

    ```bash
    # user1 のパスワードをリセット
    sudo passwd user1
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害されたアカウントを即座に無力化する（封じ込め）。
* **組み合わせ**: `sudo passwd -l <compromised_user>`
* **解説**: インシデントレスポンスにおいて、攻撃者がアカウントをこれ以上悪用できないように、即座にアカウントをロックします。これにより、攻撃者のセッションが切れた後の再侵入を防ぎます。
* **例**:

    ```bash
    # 侵害されたアカウント 'user1' を即座にロックし、状態を確認
    sudo passwd -l user1
    sudo passwd -S user1
    # -> user1 L ... (LはLockedを示す)
    ```

### 3. レッドチーム視点

* **タスク**: 永続化と管理者アクセスの確保。
* **組み合わせ**: `sudo passwd root`
* **解説**: 攻撃者が `sudo` 権限などを通じてroot権限を奪取した後、`root` ユーザーのパスワードを自身が知っているものに変更します。これにより、元の管理者を締め出し、システムの完全な制御を維持しようとします。
* **例**:

    ```bash
    # rootのパスワードを変更して管理者アクセスを奪取
    sudo passwd root
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `passwd: Authentication token manipulation error`
    * **考えられる原因**: パスワードの変更に失敗しました。現在のパスワードの入力間違い、ディスクの空き容量不足、`/etc/shadow` ファイルのパーミッションの問題などが考えられます。
    * **解決策**: まずは現在のパスワードを正しく入力しているか確認します。管理者であれば、`sudo passwd <user>` で強制的に上書きできます。

2. **エラーメッセージ例 2**: `BAD PASSWORD: The password is shorter than 8 characters`
    * **考えられる原因**: 設定しようとしたパスワードが、システムのパスワード複雑性ポリシーを満たしていません。
    * **解決策**: より長く、複雑なパスワードを設定してください。ポリシーは通常、PAM (`/etc/pam.d/common-password`など) によって定義されます。

## 環境変数と設定ファイル

* `passwd` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/shadow`**: ハッシュ化されたパスワードとポリシー情報が書き込まれる、`root` のみが読み取り可能なファイル。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `passwd` 自体ではなく、**弱いパスワードポリシー**や**不適切な `sudo` 権限**がリスクとなります。
* **悪用シナリオ**: 攻撃者が `sudo passwd` の実行権限を持っている場合、rootを含む任意のユーザーのパスワードを変更し、アカウントを乗っ取ることができます。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: `passwd` コマンド自体にSUIDビットが設定されているのは正常な動作です（一般ユーザーが自身のパスワードファイル `/etc/shadow` を更新する必要があるため）。`sudo` の設定で `sudo passwd` が許可されていて、かつ `root` のパスワード変更が許可されている場合、攻撃者は `sudo passwd root` を実行してrootのパスワードを変更し、権限を奪取できます。
* **参照**: `https://gtfobins.github.io/gtfobins/passwd/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: PAM (`pam_pwquality`など) を使って強力なパスワードポリシーを強制する。`sudoers` の設定を厳格にし、一般ユーザーに `passwd` の `sudo` 権限を与えない。
* **Detection (検知)**: `auditd` で `/etc/shadow` への変更を監視する。`passwd` コマンドの実行ログを監査し、意図しないパスワード変更がないか確認する。

## 注意点・補足

* **パスワードなしアカウント**: `passwd -d <user>` は、そのユーザーのパスワード認証を無効にします。セキュリティ上、極めて危険なため、特別な理由がない限り使用すべきではありません。アカウントを無効化したい場合は `-l` (`--lock`) を使います。
