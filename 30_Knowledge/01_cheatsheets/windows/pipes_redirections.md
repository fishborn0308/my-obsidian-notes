# Windows パイプとリダイレクション

## 概要

パイプ (`|`) とリダイレクション (`>`, `>>`, `<`) は、コマンドの標準入力 (stdin)、標準出力 (stdout)、標準エラー出力 (stderr) の流れを制御するための、コマンドラインにおける基本的な機能です。

* **リダイレクション**: コマンドの出力を、通常表示される画面ではなくファイルに書き込んだり、逆にファイルの内容をコマンドの入力として使ったりします。
* **パイプ**: あるコマンドの出力を、別のコマンドの入力として直接渡します。これにより、複数のコマンドを連結させて、より複雑な処理を一行で実現できます。

これらの機能を使いこなすことは、コマンドラインでの効率的な操作とスクリプト作成の鍵となります。

---

## 1. リダイレクション (Redirection)

コマンドの出力先や入力元を切り替えます。

| 演算子 | 説明 | 使用例 |
| :--- | :--- | :--- |
| **`>`** | **出力の上書きリダイレクト**。コマンドの標準出力 (stdout) をファイルに書き込みます。ファイルが既に存在する場合は、その内容を**上書き**します。 | `dir > file_list.txt` |
| **`>>`** | **出力の追記リダイレクト**。コマンドの標準出力 (stdout) をファイルの末尾に**追記**します。ファイルが存在しない場合は新規作成されます。 | `echo Log message >> app.log` |
| **`<`** | **入力のリダイレクト**。ファイルの内容をコマンドの標準入力 (stdin) として渡します。 | `sort < unsorted_list.txt` |
| **`2>`** | **標準エラー出力の上書きリダイレクト**。コマンドのエラー出力 (stderr) のみをファイルに書き込みます。 | `dir non_existent_file 2> error.log` |
| **`2>>`** | **標準エラー出力の追記リダイレクト**。コマンドのエラー出力 (stderr) のみをファイルの末尾に追記します。 | `command 2>> error_log.txt` |
| **`&>`** | **標準出力と標準エラー出力の両方をリダイレクト**。`>` と `2>` をまとめたものです。(`command > file.txt 2>&1` と同等) | `command &> all_output.log` |
| **`nul`** | **nullデバイス**。出力を捨てたい場合に使用する特殊なデバイス名です。 | `pause > nul` (メッセージを非表示) |

### シナリオ例1 (インフラ構築・運用視点)

* **目的**: コマンドの実行結果とエラーの両方を、1つのログファイルにまとめて記録する。
* **コマンド例**:

    ```cmd
    REM robocopyの成功・失敗メッセージを全てbackup.logに記録
    robocopy C:\source D:\backup /S > backup.log 2>&1
    ```

## 2. パイプ (Pipe)

あるコマンドの標準出力を、次のコマンドの標準入力に直接接続します。

| 演算子 | 説明 | 使用例 |
| :--- | :--- | :--- |
| **`\|`** | **パイプ**。左側のコマンドの標準出力を、右側のコマンドの標準入力に渡します。 | `dir \| find "txt"` |

### シナリオ例2 (インフラ構築・運用視点)

* **目的**: 実行中のサービスの中から、"Windows" という単語を含むサービスだけをフィルタリングして表示する。
* **連携コマンド**: `sc`, `findstr`
* **解説**: `sc query state= all` で全てのサービスをリストアップし、その大量の出力をパイプで `findstr` に渡して、キーワードに一致する行だけを絞り込みます。
* **コマンド例**:

    ```cmd
    REM 全サービスの中から "Windows" を含むものを検索
    sc query state= all | findstr /I "Windows"
    ```

## セキュリティに関する考慮事項

### レッドチーム視点

* **偵察 (Reconnaissance)**: パイプとリダイレクションは、偵察活動で得られた大量の情報から、目的のキーワード（"password", "admin"など）を効率的に絞り込むために不可欠です。

    ```cmd
    REM システム情報からプロセッサ情報を抜き出す
    systeminfo | findstr /B /C:"Processor(s)"
    ```

* **防御回避 (Defense Evasion)**:
  * `> nul` や `2> nul` を使うことで、スクリプト実行時にコンソールに表示されるエラーメッセージや出力を抑制し、スクリプトの真の目的を隠蔽しようとします。
  * `echo` と `>` を使って、悪意のあるスクリプトファイル (`.vbs`, `.ps1`) をターゲットシステム上に「組み立てる」ことがあります。これは、ファイル転送が監視されている環境を回避する手法です。

    ```cmd
    REM 悪意のあるVBScriptを一行ずつ書き出す
    echo Set oShell = CreateObject("WScript.Shell") > %TEMP%\evil.vbs
    echo oShell.Run "calc.exe" >> %TEMP%\evil.vbs
    ```

### ブルーチーム視点

* **ログ分析**: `type`, `findstr`, `sort`, `more` などをパイプで連結することで、巨大なログファイルから特定のイベントを効率的に抽出し、調査できます。

    ```cmd
    REM セキュリティログのバックアップから、特定のイベントIDを検索してページごとに表示
    type security.evtx.txt | findstr "4625" | more
    ```

* **監視**: 悪意のあるスクリプトはしばしば出力を `> nul` で隠蔽します。コマンドライン引数のロギング（イベントID 4688）を有効にしている場合、`> nul` のようなリダイレクションを含む不審なコマンド実行を検知のトリガーとできます。
