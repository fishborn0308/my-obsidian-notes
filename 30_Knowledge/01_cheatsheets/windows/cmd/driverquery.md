# `driverquery` - インストールされているデバイスドライバーを表示する

## 概要

`driverquery` コマンドは、システムにインストールされている全てのデバイスドライバーに関する情報を一覧表示するためのユーティリティです。ドライバーの表示名、種類、状態、ロードされた日時などを確認でき、システムのトラブルシューティングや、ハードウェアが正しく認識されているかの確認に役立ちます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`systeminfo`** | `systeminfo` はOSのバージョンやCPU情報など、より広範なシステム情報を表示します。`driverquery` は**デバイスドライバーに特化**しています。 | - |
| **`sc query`** | **サービス**の状態を照会します。多くのドライバーはサービスとして動作しますが、`driverquery` はよりドライバー中心の視点で情報を提供します。 | `sc` (サービス) |
| **PowerShell** | `Get-WmiObject Win32_SystemDriver` や `Get-Service` を使うことで、ドライバー情報をオブジェクトとして取得し、より高度なフィルタリングや処理が可能です。 | PowerShell (スクリプト) |

## よく連携されるコマンド

`driverquery` の出力は非常に多いため、`findstr` と連携して特定のドライバー情報を絞り込むのが一般的です。

### シナリオ例: 特定の種類のドライバーをリストアップする (インフラ構築・運用視点)

* **目的**: システムにインストールされている、ファイルシステムに関連するドライバーのみをリストアップする。
* **連携コマンド**: `findstr`
* **解説**: `driverquery /V` で詳細な情報を出力し、その結果をパイプで `findstr` に渡して "File System" という文字列でフィルタリングします。
* **コマンド例**:

    ```cmd
    REM ファイルシステムドライバーのみを抽出して表示
    driverquery /V | findstr /I "File System"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **接続先指定** | |
| `/S <system>` | 接続するリモートコンピュータを指定します。 |
| `/U <user>` | 実行するユーザーコンテキストを指定します。 |
| `/P <password>` | 指定したユーザーのパスワードを指定します。 |
| **出力形式** | |
| ⭐ `/FO <format>` | 出力形式を指定します。`TABLE` (デフォルト), `LIST`, `CSV` があります。 |
| ⭐ `/NH` | (No Header) テーブル出力のヘッダー行を表示しません。 |
| **表示内容** | |
| ⭐ `/V` | 詳細な情報を表示します。ドライバーの状態 (Running/Stopped) などが含まれます。 |
| `/SI` | 署名情報を持つドライバーに関する情報を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システムのトラブルシューティングのため、現在実行中の全ドライバーの一覧をCSV形式で取得する。
* **組み合わせ**: `/V /FO CSV`
* **解説**: 詳細なドライバー情報 (`/V`) をCSV形式 (`/FO CSV`) で出力することで、Excelなどで簡単にソートやフィルタリングができ、問題のあるドライバーの特定が容易になります。
* **例**:

    ```cmd
    REM 実行中の全ドライバーのリストを drivers.csv として保存
    driverquery /V /FO CSV > drivers.csv
    ```

### 2. ブルーチーム視点

* **タスク**: ルートキットや不審なカーネルモードドライバーがインストールされていないか監査する。
* **組み合わせ**: `driverquery /V`
* **解説**: `driverquery /V` で得られるリストを、正常な状態のベースラインと比較します。見慣れない名前のドライバーや、発行元が不明なドライバーが `Running` 状態である場合、マルウェアの可能性があります。特に `Kernel` タイプのドライバーは注意深く調査する必要があります。
* **例**:

    ```cmd
    REM 現在ロードされているカーネルドライバーをリストアップして確認
    driverquery /V | findstr "Kernel"
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストにインストールされているセキュリティ製品を特定する。
* **組み合わせ**: `driverquery`
* **解説**: 攻撃者は侵入後、`driverquery` を実行して、アンチウイルス、EDR、HIPSなどのセキュリティ製品がどのようなドライバーをロードしているかを特定します。`symantec`, `mcafee`, `crowdstrike` といったベンダー名を含むドライバーが見つかれば、それらの製品のバイパスや無効化を試みるための次のステップに進みます。
* **例**:

    ```cmd
    REM セキュリティ製品に関連しそうなキーワードでドライバーを検索
    driverquery | findstr /I "firewall antivirus defender filter"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ例 1**: `ERROR: The RPC server is unavailable.`
    * **考えられる原因**: リモートコンピュータを指定して実行 (`/S`) しましたが、ファイアウォールでブロックされているか、必要なサービスがリモート側で実行されていません。
    * **解決策**: リモートコンピュータのWindows Firewallで「リモート管理」が許可されていること、および "Remote Procedure Call (RPC)" サービスが実行中であることを確認してください。

## 環境変数と設定ファイル

* `driverquery` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `driverquery` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、攻撃者はこのコマンドの出力からシステムのセキュリティ体制を詳細に把握し、検知を回避するための計画を立てます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `driverquery` は、攻撃者が侵入後の情報収集フェーズで、セキュリティ製品を特定するために利用する典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `driverquery.exe` の実行をプロセス監査ログ (イベントID 4688) で監視します。特に、一般ユーザーによる実行や、不審なプロセスからの呼び出しはアラートの対象とすべきです。

## 注意点・補足

* **情報量**: `/V` スイッチを付けると非常に多くの情報が出力されるため、通常は `findstr` と組み合わせるか、CSV形式でファイルに出力して分析するのが効率的です。
