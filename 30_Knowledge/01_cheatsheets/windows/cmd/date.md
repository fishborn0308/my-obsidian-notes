# `date` - システムの現在の日付を表示・設定する

## 概要

`date` コマンドは、システムの現在の日付を表示、または（管理者権限で）設定するための基本的なコマンドです。引数なしで実行すると、現在の日付を表示し、新しい日付の入力を求めます。バッチファイル内で `%DATE%` 変数と組み合わせて、ログやファイル名にタイムスタンプを付与するためによく利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`time`** | システムの**時刻**を表示・設定します。`date` は**日付**を扱います。 | - |
| **PowerShell** | `Get-Date` コマンドレットは、日付と時刻をオブジェクトとして取得でき、`-Format` オプションで遥かに柔軟な書式設定が可能です。 | `Get-Date` (PowerShell) |

## よく連携されるコマンド

`date` は、その出力をファイル名やログメッセージに埋め込むために、`echo` コマンドやリダイレクションと連携するのが一般的です。

### シナリオ例: タイムスタンプ付きのバックアップファイル名を作成する (インフラ構築・運用視点)

* **目的**: どの日のバックアップか一目で分かるように、ファイル名に現在の日付を含める。
* **連携コマンド**: `copy`, `%DATE%` 変数
* **解説**: `%DATE%` 変数は `YYYY/MM/DD` のような形式で現在の日付を保持しています。これをファイル名の一部として利用します。ただし、`/` はファイル名に使えないため、置換構文 (`%DATE:/=%`) を使って取り除きます。
* **コマンド例**:

    ```cmd
    REM config.ini を config_YYYYMMDD.ini の形式でコピー
    copy config.ini config_%DATE:/=%.ini
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示と設定** | |
| ⭐ `(引数なし)` | 現在の日付を表示し、新しい日付の入力を求めます。何も入力せずEnterキーを押すと、日付は変更されません。 |
| ⭐ `/T` | 新しい日付の入力を求めず、**現在の日付のみを表示**してコマンドを終了します。 |
| **`<date>`** | (管理者権限要) `MM-DD-YY` 形式で、システムに設定する新しい日付を直接指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: バッチスクリプトのログに、実行日のタイムスタンプを記録する。
* **組み合わせ**: `echo %DATE%`
* **解説**: スクリプトの実行開始時や終了時に、現在の日付をログファイルに追記することで、いつ処理が行われたかを記録します。
* **例**:

    ```cmd
    REM ログファイルに日付ヘッダーを追記
    echo ========== Log for %DATE% ========== >> C:\Logs\app_log.txt
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンスの初期対応**として、調査対象システムの正確な現在時刻を記録する。
* **組み合わせ**: `date /T && time /T`
* **解説**: インシデント調査の開始時に、システムのタイムスタンプが正確であるか、あるいは攻撃者によって変更されていないかを確認し、証拠保全の一環として記録します。これは、イベントログのタイムラインを正確に把握するための基本です。
* **例**:

    ```cmd
    REM 調査開始時の日時を記録
    echo Investigation started at: > C:\Forensics\timeline.txt
    date /T >> C:\Forensics\timeline.txt
    time /T >> C:\Forensics\timeline.txt
    ```

### 3. レッドチーム視点

* **タスク**: **タイムスタンプ操作 (Timestomping)** の準備、またはシステムの時刻設定を調査する。
* **組み合わせ**: `date`
* **解説**: 攻撃者は侵入後、`date` や `time` を実行してシステムの現在時刻を確認します。これは、ログのタイムスタンプを偽装したり、時間ベースのセキュリティメカニズムを回避したりするための偵察活動です。管理者権限を奪取した後は、`date <new_date>` を実行してシステムの時刻を意図的にずらし、ログの追跡を困難にさせようと試みます。
* **例**:

    ```cmd
    REM (権限昇格後) システムの日付を変更してログのタイムスタンプを混乱させる
    date 01-01-2000
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ例 1**: `A required privilege is not held by the client.`
    * **考えられる原因**: 日付を変更しようとしましたが、コマンドプロンプトが管理者として実行されていません。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `date` コマンドの出力形式は、OSの地域設定に影響されます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

### 影響を与える環境変数

* **`%DATE%`**: 現在の日付を `YYYY/MM/DD` (地域設定による) 形式の文字列で保持する動的な環境変数。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `date` 自体ではなく、**システム時刻の不正な変更**がセキュリティリスクとなります。
* **悪用シナリオ**:
  * **ログの無効化**: 攻撃者がシステムの時刻を未来や過去のありえない日時に設定すると、ログのタイムスタンプが不正確になり、インシデントの追跡が非常に困難になります。
  * **認証のバイパス**: Kerberos認証など、時刻同期が重要な認証プロトコルを妨害する。
  * **証明書の無効化**: 時刻をずらすことで、TLS/SSL証明書を意図的に有効期限切れの状態にし、通信エラーを引き起こす。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**
* **解説**: `date` は、攻撃者がログのタイムスタンプを操作してフォレンジック調査を妨害するために利用される環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: NTP (Network Time Protocol) を使って、ドメインコントローラや信頼できるタイムサーバーとシステム時刻を常に同期させる。一般ユーザーにシステム時刻の変更権限を与えない。
* **Detection (検知)**: システム時刻の大幅な変更を監視する (イベントID 4616)。`date` コマンドの実行をプロセス監査で監視する。

## 注意点・補足

* **書式の違い**: `%DATE%` 変数の書式は、Windowsの「地域と言語」の設定に依存します。`YYYY/MM/DD` 形式を前提としたバッチファイルは、異なる地域設定のPCでは意図通りに動作しない可能性があるため注意が必要です。
