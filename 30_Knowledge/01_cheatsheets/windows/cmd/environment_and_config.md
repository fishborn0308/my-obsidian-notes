# Windows CMD 環境変数と設定

このドキュメントでは、Windowsのコマンドプロンプト (CMD) およびシステム全体に影響を与える主要な環境変数と設定について解説します。

---

## 1. 主要な環境変数

| 環境変数名 | 説明 | セキュリティに関する考慮事項 |
| :--- | :--- | :--- |
| `Path` | コマンドが実行された際に、CMDが実行可能ファイルを探すディレクトリのリスト。`;` で区切られます。 | **不審なパスの追加**: 攻撃者が悪意のあるディレクトリを`Path`に追加し、正規のコマンドに見せかけた悪意のある実行ファイルを優先的にロードさせることがあります。<br>**環境変数の汚染**: システムまたはユーザーレベルで`Path`への変更を監視すべきです。 |
| `TEMP` / `TMP` | 一時ファイルが作成されるディレクトリのパス。 | 一時ディレクトリに機密情報が書き込まれる可能性があります。また、マルウェアが一時ファイルを生成する場所としても利用されやすいため、監視対象となります。 |
| `USERNAME` | 現在のユーザー名。 | ユーザー名の表示に使用されます。 |
| `COMPUTERNAME` | コンピュータ名。 | ネットワーク上での識別に使用されます。 |
| `windir` | Windowsのシステムディレクトリのパス(例: `C:\Windows`)。 | システムファイルへのアクセスパスを提供します。 |
| `APPDATA` | アプリケーションデータが保存されるディレクトリのパス(ユーザー固有)。 | 多くのアプリケーションが設定やデータをここに保存するため、マルウェアの永続化メカニズムや設定ファイルの場所として悪用されることがあります。 |
| `LOCALAPPDATA` | アプリケーションデータが保存されるディレクトリのパス(ユーザー固有、ローミングされない)。 | `APPDATA`と同様に、攻撃者が情報を隠したり、実行ファイルを置いたりする場所として利用される可能性があります。 |
| `PUBLIC` | 公開共有ドキュメントのパス。 | ユーザー間でファイルを共有するために利用されますが、不適切なファイル共有設定は情報漏洩のリスクを高めます。 |
| `PROGRAMFILES` | 64ビットプログラムのインストールディレクトリのパス。 | 実行可能ファイルが保存される主要な場所です。 |
| `PROGRAMFILES(X86)` | 32ビットプログラムのインストールディレクトリのパス。 | 実行可能ファイルが保存される主要な場所です。 |

## 2. 主要な設定とファイル

| 設定/ファイル名 | 説明 | セキュリティに関する考慮事項 |
| :--- | :--- | :--- |
| **レジストリ** | Windowsの設定情報を階層構造で保存するデータベース。アプリケーション設定、システム設定、ユーザー設定などが含まれます。`reg query`, `reg add`, `reg delete`などのコマンドで操作できます。 | **永続化メカニズム**: 攻撃者がシステムの起動時やユーザーログイン時に悪意のあるプログラムを実行させるための永続化(Runキー、Servicesなど)に頻繁に悪用されます。<br>**情報窃取**: 認証情報や設定情報がレジストリに保存されている場合があり、これらを窃取される可能性があります。 |
| `hosts`ファイル (`C:\Windows\System32\drivers\etc\hosts`) | ホスト名とIPアドレスのマッピングを定義するファイル。DNSよりも優先されます。 | **ホスト名偽装**: 攻撃者がこのファイルを改ざんし、正規のドメイン名を悪意のあるIPアドレスに誘導する可能性があります。これにより、フィッシングやマルウェアのダウンロードに利用されることがあります。<br>**適切なアクセス制御**: 信頼できるユーザーのみが編集できるよう、パーミッションを厳しく設定すべきです。 |
| **イベントログ** | システム、セキュリティ、アプリケーションなどのカテゴリに分けられたイベント(ログ)を記録します。イベントビューアー (`eventvwr.msc`) で確認できます。 | **監視の基盤**: セキュリティイベントログは、不正なログイン試行、特権昇格、プロセスの作成、ファイルアクセスなどの重要な活動を記録するため、監視の最も重要な情報源です。<br>**ログの改ざん/削除**: 攻撃者は自身の足跡を消すためにログをクリアしようとします。ログのバックアップ、集中ログ管理(SIEM)、ログ保全(Immutable属性、Systemmon)が重要です。 |
| **グループポリシー** | Active Directory環境において、ユーザーやコンピュータのセキュリティ設定、ソフトウェアインストール、スクリプト実行などを集中管理するための仕組み。`gpresult`, `gpupdate`などのコマンドで確認・適用できます。 | **セキュリティベースライン**: 組織のセキュリティポリシーをシステムに適用する主要な手段です。不適切な設定はシステム全体のセキュリティレベルを低下させます。<br>**悪用**: 攻撃者がグループポリシーの不備を利用して特権昇格や永続化を行う可能性があります。 |
| `boot.ini` (`C:\boot.ini`, 旧Windows) / **BCD (Boot Configuration Data)** (`bcdedit`) | システムの起動設定ファイル。OSの選択、起動オプションなどが定義されます。`boot.ini`は古いWindowsで、新しいWindowsではBCDが使われます。 | **起動オプションの変更**: 攻撃者が起動オプションを変更し、セーフモードで起動させたり、デバッグモードを有効にしたりする可能性があります。これにより、システムへの不正アクセスが容易になることがあります。 |
| `AUTOEXEC.BAT` (`C:\AUTOEXEC.BAT`, 旧Windows) / **スタートアップフォルダー** | 古いWindowsの起動時に自動実行されるバッチファイル。現在ではスタートアップフォルダーやレジストリのRunキーが同様の目的で使用されます。 | **永続化**: 攻撃者がこの場所に悪意のあるスクリプトや実行ファイルを配置し、システム起動時に自動的に実行させることで永続化を図ることがあります。スタートアップフォルダー、レジストリRunキー、サービスなどを定期的に確認すべきです。 |
| `schtasks` (タスクスケジューラ) | 特定の時刻やイベントでプログラムを自動実行するための機能。CMDの`schtasks`コマンドで管理できます。 | **永続化**: 攻撃者がタスクスケジューラに悪意のあるタスクを登録し、定期的にマルウェアを実行したり、コマンドを呼び出したりすることで、システムへの永続的なアクセスを維持することがあります。 |
