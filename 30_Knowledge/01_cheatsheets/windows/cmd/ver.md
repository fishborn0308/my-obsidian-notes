# `ver` - Windowsのバージョン情報を表示する

## 概要

`ver` (version) コマンドは、現在実行中のコマンドプロンプトがどのWindowsバージョンで動作しているかを、1行のシンプルなテキストで表示するためのコマンドです。出力には、OSのメジャーバージョン、マイナーバージョン、ビルド番号などが含まれます。スクリプトの互換性を確認したり、システムの基本的な情報を素早く取得したりするために使用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`systeminfo`** | `systeminfo` は、OSバージョンに加え、インストール日、CPU、メモリ、パッチレベルなど、はるかに包括的な情報を表示します。`ver` はバージョン番号のみを素早く表示します。 | `systeminfo` (詳細情報) |
| **PowerShell** | `$PSVersionTable` はPowerShell自体のバージョン情報を、`Get-ComputerInfo` はOSの詳細情報をオブジェクトとして取得できます。 | `Get-ComputerInfo` |

## よく連携されるコマンド

`ver` コマンドの出力は、バッチファイル内で特定のWindowsバージョンでのみ処理を実行したい場合に、`findstr` と連携して利用されることがあります。

### シナリオ例: OSバージョンに基づいた処理の分岐 (インフラ構築・運用視点)

* **目的**: 実行環境が特定のWindowsバージョン（例: Version 10.x）であるかどうかを確認し、処理を分岐させる。
* **連携コマンド**: `findstr`, `if`
* **解説**: `ver` の出力をパイプで `findstr` に渡し、特定のバージョン文字列が含まれているかをチェックします。`ERRORLEVEL` を使って、`findstr` が文字列を見つけたかどうかを判定し、`if` 文で処理を分岐させます。
* **コマンド例**:

    ```cmd
    @echo off
    ver | findstr /I "Version 10." > nul
    if %ERRORLEVEL% == 0 (
        echo This is a Windows 10 or later system.
        REM (Windows 10以降向けの処理)
    ) else (
        echo This is an older Windows system.
        REM (古いOS向けの処理)
    )
    ```

## スイッチ/オプション説明

`ver` コマンドには、コマンドラインスイッチやオプションはありません。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| ⭐ `(なし)` | 現在のWindowsのバージョン情報を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートサーバーに接続した際に、そのOSのバージョンを素早く確認する。
* **組み合わせ**: `ver`
* **解説**: `systeminfo` よりも遥かに高速に、OSの基本的なバージョンを把握できます。
* **例**:

    ```cmd
    REM 現在のOSバージョンを確認
    ver
    REM > Microsoft Windows [Version 10.0.19045.2006]
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンスの初期対応**として、調査対象ホストのOSバージョンを記録する。
* **組み合わせ**: `ver`
* **解説**: `systeminfo` と共に `ver` を実行し、OSのビルド番号を正確に記録します。これは、そのビルドに存在する既知の脆弱性を後から調査する上で重要な情報となります。
* **例**:

    ```cmd
    REM 調査対象ホストのバージョンを記録
    ver > C:\Forensics\os_version.txt
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストのOSバージョンを特定し、利用可能な脆弱性を探す。
* **組み合わせ**: `ver`
* **解説**: 攻撃者は侵入後、`systeminfo` と共に `ver` を実行し、OSのビルド番号を特定します。この情報を基に、特定のビルドにのみ影響する権限昇格の脆弱性（カーネルエクスプロイトなど）が存在するかどうかを判断します。
* **例**:

    ```cmd
    REM 侵入したホストのOSバージョンを偵察
    ver
    ```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ver` 自体は読み取り専用のツールですが、その出力が**重大な情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、OSの正確なバージョンとビルド番号は、攻撃者が権限昇格のための脆弱性を探し、悪用するための最も重要な情報の1つです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `ver` は `cmd.exe` の内部コマンドであり、攻撃者が侵入後の情報収集フェーズで、システムの脆弱性を評価するために利用する、最も基本的な環境寄生型の手法です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 定期的にセキュリティパッチを適用し、システムを最新の状態に保つことが、`ver` によって得られる情報が悪用されるリスクを低減する最も効果的な対策です。
* **Detection (検知)**: `ver` (`cmd.exe`) の実行は非常に一般的であるため、単体での検知は困難です。しかし、不審なプロセスからの実行や、他の偵察コマンド (`whoami`, `hostname`, `systeminfo`など) と連続して実行される場合は、攻撃の兆候である可能性があります。

## 注意点・補足

* **`cmd.exe` のバージョン**: `ver` はOSのバージョンを表示しますが、コマンドプロンプト自体のバージョンは `cmd /?` の出力の先頭行などで確認できます。
