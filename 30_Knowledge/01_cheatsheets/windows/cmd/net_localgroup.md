# `net localgroup` - ローカルグループを管理する

## 概要

`net localgroup` コマンドは、ローカルコンピュータ上のユーザーグループを追加、表示、変更するために使用されます。このコマンドを使って、特定の権限を持つローカルグループを作成したり、そのグループにローカルユーザーやドメインユーザーを追加・削除したりできます。スタンドアロンのサーバーやワークステーションのアクセス権管理における基本的なツールです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`net group`** | Active Directory **ドメイン**のグローバルグループを管理します。`net localgroup` は**ローカルコンピュータ**上のグループを管理します。 | - |
| **PowerShell** | `Get-LocalGroup`, `New-LocalGroup`, `Add-LocalGroupMember` などのコマンドレットは、グループ管理をオブジェクト指向で行え、より強力でスクリプト化に適しています。 | `*-LocalGroup*` (PowerShell) |

## よく連携されるコマンド

`net localgroup` は、システムの権限構造を調査するために、他の `net` コマンドと連携して偵察活動で多用されます。

### シナリオ例: ローカルのAdministratorsグループのメンバーを確認する (ブルーチーム/レッドチーム視点)

* **目的**: そのコンピュータで管理者権限を持つアカウントを特定する。
* **連携コマンド**: (なし)
* **解説**: システム侵入後や監査時に、誰がそのマシンの管理者権限を持っているかを特定するための最も基本的なコマンドの1つです。
* **コマンド例**:

    ```cmd
    REM "Administrators" グループのメンバーを一覧表示
    net localgroup "Administrators"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | サーバー上の全てのローカルグループ名を表示します。 |
| ⭐ `<groupname>` | 指定したグループのメンバーを一覧表示します。 |
| `<groupname> /add` | 新しいローカルグループを作成します。 |
| `<groupname> /delete` | ローカルグループを削除します。 |
| **メンバー管理** | |
| ⭐ `<groupname> <username> /add` | ユーザーをローカルグループに追加します。 |
| ⭐ `<groupname> <username> /delete` | ユーザーをローカルグループから削除します。 |
| **その他** | |
| `/comment:"<text>"` | グループにコメントを追加します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートデスクトップ接続を許可するためのグループに、特定のユーザーを追加する。
* **組み合わせ**: `<groupname> <username> /add`
* **解説**: "Remote Desktop Users" グループにユーザーを追加することで、そのユーザーにリモートデスクトップでのログイン権限を付与します。
* **例**:

    ```cmd
    REM ユーザー 'devuser01' を "Remote Desktop Users" グループに追加
    net localgroup "Remote Desktop Users" devuser01 /add
    ```

### 2. ブルーチーム視点

* **タスク**: **権限昇格の監査**。特権グループのメンバーシップを確認する。
* **組み合わせ**: `net localgroup "<privileged_group>"`
* **解説**: **アクセス権管理の基本**。`Administrators` や `Remote Desktop Users` などの強力な権限を持つグループのメンバーを定期的に確認し、意図しないユーザー（特に攻撃者によって作成されたアカウント）が所属していないかを監査します。
* **例**:

    ```cmd
    REM 管理者グループのメンバーを確認
    net localgroup "Administrators"
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)**。自身が制御するアカウントを特権グループに追加する。
* **組み合わせ**: `net localgroup <groupname> <user> /add`
* **解説**: 攻撃者は権限昇格に成功した後、バックドアとして作成したアカウントや、既に存在するアカウントを `Administrators` や `Remote Desktop Users` グループに追加します。これにより、システムの再起動後も管理者権限でのアクセスやリモートログインを維持しようとします。
* **例**:

    ```cmd
    REM 攻撃者が作成した 'backdoor_user' を管理者グループに追加
    net localgroup Administrators backdoor_user /add
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'net'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: グループの作成やメンバーの変更には、管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

2. **エラーメッセージ例 1**: `The user name could not be found.`
    * **考えられる原因**: グループに追加しようとしたユーザーアカウントが存在しません。
    * **解決策**: `net user` コマンドでユーザー名が正しいか確認してください。

## 環境変数と設定ファイル

* `net localgroup` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `net localgroup` 自体ではなく、**不適切なグループメンバーシップの管理**がセキュリティリスクとなります。
* **悪用シナリオ**: 攻撃者は、権限昇格に成功すると、自身が制御するアカウントを `Administrators` グループに追加して永続化を図ります。これは攻撃の常套手段です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **永続化 (Persistence)**
* **解説**: `net localgroup` は、攻撃者がシステムの権限構造を理解し、特権グループを発見するために、侵入後の初期段階で必ずと言っていいほど使用する環境寄生型の偵察ツールです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、特権グループへの所属は必要最小限のユーザーに限定する。
* **Detection (検知)**: Windows セキュリティイベントログを監視し、**イベントID 4732 (メンバーがセキュリティが有効なローカル グループに追加されました)** の発生を厳重に監視する。特に、意図しないアカウントが `Administrators` グループに追加された場合は、即座にアラートを発報するように設定する。

## 注意点・補足

* **グループ名にスペースがある場合**: `net localgroup "Remote Desktop Users"` のように、グループ名はダブルクォートで囲む必要があります。
