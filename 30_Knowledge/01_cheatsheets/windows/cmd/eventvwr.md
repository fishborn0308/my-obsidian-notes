# `eventvwr` - Windows イベントビューアーを起動する

## 概要

`eventvwr` コマンドは、Windowsの**イベントビューアー** (`eventvwr.msc`) を起動するためのコマンドです。イベントビューアーは、OS、サービス、アプリケーションが動作中に記録したイベント（ログ）を閲覧、検索、フィルタリング、エクスポートするためのグラフィカルな管理ツールです。システムのトラブルシューティングからセキュリティインシデントの調査まで、Windows管理における最も重要な情報源の1つです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`wevtutil`** | イベントログを操作するための**コマンドライン**版ツール。`eventvwr` は **GUI** 版です。スクリプトによる自動処理には `wevtutil` が使われます。 | `wevtutil` (スクリプト) |
| **PowerShell** | `Get-WinEvent` コマンドレットは、ログをオブジェクトとして取得できるため、フィルタリングや集計が非常に強力です。 | `Get-WinEvent` (PowerShell) |
| **`auditpol`** | **何をログに記録するか**という監査ポリシーを設定します。`eventvwr` は**記録されたログを見る**ためのツールです。 | - |

## よく連携されるコマンド

`eventvwr` はGUIツールのため、直接的なコマンド連携はありません。しかし、他のコマンドで問題が発生した際に、その原因を調査するために起動されることが非常に多いです。

### シナリオ例: サービス起動失敗の原因を調査する (インフラ構築・運用視点)

* **目的**: `sc start` や `net start` でサービスの起動に失敗した際に、その詳細なエラー内容を確認する。
* **連携コマンド**: `sc`, `net start`
* **解説**: サービス起動に失敗すると、CMD上には「サービスを開始できませんでした」といった一般的なエラーしか表示されないことが多いです。`eventvwr` を起動し、「Windows ログ」>「システム」を開くことで、サービスコントロールマネージャーが記録した詳細なエラーイベント（イベントID 7000番台など）を確認できます。
* **コマンド例**:

    ```cmd
    REM サービスの起動を試みる
    net start "My Custom Service"
    REM > サービスを開始できませんでした。

    REM 詳細を調査するためにイベントビューアーを起動
    eventvwr
    ```

## スイッチ/オプション説明

`eventvwr` コマンド自体には一般的に使用されるスイッチはありません。`eventvwr.msc` を直接実行するのと同義です。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| `(なし)` | イベントビューアーのGUIを起動します。 |
| `/l:<ログファイル.evtx>` | 指定したログファイルを直接開きます。 |

### 2. ブルーチーム視点

* **タスク**: **不正アクセスの痕跡を調査する**。
* **組み合わせ**: (GUI操作: ログのフィルタリング)
* **解説**: **インシデント対応の基本**。`eventvwr` を起動し、「Windows ログ」>「セキュリティ」を開きます。右クリックメニューの「現在のログをフィルター」を使い、特定のイベントIDでログを絞り込みます。

  * **イベントID 4688**: プロセス作成 (不審なコマンド実行の追跡)
  * **イベントID 4720**: ユーザーアカウント作成 (バックドアアカウントの発見)

* **タスク**: **偵察 (Reconnaissance)** と**痕跡消去 (Anti-Forensics)**。
* **組み合わせ**: (GUI操作 / `wevtutil cl`)

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **現象**: イベントログに目的のイベントが記録されていない。
    * **考えられる原因**: 監査ポリシーが有効になっていません。例えば、プロセス作成を追跡するには、「プロセス作成の監査」ポリシーを有効にする必要があります。
    * **解決策**: `auditpol /set` コマンドやローカルセキュリティポリシー (`secpol.msc`) を使って、必要な監査ポリシーを有効化してください。

## 環境変数と設定ファイル

* イベントログファイルは通常 `C:\Windows\System32\winevt\Logs\` 配下に `.evtx` 形式で保存されます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `eventvwr.exe` 自体が、UACバイパスの手法として悪用されることがあります。
* **悪用シナリオ**: 攻撃者はレジストリを改ざんし、`eventvwr.exe` がヘルプファイルを開く際のパスを、悪意のある実行ファイルへのパスに差し替えます。これにより、`eventvwr.exe` を（管理者権限で）実行した際に、裏で悪意のあるコードが実行されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **UACバイパス**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Eventvwr/](https://lolbas-project.github.io/lolbas/Binaries/Eventvwr/)
* **解説**: `eventvwr.exe` はMicrosoft署名付きの信頼されたバイナリであるため、UACプロンプトなしで管理者権限に昇格できます。この性質を悪用し、レジストリ操作と組み合わせることで、権限昇格の足がかりとして利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**:
  * **ログの集中管理**: 最も重要な対策。イベントログをリアルタイムでSIEMなどの外部ログサーバーに転送することで、ホスト上でログがクリアされても証拠を保全できます。
  * **ログクリアの監視**: イベントログがクリアされると、**イベントID 1102** が記録されます。このイベントを検知したら、即座にアラートを発報するように設定します。
  * `eventvwr.exe` による不審な子プロセスの生成をEDRなどで監視します。

## 注意点・補足

* **ログの量**: 詳細な監査ポリシーを有効にすると、イベントログは非常に速く肥大化します。ログの最大サイズを適切に設定し、定期的にアーカイブする運用が必要です。
