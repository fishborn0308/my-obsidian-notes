# `icacls` - ファイルとフォルダーのアクセス制御リスト (ACL) を管理する

## 概要

`icacls` (Integrity Control Access Control List) コマンドは、ファイルやフォルダーに対するアクセス許可（パーミッション）を定義する**アクセス制御リスト (ACL)** を表示、変更、バックアップ、復元するための強力なコマンドラインツールです。旧来の `cacls` コマンドの後継であり、NTFSファイルシステムのセキュリティを詳細に制御する上で不可欠です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`cacls`** | `icacls` の前身となった古いコマンド。`cacls` は継承の扱いや拒否ルールの設定に制限があり、現在は `icacls` の使用が強く推奨されます。 | `icacls` |
| **`attrib`** | ファイルの**属性**（読み取り専用、隠しファイルなど）を変更します。`icacls` は「誰が」「何ができるか」という**アクセス権**を制御します。 | - |
| **PowerShell** | `Get-Acl` と `Set-Acl` コマンドレットは、ACLをオブジェクトとして扱えるため、より複雑なロジックやスクリプトでの自動化に適しています。 | `Get-Acl`/`Set-Acl` |

## よく連携されるコマンド

`icacls` は、ファイルの所有権がないとパーミッションを変更できない場合に、`takeown` コマンドと連携して使われます。

### シナリオ例: アクセス権のリセット (インフラ構築・運用視点)

* **目的**: 誤ったアクセス権設定がされたフォルダーの所有権を管理者に変更し、パーミッションを既定の状態にリセットする。
* **連携コマンド**: `takeown`
* **解説**: まず `takeown` でフォルダーの所有権を Administrators グループに取得します。その後、`icacls` の `/reset` スイッチで、親から継承された既定のアクセス権にリセットします。
* **コマンド例**:

    ```cmd
    REM D:\data フォルダーの所有権を Administrators に変更
    takeown /F D:\data /R /A

    REM D:\data フォルダーのアクセス権を既定の状態にリセット
    icacls D:\data /reset /T
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `<object>` | 対象となるファイル名またはフォルダー名を指定します。 |
| `/grant <user>:<perms>` | 指定したユーザーにアクセス許可を与えます。既存の許可は維持されます。 |
| `/deny <user>:<perms>` | 指定したユーザーにアクセス拒否を明示的に設定します。DenyはGrantより優先されます。 |
| `/remove <user>` | 指定したユーザーの全ての許可および拒否のエントリを削除します。 |
| ⭐ `/reset` | ACLを既定の継承されたACLに置き換えます。 |
| **再帰と継承** | |
| ⭐ `/T` | 指定したディレクトリ配下の全てのファイルとサブディレクトリに対して操作を**再帰的に**実行します。 |
| `/inheritance:e` | 継承を**有効 (enable)** にします。 |
| `/inheritance:d` | 継承を**無効 (disable)** にし、継承されたACLをコピーします。 |
| `/inheritance:r` | 継承を**無効 (disable)** にし、継承されたACLを**削除 (remove)** します。 |
| **保存と復元** | |
| `/save <acl_file>` | ACLをテキストファイルにバックアップします。 |
| `/restore <acl_file>` | ファイルからACLを復元します。 |

### 主なアクセス許可 (`<perms>`)

* **F** - フルコントロール
* **M** - 変更
* **RX** - 読み取りと実行
* **R** - 読み取り専用
* **W** - 書き込み専用

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 共有フォルダー `D:\Share` に対して、`Users` グループに読み取りと実行権限を、`Admins` グループにフルコントロールを与える。
* **組み合わせ**: `/grant`
* **解説**: `/grant` を複数回使用して、ユーザーやグループごとに異なる権限を付与します。
* **例**:

    ```cmd
    REM まず継承を無効化して既存の権限をコピー
    icacls D:\Share /inheritance:d

    REM Users グループに読み取りと実行を許可 (OI)(CI)は継承設定
    icacls D:\Share /grant "Users":(OI)(CI)RX

    REM Admins グループにフルコントロールを許可
    icacls D:\Share /grant "Administrators":(OI)(CI)F
    ```

### 2. ブルーチーム視点

* **タスク**: **偵察 (Reconnaissance)**。システム上の重要なファイルやディレクトリのACLを監査する。
* **組み合わせ**: (引数なし) `icacls <file>`
* **解説**: `icacls C:\Windows\System32\cmd.exe` などを実行し、重要なシステムファイルに対して `Everyone` や `Authenticated Users` に書き込み権限のような過剰な許可が与えられていないかを確認します。これは設定ミスや攻撃者による改ざんの可能性があります。
* **例**:

    ```cmd
    REM C:\Windows\System32 ディレクトリのアクセス権を確認
    icacls C:\Windows\System32
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のため、ファイルのアクセス権を変更する。
* **組み合わせ**: `/grant`
* **解説**: 攻撃者は、自身が制御するユーザーアカウント（または `Everyone`）に対して、バックドアとして設置した実行ファイルやスクリプトへの実行権限を付与します。また、正規のプログラムを改ざんするために、一時的に自身へフルコントロールを付与することもあります。
* **例**:

    ```cmd
    REM C:\Users\Public\payload.exe に Everyone のフルコントロールを付与
    icacls C:\Users\Public\payload.exe /grant Everyone:F
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Successfully processed 0 files; Failed processing 1 files` および `Access is denied.`
    * **考えられる原因**: ファイルのACLを変更しようとしましたが、あなたのアカウントがそのファイルの**所有者**ではないか、または「アクセス許可の変更」権限を持っていません。
    * **解決策**: まず `takeown` コマンドでファイルの所有権を取得してから、再度 `icacls` を実行してください。コマンドプロンプトを「管理者として実行」する必要がある場合も多いです。

## 環境変数と設定ファイル

* `icacls` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `icacls` 自体ではなく、**不適切なアクセス権設定（弱いACL）**が重大な脆弱性となります。
* **悪用シナリオ**: 書き込み可能なシステムディレクトリや実行ファイルは、攻撃者によるマルウェア設置やDLLハイジャックの温床となります。`icacls` は、このような脆弱な設定を調査し、また悪用するための両面で使われるツールです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **防御回避 (Defense Evasion)**
* **解説**: `icacls` は、攻撃者が侵入後の偵察フェーズで、書き込み可能なファイルやディレクトリを探すために多用します。また、管理者権限を奪取した後は、`icacls` を使ってファイルのパーミッションを変更しバックドアを設置したり、セキュリティソフトのファイルを改ざんしたりします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **最小権限の原則**を徹底し、ユーザーやグループには必要最小限のアクセス権のみを付与する。
* **Detection (検知)**: `Sysmon` (イベントID 11: FileCreate, イベントID 13: RegistryEventなどに関連) やファイル整合性監視 (FIM) ツールで、重要なファイルやディレクトリのACLへの変更 (セキュリティ記述子の変更 - イベントID 4670) を監視する。

## 注意点・補足

* **継承の理解**: Windowsのアクセス権は、親フォルダーから子オブジェクトへ**継承**されるのが基本です。`/inheritance` スイッチの挙動を正しく理解しないと、意図しないアクセス権設定になる可能性があるため注意が必要です。
