# `attrib` - ファイル属性の表示・変更

## 概要

`attrib` コマンドは、ファイルやディレクトリが持つ「属性」を表示または変更するために使用されます。ファイル属性は、そのファイルをOSがどのように扱うかを制御するためのメタデータです。例えば、ファイルを誤って削除しないように「読み取り専用」にしたり、通常の `dir` コマンドでは表示されないように「隠しファイル」にしたりできます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`icacls`** | `icacls` は、より詳細な**アクセス権（パーミッション）**を制御します。`attrib` は、隠しファイルや読み取り専用といった、より単純な**属性**を扱います。 | `icacls` (アクセス権) |
| **PowerShell** | `Get-ItemProperty` や `Set-ItemProperty` を使うことで、属性をより柔軟に、スクリプト内でオブジェクトとして扱うことができます。 | PowerShell (スクリプト) |

## よく連携されるコマンド

`attrib` は、`dir` コマンドや `for` ループと連携して、属性に基づいたファイル操作を行う際に利用されます。

### シナリオ例: ディレクトリ内の全隠しファイルを表示する (ブルーチーム視点)

* **目的**: 特定のディレクトリとそのサブディレクトリに含まれる全ての隠しファイル (`+H`) をリストアップする。
* **連携コマンド**: `dir`
* **解説**: `dir` コマンドの `/A:H` スイッチは隠しファイルを表示し、`/S` はサブディレクトリも対象とします。これは `attrib` で `+H` が設定されたファイルを探すのと同じ目的を果たします。
* **コマンド例**:

    ```cmd
    REM C:\Users\Public ディレクトリ以下の全隠しファイルをリストアップ
    dir C:\Users\Public /A:H /S
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **属性の指定** | `+` で属性を設定、`-` で解除します。 |
| ⭐ `R` | **読み取り専用 (Read-only)** 属性。ファイルの内容変更や削除から保護します。 |
| ⭐ `A` | **アーカイブ (Archive)** 属性。ファイルが変更されたことを示し、バックアップソフトが利用します。 |
| ⭐ `S` | **システム (System)** 属性。OSが使用する重要なファイルであることを示します。 |
| ⭐ `H` | **隠しファイル (Hidden)** 属性。デフォルトのエクスプローラーや `dir` コマンドでは表示されなくなります。 |
| **操作対象** | |
| ⭐ `/S` | カレントディレクトリとその全てのサブディレクトリにあるファイルを処理します。 |
| ⭐ `/D` | ディレクトリ自体も処理の対象に含めます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 重要な設定ファイルを誤って編集・削除しないように保護する。
* **組み合わせ**: `+R`
* **解説**: `+R` で読み取り専用属性を付与することで、意図しない変更を防ぐ簡易的な安全策となります。
* **例**:

    ```cmd
    REM C:\App\config.ini を読み取り専用に設定
    attrib +R C:\App\config.ini
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによって隠されたファイルを発見する。
* **組み合わせ**: `attrib -H -S /S /D`
* **解説**: **フォレンジック調査の基本**。マルウェアは自身の存在を隠蔽するため、ファイルに `+H` (隠し) や `+S` (システム) 属性を付与することがよくあります。調査対象のディレクトリでこのコマンドを実行し、全てのファイルを表示させることで、隠されたマルウェアやツールキットを発見します。
* **例**:

    ```cmd
    REM 侵害されたUSBドライブ (E:) の全ファイル属性を解除して可視化する
    attrib -H -S E:\*.* /S /D
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** と**防御回避 (Defense Evasion)** のため、ペイロードやツールを隠蔽する。
* **組み合わせ**: `+H +S`
* **解説**: 攻撃者は、システムに設置したバックドアやツールキット、窃取したデータを一時保管するディレクトリなどを、`+H` (隠し) および `+S` (システム) 属性を付けて隠します。これにより、通常のGUIや `dir` コマンドでは発見されにくくなります。
* **例**:

    ```cmd
    REM C:\Users\Public\payload.exe を隠しファイルかつシステムファイルに設定
    attrib +H +S C:\Users\Public\payload.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'attrib'` is not recognized... は、[CMD共通のトラブルシューティングファイル](./troubleshooting_common_errors.md)で解説。

1. **エラーメッセージ例 1**: `Access denied - <ファイルパス>`
    * **考えられる原因**: ファイルの属性を変更するための権限がありません。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `attrib` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `attrib` 自体ではなく、ファイル属性の**隠蔽機能**が悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`attrib +H +S` は、マルウェアが自身の存在をユーザーや管理者から隠蔽するための古典的で非常に一般的な手法です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**
* **解説**: `attrib` はWindowsに標準搭載されている正規のコマンドであり、攻撃者はアンチウイルス製品に検知されにくい「環境寄生型」の攻撃手法としてこれを利用し、悪意のあるファイルを隠蔽します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**:
  * Windowsのエクスプローラーで「保護されたオペレーティングシステムファイルを表示しない」のチェックを外し、「隠しファイル、隠しフォルダー、および隠しドライブを表示する」を有効にする運用を徹底する。
  * `Sysmon` (イベントID 11: FileCreate) のログや、ファイル整合性監視(FIM)ツールで、予期せぬファイル作成と `attrib` コマンドの実行を監視する。

## 注意点・補足

* **PowerShellとの比較**: PowerShellでは `(Get-Item <ファイル>).Attributes` で属性を確認し、`$file.Attributes = "Hidden, System"` のようにしてより柔軟に属性を変更できます。
