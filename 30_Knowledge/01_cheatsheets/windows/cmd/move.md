# `move` - ファイルやディレクトリを移動・名前変更する

## 概要

`move` コマンドは、ファイルやディレクトリをある場所から別の場所へ**移動**したり、**名前を変更**したりするために使用されます。`ren` (rename) コマンドと異なり、`move` はディレクトリの名前変更も可能です。また、ファイルを別のドライブに移動することもできます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`copy`** | ファイルを**コピー**します。元のファイルはそのまま残ります。`move` はファイルを**移動**し、元のファイルはなくなります。 | - |
| **`ren` / `rename`** | ファイルやディレクトリの**名前変更のみ**を行います。別のディレクトリへの移動はできません。 | - |
| **`robocopy`** | 高機能なファイル/ディレクトリのコピー・移動ツール。`/MOVE` や `/MOV` オプションで移動が可能です。リトライ機能などがあり、より堅牢です。 | `robocopy` (堅牢な移動) |

## よく連携されるコマンド

`move` は、`for` ループと組み合わせて、特定の条件に一致するファイルを一括で別のディレクトリに移動させるバッチ処理でよく使われます。

### シナリオ例: 日付별로 로그 파일을 정리 (인프라 구축 및 운영 관점)

* **目的**: 今日の日付のディレクトリを作成し、現在の場所にある全ての `.log` ファイルをその中に移動する。
* **連携コマンド**: `mkdir`, `for`
* **解説**: `%DATE%` 変数を使って日付名のディレクトリを作成し、`for` ループ（またはワイルドカード）を使って対象のファイルを `move` します。
* **コマンド例**:

    ```cmd
    REM 今日の日付でディレクトリを作成 (YYYYMMDD形式)
    SET TODAY=%DATE:/=%
    mkdir %TODAY%

    REM 全ての.logファイルを新しいディレクトリに移動
    move *.log %TODAY%\
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **動作制御** | |
| ⭐ `/Y` | 移動先に同名のファイルが既に存在する場合、**上書きの確認プロンプトを表示しません**。 |
| `/-Y` | 移動先に同名のファイルが既に存在する場合、**上書きの確認プロンプトを表示します**。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ファイルの名前を変更する。
* **組み合わせ**: `move <old_name> <new_name>`
* **解説**: ファイルやディレクトリの名前を変更する最も基本的な使い方です。
* **例**:

    ```cmd
    REM report.txt を report_final.txt に名前変更
    move report.txt report_final.txt

    REM Old_Logs ディレクトリを Logs_Archived に名前変更
    move D:\Logs\Old_Logs D:\Logs\Logs_Archived
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアを隔離し、無力化する（封じ込め）。
* **組み合わせ**: `move <malware_path> <quarantine_path>`
* **解説**: `del` で削除すると証拠が失われるため、代わりに `move` を使って、マルウェアを隔離用の安全なディレクトリに移動させます。これにより、マルウェアを元の場所から無力化しつつ、後の詳細なフォレンジック分析のためにファイルを保全できます。
* **例**:

    ```cmd
    REM 不審なファイルを隔離ディレクトリに移動
    move C:\Users\Public\svchost.exe C:\Quarantine\
    ```

### 3. レッドチーム視点

* **タスク**: 正規のプログラムを悪意のあるものに置き換える（ハイジャック）。
* **組み合わせ**: `move`
* **解説**: **永続化 (Persistence)** や**防御回避 (Defense Evasion)** の一環。攻撃者は、まず正規のプログラム（例: `Util.exe`）を別の場所に移動またはリネームし、その場所にバックドアを仕込んだ同名の悪意のあるプログラムを `move` で配置します。
* **例**:

    ```cmd
    REM 正規のヘルパーツールを退避
    move C:\App\helper.exe C:\App\helper.exe.bak

    REM 悪意のあるペイロードを正規のツール名にリネームして配置
    move C:\Users\Public\payload.exe C:\App\helper.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'move'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `The process cannot access the file because it is being used by another process.`
    * **考えられる原因**: 移動しようとしているファイルが、別のプログラムによって開かれている（ロックされている）ため、移動できません。
    * **解決策**: Process Explorer (Sysinternals) などのツールで、どのプロセスがファイルをロックしているかを確認し、そのプロセスを終了させてから再試行します。

2. **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: ファイルを移動するための権限がありません。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* **`COPYCMD`**: `/Y` または `/-Y` のデフォルト設定をこの環境変数で指定できますが、`move` はこの変数を参照しない場合があります。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `move` 自体ではなく、**ファイルの不正な移動や置き換え**が悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、正規の実行ファイルを悪意のあるものに置き換える「実行パスハイジャック」や、設定ファイルを改ざんする目的で利用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**
* **解説**: `move` はWindowsの基本コマンドであり、その実行自体は警告を発しません。攻撃者はこのコマンドを悪用して、監視を逃れながら悪意のあるファイルを正規の場所に配置したり、既存のファイルを置き換えたりします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイルシステムに適切なACL（アクセス制御リスト）を設定し、重要なシステムファイルやアプリケーションディレクトリへの書き込み権限を最小限に抑える。
* **Detection (検知)**: `Sysmon` (イベントID 11: FileCreate) やファイル整合性監視(FIM)ツールで、重要なファイルの移動、名前変更、上書きを監視する。

## 注意点・補足

* **ディレクトリの移動**: `move` コマンドでディレクトリを移動する場合、そのディレクトリとその中身が全て移動します。
* **ドライブ間の移動**: `move` は異なるドライブ間（例: `C:` から `D:`）でファイルやディレクトリを移動できます。この場合、内部的にはコピー＆デリートの動作となります。
