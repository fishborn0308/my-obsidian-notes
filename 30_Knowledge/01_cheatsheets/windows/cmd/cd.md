# `cd` - カレントディレクトリの表示・変更

## 概要

`cd` (Change Directory) コマンドは、コマンドプロンプトの作業場所（カレントディレクトリ）を変更するための、最も基本的で頻繁に使用されるコマンドです。引数なしで実行すると、現在のカレントディレクトリのパスを表示します。`chdir` は `cd` のエイリアスであり、全く同じ機能です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`pushd` / `popd`** | `pushd` はディレクトリをスタックに保存してから移動し、`popd` で元の場所に戻れます。複数のディレクトリを頻繁に行き来する複雑な作業で便利です。 | - |
| **PowerShell** | PowerShellでは `Set-Location` (エイリアス: `sl`, `cd`) を使います。ドライブ間 (`C:` から `D:`) の移動が `cd D:\` だけでできるなど、より直感的です。 | PowerShell |

## よく連携されるコマンド

`cd` は、移動した先のディレクトリの内容を確認するために `dir` と連携するのが基本的な操作フローです。

### シナリオ例: ディレクトリを移動して内容を確認する (インフラ構築・運用視点)

* **目的**: システムのログファイルが格納されているディレクトリに移動し、ファイルの一覧を確認する。
* **連携コマンド**: `dir`
* **解説**: まず `cd` で目的のディレクトリに移動し、次に `dir` を実行してそのディレクトリの内容をリスト表示します。これは日々の運用で最も頻繁に行われる操作の1つです。
* **コマンド例**:

    ```cmd
    REM Windowsのログディレクトリに移動して内容を確認
    cd %windir%\System32\LogFiles
    dir
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **動作制御** | |
| ⭐ `/D` | 現在のドライブとは異なるドライブ上のディレクトリへ移動する場合に、ドライブも同時に変更します。 |
| **特別な引数** | |
| ⭐ `(引数なし)` | 現在のカレントディレクトリのフルパスを表示します。 |
| ⭐ `..` | 親ディレクトリ（1つ上の階層）に移動します。 |
| `\` | ドライブのルートディレクトリに移動します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 別のドライブにあるアプリケーションのディレクトリに移動する。
* **組み合わせ**: `/D`
* **解説**: `/D` スイッチを使うことで、`D:` と `cd \path` の2つのコマンドを実行する代わりに、1つのコマンドでドライブとディレクトリを同時に変更できます。
* **例**:

    ```cmd
    REM CドライブからDドライブのAppLogsディレクトリへ一度に移動
    cd /D D:\AppLogs
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害されたユーザープロファイル内の不審なディレクトリを調査する。
* **組み合わせ**: `cd "%USERPROFILE%\AppData\Local\Temp"`
* **解説**: マルウェアはしばしば一時ファイルディレクトリやAppDataディレクトリにファイルを隠します。環境変数 (`%USERPROFILE%` など) と組み合わせてこれらの場所に素早く移動し、`dir` や `attrib` で調査を開始します。
* **例**:

    ```cmd
    REM 現在のユーザーのスタートアップフォルダに移動して永続化設定を調査
    cd "%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup"
    dir
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入後、書き込み可能で監視が手薄なディレクトリを探して移動する。
* **組み合わせ**: `cd %PUBLIC%\Documents`
* **解説**: 攻撃者はシステム侵入後、`cd` と `dir` を駆使してファイルシステムを探索します。特に、全ユーザーが書き込み可能な `Public` ディレクトリや `C:\Users\Public`、`C:\Temp` などは、ツールを配置したり、窃取したデータを一時的に保存したりする（ステージング）ための格好の場所となります。
* **例**:

    ```cmd
    REM 全ユーザーがアクセス可能なPublicディレクトリに移動
    cd C:\Users\Public\Music
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'cd'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `The system cannot find the path specified.`
    * **考えられる原因**: 指定したディレクトリが存在しないか、パスの入力に誤りがあります。
    * **解決策**:
      * `dir` コマンドでパスの存在を確認します。
      * パスにスペースが含まれている場合は、`cd "C:\Program Files"` のようにパス全体をダブルクォートで囲んでください。
      * **Tabキーによる補完機能**を積極的に利用し、タイプミスを防ぎましょう。

## 環境変数と設定ファイル

* `cd` コマンドの挙動は、いくつかの環境変数の影響を受けます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

### 影響を与える環境変数

* **`%CD%`**: 現在のカレントディレクトリのパスを格納している動的な環境変数。
* **`%HOMEDRIVE%`, `%HOMEPATH%`**: `cd %HOMEPATH%` のように、ユーザーのホームディレクトリに移動するために使われます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `cd` 自体に脆弱性はありませんが、バッチファイル内での不適切な使用が問題となることがあります。
* **悪用シナリオ**: 攻撃者は、`cd` と `dir` を使ってシステム内を探索し、機密情報（パスワードが書かれた設定ファイルなど）や、パーミッションが緩いディレクトリを探します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* `cd` はファイルシステムを移動するための基本的なコマンドであり、それ自体が直接的な攻撃機能を持つわけではありませんが、LOLBASを利用したほぼ全ての攻撃活動の**偵察フェーズ**で不可欠な役割を果たします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、ユーザーやプロセスに不要なディレクトリへのアクセス権を与えない。
* **Detection (検知)**: `auditpol` でファイルシステムアクセスの監査を有効にし、重要なディレクトリへの予期せぬ `cd` (アクセス) がないかイベントログで監視する。

## 注意点・補足

* **ドライブの変更**: `cd D:\data` と入力しても、カレントドライブは `C:` のままです。`D:` ドライブのカレントディレクトリが `\data` に変更されるだけです。ドライブ自体を移動するには、`D:` と入力するか、`cd /D D:\data` を使用します。
