# `taskkill` - 実行中のプロセスを終了する

## 概要

`taskkill` コマンドは、ローカルまたはリモートのシステムで実行中のプロセスを終了させるためのコマンドラインツールです。応答しなくなったアプリケーションを強制終了したり、バッチファイルで特定のプロセスをクリーンアップしたり、インシデント対応で悪意のあるプロセスを停止させたりする際に使用されます。プロセスは、プロセスID (PID) またはイメージ名 (実行ファイル名) で指定できます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`tasklist`** | `tasklist` はプロセスを**一覧表示**します。`taskkill` はプロセスを**終了**させます。これらはセットで使われることが非常に多いです。 | - |
| **`sc stop` / `net stop`** | **サービス**を正常に停止させようとします。`taskkill` は、サービスを含むあらゆる**プロセス**をより強制的に終了させることができます。 | `sc stop` (サービスの場合) |
| **PowerShell** | `Stop-Process` (エイリアス: `kill`, `spps`) は、プロセスをオブジェクトとして扱え、パイプラインを通じて `Get-Process` と直接連携できるため、スクリプトでの利用が非常に強力です。 | `Stop-Process` |

## よく連携されるコマンド

`taskkill` は、`tasklist` と `findstr` を組み合わせて、特定の条件に一致するプロセスを特定し、そのPIDを使って終了させるという流れでよく利用されます。

### シナリオ例: 特定のユーザーが実行している全プロセスを終了する (インフラ構築・運用視点)

* **目的**: リモートデスクトップサーバーで、特定のユーザー `user01` が実行している全ての `chrome.exe` プロセスを強制終了する。
* **連携コマンド**: `tasklist`, `findstr`, `for` (より高度な例)
* **解説**: `/FI` (Filter) オプションを使って、`USERNAME` と `IMAGENAME` の両方を満たすプロセスを直接 `taskkill` の対象とします。
* **コマンド例**:

    ```cmd
    REM ユーザー 'user01' が実行している全ての 'chrome.exe' を強制終了
    taskkill /S <servername> /FI "USERNAME eq user01" /FI "IMAGENAME eq chrome.exe" /F
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **ターゲット指定** | |
| ⭐ `/PID <processid>` | 終了するプロセスの**プロセスID (PID)** を指定します。 |
| ⭐ `/IM <imagename>` | 終了するプロセスの**イメージ名 (Image Name)** を指定します。ワイルドカード `*` が使用できます。 |
| **動作制御** | |
| ⭐ `/F` | プロセスを**強制的に (Forcefully)** 終了します。正常な終了要求を無視するプロセスに有効です。 |
| ⭐ `/T` | 指定したプロセスと、それが開始した全ての子プロセスを**ツリー状に (Tree)** 終了します。 |
| **フィルタリング** | |
| `/FI <filter>` | `tasklist` と同様のフィルターを適用して、終了するプロセスを指定します。 |
| **リモート操作** | |
| `/S <computer>` | 操作対象のリモートコンピュータを指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 応答しなくなったアプリケーション (`notepad.exe` など) を強制終了する。
* **組み合わせ**: `/IM <imagename> /F`
* **解説**: GUIのタスクマネージャーを開かずに、コマンドラインから素早く特定のアプリケーションを強制終了させます。
* **例**:

    ```cmd
    REM 全てのメモ帳プロセスを強制終了
    taskkill /IM notepad.exe /F
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデント対応における封じ込め**。`netstat` や `resmon` で特定した悪意のあるプロセスのPIDを使って、そのプロセスを即座に終了させる。
* **組み合わせ**: `/PID <pid> /F`
* **解説**: マルウェアのC2通信やデータ窃取活動を行っているプロセスを特定したら、そのPIDを使って `taskkill` で即座に活動を停止させます。`/T` を付けて子プロセスごと終了させるのも有効です。
* **例**:

    ```cmd
    REM PID 1234 の不審なプロセスを、子プロセスも含めて強制終了
    taskkill /PID 1234 /T /F
    ```

### 3. レッドチーム視点

* **タスク**: **防御回避 (Defense Evasion)**。アンチウイルスやEDRなどのセキュリティ製品のプロセスを無効化する。
* **組み合わせ**: `/IM <security_product.exe> /F`
* **解説**: 攻撃者は管理者権限を奪取した後、自身の活動が検知・ブロックされるのを防ぐために、`taskkill` を使ってセキュリティ製品に関連するプロセスを強制終了しようと試みます。多くのセキュリティ製品は自己防衛機能を持っているため、これは成功しないことも多いです。
* **例**:

    ```cmd
    REM (試行として) Windows Defender のプロセスを強制終了しようとする
    taskkill /IM MsMpEng.exe /F
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'taskkill'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `ERROR: The process with PID <number> could not be terminated. Reason: Access is denied.`
    * **考えられる原因**: プロセスを終了させるための権限が不足しています。対象がシステムプロセスであるか、他のユーザーが実行しているプロセスである可能性があります。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `taskkill` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `taskkill` コマンドの**サービス妨害**や**防御回避**への悪用。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、攻撃者はセキュリティ製品やログ収集エージェントのプロセスを `taskkill` で停止させ、自身の活動を隠蔽しようとします。また、基幹アプリケーションのプロセスを終了させてサービス妨害を引き起こすことも可能です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `taskkill.exe` は、攻撃者がシステムのセキュリティ機能を無効化したり、サービスを妨害したりするために利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底する。セキュリティ製品には、自身のプロセスを終了から保護する自己防衛機能が備わっていることが重要です。
* **Detection (検知)**: Windows セキュリティイベントログや `Sysmon` で、プロセスの終了イベント (イベントID 4689) を監視する。特に、`taskkill.exe` によってセキュリティ関連のプロセスが終了させられた場合は、高優先度のアラートを発報するように設定する。

## 注意点・補足

* **強制終了のリスク**: `/F` スイッチを使った強制終了は、データが保存されずに失われる可能性があります。`svchost.exe` のような重要なシステムプロセスをむやみに終了させると、システムが不安定になったり、シャットダウンしたりする可能性があるため、注意が必要です。
