# `net share` - 共有リソースを管理する

## 概要

`net share` コマンドは、サーバーやワークステーション上で、ファイル共有やプリンター共有などの共有リソースを作成、表示、削除するために使用されます。このコマンドを使うことで、どのフォルダーがネットワークに公開されているかを確認したり、新しい共有フォルダーを作成したり、不要になった共有を削除したりできます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`net use`** | `net use` は、他のコンピュータが公開している共有リソースに**接続（クライアント側）**します。`net share` は、自身のコンピュータでリソースを**公開（サーバー側）**します。 | - |
| **PowerShell** | `Get-SmbShare`, `New-SmbShare`, `Remove-SmbShare` などのコマンドレットは、共有管理をオブジェクト指向で行え、より強力でスクリプト化に適しています。 | `*-SmbShare` (PowerShell) |

## よく連携されるコマンド

`net share` で共有を作成した後は、`icacls` を使って、そのフォルダーに対するNTFSアクセス許可を詳細に設定するのが一般的なセキュリティプラクティスです。

### シナリオ例: 安全な共有フォルダーの作成 (インフラ構築・運用視点)

* **目的**: 特定の部署（例: Sales）専用の共有フォルダーを作成し、適切なアクセス権を設定する。
* **連携コマンド**: `mkdir`, `icacls`
* **解説**: `mkdir` でフォルダーを作成し、`net share` で共有を設定します。その後、`icacls` を使って、共有権限とは別に、ファイルシステムレベルでの詳細なアクセス権（NTFSアクセス許可）を設定します。
* **コマンド例**:

    ```cmd
    REM 1. 共有用のフォルダーを作成
    mkdir D:\SalesData

    REM 2. "Sales" という共有名でフォルダーを共有
    net share Sales=D:\SalesData /grant:"Everyone",READ /grant:"Sales-Admins",FULL

    REM 3. NTFSアクセス許可を設定（Sales-Adminsにフルコントロール、Sales-Usersに変更権限）
    icacls D:\SalesData /inheritance:r /grant "Administrators:(OI)(CI)F" /grant "Sales-Admins:(OI)(CI)F" /grant "Sales-Users:(OI)(CI)M"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | 現在のコンピュータで作成されている全ての共有リソースを一覧表示します。 |
| ⭐ `<sharename>` | 指定した共有リソースの詳細情報を表示します。 |
| ⭐ `<sharename>=<drive>:<path>` | 新しい共有を作成します。 |
| ⭐ `<sharename> /delete` | 指定した共有を削除します。 |
| **アクセス許可** | |
| `/grant:<user>,<permission>` | 指定したユーザーまたはグループに共有アクセス許可を与えます。`<permission>` は `READ`, `CHANGE`, `FULL` のいずれか。 |
| **その他** | |
| `/users:<number>` | 共有に同時にアクセスできる最大ユーザー数を設定します。 |
| `/unlimited` | 同時アクセスユーザー数を無制限にします。 |
| `/remark:"<text>"` | 共有に関するコメントを追加します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー上で現在どのような共有が設定されているかを監査する。
* **組み合わせ**: `net share`
* **解説**: 定期的にこのコマンドを実行し、意図しないフォルダー（特に管理者共有 `C$` 以外）が公開されていないかを確認します。
* **例**:

    ```cmd
    REM 現在の共有設定を一覧表示
    net share
    ```

### 2. ブルーチーム視点

* **タスク**: **ラテラルムーブメントの痕跡**を調査する。
* **組み合わせ**: `net share`
* **解説**: 攻撃者はしばしば、侵入したサーバー上に新しい共有を作成し、他のシステムからツールをコピーしたり、窃取したデータを集約したりするための中継地点として利用します。`net share` で見慣れない共有が作成されていないかを確認することは、インシデント調査の重要なステップです。
* **例**:

    ```cmd
    REM サーバーの共有設定を監査し、不審な共有がないか確認
    net share
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したネットワーク内でアクセス可能な共有を探す。
* **組み合わせ**: `net view \\<computername>` → `net share`
* **解説**: `net view` でネットワーク上のコンピュータをリストアップした後、各コンピュータに対して `net share` を実行（直接は不可、`net view \\<computername>` で代用）し、アクセス可能な共有フォルダーを探します。特に、パーミッション設定が緩い共有は、情報窃取やマルウェア拡散の格好の標的となります。
* **例**:

    ```cmd
    REM ファイルサーバー 'FS-01' 上の共有リソースを偵察
    net view \\FS-01
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `System error 5 has occurred. Access is denied.`
    * **考えられる原因**: `net share` で共有を作成または削除するには管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `net share` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **不適切な共有アクセス許可**。特に `Everyone` グループに対して `FULL` (フルコントロール) や `CHANGE` (変更) の権限を与えることは、ランサムウェアの拡散やデータ改ざんの温床となり、極めて危険です。
* **悪用シナリオ**: 攻撃者は、ネットワーク内で `Everyone:F` のような設定がされた共有を探し出し、そこを足がかりにマルウェアを拡散させたり、機密情報を窃取したりします。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **ラテラルムーブメント (Lateral Movement)**
* **解説**: `net share` は、攻撃者が侵入後の偵察フェーズで、ネットワーク共有リソースをマッピングするために利用する典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **最小権限の原則**を徹底し、共有アクセス許可は `Everyone` を避け、認証された特定のユーザーやグループに必要な最小限の権限 (`READ` など) のみを与える。
* **Detection (検知)**: 定期的に `net share` を実行して共有設定を監査するスクリプトを運用する。Windows セキュリティイベントログで、共有の作成 (イベントID 5142) やアクセス (イベントID 5140) を監視する。

## 注意点・補足

* **共有アクセス許可 vs NTFSアクセス許可**: ユーザーが共有フォルダー内のファイルにアクセスするには、**共有アクセス許可**と**NTFSアクセス許可**の両方が必要です。両方のうち、**より厳しい方の権限**が適用されます。`net share` は前者のみを設定するため、後者の `icacls` での設定も不可欠です。
