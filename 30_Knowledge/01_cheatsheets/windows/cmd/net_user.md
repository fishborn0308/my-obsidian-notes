# `net user` - ユーザーアカウントを管理する

## 概要

`net user` コマンドは、コンピュータ上のユーザーアカウントを管理するための基本的なコマンドです。このコマンドを使って、新しいユーザーの作成、既存のユーザーのパスワード変更やプロパティの変更、アカウントの削除などが行えます。Active Directoryドメイン環境では、ドメインユーザーアカウントに対しても一部の操作が可能です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`net localgroup`** | グループのメンバーシップを管理します。`net user` はユーザーアカウント自体を管理します。 | - |
| **PowerShell** | `Get-LocalUser`, `New-LocalUser`, `Set-LocalUser` などのコマンドレットは、ユーザーアカウントをオブジェクトとして扱え、より強力でスクリプト化に適しています。 | `*-LocalUser` (PowerShell) |

## よく連携されるコマンド

`net user` は、作成したユーザーを特定のグループに追加するために `net localgroup` と連携するのが一般的なワークフローです。

### シナリオ例: 新規ユーザーの作成とグループへの追加 (インフラ構築・運用視点)

* **目的**: 新しい開発者アカウントを作成し、リモートデスクトップ接続ができるように権限を付与する。
* **連携コマンド**: `net localgroup`
* **解説**: まず `net user` でユーザーアカウントとパスワードを作成し、次に `net localgroup` を使ってそのユーザーを "Remote Desktop Users" グループに追加します。
* **コマンド例**:

    ```cmd
    REM ユーザー 'dev01' をパスワード 'P@ssw0rd123' で作成
    net user dev01 P@ssw0rd123 /add

    REM 作成したユーザーを "Remote Desktop Users" グループに追加
    net localgroup "Remote Desktop Users" dev01 /add
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | このコンピュータの全てのローカルユーザーアカウントを一覧表示します。 |
| ⭐ `<username>` | 指定したユーザーアカウントの詳細情報を表示します。 |
| ⭐ `<username> <password> /add` | 指定したユーザー名とパスワードで新しいアカウントを作成します。 |
| ⭐ `<username> /delete` | 指定したユーザーアカウントを削除します。 |
| **パスワードとアカウント制御** | |
| ⭐ `<username> *` | 指定したユーザーのパスワードを変更します。`*` を指定すると、対話的にパスワードの入力を求められます。 |
| `/active:{yes\|no}` | アカウントを有効または無効にします。 |
| `/passwordchg:{yes\|no}` | ユーザーが自身のパスワードを変更できるかどうかを設定します。 |
| `/expires:<date>` | アカウントの有効期限を設定します。 |
| **ドメイン操作** | |
| `/domain` | Active Directory ドメインコントローラー上でコマンドを実行します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーション用のサービスアカウントを作成し、パスワードを無期限に設定する。
* **組み合わせ**: `/add /expires:never /passwordchg:no`
* **解説**: サービスアカウントが意図せず無効になったり、パスワード変更を求められたりするのを防ぎます。
* **例**:

    ```cmd
    REM 'svc_myapp' というサービスアカウントを作成
    net user svc_myapp Str0ngP@ssw0rd /add /expires:never /passwordchg:no /comment:"Service account for MyApp"
    ```

### 2. ブルーチーム視点

* **タスク**: **不正なアカウントの発見と無効化**。
* **組み合わせ**: `net user` と `net user <username> /active:no`
* **解説**: **インシデント対応の基本**。まず `net user` でシステム上の全アカウントをリストアップし、ベースラインと比較して見慣れないアカウント（攻撃者が作成したバックドアアカウントなど）がないかを確認します。発見した場合、`net user <username> /active:no` で即座にアカウントを無効化し、攻撃者のアクセスを遮断します。
* **例**:

    ```cmd
    REM 不審なアカウント 'hacker' を無効化
    net user hacker /active:no
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)** と**永続化 (Persistence)**。
* **組み合わせ**: `net user` と `net user <username> <password> /add`
* **解説**:
  * **偵察**: 攻撃者は侵入後、`net user` を実行してシステム上のアカウントをリストアップし、次の攻撃対象となるアカウント（特に管理者アカウント）を特定します。
  * **永続化**: 管理者権限を奪取した後、`net user attacker P@ssw0rd /add` と `net localgroup administrators attacker /add` を実行して、バックドアとなる管理者アカウントを作成し、永続的なアクセスを確保します。
* **例**:

    ```cmd
    REM システム上のユーザーを偵察
    net user

    REM 偵察後、特定のユーザー 'admin' の詳細情報を確認
    net user admin
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'net'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `System error 5 has occurred. Access is denied.`
    * **考えられる原因**: ユーザーの作成、変更、削除には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `net user` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `net user` 自体ではなく、**不適切なアカウント管理**がセキュリティリスクとなります。
* **悪用シナリオ**: 攻撃者は権限昇格後、このコマンドを使ってバックドアアカウントを作成し、永続化を図ります。また、偵察段階で `net user` を使い、休眠アカウントや弱いパスワードポリシーを持つアカウントを探し出します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **永続化 (Persistence)**
* **解説**: `net user` は、攻撃者がシステムのユーザーアカウント情報を収集したり、新しいアカウントを作成したりするために、侵入後の活動で必ずと言っていいほど使用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、不要なアカウントは作成しない。強力なパスワードポリシーを適用する。
* **Detection (検知)**: Windows セキュリティイベントログを監視し、**イベントID 4720 (ユーザーアカウントが作成されました)** や **イベントID 4722 (ユーザーアカウントが有効になりました)** などのアカウント作成・変更イベントを厳重に監視する。`net.exe` の実行自体をプロセス監査ログ (イベントID 4688) で監視することも有効です。

## 注意点・補足

* **パスワードの平文**: バッチファイル内で `net user <username> <password> /add` のようにパスワードを直接記述すると、そのファイルを見ることができる誰にでもパスワードが知られてしまいます。対話的な操作では `net user <username> *` を使うのが安全です。
