# `mkdir` - 新しいディレクトリを作成する

## 概要

`mkdir` (Make Directory) コマンドは、新しいディレクトリ（フォルダ）を作成するために使用されます。`md` は `mkdir` のエイリアスであり、全く同じ機能です。複数のディレクトリを一度に作成したり、パスを指定して、存在しない親ディレクトリ（中間ディレクトリ）も同時に作成したりできます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`rmdir` / `rd`** | ディレクトリを**削除**します。`mkdir` は**作成**します。 | - |
| **`copy con <file>`** | 新しい**ファイル**を作成します。`mkdir` は**ディレクトリ**を作成します。 | - |
| **PowerShell** | `New-Item -ItemType Directory` (エイリアス: `mkdir`, `md`) は、ディレクトリをオブジェクトとして作成し、パイプラインで後続の処理に渡すことができます。 | `New-Item` (PowerShell) |

## よく連携されるコマンド

`mkdir` は、ディレクトリを作成した直後にそのディレクトリに移動する `cd` と連携するのが基本的な操作フローです。

### シナリオ例: プロジェクト用のディレクトリを作成してすぐに移動する (インフラ構築・運用視点)

* **目的**: 新しいプロジェクト用のディレクトリを作成し、すぐにその中を作業場所として使い始める。
* **連携コマンド**: `cd`
* **解説**: まず `mkdir` でディレクトリを作成し、次に `cd` を実行して作成したディレクトリに移動します。
* **コマンド例**:

    ```cmd
    REM "NewProject" ディレクトリを作成し、その中に移動する
    mkdir NewProject
    cd NewProject
    ```

## スイッチ/オプション説明

Windowsの `mkdir` コマンドには、Linux版のような `-p` や `-m` といったスイッチはありません。主な機能は引数で制御します。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| ⭐ `<drive>:<path>` | 作成するディレクトリのパスを指定します。パス内に存在しないディレクトリがあれば、それらもまとめて作成されます。 |
| `(複数の引数)` | `mkdir dir1 dir2 dir3` のようにスペースで区切ることで、複数のディレクトリを一度に作成できます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのログやバックアップを保存するための、階層化されたディレクトリ構造を作成する。
* **組み合わせ**: `mkdir <path>`
* **解説**: `mkdir C:\AppLogs\WebApp\2025-10` のようにパスを指定すると、`AppLogs` や `WebApp` が存在しない場合でも自動的に作成されます。これはLinuxの `mkdir -p` と同様の便利な機能です。
* **例**:

    ```cmd
    REM アプリケーションのデータディレクトリ構造を一括で作成
    mkdir C:\ProgramData\MyApp\config C:\ProgramData\MyApp\data C:\ProgramData\MyApp\logs
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠ファイルを隔離するための、保全用ディレクトリを作成する。
* **組み合わせ**: `mkdir <quarantine_dir>`
* **解説**: インシデント対応中に、マルウェアや不審なファイルを隔離するための専用ディレクトリを作成します。作成後、`icacls` でアクセス権を厳格に設定することが重要です。
* **例**:

    ```cmd
    REM フォレンジック調査用のディレクトリを作成
    mkdir D:\Forensics\Case001\Evidence
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)** と**永続化 (Persistence)** のための隠しディレクトリを作成する。
* **組み合わせ**: `mkdir`
* **解説**: 攻撃者は侵入後、自身のツールや窃取したデータを保管するために、正規のディレクトリに見せかけた名前のディレクトリ (`C:\ProgramData\Microsoft Help` のようにスペースを入れるなど) や、あまり監視されていない場所 (`C:\Users\Public`など) にディレクトリを作成します。
* **例**:

    ```cmd
    REM ツールを隠すためのディレクトリを作成
    mkdir C:\Users\Public\Downloads\updates
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `A subdirectory or file <dirname> already exists.`
    * **考えられる原因**: 作成しようとしたディレクトリまたは同名のファイルが既に存在します。
    * **解決策**: 別の名前を使用するか、既存のディレクトリが不要であれば `rd /S /Q` で削除してから再試行してください。

2. **エラーメッセージ**: `Access is denied.`
    * **考えられる原因**: ディレクトリを作成するための権限がありません（例: `C:\Program Files` 配下など）。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `mkdir` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `mkdir` 自体ではなく、**書き込み権限の緩いディレクトリ**が悪用されます。
* **悪用シナリオ**: 攻撃者は、誰でも書き込めるディレクトリ (`C:\Users\Public` や `C:\Temp` など) に `mkdir` でディレクトリを作成し、そこをマルウェアの配置場所やデータ窃取のための一時保管場所（ステージングエリア）として利用します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **防御回避 (Defense Evasion)**
* **解説**: `mkdir` は `cmd.exe` の内部コマンドであり、攻撃者が活動の準備段階でディレクトリを作成するために利用する環境寄生型の手法です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に基づき、ファイルシステムに適切なACLを設定する。
* **Detection (検知)**: `Sysmon` (イベントID 11: FileCreate) やファイル整合性監視(FIM)ツールで、重要なシステムディレクトリや、通常は書き込みが発生しない場所での予期せぬディレクトリ作成を監視する。

## 注意点・補足

* **親ディレクトリの自動作成**: `mkdir a\b\c` と実行すると、`a` と `b` が存在しなくても自動的に作成されます。これは `mkdir` の非常に便利な機能です。
