# `vssadmin` - ボリュームシャドウコピーサービスを管理する

## 概要

`vssadmin` (Volume Shadow Copy Service Administration) コマンドは、Windowsの**ボリュームシャドウコピーサービス (VSS)** を管理するためのコマンドラインツールです。VSSは、特定の時点でのボリュームの静的なスナップショット（シャドウコピー）を作成する機能です。これにより、ファイルが使用中であっても、整合性の取れたバックアップを作成できます。

`vssadmin` は、これらのシャドウコピーの一覧表示、作成、削除、およびシャドウコピーが使用するディスク領域の管理に使用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`diskshadow`** | `vssadmin` よりも対話的で高機能なスクリプティングが可能なVSS管理ツール。より複雑なVSS操作に適しています。 | `diskshadow` (スクリプト) |
| **PowerShell** | `Get-CimInstance -ClassName Win32_ShadowCopy` や WMI を通じてシャドウコピーを操作できますが、`vssadmin` の方が手軽な場合があります。 | - |

## よく連携されるコマンド

`vssadmin` は、バックアップソフトがシャドウコピーを作成・削除する際に内部的に呼び出されます。また、攻撃者がデータを暗号化する前に、復旧を妨害する目的で `vssadmin` を実行することが知られています。

### シナリオ例: 既存のシャドウコピーを確認する (インフラ構築・運用視点)

* **目的**: システムに現在どのようなシャドウコピー（復元ポイント）が存在するかを確認する。
* **連携コマンド**: (なし)
* **解説**: `list shadows` は、バックアップやシステムの復元ポイントが正しく作成されているかを確認するための基本的なコマンドです。
* **コマンド例**:

    ```cmd
    REM 存在する全てのシャドウコピーを一覧表示
    vssadmin list shadows
    ```

## スイッチ/オプション説明

`vssadmin` は `vssadmin <command> [options]` の形式で使用します。

| コマンド (Operation) | 説明 |
| :--- | :--- |
| **シャドウコピー管理** | |
| ⭐ `list shadows` | 既存の全てのシャドウコピーを一覧表示します。 |
| ⭐ `delete shadows` | 指定したボリュームのシャドウコピーを削除します。 |
| `create shadow` | 新しいシャドウコピーを作成します。 |
| **ストレージ管理** | |
| `list shadowstorage` | シャドウコピーストレージの関連付け（どこに、どれくらいのサイズで確保されているか）を表示します。 |
| `resize shadowstorage` | シャドウコピーストレージの最大サイズを変更します。 |
| **その他** | |
| `list providers` | 登録されているVSSプロバイダー（シャドウコピーを作成するソフトウェアコンポーネント）を一覧表示します。 |
| `list writers` | VSSに参加しているライター（SQL ServerやExchangeなど、自身のデータを静止させるアプリケーション）の状態を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディスク容量を確保するため、古いシャドウコピーを削除する。
* **組み合わせ**: `delete shadows /for=<volume> /oldest`
* **解説**: `/for` で対象ボリュームを指定し、`/oldest` で最も古いシャドウコピーから削除します。`/all` を指定すると全て削除されます。
* **例**:

    ```cmd
    REM Cドライブの最も古いシャドウコピーを削除
    vssadmin delete shadows /for=C: /oldest
    ```

### 2. ブルーチーム視点

* **タスク**: **ランサムウェア攻撃の兆候を検知する**。
* **組み合わせ**: `list shadows`
* **解説**: `vssadmin list shadows` を実行し、「一致するシャドウ コピーが見つかりませんでした。」と表示された場合、それは極めて不審な兆候です。多くのランサムウェアは、暗号化の前にまず `vssadmin delete shadows` を実行して、システムの復元ポイントを削除し、復旧を妨害するためです。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **影響 (Impact)**。ランサムウェア攻撃のシミュレーションとして、ファイルの暗号化前に復旧手段を無効化する。
* **組み合わせ**: `delete shadows /all /quiet`
* **解説**: 攻撃者はペイロードの実行初期段階で、このコマンドを実行して全てのシャドウコピーを警告なし (`/quiet`) で削除します。これにより、被害者はOSの標準機能（「以前のバージョンの復元」など）を使ってファイルを復元できなくなります。
* **例**:

    ```cmd
    REM 全てのボリュームの全シャドウコピーを警告なしで削除
    vssadmin delete shadows /all /quiet
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Error: You must be an administrator to run this command.`
    * **考えられる原因**: `vssadmin` の実行には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `vssadmin` の**シャドウコピー削除機能**が、ランサムウェアによって悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`vssadmin delete shadows` は、ランサムウェアが活動を開始する際の定型的な最初のステップです。これにより、被害者の復旧選択肢を大幅に狭め、身代金を支払わせる可能性を高めます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `vssadmin.exe` は、攻撃者がシステムのバックアップを破壊し、検知を困難にするために利用する、極めて重要な環境寄生型バイナリです。正規のシステム管理ツールであるため、実行自体をブロックするのは困難です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: シャドウコピーとは別に、オフラインまたはイミュータブル（変更不能）なバックアップを取得する。
* **Detection (検知)**: **`vssadmin.exe` の実行、特に `delete shadows` という引数を伴う実行を、プロセス監査ログ (イベントID 4688) やEDRで最優先で監視し、即座にアラートを発報するように設定する**。これはランサムウェア攻撃の非常に強力な兆候です。

## 注意点・補足

* **シャドウコピーの性質**: シャドウコピーは差分スナップショットであり、元のボリューム上に保存されます。そのため、物理的なディスク障害に対するバックアップにはなりません。
