# `sc` - Windowsサービスを総合的に管理する

## 概要

`sc` (Service Control) コマンドは、Windowsのサービスコントロールマネージャーと通信するための、非常に強力で包括的なコマンドラインツールです。`net start` や `net stop` がサービスの開始/停止に限定されているのに対し、`sc` はサービスの照会 (`query`)、作成 (`create`)、削除 (`delete`)、設定変更 (`config`)、依存関係の表示 (`qc`) など、サービスに関するほぼ全ての操作を実行できます。システムの管理や、攻撃者による永続化の手法として、幅広く利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`net start` / `net stop`** | `net` コマンドはサービスの開始/停止など基本的な操作のみです。`sc` はサービスの作成や設定変更など、はるかに詳細で広範な管理機能を提供します。 | `sc` (詳細管理) |
| **`services.msc`** | GUIベースのサービス管理コンソール。`sc` はその機能をCUIで提供し、スクリプトによる自動化を可能にします。 | `sc` (スクリプト) |
| **PowerShell** | `Get-Service`, `New-Service`, `Set-Service` などのコマンドレットは、サービスをオブジェクトとして扱え、パイプラインと連携できるため、よりモダンで強力なスクリプティングが可能です。 | `*-Service` (PowerShell) |

## よく連携されるコマンド

`sc` の出力は詳細であるため、`findstr` と連携して特定の情報をフィルタリングすることがよくあります。

### シナリオ例: 実行中のサービスのみをリストアップする (インフラ構築・運用視点)

* **目的**: システム上で現在 `RUNNING` 状態にあるサービスをすべて表示する。
* **連携コマンド**: `findstr`
* **解説**: `sc query` は全てのサービスの状態を出力しますが、そのままだと情報が多すぎます。出力をパイプで `findstr` に渡し、"RUNNING" を含む行だけをフィルタリングします。
* **コマンド例**:

    ```cmd
    REM STATE が RUNNING のサービスのみをフィルタリング表示
    sc query state= all | findstr "STATE" | findstr "RUNNING"
    ```

## スイッチ/オプション説明

`sc` は `sc <command> [service_name] [options]` の形式で使用します。

| コマンド (Operation) | 説明 |
| :--- | :--- |
| ⭐ **`query`** | 1つまたは全てのサービスの状態を照会します。 |
| ⭐ **`qc`** | サービスの詳細な設定情報（実行ファイルのパス、依存関係など）を照会します。 |
| ⭐ **`create`** | 新しいサービスをレジストリに登録します。 |
| ⭐ **`delete`** | サービスを削除します。 |
| ⭐ **`start`** | サービスを開始します。 |
| ⭐ **`stop`** | サービスを停止します。 |
| **`config`** | サービスのプロパティ（スタートアップの種類、実行アカウントなど）を変更します。 |
| **`qfailure`** | サービスの失敗時のアクションを照会します。 |

| 主なオプション (`create`, `config`で使用) | 説明 |
| :--- | :--- |
| ⭐ `binPath=` | サービスの実行可能ファイルの完全パスを指定します。 |
| `type=` | サービスの種類を指定します (`own`, `share`, `kernel` など)。 |
| ⭐ `start=` | サービスのスタートアップの種類を指定します (`boot`, `system`, `auto`, `demand`, `disabled`)。 |
| `DisplayName=` | `services.msc` などで表示されるサービスの名前。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 特定のサービス (例: `wuauserv`) の実行ファイルのパスやスタートアップの種類を確認する。
* **組み合わせ**: `qc`
* **解説**: `sc qc <service_name>` は、サービスの構成情報を詳細に表示するため、トラブルシューティングや設定監査に不可欠です。
* **例**:

    ```cmd
    REM Windows Update サービスの設定情報を確認
    sc qc wuauserv
    ```

### 2. ブルーチーム視点

* **タスク**: **永続化されたマルウェアの発見**。システムに登録されている不審なサービスを調査する。
* **組み合わせ**: `query` と `qc`
* **解説**: **インシデント対応の基本**。まず `sc query type= service state= all` ですべてのサービスをリストアップし、見慣れないサービス名や説明がないサービスを探します。不審なものを見つけたら `sc qc <service_name>` で `BINARY_PATH_NAME` を確認し、そのパスや実行ファイルが正当なものか（例: `C:\Windows\Temp` のような場所ではないか）を調査します。
* **例**:

    ```cmd
    REM 不審なサービス 'SysHelper' の詳細設定を確認
    sc qc SysHelper
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** と**権限昇格 (Privilege Escalation)** のため、悪意のあるサービスを登録する。
* **組み合わせ**: `create`
* **解説**: 攻撃者は管理者権限を奪取した後、`sc create` を使って自身のペイロードをサービスとして登録します。`start= auto` に設定すればシステム起動時に自動実行され、`binPath=` に巧妙に偽装したコマンドを記述することで、永続的なアクセスや権限昇格を維持します。
* **例**:

    ```cmd
    REM 'Updater' という偽のサービス名で、リバースシェルを起動するサービスを作成
    sc create Updater binPath= "C:\Users\Public\reverse.exe" start= auto DisplayName= "System Updater Service"
    sc start Updater
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `[SC] OpenSCManager FAILED 5: Access is denied.`
    * **考えられる原因**: サービスの作成、削除、設定変更には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

2. **エラーメッセージ**: `[SC] OpenService FAILED 1060: The specified service does not exist as an installed service.`
    * **考えられる原因**: 操作対象として指定したサービス名が存在しません。
    * **解決策**: `sc query` で正しいサービス名を確認してください。表示名ではなく、サービス名で指定する必要があります。

## 環境変数と設定ファイル

* `sc` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。サービスの設定はレジストリに保存されます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **不適切なサービスパーミッション**や**引用符で囲まれていないサービスパス**。
* **悪用シナリオ**:
  * **永続化**: レッドチームのシナリオで述べた通り、`sc create` は攻撃者がバックドアを仕掛けるための最も一般的な手法の1つです。
  * **権限昇格**: `binPath=` が `C:\Program Files\Some App\service.exe` のように引用符で囲まれていない場合、攻撃者が `C:\Program Files\Some.exe` というファイルを置くことで、正規のサービスより先に悪意のあるプログラムを実行させることが可能です (Unquoted Service Path)。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **永続化 (Persistence)**, **権限昇格 (Privilege Escalation)**, **コード実行 (Execution)**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Sc/](https://lolbas-project.github.io/lolbas/Binaries/Sc/)
* **解説**: `sc.exe` は、攻撃者がシステムに深く潜伏し、活動を維持するために利用する、極めて重要な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: サービスの実行可能ファイルのパーミッションを厳格に管理する。開発時にサービスのパスを必ず引用符で囲む。
* **Detection (検知)**: Windows セキュリティイベントログで、**新しいサービスのインストール (イベントID 4697/7045)** を厳重に監視する。`sc.exe` の実行、特に `create` コマンドをプロセス監査ログで監視する。

## 注意点・補足

* **サービス名と表示名**: `sc` コマンドで操作する際には、GUIの「表示名」ではなく、プロパティで確認できる「サービス名」を使用する必要があります。
