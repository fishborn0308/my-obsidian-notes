# `net stop` - Windowsサービスを停止する

## 概要

`net stop` コマンドは、現在実行中のWindowsサービスを停止するために使用されます。サービスを停止することで、関連するプロセスが終了し、サービスが確保していたリソースが解放されます。システムのメンテナンス、設定変更の適用、または問題を引き起こしているサービスを無力化する際に利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`sc stop`** | `sc` は、サービスの停止、開始、設定変更など、より包括的なサービス管理機能を提供します。`sc` は依存関係にあるサービスも停止できる場合があります。 | `sc` |
| **PowerShell** | `Stop-Service` コマンドレットは、サービスをオブジェクトとして扱え、パイプラインを通じて他のコマンドレットと連携できるため、スクリプトでの利用に非常に強力です。 | `Stop-Service` |

## よく連携されるコマンド

`net stop` は、サービスを停止した後に、その状態を確認する `sc query` と連携して使われます。

### シナリオ例: サービスの停止と状態確認 (インフラ構築・運用視点)

* **目的**: メンテナンスのためにWebサービス (W3SVC) を停止し、正しく停止したことを確認する。
* **連携コマンド**: `sc query`
* **解説**: まず `net stop` でサービスを停止し、その後 `sc query` を使ってサービスの `STATE` が `STOPPED` になっていることを確認します。
* **コマンド例**:

    ```cmd
    REM World Wide Web Publishing Service を停止
    net stop W3SVC

    REM サービスが停止したことを確認
    sc query W3SVC
    ```

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルを変更するために、一時的にサービスを停止する。
* **組み合わせ**: `net stop <service>`
* **解説**: アプリケーションの設定変更やアップデートを適用する前に、関連するサービスを安全に停止します。
* **例**:

    ```cmd
    REM メンテナンスのため、SQL Server サービスを停止
    net stop MSSQLSERVER
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデント対応における封じ込め**。マルウェアが利用しているサービスを停止する。
* **組み合わせ**: `net stop <malicious_service>`
* **解説**: 調査の結果、攻撃者が永続化のために設置した悪意のあるサービスや、マルウェアの活動に利用されている正規のサービス（例: `WinRM`）を特定した場合、`net stop` で即座にサービスを停止させ、攻撃者の活動を無力化します。
* **例**:

    ```cmd
    REM 攻撃者がC2通信に利用している 'evilsvc' サービスを停止
    net stop evilsvc
    ```

### 3. レッドチーム視点

* **タスク**: **サービス妨害 (DoS)** または **防御回避 (Defense Evasion)**。
* **組み合わせ**: `net stop <service_name>`
* **解説**: 攻撃者は管理者権限を奪取した後、`net stop` を使って標的の基幹サービス（データベース、Webサーバーなど）を停止させ、サービス妨害を引き起こします。また、`WinDefend` (Windows Defender) やEDR製品のサービスを停止させ、検知を回避して活動しようとします。
* **例**:

    ```cmd
    REM Windows Defender のウイルス対策サービスを停止
    net stop WinDefend
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'net'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `System error 5 has occurred. Access is denied.`
    * **考えられる原因**: サービスの停止には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

2. **エラーメッセージ例 2**: `The service is not started.`
    * **考えられる原因**: 停止しようとしたサービスは、既に停止しています。
    * **解決策**: `sc query <service>` で現在の状態を確認してください。

## 環境変数と設定ファイル

* `net stop` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `net stop` 自体ではなく、**サービスの停止権限の不適切な付与**がリスクとなります。
* **悪用シナリオ**: 攻撃者は、`net stop` を使ってセキュリティ製品（アンチウイルス、EDR）やログ収集サービスを無効化し、検知を逃れてシステム内で自由に活動しようとします。また、基幹サービスを停止させることによるDoS攻撃も常套手段です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `net stop` は、攻撃者がセキュリティ機構を無効化したり、サービス妨害を行ったりするために利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: サービスのパーミッションを適切に設定し、一般ユーザーが重要なサービスを停止できないようにする。セキュリティ製品には、自身のサービスを停止から保護する自己防衛機能が備わっていることが多いです。
* **Detection (検知)**: Windows セキュリティイベントログで、サービスの停止 (イベントID 7036) を監視する。特に、セキュリティ関連のサービスが意図せず停止した場合は、高優先度のアラートを発報するように設定する。

## 注意点・補足

* **依存関係**: あるサービスが他のサービスに依存している場合、`net stop` はそのサービスのみを停止しようとします。依存されているサービスは停止しません。`sc stop` は依存関係をより考慮して動作することがあります。
