# `taskmgr` - タスクマネージャーを起動する

## 概要

`taskmgr` コマンドは、Windowsの**タスクマネージャー** (`taskmgr.exe`) のGUIを起動します。タスクマネージャーは、コンピュータで実行されているプログラム、プロセス、サービスに関する情報をリアルタイムで提供する中心的なシステムユーティリティです。

主な機能には、リソース（CPU、メモリ、ディスク、ネットワーク）の使用状況の監視、応答しないアプリケーションの終了、プロセスの詳細情報の確認、スタートアッププログラムの管理などが含まれます。システムのパフォーマンス問題の診断から、不審なプロセスの調査まで、幅広い用途で利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`resmon`** | リソースモニターは、タスクマネージャーよりも**詳細な**リソース監視機能を提供します。特定のプロセスがどのファイルにアクセスしているか、どのTCPポートで通信しているかなどをリアルタイムで追跡できます。 | `resmon` (詳細なリアルタイム分析) |
| ⭐ **`tasklist` / `taskkill`** | `tasklist` と `taskkill` は、タスクマネージャーのプロセス管理機能をCUIで提供します。スクリプトによる自動化に適しています。 | `tasklist` / `taskkill` (CUI/スクリプト) |
| **`perfmon`** | パフォーマンスモニターは、パフォーマンスカウンターを使って**長期間にわたる**データ収集とレポート作成を得意とします。`taskmgr` は**リアルタイム**の概要監視に適しています。 | `perfmon` (長期監視) |

## よく連携されるコマンド

`taskmgr` はGUIツールのため、他のコマンドと直接連携することはありません。しかし、`taskmgr` で特定した不審なプロセス名やPIDを、`taskkill` や `sc` などのコマンドラインツールでさらに操作する、という流れで利用されます。

### シナリオ例: プロセスの特定と強制終了 (インフラ構築・運用視点)

* **目的**: GUIで応答しなくなったアプリケーションを特定し、CUIから強制終了する。
* **連携コマンド**: `taskkill`
* **解説**: `taskmgr` を起動し、「プロセス」タブまたは「詳細」タブで問題のアプリケーションのプロセス名 (`IMAGENAME`) またはPIDを特定します。その後、コマンドプロンプトから `taskkill` を使ってそのプロセスを終了させます。
* **コマンド例**:

    ```cmd
    REM (taskmgrで 'badapp.exe' の PIDが1234であることを確認)

    REM PIDを指定して強制終了
    taskkill /PID 1234 /F

    REM または、イメージ名を指定して強制終了
    taskkill /IM badapp.exe /F
    ```

## スイッチ/オプション説明

`taskmgr` コマンドには、一般的に使用されるコマンドラインスイッチはほとんどありません。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| `(なし)` | タスクマネージャーのGUIを起動します。 |
| `/7` | (Windows 10/11) クラシックなWindows 7スタイルのタスクマネージャーを起動します。 |

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `タスクマネージャーは管理者によって無効にされています。`
    * **考えられる原因**: グループポリシーまたはレジストリ設定によって、タスクマネージャーの実行が禁止されています。これは、マルウェアが自身の終了を防ぐために行うことがあります。
    * **解決策**: グループポリシーエディタ (`gpedit.msc`) またはレジストリエディタ (`regedit`) で、タスクマネージャーを無効化する設定を解除する必要があります。

## 環境変数と設定ファイル

* `taskmgr` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `taskmgr` 自体は監視ツールですが、その情報が**情報漏洩**に繋がり、また**防御回避**のために悪用されます。
* **悪用シナリオ**:
  * **偵察**: レッドチームのシナリオで述べた通り、攻撃者は `taskmgr` を使ってセキュリティ製品の存在を把握します。
  * **防御回避**: 攻撃者は、`taskmgr` を使ってセキュリティ製品のプロセスを終了させようと試みることがあります（多くの製品には自己防衛機能があるため、成功しないことも多いです）。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `taskmgr.exe` は、攻撃者が侵入後の情報収集フェーズで、システムの状況やセキュリティ対策を把握するために利用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `taskmgr.exe` の実行自体は非常に一般的ですが、不審なプロセス（Webサーバーのワーカープロセスなど）からの実行は監視対象となります。また、`taskmgr` がグループポリシーで無効化された場合は、マルウェアによる操作の可能性があるため、アラートを発報すべきです。

## 注意点・補足

* **管理者権限**: `taskmgr` は一般ユーザーでも起動できますが、「管理者として実行」することで、全てのユーザーのプロセスやサービスの詳細情報を表示・操作できるようになります。
