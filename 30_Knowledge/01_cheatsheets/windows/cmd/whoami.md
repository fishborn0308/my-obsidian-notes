# `whoami` - 現在のユーザー情報を表示する

## 概要

`whoami` コマンドは、コマンドを実行している現在のユーザーのコンテキスト情報を表示するための基本的なコマンドです。「今、自分は誰としてこの操作を行っているのか？」を素早く確認するために使用されます。デフォルトではドメインとユーザー名 (`DOMAIN\user`) を表示しますが、オプションを付けることで、所属しているグループ、持っている権限 (privileges)、ログオンIDなどより詳細なセキュリティ情報を表示できます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`%USERNAME%`** | 環境変数 `%USERNAME%` はユーザー名のみを保持しています。`whoami` はドメイン名や、`/all` スイッチでSIDや権限など、より多くの情報を表示できます。 | `whoami` (詳細情報) |
| **PowerShell** | `whoami.exe` を直接実行するか、`$env:USERNAME` や `[System.Security.Principal.WindowsIdentity]::GetCurrent()` を使います。PowerShellでは、これらの情報をオブジェクトとして取得できます。 | `[WindowsIdentity]` |

## よく連携されるコマンド

`whoami` の出力は、他のコマンドの実行結果と組み合わせて、現在の権限レベルを把握するために使われます。

### シナリオ例: 現在の権限レベルを確認する (インフラ構築・運用視点)

* **目的**: 現在のユーザーがローカルの管理者グループに所属しているかどうかを素早く確認する。
* **連携コマンド**: (なし)
* **解説**: `whoami /groups` を実行し、出力の中に "BUILTIN\Administrators" が含まれているかを確認します。これにより、管理者権限が必要な操作を実行できるかどうかを判断します。
* **コマンド例**:

    ```cmd
    REM 現在のユーザーが所属するグループを一覧表示
    whoami /groups
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示内容** | |
| ⭐ `(引数なし)` | 現在のユーザー名を `domain\user` 形式で表示します。 |
| ⭐ `/USER` | 現在のユーザー名とセキュリティ識別子 (SID) を表示します。 |
| ⭐ `/GROUPS` | 現在のユーザーが所属するグループ、SID、属性を表示します。 |
| ⭐ `/PRIV` | 現在のユーザーのセキュリティ権限 (privileges) を表示します。 |
| ⭐ `/ALL` | `/USER`, `/GROUPS`, `/PRIV` の全ての情報をまとめて表示します。**非常に有用です**。 |
| `/LOGONID` | 現在のユーザーのログオンIDを表示します。 |
| `/FQDN` | ユーザー名を完全修飾ドメイン名 (FQDN) 形式 (`user@corp.example.com`) で表示します。 |
| **出力形式** | |
| `/FO <format>` | 出力形式を指定します (`TABLE`, `LIST`, `CSV`)。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スクリプトを実行するユーザーが十分な権限を持っているかを確認する。
* **組み合わせ**: `/priv`
* **解説**: `/priv` を使って、`SeBackupPrivilege` (バックアップ権限) や `SeShutdownPrivilege` (シャットダウン権限) など、特定のタスクに必要な権限を持っているかを確認します。
* **例**:

    ```cmd
    REM 現在のセッションが持つ権限を一覧表示
    whoami /priv
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデントレスポンスの初期対応**。侵害されたホストで、攻撃者がどのユーザーコンテキストで活動しているかを特定する。
* **組み合わせ**: `/all`
* **解説**: `whoami /all > whoami_output.txt` を実行し、攻撃者が使用しているアカウントのユーザー名、SID、所属グループ、権限を正確に記録します。これにより、攻撃者がどのような権限レベルで活動しているか（一般ユーザーか、管理者か、SYSTEMか）を判断します。
* **例**:

    ```cmd
    REM 調査対象ホストで実行中のユーザーの全情報を取得
    whoami /all
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入後の状況確認として、現在の権限レベルを把握する。
* **組み合わせ**: `/all`
* **解説**: 攻撃者はシステムに侵入後、ほぼ確実に `whoami` と `hostname` を実行します。`whoami /all` の結果から、現在のユーザーがローカル管理者か、ドメインの特権グループに所属しているか、あるいは `SeImpersonatePrivilege` のような権限昇格に利用できる権限を持っているかなどを確認し次の行動を計画します。
* **例**:

    ```cmd
    REM 侵入後の初期偵察
    whoami /all
    ```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `whoami` 自体は読み取り専用のツールですが、その出力が**重大な情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`whoami` の出力は、攻撃者が現在の権限レベルを把握し、次の権限昇格やラテラルムーブメントの計画を立てるための最も基本的な情報源です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `whoami.exe` は、攻撃者が侵入後の情報収集フェーズで、ほぼ間違いなく実行する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `whoami.exe` の実行は非常に一般的であるため、単体での検知は困難です。しかし、不審なプロセス（例: Webサーバーのワーカープロセス `w3wp.exe`）から `whoami.exe` が実行されるといった、通常ではありえないコンテキストでの実行をEDRなどで監視することは有効です。また、他の偵察コマンドと連続して実行される場合も攻撃の兆候です。

## 注意点・補足

* **権限の確認**: `whoami /priv` は現在の**セッション**が持つ権限を表示します。管理者グループに所属していても、UAC (User Account Control) によって昇格されていないセッション（通常のコマンドプロンプト）では、表示される権限が制限されていることに注意が必要です。
