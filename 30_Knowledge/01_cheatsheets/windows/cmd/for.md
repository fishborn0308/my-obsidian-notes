# `for` - ファイルやコマンド出力などを反復処理する

## 概要

`for` コマンドは、バッチファイルやコマンドプロンプトにおいて、繰り返し（ループ）処理を実現するためのコマンドです。ファイルの一覧、ディレクトリの一覧、数値の範囲、ファイルの内容、コマンドの出力結果など、さまざまなデータセットを1つずつ処理できます。バッチ処理による自動化の根幹をなす、非常に強力で多機能なコマンドです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **PowerShell** | `ForEach-Object` や `foreach` ループは、オブジェクトのプロパティを扱えるため、より高度で柔軟なループ処理が可能です。`for` はテキストベースの処理に特化しています。 | `ForEach-Object` (PowerShell) |
| **`goto`** | `goto` は指定したラベルにジャンプするだけです。`for` はデータセットの各要素に対して繰り返し処理を行う、より構造化されたループです。 | `for` |

## よく連携されるコマンド

`for` は、そのループ内で `echo`, `copy`, `del`, `call` など、ほぼ全てのコマンドと連携して、一連の自動化処理を構築します。

### シナリオ例: 複数サーバーへの疎通確認 (インフラ構築・運用視点)

* **目的**: `servers.txt` にリストされた全てのサーバーに対して `ping` を実行し、疎通確認を行う。
* **連携コマンド**: `ping`, `type`
* **解説**: `for /F` を使って、`servers.txt` の内容を一行ずつ読み込み、各行（サーバー名）に対して `ping` コマンドを実行します。
* **コマンド例**:

    ```cmd
    REM servers.txt の各行に対してpingを実行
    FOR /F "tokens=*" %%A IN (servers.txt) DO (
        echo.
        echo Checking status of %%A...
        ping -n 2 %%A
    )
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **ループの種類** | |
| ⭐ `%%variable` | (バッチファイル内) ループで使う変数を定義します。コマンドプロンプトから直接実行する場合は `%variable` のように `%` は1つです。 |
| `(set)` | 処理対象のリストを指定します。ワイルドカード (`*`, `?`) が使えます。（例: `(*.txt)`) |
| ⭐ `/F ["options"]` | **最も強力なオプション**。ファイルの内容、文字列、またはコマンドの出力を一行ずつ処理します。`"options"` で解析方法を細かく指定できます。 |
| `/D` | 指定したパターンに一致する**ディレクトリ**を反復処理します。 |
| `/R [[drive:]path]` | 指定したパスとその全ての**サブディレクトリ**を再帰的に処理します。 |
| `/L` | 指定した範囲で**数値**を増減させながらループします (`start`, `step`, `end`)。 |
| **FOR /F の "options"** | |
| `tokens=x,y-z` | 各行から抽出するトークン（列）を指定します。 |
| `delims=xyz` | トークンを区切る文字を指定します。デフォルトはスペースとタブです。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複数のファイルを一括でリネームする（例: `_backup` を付与）。
* **組み合わせ**: `for %%A in (*.log) do ren "%%A" "%%~nA_backup%%~xA"`
* **解説**: カレントディレクトリの全 `.log` ファイルをループ処理し、`ren` (rename) コマンドで名前を変更します。`%%~nA` はファイル名、`%%~xA` は拡張子を表す修飾子です。
* **例**:

    ```cmd
    REM カレントディレクトリの全 .txt ファイルを .bak にリネーム
    for %%F in (*.txt) do ren "%%F" "%%~nF.bak"
    ```

### 2. ブルーチーム視点

* **タスク**: 複数のログファイルから、特定のキーワードを含む行を検索し、結果を1つのファイルにまとめる。
* **組み合わせ**: `for /R <logdir> %%F in (*.log) do findstr /I "error" "%%F" >> all_errors.log`
* **解説**: `/R` でログディレクトリを再帰的に探索し、見つかった各ログファイルに対して `findstr` を実行。結果を `>>` で1つのファイルに追記します。
* **例**:

    ```cmd
    REM (C:\Logs に移動後)
    REM 全ての.logファイルから "Access Denied" を検索し、結果を denied.txt に集約
    for /R . %%F in (*.log) do findstr /I "Access Denied" "%%F" >> denied.txt
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。システムのユーザー一覧を取得し、各ユーザーの詳細情報を照会する。
* **組み合わせ**: `for /F "tokens=1" %%U in ('net user ^| find "User accounts"') do net user %%U`
* **解説**: `net user` コマンドの実行結果を `for /F` で解析し、取得したユーザー名 (`%%U`) を `net user` コマンドに渡して、各ユーザーの所属グループや最終ログイン日時などを列挙します。
* **例**:

    ```cmd
    REM システム上の全サービスを列挙
    for /F "tokens=2" %%S in ('sc query state^= all ^| find "SERVICE_NAME"') do echo %%S
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **現象**: バッチファイルを実行すると、`%%A` が予期せず終了する。
    * **考えられる原因**: コマンドプロンプトから直接 `for` コマンドを実行する場合、ループ変数は `%A` のように `%` が1つです。しかし、**バッチファイル内では `%%A` のように `%` を2つ重ねる必要があります**。
    * **解決策**: バッチファイル内のループ変数が `%%` になっているか確認してください。

2. **現象**: `FOR /F` でファイルパスを処理する際、パスにスペースが含まれていると正しく動作しない。
    * **考えられる原因**: `usebackq` オプションが指定されていないか、ファイルパスがダブルクォートで囲まれていません。
    * **解決策**: `for /F "usebackq tokens=*" %%A in ("file with spaces.txt") do ...` のように、`usebackq` を使い、ファイルパスをダブルクォートで囲みます。

## 環境変数と設定ファイル

* `for` は、`setlocal enabledelayedexpansion` と連携して、ループ内で変更される変数を `!var!` の形式で扱うことができます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `for` 自体ではなく、それが呼び出すコマンドや処理するデータに起因する問題。
* **悪用シナリオ**: `for` ループは、システム上のファイルを列挙したり、コマンドを繰り返し実行したりするため、攻撃者による偵察、データ破壊、サービス妨害（プロセスを大量に起動するなど）のスクリプトで中心的な役割を果たします。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **コード実行 (Execution)**, **防御回避 (Defense Evasion)**
* **解説**: `for.exe` というバイナリは存在しませんが、`cmd.exe` の `for` ループ機能は、LOLBASとして広く悪用されます。例えば、`for /L` を使ってポートスキャンを行ったり、`for /F` でコマンド出力を解析して情報を窃取したりします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `cmd.exe` のプロセス作成イベント (イベントID 4688) とコマンドライン引数を監視します。`for` ループを使った異常な繰り返し処理や、偵察活動（`net user` の繰り返し実行など）を検知ルールとして設定することが有効です。

## 注意点・補足

* **構文の複雑さ**: `for` コマンド、特に `for /F` の構文は非常に複雑で癖があります。PowerShellの `ForEach-Object` を使える環境であれば、そちらの方がはるかに直感的で強力です。
