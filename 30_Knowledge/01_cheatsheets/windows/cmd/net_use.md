# `net use` - ネットワークリソースに接続する

## 概要

`net use` コマンドは、リモートコンピュータ上の共有フォルダーやプリンターなどの共有リソースに接続するために使用されます。最も一般的な使い方は、共有フォルダーをローカルのドライブ文字（例: `Z:`）に割り当てる（マッピングする）ことです。また、現在のネットワーク接続の一覧表示や、接続の切断も行えます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`net share`** | `net share` は、自身のコンピュータでリソースを**公開（サーバー側）**します。`net use` は、他のコンピュータが公開しているリソースに**接続（クライアント側）**します。 | - |
| **PowerShell** | `New-PSDrive -Persist` や `Get-PSDrive` を使うことで、ネットワークドライブをオブジェクトとして、より柔軟に管理できます。 | `New-PSDrive` (PowerShell) |
| **`pushd`/`popd`** | `pushd \\server\share` は、一時的にネットワークドライブを割り当てて、そこに移動します。`popd` で元の場所に戻ると割り当てが解除されます。 | `pushd` (一時的な利用) |

## よく連携されるコマンド

`net use` は、`net view` で発見した共有リソースに接続したり、接続後に `dir` や `copy` でファイルを操作したりする一連の流れで使われます。

### シナリオ例: ファイルサーバーに接続してファイル操作を行う (インフラ構築・運用視点)

* **目的**: ファイルサーバー上の共有フォルダーを `Z:` ドライブとして割り当て、ファイルのコピーを行う。
* **連携コマンド**: `dir`, `copy`
* **解説**: `net use` でネットワークドライブを割り当てた後は、そのドライブ文字を使ってローカルドライブと同じようにファイル操作ができます。
* **コマンド例**:

    ```cmd
    REM \\fileserver\share を Z: ドライブに割り当てる
    net use Z: \\fileserver\share

    REM Z: ドライブの内容を確認
    dir Z:

    REM ローカルのファイルを Z: ドライブにコピー
    copy C:\Reports\report.docx Z:\

    REM 使い終わったら切断
    net use Z: /delete
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | 現在のネットワーク接続を一覧表示します。 |
| ⭐ `<devicename> \\<computername>\<sharename>` | 共有リソースを指定したデバイス名（ドライブ文字など）に割り当てます。 |
| ⭐ `*` | デバイス名として `*` を指定すると、次に利用可能なドライブ文字が自動的に割り当てられます。 |
| ⭐ `/delete` | 指定したネットワーク接続を解除します。`*` を指定すると全ての接続を解除します。 |
| **認証** | |
| `<password>` | パスワードを指定します。セキュリティ上、`*` を使って対話的に入力することが推奨されます。 |
| `/user:<username>` | 接続に使用するユーザー名を指定します。ドメインユーザーの場合は `DOMAIN\user` 形式です。 |
| **永続性** | |
| ⭐ `/persistent:{yes\|no}` | 接続を永続的にするかどうかを設定します。`yes` の場合、次回のログオン時にも接続が復元されます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログオンスクリプトで、ユーザーごとにホームディレクトリをネットワークドライブとして割り当てる。
* **組み合わせ**: `/persistent:yes`
* **解説**: ユーザーが常にアクセスする必要があるファイルサーバー上の共有を、ログイン時に自動的に割り当てます。
* **例**:

    ```cmd
    REM ユーザーのホームディレクトリを H: ドライブに割り当て
    net use H: \\fileserver\home\%USERNAME% /persistent:yes
    ```

### 2. ブルーチーム視点

* **タスク**: **ラテラルムーブメントの痕跡調査**。不審なネットワーク接続がないか確認する。
* **組み合わせ**: `net use`
* **解説**: インシデント発生時に、侵害されたホストで `net use` を実行します。もし、見慣れないサーバーや `IPC$` 共有への接続があれば、それは攻撃者が他のサーバーを偵察したり、攻撃ツールを展開したりするために確立した接続である可能性が高いです。
* **例**:

    ```cmd
    REM 現在確立されているネットワーク接続を監査
    net use
    ```

### 3. レッドチーム視点

* **タスク**: **ラテラルムーブメント (Lateral Movement)**。窃取した認証情報を使って、他のサーバーの管理共有に接続する。
* **組み合わせ**: `/user:<user> <password>`
* **解説**: 攻撃者は、`mimikatz` などで窃取した管理者アカウントのパスワードやハッシュを使い、`net use` で他のサーバーの `C$` や `ADMIN$` といった管理者共有に接続します。これにより、ネットワーク内の他のサーバーに侵入を拡大します。
* **例**:

    ```cmd
    REM 窃取した管理者 'AdminUser' のパスワードでターゲットサーバーの C$ 共有に接続
    net use \\TargetServer\C$ "P@ssw0rd123" /user:CORP\AdminUser

    REM (接続後) psexec などでコマンドを実行
    psexec \\TargetServer cmd.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `System error 53 has occurred. The network path was not found.`
    * **考えられる原因**: ホスト名が間違っている、サーバーがオフライン、またはファイアウォールでファイル共有 (SMB, TCP/445) がブロックされています。
    * **解決策**: `ping <server>` で疎通確認を行い、サーバー名とファイアウォールの設定を確認してください。

2. **エラーメッセージ**: `System error 5 has occurred. Access is denied.`
    * **考えられる原因**: 認証情報（ユーザー名/パスワード）が間違っているか、その共有リソースへのアクセス権がありません。
    * **解決策**: `/user` スイッチで正しい認証情報を指定してください。共有権限とNTFS権限の両方を確認する必要があります。

## 環境変数と設定ファイル

* `net use` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **平文パスワードの露呈**と**認証情報の悪用**。
* **悪用シナリオ**:
  * スクリプト内にパスワードを平文で記述すると、誰でもそのスクリプトを読めばパスワードが分かってしまいます。
  * レッドチームのシナリオで述べた通り、`net use` はラテラルムーブメントのための最も基本的なツールの1つです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **ラテラルムーブメント (Lateral Movement)**
* **解説**: `net use` は、攻撃者がネットワーク内のリソースを探索し、アクセスを拡大するために利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、不要なファイル共有を行わない。管理者共有 (`C$`) へのアクセスを厳しく制限する。
* **Detection (検知)**: Windows セキュリティイベントログで、ネットワーク共有への接続イベント (イベントID 5140) や、`net.exe` の実行 (プロセス作成イベントID 4688) を監視する。特に、短時間に多数のサーバーへの `net use` 接続が行われた場合は、ラテラルムーブメントの兆候である可能性が高いです。

## 注意点・補足

* **パスワードプロンプト**: `net use Z: \\server\share *` のように、パスワードの代わりに `*` を指定すると、コマンド実行時に安全なパスワード入力プロンプトが表示されます。バッチファイルでの使用には向きませんが、対話的な操作ではこの方法が推奨されます。
