# `if` - バッチファイルで条件分岐処理を行う

## 概要

`if` コマンドは、バッチファイル内で条件分岐を行うための基本的な制御構文です。指定した条件（エラーレベル、文字列の比較、ファイルの存在など）を評価し、その結果が真 (true) であれば、後続のコマンドを実行します。`else` 句と組み合わせることで、条件が偽 (false) の場合の処理も記述でき、スクリプトに柔軟なロジックを実装することが可能になります。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **PowerShell** | PowerShellの `If-ElseIf-Else` ステートメントは、より複雑な比較演算子 (`-gt`, `-lt`, `-match`など) を持ち、オブジェクトのプロパティを評価できるなど、はるかに高機能で柔軟です。 | `If` (PowerShell) |
| **`goto`** | `if` が条件を**評価**するのに対し、`goto` は指定したラベルに**ジャンプ**します。この2つを組み合わせることで、古典的な条件分岐ロジックを構築します。 | - |

## よく連携されるコマンド

`if` は、その評価結果に基づいてスクリプトの実行フローを変更するため、`goto` コマンドと密接に連携します。

### シナリオ例: エラーレベルに基づいた処理の分岐 (インフラ構築・運用視点)

* **目的**: `robocopy` などのコマンド実行結果（エラーレベル）に応じて、成功または失敗のメッセージを表示する。
* **連携コマンド**: `goto`, `%ERRORLEVEL%`
* **解説**: コマンドが成功すると `%ERRORLEVEL%` は `0` に、失敗すると `0` 以外になります。`if` を使ってこの値を評価し、`goto` でそれぞれの処理セクションにジャンプさせます。
* **コマンド例**:

    ```cmd
    @echo off
    robocopy C:\Source D:\Backup /E

    REM robocopyの成功は8未満
    if %ERRORLEVEL% LSS 8 (
        goto :success
    ) else (
        goto :failure
    )

    :success
    echo Backup completed successfully.
    goto :end

    :failure
    echo An error occurred during backup! Error level was %ERRORLEVEL%.
    goto :end

    :end
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **比較の種類** | |
| ⭐ `[NOT] ERRORLEVEL <number>` | 直前のコマンドの終了コードが、指定した数値**以上**であるか (またはそうでないか) を評価します。 |
| ⭐ `[NOT] "string1"=="string2"` | 2つの文字列が等しいか (または等しくないか) を評価します。 |
| ⭐ `[NOT] EXIST <filename>` | 指定したファイルが存在するか (またはしないか) を評価します。 |
| `/I` | (`"string1"=="string2"` と併用) 文字列比較の際に、**大文字と小文字を区別しません**。 |
| **比較演算子** | 数値や文字列の比較に使います。 |
| `EQU` | 等しい |
| `NEQ` | 等しくない |
| `LSS` | より小さい |
| `LEQ` | 以下 |
| `GTR` | より大きい |
| `GEQ` | 以上 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スクリプトの引数が指定されているかどうかを確認する。
* **組み合わせ**: `"%1" == ""`
* **解説**: バッチファイルの最初の引数 (`%1`) が空文字列 (`""`) と等しいかどうかをチェックし、引数がなければ使い方を表示して終了する、という定型的な処理です。
* **例**:

    ```cmd
    @echo off
    if "%1"=="" (
        echo Usage: %0 ^<servername^>
        goto :EOF
    )
    echo Pinging server: %1
    ping %1
    ```

### 2. ブルーチーム視点

* **タスク**: 特定のIOC (Indicator of Compromise) ファイルが存在するかどうかで処理を分岐させる簡易的な調査スクリプトを作成する。
* **組み合わせ**: `EXIST`
* **解説**: `if exist` を使って、マルウェアが作成する特定のファイルやアーティファクトの存在を確認し、存在した場合にのみアラートを出すなどの処理を行います。
* **例**:

    ```cmd
    REM 攻撃者がよく作成するファイルの存在を確認
    if exist C:\Users\Public\payload.exe (
        echo [ALERT] Suspicious file found: C:\Users\Public\payload.exe
    )
    ```

### 3. レッドチーム視点

* **タスク**: 自身が管理者権限で実行されているかを確認し、権限に応じてペイロードを切り替える。
* **組み合わせ**: `net session`, `if errorlevel`
* **解説**: `net session` コマンドは管理者権限がないと失敗 (`ERRORLEVEL` が `0` 以外になる) します。これを利用して、スクリプトが昇格された権限で実行されているかを判定し、その後の攻撃フローを変更します。
* **例**:

    ```cmd
    @echo off
    net session >nul 2>&1
    if %ERRORLEVEL% == 0 (
        echo Running as Administrator. Executing privileged payload...
        REM (管理者権限での処理)
    ) else (
        echo Running as standard user. Attempting UAC bypass...
        REM (一般ユーザー権限での処理)
    )
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **現象**: `if` 文の `(` や `)` が正しく解釈されない、または `The syntax of the command is incorrect.` エラーが出る。
    * **考えられる原因**:
      * `if condition (...)` のように、条件と開き括弧の間にスペースがない。
      * `else` が `)` と同じ行にない (`... ) else ( ...`)。
    * **解決策**:
      * `if condition (...)` -> `if condition (...)` とスペースを入れます。
      * `)` と `else` は `) else (` のように同じ行に記述する必要があります。

## 環境変数と設定ファイル

* `if` は、環境変数の値を比較するためによく使われます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

### 影響を与える環境変数

* **`%ERRORLEVEL%`**: 多くの `if` 条件分岐で評価の対象となります。
* **`%<変数名>%`**: `if "%VAR%"=="value"` のように、環境変数の値を比較できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `if` 自体ではなく、比較する変数にユーザー入力をそのまま使用し、かつ引用符で囲んでいない場合に、**コマンドインジェクション**の可能性があります。
* **悪用シナリオ**: `if %input%==value do_something` というコードで、攻撃者が `value do_something_else & malicious_command` のような文字列を入力すると、予期せぬコマンドが実行される可能性があります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **コード実行 (Execution)**
* **解説**: `if` は `cmd.exe` の内部コマンドであり、攻撃者が作成する悪意のあるバッチスクリプトのロジック制御（権限チェック、環境チェックなど）の中心的な役割を果たします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: バッチファイル内で変数を比較する際は、`if "%VAR%"=="value"` のように**必ずダブルクォートで囲む**。
* **Detection (検知)**: PowerShellのスクリプトブロックロギングやコマンドライン引数の監査を有効化し、バッチファイルから実行される不審なコマンドを捉える。

## 注意点・補足

* **遅延環境変数**: `for` ループ内でカウンタ変数を `if` で評価する場合など、ループ内で変化する変数を正しく扱うには、スクリプト冒頭で `setlocal enabledelayedexpansion` を宣言し、変数を `%VAR%` の代わりに `!VAR!` で参照する必要があります。
