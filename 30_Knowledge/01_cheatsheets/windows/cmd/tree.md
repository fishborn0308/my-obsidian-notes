# `tree` - ディレクトリ構造をツリー形式で表示する

## 概要

`tree` コマンドは、指定したディレクトリ配下のサブディレクトリと（オプションで）ファイルの構造を、視覚的に分かりやすいツリー形式で表示するコマンドです。プロジェクトのフォルダー構成を文書化したり、ディレクトリ構造の全体像を素早く把握したりするのに便利です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`dir /S /B`** | `dir` はファイルとディレクトリを**リスト形式**で表示します。`tree` は**階層的なツリー構造**で視覚的に表示します。構造を把握するには `tree` が適しています。 | `tree` (構造把握) |
| **PowerShell** | `Get-ChildItem -Recurse` はオブジェクトのリストを返します。`tree` のような視覚的なツリー表示を得るには、追加のフォーマット処理が必要です。 | - |

## よく連携されるコマンド

`tree` の出力は長くなることが多いため、リダイレクション (`>`) を使ってファイルに保存するのが一般的です。

### シナリオ例: ディレクトリ構造をドキュメント化する (インフラ構築・運用視点)

* **目的**: アプリケーションのディレクトリ構造をテキストファイルに出力し、構成ドキュメントの一部として保存する。
* **連携コマンド**: リダイレクション (`>`)
* **解説**: `tree` コマンドの出力を `>` を使ってファイルにリダイレクトします。`/F` でファイルも表示し、`/A` でASCII文字を使うことで、テキストエディタで文字化けなく表示できるようにします。
* **コマンド例**:

    ```cmd
    REM C:\Program Files\MyApp の構造を directory_structure.txt に保存
    tree "C:\Program Files\MyApp" /F /A > directory_structure.txt
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示内容** | |
| ⭐ `/F` | 各ディレクトリ内の**ファイル (Files)** も表示します。 |
| ⭐ `/A` | ツリーの描画に、拡張文字ではなく **ASCII** 文字 (`+`, `-`, `\`) を使用します。テキストファイルへのリダイレクトや、異なる文字コード環境での表示の際に文字化けを防ぎます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 現在のディレクトリ構造を確認する。
* **組み合わせ**: `tree /F`
* **解説**: サブディレクトリとファイルを含めた現在のディレクトリ構造を素早く視覚的に把握します。
* **例**:

    ```cmd
    REM 現在のディレクトリの構造をファイルも含めて表示
    tree /F
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによって作成されたファイルやディレクトリの全体像を把握する。
* **組み合わせ**: `tree <path> /F /A > suspicious_dir_tree.txt`
* **解説**: 攻撃者が作成したと思われるディレクトリ（例: `C:\Users\Public\payloads`）に対して `tree /F` を実行し、どのようなファイルが、どのような階層構造で配置されているかを記録・分析します。
* **例**:

    ```cmd
    REM 不審なディレクトリの構造を調査
    tree C:\Users\Public\Intel /F /A
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したシステムのファイルサーバーなどで、興味深いディレクトリ構造を探す。
* **組み合わせ**: `tree`
* **解説**: 攻撃者は侵入後、`dir` だけでなく `tree` も使って、`data`, `backup`, `config`, `users` といった名前のディレクトリを探し、ファイルシステムの全体像を素早く把握しようとします。これにより、情報価値の高いファイルがどこにありそうかを見定めます。
* **例**:

    ```cmd
    REM 'D:\shares' のディレクトリ構造を偵察
    tree D:\shares
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'tree'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `No subfolders exist`
    * **考えられる原因**: 指定したディレクトリにサブディレクトリが存在しません。
    * **解決策**: これはエラーではありません。`/F` スイッチを付けてファイルを表示するか、別のディレクトリを指定してください。

## 環境変数と設定ファイル

* `tree` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `tree` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`tree` の出力は、攻撃者がファイルシステムの構造やディレクトリの命名規則を把握し、重要な情報がどこに保存されているかを推測するための重要な偵察情報となります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `tree.com` は、攻撃者が侵入後の情報収集フェーズで、ファイルシステムの構造を把握するために利用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に基づき、ファイルやディレクトリに適切なACL（アクセス制御リスト）を設定し、ユーザーに不要なディレクトリへのアクセス権を与えない。
* **Detection (検知)**: `tree.com` の実行、特に重要なファイルサーバーやシステムディレクトリに対する実行を、プロセス監査ログ（イベントID 4688）やEDRで監視する。

## 注意点・補足

* **出力の量**: 非常に深い階層や多数のファイルを持つディレクトリに対して `/F` スイッチ付きで実行すると、出力が膨大になることがあります。`| more` を使うか、ファイルにリダイレクトすることを推奨します。
