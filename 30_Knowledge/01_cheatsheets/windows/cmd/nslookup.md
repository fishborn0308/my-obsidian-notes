# `nslookup` - DNSサーバーに問い合わせる

## 概要

`nslookup` (name server lookup) は、ドメイン名からIPアドレスを調べる（正引き）、またはIPアドレスからドメイン名を調べる（逆引き）など、DNS (Domain Name System) サーバーに問い合わせを行うためのコマンドラインツールです。対話モードと非対話モードの2つのモードを持ち、様々な種類のDNSレコードを照会できます。ネットワークのトラブルシューティングにおいて、名前解決の問題を診断する基本的な手段です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`ping`** | `ping` は名前解決を試みた後、ICMPパケットで疎通確認まで行います。`nslookup` は名前解決の**問い合わせのみ**に特化しています。 | - |
| **PowerShell** | `Resolve-DnsName` コマンドレットは、より詳細な情報をオブジェクトとして取得でき、スクリプトでの利用に適しています。 | `Resolve-DnsName` |
| **`dig` (Linux)** | Linux環境で標準的に使われる高機能なDNSクエリツール。`nslookup` よりも出力が詳細で、トラブルシューティングに優れています。 | `dig` (Linux環境) |

## よく連携されるコマンド

`nslookup` は、`ipconfig /flushdns` と連携して、DNSキャッシュをクリアした後の名前解決の動作を確認する際などに使われます。

### シナリオ例: 特定のDNSサーバーを使って名前解決を行う (インフラ構築・運用視点)

* **目的**: 社内のDNSサーバーではなく、Googleの公開DNSサーバー (`8.8.8.8`) を使って、外部からドメインがどのように見えるかを確認する。
* **連携コマンド**: (なし)
* **解説**: `nslookup` の引数として、解決したいドメイン名の後に、問い合わせ先のDNSサーバーを指定します。これにより、DNS設定の伝播状況などを確認できます。
* **コマンド例**:

    ```cmd
    REM [www.example.com](https://www.example.com) を 8.8.8.8 に問い合わせる
    nslookup [www.example.com](https://www.example.com) 8.8.8.8
    ```

## スイッチ/オプション説明

`nslookup` は対話モードと非対話モードでオプションの使い方が異なります。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **非対話モード** | |
| ⭐ `<domain>` | 問い合わせたいドメイン名またはIPアドレス。 |
| `[server]` | 問い合わせ先のDNSサーバーを明示的に指定します。 |
| ⭐ `-type=<record>` | 問い合わせるレコードタイプを指定します。(`A`, `MX`, `NS`, `SOA`, `CNAME`, `PTR` など) |
| **対話モード** (`nslookup` を引数なしで起動) | |
| `server <ip>` | 問い合わせ先のDNSサーバーを変更します。 |
| `set type=<record>` | 問い合わせるレコードタイプを変更します。 |
| `exit` | 対話モードを終了します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しく設定したWebサーバーのDNSレコード (Aレコード) が正しく引けるか確認する。
* **組み合わせ**: `nslookup <domain>`
* **解説**: ドメイン名の正引きを行い、意図したIPアドレスが返ってくるかを確認する最も基本的な使い方です。
* **例**:

    ```cmd
    REM [www.example.com](https://www.example.com) のIPアドレスを確認
    nslookup [www.example.com](https://www.example.com)
    ```

### 2. ブルーチーム視点

* **タスク**: ファイアウォールログで検知した不審なIPアドレスの素性を調査する。
* **組み合わせ**: `nslookup <suspicious_ip>`
* **解説**: 不審なIPアドレスの逆引き（PTRレコード問い合わせ）を行い、どのようなドメイン名が割り当てられているかを確認します。これにより、そのIPアドレスの所有組織などを推測する手がかりとなります。
* **例**:

    ```cmd
    REM 8.8.8.8 の逆引きを実行
    nslookup 8.8.8.8
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)** の一環として、標的組織のメールサーバー (MX) やネームサーバー (NS) を特定する。
* **組み合わせ**: `-type=MX <domain>`
* **解説**: 標的ドメインのMXレコードを調べることで、`mail.example.com` のようなメールサーバーのホスト名を特定します。メールサーバーは、フィッシング攻撃や認証情報を狙ったブルートフォース攻撃の格好の標的となります。
* **例**:

    ```cmd
    REM 標的ドメインのメールサーバーを特定
    nslookup -type=MX example.com

    REM 標的ドメインのネームサーバーを特定
    nslookup -type=NS example.com
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'nslookup'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `*** <server_name> can't find <domain>: Non-existent domain`
    * **考えられる原因**: `NXDOMAIN`。問い合わせたドメイン名はDNSサーバーに存在しません。
    * **解決策**: ドメイン名のスペルが正しいか確認してください。

2. **エラーメッセージ**: `*** <server_name> can't find <domain>: Server failed`
    * **考えられる原因**: `SERVFAIL`。DNSサーバーが何らかの理由で応答に失敗しました。
    * **解決策**: 別のDNSサーバー (`nslookup <domain> 8.8.8.8`など) を指定して再度試み、問題の切り分けを行います。

## 環境変数と設定ファイル

* `nslookup` がデフォルトで使用するDNSサーバーは、ネットワークアダプターのTCP/IP設定に基づきます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `nslookup` 自体ではなく、DNSプロトコルの仕組みやサーバーの設定不備が悪用されます。
* **悪用シナリオ**: `nslookup` は、攻撃者が**情報収集（偵察）**フェーズで、標的のインフラ（メールサーバー、DNSサーバー、サブドメインなど）をマッピングするために利用します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `nslookup.exe` は、攻撃者が侵入したWindowsシステムから、追加のツールなしでDNS偵察を行うために利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 権威DNSサーバーの設定で、ゾーン転送を信頼できるホストのみに制限する。
* **Detection (検知)**: 内部から外部への異常なDNSクエリ（特に `ANY` や `AXFR` タイプのリクエスト）をネットワーク監視ツールやDNSログで監視することで、偵察活動を検知できる可能性があります。

## 注意点・補足

* **対話モード**: `nslookup` を引数なしで実行すると、対話モードに入ります。このモード内で `set type=mx` や `server 8.8.8.8` のように設定を切り替えながら、連続してクエリを実行できます。
