# `net start` - Windowsサービスを開始する

## 概要

`net start` コマンドは、Windowsのサービスを開始するために使用されます。引数としてサービス名を指定するとそのサービスを起動し、引数なしで実行すると現在実行中の全てのサービスを一覧表示します。サービスのトラブルシューティングや、バッチファイルによるサービスの制御に利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`sc start`** | `sc` は、サービスの開始、停止、設定変更、依存関係の照会など、より包括的なサービス管理機能を提供します。`net start` はサービスの開始に特化しています。 | `sc` |
| **PowerShell** | `Start-Service` コマンドレットは、サービスをオブジェクトとして扱え、パイプラインを通じて他のコマンドレットと連携できるため、スクリプトでの利用に非常に強力です。 | `Start-Service` |

## よく連携されるコマンド

`net start` は、サービスの状態を確認する `sc query` や `net start` (引数なし) と連携して使われます。

### シナリオ例: サービスの起動と状態確認 (インフラ構築・運用視点)

* **目的**: 停止しているWebサービス (W3SVC) を起動し、正しく実行中になったことを確認する。
* **連携コマンド**: `findstr`
* **解説**: まず `net start` でサービスを起動し、その後 `net start` を引数なしで実行した結果を `findstr` でフィルタリングし、目的のサービスがリストに含まれているかを確認します。
* **コマンド例**:

    ```cmd
    REM World Wide Web Publishing Service を開始
    net start W3SVC

    REM サービスが実行中の一覧に含まれているか確認
    net start | findstr /I "World Wide Web Publishing Service"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | 現在実行中のサービスの一覧を表示します。 |
| ⭐ `<service>` | 開始するサービスの**サービス名**を指定します。サービス名にスペースが含まれる場合は、ダブルクォートで囲みます (例: `"Windows Update"`)。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: メンテナンスで停止したサービスを手動で開始する。
* **組み合わせ**: `net start <service>`
* **解説**: GUI (services.msc) を開かずに、コマンドラインから素早くサービスを開始します。
* **例**:

    ```cmd
    REM Windows Update サービスを開始
    net start "Windows Update"

    REM 実行中のサービス一覧を確認
    net start
    ```

### 2. ブルーチーム視点

* **タスク**: 調査のために一時的に停止させたサービスを再開する。
* **組み合わせ**: `net start <service>`
* **解説**: インシデント対応中に、影響調査のために特定のサービスを `net stop` で停止させた後、安全が確認された時点でサービスを再開するために使用します。
* **例**:

    ```cmd
    REM 調査完了後、Print Spooler サービスを再開
    net start Spooler
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** と**防御回避 (Defense Evasion)** のため、自身が作成または改ざんした悪意のあるサービスを開始する。
* **組み合わせ**: `net start <malicious_service_name>`
* **解説**: 攻撃者は、権限昇格後に `sc create` などでバックドアとなるサービスを登録します。その後、`net start` を使ってそのサービスを起動し、C2サーバーへの接続などを確立します。また、セキュリティ製品のサービスを `net stop` で停止させた後、再度 `net start` で起動して正常に動作しているように見せかけることもあります。
* **例**:

    ```cmd
    REM 攻撃者が 'SysHelper' という名前で登録した悪意のあるサービスを開始
    net start SysHelper
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'net'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `System error 5 has occurred. Access is denied.`
    * **考えられる原因**: サービスの開始には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

2. **エラーメッセージ例 2**: `The service name is invalid.`
    * **考えられる原因**: 指定したサービス名が存在しないか、スペルが間違っています。
    * **解決策**: `services.msc` や `sc query` を使って、正しい**サービス名**を確認してください。表示名ではないことに注意が必要です。

## 環境変数と設定ファイル

* `net start` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `net start` 自体ではなく、**不適切なサービス権限**や**脆弱なサービス実行可能ファイル**が悪用されます。
* **悪用シナリオ**: 攻撃者は、書き込み権限が緩いディレクトリに配置されたサービス実行可能ファイルを悪意のあるものに置き換え、`net start` でそれを起動させることで権限昇格を図ります (Path Interception)。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **コード実行 (Execution)**, **永続化 (Persistence)**
* **解説**: `net start` は、攻撃者がシステムに登録した悪意のあるサービスを起動するための環境寄生型の手法として利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: サービスの実行可能ファイルのパーミッションを厳格に管理し、不要な書き込み権限を与えない。AppLockerやWDACで、信頼できないサービスの実行を禁止する。
* **Detection (検知)**: Windows セキュリティイベントログで、サービスの開始 (イベントID 7036) と、新しいサービスのインストール (イベントID 4697/7045) を監視する。

## 注意点・補足

* **サービス名 vs 表示名**: `net start` で指定するのは、「サービス」管理ツール (`services.msc`) で表示される「サービス名」です。「表示名」ではないことに注意してください（例: "Print Spooler" のサービス名は "Spooler"）。
