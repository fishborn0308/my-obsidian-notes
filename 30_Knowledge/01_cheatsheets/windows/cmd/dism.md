# `dism` - Windowsイメージのサービスと管理

## 概要

`DISM` (Deployment Image Servicing and Management) は、Windowsイメージ (`.wim`, `.vhd`, `.vhdx`) のサービスと管理を行うためのコマンドラインツールです。主な用途は2つあります。

1. **オフラインイメージの編集**: Windowsインストールメディアやリカバリイメージをマウントし、ドライバー、パッケージ、言語パックなどを追加・削除する。
2. **オンラインイメージの修復**: 現在実行中のWindows (`/Online`) のコンポーネントストアをチェックし、破損したシステムファイルを修復する。これは `sfc /scannow` が失敗した場合の、より強力な修復手段として利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`sfc`** | `sfc` は保護された**システムファイル**の整合性をチェックし、ローカルのキャッシュから修復します。`dism` はその修復ソースとなる**コンポーネントストア**自体の破損をチェック・修復できます。 | `sfc` (一次対応) -> `dism` (二次対応) |
| **`chkdsk`** | **ファイルシステム**のエラーをチェック・修復します。`dism` はWindowsの**イメージとコンポーネント**を対象とします。 | - |
| **PowerShell** | `Repair-WindowsImage` などのコマンドレットが `dism` と同等の機能を提供し、スクリプトでの自動化に適しています。 | `Repair-WindowsImage` (PowerShell) |

## よく連携されるコマンド

`dism` は、`sfc` が修復に失敗した場合に、その修復ソースを直すために連携して使われます。

### シナリオ例: 破損したWindowsシステムファイルを修復する (インフラ構築・運用視点)

* **目的**: システムファイルチェッカー (`sfc`) が修復不可能なエラーを報告した場合に、`dism` を使ってコンポーネントストアを修復し、再度 `sfc` を実行する。
* **連携コマンド**: `sfc`
* **解説**: `sfc` が「Windowsリソース保護は、破損したファイルを検出しましたが、それらの一部を修正できませんでした。」と報告した場合、まず `dism /Online /Cleanup-Image /RestoreHealth` で修復ソースを健全な状態に戻してから、再度 `sfc /scannow` を実行するのが標準的な修復手順です。
* **コマンド例**:

    ```cmd
    REM 1. DISM を使って、Windows Update から健全なファイルを取得し、コンポーネントストアを修復
    dism /Online /Cleanup-Image /RestoreHealth

    REM 2. DISM完了後、再度システムファイルチェッカーを実行
    sfc /scannow
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **イメージ指定** | |
| ⭐ `/Online` | 現在実行中のオペレーティングシステムを対象とします。 |
| `/Image:<path>` | オフラインのWindowsイメージファイル (`.wim`) のパスを指定します。 |
| **クリーンアップと修復** | |
| ⭐ `/Cleanup-Image` | コンポーネントストアのクリーンアップと修復操作を行います。 |
| ⭐ `/CheckHealth` | コンポーネントストアに破損が記録されているかを**高速に**チェックします。スキャンは実行しません。 |
| `/ScanHealth` | コンポーネントストアの破損をスキャンします。`/CheckHealth` より時間がかかります。 |
| ⭐ `/RestoreHealth` | コンポーネントストアの破損をスキャンし、**自動的に修復**を試みます。 |
| `/Source:<path>` | `/RestoreHealth` と併用。Windows Updateの代わりに、指定した場所から修復用ファイルを取得します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Windowsの動作が不安定な場合に、システムの健全性を確認し、修復する。
* **組み合わせ**: `/Online /Cleanup-Image /RestoreHealth`
* **解説**: システムファイルやコンポーネントストアの破損を修復するための最も一般的なコマンドです。定期的なメンテナンスや、トラブルシューティングの際に実行されます。
* **例**:

    ```cmd
    REM 実行中のWindowsの健全性を確認し、問題があれば修復する
    dism /Online /Cleanup-Image /RestoreHealth
    ```

### 2. ブルーチーム視点

* **タスク**: システムファイルの改ざんが疑われる場合に、システムの整合性を検証する。
* **組み合わせ**: `dism /Online /Cleanup-Image /ScanHealth`
* **解説**: インシデント対応中に、マルウェアによってシステムファイルが改ざんされていないかを確認するために、まず修復を伴わないスキャンを実行します。ここで破損が検出されれば、攻撃による改ざんの可能性があります。
* **例**:

    ```cmd
    REM 現在のシステムのコンポーネントストアをスキャン
    dism /Online /Cleanup-Image /ScanHealth
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。システムのパッケージや機能をリストアップする。
* **組み合わせ**: `dism /Online /Get-Features`
* **解説**: 攻撃者は侵入後、このコマンドでシステムにインストールされているWindowsの機能や役割（例: `NetFx3`, `IIS-WebServerRole`）を特定します。これにより、システムの役割を推測し、有効化されている機能に存在する脆弱性を探して攻撃の足がかりとします。
* **例**:

    ```cmd
    REM インストールされているWindowsの機能一覧を取得
    dism /Online /Get-Features
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Error: 0x800f081f - The source files could not be found.`
    * **考えられる原因**: `/RestoreHealth` を実行した際に、Windows Update サービスに接続できないか、必要な修復ファイルを見つけられませんでした。
    * **解決策**:
      * インターネット接続を確認します。
      * `/Source` オプションを使い、Windowsインストールメディア (`install.wim`) などを修復ソースとして明示的に指定します。

        ```cmd
        dism /Online /Cleanup-Image /RestoreHealth /Source:WIM:D:\sources\install.wim:1 /limitaccess
        ```

## 環境変数と設定ファイル

* `dism` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `dism` 自体ではなく、**オフラインイメージへのマルウェア注入**が悪用される可能性があります。
* **悪用シナリオ**: 攻撃者が企業の展開用Windowsイメージ (`.wim`) にアクセスできた場合、`dism` を使ってオフラインでイメージにバックドアやマルウェアを注入する可能性があります。これにより、今後そのイメージから展開される全てのPCが、最初から侵害された状態になってしまいます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **防御回避 (Defense Evasion)**
* **解説**: `dism` は、システムの機能やパッケージをリストアップする偵察ツールとして利用されます。また、特定の条件下では、`dism` を使ってPEファイルをインポートし、セキュリティ製品のスキャンを回避してコードを実行するような高度な手法も存在します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 展開用のマスターイメージへのアクセス制御を厳格にし、定期的にイメージのハッシュ値を確認して改ざんがないか監査する。
* **Detection (検知)**: `dism` コマンドの実行、特にオフラインイメージ (`/Image:`) に対する操作をプロセス監査ログで監視する。

## 注意点・補足

* **管理者権限**: `dism` を使ってシステムの修復や変更を行うには、コマンドプロンプトを「管理者として実行」する必要があります。
