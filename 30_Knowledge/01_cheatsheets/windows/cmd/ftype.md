# `ftype` - ファイルタイプに関連付けられた実行コマンドを表示・変更する

## 概要

`ftype` (file type) コマンドは、「ファイルタイプ」と、その種類のファイルを開く際に実行されるコマンド文字列との関連付けを表示または変更するために使用されます。

このコマンドは、`assoc` コマンドと対になる関係です。`assoc` がファイル拡張子 (`.txt`) とファイルタイプ (`txtfile`) を紐付けるのに対し、`ftype` はそのファイルタイプ (`txtfile`) と実行するプログラム (`notepad.exe %1`) を紐付けます。この2段階の仕組みによって、Windowsはファイルをダブルクリックしたときにどのプログラムを起動するかを決定しています。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`assoc`** | `ftype` が「ファイルタイプ」と「実行コマンド」を紐付けるのに対し、`assoc` は「ファイル拡張子」と「ファイルタイプ」を紐付けます。これらは常にセットで使われます。 | - |
| **PowerShell** | `Get-Item HKCR:\txtfile\shell\open\command` のようにレジストリを直接操作することで、より詳細な関連付け情報を取得・設定できます。 | PowerShell (スクリプト) |

## よく連携されるコマンド

`ftype` は、ファイル拡張子の関連付けを管理する `assoc` と密接に連携します。

### シナリオ例: ファイルを開いたときの動作を理解する (インフラ構築・運用視点)

* **目的**: `.txt` ファイルをダブルクリックしたときに `notepad.exe` が起動する仕組みを、`assoc` と `ftype` を使って完全に理解する。
* **連携コマンド**: `assoc`
* **解説**: まず `assoc` で拡張子に対応するファイルタイプを調べ、次に `ftype` でそのファイルタイプに対応する実行コマンドを調べます。
* **コマンド例**:

    ```cmd
    REM 1. .txt 拡張子に関連付けられているファイルタイプを確認
    assoc .txt
    REM > .txt=txtfile

    REM 2. "txtfile" というファイルタイプがどのコマンドを実行するかを確認
    ftype txtfile
    REM > txtfile=%SystemRoot%\system32\NOTEPAD.EXE %1
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示と設定** | |
| ⭐ `(引数なし)` | 定義されている全てのファイルタイプとその実行コマンドの一覧を表示します。 |
| ⭐ `ftype <filetype>` | 指定したファイルタイプの現在の実行コマンドを表示します。 |
| ⭐ `ftype <filetype>=<command>` | (`管理者権限要`) 指定したファイルタイプに、実行するコマンド文字列を関連付けます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 特定のファイルタイプがどのプログラムで開かれるように設定されているかを確認する。
* **組み合わせ**: `ftype <filetype>`
* **解説**: ソフトウェアのインストールやアンインストールによって、意図しないプログラムに関連付けが変更されていないかを確認する際に使用します。
* **例**:

    ```cmd
    REM "Python.File" というファイルタイプがどのPythonインタプリタで実行されるか確認
    ftype Python.File
    ```

### 2. ブルーチーム視点

* **タスク**: **ファイルタイプの関連付けハイジャック**がされていないか監査する。
* **組み合わせ**: `ftype <filetype>`
* **解説**: 攻撃者が `ftype` を使って、`txtfile` のような無害なファイルタイプの実行コマンドを、悪意のあるプログラムに書き換えていないかを確認します。これは `assoc` コマンドの監査とセットで行います。
* **例**:

    ```cmd
    REM テキストファイルのファイルタイプの定義を確認
    ftype txtfile
    REM > 正常であれば notepad.exe になっているはず
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のために、既存のファイルタイプをハイジャックする。
* **組み合わせ**: `ftype <filetype>=<malicious_command>`
* **解説**: 攻撃者は、ユーザーが頻繁に開くファイルタイプ（例: `txtfile`）の実行コマンドを、自身のペイロード（リバースシェルなど）に書き換えます。これにより、ユーザーが次にテキストファイルを開いたときに、メモ帳ではなく攻撃者のプログラムが実行されるようになります。
* **例**:

    ```cmd
    REM txtfileの関連付けを、C:\Users\Publicにあるペイロードに書き換える
    ftype txtfile=C:\Users\Public\payload.exe %1
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: `ftype` で関連付けを変更するには管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

`ftype` のコマンド文字列内では、特殊な変数が利用できます。

### 影響を与える環境変数

* **`%1`**: 実行時に、開こうとしたファイルのフルパスに置き換えられます。
* **`%*`**: `%1` と同様に、全てのパラメータに置き換えられます。

### 関連するレジストリ/設定ファイル

* **`HKEY_CLASSES_ROOT\<filetype>\shell\open\command`**: `ftype` コマンドは、このレジストリキーの値を読み書きしています。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **ファイル関連付けのハイジャック**: レッドチームのシナリオで述べた通り、`ftype` と `assoc` の組み合わせは、ユーザーの操作をトリガーとしてマルウェアを実行させるための古典的で強力な永続化手法です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **永続化 (Persistence)**, **防御回避 (Defense Evasion)**
* **解説**: `ftype` はWindowsに標準で搭載されている正規のコマンドであり、攻撃者はアンチウイルス製品に検知されにくい「環境寄生型」の攻撃手法としてこれを利用します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: AppLockerやWDACを使い、信頼されていないパスからの実行ファイルの起動を禁止する。
* **Detection (検知)**: `HKEY_CLASSES_ROOT` 配下のレジストリキーへの変更、特に `shell\open\command` の値の変更を `Sysmon` (イベントID 13) などで厳重に監視します。

## 注意点・補足

* **元に戻す**: 関連付けを元に戻したい場合は、正常なPCで `ftype <filetype>` を実行して正しいコマンド文字列を確認し、それを問題のPCで再設定します。変更前には必ず元の設定を控えておきましょう。
