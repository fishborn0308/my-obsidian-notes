# `ftp` - ファイル転送プロトコル (非推奨)

## 概要

`ftp` コマンドは、FTP (File Transfer Protocol) を使用して、リモートのFTPサーバーとの間でファイルを送受信するための対話的なクライアントです。

**⚠️ 重要**: このコマンドは**レガシー（旧式）**です。FTPプロトコルは、ユーザー名、パスワード、転送されるデータを含む**全ての通信を暗号化せずに平文で送信します**。そのため、現代のネットワーク環境でこれを使用することは**極めて危険**です。安全なファイル転送には `sftp` (SSH File Transfer Protocol) や `scp`、あるいはWebDAV over HTTPSなどの暗号化されたプロトコルの使用が強く推奨されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`sftp` / `scp`** | SSHプロトコル上で動作し、全ての通信を暗号化します。`ftp` の安全な代替手段です。 | `sftp`, `scp` |
| **PowerShell** | `Invoke-WebRequest` や、`Posh-SSH` などのモジュールを使うことで、より高度で安全なファイル転送をスクリプト化できます。 | PowerShell |

## よく連携されるコマンド

`ftp` は、`-s` スイッチを使ってコマンドを記述したテキストファイルを読み込ませることで、非対話的なバッチ処理（自動アップロード/ダウンロード）に使われることがあります。

### シナリオ例: スクリプトによるファイルの自動アップロード (インフラ構築・運用視点)

* **目的**: バッチファイルから、生成したレポートファイルをFTPサーバーに自動でアップロードする。
* **連携コマンド**: (バッチファイル内)
* **解説**: `open`, `user`, `put` などのFTPコマンドを記述したスクリプトファイルを作成し、`ftp -s` で実行します。**（注: パスワードが平文で見えるため、非常に危険な方法です）**
* **コマンド例**:

    ```cmd
    REM ftp_script.txt の中身:
    open ftp.example.com
    myusername
    MyS3cr3tP@ssw0rd
    binary
    cd /reports
    put daily_report.zip
    bye

    REM バッチファイルから上記スクリプトを実行
    ftp -n -s:ftp_script.txt
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **起動オプション** | |
| ⭐ `-s:<filename>` | 指定したテキストファイルに含まれるFTPコマンドを実行します（スクリプト実行）。 |
| `-i` | 複数ファイルを転送する際の対話的なプロンプトを無効にします。 |
| `-n` | 初期接続時の自動ログインを無効にします。 |
| **対話モードの主なコマンド** | `ftp` 実行後に `ftp>` プロンプトで入力 |
| `open <host>` | FTPサーバーに接続します。 |
| `get <remote-file>` | ファイルをダウンロードします。 |
| `put <local-file>` | ファイルをアップロードします。 |
| `mget`, `mput` | 複数ファイルを一括でダウンロード/アップロードします。 |
| `ls`, `dir` | リモートのファイル一覧を表示します。 |
| `cd <dir>` | リモートのディレクトリを移動します。 |
| `binary` / `ascii` | 転送モードをバイナリ/アスキーに設定します。 |
| `bye` / `quit` | セッションを終了します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: どうしてもFTPしか使えないレガシーな機器（古いネットワーク機器など）からファームウェアやログをダウンロードする。
* **組み合わせ**: 対話モード
* **解説**: 限定的かつ信頼できる閉域網内で、やむを得ずFTPを使用するケース。
* **例**:

    ```cmd
    C:\> ftp
    ftp> open 192.168.1.254
    ftp> (ユーザー名とパスワードを入力)
    ftp> get config.bak
    ftp> bye
    ```

### 2. ブルーチーム視点

* **タスク**: **ネットワークトラフィックの監査**。平文のFTP通信を検知する。
* **組み合わせ**: (Wiresharkなどのパケットキャプチャツール)
* **解説**: ネットワーク上でFTP (TCP/21) 通信が流れていることを検知した場合、それはセキュリティポリシー違反の明確な兆候です。Wiresharkで通信をキャプチャすれば、`USER` や `PASS` コマンドでやり取りされる認証情報が平文で読み取れてしまうことを実証できます。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **情報窃取 (Data Exfiltration)** と**ペイロードのダウンロード**。
* **組み合わせ**: `ftp -s:<script>`
* **解説**: 攻撃者は、侵入したシステムから外部へデータを持ち出すための簡易な方法として `ftp.exe` を利用します。また、ファイアウォールの設定が緩く、アウトバウンドのFTPが許可されている場合、攻撃ツールを外部のFTPサーバーからダウンロードするためにも使われます。
* **例**:

    ```cmd
    REM C2サーバーからツールをダウンロードするためのスクリプトを実行
    ftp -n -s:C:\Users\Public\download.txt
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `530 Login incorrect.`
    * **考えられる原因**: ユーザー名またはパスワードが間違っています。
    * **解決策**: 正しい認証情報を再入力してください。

2. **エラーメッセージ**: `Connection timed out.`
    * **考えられる原因**: サーバーに到達できないか、ファイアウォールでポート21 (FTP制御ポート) がブロックされています。
    * **解決策**: `ping` でサーバーへの疎通を確認し、ファイアウォールの設定を見直してください。

## 環境変数と設定ファイル

* `ftp` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **平文での認証情報とデータ転送**。これがFTPの根本的かつ致命的な脆弱性です。
* **悪用シナリオ**: ネットワーク経路上でパケットを盗聴（スニッフィング）することで、攻撃者はFTPのユーザー名とパスワードを容易に窃取し、サーバーに不正アクセスできます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **データ窃取 (Data Exfiltration)**, **ダウンロード (Download)**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Ftp/](https://lolbas-project.github.io/lolbas/Binaries/Ftp/)
* **解説**: `ftp.exe` は、攻撃者が追加のツールをインストールすることなく、侵入したWindowsシステムから外部へデータを持ち出したり、逆に外部からツールを持ち込んだりするために利用する典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **FTPサーバーの使用を禁止し、SFTP/FTPSへ移行する**。境界ファイアウォールで、不要なFTP (TCP/21, 20) のアウトバウンド通信をブロックする。
* **Detection (検知)**: ネットワーク侵入検知システム(IDS)でFTP通信を監視し、アラートを発報する。`ftp.exe` の実行をプロセス監査ログ (イベントID 4688) で監視する。

## 注意点・補足

* **アクティブモード vs パッシブモード**: FTPには2つの接続モードがあり、ファイアウォール環境下ではパッシブモード (`passive` コマンド) を使う必要があることが多いです。
