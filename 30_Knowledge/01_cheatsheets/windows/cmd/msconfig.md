# `msconfig` - システム構成ユーティリティを起動する

## 概要

`msconfig` コマンドは、**システム構成ユーティリティ**のGUI (`msconfig.exe`) を起動します。このツールは、Windowsの起動プロセスをトラブルシューティングし、システムのパフォーマンスに影響を与える可能性のあるサービスやスタートアッププログラムを管理するために使用されます。主に、システムの起動モード（通常、診断、セレクティブスタートアップ）の変更や、問題の原因を切り分けるために一時的にサービスやプログラムを無効化するのに役立ちます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`taskmgr` (タスクマネージャー)** | モダンなWindowsでは、**スタートアップタブ**が `msconfig` のスタートアップ管理機能を引き継いでおり、より高機能です。 | `taskmgr` (スタートアップ管理) |
| **`services.msc`** | サービスの一覧表示と管理に特化したGUIツール。`msconfig` よりも詳細なサービスの設定が可能です。 | `services.msc` (サービス管理) |
| **`bcdedit`** | `msconfig` の**ブートタブ**で設定できる内容を、コマンドラインからより詳細に制御できます。 | `bcdedit` (ブート設定) |

## よく連携されるコマンド

`msconfig` はGUIツールであり、他のコマンドと直接連携することは稀です。むしろ、システムのトラブルシューティングの一環として、他の診断ツールと組み合わせて使われます。

### シナリオ例: クリーンブートによる問題の切り分け (インフラ構築・運用視点)

* **目的**: システムの動作が不安定な原因が、サードパーティ製のサービスやスタートアッププログラムにあるかを特定する。
* **連携コマンド**: (GUI操作)
* **解説**: `msconfig` を起動し、「サービス」タブで「Microsoft のサービスをすべて隠す」にチェックを入れ、残りのサービスをすべて無効化します。次に「スタートアップ」タブ（またはタスクマネージャー）で全てのスタートアップ項目を無効化してから再起動します。これで問題が解決すれば、無効化した項目の中に原因があることが分かります。
* **コマンド例**:

    ```cmd
    REM システム構成ユーティリティを起動
    msconfig
    ```

## スイッチ/オプション説明

`msconfig` コマンド自体にはスイッチはありません。実行するとGUIアプリケーションが起動します。以下はGUI内の主要なタブです。

| タブ名 | 説明 |
| :--- | :--- |
| **全般 (General)** | スタートアップの種類を選択します（通常、診断、スタートアップのオプションを選択）。 |
| **ブート (Boot)** | OSの起動オプション（セーフブート、GUIブートなしなど）を設定します。 |
| **サービス (Services)** | システム起動時に読み込まれるサービスを有効/無効にします。 |
| **スタートアップ (Startup)** | ユーザーログイン時に自動的に開始されるプログラムを管理します（新しいWindowsではタスクマネージャーへのリンクになっています）。 |
| **ツール (Tools)** | イベントビューアーやパフォーマンスモニターなど、他の管理ツールを起動するためのランチャー。 |

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `この操作を実行するには、管理者特権が必要です。`
    * **考えられる原因**: `msconfig` はシステム全体の設定を変更するため、管理者権限が必要です。
    * **解決策**: `msconfig` を管理者として実行してください（スタートメニューで `msconfig` と入力し、右クリックして「管理者として実行」を選択）。

## 環境変数と設定ファイル

* `msconfig` が変更する設定は、主にレジストリやBCD (Boot Configuration Data) ストアに保存されます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `msconfig.exe` 自体が、UACバイパスの手法として悪用されることがあります。
* **悪用シナリオ**: 攻撃者はレジストリを改ざんし、`msconfig.exe` のヘルプ機能などが呼び出すプログラムのパスを、悪意のある実行ファイルに差し替えます。これにより、正規の `msconfig.exe` を（管理者権限で）実行した際に、裏で悪意のあるコードが実行されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **UACバイパス**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Msconfig/](https://lolbas-project.github.io/lolbas/Binaries/Msconfig/)
* **解説**: `msconfig.exe` はMicrosoft署名付きの信頼されたバイナリであり、UACプロンプトなしで管理者権限に昇格できます。この性質を悪用し、レジストリ操作と組み合わせることで、権限昇格の足がかりとして利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `msconfig.exe` による不審なレジストリキーの変更や、子プロセスの生成をEDRなどで監視する。スタートアップ項目の変更を監視し、意図しないプログラムが追加されていないか定期的に監査する。

## 注意点・補足

* **タスクマネージャーへの移行**: Windows 8以降、スタートアップ項目の管理機能はタスクマネージャー (`taskmgr`) の「スタートアップ」タブに移行されました。`msconfig` のスタートアップタブは、タスクマネージャーを開くためのショートカットになっています。
