# `chkdsk` - ディスクのエラーをチェックし修復する

## 概要

`chkdsk` (Check Disk) コマンドは、ディスクボリューム（例: C:ドライブ）のファイルシステムとメタデータの論理的な整合性をチェックし、問題を検出した場合には修復を試みるユーティリティです。予期せぬシャットダウンやソフトウェアのクラッシュによって発生したファイルシステムの不整合、クロスリンクされたファイル、失われたクラスターなどを修正できます。また、物理的なディスク表面をスキャンして不良セクタを特定し、データの回復を試みることも可能です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`sfc /scannow`** | `sfc` は**Windowsの保護されたシステムファイル**の整合性をチェックし、破損している場合は正しいバージョンに置き換えます。`chkdsk` は**ディスク全体のファイルシステム**のエラーをチェックします。 | `sfc` (システムファイル) |
| **`format`** | `format` はディスクを**初期化**し、全てのデータを消去します。`chkdsk` はデータを**保持したまま**修復を試みます。 | - |
| **PowerShell** | `Repair-Volume` コマンドレットが `chkdsk` と同様の機能を提供します。 | PowerShell (スクリプト) |

## よく連携されるコマンド

`chkdsk` は、実行結果（特に修復が必要な場合）がイベントログに記録されるため、イベントビューアー (`eventvwr.msc`) と連携して結果を確認します。

### シナリオ例: システムドライブのスキャン結果を確認する (インフラ構築・運用視点)

* **目的**: 次回の再起動時にスケジュールされた `chkdsk /f` の実行結果を確認する。
* **連携コマンド**: `eventvwr.msc`
* **解説**: システムドライブ (C:) に対して修復オプション付きで `chkdsk` を実行すると、通常は次回の再起動時にスキャンがスケジュールされます。スキャン完了後、Windowsイベントログに結果が記録されるため、イベントビューアーでその内容を確認します。
* **コマンド例**:

    ```cmd
    REM イベントビューアーを起動
    eventvwr.msc
    REM Windows ログ -> Application を開き、ソースが "Wininit" または "Chkdsk" のイベントを探す
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `<volume>` | チェックするドライブ文字を指定します。（例: `C:`, `D:`） |
| ⭐ `/F` | ディスク上のエラーを**修正 (Fix)** します。このスイッチがない場合、`chkdsk` は読み取り専用モードで実行されます。 |
| ⭐ `/R` | 不良セクタを見つけ、読み取り可能な情報を**回復 (Recover)** します。`/F` の機能を含みます。 |
| **詳細操作** | |
| `/X` | 必要であれば、スキャン前にボリュームを強制的に**マウント解除**します。`/F` と共に使われることが多いです。 |
| `/B` | (NTFSのみ) ボリューム上の不良クラスターのリストをクリアし、既知の不良クラスターを再スキャンします。`/R` の機能を含みます。 |
| `/scan` | (NTFSのみ) オンラインスキャンを実行します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システムの動作が不安定な場合に、ディスクのファイルシステムに問題がないかを確認し、修復する。
* **組み合わせ**: `/F /R`
* **解説**: `/F` でファイルシステムエラーを修正し、`/R` で物理的な不良セクタのスキャンと回復も同時に行います。最も包括的なスキャンですが、ディスクサイズによっては非常に時間がかかることがあります。
* **例**:

    ```cmd
    REM Dドライブを包括的にチェックし、エラーを修正
    chkdsk D: /F /R
    ```

### 2. ブルーチーム視点

* **タスク**: **フォレンジックイメージの整合性確認**。
* **組み合わせ**: `chkdsk <mounted_image_drive_letter>`
* **解説**: 調査対象のディスクイメージを読み取り専用でマウントした後、`chkdsk` を（修復オプションなしで）実行します。これにより、ファイルシステムが破損していないか、あるいは攻撃者によって意図的に破損させられた痕跡がないかを確認します。
* **例**:

    ```cmd
    REM マウントしたEドライブのイメージを読み取り専用でスキャン
    chkdsk E:
    ```

### 3. レッドチーム視点

* **タスク**: **サービス妨害 (Denial of Service, DoS)**。
* **組み合わせ**: `chkdsk C: /f`
* **解説**: 攻撃者が管理者権限を奪取した後、このコマンドを実行することで、システムの再起動を強制し、`chkdsk` が完了するまでの間、長時間システムをオフラインにさせることができます。これは直接的な破壊活動ではありませんが、ビジネスに大きな影響を与える可能性があります。

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Chkdsk cannot run because the volume is in use by another process. Would you like to schedule this volume to be checked the next time the system restarts? (Y/N)`
    * **考えられる原因**: チェック対象がOSのシステムドライブ (C:) であるか、他のプロセスがファイルを開いているため、ロックをかけられません。
    * **解決策**: `Y` を入力してEnterキーを押します。これにより、次回のWindows起動時に `chkdsk` が実行されます。

## 環境変数と設定ファイル

* `chkdsk` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `chkdsk` 自体に脆弱性があることは稀ですが、その動作がサービス妨害に悪用される可能性があります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、攻撃者は `chkdsk` を実行させてシステムの再起動を誘発し、長時間にわたるダウンタイムを引き起こす可能性があります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* `chkdsk` はファイルシステムのチェックと修復に特化しており、コード実行やファイルダウンロードなどの機能を持たないため、LOLBASのリストには通常含まれません。しかし、DoS攻撃の文脈で利用される可能性はあります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 管理者権限へのアクセスを厳格に管理する。
* **Detection (検知)**: `chkdsk` の実行 (プロセス作成イベントID 4688) や、再起動時のスキャン予約に関連するイベントログ (ソース: Wininit) を監視し、計画外の `chkdsk` が実行されていないかを確認する。

## 注意点・補足

* **時間**: `/R` オプションを付けた `chkdsk` は、ディスクの容量によっては完了までに数時間以上かかることがあります。実行は慎重に計画してください。
* **データ損失のリスク**: `chkdsk` はエラーを修復しようとしますが、ファイルシステムが深刻に破損している場合、修復プロセス中にデータの一部が失われる可能性があります。実行前には重要なデータのバックアップを取ることが強く推奨されます。
