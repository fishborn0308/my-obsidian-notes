# `takeown` - ファイルの所有権を取得する

## 概要

`takeown` コマンドは、アクセスが拒否されたファイルやフォルダーの**所有権**を、管理者権限を持つユーザーが強制的に取得するためのコマンドラインツールです。ファイルのアクセス許可 (ACL) を変更するには、まずそのファイルの所有者である必要があります。`takeown` を使用して所有権を取得した後、`icacls` コマンドでアクセス許可を変更するというのが一般的な流れです。これにより、ロックアウトされたファイルやシステムファイルへのアクセスを回復できます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`icacls`** | `icacls` はファイルの**アクセス許可 (パーミッション)** を変更します。`takeown` はその前段階として、ファイルの**所有権**を変更します。所有者でなければ `icacls` でパーミッションを変更できないことが多いです。 | - |
| **PowerShell** | `Get-Acl` と `Set-Acl` を組み合わせて所有者を変更できますが、`takeown` の方が単一のコマンドで完結するため、手軽な場合があります。 | `Set-Acl` (スクリプト) |

## よく連携されるコマンド

`takeown` は、所有権を取得した後にアクセス権を変更するため、`icacls` と密接に連携します。

### シナリオ例: アクセス不能になったフォルダーへのアクセス権を回復する (インフラ構築・運用視点)

* **目的**: 何らかの理由でアクセスできなくなったフォルダーの所有権を管理者に移し、フルコントロール権限を付与する。
* **連携コマンド**: `icacls`
* **解説**: まず `takeown` でフォルダーとその中身の所有権を Administrators グループに取得します。その後、`icacls` を使って Administrators グループにフルコントロールを与えます。
* **コマンド例**:

    ```cmd
    REM D:\LockedFolder の所有権をAdministratorsグループに再帰的に設定
    takeown /F "D:\LockedFolder" /A /R

    REM 所有権取得後、Administratorsグループにフルコントロールを再帰的に付与
    icacls "D:\LockedFolder" /grant Administrators:F /T
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **対象指定** | |
| ⭐ `/F <filename>` | 所有権を取得するファイル名またはフォルダー名を指定します。 |
| **動作制御** | |
| ⭐ `/A` | 所有権を現在のユーザーではなく、**Administrators グループ**に与えます。 |
| ⭐ `/R` | ディレクトリ内のファイルとサブディレクトリに対して、**再帰的に**操作を実行します。 |
| `/D {Y\|N}` | 確認プロンプトに対するデフォルトの応答を指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ユーザープロファイルの移行やバックアップで、アクセス権エラーが発生するファイルの所有権を取得する。
* **組み合わせ**: `/F <path> /A /R`
* **解説**: ユーザープロファイルには、他の管理者アカウントから直接アクセスできないファイルが含まれていることがあります。`/A` (Administrators) と `/R` (再帰) を使ってディレクトリ全体の所有権を取得することで、`robocopy` などでのコピーが可能になります。
* **例**:

    ```cmd
    REM C:\Users\OldUser プロファイルの所有権を Administrators に取得
    takeown /F C:\Users\OldUser /A /R
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによってロックされたり、アクセス権が変更されたりしたシステムファイルやログファイルにアクセスする。
* **組み合わせ**: `/F <file>`
* **解説**: インシデント対応中に、マルウェアが自身の削除を防ぐために変更したファイルの所有権を `takeown` で奪い返します。これにより、ファイルの隔離 (`move`) や削除 (`del`)、あるいは内容の分析が可能になります。
* **例**:

    ```cmd
    REM マルウェアによって改ざんされた hosts ファイルの所有権を取得
    takeown /F C:\Windows\System32\drivers\etc\hosts
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** や**防御回避 (Defense Evasion)** のため、保護されたシステムファイルを改ざんする。
* **組み合わせ**: `takeown` → `icacls` → `copy`/`move`
* **解説**: 攻撃者は管理者権限を奪取した後、`sfc` (システムファイルチェッカー) によって保護されているファイルを置き換えるために、まず `takeown` でファイルの所有権を奪います。次に `icacls` で自身に書き込み権限を与え、`sethc.exe` (固定キー機能) のようなアクセシビリティツールを `cmd.exe` のコピーに置き換えて、ログイン画面からバックドアを起動できるようにします。
* **例**:

    ```cmd
    REM sethc.exe の所有権を奪い、アクセス権を変更してバックドアを仕掛ける
    takeown /F C:\Windows\System32\sethc.exe
    icacls C:\Windows\System32\sethc.exe /grant Administrators:F
    copy C:\Windows\System32\cmd.exe C:\Windows\System32\sethc.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `ERROR: The current logged on user does not have administrative privileges.`
    * **考えられる原因**: `takeown` を実行するには管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `takeown` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `takeown` は、システムの保護メカニズムをバイパスするための強力なツールであり、そのものが悪用対象となります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`takeown` は保護されたシステムファイルを改ざんし、バックドアを設置するための攻撃チェーンの重要な一部です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **権限昇格 (Privilege Escalation)**, **防御回避 (Defense Evasion)**
* **解説**: `takeown.exe` は、攻撃者がシステムの保護を無力化し、ファイルを改ざんするために利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底する。
* **Detection (検知)**: `Sysmon` やファイル整合性監視 (FIM) ツールで、重要なシステムファイル (`sethc.exe`, `utilman.exe`など) の所有権の変更を監視する。`takeown.exe` の実行、特にシステムディレクトリ内のファイルに対する実行をプロセス監査ログで監視する。

## 注意点・補足

* **TrustedInstaller**: Windowsの多くのシステムファイルは、デフォルトで `TrustedInstaller` という特別なアカウントによって所有されています。これらのファイルの所有権を `takeown` で変更すると、システムの安定性に影響を与える可能性があるため、操作は慎重に行う必要があります。
