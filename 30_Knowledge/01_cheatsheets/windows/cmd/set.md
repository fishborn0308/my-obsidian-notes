# `set` - 環境変数を表示・設定・削除する

## 概要

`set` コマンドは、Windowsのコマンドプロンプト環境で**環境変数**を操作するための中心的なコマンドです。引数なしで実行すると、現在の全ての環境変数とその値を表示します。バッチファイル内では、ファイルパスや設定値などを一時的に保存する**変数**を定義したり、`/P` スイッチでユーザーからの入力を受け取ったり、`/A` スイッチで簡単な算術演算を行ったりとスクリプトの動作を動的に制御するために不可欠な役割を果たします。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **PowerShell** | PowerShellでは `$variable = "value"` の形式で変数を扱います。変数は厳密に型付けでき、文字列だけでなく複雑なオブジェクトも格納できます。`set` は文字列ベースの単純な操作に限定されます。 | `$` (PowerShell) |

## よく連携されるコマンド

`set` で定義した変数は、`echo` で表示したり、`if` で条件分岐に使ったり、`for` ループの制御に使ったりとあらゆるバッチ処理コマンドと連携します。

### シナリオ例: ファイル名に日付スタンプを付与する (インフラ構築・運用視点)

* **目的**: バックアップファイル名などに、実行時の日付を含んだ一意の名前を付ける。
* **連携コマンド**: `echo`, `copy`
* **解説**: `%DATE%` という組み込み変数から日付を取得し、`set` の置換機能を使ってファイル名に使用できない `/` を取り除き、新しい変数に格納します。
* **コマンド例**:

    ```cmd
    REM %DATE% は "YYYY/MM/DD" 形式なので、"/" を空文字に置換
    set TODAY=%DATE:/=%

    REM "backup_YYYYMMDD.zip" というファイル名を作成
    set BACKUP_FILENAME=backup_%TODAY%.zip

    echo Creating backup file: %BACKUP_FILENAME%
    copy data.db %BACKUP_FILENAME%
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | 現在定義されている全ての環境変数を表示します。 |
| ⭐ `<variable>=[string]` | 指定した変数に文字列を代入します。`string` を空にすると、変数を削除します。 |
| **機能拡張** | |
| ⭐ `/A` | 式を評価する**算術 (Arithmetic)** 計算を行います。 |
| ⭐ `/P` | ユーザーからの入力を受け取り、変数に格納する**プロンプト (Prompt)** を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ユーザーに対話的に設定値を入力させ、スクリプトの動作を制御する。
* **組み合わせ**: `/P`
* **解説**: `/P` を使ってユーザーからの入力を変数に格納し、その後の `if` 文などで処理を分岐させます。
* **例**:

    ```cmd
    @echo off
    set /p AREYOUSURE="Are you sure you want to proceed? (Y/N): "
    if /I "%AREYOUSURE%" NEQ "Y" (
        echo Operation cancelled.
        goto :EOF
    )
    echo Proceeding with operation...
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンス**。侵害されたホストの環境変数一覧を調査し、不審な設定がないか確認する。
* **組み合わせ**: `set > variables.txt`
* **解説**: `set` を実行して出力をファイルに保存し、`Path` 環境変数に見慣れないディレクトリが追加されていないか、`PSModulePath` が改ざんされていないか、あるいは攻撃者が活動の目印として設定した不審な変数がないかなどを調査します。
* **例**:

    ```cmd
    REM 環境変数の一覧を取得してファイルに保存
    set > C:\Forensics\environment_vars.txt
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)** と**防御回避 (Defense Evasion)**。
* **組み合わせ**: `set`
* **解説**:
  * **偵察**: 攻撃者は侵入後、`set` を実行して `LOGONSERVER` や `USERDOMAIN` などの変数からドメインコントローラーの情報を得たり、`Path` からインストールされているソフトウェアを推測したりします。
  * **防御回避**: `set` を使って悪意のあるスクリプトの一部を環境変数に隠し、ファイルベースのスキャンを回避しようとすることがあります (例: `set payload=iex...` → `powershell !payload!`)。
* **例**:

    ```cmd
    REM ログオンサーバーを確認してドメインコントローラーを特定
    set LOGONSERVER
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **現象**: `for` ループ内で変数を変更しても、ループ内でその変更が反映されない。
    * **考えられる原因**: CMDの通常の変数展開 (`%VAR%`) は、コマンドブロックが解釈される時点で一度に行われるためです。
    * **解決策**: スクリプトの冒頭で `setlocal enabledelayedexpansion` を宣言し、ループ内で変更される変数を `%VAR%` の代わりに `!VAR!` で参照します（**遅延環境変数展開**）。

## 環境変数と設定ファイル

* `set` は環境変数そのものを操作するためのコマンドです。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `set` 自体ではなく、環境変数の内容が**情報漏洩**に繋がったり、**不正なパス**が設定されたりすることが問題となります。
* **悪用シナリオ**:
  * **Path Hijacking**: 攻撃者が `Path` の先頭に `C:\Temp` などを追加し、そこに `powershell.exe` のような正規のツールと同名の悪意のあるファイルを置くことで、コマンド実行時にそちらが優先的に呼び出されるように仕向けます。
  * **情報窃取**: 環境変数にAPIキーやパスワードが平文で設定されている場合、`set` コマンド一発でそれらが窃取されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **防御回避 (Defense Evasion)**
* **解説**: `set` は `cmd.exe` の内部コマンドであり、攻撃者がシステムの構成情報を収集したり、悪意のあるコードを隠蔽したりするために利用する、基本的な環境寄生型の手法です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 環境変数に機密情報を平文で保存しない。システムの `Path` 変数への書き込み権限を厳格に管理する。
* **Detection (検知)**: 環境変数の変更、特に `Path` への変更を監視する (`Sysmon` イベントID 13: RegistryEvent で `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment` などを監視)。`set.exe` (実際には`cmd.exe`) の実行をプロセス監査ログで監視する。

## 注意点・補足

* **スコープ**: バッチファイル内で `set` を使って変更した変数は、そのコマンドプロンプトのセッション内でのみ有効です。永続的に変更するには、システムのプロパティ (`sysdm.cpl`) から設定するか、`setx` コマンドを使用します。
