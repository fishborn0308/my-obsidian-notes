# `hostname` - コンピュータのホスト名を表示する

## 概要

`hostname` コマンドは、現在のコンピュータのホスト名を標準出力に表示する、非常にシンプルで基本的なコマンドです。「今、自分はどのサーバーに接続しているのか？」を素早く確認するために使用されます。スクリプト内で、処理を実行しているサーバー名をログに記録したり、ファイル名に含めたりする際にも利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`%COMPUTERNAME%`** | 環境変数 `%COMPUTERNAME%` もホスト名を保持しています。`hostname` はコマンドであり、`%COMPUTERNAME%` は変数の参照です。結果は同じですが、`hostname` はリモート実行などでより明確です。 | - |
| **PowerShell** | `Get-ComputerInfo` や `$env:COMPUTERNAME` を使います。PowerShellでは、これらの情報をオブジェクトの一部として取得できます。 | `Get-ComputerInfo` (PowerShell) |

## よく連携されるコマンド

`hostname` は、その出力を他のコマンドの引数やファイル名の一部として利用するために、リダイレクションや環境変数と連携します。

### シナリオ例: ホスト名を含んだログファイルを作成する (インフラ構築・運用視点)

* **目的**: どのサーバーで生成されたログか一目で分かるように、ファイル名にホスト名を含める。
* **連携コマンド**: `echo`, リダイレクション (`>`)
* **解説**: `hostname` コマンドの実行結果を、リダイレクションを使ってファイルに書き出したり、バッチファイル内で変数に格納して利用したりします。
* **コマンド例**:

    ```cmd
    REM 現在のホスト名を server_name.txt として保存
    hostname > server_name.txt

    REM バッチファイル内でファイル名として利用する
    SET LOGFILE=C:\Logs\applog_%COMPUTERNAME%.log
    echo Process started at %TIME% >> %LOGFILE%
    ```

## スイッチ/オプション説明

`hostname` コマンド自体には、一般的に使用されるコマンドラインスイッチはありません。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| ⭐ `(引数なし)` | 現在のコンピュータのホスト名を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複数のリモートデスクトップやSSHセッションを開いているときに、現在操作しているサーバーを正確に識別する。
* **組み合わせ**: `hostname`
* **解説**: 複数のサーバーを同時にメンテナンスする際に、意図しないサーバーで破壊的なコマンドを実行してしまうミスを防ぐための基本的な確認作業です。
* **例**:

    ```cmd
    REM 現在作業中のホスト名を確認
    hostname
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデントレスポンスの初期対応**として、調査対象のホストを正確に特定・記録する。
* **組み合わせ**: `hostname`
* **解説**: 不審なアラートを検知し、対象のサーバーに接続した後、最初に `hostname` を実行します。これにより、調査しているのが正しいサーバーであることを確認し、その後の調査記録（フォレンジックレポートなど）にホスト名を正確に記載します。
* **例**:

    ```cmd
    REM 調査対象ホストのIDを記録
    hostname
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入後の状況確認として、ホストの役割を推測する。
* **組み合わせ**: `hostname`
* **解説**: 攻撃者はシステムに侵入後、ほぼ確実に `whoami` と `hostname` を実行します。ホスト名が `PROD-DB-MASTER` や `AD-CONTROLLER-01` のようであれば、そこが非常に価値の高いターゲットであることが分かり、`DEV-TEST-INSTANCE` のようであれば、そこからさらに本番環境へ侵入するための足がかりを探します。
* **例**:

    ```cmd
    REM 侵入したホストの名前を確認
    hostname
    ```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `hostname` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: ホスト名は、攻撃者にとってそのサーバーの役割や重要度を推測するための重要な情報源です。`prod-db-master` や `dev-ci-runner` のような分かりやすい命名規則は、管理者にとっては便利ですが、攻撃者にも同様に有益な情報を与えてしまいます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `hostname` は、攻撃者が侵入後の情報収集フェーズで、ほぼ間違いなく実行する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: サーバーの役割が直接的に推測できないような、無機質な命名規則（例: `SRV-TKY-001`）を採用することを検討する。
* **Detection (検知)**: `hostname.exe` の実行は非常に一般的であるため、単体での検知は困難です。しかし、不審なプロセス（例: Webサーバーのワーカープロセス）から `hostname.exe` が実行されるといった、通常ではありえないコンテキストでの実行をEDRなどで監視することは有効です。

## 注意点・補足

* **ホスト名の変更**: `hostname` コマンドでホスト名を変更することはできません。ホスト名を変更するには、システムのプロパティ (`sysdm.cpl`) のGUIを使用するか、管理者権限で `netdom` コマンドやPowerShellの `Rename-Computer` コマンドレットを使用する必要があります。
