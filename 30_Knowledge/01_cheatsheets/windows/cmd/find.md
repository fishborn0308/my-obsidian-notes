# `find` - ファイル内の文字列を検索する

## 概要

`find` コマンドは、1つまたは複数のファイルの中から、指定したテキスト文字列を含む行を検索し、その行を表示するためのコマンドです。Linuxの `grep` に似ていますが、機能はより限定的です。ログファイルや設定ファイルから特定の情報を素早く見つけ出すのに利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`findstr`** | `find` の高機能版。正規表現、複数ファイル検索、ディレクトリ再帰検索など、`find` にはない多くの機能を持ちます。より複雑な検索には `findstr` が推奨されます。 | `findstr` |
| **PowerShell** | `Select-String` (エイリアス: `sls`) は、オブジェクト指向のパイプラインと連携でき、正規表現も強力で、より柔軟なテキスト検索が可能です。 | `Select-String` |
| **`type`** | ファイルの**内容全体**を表示します。`find` はファイルの中から**特定の行のみ**を抽出します。 | - |

## よく連携されるコマンド

`find` は、他のコマンドの出力をパイプ (`|`) で受け取り、その中から特定のキーワードを含む行だけをフィルタリングするためによく使われます。

### シナリオ例: 実行中のプロセスから特定のものを絞り込む (インフラ構築・運用視点)

* **目的**: 現在実行中の全プロセスの中から、"explorer.exe" という名前のプロセスだけを表示する。
* **連携コマンド**: `tasklist`, パイプ (`|`)
* **解説**: `tasklist` で全プロセスのリストを出力し、その結果をパイプで `find` に渡して "explorer.exe" という文字列を含む行だけをフィルタリングします。
* **コマンド例**:

    ```cmd
    REM tasklistの出力から "explorer.exe" プロセスを検索
    tasklist | find /I "explorer.exe"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **検索動作** | |
| ⭐ `"<string>"` | 検索する文字列を指定します。必ずダブルクォーテーションで囲みます。 |
| ⭐ `/V` | 指定した文字列を**含まない**行を表示します (Invert)。 |
| ⭐ `/C` | 一致した行の**総数 (Count)** のみを表示します。 |
| ⭐ `/N` | 一致した行の**行番号 (Number)** を付けて表示します。 |
| ⭐ `/I` | 検索時に**大文字と小文字を区別しません** (Ignore case)。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルから特定のキーを持つ設定行を見つけ出す。
* **組み合わせ**: `/I`
* **解説**: `/I` を使って大文字・小文字を区別せずに検索することで、`LogLevel` と `loglevel` の両方にマッチさせることができます。
* **例**:

    ```cmd
    REM httpd.conf から "Listen" ディレクティブを探す
    find /I "Listen" "C:\Apache24\conf\httpd.conf"
    ```

### 2. ブルーチーム視点

* **タスク**: ログファイルに特定のエラーコードが出現した回数をカウントする。
* **組み合わせ**: `/C`
* **解説**: 大量のログファイルに対して、まず `/C` でエラーの発生回数を確認し、回数が多ければ、次に `/C` を外して具体的なエラー内容を調査するという段階的な分析を行います。
* **例**:

    ```cmd
    REM アプリケーションログに "Error Code 500" が何回出現したかカウント
    find /C "Error Code 500" "C:\App\Logs\app.log"
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。設定ファイルの中から "password" という文字列を含む行を探し出す。
* **組み合わせ**: `type <file> | find /I "password"`
* **解説**: 攻撃者はシステム侵入後、`find` や `findstr` を駆使して、設定ファイル (`.ini`, `.conf`, `.xml` など) やバックアップファイルにハードコードされたパスワードや接続文字列を探します。
* **例**:

    ```cmd
    REM Webサーバーの設定ファイルからパスワードを探す
    type C:\App\web.config | find /I "password"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'find'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `FIND: Parameter format not correct`
    * **考えられる原因**: 検索文字列がダブルクォーテーションで囲まれていないか、スイッチの使い方が間違っています。
    * **解決策**: `find "search string" file.txt` のように、検索文字列を必ず `"` で囲んでください。

2. **エラーメッセージ例 2**: `File not found - <ファイル名>`
    * **考えられる原因**: 指定したファイルが存在しないか、パスが間違っています。
    * **解決策**: `dir` コマンドでファイルが存在することと、パスが正しいことを確認してください。

## 環境変数と設定ファイル

* `find` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `find` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`find` は攻撃者がシステム内に平文で不用意に保存されたパスワードやAPIキーなどの機密情報を発見するための強力なツールです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `find` はWindowsの基本コマンドであり、その実行自体は警告を発しません。攻撃者はこのコマンドを悪用して、システム内の機密情報を探し出します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードやAPIキーを設定ファイルやソースコードにハードコードしない。機密情報は暗号化して保存するか、専用のシークレット管理ツールを使用する。
* **Detection (検知)**: `find.exe` の実行、特に機密情報を含む可能性のあるファイル（`.config`, `.xml`, `.ini`など）に対するスキャンを、プロセス監査ログやEDRで監視する。

## 注意点・補足

* **正規表現の非対応**: `find` コマンドは正規表現をサポートしていません。正規表現を使ったより複雑な検索が必要な場合は、`findstr` コマンドを使用してください。
* **複数ファイルの検索**: `find "string" file1.txt file2.txt` のように、複数のファイルを一度に検索対象として指定できます。
