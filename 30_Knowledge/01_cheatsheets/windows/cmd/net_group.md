# `net group` - ドメインのグローバルグループを管理する

## 概要

`net group` コマンドは、Active Directory ドメイン環境において、サーバー（通常はドメインコントローラー）上のグローバルグループを追加、表示、変更するために使用されます。このコマンドを使って、ドメイン全体のユーザーグループを作成したり、グループにドメインユーザーを追加・削除したりできます。

**注**: このコマンドはドメイン環境でのみ機能します。スタンドアロンのサーバーやクライアントPCのローカルグループを管理するには `net localgroup` を使用します。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`net localgroup`** | **ローカルコンピュータ**上のグループを管理します。`net group` は**ドメイン**全体のグローバルグループを管理します。 | - |
| **PowerShell** | `Get-ADGroup`, `New-ADGroup`, `Add-ADGroupMember` などのActive Directoryモジュールのコマンドレットは、グループ管理をオブジェクト指向で行え、より強力でスクリプト化に適しています。 | `Active Directory Module` |

## よく連携されるコマンド

`net group` は、ドメイン内の権限構造を調査するために、他の `net` コマンドと連携して偵察活動で多用されます。

### シナリオ例: ドメイン管理者をリストアップする (レッドチーム視点)

* **目的**: ドメイン内で最も高い権限を持つ「Domain Admins」グループに所属するユーザーを特定する。
* **連携コマンド**: (なし)
* **解説**: 攻撃者はシステム侵入後、権限昇格の最終目標としてドメイン管理者を特定するために、まずこのコマンドを実行します。
* **コマンド例**:

    ```cmd
    REM "Domain Admins" グループのメンバーを一覧表示
    net group "Domain Admins" /domain
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `(引数なし)` | サーバー上の全てのグローバルグループ名を表示します。 |
| ⭐ `<groupname>` | 指定したグループのメンバーを一覧表示します。 |
| `<groupname> /add` | 新しいグループを作成します。 |
| `<groupname> /delete` | グループを削除します。 |
| **メンバー管理** | |
| `<groupname> <username> [...] /add` | 1人または複数のユーザーをグループに追加します。 |
| `<groupname> <username> [...] /delete` | 1人または複数のユーザーをグループから削除します。 |
| **その他** | |
| `/comment:"<text>"` | グループにコメントを追加します。 |
| ⭐ `/domain` | ドメインコントローラー上でコマンドを実行します。ドメインメンバーから実行する際に必要です。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しいドメイングループを作成し、ユーザーを追加する。
* **組み合わせ**: `/add`
* **解説**: ファイル共有などのアクセス権を管理するために、特定の役割（例: `Sales-Users`）を持つグループを作成し、メンバーを追加します。
* **例**:

    ```cmd
    REM "Sales-Users" グループをドメインに作成
    net group "Sales-Users" /add /comment:"Sales Department Users" /domain

    REM ユーザー 'tanaka' を "Sales-Users" グループに追加
    net group "Sales-Users" tanaka /add /domain
    ```

### 2. ブルーチーム視点

* **タスク**: 特権グループのメンバーシップを定期的に監査する。
* **組み合わせ**: `net group "<privileged_group>"`
* **解説**: **アクセス権管理の基本**。`Domain Admins`, `Enterprise Admins`, `Schema Admins` などの強力な権限を持つグループのメンバーを定期的に確認し、意図しないユーザーが所属していないかを監査します。
* **例**:

    ```cmd
    REM ドメインの管理者グループのメンバーを確認
    net group "Domain Admins" /domain
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。ドメインの権限構造をマッピングする。
* **組み合わせ**: `net group`
* **解説**: 攻撃者は侵入後、`net group` でグループ一覧を取得し、`Domain Admins`, `Administrators`, `SQLAdmins` のような興味深い名前のグループを探します。その後、各グループのメンバーを `net group "<groupname>"` で特定し、次の攻撃対象となる特権ユーザーをリストアップします。
* **例**:

    ```cmd
    REM ドメイン内の全てのグループ名をリストアップ
    net group /domain

    REM 特定のグループのメンバーをリストアップ
    net group "Enterprise Admins" /domain
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'net'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: グループの作成やメンバーの変更には、ドメインの管理者権限が必要です。
    * **解決策**: ドメイン管理者アカウントでコマンドプロンプトを実行してください。

2. **エラーメッセージ例 1**: `The group name could not be found.`
    * **考えられる原因**: 指定したグループが存在しないか、ドメインメンバーから実行している場合に `/domain` スイッチを付け忘れています。
    * **解決策**: グループ名を確認し、必要であれば `/domain` スイッチを付けてください。

## 環境変数と設定ファイル

* `net group` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `net group` 自体ではなく、**不適切なグループメンバーシップの管理**がセキュリティリスクとなります。
* **悪用シナリオ**: 攻撃者は、権限昇格に成功すると、自身が制御するアカウントを `Domain Admins` グループに追加して永続化を図ります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `net group` は、攻撃者がActive Directory環境の権限構造を理解するために、侵入後の初期段階で必ずと言っていいほど使用する環境寄生型の偵察ツールです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、特権グループへの所属は必要最小限のユーザーに限定する。
* **Detection (検知)**: Active Directoryの監査を有効にし、**イベントID 4728 (メンバーがセキュリティ対応グローバルグループに追加されました)** や **イベントID 4732 (メンバーがセキュリティ対応ローカルグループに追加されました)** などのグループメンバーシップ変更イベントを厳重に監視する。

## 注意点・補足

* **ドメインコントローラー**: `net group` コマンドによる変更は、ドメインコントローラー上で行われます。変更がドメイン全体に複製されるまでには、若干の時間がかかる場合があります。
