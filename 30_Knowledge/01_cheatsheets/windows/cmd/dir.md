# `dir` - ファイルとディレクトリの一覧を表示する

## 概要

`dir` コマンドは、指定したディレクトリ内に存在するファイルとサブディレクトリの一覧を表示するための、最も基本的で頻繁に使用されるコマンドです。ファイル名だけでなく、最終更新日時、サイズ、属性などの情報も表示でき、オプションを組み合わせることで、表示形式や順序、対象とするファイルを柔軟に制御できます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`tree`** | ディレクトリ構造を**ツリー形式**で視覚的に分かりやすく表示します。`dir /S` は**リスト形式**で再帰的に表示します。 | `tree` (階層表示) |
| **PowerShell** | `Get-ChildItem` (エイリアス: `gci`, `ls`, `dir`) は、ファイルやディレクトリをオブジェクトとして取得するため、パイプラインで `Sort-Object` や `Where-Object` に渡し、より高度なフィルタリングやソートが可能です。 | `Get-ChildItem` (PowerShell) |

## よく連携されるコマンド

`dir` コマンドの出力は、パイプ (`|`) を通じて他のコマンドに渡され、結果をフィルタリングしたり、処理したりするのに使われます。

### シナリオ例: 特定のサービスに関連するファイルを探す (インフラ構築・運用視点)

* **目的**: システムディレクトリ内で、名前に "apache" を含む設定ファイルやログファイルを再帰的に検索する。
* **連携コマンド**: `findstr`
* **解説**: `dir /S` で指定したディレクトリ以下を再帰的にリストアップし、その結果をパイプで `findstr` に渡して、キーワードに一致する行だけを絞り込みます。
* **コマンド例**:

    ```cmd
    REM C:\Program Files 以下で "apache" を含むファイル/ディレクトリを検索
    dir "C:\Program Files" /S | findstr /I "apache"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `/P` | 1画面ごとに表示を一時停止します (ページャ機能)。 |
| `/W` | ワイド形式で、ファイル名のみを複数列で表示します。 |
| ⭐ `/B` | ファイル名のみを、ヘッダーやサマリーなしで表示します (Bare format)。スクリプトでの利用に便利です。 |
| **フィルタリングと順序** | |
| ⭐ `/A[[:]attributes]` | 特定の属性を持つファイルのみを表示します。（例: `/A:H` は隠しファイル, `/A:-D` はディレクトリ以外） |
| ⭐ `/O[[:]sortorder]` | 表示順序を指定します。（例: `/O:N` 名前順, `/O:S` サイズ順, `/O:D` 日付順, `/O:G` ディレクトリ先） |
| **操作対象** | |
| ⭐ `/S` | 指定したディレクトリとその全ての**サブディレクトリ**にあるファイルを再帰的に表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディレクトリ内のファイルを、更新日時が新しい順に詳細表示する。
* **組み合わせ**: `/O:-D`
* **解説**: `/O:D` が日付の古い順なので、接頭辞 `-` を付けて逆順（新しい順）にソートします。最近変更された設定ファイルやログファイルを確認するのに便利です。
* **例**:

    ```cmd
    REM Windows Updateのログディレクトリを新しい順に表示
    dir %windir%\Logs\CBS /O:-D
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによって隠されたファイルや、システム属性が付与されたファイルを発見する。
* **組み合わせ**: `/A:HS`
* **解説**: **フォレンジック調査の基本**。`/A` スイッチで隠しファイル (`H`) とシステムファイル (`S`) を明示的に表示させます。攻撃者はしばしばマルウェアをこれらの属性で隠蔽するため、通常の `dir` では見逃してしまいます。
* **例**:

    ```cmd
    REM C:\Users\Public ディレクトリ以下の隠しファイルとシステムファイルを再帰的に表示
    dir C:\Users\Public /A:HS /S
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したシステムで、機密情報が含まれていそうなファイルを探す。
* **組み合わせ**: `/S /B`
* **解説**: 攻撃者はシステム侵入後、`dir /S /B *pass*` や `dir /S /B *.config` のように、ファイル名に特定のキーワードを含むファイルをシステム全体から探し出し、情報窃取の足がかりとします。`/B` を使うことで、ノイズの少ないリストを得られます。
* **例**:

    ```cmd
    REM 全ユーザープロファイルから、名前に "config" を含むファイルを検索
    dir C:\Users\*config* /S /B
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'dir'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `File Not Found`
    * **考えられる原因**: 指定したパスにファイルやディレクトリが存在しないか、ワイルドカードに一致するファイルがありません。
    * **解決策**: `cd` コマンドでパスが正しいか確認してください。ファイル名やパターンが正しいか再確認します。

## 環境変数と設定ファイル

* **`DIRCMD`**: `dir` コマンドのデフォルトスイッチをこの環境変数で設定できます。例えば `set DIRCMD=/A` と設定すると、`dir` を実行するたびに常に `/A` が付与された状態で動作します。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `dir` 自体は読み取り専用のコマンドですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: 攻撃者は `dir` を駆使してシステム内を探索し、ファイルシステムの構造、存在するアプリケーション、設定ファイル、ユーザーデータなどを把握します。これは、さらなる攻撃計画を立てるための重要な偵察活動です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `dir` は、攻撃者が侵入後の情報収集フェーズで、ほぼ間違いなく実行する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に基づき、ファイルやディレクトリに適切なACL（アクセス制御リスト）を設定し、ユーザーに不要な読み取り権限を与えない。
* **Detection (検知)**: `auditpol` でファイルシステムアクセスの監査を有効にし、重要なディレクトリ (例: `C:\Windows\System32`) やユーザープロファイルに対する予期せぬ `dir` (アクセス) がないかイベントログで監視する。

## 注意点・補足

* **ファイル名の表示**: デフォルトでは `8.3形式` の短いファイル名は表示されません。表示するには `/X` スイッチを使います。
* **所有者の表示**: ファイルの所有者を表示するには `/Q` スイッチを使います。
