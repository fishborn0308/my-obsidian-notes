# `systeminfo` - システムの構成情報を表示する

## 概要

`systeminfo` コマンドは、ローカルまたはリモートのコンピュータに関する詳細な構成情報を表示するためのユーティリティです。このコマンドの出力には、OSの名称とバージョン、インストール日、システム起動時間、CPU情報、物理および仮想メモリの搭載量と空き容量、ネットワークカードの詳細、そして適用済みのWindows Update修正プログラム（Hotfix）の一覧などが含まれます。システムのインベントリ収集や、トラブルシューティングの初期調査に不可欠なツールです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`wmic`** | `wmic` はWMI (Windows Management Instrumentation) を使って、より広範で詳細なハードウェアやソフトウェアの情報を取得できます。`systeminfo` は主要な情報を要約して表示します。 | `wmic` (詳細情報) |
| **PowerShell** | `Get-ComputerInfo` コマンドレットは、`systeminfo` と同等以上の情報をオブジェクトとして取得でき、必要な情報を選択したり、フィルタリングしたりするのが容易です。 | `Get-ComputerInfo` |

## よく連携されるコマンド

`systeminfo` の出力は多岐にわたるため、`findstr` と連携して特定の情報（例: Hotfix）を絞り込むのが一般的です。

### シナリオ例: 特定のセキュリティパッチが適用されているか確認する (ブルーチーム視点)

* **目的**: 脆弱性情報で公開された特定のKB番号（例: `KB5005565`）の修正プログラムが、システムに適用済みであるかを確認する。
* **連携コマンド**: `findstr`
* **解説**: `systeminfo` の出力には適用済みのHotfix一覧が含まれています。その出力をパイプで `findstr` に渡し、目的のKB番号でフィルタリングします。
* **コマンド例**:

    ```cmd
    REM システム情報から 'KB5005565' の文字列を検索
    systeminfo | findstr "KB5005565"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **接続先指定** | |
| `/S <system>` | 接続するリモートコンピュータを指定します。 |
| `/U <user>` | 実行するユーザーコンテキストを指定します。 |
| `/P <password>` | 指定したユーザーのパスワードを指定します。 |
| **出力形式** | |
| ⭐ `/FO <format>` | 出力形式を指定します。`TABLE` (デフォルト), `LIST`, `CSV` があります。 |
| ⭐ `/NH` | (No Header) `CSV` 形式の出力で、ヘッダー行を表示しません。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複数のサーバーのインベントリ情報（OSバージョン、メモリ容量など）をCSV形式で収集する。
* **組み合わせ**: `/S <system> /FO CSV`
* **解説**: `/S` でリモートサーバーを指定し、`/FO CSV` で出力をCSV形式にすることで、複数のサーバー情報を簡単に収集し、Excelなどで一覧表を作成できます。
* **例**:

    ```cmd
    REM リモートサーバー 'SRV-01' の情報を inventory.csv に追記
    systeminfo /S SRV-01 /FO CSV >> inventory.csv
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデントレスポンスの初期対応**として、侵害されたホストの基本情報を迅速に収集する。
* **組み合わせ**: `systeminfo`
* **解説**: 調査対象のホストで `systeminfo` を実行し、OSのバージョン、インストール日、起動時間などを記録します。特に「System Boot Time」は、攻撃がいつ発生したかを推測する上で重要なタイムライン情報となります。「Hotfix(s)」セクションは、システムが脆弱な状態であったかどうかを判断する手がかりになります。
* **例**:

    ```cmd
    REM 調査対象ホストのシステム情報をファイルに保存
    systeminfo > C:\Forensics\system_info.txt
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストのOSバージョンとパッチレベルを特定し、既知の脆弱性を探す。
* **組み合わせ**: `systeminfo`
* **解説**: 攻撃者は侵入後、ほぼ確実に `systeminfo` を実行します。OSのビルド番号や適用されていないHotfixの情報を基に、権限昇格に利用できるカーネルエクスプロイトなどの脆弱性を特定します。
* **例**:

    ```cmd
    REM OSバージョンとHotfix情報を偵察
    systeminfo
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'systeminfo'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `ERROR: The RPC server is unavailable.`
    * **考えられる原因**: リモートコンピュータを指定して実行 (`/S`) しましたが、ファイアウォールでブロックされているか、必要なサービスがリモート側で実行されていません。
    * **解決策**: リモートコンピュータのWindows Firewallで「リモート管理」が許可されていること、および "Remote Procedure Call (RPC)" サービスが実行中であることを確認してください。

## 環境変数と設定ファイル

* `systeminfo` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `systeminfo` 自体は読み取り専用のツールですが、その出力が**重大な情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、OSのバージョンやパッチレベルは、攻撃者が権限昇格のための脆弱性を探す上で最も重要な情報の1つです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `systeminfo.exe` は、攻撃者が侵入後の情報収集フェーズで、システムの脆弱性を評価するために必ずと言っていいほど使用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 定期的にセキュリティパッチを適用し、システムを最新の状態に保つことが最も重要です。
* **Detection (検知)**: `systeminfo.exe` の実行、特に不審なプロセス（例: Webサーバーのワーカープロセス）からの実行をプロセス監査ログ（イベントID 4688）やEDRで監視する。

## 注意点・補足

* **情報量**: `systeminfo` はHotfixのリストを含め、大量の情報を表示します。`findstr` を使って必要な情報だけを抜き出すか、PowerShellの `Get-ComputerInfo` を使って特定のプロパティを選択するのが効率的です。
