# `type` - テキストファイルの内容を表示する

## 概要

`type` コマンドは、指定したテキストファイルの内容をコマンドプロンプトの画面に表示するための基本的なコマンドです。Linuxの `cat` コマンドに相当する機能を持っています。主に、設定ファイルやログファイル、バッチファイルの中身を素早く確認するために使用されます。バイナリファイルに対して使用すると、文字化けしたデータが表示されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`more`** | `more` はファイルの内容を1画面ずつページ単位で表示します。長いファイルを読むのに適しています。`type` はファイル全体を一度に表示しようとします。 | `more` (長いファイル) |
| **`echo`** | `echo` は引数で与えられた**文字列**を表示します。`type` は**既存のファイル**の内容を表示します。 | - |
| **PowerShell** | `Get-Content` (エイリアス: `cat`, `gc`, `type`) は、ファイルの内容を1行ずつオブジェクトとしてパイプラインに渡せるため、後続のコマンドレットでの処理が非常に強力です。 | `Get-Content` (PowerShell) |

## よく連携されるコマンド

`type` コマンドは、その出力をパイプ (`|`) で `find` や `findstr` に渡し、ファイルの中から特定の文字列を含む行だけをフィルタリングするためによく使われます。

### シナリオ例: ログファイルからエラー行を抽出する (インフラ構築・運用視点)

* **目的**: アプリケーションのログファイルから "ERROR" という文字列が含まれる行だけを抜き出して表示する。
* **連携コマンド**: `findstr`, パイプ (`|`)
* **解説**: `type` でログファイルの内容を全て出力し、その結果をパイプで `findstr` に渡して "ERROR" を含む行だけをフィルタリングします。
* **コマンド例**:

    ```cmd
    REM app.log の内容から "ERROR" を含む行を検索
    type C:\App\Logs\app.log | findstr /I "ERROR"
    ```

## スイッチ/オプション説明

`type` コマンドには、コマンドラインスイッチやオプションはありません。

| スイッチ/オプション | 説明 |
| :--- | :--- |
| ⭐ `<filename(s)>` | 内容を表示したい1つまたは複数のファイル名を指定します。ワイルドカード (`*`, `?`) が使用できます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイル (`.ini`, `.conf` など) の内容を素早く確認する。
* **組み合わせ**: `type <config_file>`
* **解説**: GUIのテキストエディタを開くよりも手軽に、コマンドライン上で設定ファイルの中身を確認できます。
* **例**:

    ```cmd
    REM hostsファイルの内容を確認
    type C:\Windows\System32\drivers\etc\hosts
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンス**。マルウェアの疑いがあるスクリプトファイル（`.bat`, `.vbs`など）の中身を安全に確認する。
* **組み合わせ**: `type <suspicious_script>`
* **解説**: スクリプトファイルを直接ダブルクリックで実行するとマルウェアが活動を開始してしまうため、`type` を使ってそのソースコードを画面に表示し、どのような処理が記述されているかを安全に調査します。
* **例**:

    ```cmd
    REM 不審なバッチファイルの中身を表示して調査
    type C:\Users\Public\update.bat
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。設定ファイルや保存されたスクリプトから、パスワードや接続文字列などの機密情報を探す。
* **組み合わせ**: `type <file> | findstr /I "password"`
* **解説**: 攻撃者は侵入後、`type` と `findstr` を組み合わせて、`.config`, `.xml`, `.ini` などのファイルの中からハードコードされた認証情報を探し出します。
* **例**:

    ```cmd
    REM Webアプリケーションの設定ファイルから 'password' という文字列を探す
    type C:\inetpub\wwwroot\webapp\web.config | findstr /I "password"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'type'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `The system cannot find the file specified.`
    * **考えられる原因**: 指定したファイルが存在しないか、パスが間違っています。
    * **解決策**: `dir` コマンドでファイルが存在することと、パスが正しいことを確認してください。パスにスペースが含まれる場合は、`"C:\Program Files\..."` のようにダブルクォートで囲みます。

## 環境変数と設定ファイル

* `type` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `type` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`type` は攻撃者がシステム内に平文で不用意に保存されたパスワードやAPIキーなどの機密情報を発見するための基本的なツールです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Re-connaissance)**
* **解説**: `type` は `cmd.exe` の内部コマンドであり、攻撃者が侵入後の情報収集フェーズで、ファイルの内容を調査するために利用する最も基本的な環境寄生型の手法です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードやAPIキーなどの機密情報を設定ファイルに平文で保存しない。可能であれば暗号化するか、専用のシークレット管理ツールを使用する。
* **Detection (検知)**: ファイルアクセス監査を有効にし、重要な設定ファイルへの予期せぬ読み取りアクセスを監視する。`type` (`cmd.exe`) の実行と、その対象となるファイルをプロセス監査ログで監視する。

## 注意点・補足

* **ワイルドカード**: `type *.log` のようにワイルドカードを使うと、一致した全てのファイルの内容が連続して表示されます。
