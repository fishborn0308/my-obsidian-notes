# `rd` / `rmdir` - ディレクトリを削除する

## 概要

`rd` (Remove Directory) コマンドは、ディレクトリ（フォルダ）を削除するためのコマンドです。`rmdir` は `rd` のエイリアスであり、全く同じ機能です。デフォルトでは**空のディレクトリのみ**を削除できますが、`/S` スイッチを使うことで、ディレクトリとその中に含まれる全てのファイルおよびサブディレクトリを**再帰的に削除**できます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`del`** | **ファイル**を削除します。`rd` は**ディレクトリ**を削除します。 | - |
| **PowerShell** | `Remove-Item` (エイリアス: `rm`, `rd`, `ri`) は、ファイル、ディレクトリ、レジストリキーなど、より多くのオブジェクトを統一的に削除できる高機能なコマンドレットです。`-Recurse` と `-Force` を組み合わせることが多いです。 | `Remove-Item` (PowerShell) |

## よく連携されるコマンド

`rd` は、一時的なディレクトリを作成して処理を行い、最後にそのディレクトリをクリーンアップする、という一連のバッチ処理で `mkdir` や `move` と連携されます。

### シナリオ例: 一時作業ディレクトリのクリーンアップ (インフラ構築・運用視点)

* **目的**: バッチ処理の最後に、作業用として作成した一時ディレクトリとその中身を全て削除する。
* **連携コマンド**: `if exist`
* **解説**: スクリプトの最後に、作業用ディレクトリが存在する場合にのみ、`rd /S /Q` を使って確認プロンプトなしでディレクトリツリー全体を削除します。
* **コマンド例**:

    ```cmd
    REM スクリプトの最後で一時ディレクトリをクリーンアップ
    IF EXIST "C:\Temp\MyWorkDir" (
        echo Cleaning up temporary directory...
        rd /S /Q "C:\Temp\MyWorkDir"
    )
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `<path>` | 削除するディレクトリのパスを指定します。 |
| **動作制御** | |
| ⭐ `/S` | 指定したディレクトリに加えて、その中にある全てのファイルとサブディレクトリを削除します。ディレクトリツリーを削除する際に使用します。 |
| ⭐ `/Q` | 静観 (Quiet) モード。`/S` と共に使用し、ディレクトリツリーを削除する際の**確認プロンプトを表示しません**。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 不要になったプロジェクトのディレクトリを、中身ごと完全に削除する。
* **組み合わせ**: `/S /Q`
* **解説**: **非常に強力で破壊的な組み合わせ**。確認なしでディレクトリツリー全体を削除します。実行前には、対象のパスが正しいことを何度も確認することが不可欠です。
* **例**:

    ```cmd
    REM 古いバージョンのアプリケーションディレクトリを完全に削除
    rd /S /Q "C:\Applications\OldApp_v1"
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠保全完了後、マルウェアが作成したディレクトリをシステムから除去する。
* **組み合わせ**: `/S /Q`
* **解説**: インシデント対応の修復 (Remediation) フェーズで使用します。攻撃者が作成したディレクトリツリー（例: `C:\Users\Public\payloads`）を、関連する証拠を全て収集した後、このコマンドで完全に削除します。
* **例**:

    ```cmd
    REM 攻撃者が作成したステージングディレクトリを削除
    rd /S /Q "C:\Users\Public\Intel"
    ```

### 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)** または **破壊活動 (Sabotage)**。
* **組み合わせ**: `/S /Q`
* **解説**:
  * **痕跡消去**: 攻撃者は、自身の活動のために作成したツールやログが格納されたディレクトリを、活動終了時に `rd /S /Q` で削除し、調査を困難にします。
  * **破壊活動**: 管理者権限を奪取した後、`rd /S /Q C:\Windows\System32` のようなコマンドでシステムの重要ディレクトリを破壊し、サービス妨害 (DoS) を引き起こします。
* **例**:

    ```cmd
    REM 攻撃ツールを格納していた一時ディレクトリを痕跡消去のために削除
    rd /S /Q "%TEMP%\mytools"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'rd'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `The directory is not empty.`
    * **考えられる原因**: 中身の入ったディレクトリを、`/S` スイッチなしで削除しようとしました。
    * **解決策**: ディレクトリが空であることを確認してから再実行するか、中身ごと削除してよい場合は `rd /S <dirname>` を使用してください。

2. **エラーメッセージ例 2**: `The process cannot access the file because it is being used by another process.`
    * **考えられる原因**: 削除しようとしているディレクトリ、またはその中のファイルが別のプログラムによって開かれている（ロックされている）ため、削除できません。
    * **解決策**: Process Explorer (Sysinternals) などのツールで、どのプロセスがロックしているかを確認し、そのプロセスを終了させてから再試行します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `rd /S /Q` コマンドの**破壊力**そのものがリスクとなります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、ランサムウェアやワイパー型マルウェアが、システムの復旧を妨害したり、データを破壊したりするためにこのコマンドを悪用します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `rd` は `cmd.exe` の内部コマンドであり、攻撃者が痕跡を消したり、データを破壊したりするために利用する環境寄生型の機能です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要なディレクトリに適切なACL（アクセス制御リスト）を設定し、最小権限の原則を徹底する。定期的なバックアップを取得する。
* **Detection (検知)**: `Sysmon` やEDRツールで、重要なシステムディレクトリに対する `rd` (`cmd.exe`による削除操作) の実行を監視する。

## 注意点・補足

* **ワイルドカードの危険性**: `rd /S /Q *` のようなワイルドカードの使用は非常に危険です。カレントディレクトリを `cd` で正確に確認してから実行してください。意図しない場所で実行すると、大惨事を引き起こします。
