# `assoc` - ファイル拡張子の関連付けを表示・変更する

## 概要

`assoc` (associate) コマンドは、ファイル拡張子（例: `.txt`）と、そのファイルの種類を示す「ファイルタイプ」（例: `txtfile`）との間の関連付けを表示または変更するために使用されます。

このコマンドは、ファイルを開いたときにどのプログラムが起動するかを決定する2段階のプロセスのうち、最初のステップを担います。`assoc` で拡張子とファイルタイプをリンクし、次に関連コマンドである `ftype` でそのファイルタイプと実行コマンドをリンクします。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`ftype`** | `assoc` が拡張子と「ファイルタイプ名」を紐付けるのに対し、`ftype` はその「ファイルタイプ名」と「実行するプログラム」を紐付けます。これらは常にセットで使われます。 | - |
| **PowerShell** | `Get-Item HKCR:\.txt` のようにレジストリを直接操作することで、より詳細な関連付け情報を取得・設定できます。スクリプトによる高度な管理にはPowerShellが適しています。 | PowerShell |

## よく連携されるコマンド

`assoc` の真価は、`ftype` コマンドと連携することで発揮されます。

### シナリオ例: ファイルを開いたときの動作を理解する (インフラ構築・運用視点)

* **目的**: `.txt` ファイルをダブルクリックしたときに、なぜ `notepad.exe` が起動するのか、その仕組みを理解する。
* **連携コマンド**: `ftype`
* **解説**:
    1. `assoc` で、`.txt` 拡張子がどのファイルタイプに関連付けられているかを調べる。
    2. `ftype` で、そのファイルタイプがどの実行コマンドに紐付いているかを調べる。
* **コマンド例**:

    ```cmd
    REM 1. .txt拡張子のファイルタイプを調べる
    assoc .txt
    REM -> .txt=txtfile

    REM 2. "txtfile" というファイルタイプがどのコマンドを実行するか調べる
    ftype txtfile
    REM -> txtfile=%SystemRoot%\system32\NOTEPAD.EXE %1
    ```

    **結果**: これにより、`.txt` ファイルは `txtfile` として扱われ、`txtfile` は `notepad.exe` を起動するように設定されていることが分かります。

## 基本構文

| 構文 | 説明 |
| :--- | :--- |
| `assoc` | 全てのファイル拡張子とファイルタイプの関連付けを一覧表示します。 |
| `assoc .<ext>` | 指定した拡張子 (`.ext`) のファイルタイプを表示します。 |
| `assoc .<ext>=<filetype>` | (`管理者権限要`) 指定した拡張子を、指定したファイルタイプに関連付けます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 現在の関連付け設定を確認する。
* **組み合わせ**: `assoc`
* **解説**: システムにどのようなファイル拡張子の関連付けが存在するかを一覧で確認し、システムの現在の状態を把握します。
* **例**:

    ```cmd
    REM .log ファイルがどのファイルタイプに関連付けられているか確認
    assoc .log
    ```

### 2. ブルーチーム視点

* **タスク**: **ファイル拡張子のハイジャック**がされていないか監査する。
* **組み合わせ**: `assoc .<ext>`
* **解説**: マルウェアは、ユーザーが普段使う無害な拡張子（`.txt`, `.html`など）の関連付けを乗っ取り、ユーザーがファイルを開いたときに自身が実行されるように仕掛けることがあります。
* **シナリオ**: `assoc .txt` を実行し、結果が `txtfile` 以外の不審なファイルタイプ（例: `VBSFile`）になっていないかを確認します。もし変更されていれば、`ftype` でそのファイルタイプが何を指しているかをさらに調査します。
* **例**:

    ```cmd
    REM .html ファイルの関連付けを確認
    assoc .html
    REM -> htmlfile (正常)
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のためにファイル拡張子をハイジャックする。
* **組み合わせ**: `assoc .<ext>=<malicious_filetype>`
* **解説**: 攻撃者は、`ftype` で悪意のあるコマンド（リバースシェルなど）を実行する新しいファイルタイプを作成した後、`assoc` を使って `.txt` のような一般的な拡張子をその悪意のあるファイルタイプに関連付けます。
* **シナリオ**: ユーザーが次に `.txt` ファイルを開いたときに、メモ帳ではなく、攻撃者のペイロードが実行されるように仕掛けます。これはユーザーの通常の操作を利用した、検知されにくい永続化の手法です。
* **例**:

    ```cmd
    REM 1. 悪意のあるコマンドを実行するファイルタイプを作成
    ftype MaliciousPayload="c:\users\public\payload.exe" "%1"

    REM 2. .txt ファイルの関連付けを、作成した悪意のあるタイプに乗っ取る
    assoc .txt=MaliciousPayload
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'assoc' is not recognized as an internal or external command...` は[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `Access is denied.`
    * **考えられる原因**: ファイル拡張子の関連付けを変更するには管理者権限が必要です。一般ユーザー権限のCMDで実行しようとしました。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 関連するレジストリ

* **`HKEY_CLASSES_ROOT` (`HKCR`)**: ファイル拡張子の関連付けに関する情報は、Windowsレジストリのこのハイブに格納されています。`assoc` と `ftype` は、このレジストリキーを読み書きするためのコマンドラインインターフェイスです。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **ファイル拡張子のハイジャックによる永続化**。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`assoc` と `ftype` の組み合わせは、ユーザーの操作をトリガーとしてマルウェアを実行させるための古典的で強力な永続化手法です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **永続化 (Persistence)**, **防御回避 (Defense Evasion)**
* **解説**: `assoc` はWindowsに標準で搭載されている正規のコマンドであり、攻撃者はアンチウイルス製品に検知されにくい「環境寄生型」の攻撃手法としてこれを利用します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: AppLockerやWindows Defender Application Control (WDAC) を使い、信頼されていないパス（例: `c:\users\public`）からの実行ファイルの起動を禁止する。
* **Detection (検知)**: `HKEY_CLASSES_ROOT` 配下のレジストリキーへの変更を、`Sysmon` (イベントID 13) などで監視します。特に、`.txt` や `.lnk` といった一般的な拡張子の関連付けが変更された場合は、アラートを発報するように設定します。

## 注意点・補足

* **`ftype`との連携**: `assoc` 単体では、ファイルを開いたときの動作は変わりません。必ず `ftype` とセットで理解・使用する必要があります。
* **システムへの影響**: `assoc` による変更は、レジストリを直接書き換えるため、システム全体（またはユーザープロファイル全体）に影響します。変更は慎重に行ってください。
