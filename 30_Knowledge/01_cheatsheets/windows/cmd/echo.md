# `echo` - メッセージを表示し、コマンドエコーを制御する

## 概要

`echo` は、引数として渡された文字列を標準出力（通常は画面）に表示するための非常に基本的なコマンドです。バッチファイル内でユーザーに進捗状況を伝えたり、リダイレクション (`>` や `>>`) と組み合わせて新しいファイルを作成したり、既存のファイルに追記したりする際に不可欠な役割を果たします。また、`echo on` / `echo off` は、バッチファイル実行時にコマンド自体を画面に表示するかどうかを制御します。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`type`** | **既存の**テキストファイルの内容を表示します。`echo` は引数で与えられた**文字列**を表示したり、新しいファイルを作成したりします。 | - |
| **PowerShell** | `Write-Output` (エイリアス: `echo`) はオブジェクトをパイプラインに出力し、`Write-Host` はコンソールに直接書き込みます。PowerShellの `echo` はより高機能です。 | `Write-Output` (PowerShell) |

## よく連携されるコマンド

`echo` の真価は、リダイレクション演算子 (`>` と `>>`) と連携することで発揮されます。

### シナリオ例: 新しい設定ファイルを作成する (インフラ構築・運用視点)

* **目的**: 1行の設定を含む新しいテキストファイルをコマンドラインから直接作成する。
* **連携コマンド**: リダイレクション (`>`)
* **解説**: `echo` で出力したい文字列を指定し、`>` を使ってその出力を新しいファイルにリダイレクト（上書き保存）します。
* **コマンド例**:

    ```cmd
    REM "enabled=true" という内容で C:\App\settings.conf ファイルを新規作成
    echo enabled=true > C:\App\settings.conf

    REM 既存のログファイルに追記する場合は >> を使用
    echo Log entry: Process completed. >> C:\App\logs\app.log
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **コマンドエコー制御** | |
| ⭐ `on` | バッチファイル内で、実行されるコマンドを画面に表示するように設定します。 |
| ⭐ `off` | バッチファイル内で、実行されるコマンドを画面に表示しないように設定します (`@echo off` が一般的)。 |
| **メッセージ表示** | |
| ⭐ `<message>` | 画面に表示する文字列を指定します。 |
| ⭐ `.` | (ドット) `echo.` と入力すると、改行（空行）を出力します。 |
| `(引数なし)` | `echo` が現在 ON か OFF かを表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: バッチファイルの実行状況をユーザーに分かりやすく表示する。
* **組み合わせ**: `echo <メッセージ>`
* **解説**: スクリプトの開始、終了、重要なステップの前後でメッセージを表示し、処理がどこまで進んでいるかを明確にします。`@echo off` をスクリプトの先頭に置くのが定石です。
* **例**:

    ```cmd
    @echo off
    echo.
    echo バックアップ処理を開始します...
    robocopy C:\Source D:\Backup /E
    echo バックアップが完了しました。
    echo.
    ```

### 2. ブルーチーム視点

* **タスク**: ログインした攻撃者に対して警告メッセージを表示する。
* **組み合わせ**: `echo <警告メッセージ> > /etc/issue` (Linuxの例)
* **解説**: `echo` を使って、ログインバナー (`/etc/issue`など) やMOTD (Message of the Day) を編集し、不正アクセスが監視されていることを示す警告を設置します。これは抑止力として機能します。
* **例**:

    ```cmd
    REM (管理者権限で実行)
    REM ネットワークドライブのルートに警告ファイルを作成
    echo "この領域へのアクセスはすべて記録されています。" > \\nas-server\share\WARNING.txt
    ```

### 3. レッドチーム視点

* **タスク**: **悪意のあるスクリプトやペイロードをファイルに書き出す**。
* **組み合わせ**: `echo <payload> > <script_file>`
* **解説**: **防御回避 (Defense Evasion)** の一環。攻撃者は `curl` や `wget` のようなツールが使えないあるいは監視されている環境で、`echo` コマンドを複数回使って、悪意のあるVBScriptやPowerShellスクリプト、あるいはバイナリの断片をターゲットシステム上に「組み立てる」ことがあります。
* **例**:

    ```cmd
    REM 簡単なVBScriptを temp.vbs として作成
    echo Set oShell = CreateObject("WScript.Shell") > %TEMP%\temp.vbs
    echo oShell.Run "calc.exe" >> %TEMP%\temp.vbs
    REM 作成したスクリプトを実行
    cscript %TEMP%\temp.vbs
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `echo` 自体は非常にシンプルなため、エラーは稀です。多くは特殊文字の扱いやリダイレクション先への権限不足に起因します。

1. **現象**: `>` や `<`、`|` などの記号が正しく表示されない、または `The process tried to write to a nonexistent pipe.` のようなエラーが出る。
    * **考えられる原因**: これらの文字はCMDによってリダイレクションやパイプとして解釈されてしまいます。
    * **解決策**: キャレット (`^`) を使って特殊文字をエスケープします。

        ```cmd
        REM ">" という文字自体を表示する
        echo ^>
        ```

## 環境変数と設定ファイル

* `echo` は環境変数を表示するためによく使われます。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

### 影響を与える環境変数

* **`%<変数名>%`**: `echo %USERNAME%` のように、環境変数を `%` で囲むと、その値が展開されて表示されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `echo` 自体ではなく、**ファイル書き込み機能**が悪用されます。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`echo` とリダイレクションは、他のダウンロードツールが使えない環境で悪意のあるスクリプトを作成するための強力な手段です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**
* **解説**: `echo` はWindowsの基本コマンドであり、その実行自体は警告を発しません。攻撃者はこのコマンドを悪用して、ディスク上に悪意のあるファイルを作成し、次の攻撃の足がかりとします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: AppLockerやWDACで、信頼できないディレクトリ（`%TEMP%`, `C:\Users\Public`など）からのスクリプトファイルの実行を禁止する。
* **Detection (検知)**: `Sysmon` (イベントID 11: FileCreate) を使って、`cmd.exe` や `powershell.exe` の `echo` コマンドによる `.vbs`, `.ps1`, `.bat` などのスクリプトファイルの作成を監視する。

## 注意点・補足

* **末尾のスペース**: `echo text > file.txt` を実行すると、`file.txt` には "text " のように末尾にスペースが含まれることがあります。厳密な設定ファイルを作成する場合は注意が必要です。
