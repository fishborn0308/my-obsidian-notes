# `del` - 1つ以上のファイルを削除する

## 概要

`del` (delete) コマンドは、1つまたは複数のファイルを指定して**恒久的に削除**するための基本的なコマンドです。`erase` は `del` のエイリアスであり、全く同じ機能です。このコマンドで削除されたファイルは、通常の方法では復元できません（ごみ箱には移動しません）。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`rmdir` / `rd`** | **空のディレクトリ**を削除します。`del` は**ファイル**を削除します。中身を含むディレクトリを削除するには `rd /s` を使います。 | `rd /s` (ディレクトリ) |
| **PowerShell** | `Remove-Item` (エイリアス: `rm`, `del`) は、ファイル、ディレクトリ、レジストリキーなど、より多くのオブジェクトを統一的に削除できる高機能なコマンドレットです。 | `Remove-Item` (PowerShell) |

## よく連携されるコマンド

`del` は、`for` ループや `if exist` と連携して、特定の条件に一致するファイルをクリーンアップするバッチファイルで多用されます。

### シナリオ例: 古いログファイルを削除する (インフラ構築・運用視点)

* **目的**: ログディレクトリ内で、特定のパターンに一致する古いログファイルを削除する。
* **連携コマンド**: `forfiles` (よりモダンな方法)
* **解説**: `forfiles` コマンドを使うと、日付に基づいたファイルの検索と、見つかったファイルに対するコマンドの実行を簡単に行えます。
* **コマンド例**:

    ```cmd
    REM C:\Logs ディレクトリで、最終更新日から30日以上経過した .log ファイルを削除
    forfiles /P C:\Logs /S /M *.log /D -30 /C "cmd /c del @path"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **ファイル指定** | |
| ⭐ `<names>` | 削除する1つ以上のファイル名を指定します。ワイルドカード (`*`, `?`) が使用できます。 |
| **動作制御** | |
| ⭐ `/P` | 各ファイルを削除する前に、**確認のプロンプト (Prompt)** を表示します。 |
| ⭐ `/F` | 読み取り専用属性のファイルも**強制的に (Force)** 削除します。 |
| ⭐ `/S` | 指定したディレクトリだけでなく、全ての**サブディレクトリ (Subdirectories)** からも指定したファイルを削除します。 |
| ⭐ `/Q` | **静観 (Quiet)** モード。ワイルドカード (`*.*`) で全ファイルを削除する際の確認プロンプトを表示しません。 |
| **属性指定** | |
| `/A[[:]attributes]` | 特定の属性を持つファイルを削除対象とします。（例: `/A:H` は隠しファイルのみを対象） |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションの一時ディレクトリの中身を全てクリーンアップする。
* **組み合わせ**: `/S /Q`
* **解説**: サブディレクトリ内のファイルも含め (`/S`)、確認プロンプトなしで (`/Q`) 全てのファイルを削除します。デプロイスクリプトやクリーンアップバッチで定番の使い方です。
* **例**:

    ```cmd
    REM C:\App\Temp ディレクトリの中身を（サブディレクトリも含めて）全て削除
    del /S /Q C:\App\Temp\*
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠保全と分析が完了したマルウェアをシステムから完全に除去する。
* **組み合わせ**: `/F /A:H`
* **解説**: マルウェアはしばしば隠しファイル属性 (`+H`) や読み取り専用属性 (`+R`) を持っています。`/F` で読み取り専用を無視し、`/A` で属性を指定して、これらの隠れたマルウェアを確実に削除します。
* **例**:

    ```cmd
    REM 隠し属性を持つ悪意のあるDLLを強制的に削除
    del /F /A:H C:\Users\Public\evil.dll
    ```

### 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)** のため、自身の活動の証拠となるファイルを削除する。
* **組み合わせ**: `del <log_or_tool_file>`
* **解説**: 攻撃者は侵入後、自身の活動を隠蔽するために `del` を多用します。ダウンロードしたツール、生成した一時ファイル、改ざんしたファイルのバックアップなどを `del` で消去し、調査を困難にしようとします。
* **例**:

    ```cmd
    REM 攻撃に使用したツールキットとログを削除
    del C:\Users\Public\nc.exe
    del C:\Users\Public\keylog.txt
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'del'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: ファイルを削除するための権限がありません。または、ファイルが別のプロセスによって使用中です。
    * **解決策**:
      * コマンドプロンプトを「管理者として実行」します。
      * ファイルが読み取り専用属性を持つ場合は、`/F` スイッチを使います。
      * Process Explorerなどのツールで、どのプロセスがファイルをロックしているかを確認し、そのプロセスを終了させます。

## 環境変数と設定ファイル

* `del` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `del` コマンドの**破壊力**そのものがリスクとなります。
* **悪用シナリオ**:
  * **破壊活動 (Sabotage) / ランサムウェア**: 攻撃者が管理者権限を奪取した後、`del /S /Q C:\*` のようなコマンドで重要なシステムファイルやユーザーデータを削除し、システムを破壊したり、身代金を要求したりします。
  * **痕跡消去**: 攻撃者がイベントログファイル (`.evtx`) などを削除して、侵入の痕跡を隠蔽します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `del` はWindowsの基本コマンドであり、その実行自体は警告を発しません。攻撃者はこのコマンドを悪用して、ログファイルや自身のツールを削除して痕跡を消したり、重要なファイルを削除してシステムに損害を与えたりします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要なファイルのバックアップを定期的に取得する。ファイルやディレクトリに適切なACL（アクセス制御リスト）を設定し、最小権限の原則を徹底する。
* **Detection (検知)**: `Sysmon` やEDRツールで、重要なシステムファイルや大量のファイルに対する `del` コマンドの実行を監視する。イベントログの集中管理を行い、ホスト上でのログ削除を検知できるようにする。

## 注意点・補足

* **ワイルドカードの危険性**: `del *.*` のようなワイルドカードを使用する際は、`cd` コマンドでカレントディレクトリを正確に確認してから実行してください。意図しない場所で実行すると、必要なファイルを削除してしまう可能性があります。安全のため、`/P` スイッチを付けて確認する癖をつけるのが良いでしょう。
