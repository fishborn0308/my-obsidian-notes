# `copy` - 1つ以上のファイルをコピーする

## 概要

`copy` コマンドは、1つ以上のファイルを、ある場所から別の場所へコピーするための、最も基本的なファイル操作コマンドの1つです。ファイルの複製や、簡易的なバックアップ作成、複数のテキストファイルの連結などに使用されます。ただし、サブディレクトリを再帰的にコピーする機能はありません。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`xcopy`** | `copy` の高機能版。`/S` スイッチでサブディレクトリを再帰的にコピーできます。 | `robocopy` |
| ⭐ **`robocopy`** | `xcopy` のさらに高機能な後継。ミラーリング、リトライ、ログ出力など、堅牢なファイルコピー機能を提供します。ディレクトリのコピーやバックアップには `robocopy` が最適です。 | `robocopy` |
| **`move`** | ファイルを**移動**します。元のファイルは残りません。`copy` はファイルを**複製**し、元のファイルはそのまま残ります。 | - |

## よく連携されるコマンド

`copy` は、複数のテキストファイルを1つにまとめる（連結する）際に、ワイルドカードと組み合わせて使われるのが特徴的です。

### シナリオ例: 複数のログファイルを1つに結合する (インフラ構築・運用視点)

* **目的**: 分割された複数のログファイル (`log_part1.txt`, `log_part2.txt`...) を、時系列で1つのファイル (`full_log.txt`) にまとめる。
* **連携コマンド**: ワイルドカード (`*`)
* **解説**: `copy` コマンドは `+` 記号やワイルドカードを使うことで、複数のファイルを指定した順番で連結できます。
* **コマンド例**:

    ```cmd
    REM log_part*.txt に一致する全ファイルを full_log.txt として結合
    copy log_part*.txt full_log.txt
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **ファイル形式** | |
| ⭐ `/A` | ファイルをASCIIテキストファイルとして扱います。 |
| ⭐ `/B` | ファイルをバイナリファイルとして扱います。 |
| **動作制御** | |
| `/V` | コピー後に、新しいファイルが正しく書き込まれたか**検証 (Verify)** します。 |
| ⭐ `/Y` | コピー先に同名のファイルが存在する場合、**上書きの確認プロンプトを表示しません**。 |
| `/-Y` | コピー先に同名のファイルが存在する場合、**上書きの確認プロンプトを表示します**。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルを編集する前に、バックアップを作成する。
* **組み合わせ**: `copy <source> <destination>`
* **解説**: `copy config.ini config.ini.bak` のように、元のファイルを別の名前で複製する、最も基本的な使い方です。
* **例**:

    ```cmd
    REM App.config ファイルのバックアップを作成
    copy C:\App\App.config C:\App\App.config.20250929.bak
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠保全のため、不審なファイルを調査用ディレクトリに複製する。
* **組み合わせ**: `copy <source> <destination> /V`
* **解説**: `/V` スイッチで検証を行うことで、コピープロセス中にファイルが破損していないことを保証し、証拠の完全性を高めます。
* **例**:

    ```cmd
    REM C:\Users\Public\suspicious.exe を証拠保全用ディレクトリにコピー
    copy C:\Users\Public\suspicious.exe D:\Forensics\Case001\ /V
    ```

### 3. レッドチーム視点

* **タスク**: 正規のシステムツールを書き込み可能なディレクトリにコピーし、それを悪用する。
* **組み合わせ**: `copy %windir%\system32\cmd.exe C:\Users\Public\cmd.exe`
* **解説**: **防御回避 (Defense Evasion)** の一環。AppLockerなどで `system32` からの実行が制限されている場合に、`cmd.exe` などを書き込み可能な別の場所にコピーし、そこから実行することで制限をバイパスしようとします。
* **例**:

    ```cmd
    REM cmd.exe をPublicディレクトリにコピー
    copy %windir%\system32\cmd.exe %PUBLIC%\cmd.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'copy'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ例 1**: `The system cannot find the file specified.`
    * **考えられる原因**: コピー元として指定したファイルが存在しないか、パスが間違っています。
    * **解決策**: `dir` コマンドでファイルが存在することと、パスが正しいことを確認してください。パスにスペースが含まれる場合は、`"C:\Program Files\..."` のようにダブルクォートで囲みます。

2. **エラーメッセージ例 2**: `Access is denied.`
    * **考えられる原因**: コピー元のファイルへの読み取り権限、またはコピー先のディレクトリへの書き込み権限がありません。
    * **解決策**: コマンドプロンプトを「管理者として実行」するか、`icacls` コマンドでアクセス権を確認・変更してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `copy` 自体ではなく、**ファイルの不正な配置**や**置き換え**が悪用されます。
* **悪用シナリオ**:
  * **データ窃取**: 攻撃者は、`copy \\fileserver\share\secret.docx C:\Users\Public\` のように、機密ファイルを外部からアクセスしやすい場所にコピーして、情報窃取の準備をします。
  * **永続化**: 正規の実行ファイルを、悪意のあるバックドアが仕込まれた同名のファイルで上書きする。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **防御回避 (Defense Evasion)**, **データ窃取 (Data Exfiltration) の準備**
* **解説**: `copy` はWindowsの基本コマンドであり、その実行自体は警告を発しないため、攻撃者は監視を逃れながらファイルを移動・複製するために多用します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: AppLockerやWDACで、信頼できないディレクトリ（`C:\Users\Public`など）からの実行ファイルの実行を禁止する。
* **Detection (検知)**: `Sysmon` (イベントID 11: FileCreate) を使って、重要なシステムディレクトリにあるファイルが予期せぬ場所にコピーされていないか、あるいは上書きされていないかを監視する。

## 注意点・補足

* **ディレクトリのコピー**: `copy` コマンドはディレクトリをコピーできません。ディレクトリをコピーする場合は `xcopy` または `robocopy` を使用してください。
