# `findstr` - ファイル内の文字列を高度に検索する

## 概要

`findstr` は、ファイルや標準入力から指定した文字列やパターンを検索するための、`find` コマンドより高機能なユーティリティです。`find` が単純な文字列検索しかできないのに対し、`findstr` は**正規表現**を使った柔軟なパターンマッチング、複数ファイルやディレクトリを横断した検索が可能です。Linuxの `grep` コマンドに相当する強力なツールです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`find`** | `find` は単純な文字列検索のみで、正規表現は使えません。`findstr` は正規表現や複数条件での検索が可能で、ほぼ上位互換です。 | `findstr` |
| **PowerShell** | `Select-String` (エイリアス: `sls`) は、オブジェクトをパイプラインで扱え、より強力な正規表現エンジン（.NET regex）を持ちます。スクリプト内での柔軟性は `Select-String` が優れています。 | `Select-String` (PowerShell) |

## よく連携されるコマンド

`findstr` は、他のコマンドの出力をパイプ (`|`) で受け取り、その中から特定のキーワードを含む行だけをフィルタリングするためによく使われます。

### シナリオ例: ネットワーク接続状態から特定のポートを探す (インフラ構築・運用視点)

* **目的**: `netstat` の出力結果から、`LISTENING`（待ち受け）状態のポートのみを表示する。
* **連携コマンド**: `netstat`, パイプ (`|`)
* **解説**: `netstat -ano` で全ての接続とプロセスIDを表示し、その結果をパイプで `findstr` に渡して "LISTENING" という文字列を含む行だけをフィルタリングします。
* **コマンド例**:

    ```cmd
    REM LISTENING状態のポートのみを表示
    netstat -ano | findstr "LISTENING"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **検索動作** | |
| ⭐ `/S` | カレントディレクトリとその全ての**サブディレクトリ**にあるファイルを検索します。 |
| ⭐ `/I` | 検索時に**大文字と小文字を区別しません** (Ignore case)。 |
| ⭐ `/V` | 指定した文字列を**含まない**行を表示します (Invert)。 |
| ⭐ `/N` | 一致した行の**行番号 (Number)** を付けて表示します。 |
| `/C:"string"` | ダブルクォートで囲まれた文字列を、スペースを含め一つのリテラル文字列として検索します。 |
| `/R` | 検索文字列を**正規表現**として扱います。デフォルトの動作に近いですが、明示的に指定する場合に使います。 |
| `/L` | 検索文字列を**リテラル**として扱います。正規表現のメタ文字をただの文字として検索したい場合に使います。 |
| **ファイル指定** | |
| `/P` | 印刷不可能な文字を含むファイルをスキップします。 |
| `/D:<dirlist>` | 検索するディレクトリをセミコロン区切りで指定します。 |
| `/F:<file>` | 検索対象のファイルリストをファイルから読み込みます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複数の設定ファイルから、特定のIPアドレスが記述されているファイルをすべて見つけ出す。
* **組み合わせ**: `/S /I`
* **解説**: `/S` でディレクトリを再帰的に検索し、`/I` で大文字小文字を無視します。これにより、広範囲のファイルから目的の記述を効率的に探し出せます。
* **例**:

    ```cmd
    REM C:\App\Config ディレクトリ配下で "192.168.1.100" を含むファイルをすべて検索
    findstr /S /I "192.168.1.100" C:\App\Config\*.*
    ```

### 2. ブルーチーム視点

* **タスク**: 大量のログファイルの中から、複数の不審なキーワードのいずれかを含む行を抽出する。
* **組み合わせ**: `/S /N /I`
* **解説**: 検索文字列をスペースで区切るとOR検索になります。`/S` で全ログファイルを対象とし、`/N` で行番号を特定、`/I` で大文字小文字を無視して、攻撃の痕跡を広範囲に調査します。
* **例**:

    ```cmd
    REM C:\Logs 配下で "error", "failed", "denied" のいずれかを含む行を行番号付きで検索
    findstr /S /N /I "error failed denied" C:\Logs\*.log
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。システム内の全ファイルから、ハードコードされたパスワードを探す。
* **組み合わせ**: `/S /I /P`
* **解説**: 攻撃者はシステム侵入後、`findstr` を使ってシステム全体から `password`, `secret`, `apikey` などのキーワードを検索します。`/S` で再帰的に、`/I` で大文字小文字を無視し、`/P` でバイナリファイルをスキップすることで、効率的に機密情報を探します。
* **例**:

    ```cmd
    REM Cドライブ全体から "password" という文字列を含むファイルを探す
    findstr /S /I /P "password" C:\*.*
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'findstr'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **現象**: スペースを含む文字列を検索できない。
    * **考えられる原因**: 検索文字列が `/C` スイッチなしで指定されています。
    * **解決策**: `findstr /C:"search string with spaces" <filename>` のように、`/C:` スイッチを付けて文字列をダブルクォートで囲んでください。

## 環境変数と設定ファイル

* `findstr` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `findstr` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`findstr` は攻撃者がシステム内に平文で保存されたパスワードやAPIキーなどの機密情報を発見するための強力なツールです。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Findstr/](https://lolbas-project.github.io/lolbas/Binaries/Findstr/)
* **解説**: `findstr` はWindowsの基本コマンドであり、その実行自体は警告を発しません。攻撃者はこのコマンドを悪用して、システム内の機密情報を探し出します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードやAPIキーを設定ファイルやソースコードにハードコードしない。機密情報は暗号化して保存するか、専用のシークレット管理ツールを使用する。
* **Detection (検知)**: `findstr.exe` の実行、特にシステム全体を対象とするような広範囲のスキャン (`/S C:\*.*`) を、プロセス監査ログやEDRで監視する。

## 注意点・補足

* **正規表現**: `findstr` の正規表現は限定的です。`^` (行頭), `$` (行末), `.` (任意の一文字), `*` (0回以上の繰り返し) などが使えますが、`+` や `?` といった文字はサポートしていません。より高度な正規表現が必要な場合は、PowerShellの `Select-String` を利用するのが良いでしょう。
