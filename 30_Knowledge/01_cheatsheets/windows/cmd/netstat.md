# `netstat` - ネットワーク統計情報を表示する

## 概要

`netstat` (network statistics) コマンドは、アクティブなネットワーク接続（インバウンド、アウトバウンド）、待ち受け中のポート、IPルーティングテーブル、ネットワークインターフェイスの統計情報などを表示するための、古典的で強力なネットワーク診断ツールです。「どのポートが開いているか？」「どのプロセスがどのIPアドレスと通信しているか？」といった、システムのネットワーク活動を調査する際に不可欠です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **PowerShell** | `Get-NetTCPConnection` コマンドレットは、`netstat` と同等の情報をオブジェクトとして取得できるため、ソート、フィルタリング、集計が遥かに容易で強力です。 | `Get-NetTCPConnection` |
| **`route print`** | `netstat -r` と同じくルーティングテーブルを表示します。機能は同等です。 | - |

## よく連携されるコマンド

`netstat` の出力は非常に多いため、パイプ (`|`) を通じて `findstr` に渡し、目的の情報を絞り込むのが一般的な使い方です。

### シナリオ例: 特定のポートを使用しているプロセスを特定する (インフラ構築・運用視点)

* **目的**: ポート 8080 をどのアプリケーションが使用（待ち受け）しているかを特定する。
* **連携コマンド**: `findstr`
* **解説**: `netstat -ano` で全ての接続と待ち受けポートを、関連するプロセスID (PID) と共に表示します。その出力から `findstr` を使って `:8080` を含む行をフィルタリングすることで、目的のプロセスを素早く特定できます。
* **コマンド例**:

    ```cmd
    REM ポート 8080 を使用している接続やプロセスを検索
    netstat -ano | findstr ":8080"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示内容** | |
| ⭐ `-a` | 全てのアクティブなTCP接続と、TCPおよびUDPの待ち受けポートを表示します。 |
| ⭐ `-n` | アドレスとポート番号を名前解決せず、**数値形式**で表示します。実行が高速になります。 |
| ⭐ `-o` | 各接続に関連付けられた**プロセスID (PID)** を表示します。**非常に重要です**。 |
| ⭐ `-b` | (`管理者権限要`) `-o` と似ていますが、PIDではなく**実行可能ファイル名 (.exe)** を表示します。 |
| `-r` | ルーティングテーブルの内容を表示します (`route print` と同義)。 |
| `-e` | イーサネットの統計情報（送受信バイト数など）を表示します。 |
| **動作制御** | |
| `<interval>` | 指定した秒数ごとに、情報を繰り返し表示します。`Ctrl+C` で停止します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー上でどのポートが待ち受け（LISTENING）状態になっているかを確認する。
* **組み合わせ**: `-ano`
* **解説**: `netstat -ano` は、システムの全接続状態をPID付きで把握するための万能な組み合わせです。`findstr "LISTENING"` と組み合わせることで、意図しないポートが開いていないか、目的のサービスが正しくポートをバインドしているかを確認できます。
* **例**:

    ```cmd
    REM 現在待ち受け中のポートと、それに対応するプロセスIDを全て表示
    netstat -ano | findstr "LISTENING"
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンスの基本**。マルウェアのC2通信など、不審な外部へのネットワーク接続を発見する。
* **組み合わせ**: `netstat -ano`
* **解説**: 侵害されたホストでこのコマンドを実行し、`ESTABLISHED`（接続済み）のリストを精査します。見慣れないリモートIPアドレスやポートへの接続がないか、また、その接続を行っているプロセス (PID) は何かを特定します。特定したPIDは `tasklist` でさらに詳細を調査します。
* **例**:

    ```cmd
    REM 5秒ごとにネットワーク接続を監視し、不審な接続がないかリアルタイムで確認
    netstat -ano 5
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストのネットワーク活動を把握する。
* **組み合わせ**: `netstat -an`
* **解説**: 攻撃者は侵入後、`netstat -an` を実行して、そのサーバーがどの他のサーバー（データベース、ドメインコントローラーなど）と通信しているかを特定します。これにより、内部ネットワークの構成をマッピングし、次のラテラルムーブメント（横展開）の標的を探します。
* **例**:

    ```cmd
    REM 確立済みの接続をリストアップし、内部の通信相手を特定
    netstat -an | findstr "ESTABLISHED"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'netstat'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **現象**: `-b` スイッチを使っても、一部の実行可能ファイル名が表示されない。
    * **考えられる原因**: `-b` スイッチは多くのリソースを消費し、時間がかかることがあります。また、権限が不足していると情報を取得できない場合があります。
    * **解決策**: コマンドプロンプトを「管理者として実行」してください。より確実なのは `-o` でPIDを特定し、`tasklist /FI "PID eq <pid>"` でプロセス名を確認する方法です。

## 環境変数と設定ファイル

* `netstat` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `netstat` 自体は読み取り専用のツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`netstat` の出力は、攻撃者がシステムのネットワーク構成、稼働中のサービス、通信相手などを把握するための非常に価値の高い偵察情報です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `netstat` は、攻撃者が侵入後の情報収集フェーズで、ネットワーク環境をマッピングするために必ずと言っていいほど使用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `netstat.exe` の実行、特に不審なプロセスからの実行をプロセス監査ログ（イベントID 4688）やEDRで監視する。予期せぬポートでの `LISTENING` や、不審な外部IPへの `ESTABLISHED` 接続をネットワーク監視ツールやEDRで検知する。

## 注意点・補足

* **`-b`スイッチの負荷**: `/b` スイッチは、各接続に対応する実行可能ファイルを作成するために時間がかかることがあります。管理者権限が必要で、多数の接続があるシステムでは応答が非常に遅くなる可能性があります。
