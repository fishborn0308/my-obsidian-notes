# `shutdown` - Windowsをシャットダウンまたは再起動する

## 概要

`shutdown` コマンドは、Windowsコンピュータをコマンドラインからシャットダウン、再起動、ログオフ、または休止状態にするためのユーティリティです。タイマーを設定して指定時間後に操作を実行したり、リモートのコンピュータに対して操作を行ったり、ユーザーに通知メッセージを表示したりと、柔軟な制御が可能です。サーバーの計画的な再起動や、スクリプトによる自動シャットダウンなどで利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`logoff`** | 現在のユーザーセッションを**ログオフ**するだけで、コンピュータの電源は切れません。`shutdown /l` と同等です。 | - |
| **PowerShell** | `Stop-Computer` や `Restart-Computer` コマンドレットは、より高度なスクリプト機能（複数のコンピュータへの一括実行など）を提供します。 | `Stop-Computer` / `Restart-Computer` |

## よく連携されるコマンド

`shutdown` は、パッチ適用や設定変更のバッチ処理の最後に、変更を適用するために実行されることが多いです。

### シナリオ例: パッチ適用後のサーバー自動再起動 (インフラ構築・運用視点)

* **目的**: サーバーにパッチを適用するバッチスクリプトの最後に、60秒の猶予期間を設けて自動的に再起動させる。
* **連携コマンド**: (バッチファイル内)
* **解説**: `/r` で再起動を指定し、`/t 60` でタイムアウトを60秒に設定します。`/c` で再起動の理由をユーザーに通知できます。
* **コマンド例**:

    ```cmd
    REM ... パッチ適用処理 ...

    echo.
    echo Patch application complete. The server will restart in 60 seconds.
    shutdown /r /t 60 /c "System is restarting to apply security patches."
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **主な操作** | |
| ⭐ `/s` | コンピュータを**シャットダウン (Shutdown)** します。 |
| ⭐ `/r` | コンピュータを**再起動 (Restart)** します。 |
| ⭐ `/l` | 現在のユーザーを**ログオフ (Logoff)** します。 |
| `/h` | コンピュータを**休止状態 (Hibernate)** にします。 |
| ⭐ `/a` | (`/s` や `/r` とは独立) システムシャットダウンを**中止 (Abort)** します。タイマーが作動中にのみ有効です。 |
| **オプション** | |
| ⭐ `/t <seconds>` | 操作を実行するまでの**タイムアウト (Time)** 期間を秒単位で指定します。デフォルトは30秒です。 |
| ⭐ `/m \\<computer>` | 操作の対象となるリモートコンピュータを指定します。 |
| `/c "<comment>"` | シャットダウンの理由を**コメント**として指定します（最大512文字）。 |
| ⭐ `/f` | 実行中のアプリケーションをユーザーに警告せずに**強制的に (Force)** 閉じます。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートのサーバー `SRV-01` を、夜間に即時強制再起動する。
* **組み合わせ**: `/r /m \\<computer> /f /t 0`
* **解説**: `/m` でリモートコンピュータを指定し、`/r` で再起動、`/f` でアプリケーションを強制終了、`/t 0` で即時実行します。
* **例**:

    ```cmd
    REM リモートサーバー SRV-01 を強制的に即時再起動
    shutdown /r /m \\SRV-01 /f /t 0
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデント対応における封じ込め**。ランサムウェアの拡散など、緊急事態において侵害されたホストをネットワークから切り離すために即時シャットダウンする。
* **組み合わせ**: `/s /m \\<computer> /f /t 0`
* **解説**: マルウェアの活動を即座に停止させるための最終手段の1つ。リモートから `shutdown` コマンドを送り、ホストをシャットダウンすることで、被害の拡大を食い止めます。
* **例**:

    ```cmd
    REM 侵害されたホスト 'VICTIM-PC' を即時シャットダウン
    shutdown /s /m \\VICTIM-PC /f /t 0
    ```

### 3. レッドチーム視点

* **タスク**: **サービス妨害 (DoS, Denial of Service)**。
* **組み合わせ**: `/r /f`
* **解説**: 攻撃者は管理者権限を奪取した後、`shutdown` コマンドを使ってサーバーを予期せず再起動またはシャットダウンさせ、業務を妨害します。これを `while` ループなどで繰り返すことで、持続的なサービス妨害を引き起こすことも可能です。
* **例**:

    ```cmd
    REM サーバーを5分後に再起動するように仕掛ける
    shutdown /r /t 300 /c "Unscheduled maintenance is required."
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Access is denied.(5)`
    * **考えられる原因**: シャットダウンや再起動の実行、またはリモートコンピュータへの操作には管理者権限が必要です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。リモートの場合は、実行元のアカウントがターゲットコンピュータの管理者である必要があります。

## 環境変数と設定ファイル

* `shutdown` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `shutdown` コマンドの**サービス妨害**への悪用。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、攻撃者はシステムやサービスを停止させるためにこのコマンドを悪用します。正規のコマンドであるため、アンチウイルス製品による検知が難しい場合があります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **影響 (Impact)**
* **解説**: `shutdown.exe` は、攻撃者がシステムの可用性を損なう（サービス妨害）目的で利用する、典型的な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、一般ユーザーにシャットダウン権限を与えない。ユーザー権利の割り当て (`secpol.msc`) で、「リモートシステムからの強制シャットダウン」権限を Administrators のみに制限する。
* **Detection (検知)**: Windows セキュリティイベントログで、システムのシャットダウンまたは再起動のイベント (イベントID 1074) や、`shutdown.exe` の実行 (プロセス監査イベントID 4688) を監視する。

## 注意点・補足

* **中止コマンド**: `shutdown /s /t 300` のようにタイマーを設定した場合、`shutdown /a` を実行することで、タイムアウト期間内であればシャットダウンをキャンセルできます。
