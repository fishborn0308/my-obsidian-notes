# `runas` - 別のユーザーとしてプログラムを実行する

## 概要

`runas` コマンドは、現在のユーザーとしてログインしたまま、一時的に別のユーザーアカウントの権限でプログラムやコマンドを実行するためのコマンドです。Linuxの `sudo` に似た目的で使用されます。

日常業務では一般ユーザーアカウントを使用し、管理者権限が必要な特定のタスク（ソフトウェアのインストール、システム設定の変更など）を実行する場合に、`runas` を使って管理者アカウントとしてコマンドプロンプトや特定のプログラムを起動します。これにより、最小権限の原則を実践し、システムのセキュリティを向上させることができます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **右クリック -> 「管理者として実行」** | GUI操作。現在のユーザーアカウントが管理者グループに属している場合に、UACプロンプトを経て権限を昇格させます。`runas` は、**全く別のユーザー**の資格情報を使用できます。 | - |
| **PowerShell** | `Start-Process -Verb RunAs` はUAC昇格を、`Start-Process -Credential` は別ユーザーとしての実行をより柔軟に、スクリプト内でオブジェクト指向に扱えます。 | `Start-Process` (PowerShell) |

## よく連携されるコマンド

`runas` は、管理者権限が必要な各種コマンド (`cmd.exe`, `regedit`, `mmc` など) を起動するために連携します。

### シナリオ例: 管理者権限のコマンドプロンプトを起動する (インフラ構築・運用視点)

* **目的**: 通常のユーザーセッションから、システム設定を変更するために管理者権限を持つコマンドプロンプトを起動する。
* **連携コマンド**: `cmd`
* **解説**: `runas` を使って、ローカルの `Administrator` アカウントとして新しいコマンドプロンプトウィンドウ (`cmd.exe`) を起動します。パスワードの入力を求められ、認証が成功すると管理者権限を持つ新しいウィンドウが開きます。
* **コマンド例**:

    ```cmd
    REM ローカルのAdministratorとしてコマンドプロンプトを起動
    runas /user:Administrator cmd.exe

    REM ドメインの管理者としてコマンドプロンプトを起動
    runas /user:CORP\DomainAdmin cmd.exe
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **ユーザー指定** | |
| ⭐ `/user:<UserName>` | プログラムの実行に使用するユーザー名を指定します。`user@domain` または `domain\user` の形式です。 |
| **動作モード** | |
| `/netonly` | 資格情報をリモートアクセス専用とすることを指定します。プログラムはローカルコンピュータ上では現在のユーザー権限で実行されますが、リモートリソースにアクセスする際は指定したユーザーの権限が使われます。 |
| `/noprofile` | 指定したユーザーのプロファイルを読み込みません。起動が高速になる場合がありますが、プロファイルに依存するアプリケーションでは問題が発生する可能性があります。 |
| `/env` | 現在のユーザーの環境変数を引き継ぎます。 |
| **資格情報の保存 (非推奨)** | |
| ⭐ `/savecred` | ユーザーが入力したパスワードを保存します。**セキュリティリスクが高いため、使用には細心の注意が必要です。** |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 一般ユーザーでPCを操作中に、ドメインコントローラーの「Active Directory ユーザーとコンピューター」(`dsa.msc`) を起動する。
* **組み合わせ**: `/user:<domain>\<admin>`
* **解説**: 日常業務用の一般アカウントからログアウトすることなく、ドメイン管理者アカウントの権限で特定の管理ツール (`mmc` のスナップインなど) を起動します。
* **例**:

    ```cmd
    REM ドメイン管理者として 'dsa.msc' を起動
    runas /user:CORP\DomainAdmin "mmc dsa.msc"
    ```

### 2. ブルーチーム視点

* **タスク**: `runas.exe` の実行履歴を調査し、不正な権限昇格やラテラルムーブメントの試みがないか確認する。
* **組み合わせ**: (イベントログ分析)
* **解説**: `runas` が実行されると、Windows セキュリティイベントログにログオンイベント (イベントID 4624) やプロセス作成イベント (イベントID 4688) が記録されます。どのユーザーが、どのターゲットユーザーとして、どのプログラムを実行しようとしたかを監視することで、攻撃の兆候を捉えることができます。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **ラテラルムーブメント (Lateral Movement)**。窃取した認証情報を使って、別のユーザーとしてコマンドを実行する。
* **組み合わせ**: `/user:<stolen_user>`
* **解説**: 攻撃者は、`mimikatz` などで窃取したドメイン管理者などの認証情報を使い、`runas` を使ってそのユーザーの権限で `cmd.exe` や `powershell.exe` を起動します。これにより、現在のホスト上で権限を昇格させたり、他のサーバーへのアクセスを試みたりします。
* **例**:

    ```cmd
    REM 窃取したドメイン管理者の資格情報でPowerShellを起動
    runas /user:CORP\StolenAdmin powershell.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `RUNAS ERROR: Unable to run - <program>. 1326: Logon failure: unknown user name or bad password.`
    * **考えられる原因**: ユーザー名またはパスワードが間違っています。
    * **解決策**: ユーザー名 (`domain\user` 形式を含む) とパスワードが正しいか再確認してください。

2. **エラーメッセージ**: `RUNAS ERROR: Unable to run - <program>. 740: The requested operation requires elevation.`
    * **考えられる原因**: `runas` で起動しようとしているプログラム自体が、さらに管理者権限への昇格（UACプロンプト）を要求していますが、`runas` のセッションからはUACプロンプトを直接操作できないためです。
    * **解決策**: まず `runas` で管理者権限の `cmd.exe` を起動し、その新しいコマンドプロンプト内から目的のプログラムを実行してみてください。

## 環境変数と設定ファイル

* `runas` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `/savecred` オプションの**安易な使用**。
* **悪用シナリオ**: 管理者が `/savecred` を使って一度 `runas` を実行すると、その資格情報がWindows資格情報マネージャーに保存されます。攻撃者がそのホストに侵入した場合、保存された資格情報を悪用して、パスワードなしで管理者権限のコマンドを実行できてしまいます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **権限昇格 (Privilege Escalation)**, **ラテラルムーブメント (Lateral Movement)**, **防御回避 (Defense Evasion)**
* **解説**: `runas` は、攻撃者が窃取した認証情報を利用して権限を昇格させたり、別のユーザーのコンテキストで活動して検知を回避したりするための、極めて重要な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `/savecred` オプションの使用を禁止するポリシーを適用する。最小権限の原則を徹底し、管理者アカウントの認証情報を厳格に管理する。
* **Detection (検知)**: Windows セキュリティイベントログで、`runas` によるログオンイベント (イベントID 4624, ログオンタイプ 2 - Interactive) や、`runas.exe` のプロセス作成イベント (イベントID 4688) を監視する。

## 注意点・補足

* **GUIアプリケーション**: `runas` は主にCUIアプリケーションの実行に使われます。`explorer.exe` のようなGUIシェルの一部を `runas` で起動しようとすると、期待通りに動作しないことがあります。
