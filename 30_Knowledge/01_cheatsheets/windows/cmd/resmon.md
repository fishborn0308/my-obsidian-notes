# `resmon` - リソースモニターを起動する

## 概要

`resmon` コマンドは、**リソースモニター** (`resmon.exe`) のGUIを起動します。リソースモニターは、タスクマネージャーよりも詳細なパフォーマンスデータとリソース使用状況をリアルタイムで提供する高度な監視ツールです。CPU、メモリ、ディスク、ネットワークの4つの主要なリソースについて、どのプロセスが、どのファイルやネットワーク接続に、どれくらいアクセスしているかを詳細に分析できます。

システムのパフォーマンスボトルネックを特定したり、不審なプロセスの活動を調査したりする際に非常に強力なツールです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`taskmgr`** | タスクマネージャーはシステムの全体的な概要を素早く把握するのに適しています。リソースモニターは、特定のプロセスが**どのディスクファイルにアクセスしているか**、**どのTCPポートで通信しているか**など、より深いレベルの詳細情報を提供します。 | `resmon` (詳細分析) |
| **`perfmon`** | パフォーマンスモニターは、パフォーマンスカウンターを使って長期間にわたるデータ収集と分析、レポート作成を得意とします。`resmon` は**リアルタイム**の「今、何が起きているか」の分析に特化しています。 | `perfmon` (長期監視) |
| **PowerShell** | `Get-Process`, `Get-NetTCPConnection` などを組み合わせることで、`resmon` が表示する情報をCUIで取得できますが、`resmon` はそれらを相関させて視覚的に表示する点で優れています。 | - |

## よく連携されるコマンド

`resmon` はGUIツールであり、他のコマンドと直接連携することはありません。しかし、`tasklist` や `netstat` で不審なプロセスや接続を発見した後に、その詳細を分析するために起動されることがよくあります。

### シナリオ例: 不明なプロセスの活動を特定する (ブルーチーム視点)

* **目的**: `tasklist` で見つけた不審なプロセス `svchost.exe` (多数存在する中の1つ) が、実際にどのようなネットワーク活動を行っているかを特定する。
* **連携コマンド**: `tasklist`
* **解説**: まず `tasklist /svc` などで不審なプロセスのPIDを特定します。次に `resmon` を起動し、「ネットワーク」タブの「TCP接続」ペインでそのPIDを探し、どのリモートアドレスと通信しているか、どれくらいのデータを送受信しているかをリアルタイムで監視します。
* **コマンド例**:

    ```cmd
    REM リソースモニターを起動して詳細な調査を開始
    resmon
    ```

## スイッチ/オプション説明

`resmon` コマンド自体にはスイッチはありません。実行するとGUIアプリケーションが起動します。以下はGUI内の主要なタブです。

| タブ名 | 説明 |
| :--- | :--- |
| **概要 (Overview)** | CPU、メモリ、ディスク、ネットワークの主要なメトリクスをまとめて表示します。 |
| **CPU** | プロセスごとのCPU使用率、関連付けられたサービス、ハンドル、モジュールを表示します。 |
| **メモリ (Memory)** | プロセスごとの物理メモリ使用量、ハードフォールトなどを詳細に表示します。 |
| ⭐ **ディスク (Disk)** | プロセスごとのディスクI/O (読み取り/書き込みバイト数/秒) と、アクセスしているファイルを表示します。 |
| ⭐ **ネットワーク (Network)** | プロセスごとのネットワークI/O、TCP接続、待ち受けポートを表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのパフォーマンスが低下している原因が、ディスクI/Oのボトルネックにあるかを調査する。
* **組み合わせ**: (GUI操作) `ディスク` タブ
* **解説**: `resmon` を起動し、「ディスク」タブを開きます。「ディスク活動」ペインをI/O合計（B/秒）でソートし、どのプロセスがディスクに最も負荷をかけているか、どのファイルにアクセスが集中しているかを特定します。

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンス**。マルウェアの疑いがあるプロセスのネットワーク活動をリアルタイムで監視する。
* **組み合わせ**: (GUI操作) `ネットワーク` タブ
* **解説**: `resmon` の「ネットワーク」タブを開き、不審なプロセスにチェックを入れます。すると、そのプロセスに関連するネットワーク活動（TCP接続、待ち受けポートなど）だけがフィルタリング表示されます。これにより、マルウェアのC2サーバーとの通信先IPアドレスやポートを特定できます。

* **解説**: 攻撃者は侵入後、`resmon` を起動して、システムでどのようなアプリケーションが動作しているか、どのようなネットワーク接続が常時確立されているか（ログ転送など）、そしてEDRやアンチウイルス製品のプロセスがリソースを監視していないかなどを調査します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `resmon` 自体は読み取り専用の監視ツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、`resmon` は攻撃者がシステムの内部構成やセキュリティ対策を把握するための非常に強力な偵察ツールです。プロセス名、接続先IP、アクセスしているファイルパスなど、機密情報が豊富に表示されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `resmon.exe` は、攻撃者が侵入後の情報収集フェーズで、システムの詳細な動作状況を把握するために利用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `resmon.exe` の実行をプロセス監査ログ (イベントID 4688) で監視します。特に、一般ユーザーによる実行や、不審なプロセス（Webサーバーのワーカープロセスなど）からの呼び出しはアラートの対象とすべきです。

## 注意点・補足

* **管理者権限**: `resmon` は一般ユーザーでも起動できますが、「管理者として実行」することで、全てのプロセスの詳細情報を表示できるようになります。
