# `reg` - Windowsレジストリをコマンドラインから操作する

## 概要

`reg` コマンドは、Windowsのシステム設定やアプリケーション設定が格納されている階層型データベースである**レジストリ**を、コマンドラインから直接操作するための強力なツールです。GUIツールであるレジストリエディタ (`regedit.exe`) と同様の操作を、スクリプトやバッチファイルで自動化するために使用されます。レジストリの変更はシステムに深刻な影響を与える可能性があるため、操作には細心の注意が必要です。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`regedit`** | GUIベースのレジストリエディタ。対話的な操作や検索に適しています。`reg` はCUIベースであり、スクリプトによる自動化に適しています。 | `regedit` (対話的操作) |
| **PowerShell** | `Get-ItemProperty`, `Set-ItemProperty`, `New-Item` などのコマンドレットは、レジストリをドライブのように (`HKLM:\...`) 扱え、オブジェクト指向で非常に柔軟な操作が可能です。 | PowerShell (スクリプト) |

## よく連携されるコマンド

`reg` コマンドは、`for` ループと連携して、複数のレジストリキーや値を一度に処理するバッチファイルなどで利用されます。

### シナリオ例: スタートアッププログラムの一覧を取得する (ブルーチーム視点)

* **目的**: 現在のユーザーのスタートアップ (`Run` キー) に登録されているプログラムの一覧を取得する。
* **連携コマンド**: (なし)
* **解説**: `reg query` を使って、自動実行プログラムが登録される代表的なレジストリキーの内容を照会します。
* **コマンド例**:

    ```cmd
    REM 現在のユーザーのRunキーに登録されているプログラムを照会
    reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run"
    ```

## スイッチ/オプション説明

`reg` は `reg <operation> [parameters]` の形式で使用します。

| 操作 (Operation) | 説明 |
| :--- | :--- |
| ⭐ **`query`** | レジストリキー配下の値やサブキーを照会します。 |
| ⭐ **`add`** | 新しいサブキーまたは値のエントリを追加します。 |
| ⭐ **`delete`** | サブキーまたは値のエントリを削除します。 |
| **`copy`** | レジストリエントリを別の場所にコピーします。 |
| **`save`** / **`load`** | レジストリキーの一部をファイルとして保存/復元します。 |
| **`export`** / **`import`** | レジストリキー全体を `.reg` ファイルとしてエクスポート/インポートします。 |

| 一般的なパラメータ | 説明 |
| :--- | :--- |
| ⭐ `<KeyName>` | `HKLM\Software\...` のような完全なキー名を指定します。 |
| ⭐ `/v <ValueName>` | 操作対象の値の名前を指定します。 |
| `/ve` | キーの既定の値を操作対象とします。 |
| ⭐ `/t <Type>` | 値のデータ型を指定します (`REG_SZ`, `REG_DWORD`など)。 |
| ⭐ `/d <Data>` | 値に設定するデータを指定します。 |
| ⭐ `/f` | 確認プロンプトを表示せずに、強制的に操作を実行します。 |
| `/s` | 指定したキー配下の全てのサブキーと値を再帰的に照会します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのインストールスクリプトで、必要なレジストリ値を設定する。
* **組み合わせ**: `add`
* **解説**: アプリケーションの動作に必要な設定値を、バッチファイルを使って自動的にレジストリに追加します。
* **例**:

    ```cmd
    REM アプリケーションのライセンスキーを設定
    reg add "HKLM\Software\MyApp" /v LicenseKey /t REG_SZ /d "1234-5678-90AB-CDEF" /f
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによる**永続化**の一般的な手口である `Run` キーを監査する。
* **組み合わせ**: `query`
* **解説**: `HKEY_CURRENT_USER` (HKCU) と `HKEY_LOCAL_MACHINE` (HKLM) の両方の `Run` キーを調査し、見慣れないプログラムや不審なパスの実行ファイルが登録されていないかを確認します。
* **例**:

    ```cmd
    REM システム全体のスタートアップ項目を調査
    reg query "HKLM\Software\Microsoft\Windows\CurrentVersion\Run"
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のため、`Run` キーにバックドアを登録する。
* **組み合わせ**: `add`
* **解説**: 攻撃者は管理者権限を奪取した後、システムの起動時やユーザーのログイン時に自身のペイロードが自動実行されるように、`Run` キーに値を書き込みます。これは最も古典的で一般的な永続化の手法です。
* **例**:

    ```cmd
    REM ユーザーログイン時に悪意のあるペイロードを実行するように登録
    reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "Windows Update" /t REG_SZ /d "C:\Users\Public\payload.exe" /f
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `ERROR: Access is denied.`
    * **考えられる原因**: レジストリの変更には管理者権限が必要です。特に `HKLM` ハイブの変更は管理者権限が必須です。
    * **解決策**: コマンドプロンプトを「管理者として実行」して、再度コマンドを実行してください。

## 環境変数と設定ファイル

* `reg` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `reg` 自体ではなく、**レジストリの不適切な変更**が重大なセキュリティリスクとなります。
* **悪用シナリオ**: 攻撃者は `reg` コマンドを駆使して、以下のような様々な悪意のある活動を行います。
  * **永続化**: `Run` キー、サービス、AppInit_DLLsなどへの登録。
  * **防御回避**: UACの無効化、Windows Defenderのリアルタイム保護の無効化。
  * **情報窃取**: 保存された認証情報（例: Winlogon）の窃取。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **永続化 (Persistence)**, **防御回避 (Defense Evasion)**
* **参照**: [https://lolbas-project.github.io/lolbas/Binaries/Reg/](https://lolbas-project.github.io/lolbas/Binaries/Reg/)
* **解説**: `reg.exe` は、攻撃者がシステムの構成を変更し、自身の活動を隠蔽・維持するために利用する、極めて重要な環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、一般ユーザーにレジストリの書き込み権限を与えない。
* **Detection (検知)**: `Sysmon` を導入し、レジストリキーの作成、変更、削除（特に `Run` キーなど永続化に使われるキー）を **イベントID 12, 13, 14** で厳重に監視する。

## 注意点・補足

* **ハイブの短縮形**: `HKEY_LOCAL_MACHINE` は `HKLM`、`HKEY_CURRENT_USER` は `HKCU` のように短縮して記述できます。
* **レジストリのバックアップ**: `reg` コマンドで重要なキーを変更する前には、`reg export` コマンドで対象のキーをバックアップしておくことが強く推奨されます。
