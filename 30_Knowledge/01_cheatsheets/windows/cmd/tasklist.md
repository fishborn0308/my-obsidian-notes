# `tasklist` - 実行中のプロセスを一覧表示する

## 概要

`tasklist` コマンドは、システムで現在実行中の全てのプロセスに関する情報を表示するためのコマンドラインツールです。各プロセスについて、イメージ名（実行ファイル名）、プロセスID (PID)、セッション名、メモリ使用量などを一覧表示します。システムのパフォーマンスを監視したり、応答しないアプリケーションを特定したり、不審なプロセスが実行されていないかを調査したりする際に、最初に使用される基本的なコマンドの1つです。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ **`taskkill`** | `tasklist` がプロセスを**一覧表示**するのに対し、`taskkill` はプロセスを**終了**させます。これらは対になるコマンドです。 | - |
| **PowerShell** | `Get-Process` (エイリアス: `ps`, `gps`) は、プロセスをオブジェクトとして取得するため、ソート、フィルタリング、パイプラインでの後続処理が遥かに強力で柔軟です。 | `Get-Process` |

## よく連携されるコマンド

`tasklist` の出力は情報量が多いため、`findstr` と連携して特定のプロセス情報を絞り込んだり、その結果を `taskkill` に渡してプロセスを終了させたりするのが一般的な使い方です。

### シナリオ例: 特定のプロセスID (PID) を特定して終了させる (インフラ構築・運用視点)

* **目的**: 複数実行されている `chrome.exe` のうち、メモリを大量に消費している特定のプロセスを見つけて終了させる。
* **連携コマンド**: `findstr`, `taskkill`
* **解説**: `tasklist` で全プロセスを表示し、`findstr` で "chrome.exe" を含む行をフィルタリングします。その中からPIDを特定し、`taskkill` コマンドでそのプロセスを終了させます。
* **コマンド例**:

    ```cmd
    REM chrome.exe のプロセスリストを表示
    tasklist | findstr /I "chrome.exe"

    REM (上記で特定したPID 5678 のプロセスを強制終了)
    taskkill /PID 5678 /F
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示内容** | |
| ⭐ `/V` | 各プロセスの**詳細 (Verbose)** な情報（ウィンドウタイトル、実行ユーザーなど）を表示します。 |
| ⭐ `/SVC` | 各プロセスでホストされている**サービス (Services)** を表示します。`svchost.exe` の中身を特定するのに不可欠です。 |
| `/M [<module>]` | 指定したDLL**モジュール (Module)** を使用している全てのプロセスをリストアップします。 |
| **出力形式** | |
| ⭐ `/FO <format>` | 出力形式を指定します。`TABLE` (デフォルト), `LIST`, `CSV` があります。 |
| ⭐ `/NH` | (No Header) `TABLE` または `CSV` 形式の出力で、ヘッダー行を表示しません。 |
| **フィルタリング** | |
| ⭐ `/FI <filter>` | 表示するプロセスを指定する**フィルター (Filter)** を適用します。 (例: `/FI "IMAGENAME eq cmd.exe"`) |
| **リモート操作** | |
| `/S <computer>` | 操作対象のリモートコンピュータを指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: `svchost.exe` として実行されている各サービスの実体を確認する。
* **組み合わせ**: `/SVC`
* **解説**: `svchost.exe` は多くのWindowsサービスをホストする汎用的なプロセスです。`/SVC` を使うことで、どのPIDの `svchost.exe` がどのサービス（例: `wuauserv`, `Dhcp`）に対応しているかを特定できます。
* **例**:

    ```cmd
    REM svchost.exe がホストしているサービスを一覧表示
    tasklist /SVC | findstr /I "svchost.exe"
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンス**。侵害されたホストで実行中の不審なプロセスを特定する。
* **組み合わせ**: `/FO CSV /V`
* **解説**: **インシデント対応の基本**。`tasklist /FO CSV /V > process_list.csv` を実行して、全プロセスの詳細情報をCSVとして取得します。Excelなどで開き、見慣れないプロセス名、`Image Path` が不審な場所（例: `C:\Users\Public`）になっていないか、`User Name` が予期せぬもの（例: `SYSTEM` で動作する非標準プロセス）でないかなどを調査します。
* **例**:

    ```cmd
    REM 全プロセスの詳細をCSVで出力
    tasklist /V /FO CSV > tasklist_verbose.csv
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストで実行されているセキュリティ製品を特定する。
* **組み合わせ**: `tasklist /FO LIST`
* **解説**: 攻撃者は侵入後、ほぼ確実に `tasklist` を実行します。`cb.exe` (Carbon Black), `MsMpEng.exe` (Windows Defender), `CylanceSvc.exe` (Cylance) のような、アンチウイルスやEDR製品のプロセス名を探し、その後の攻撃でそれらの検知を回避する方法を計画します。
* **例**:

    ```cmd
    REM 実行中のプロセスを偵察し、セキュリティ製品を探す
    tasklist | findstr /I "defender Cylance carbon"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'tasklist'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `ERROR: The RPC server is unavailable.`
    * **考えられる原因**: リモートコンピュータを指定して実行 (`/S`) しましたが、ファイアウォールでブロックされているか、必要なサービスがリモート側で実行されていません。
    * **解決策**: リモートコンピュータのWindows Firewallで「リモート管理」が許可されていること、および "Remote Procedure Call (RPC)" サービスが実行中であることを確認してください。

## 環境変数と設定ファイル

* `tasklist` は、その動作に影響を与える特定の環境変数や設定ファイルを使用しません。
* 一般的な環境変数は、[CMD共通の環境変数・設定ファイル](./environment_and_config.md)で解説。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `tasklist` 自体は読み取り専用のツールですが、その出力が**重大な情報漏洩**に繋がります。
* **悪用シナリオ**: レッドチームのシナリオで述べた通り、実行中のプロセスのリストは、攻撃者がシステムの構成、稼働中のアプリケーション、そして最も重要な**セキュリティ対策ソフト**を把握するための非常に価値の高い偵察情報です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `tasklist.exe` は、攻撃者が侵入後の情報収集フェーズで、システムの状況を把握するために必ずと言っていいほど使用する環境寄生型バイナリです。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `tasklist.exe` の実行、特に不審なプロセス（例: Webサーバーのワーカープロセス）や一般ユーザーからの実行をプロセス監査ログ（イベントID 4688）やEDRで監視する。

## 注意点・補足

* **`wmic`との比較**: `wmic process list full` コマンドは、`tasklist` よりもさらに詳細な情報（コマンドライン引数など）を取得できます。より深い調査には `wmic` やPowerShellの `Get-Process` が有効です。
