# `ping` - ネットワークホストの疎通を確認する

## 概要

`ping` コマンドは、指定したホスト（IPアドレスまたはホスト名）がネットワーク的に**到達可能か**どうかをテストするための、最も基本的なネットワーク診断ツールです。`ping` は、**ICMP (Internet Control Message Protocol)** の「エコー要求 (echo request)」パケットをターゲットホストに送信し、そのホストからの「エコー応答 (echo reply)」を待ち受けます。

応答が返ってくることで、ホストが稼働していること、そしてそこまでのネットワーク経路に問題がないことを確認できます。また、応答時間 (**latency** or **RTT**) も表示されるため、ネットワークの遅延を測定するのにも利用されます。

(出自: `Windows標準搭載`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`tracert` / `pathping`** | 宛先までの**ネットワーク経路**を調査します。`ping` は**最終的な宛先**との疎通のみを確認します。「繋がらない」ときに、どこが問題かを調べるのが `tracert` や `pathping` です。 | `pathping` (詳細診断) |
| **PowerShell** | `Test-NetConnection` は、ICMP (`ping`) だけでなく、特定のTCPポートへの接続テストも行える、より高機能なコマンドレットです。 | `Test-NetConnection` |

## よく連携されるコマンド

`ping` は、バッチファイルの `for` ループと連携して、ネットワーク内の複数のホストの生存確認（Pingスイープ）を行う際に利用されます。

### シナリオ例: ネットワーク内の生存ホストを探索する (レッドチーム視点)

* **目的**: ネットワーク内のどのホストがオンラインであるかを探索する。
* **連携コマンド**: `for`
* **解説**: `for /L` ループを使ってサブネット内の全てのIPアドレスに対して `ping` を送信し、応答があったホストのみを `findstr` でフィルタリングして表示します。
* **コマンド例**:

    ```cmd
    REM 192.168.1.1 から 192.168.1.254 までPingスイープを実行
    FOR /L %i IN (1,1,254) DO ping -n 1 192.168.1.%i | findstr "Reply"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **動作制御** | |
| ⭐ `-t` | `Ctrl+C` で中断されるまで、Pingを**無期限に (to infinity)** 送信し続けます。 |
| ⭐ `-n <count>` | エコー要求を送信する**回数 (number)** を指定します。デフォルトは4回です。 |
| `-l <size>` | 送信するパケットのデータフィールドの**サイズ (length)** をバイト単位で指定します。 |
| ⭐ `-w <timeout>` | 各応答を待つ**タイムアウト時間 (wait)** をミリ秒単位で指定します。 |
| **名前解決** | |
| `-a` | 宛先のIPアドレスからホスト名への**名前解決 (address)** を試みます。 |
| **IPバージョン** | |
| `-4` | IPv4の使用を強制します。 |
| `-6` | IPv6の使用を強制します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーやネットワーク機器の死活監視を継続的に行う。
* **組み合わせ**: `-t`
* **解説**: ネットワークの変更作業中に、 `-t` を使って重要なサーバーへの接続性を継続的に監視します。応答が途絶えれば、変更が原因で問題が発生したと即座に判断できます。
* **例**:

    ```cmd
    REM ゲートウェイへの接続を継続的に監視
    ping 192.168.1.1 -t
    ```

### 2. ブルーチーム視点

* **タスク**: 既知の悪意のあるIPが、現在も活動中（オンライン）であるかを確認する。
* **組み合わせ**: `-n`
* **解説**: 脅威インテリジェンス情報から得られたマルウェアのC2サーバーのIPアドレスリストに対し、`-n 1` で一度だけ `ping` を実行して疎通を確認します。ただし、応答がないからといって活動していないとは限りません（ICMPをブロックしている可能性があるため）。
* **例**:

    ```cmd
    REM C2サーバーの疑いがあるIPに疎通確認
    ping -n 2 198.51.100.10
    ```

### 3. レッドチーム視点

* **タスク**: 侵入したネットワーク内の生存ホストを探索（Pingスイープ）する。
* **組み合わせ**: `for /L %i IN (1,1,254) DO ping -n 1 192.168.1.%i`
* **解説**: 上記「よく連携されるコマンド」のシナリオ例を参照。`ping` は、ネットワーク上のアクティブなホストを発見するための最も基本的な手段です。

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

* `'ping'` is not recognized... は、[CMD共通のトラブルシューティング](./troubleshooting_common_errors.md)を参照。

1. **エラーメッセージ**: `Request timed out.`
    * **考えられる原因**:
        * 宛先ホストがシャットダウンしている。
        * 途中または宛先の**ファイアウォールがICMPパケットをブロックしている**。（非常によくある）
        * ネットワーク経路に問題がある。
    * **解決策**: 宛先ホストがICMPに応답しないように設定されている可能性を考慮します。PowerShellの `Test-NetConnection` など、別のツールでTCPポートへの接続を試みます。

2. **エラーメッセージ**: `Destination host unreachable.`
    * **考えられる原因**: 自分からゲートウェイまでの経路上にあるルーターが、宛先への経路を知らないと判断しています。
    * **解決策**: `ipconfig` で自身のIP設定を確認し、`route print` でルーティングテーブルを確認する必要があります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ping` 自体ではなく、それが利用するICMPプロトコルが悪用されることがあります。
* **悪用シナリオ**:
  * **偵察**: `ping` スイープは、ネットワーク上のアクティブなホストを発見するための最も基本的な手段です。
  * **サービス妨害攻撃 (DoS)**: 大量のICMPパケットを送りつける「Pingフラッド攻撃」や、巨大なパケットを送りつける「Ping of Death」は、古典的なDoS攻撃の一種です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: **偵察 (Reconnaissance)**, **データ窃取 (Data Exfiltration)**
* **解説**: `ping.exe` は、生存ホストの探索だけでなく、ICMPパケットのデータ部分に少量のデータを埋め込んで外部に送信することで、隠密なデータ窃取にも利用される可能性があります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 境界ファイアウォールで、外部からの不要なICMPエコー要求をブロックまたはレート制限する。
* **Detection (検知)**: 内部ネットワークから外部への大量のICMPパケットや、単一ホストからの広範囲なPingスイープをネットワーク侵入検知システム(IDS/IPS)で検知する。

## 注意点・補足

* **応答がない ≠ オフライン**: ファイアウォールによってICMPがブロックされていることは非常に一般的です。`ping` の応答がないからといって、そのホストがダウンしていると結論付けるのは早計です。
