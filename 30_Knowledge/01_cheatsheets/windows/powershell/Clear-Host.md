# `Clear-Host` - コンソール画面をクリアする

## 概要

`Clear-Host` コマンドレットは、現在のPowerShellコンソール画面（ホスト）に表示されているすべてのテキストを消去します。スクロールバックバッファの内容もクリアされることがあります（ホストアプリケーションに依存）。これは、CMDの `cls` コマンドと機能的に同等です。画面を整理し、新しいコマンドの出力をクリーンな状態で見たい場合に使用されます。

このコマンドレットには、`cls` および `clear` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`cls` (CMD)** | CMDの画面クリアコマンド。PowerShellでは `Clear-Host` のデフォルトエイリアスとして `cls` が定義されているため、CMDと同じ感覚で使用できます。 |
| **`Remove-Item`** | `Remove-Item` はファイルや変数などのアイテムを削除します。`Clear-Host` は画面表示のみをクリアします。 |

## よく連携されるコマンドレット

### シナリオ例: スクリプトの開始時に画面を整理する (インフラ構築・運用視点)

* **目的**: ユーザー対話型のスクリプトを実行する前に、以前のコマンド出力を消去して、スクリプトの表示をクリーンな状態から始める。
* **連携コマンドレット**: `Write-Host`
* **解説**: スクリプトの冒頭で `Clear-Host` を実行することで、ユーザーはスクリプトの出力に集中できます。
* **コマンド例**:

    ```powershell
    # 画面をクリア
    Clear-Host

    # スクリプトの開始をユーザーに通知
    Write-Host "=================================="
    Write-Host "System Maintenance Script Started."
    Write-Host "=================================="
    ```

## パラメータ説明

`Clear-Host` コマンドレットには、一般的に使用されるパラメータはありません。

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 対話的な操作中に、画面がコマンド出力で乱雑になった際にリセットする。
* **組み合わせ**: `cls` (エイリアス)
* **解説**: 最もシンプルで一般的な使い方。コマンドをいくつか実行した後、画面をクリアにして次の作業に移ります。
* **例**:

    ```powershell
    # 画面をクリアする (最も一般的なエイリアス)
    cls
    ```

### 2. ブルーチーム視点

* **タスク**: ライブレスポンス中に、調査対象のホストでコマンドを実行する前に画面をクリアし、出力のスクリーンショットを撮りやすくする。
* **組み合わせ**: `Clear-Host`
* **解説**: 証拠保全の一環としてコマンド実行結果を記録する際に、不要な情報が画面に表示されないようにするため、各コマンドの実行前に画面をクリアします。
* **例**:

    ```powershell
    Clear-Host
    Get-Process | Where-Object { $_.ProcessName -like "*suspicious*" }
    # (スクリーンショットを撮影)
    ```

### 3. レッドチーム視点

* **タスク**: 対話的なシェルセッション中に、実行したコマンドの痕跡を（画面上から）隠蔽する。
* **組み合わせ**: `cls`
* **解説**: 攻撃者は、システム上で偵察コマンドなどを実行した後、`cls` を実行して画面をクリアにすることで、背後から管理者に見られた場合に何を実行していたかを分かりにくくします。ただし、これはコマンド履歴 (`Get-History`) やPowerShellのログ自体を消すものではありません。

## エラーメッセージとトラブルシューティング

* `Clear-Host` は現在のホストアプリケーションに画面クリアを要求するだけなので、エラーが発生することはほとんどありません。

## セキュリティに関する考慮事項

### 悪用事例

* **画面上の痕跡隠蔽**: レッドチームのシナリオで述べた通り、攻撃者はショルダーサーフィン（覗き見）対策として、実行したコマンドの出力を画面から消去するためにこのコマンドを使用します。

### LOLBASにおける利用例

* `Clear-Host` はPowerShellの基本的な機能であり、直接的な攻撃ペイロードとして使われることはありませんが、攻撃者が対話的に活動する際の補助的なコマンドとして利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: **PowerShellトランスクリプションログ**や**スクリプトブロックロギング (イベントID 4104)** を有効化することが最も重要です。これにより、`Clear-Host` が実行されて画面から出力が消去されても、実行された全てのコマンドと（多くの場合）その出力がログとして記録されます。

## 注意点・補足

* `Clear-Host` の実際の動作（スクロールバックバッファをクリアするかどうかなど）は、使用しているホストアプリケーション（PowerShellコンソール、Windows Terminal, VS Codeのターミナルなど）に依存します。
