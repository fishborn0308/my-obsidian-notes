# `Remove-Item` - アイテムを削除する

## 概要

`Remove-Item` コマンドレットは、ファイル、ディレクトリ、レジストリキー、変数、関数など、さまざまなPowerShellプロバイダーのアイテムを削除します。CMDの `del` と `rd /s` の両方の機能を兼ね備えた、非常に強力で破壊的なコマンドです。

削除されたアイテムはごみ箱には移動せず、通常の方法では復元できません。そのため、`-WhatIf` パラメータを使った事前確認が強く推奨されます。

このコマンドレットには、`ri`, `rm`, `rmdir`, `del`, `erase`, `rd` という多数の便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`del` / `rd` (CMD)** | CMDのコマンドはファイルとディレクトリでコマンドが分かれています。`Remove-Item` は `-Recurse` を付けるだけで、ファイルと中身の入ったディレクトリの両方を統一的に削除できます。 |
| **`Clear-Content`** | `Clear-Content` はファイルの**内容のみを削除**し、ファイル自体は空の状態で残します。`Remove-Item` はファイル**自体を削除**します。 |

## よく連携されるコマンドレット

### シナリオ例: 古いログファイルを検索して削除する (インフラ構築・運用視点)

* **目的**: ログディレクトリ内にある、最終更新日時が30日以上前の `.log` ファイルを全て検索し、一括で削除する。
* **連携コマンドレット**: `Get-ChildItem`, `Where-Object`
* **解説**: `Get-ChildItem` で全てのログファイルオブジェクトを取得し、パイプラインで `Where-Object` に渡して日付の条件でフィルタリングします。最後に、条件に一致したファイルオブジェクトを `Remove-Item` に渡して削除します。
* **コマンド例**:

    ```powershell
    # C:\Logs 以下の .log ファイルを検索し、30日以上前のものを削除（-WhatIfで事前確認）
    Get-ChildItem -Path "C:\Logs" -Filter "*.log" -Recurse | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } | Remove-Item -Force -WhatIf
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Path`** | `String[]` | 削除するアイテムのパスを指定します。ワイルドカードが使用できます。 |
| ⭐ **`-Recurse`** | `SwitchParameter` | ディレクトリなどのコンテナーアイテムを削除する際に、その中に含まれる全ての子アイテムも再帰的に削除します。 |
| ⭐ **`-Force`** | `SwitchParameter` | 読み取り専用や隠しファイルなど、通常は削除できないアイテムを強制的に削除します。確認プロンプトも抑制します。 |
| **`-LiteralPath`** | `String[]` | ワイルドカードを解釈せず、パスを文字通りに受け取ります。 |
| ⭐ **`-WhatIf`** | `SwitchParameter` | コマンドを実際には実行せず、何が起こるかを表示します。破壊的な操作の前に安全確認のために必須です。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スクリプトの実行後に作成された一時ディレクトリを、中身ごと完全にクリーンアップする。
* **組み合わせ**: `Remove-Item -Path <path> -Recurse -Force`
* **解説**: `-Recurse` でディレクトリ内の全アイテムを、`-Force` で読み取り専用ファイルなども含めて、確認なしで強制的に削除します。
* **例**:

    ```powershell
    # 'C:\Temp\MyWorkDir' ディレクトリとその中身を全て削除
    Remove-Item -Path "C:\Temp\MyWorkDir" -Recurse -Force
    ```

### 2. ブルーチーム視点

* **タスク**: **インシデント対応における修復 (Remediation)**。証拠保全が完了したマルウェアや、攻撃者が作成したファイルをシステムから除去する。
* **組み合わせ**: `Remove-Item -Path <malware_path> -Force`
* **解説**: マルウェアはしばしば隠し属性や読み取り専用属性を持っているため、`-Force` スイッチを使って確実に削除します。
* **例**:

    ```powershell
    # 証拠保全後の不審なファイルを削除
    Remove-Item -Path "C:\Users\Public\payload.exe" -Force
    ```

### 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)** または **破壊活動 (Impact)**。
* **組み合わせ**: `Remove-Item -Path <path> -Recurse -Force`
* **解説**:
  * **痕跡消去**: 攻撃者は、自身の活動のためにダウンロードしたツールや、生成したログ、作成したバックドアアカウントなどを、活動終了時に `Remove-Item` で削除し調査を困難にします。
  * **破壊活動**: ランサムウェアやワイパー型マルウェアは、`Remove-Item` を使ってユーザーの重要なファイルやシステムファイルを削除し、システムを破壊します。
* **例**:

    ```powershell
    # 攻撃ツールキットを格納していたディレクトリを痕跡消去のために削除
    Remove-Item -Path "$env:TEMP\mytools" -Recurse -Force
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Remove-Item: Cannot remove item ... because it is in use.`
    * **考えられる原因**: 削除しようとしているファイルが、別のプロセスによって開かれている（ロックされている）ため、削除できません。
    * **解決策**: Process Explorer (Sysinternals) などのツールで、どのプロセスがファイルをロックしているかを確認し、そのプロセスを終了させてから再試行します。

## セキュリティに関する考慮事項

### 悪用事例

* **痕跡消去とデータ破壊**: レッドチームのシナリオで述べた通り、`Remove-Item` はランサムウェアやワイパー型マルウェアがデータを破壊したり、攻撃者が自身の活動の痕跡を消去したりするための最も直接的な手段です。

### LOLBASにおける利用例

* **機能**: **防御回避 (Defense Evasion)**, **影響 (Impact)**
* **解説**: `Remove-Item` は PowerShell の標準機能であり、攻撃者が検知されにくい形でファイルやログを削除するために利用されます（Living off the Land）。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要なファイルやディレクトリに適切なACL（アクセス制御リスト）を設定し、最小権限の原則を徹底する。定期的なオフラインバックアップを取得する。
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化し、`Remove-Item` の実行、特に `-Recurse` や `-Force` を伴う広範囲な削除操作を監視する。`Sysmon` (イベントID 23: FileDelete) でファイルの削除イベントを監視する。

## 注意点・補足

* **`-WhatIf` の重要性**: `Remove-Item -Recurse` は非常に強力です。意図しないファイルを削除する大惨事を防ぐため、**本番環境で実行する前には、必ず `-WhatIf` スイッチを付けてコマンドを実行し、どのアイテムが削除対象になるかを正確に確認してください。**
