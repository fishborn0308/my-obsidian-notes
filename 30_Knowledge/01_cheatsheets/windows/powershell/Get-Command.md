# `Get-Command` - コマンドを取得する

## 概要

`Get-Command` コマンドレットは、PowerShellセッションにインストールおよびインポートされている全てのコマンドに関する情報を取得します。これには、コマンドレット、関数、エイリアス、スクリプト、 `Path` 環境変数に含まれる実行可能ファイルが含まれます。

特定の名前のコマンドを検索したり、名詞 (`Noun`) や動詞 (`Verb`) に基づいてコマンドを発見したり、特定のモジュールに含まれるコマンドを一覧表示したりするために使用されます。PowerShellで「どのようなコマンドが使えるのか？」を発見するための基本的な探索ツールです。

このコマンドレットには、`gcm` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`Get-Alias`** | `Get-Alias` は**エイリアスのみ**を検索します。`Get-Command` はエイリアスを含む**全ての種類**のコマンドを検索します。 |
| **`Get-Help`** | `Get-Help` はコマンドの**使い方（ヘルプドキュメント）**を表示します。`Get-Command` は**コマンド自体の存在とその情報（型、モジュールなど）**を取得します。 |

## よく連携されるコマンドレット

### シナリオ例: 特定の機能に関連するコマンドを探し、その使い方を調べる (インフラ構築・運用視点)

* **目的**: "Process"（プロセス）に関連する全てのコマンドレットを探し出し、その中の1つ (`Stop-Process`) の詳細なヘルプを表示する。
* **連携コマンドレット**: `Get-Help`
* **解説**: `Get-Command` の `-Noun` パラメータを使って、特定の対象を操作するコマンドレット群を簡単に見つけることができます。見つけたコマンドレットの具体的な使い方を調べるために、その結果を `Get-Help` に渡します。
* **コマンド例**:

    ```powershell
    # 'Process' という名詞を持つ全てのコマンドを検索
    Get-Command -Noun "Process"

    # 'Stop-Process' コマンドレットの詳細なヘルプを表示
    Get-Help -Name "Stop-Process" -Full
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Name`** | `String[]` | 取得するコマンドの名前を指定します。ワイルドカード (`*`) が使用できます。 |
| ⭐ **`-Verb`** | `String[]` | コマンドレットの動詞部分 (`Get`, `Set`, `New`など) を指定して検索します。 |
| ⭐ **`-Noun`** | `String[]` | コマンドレットの名詞部分 (`Process`, `Service`, `Item`など) を指定して検索します。 |
| **`-Module`** | `String[]` | 指定したモジュールに含まれるコマンドを検索します。 |
| **`-CommandType`** | `CommandTypes` | 検索するコマンドの種類を指定します (`Cmdlet`, `Function`, `Alias`, `Application`など)。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しくインストールしたモジュール (`Pester` など) に、どのようなコマンドレットが含まれているかを確認する。
* **組み合わせ**: `Get-Command -Module <ModuleName>`
* **解説**: 新しいモジュールを使い始める際に、このコマンドで利用可能なコマンドレットを一覧表示し、機能の全体像を把握します。
* **例**:

    ```powershell
    # 'Pester' モジュールに含まれる全てのコマンドレットを一覧表示
    Get-Command -Module Pester
    ```

### 2. ブルーチーム視点

* **タスク**: **環境の監査**。現在のセッションで利用可能な全てのコマンドをリストアップし、不審な、あるいは意図せずインポートされたコマンドがないかを確認する。
* **組み合わせ**: `Get-Command`
* **解説**: `Get-Command` の出力をベースラインと比較することで、攻撃者によって追加された悪意のある関数や、意図せずロードされたモジュールを発見する手がかりとなります。
* **例**:

    ```powershell
    # 現在のセッションの全てのコマンドレットと関数をリストアップして監査
    Get-Command -CommandType Cmdlet, Function | Sort-Object Source, Name
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストで利用可能なコマンドを探索し、「環境寄生」に使えるツールを探す。
* **組み合わせ**: `Get-Command`
* **解説**: 攻撃者は侵入後、`Get-Command` を使って、システムにどのようなコマンドやモジュールがインストールされているかを調査します。`ActiveDirectory` モジュール、`NetTCPIP` モジュール、あるいは `Invoke-WebRequest` のような通信機能を持つコマンドレットの存在は、次の攻撃計画を立てる上で重要な情報となります。
* **例**:

    ```powershell
    # 'Invoke' という動詞を持つコマンド (リモート実行やWebリクエストに使われることが多い) を検索
    Get-Command -Verb "Invoke"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Get-Command: The term '...' is not recognized as the name of a cmdlet, function, script file, or operable program.`
    * **考えられる原因**: 指定した名前のコマンドが存在しないか、そのコマンドが含まれるモジュールがインポートされていません。
    * **解決策**: コマンド名が正しいか確認します。必要なモジュールがインストールされているか `Get-Module -ListAvailable` で確認し、`Import-Module <ModuleName>` でインポートしてください。

## セキュリティに関する考慮事項

### 悪用事例

* **偵察 (Reconnaissance)**: レッドチームのシナリオで述べた通り、`Get-Command` は攻撃者が侵入先システムで利用可能なツールや機能を把握するための最も基本的な偵察手段です。これにより、どのような「環境寄生型攻撃 (Living off the Land)」が可能かを判断します。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `Get-Command` は PowerShell の標準機能であり、攻撃者が侵入後の情報収集フェーズで、システムの能力を把握するために利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **JEA (Just Enough Administration)** を構成し、リモートセッションでユーザーが実行できるコマンドレットを必要最小限に制限する。
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化し、`Get-Command` の実行、特に他の偵察コマンド (`Get-Process`, `Get-CimInstance`など) と連続して実行されるような挙動を監視する。

## 注意点・補足

* `Get-Command` は、現在のセッションに**インポートされている**モジュールのコマンドのみを表示するのがデフォルトの挙動です。インストールされているがまだインポートされていないモジュールのコマンドも検索するには、`Get-Module -ListAvailable | Import-Module` のように事前にモジュールをインポートしておく必要があります。
