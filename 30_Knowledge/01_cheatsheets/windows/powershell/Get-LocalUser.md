# `Get-LocalUser` - ローカルユーザーを取得する

## 概要

`Get-LocalUser` コマンドレットは、ローカルのセキュリティアカウントマネージャー (SAM) データベースに存在するユーザーアカウントを取得します。このコマンドレットは、特定のユーザーを名前やSIDで検索したり、システムに存在する全てのユーザーアカウントを一覧表示したりするために使用されます。

このコマンドレットは、`Microsoft.PowerShell.LocalAccounts` モジュールに含まれており、スタンドアロンサーバーやワークステーションのユーザーアカウント監査における基本的な役割を果たします。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`net user` (CMD)** | `net user` はユーザーアカウントの基本情報を**テキスト**として表示します。`Get-LocalUser` は、`Enabled`, `LastLogon`, `SID` などの豊富なプロパティを持つ**オブジェクト**を返すため、パイプラインでのフィルタリングやソートといったプログラム的な処理が非常に容易です。 |
| **`Get-ADUser`** | `Get-ADUser` は **Active Directory ドメイン**上のユーザーを取得します。`Get-LocalUser` は**ローカルコンピュータ**上のユーザーを取得します。 |

## よく連携されるコマンドレット

### シナリオ例: 無効化されている全てのローカルユーザーアカウントを検索する (インフラ構築・運用視点)

* **目的**: 棚卸しのために、現在システムで無効化 (`Enabled -eq $false`) されている全てのアカウントをリストアップする。
* **連携コマンドレット**: `Where-Object`
* **解説**: `Get-LocalUser` ですべてのローカルユーザーオブジェクトを取得し、その結果をパイプラインで `Where-Object` に渡します。`Enabled` プロパティが `false` であるユーザーのみをフィルタリングします。
* **コマンド例**:

    ```powershell
    # 無効化されている全てのローカルユーザーを検索
    Get-LocalUser | Where-Object { -not $_.Enabled }
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Name`** | `String[]` | 取得するユーザーアカウントの名前を指定します。ワイルドカード (`*`) が使用できます。 |
| **`-SID`** | `SecurityIdentifier` | 取得するユーザーアカウントのセキュリティ識別子 (SID) を指定します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システムに存在するビルトインアカウント以外のユーザーアカウントを全てリストアップする。
* **組み合わせ**: `Get-LocalUser | Where-Object { $_.PrincipalSource -eq 'Local' }`
* **解説**: `PrincipalSource` プロパティを確認することで、アカウントの種類（ローカル、Active Directoryなど）を判別できます。これにより、手動で作成されたアカウントのみを監査対象とできます。
* **例**:

    ```powershell
    # ローカルで作成されたアカウントのみを表示
    Get-LocalUser | Where-Object { $_.PrincipalSource -eq 'Local' } | Select-Object Name, Enabled, LastLogon
    ```

### 2. ブルーチーム視点

* **タスク**: **不正アカウントの監査**。意図せず作成された、あるいは不審なユーザーアカウントが存在しないかを確認する。
* **組み合わせ**: `Get-LocalUser`
* **解説**: **インシデント対応の基本**。`Get-LocalUser` を実行して、システム上の全アカウントをリストアップし、ベースラインと比較して見慣れないアカウントがないかを確認します。特に、`LastLogon` がない新しいアカウントや、`Description` が不審なアカウントは注意深く調査する必要があります。
* **例**:

    ```powershell
    # 全てのローカルユーザーとそのプロパティをリスト形式で詳細に表示
    Get-LocalUser | Format-List *
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。侵入したホストのローカルユーザーアカウントを列挙し、権限昇格やラテラルムーブメントの足がかりを探す。
* **組み合わせ**: `Get-LocalUser`
* **解説**: 攻撃者は侵入後、`net user` と同様に `Get-LocalUser` を実行して、そのマシンに存在するアカウントを特定します。無効化されているがパスワードが推測しやすいアカウント、特権グループに所属していそうなアカウント、あるいは他のシステムでも使われていそうなアカウント名を探します。
* **例**:

    ```powershell
    # 全てのローカルユーザー名と、アカウントが無効化されているかどうかのリストを取得
    Get-LocalUser | Select-Object Name, Enabled
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Get-LocalUser : User <username> was not found.`
    * **考えられる原因**: 指定した `-Name` のユーザーアカウントが存在しません。
    * **解決策**: ユーザー名のスペルが正しいか確認してください。引数を指定せずに `Get-LocalUser` を実行して、存在するユーザーの一覧を確認するのも有効です。

## セキュリティに関する考慮事項

### 悪用事例

* **偵察 (Reconnaissance)**: レッドチームのシナリオで述べた通り、`Get-LocalUser` は攻撃者が侵入したシステムのユーザー構成を理解し、次の攻撃ターゲット（権限昇格やパスワード推測の対象）を特定するための基本的な偵察ツールです。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `Get-LocalUser` は PowerShell の標準機能であり、`powershell.exe` を通じて実行されます。攻撃者はこの正規の管理機能を利用して、環境に溶け込みながらシステムのアカウント情報を収集します（Living off the Land）。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、不要なローカルアカウントは作成しない、または無効化する。
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化し、`Get-LocalUser` の実行を記録・監視する。特に、他の偵察コマンド (`Get-LocalGroup`, `Get-NetTCPConnection`など) と連続して実行される場合は、攻撃の兆候である可能性があります。

## 注意点・補足

* このコマンドレットは、`Microsoft.PowerShell.LocalAccounts` モジュールの一部です。このモジュールは、Windows 10、Windows Server 2016以降のバージョンにデフォルトで含まれています。
