# `Get-Item` - 指定した場所のアイテムを取得する

## 概要

`Get-Item` コマンドレットは、指定されたパスにあるアイテムを取得します。このコマンドレットは、アイテムの**内容** (`Get-Content`) や、コンテナー内の**子アイテム** (`Get-ChildItem`) を取得するのではなく、アイテム**そのもの**のプロパティ（名前、作成日時、属性など）を持つオブジェクトを返します。

ファイルやディレクトリの存在確認や、そのメタデータ（プロパティ）をプログラム的に扱いたい場合に使用されます。

このコマンドレットには、`gi` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`Get-ChildItem`** | `Get-ChildItem` は、指定した場所の**中にあるアイテム（子アイテム）**のリストを取得します。`Get-Item` は、指定した場所の**アイテム自体**を一つ取得します。 |
| **`Get-Content`** | `Get-Content` は、ファイルの**中身（テキストなど）**を読み取ります。`Get-Item` は、ファイル**そのもの（プロパティ）**を取得します。 |
| **`Test-Path`** | `Test-Path` は、パスが存在するかどうかを**真偽値 (`$true`/`$false`)** で返します。`Get-Item` は、パスが存在する場合にその**オブジェクト**を返します（存在しない場合はエラー）。条件分岐で存在確認だけしたい場合は `Test-Path` が便利です。 |

## よく連携されるコマンドレット

### シナリオ例: ファイルの属性と更新日時を確認する (インフラ構築・運用視点)

* **目的**: 特定の構成ファイル (`web.config`) のプロパティを取得し、読み取り専用属性が付いているか、いつ最終更新されたかを確認する。
* **連携コマンドレット**: `Select-Object`
* **解説**: `Get-Item` で取得したファイルオブジェクトをパイプラインで `Select-Object` に渡し、`Name`, `IsReadOnly`, `LastWriteTime` といった特定のプロパティだけを抜き出して表示します。
* **コマンド例**:

    ```powershell
    # 'web.config' ファイルのプロパティを取得し、特定の情報だけを表示
    Get-Item -Path "C:\inetpub\wwwroot\web.config" | Select-Object -Property Name, IsReadOnly, LastWriteTime
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Path`** | `String[]` | 情報を取得するアイテムのパスを指定します。ワイルドカードが使用できます。 |
| **`-LiteralPath`** | `String[]` | ワイルドカードを解釈せず、パスを文字通りに受け取ります。`[` や `]` など、ワイルドカードとして解釈されうる文字を含むパスを扱う際に使用します。 |
| ⭐ **`-Force`** | `SwitchParameter` | 隠しファイルやシステムファイルなど、通常はアクセスできないアイテムを取得します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スクリプト内で、特定のレジストリキーが存在するかどうかを確認する。
* **組み合わせ**: `Get-Item -Path "HKLM:\..."`
* **解説**: ファイルシステムだけでなく、`HKLM:` や `HKCU:` のようなレジストリプロバイダーに対しても使用できます。これにより、ソフトウェアのインストール状況や設定値をスクリプトで確認できます。
* **例**:

    ```powershell
    # 特定の .NET Framework バージョンがインストールされているかを示すレジストリキーを確認
    Get-Item -Path "HKLM:\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full"
    ```

### 2. ブルーチーム視点

* **タスク**: **タイムライン分析**。マルウェアの疑いがあるファイルの作成日時、最終アクセス日時、最終更新日時を正確に記録する。
* **組み合わせ**: `Get-Item <file> | Select-Object *`
* **解説**: `Get-Item` で取得したオブジェクトには、`CreationTime`, `LastAccessTime`, `LastWriteTime` といった詳細なタイムスタンプ情報が含まれています。これらは、攻撃者がいつシステムにファイルを配置し、いつアクセスしたかを特定するための重要な証拠となります。
* **例**:

    ```powershell
    # 不審なファイルの全てのタイムスタンプ情報を取得
    Get-Item -Path C:\Users\Public\suspicious.exe -Force | Select-Object -Property *Time*
    ```

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。特定のファイルや設定が存在するかどうかを、エラーを出さずに確認する。
* **組み合わせ**: `Get-Item -Path <Path> -ErrorAction SilentlyContinue`
* **解説**: 攻撃者は侵入後、`Get-Item` (またはエイリアスの `gi`) を使って、特定のアプリケーションの設定ファイル、バックアップ、あるいは既知の脆弱性を持つDLLなどが存在するかを調査します。`-ErrorAction SilentlyContinue` を付けることで、ファイルが存在しない場合にエラーメッセージが表示されるのを防ぎ、偵察活動をより隠密にします。
* **例**:

    ```powershell
    # Unattend.xml ファイル (自動応答ファイルで、パスワードが含まれている可能性がある) の存在を確認
    Get-Item -Path "C:\Windows\Panther\unattend.xml" -ErrorAction SilentlyContinue
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Get-Item: Cannot find path '...' because it does not exist.`
    * **考えられる原因**: 指定した `-Path` が存在しません。
    * **解決策**: パスが正しいか確認してください。存在確認のみが目的であれば、`Test-Path` を使う方がエラーハンドリングが容易です。

## セキュリティに関する考慮事項

### 悪用事例

* **偵察 (Reconnaissance)**: レッドチームのシナリオで述べた通り、`Get-Item` は攻撃者がシステムの設定状況、インストールされているソフトウェア、そして機密情報を含む可能性のあるファイルの存在を確認するための基本的な偵察ツールです。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `Get-Item` は PowerShell の標準機能であり、攻撃者が環境に溶け込みながらファイルシステムの情報を収集するために利用されます（Living off the Land）。

### 対応策・緩和策 (ブルーチーム視-

**Prevention (予防)**: -

* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化し、`Get-Item` の実行、特に機密性の高いファイルやレジストリキーに対するアクセスを監視する。他の偵察コマンドと連続して実行される場合は、攻撃の兆候である可能性があります。

## 注意点・補足

* **オブジェクトのプロパティ**: `Get-Item` が返すオブジェクトがどのようなプロパティを持っているかを知りたい場合は、`Get-Item <path> | Get-Member` のようにパイプラインで `Get-Member` に渡すと、全てのプロパティとメソッドの一覧を確認できます。
