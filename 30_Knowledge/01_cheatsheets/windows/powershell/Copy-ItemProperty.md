# `Copy-ItemProperty` - アイテムのプロパティをコピーする

## 概要

`Copy-ItemProperty` コマンドレットは、指定したアイテムのプロパティを、別の指定したアイテムにコピーします。アイテムそのもの (`Copy-Item`) ではなく、そのアイテムが持つプロパティ（ファイルシステムでは属性やタイムスタンプ、レジストリでは値など）をコピーするのが特徴です。

このコマンドレットが最も強力に機能するのは**レジストリ操作**であり、あるレジストリキーが持つ値を、別のキーにそっくりそのまま複製する際などに利用されます。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`Copy-Item`** | `Copy-Item` はアイテム**全体**（ファイル、フォルダ、レジストリキー）をコピーします。`Copy-ItemProperty` はアイテムが持つ**プロパティ（レジストリ値など）のみ**をコピーします。 |
| **`Set-ItemProperty`** | `Set-ItemProperty` はプロパティの値を**特定の値に設定**します。`Copy-ItemProperty` は、別の場所からプロパティとその値を**コピー**してきます。 |

## よく連携されるコマンドレット

### シナリオ例: レジストリキーの設定値をバックアップ・複製する (インフラ構築・運用視点)

* **目的**: アプリケーションの設定が格納されているレジストリキーの全ての値を、バックアップ用の新しいキーに複製する。
* **連携コマンドレット**: `New-Item`
* **解説**: まず `New-Item` でバックアップ先のレジストリキーを作成します。次に、`Copy-ItemProperty` を使って、元のキーから新しいキーへ、全てのプロパティ (`-Name *`) をコピーします。
* **コマンド例**:

    ```powershell
    # 元の設定キーのパス
    $sourcePath = "HKCU:\Software\MyApp\Settings"
    # バックアップ先のキーのパス
    $destinationPath = "HKCU:\Software\MyApp\Settings_Backup"

    # バックアップ先のキーを新規作成
    New-Item -Path $destinationPath -Force

    # 元のキーから全てのプロパティ(値)をバックアップ先にコピー
    Copy-ItemProperty -Path $sourcePath -Destination $destinationPath -Name *
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-Path`** | `String[]` | プロパティのコピー**元**となるアイテムのパスを指定します。 |
| ⭐ **`-Destination`** | `String` | プロパティのコピー**先**となるアイテムのパスを指定します。 |
| ⭐ **`-Name`** | `String[]` | コピーするプロパティの名前を指定します。ワイルドカード (`*`) を使うと全てのプロパティが対象になります。 |
| **`-Force`** | `SwitchParameter` | 読み取り専用のコピー先など、制限を強制的に上書きします。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: あるユーザーのソフトウェア設定（レジストリに保存されている）を、別のユーザーの設定に複製する。
* **組み合わせ**: `Copy-ItemProperty`
* **解説**: ソフトウェアの移行や複数ユーザーへの設定展開時に、GUIで1つずつ設定する代わりに、`Copy-ItemProperty` を使ってレジストリ値を直接コピーすることで、作業を自動化・効率化します。
* **例**:

    ```powershell
    # UserA のエディタ設定を UserB にコピー
    Copy-ItemProperty -Path "HKU:\S-1-5-21-...\Software\MyEditor" -Destination "HKU:\S-1-5-21-...\Software\MyEditor" -Name *
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによって変更された疑いのあるレジストリキーの値を、分析のためにバックアップする。
* **組み合わせ**: `Copy-ItemProperty -Path <source> -Destination <dest> -Name *`
* **解説**: インシデント対応中に、変更されたレジストリキー（例: `Run`キー）を修正する前に、現在の状態を別のキーにバックアップとしてコピーします。これにより、元の状態を証拠として保全しつつ、システムの修復作業を進めることができます。
* **例**:

    ```powershell
    # 変更された疑いのあるRunキーの値をバックアップ
    Copy-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run" -Destination "HKCU:\Software\Investigation\Run_Backup" -Name *
    ```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** のため、COMハイジャックを行う。
* **組み合わせ**: `Copy-ItemProperty`
* **解説**: 攻撃者は、正規のCOMサーバーのレジストリ設定を、自身が制御する別の場所にコピーします。その後、コピーしたキーのプロパティ（実行ファイルのパスなど）を悪意のあるペイロードに書き換えることで、正規のアプリケーションがCOMオブジェクトを呼び出した際に、代わりに悪意のあるコードが実行されるように仕向けます。

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1. **エラーメッセージ**: `Copy-ItemProperty: Access to the registry key '...' is denied.`
    * **考えられる原因**: レジストリキーの読み取りまたは書き込みに必要な権限がありません。特に `HKEY_LOCAL_MACHINE` (HKLM) 配下の操作には管理者権限が必須です。
    * **解決策**: PowerShellを「管理者として実行」して、再度コマンドを実行してください。

## セキュリティに関する考慮事項

### 悪用事例

* **レジストリ操作による永続化・防御回避**: レッドチームのシナリオで述べた通り、攻撃者は `Copy-ItemProperty` や `Set-ItemProperty` を使って、システムの動作を制御するレジストリ値を改ざんします。これにより、永続化、UACバイパス、セキュリティソフトの無効化など、様々な悪意のある活動を行います。

### LOLBASにおける利用例

* **機能**: **防御回避 (Defense Evasion)**
* **解説**: `Copy-ItemProperty` は PowerShell の標準機能であり、`powershell.exe` を通じて実行されます。攻撃者はこの正規の機能を利用して、検知されにくい形でレジストリを操作し、システムのセキュリティ設定を弱体化させたり、永続化の足がかりを築いたりします。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を徹底し、一般ユーザーに重要なレジストリキーへの書き込み権限を与えない。
* **Detection (検知)**: `Sysmon` を導入し、レジストリキーの値の作成・変更・削除 (イベントID 13) を厳重に監視する。特に、`Run` キーやCOM関連のキーなど、永続化に利用されやすいレジストリパスへの変更は高優先度のアラート対象とすべきです。PowerShellのスクリプトブロックロギングも有効です。

## 注意点・補足

* `Copy-ItemProperty` は、主にレジストリプロバイダーでその真価を発揮します。ファイルシステムプロバイダーでも使用できますが（例: ファイルのタイムスタンプのコピー）、その用途は限定的です。
