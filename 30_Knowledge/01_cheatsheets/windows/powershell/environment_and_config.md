# PowerShell 環境変数と設定

このドキュメントでは、PowerShell環境における主要な環境変数と設定について解説します。

## 1. 主要な環境変数

CMDの環境変数に加えて、PowerShell特有の環境変数や、PowerShellスクリプトでよく利用される変数があります。

| 環境変数名 | 説明 | セキュリティに関する考慮事項 |
| :--- | :--- | :--- |
| `Path` | コマンドレットや外部コマンドが実行された際に、PowerShellが実行可能ファイルを探すディレクトリのリスト。`;` で区切られます。 | CMDと同様に、**不審なパスの追加**は危険です。`$env:Path` で確認・設定できます。 |
| `PSModulePath` | PowerShellモジュールが格納されているディレクトリのパス。`Get-Module`などでモジュールを探す際に参照されます。 | 攻撃者がカスタムモジュールを注入し、悪意のあるコードを実行するために悪用される可能性があります。信頼できないソースからのモジュールをインストールしない、または`PSModulePath`を適切に制御すべきです。 |
| `PSHome` | PowerShellのインストールディレクトリのパス。 | PowerShellのシステム関連ファイルが格納されています。このパスへの不適切なアクセスはシステムを不安定にしたり、セキュリティを侵害したりする可能性があります。 |
| `PSVersionTable` | PowerShellのバージョン、ビルド情報、CLRバージョンなど、PowerShell環境に関する詳細情報を含むハッシュテーブル。 | システム偵察時にPowerShellのバージョン情報を取得するために利用されます。古いバージョンのPowerShellには既知の脆弱性が存在する可能性があるため、常に最新の状態に保つべきです。 |
| `ExecutionPolicy` | PowerShellスクリプトの実行ポリシー。スクリプトの実行を許可するかどうかを制御します。 (`Restricted`, `AllSigned`, `RemoteSigned`, `Unrestricted`, `Bypass`, `Undefined`) | **セキュリティの主要設定**: `Unrestricted`や`Bypass`に設定すると、悪意のあるスクリプトも実行可能になります。通常は`RemoteSigned`(ダウンロードしたスクリプトは署名が必要)またはより厳格なポリシーを設定すべきです。攻撃者はこれを変更しようとすることがあります。 |
| `PWD` | 現在の作業ディレクトリの場所。`Get-Location`コマンドレットで取得できるオブジェクトです。 | スクリプト内でのパス操作によく使われます。 |

## 2. 主要な設定とファイル

Windowsのシステムレベルの設定ファイル(レジストリ、hostsファイル、イベントログなど)はCMDと共通です。ここではPowerShell特有のプロファイル設定について詳述します。

| 設定/ファイル名 | 説明 | セキュリティに関する考慮事項 |
| :--- | :--- | :--- |
| **PowerShell プロファイル** | PowerShellが起動するたびに自動的に実行されるスクリプトファイル。ユーザーやホスト、PowerShellのバージョンによって複数のプロファイルが存在します。最も一般的なのは`$PROFILE`変数で示されるユーザー固有のプロファイル(例: `Microsoft.PowerShell_profile.ps1`)。 | **永続化メカニズム**: 攻撃者がプロファイルファイルに悪意のあるPowerShellスクリプトやコマンドを埋め込み、PowerShellが起動するたびに実行させることがあります。これにより、永続化やバックドアとして機能する可能性があります。<br>**定期的な監査**: プロファイルファイルの内容を定期的に監査し、不審な記述がないか確認すべきです。<br>**ファイルパーミッション**: プロファイルファイルに適切なパーミッションを設定し、非特権ユーザーによる改ざんを防ぎます。 |
| `Get-ExecutionPolicy` | 現在のPowerShell実行ポリシーを表示するコマンドレット。 | 攻撃者はスクリプト実行を容易にするため、`Set-ExecutionPolicy`コマンドレットを使って実行ポリシーを変更しようとすることがあります。監査や監視の対象とすべきアクションです。 |
| `Set-ExecutionPolicy` | PowerShell実行ポリシーを変更するコマンドレット。 | **特権昇格/永続化の可能性**: セキュリティレベルを低下させる設定変更は、攻撃者にとって有利に働きます。このコマンドレットの実行は厳重に監視し、許可されたユーザーのみが実行できるように制御すべきです。 |
| **PowerShellモジュール** | 特定の機能やコマンドレットをまとめたパッケージ。`Get-Module`, `Import-Module`, `Install-Module`などで管理します。 | **悪意のあるモジュール**: 攻撃者が悪意のあるPowerShellモジュールを作成し、システムにインポートさせることで、さまざまな不正行為(情報窃取、特権昇格、横展開など)を試みることがあります。信頼できないソースからのモジュールのインストールは厳禁です。<br>**スクリプトブロックロギング**: PowerShellのスクリプトブロックロギングを有効にし、モジュール内のコード実行も監視できるようにすべきです。 |
| **イベントログ** (共通) | Windowsのシステム、セキュリティ、アプリケーションログはPowerShellからも`Get-WinEvent`などでアクセスできます。 | CMDの項と同様に、セキュリティ監視の最も重要な情報源です。PowerShellのアクティビティ(スクリプト実行など)も詳細に記録されます。 |
