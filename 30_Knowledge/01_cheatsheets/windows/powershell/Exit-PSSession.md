# `Exit-PSSession` - 対話的なリモートセッションを終了する

## 概要

`Exit-PSSession` コマンドレットは、`Enter-PSSession` を使って開始した対話的なリモートPowerShellセッションを終了します。このコマンドを実行すると、リモートコンピュータとの接続が閉じられ、コマンドプロンプトがローカルコンピュータのものに戻ります。

単純に `exit` キーワードを入力するだけでも同じ効果が得られます。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`exit` (キーワード)** | 対話的なリモートセッション内で `exit` を実行すると、`Exit-PSSession` と同じくセッションを終了します。`exit` の方が短く、一般的に使われます。 |
| **`Remove-PSSession`** | `Remove-PSSession` は、`New-PSSession` で作成されたバックグラウンドの持続的なセッションを**完全に削除**します。`Exit-PSSession` は、対話モードを**抜ける**だけです。`New-PSSession` で作成したセッションに入ってから `exit` しても、セッション自体はバックグラウンドに残り続けます。 |

## よく連携されるコマンドレット

### シナリオ例: リモートサーバーでの作業とセッションの終了 (インフラ構築・運用視点)

* **目的**: リモートサーバーに接続して作業を行い、完了後にセッションを正しく終了する。
* **連携コマンドレット**: `Enter-PSSession`
* **解説**: `Enter-PSSession` でリモートセッションを開始し、必要なコマンドを実行した後、`Exit-PSSession` または `exit` でローカルセッションに戻ります。
* **コマンド例**:

    ```powershell
    # リモートサーバー 'WEB-SRV-01' に対話セッションを開始
    Enter-PSSession -ComputerName "WEB-SRV-01"

    # --- リモートセッション内での作業 ---
    [WEB-SRV-01]: PS C:\> Get-Service -Name "w3svc"
    [WEB-SRV-01]: PS C:\> Restart-Service -Name "w3svc"
    # --- 作業完了 ---

    # 対話セッションを終了してローカルに戻る
    [WEB-SRV-01]: PS C:\> Exit-PSSession

    # ローカルのプロンプトに戻る
    PS C:\>
    ```

## パラメータ説明

`Exit-PSSession` コマンドレットには、一般的に使用されるパラメータはありません。

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートでの管理作業を完了し、セッションを閉じる。
* **組み合わせ**: `exit` (エイリアス的な使い方)
* **解説**: 最も基本的な使い方。リモートでの作業が完了したら、`exit` と入力してセッションを終了します。
* **例**:

    ```powershell
    [REMOTE-HOST]: PS C:\> exit
    ```

### 2. ブルーチーム視点

* **タスク**: ライブレスポンス調査が完了した後、侵害されたホストから安全に切断する。
* **組み合わせ**: `Exit-PSSession`
* **解説**: 調査コマンドの実行と証拠収集が完了した後、`Exit-PSSession` を使ってホストから切断します。これにより、意図しないコマンドを誤って実行するリスクを減らします。
* **例**: -

### 3. レッドチーム視点

* **タスク**: ラテラルムーブメント後の痕跡整理。
* **組み合わせ**: `Exit-PSSession`
* **解説**: 攻撃者がラテラルムーブメントのために `Enter-PSSession` を使用した後、目的のタスク（ペイロードの実行、情報の窃取など）が完了したら、`Exit-PSSession` でセッションを閉じます。開いたままのセッションは、ブルーチームに発見されるリスクを高めるため、速やかに閉じるのが一般的です。

## エラーメッセージとトラブルシューティング

* `Exit-PSSession` は、`Enter-PSSession` で開始されたセッション内で実行されるため、エラーが発生することは稀です。「この操作はローカルセッションでは実行できません」といったメッセージが表示された場合は、既に対話セッションを抜けていることを意味します。

## セキュリティに関する考慮事項

### 悪用事例

* **セッションのクリーンアップ**: レッドチームのシナリオで述べた通り、攻撃者は活動の痕跡を最小限に抑えるために、使用済みのリモートセッションを閉じる目的でこのコマンドレット（または `exit`）を使用します。

### LOLBASにおける利用例

* `Exit-PSSession` は PowerShell の標準機能であり、`Enter-PSSession` や `Invoke-Command` といったラテラルムーブメント手法の一部として、セッション管理のために利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: PowerShellのリモートセッションの開始と終了は、Windowsイベントログの **PowerShell Operational Log (イベントID 400, 403など)** や **Security Log (イベントID 4624, ログオンタイプ 3)** で追跡できます。セッションの開始と終了のペアを監視することで、不審なリモートアクセスを特定する手がかりとなります。

## 注意点・補足

* **`exit` vs `Exit-PSSession`**: どちらも対話セッションを終了させる点では同じですが、`Exit-PSSession` はコマンドレットであり、`exit` はPowerShellのキーワードです。スクリプト内で明示的にセッション終了の意図を示したい場合は `Exit-PSSession` を使うと可読性が向上することがあります。
