# `Get-Member` - オブジェクトのメンバーを取得する

## 概要

`Get-Member` コマンドレットは、PowerShellのオブジェクトが持つメンバー（プロパティ、メソッド、イベントなど）の種類、名前、定義を取得します。PowerShellでは、あらゆるコマンドの出力がオブジェクトであるため、`Get-Member` を使うことで、そのオブジェクトが「どのようなデータ（プロパティ）を持っているのか」「どのような操作（メソッド）が可能か」を調べることができます。

PowerShellのオブジェクト指向の性質を理解し、パイプラインを効果的に活用するために不可欠な探索・学習ツールです。

このコマンドレットには、`gm` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`Select-Object`** | `Select-Object` はオブジェクトから特定の**プロパティを選択して新しいオブジェクトを作成**します。`Get-Member` はオブジェクトが持つ**全てのメンバーの定義をリストアップ**します。オブジェクトの構造を知るのが`Get-Member`、知っている構造からデータを抜き出すのが`Select-Object`です。 |
| **`Get-Help`** | `Get-Help` は**コマンドレット**の使い方を表示します。`Get-Member` は、そのコマンドレットが**出力するオブジェクト**の構造を表示します。 |

## よく連携されるコマンドレット

### シナリオ例: コマンドレットの出力オブジェクトを調査する (インフラ構築・運用視点)

* **目的**: `Get-Process` コマンドレットが返すプロセスオブジェクトに、どのようなプロパティが含まれているかを調べる。
* **連携コマンドレット**: `Get-Process`
* **解説**: 任意のコマンドレットの出力をパイプラインで `Get-Member` に渡すことで、そのコマンドレットがどのような種類のオブジェクトを返すのか、そしてそのオブジェクトが持つプロパティやメソッドの一覧を簡単に確認できます。
* **コマンド例**:

    ```powershell
    # Get-Process が返すオブジェクトのメンバーを調べる
    Get-Process | Get-Member

    # 文字列オブジェクトのメンバーを調べる
    "Hello, World" | Get-Member
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| ⭐ **`-InputObject`** | `PSObject` | メンバーを調べるオブジェクトを指定します。通常はパイプラインから渡されます。 |
| ⭐ **`-MemberType`** | `PSMemberTypes` | 表示するメンバーの種類を指定します (`Property`, `Method`, `AliasProperty`, `NoteProperty` など)。 |
| **`-Name`** | `String[]` | 表示するメンバーの名前を指定します。ワイルドカード (`*`) が使用できます。 |
| **`-Static`** | `SwitchParameter` | オブジェクトのインスタンスメンバーではなく、クラスの静的メンバーを取得します。 |
| **`-Force`** | `SwitchParameter` | `get_` や `set_` メソッドなど、通常は表示されない全てのメンバーを表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: `Get-Service` の出力から、特定のプロパティ（例: サービスの開始モード）だけでソートしたいが、プロパティ名がわからない場合に調べる。
* **組み合わせ**: `Get-Service | Get-Member -MemberType Property`
* **解説**: `-MemberType Property` を使ってプロパティのみに絞り込むことで、利用可能なプロパティ名 (`Name`, `Status`, `StartType`など) を簡単に見つけ出すことができます。
* **例**:

    ```powershell
    # Get-Service が返すオブジェクトのプロパティを一覧表示
    Get-Service | Get-Member -MemberType Property
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なPowerShellスクリプト内で作成されているカスタムオブジェクトの構造を分析する。
* **組み合わせ**: `... | Get-Member`
* **解説**: 攻撃者が情報を整理するために作成したオブジェクトを `Get-Member` に渡すことで、そのオブジェクトにどのような情報（プロパティ名から推測）が格納されているかを理解する手がかりとなります。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance) / 環境寄生 (Living off the Land)**。未知のオブジェクトを操作する方法を発見する。
* **組み合わせ**: `... | Get-Member`
* **解説**: 攻撃者は侵入したシステムで、COMオブジェクトや。NETオブジェクトを操作して悪意のある活動を行うことがあります。`Get-Member` を使うことで、それらのオブジェクトが持つメソッド（実行可能な操作）やプロパティ（設定可能な値）を調査し、どのように悪用できるかを探します。
* **例**:

    ```powershell
    # COMオブジェクト 'WScript.Shell' がどのような操作を行えるか調査
    $comObject = New-Object -ComObject "WScript.Shell"
    $comObject | Get-Member
    ```

## エラーメッセージとトラブルシューティング

* `Get-Member` はオブジェクトの構造を表示するだけなので、エラーが発生することは稀です。パイプラインの前にコマンドが失敗している場合は、そちらのエラーを確認する必要があります。

## セキュリティに関する考慮事項

### 悪用事例

* **偵察 (Reconnaissance)**: レッドチームのシナリオで述べた通り、`Get-Member` は攻撃者がシステムの内部コンポーネントやアプリケーションのオブジェクトを調査し、それらを操作・悪用する方法を発見するための強力な内省ツールです。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `Get-Member` は PowerShell の標準機能であり、攻撃者が他の LOLBAS 手法（COMオブジェクトの悪用など）を発見・学習するために使用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) を有効化し、`Get-Member` の実行、特に `New-Object -ComObject` などで作成された不審なオブジェクトに対して使用されているケースを監視する。

## 注意点・補足

* **PowerShellの核心**: `Get-Member` を使いこなすことは、PowerShellの「全てはオブジェクトである」という思想を理解し、その能力を最大限に引き出すための第一歩です。わからないオブジェクトがあれば、まず `| Get-Member` に渡してみるのが定石です。
