# `Get-Location` - 現在の場所を取得する

## 概要

`Get-Location` コマンドレットは、現在の作業場所（カレントディレクトリ）の情報をオブジェクトとして取得します。このコマンドレットは、ファイルシステムプロバイダー (`FileSystem::`) だけでなく、レジストリ (`Registry::`)、証明書 (`Certificate::`) など、PowerShellがサポートする他のプロバイダーの現在の場所も取得できます。

CMDの `cd` (引数なし) やLinuxの `pwd` (print working directory) と同じ目的で使用されますが、`Get-Location` はパス情報を文字列だけでなく、詳細なプロパティを持つ `PathInfo` オブジェクトとして返します。

このコマンドレットには、`gl`, `pwd` という便利なエイリアスがデフォルトで設定されています。

## 類似コマンドレット/コマンドとの差異

| 類似コマンドレット/コマンド | 差異 / 使い分けのポイント |
| :--- | :--- |
| **`cd` (CMD)** | CMDの `cd` (引数なし) は現在のディレクトリパスを**文字列**として表示します。`Get-Location` は**オブジェクト**を返すため、スクリプト内で `(Get-Location).Path` のようにパス情報だけを抽出して利用するのが容易です。 |
| **`pwd` (Linux)** | `pwd` はLinux/Unix環境での標準コマンドです。PowerShellでは、互換性のために `pwd` が `Get-Location` のデフォルトエイリアスとして定義されており、同じ感覚で使用できます。 |
| **`$PWD` (自動変数)** | `$PWD` は現在の場所のパス情報を持つ自動変数です。コマンドレットを実行する `Get-Location` よりも高速にアクセスできます。 |

## よく連携されるコマンドレット

### シナリオ例: スクリプト実行前の場所を保存し、処理後に戻る (インフラ構築・運用視点)

* **目的**: スクリプト内で一時的に別のディレクトリに移動して作業を行い、スクリプトの最後には必ず元のディレクトリに戻るようにする。
* **連携コマンドレット**: `Set-Location`
* **解説**: スクリプトの冒頭で `Get-Location` を使って現在の場所を変数に保存しておきます。処理の最後に、その変数を `Set-Location` に渡すことで、スクリプトを実行した元の場所に戻ることができます。
* **コマンド例**:

    ```powershell
    # スクリプト実行時のカレントディレクトリを保存
    $initialLocation = Get-Location

    # 作業ディレクトリに移動
    Set-Location -Path "C:\Temp\Work"
    # ... 何らかのファイル処理 ...

    # 処理完了後、元のディレクトリに戻る
    Set-Location -Path $initialLocation
    ```

## パラメータ説明

| パラメータ | 型 | 説明 |
| :--- | :--- | :--- |
| **`-Stack`** | `SwitchParameter` | `Push-Location` でスタックに保存された場所の履歴を表示します。 |
| **`-StackName`** | `String[]` | `-Stack` と共に使用し、特定の名前付きスタックの履歴を表示します。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 現在の作業ディレクトリをログに出力する。
* **組み合わせ**: `Get-Location`
* **解説**: スクリプトがどのディレクトリで実行されたかをログに記録することは、トラブルシューティングの際に重要な情報となります。
* **例**:

    ```powershell
    # ログファイルに現在のパスを記録
    Add-Content -Path "C:\Logs\script.log" -Value "Script run from: $((Get-Location).Path)"
    ```

### 2. ブルーチーム視点

* **タスク**: **ライブレスポンス**。不審なスクリプトが実行された際の、カレントディレクトリを特定する。
* **組み合わせ**: (PowerShellログ分析)
* **解説**: PowerShellのスクリプトブロックロギングを有効にしている場合、攻撃者が実行したコマンドのログが記録されます。もし攻撃者が相対パス（例: `.\payload.exe`）でコマンドを実行した場合、`Get-Location` や `$PWD` を調査することで、その基点となったディレクトリを特定する手がかりとなります。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。現在の作業ディレクトリを把握し、相対パスでの活動の基点とする。
* **組み合わせ**: `pwd` (エイリアス)
* **解説**: 攻撃者はシステムに侵入した後、`pwd` (`Get-Location`) を実行して、自分が今システムのどこにいるのかを把握します。これは、`whoami` や `hostname` と並んで、状況把握のための最も基本的な初期コマンドの1つです。
* **例**:

    ```powershell
    # 現在地を確認
    pwd
    ```

## エラーメッセージとトラブルシューティング

* `Get-Location` は現在の場所を取得するだけなので、エラーが発生することはほとんどありません。

## セキュリティに関する考慮事項

### 悪用事例

* **偵察 (Reconnaissance)**: レッドチームのシナリオで述べた通り、攻撃者は `Get-Location` (`pwd`) を使って、侵入後の活動の基点となる現在のディレクトリを把握します。これにより、相対パスを使ったファイルの探索やペイロードの実行が可能になります。

### LOLBASにおける利用例

* **機能**: **偵察 (Reconnaissance)**
* **解説**: `Get-Location` は PowerShell の標準機能であり、攻撃者が侵入後の状況確認のために利用する、最も基本的な環境寄生型の手法です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: PowerShellのスクリプトブロックロギング (イベントID 4104) やトランスクリプションログを有効化し、`Get-Location` (`pwd`) の実行を記録・監視する。このコマンド自体は極めて一般的であるため、他の偵察コマンド (`whoami`, `hostname`, `Get-ChildItem` など) との連続実行といった文脈で評価することが重要です。

## 注意点・補足

* **`Push-Location` と `Pop-Location`**: `Get-Location` と `Set-Location` を使った場所の保存・復元は一時的なものです。より複雑なディレクトリ移動を行う場合は、場所をスタックに積む `Push-Location` と、スタックから取り出して戻る `Pop-Location` を使う方が、より堅牢で管理しやすいスクリプトになります。
