# Windows CMD ワイルドカード

## 概要

ワイルドカードは、複数のファイル名やディレクトリ名を一度に表現するために使用される特殊な文字です。コマンドプロンプトで `dir`, `copy`, `del` などのファイル操作コマンドと共に使うことで、特定のパターンに一致する全てのファイルを対象とすることができ、作業を大幅に効率化します。

## 主要なワイルドカード

Windowsのコマンドプロンプトでは、主に2種類のワイルドカードが使用されます。

| ワイルドカード | 名称 | 説明 | 使用例 |
| :--- | :--- | :--- | :--- |
| **`*`** | アスタリスク | **任意の長さの任意の文字列**にマッチします。最も広く使われるワイルドカードです。 | `*.txt` (拡張子が`.txt`の全てのファイル)<br>`report*` ("report"で始まる全てのファイル) |
| **`?`** | 疑問符 | **任意の一文字**にマッチします。文字数が固定されているパターンを検索する際に便利です。 | `report?.log` (`report1.log`, `reportA.log`にはマッチするが `report10.log`にはマッチしない) |

## コマンドでの利用例

### `dir` - ファイルのリスト表示

特定のパターンに一致するファイルをリストアップします。

```cmd
REM .log拡張子を持つ全てのファイルを表示
dir *.log

REM ファイル名が "image" で始まり、拡張子が3文字の全てのファイルを表示
dir image*.???
```

### `copy` - ファイルのコピー

パターンに一致する複数のファイルを一度にコピーします。

```cmd
REM 全ての .ini ファイルを D:\backup ディレクトリにコピー
copy *.ini D:\backup\
```

### `del` - ファイルの削除

パターンに一致する複数のファイルを一度に削除します。**非常に強力で危険なため、使用には細心の注意が必要です。**

```cmd
REM 全ての .tmp ファイルを削除
del *.tmp
```

## 業務シナリオ

### 1\. インフラ構築・運用視点

* **タスク**: ログディレクトリに溜まった古いログファイル (`.log`, `.bak`など) を一括でクリーンアップする。
* **解説**: `del` コマンドとワイルドカードを組み合わせることで、不要なファイルを効率的に削除できます。実行前には `dir` で対象ファイルを確認するのが安全です。
* **コマンド例**:

    ```cmd
    REM まず、削除対象となるファイルを確認
    dir C:\Logs\*.bak

    REM 確認後、対象ファイルを一括で削除
    del C:\Logs\*.bak
    ```

### 2\. ブルーチーム視点

* **タスク**: 不審なファイルが配置されやすいディレクトリ（例: `%TEMP%`）で、特定のパターンの実行可能ファイルを検索する。
* **解説**: 攻撃者はしばしばランダムな名前の実行ファイルを作成します。`dir` とワイルドカードを使って、予期せぬ実行ファイルが存在しないかを調査します。
* **コマンド例**:

    ```cmd
    REM C:\Users\Public\Downloads 内の全ての .exe ファイルを再帰的にリストアップ
    dir C:\Users\Public\Downloads\*.exe /S /B
    ```

### 3\. レッドチーム視点

* **タスク**: **偵察 (Reconnaissance)**。ファイルシステム全体から、名前に "password" や "config" を含むファイルを探し出す。
* **解説**: 攻撃者は侵入後、ワイルドカードを駆使して機密情報が含まれていそうなファイルを効率的に検索します。
* **コマンド例**:

    ```cmd
    REM Cドライブ全体から、名前に "pass" を含む .txt または .ini ファイルを検索
    dir C:\*pass*.txt /S /B
    dir C:\*pass*.ini /S /B
    ```

## 注意点・補足

* **破壊的コマンドとの組み合わせ**: `del` や `rd /s` のような破壊的なコマンドでワイルドカードを使用する際は、**実行前に必ず `dir` コマンドで対象となるファイルやディレクトリが意図したものと一致しているかを確認してください**。`del *.*` や `del C:\Windows\*` のようなコマンドは、システムに深刻なダメージを与える可能性があります。
* **PowerShellのワイルドカード**: PowerShellも `*` や `?` をサポートしていますが、`[]` を使った文字範囲の指定など、より高度なワイルドカード機能を持っています。
