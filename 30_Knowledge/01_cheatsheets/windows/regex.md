# Windows CMD 正規表現

## 概要

正規表現（Regular Expression, regex）は、文字列の集合を単一のパターンで表現するための形式言語です。Windowsのコマンドプロンプトでは、主に `findstr` コマンドが正規表現をサポートしており、ログファイルやコマンド出力から特定のパターンを持つ行を柔軟に検索するために使用されます。

**注**: CMDの `findstr` がサポートする正規表現は、Perl互換正規表現 (PCRE) など、最新のプログラミング言語で使われるものに比べて機能が限定的です。

## `findstr` での利用

`findstr` コマンドで正規表現を使用する場合、いくつかのスイッチが関連します。

* **`/R`**: 検索文字列を正規表現として解釈します。これは `findstr` のデフォルトの動作に近いですが、明示的に指定する場合に使用します。
* **`/L`**: 検索文字列を正規表現ではなく、リテラル（文字通り）として解釈します。メタ文字をただの文字として検索したい場合に便利です。
* **`/C:"<string>"`**: スペースを含む文字列を、1つのリテラル文字列として検索します。

## 主要なメタ文字と構文

| メタ文字 | 名称 | 説明 | findstrでの使用例 |
| :--- | :--- | :--- | :--- |
| **`.`** | ドット / ピリオド | 任意の一文字にマッチします。 | `findstr "f.le" file.txt` (`file`, `fole` などにマッチ) |
| **`*`** | アスタリスク | 直前の文字の0回以上の繰り返しにマッチします。 | `findstr "ab*c" file.txt` (`ac`, `abc`, `abbc` などにマッチ) |
| **`^`** | キャレット / ハット | 行の先頭にマッチします。 | `findstr "^Error" log.txt` (行頭が"Error"で始まる行にマッチ) |
| **`$`** | ドル | 行の末尾にマッチします。 | `findstr "failed$" log.txt` (行末が"failed"で終わる行にマッチ) |
| **`[]`** | 角括弧 | 角括弧内のいずれかの一文字にマッチします。(`[abc]` は `a` または `b` または `c`) | `findstr "gr[ea]y" file.txt` (`grey`, `gray` の両方にマッチ) |
| **`[^]`** | 否定文字セット | 角括弧内のいずれの文字にもマッチしない一文字にマッチします。 | `findstr "[^0-9]" file.txt` (数字以外の文字を含む行にマッチ) |
| **`[x-y]`** | 範囲指定 | `x` から `y` までの範囲のいずれかの一文字にマッチします。 | `findstr "[0-9]" file.txt` (いずれかの数字を含む行にマッチ) |
| **`\<`** | 単語の先頭 | 単語の始まりにマッチします。 | `findstr "\<cat" file.txt` (`category` にはマッチせず `cat` にマッチ) |
| **`\>`** | 単語の末尾 | 単語の終わりにマッチします。 | `findstr "cat\>" file.txt` (`catalog` にはマッチせず `cat` にマッチ) |
| **`\`** | バックスラッシュ | メタ文字をエスケープし、ただの文字として扱います。 | `findstr "192\.168\.1\.1" log.txt` (ドットをリテラルとして扱う) |

## 業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログファイルからIPアドレスのようなパターンを持つ行を抽出する。
* **解説**: `findstr` の正規表現は限定的ですが、単純なパターンの組み合わせでIPアドレスが含まれる行を大まかに絞り込むことができます。
* **コマンド例**:

    ```cmd
    REM 3桁の数字が4回繰り返される単純なパターンでIPアドレスを検索
    type access.log | findstr "[0-9][0-9]*\.[0-9][0-9]*\.[0-9][0-9]*\.[0-9][0-9]*"
    ```

### 2. ブルーチーム視点

* **タスク**: Webサーバーのアクセスログから、SQLインジェクション攻撃の兆候（例: `UNION SELECT`）を調査する。
* **解説**: 攻撃のキーワードを正規表現パターンとして使い、大量のログから不審なリクエストを抽出します。
* **コマンド例**:

    ```cmd
    REM アクセスログから "union" と "select" が含まれる行を検索（大文字小文字無視）
    findstr /R /I "union.*select" access.log
    ```

### 3. レッドチーム視点

* **タスク**: `dir /s` の結果から、特定の拡張子（`.bak`, `.config`, `.txt`）を持つファイルのみをリストアップする。
* **解説**: 偵察段階で、機密情報が含まれていそうな拡張子を持つファイルを効率的に見つけ出します。
* **コマンド例**:

    ```cmd
    REM .bak, .config, .txt で終わる行を検索
    dir /s /b | findstr /R /I "\.bak$ \.config$ \.txt$"
    ```

## 注意点・補足

* **機能の制限**: `findstr` の正規表現は、`+` (1回以上の繰り返し), `?` (0回または1回の繰り返し), `()` (グルーピング), `|` (OR条件) といった、モダンな正規表現エンジンでは一般的な多くの機能をサポートしていません。
* **PowerShellの利用**: より高度な正規表現（PCRE互換）を使用したい場合は、CMDではなくPowerShellの `Select-String` コマンドレットの使用が強く推奨されます。PowerShellははるかに強力なテキスト処理能力を持っています。
