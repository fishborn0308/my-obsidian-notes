# Windows ファイルシステムの階層標準

## 概要

このドキュメントでは、Windowsオペレーティングシステムにおける主要なディレクトリ（フォルダー）の構造と、それぞれの役割について解説します。LinuxのFHS (Filesystem Hierarchy Standard) とは異なりますが、Windowsにも特定の目的を持った標準的なディレクトリが存在します。これらの構造を理解することは、システム管理、トラブルシューティング、そしてセキュリティ分析の基礎となります。

---

## 主要なディレクトリ構造

| パス (`%SystemDrive%` は通常 `C:`) | 環境変数 | 説明 | セキュリティに関する考慮事項 (ブルーチーム/レッドチーム視点) |
| :--- | :--- | :--- | :--- |
| **`\Users`** | | 全てのユーザープロファイルが格納される親ディレクトリ。 | **レッドチーム**: 他のユーザーのプロファイルにアクセスできれば、機密情報を窃取できる可能性があります。特に共有PCでは重要な偵察対象です。<br>**ブルーチーム**: ユーザープロファイル間のアクセス制御を適切に設定し、不要なアクセスがないか監視します。 |
| &nbsp;&nbsp;`\<Username>` | `%USERPROFILE%` | 現在のユーザーのプロファイルディレクトリ。デスクトップ、ドキュメント、ダウンロード、AppDataなどが含まれます。 | **レッドチーム**: 侵入後の主要な活動拠点。ユーザーの文書、ブラウザのCookie/パスワード、SSHキーなどを探索します。<br>**ブルーチーム**: 最も重要な監視対象の一つ。`AppData`配下での不審な実行ファイルの作成や、ドキュメントファイルのマクロ実行などを監視します。 |
| &nbsp;&nbsp;&nbsp;&nbsp;`\AppData` | `%APPDATA%` (Roaming) / `%LOCALAPPDATA%` (Local) | アプリケーションが設定やキャッシュ、一時データを保存する場所。`Roaming`はドメイン環境で同期され、`Local`はされません。 | **レッドチーム**: マルウェアの永続化（スタートアップへの登録）、ペイロードの配置場所として極めて頻繁に悪用されます。<br>**ブルーチーム**: `AppData\Local\Temp`や`AppData\Roaming`からの実行ファイルの実行は、AppLockerやWDACで制限・監視すべきです。 |
| &nbsp;&nbsp;`\Public` | `%PUBLIC%` | PC上の全てのユーザーが共有でアクセスできるディレクトリ。 | **レッドチーム**: 書き込み権限が緩いことが多く、マルウェアのステージングエリア（一時的な配置場所）として利用されやすいです。<br>**ブルーチーム**: このディレクトリへの不審なファイル（特に実行ファイル）の書き込みを厳重に監視します。 |
| **`\Windows`** | `%windir%`, `%SystemRoot%` | Windowsオペレーティングシステムの中核となるファイルが格納されています。 | **レッドチーム**: LOLBAS (Living Off The Land Binaries and Scripts) の宝庫。正規のシステムツールを悪用して攻撃活動を行います。<br>**ブルーチーム**: このディレクトリ内のファイルの変更・書き込みは、ファイル整合性監視(FIM)ツールで厳重に監視する必要があります。 |
| &nbsp;&nbsp;`\System32` | | 64ビットOSの主要なシステムファイルと実行可能ファイルが格納されています。（歴史的な経緯で"32"という名前ですが64ビットファイルです） | **レッドチーム**: `cmd.exe`, `powershell.exe`, `rundll32.exe`など、攻撃に利用するツールが多数存在します。また、DLLハイジャックの標的ともなります。<br>**ブルーチーム**: このディレクトリからの不審なプロセスの起動、特に`svchost.exe`に偽装したプロセスなどに注意します。 |
| &nbsp;&nbsp;`\SysWOW64` | | 64ビットOS上で32ビットアプリケーションを動作させるための32ビット版システムファイルが格納されています。 | **レッドチーム**: 32ビット版のLOLBASを利用する際の起点となります。 |
| &nbsp;&nbsp;`\Temp` | | システム全体が使用する一時ファイルディレクトリ。 | **レッドチーム**: `%USERPROFILE%\AppData\Local\Temp`と同様に、マルウェアの展開場所や一時ファイル置き場として悪用されます。<br>**ブルーチーム**: このディレクトリからの実行ファイルの実行は強く制限・監視すべきです。 |
| &nbsp;&nbsp;`\Logs` | | Windowsの各種コンポーネント（CBSなど）のログファイルが保存されます。 | **ブルーチーム**: システムのトラブルシューティングや、Windows Updateの失敗などを調査する際の情報源となります。 |
| **`\Program Files`** | `%ProgramFiles%` | 64ビットアプリケーションがデフォルトでインストールされる場所。 | **レッドチーム**: インストールされているアプリケーションのリストを偵察し、既知の脆弱性を持つ古いソフトウェアを探します。また、パーミッションが緩いディレクトリがあれば、DLLハイジャックの標的とします。<br>**ブルーチーム**: アプリケーションのインストールとパッチ適用を適切に管理します。このディレクトリへの予期せぬ書き込みを監視します。 |
| **`\Program Files (x86)`** | `%ProgramFiles(x86)%` | 32ビットアプリケーションがデフォルトでインストールされる場所。（64ビットOSのみ存在） | 上記 `Program Files` と同様です。 |
| **`\ProgramData`** | `%ProgramData%` | 全てのユーザーが共通で利用するアプリケーションデータが保存される場所。隠しディレクトリです。 | **レッドチーム**: マルウェアが設定ファイルやペイロードを隠す場所としてよく利用します。<br>**ブルーチーム**: `Users`ディレクトリと同様に、このディレクトリへの不審なファイル作成やディレクトリ作成を監視することが重要です。 |
| **`\PerfLogs`** | | パフォーマンスモニターやサーバーマネージャーによって生成されるパフォーマンスログが保存されるデフォルトの場所。 | 通常、直接的な攻撃対象にはなりにくいですが、システムのパフォーマンス分析を通じて、マルウェアの異常なリソース消費を間接的に発見できる可能性があります。 |
