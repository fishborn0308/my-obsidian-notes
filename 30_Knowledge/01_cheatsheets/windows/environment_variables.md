# Windows 環境変数

## 概要

このドキュメントでは、Windowsオペレーティングシステム全体で使用される主要な環境変数について解説します。環境変数は、システムの動作、コマンドプロンプトやPowerShellの挙動、アプリケーションの実行に必要なパスや設定値を格納するための動的な名前付きの値です。

## 主要な環境変数

| 環境変数名 | 説明 | セキュリティに関する考慮事項 (ブルーチーム/レッドチーム視点) |
| :--- | :--- | :--- |
| **`Path`** | 実行可能ファイルを探すディレクトリのパスをセミコロン(`;`)区切りでリストします。コマンドプロンプトでコマンドが実行されると、このリストの先頭から順に検索されます。 | **レッドチーム**: `Path`の先頭に書き込み可能なディレクトリを追加し、正規の実行可能ファイルと同名の悪意のあるファイルを配置することで、コマンド実行をハイジャックできます (Path Interception)。<br>**ブルーチーム**: `Path`環境変数の変更、特にユーザーレベルでの変更や、信頼できないパスの追加を監視することが重要です。 |
| **`TEMP` / `TMP`** | アプリケーションが一時ファイルを保存するディレクトリのパス。通常はユーザープロファイル内にあります。 | **レッドチーム**: マルウェアのダウンロード場所、展開場所、一時的なデータ保管場所として頻繁に悪用されます。監視が手薄なことが多いです。<br>**ブルーチーム**: `%TEMP%`ディレクトリからの実行ファイルの実行をAppLockerやWDACで制限・監視することが強く推奨されます。`Sysmon`でこのディレクトリへのファイル作成を監視するのも有効です。 |
| **`USERNAME`** | 現在ログオンしているユーザーの名前。 | **レッドチーム**: 侵入後の初期偵察で、現在の権限レベルを把握するために`whoami`と合わせて使用します。 |
| **`USERDOMAIN`** | 現在のユーザーが所属するドメイン名。ワークグループ環境ではコンピュータ名になります。 | **レッドチーム**: 現在のホストがドメインに参加しているかどうか、ドメイン名は何かを特定するための重要な情報源です。 |
| **`USERPROFILE`** | 現在のユーザーのプロファイルディレクトリのパス (例: `C:\Users\Username`)。 | **レッドチーム**: デスクトップ、ドキュメント、AppDataなど、ユーザーデータが集約されている場所への起点となります。情報窃取の主要なターゲットです。 |
| **`APPDATA`** | アプリケーションが設定などを保存するローミングプロファイルディレクトリのパス (例: `C:\Users\Username\AppData\Roaming`)。 | **レッドチーム**: 多くのマルウェアが永続化のために、このディレクトリ配下にファイルや設定を配置します。<br>**ブルーチーム**: このディレクトリ内の不審なファイル作成や、スタートアップに登録されるような挙動を監視すべきです。 |
| **`LOCALAPPDATA`** | アプリケーションが設定などを保存するローカルプロファイルディレクトリのパス。 | **レッドチーム**: `APPDATA`と同様に、永続化やツールキットの配置場所として利用されます。 |
| **`COMPUTERNAME`** | コンピュータのホスト名。 | **レッドチーム**: 侵入したホストの役割を推測するための初期偵察に使用します。`DB-SERVER-01`のような名前は格好のターゲットです。 |
| **`windir`** | Windowsがインストールされているディレクトリのパス (例: `C:\Windows`)。 | **レッドチーム**: `System32`など、重要なシステムファイルへのパスの基点となります。LOLBASが悪用するバイナリの多くがこの配下にあります。 |
| **`SystemRoot`** | `windir` と同じく、Windowsのルートディレクトリを指します。 | `windir` と同様です。 |
| **`SystemDrive`** | Windowsがインストールされているドライブ (例: `C:`)。 | - |
| **`ProgramFiles`** | 64ビットアプリケーションがインストールされるディレクトリのパス (例: `C:\Program Files`)。 | **レッドチーム**: 権限昇格の脆弱性を持つアプリケーションを探すための偵察対象となります。 |
| **`ProgramFiles(x86)`** | 32ビットアプリケーションがインストールされるディレクトリのパス。 | `ProgramFiles` と同様です。 |
| **`PUBLIC`** | 全てのユーザーが共通で利用できるファイルの置き場所 (例: `C:\Users\Public`)。 | **レッドチーム**: 全ユーザーから読み書き可能な場合が多く、マルウェアのステージングエリア（一時的な配置場所）や、ユーザー間でのデータ受け渡し場所として悪用されやすいです。 |
| **`PSModulePath`** | PowerShellがモジュールを検索するパスのリスト。 | **レッドチーム**: このパスに書き込み権限がある場合、悪意のあるPowerShellモジュールを配置し、正規のモジュールに見せかけてロードさせることで、コード実行や永続化を図ることができます。 |

## 環境変数の確認と設定方法

### CMD (コマンドプロンプト)

* **全ての環境変数を表示:**

  ```cmd
  set

  ```

  * **特定の環境変数を表示:**

    ```cmd
    echo %USERNAME%
    ```

  * **一時的な設定 (現在のセッションのみ有効):**

    ```cmd
    set MY_VARIABLE=Hello
    ```

  * **永続的な設定 (ユーザーレベル):**

    ```cmd
    setx MY_VARIABLE "Hello World"
    ```

  * **永続的な設定 (システムレベル):**

    ```cmd
    setx /M MY_VARIABLE "System Wide Hello"
    ```

### PowerShell

* **全ての環境変数を表示:**

    ```powershell
    Get-ChildItem Env:
    ```

* **特定の環境変数を表示:**

    ```powershell
    $env:USERNAME
    # または
    Get-Content Env:\USERNAME
    ```

* **一時的な設定 (現在のセッションのみ有効):**

    ```powershell
    $env:MY_VARIABLE = "Hello"
    ```

* **永続的な設定 (ユーザーレベル):**

    ```powershell
    [System.Environment]::SetEnvironmentVariable("MY_VARIABLE", "Hello World", "User")
    ```

* **永続的な設定 (システムレベル):**

    ```powershell
    [System.Environment]::SetEnvironmentVariable("MY_VARIABLE", "System Wide Hello", "Machine")
    ```

### GUI

1. `sysdm.cpl` を実行して「システムのプロパティ」を開きます。
2. 「詳細設定」タブをクリックします。
3. 「環境変数」ボタンをクリックします。
4. ユーザー環境変数とシステム環境変数を表示・編集できます。
