# Command - Linux - date - システムの日付と時刻を表示・設定する

## 概要

`date` は、システムの現在の時刻や日付を表示したり、スーパーユーザー権限でそれらを変更したりするためのコマンドです。最大の特徴は、非常に柔軟な「表示フォーマットの指定」と「相対的な日付計算（昨日の日付など）」ができる点にあります。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `timedatectl` | 現代のシステム（systemd）において、タイムゾーンや NTP 同期設定も含めた統合的な管理を行う。 | **現代の標準設定** |
| `hwclock` | マザーボード上のハードウェアクロック（RTC）を直接操作・表示する。 | OS起動・停止時の同期 |
| `uptime` | システムが起動してからの経過時間と平均負荷を表示する。 | 稼働状況の確認 |
| `date` | 表示フォーマットの変換や、スクリプト内での文字列生成に特化。 | **スクリプト・動的な表示** |

## よく連携されるコマンド

### シナリオ例: タイムスタンプ付きバックアップファイルの作成 (運用視点)

* **目的**: ファイルをコピーする際、名前に実行時の日付を入れて世代管理をしやすくする。
* **連携コマンド**: `cp`, `tar`
* **解説**: バッククォート `` ` `` または `$( )` を使ってコマンドの結果を文字列として展開します。
* **コマンド例**:
    ```bash
    # 現在の日付（20260222等）をファイル名に含めてコピー
    cp config.conf config.conf.$(date +%Y%m%d)
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示・計算** | |
| ⭐️ `+FORMAT` | 出力形式を指定する。 (例: `+%Y-%m-%d %H:%M:%S`) |
| ⭐️ `-d`, `--date=STRING` | 現在ではなく、指定した文字列が示す日時を表示する（例: `'yesterday'`, `'1 month ago'`). |
| ⭐️ `-u`, `--utc` | 協定世界時 (UTC) で表示・設定する。 |
| **設定 (要root)** | |
| `-s`, `--set=STRING` | システム時刻を指定した日時に設定する。 |

### 主要なフォーマット指定子

* `%Y`: 西暦4桁 (2026)
* `%m`: 月 (01..12)
* `%d`: 日 (01..31)
* `%H`: 時 (00..23)
* `%M`: 分 (00..59)
* `%S`: 秒 (00..60)
* `%s`: **Unixエポックからの経過秒数** (1970-01-01 00:00:00 UTCからの秒数)

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログ解析時のエポック秒から人間が読める形式への変換。
* **組み合わせ**: `date -d @<seconds>`
* **解説**: システムログ等に記録された秒数表示のタイムスタンプを即座に確認します。
* **例**:
    ```bash
    # Unixエポック秒を日時に変換
    date -d @1740220000
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査（フォレンジック）におけるタイムライン分析。
* **組み合わせ**: `--utc`
* **解説**: 複数のサーバーのログを比較する際、各サーバーのタイムゾーンが異なると混乱を招くため、すべて UTC に統一して突き合わせを行います。
* **例**:
    ```bash
    # 標準時で正確な時刻を確認
    date -u
    ```

### 3. レッドチーム視点

* **タスク**: アンチフォレンジック（証跡の攪乱）や、期限切れ証明書の回避。
* **組み合わせ**: `date -s`
* **解説**: 侵入後、ログに記録される時刻を過去のものに偽装するために一時的にシステム時刻を変更します。また、HTTPS通信時に証明書の期限が切れている場合、一時的に時刻を戻して通信を通すといった手法に使われます。
* **例**:
    ```bash
    # システム時刻を1日前に戻す
    sudo date -s "yesterday"
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `date: cannot set date: Operation not permitted`
    * **考えられる原因**: 一般ユーザー権限で時刻を設定（`-s`）しようとした。
    * **解決策**: `sudo` を使用する。コンテナ環境の場合は、ホスト側の時刻管理が必要な場合がある。

2.  **時刻がずれる（NTPの影響）**:
    * **事象**: `date -s` で時刻を変更しても、すぐに元の正しい時刻に戻ってしまう。
    * **解決策**: `chronyd` や `ntpd` などの同期サービスが動いているため。変更が必要な場合は一時的にそれらを停止するか、`timedatectl set-ntp false` を実行する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: 弱い権限設定。
* **悪用シナリオ**: 攻撃者が `sudo date` の実行権限を持っている場合、システム時刻を大幅にずらすことで、時刻同期に依存する認証プロトコル（Kerberos等）を破壊したり、ログの順序をバラバラにして解析を困難にさせたりします。

### GTFOBins における利用例

`date` が `sudo` で実行可能な場合、任意のファイルを読み取ることができます（ファイルの内容を日付として解析させようとする際のエラーメッセージなどを利用）。

* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo date を使ってファイルを読み取る（強引な手法）
    LFILE=/etc/shadow
    sudo date -f "$LFILE"
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/date/`

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**: システム時刻の急激な変更（Time Jump）を `auditd` や SIEM で監視する。
* **Prevention (予防)**: 一般ユーザーへの `sudo date` 許可を避け、時刻同期は信頼できる NTP サーバに限定する。

## 注意点・補足

* **スクリプトでの利用**: `$(date)` のようにスペースが含まれる出力を変数に入れる際は、クォート忘れによる意図せぬ分割に注意してください。
* **精度**: `date` はミリ秒やマイクロ秒の表示も可能ですが（`%N`）、ハードウェアやカーネルの精度に依存します。
