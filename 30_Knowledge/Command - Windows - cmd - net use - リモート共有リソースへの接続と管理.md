# Command - Windows - cmd - net use - リモート共有リソースへの接続と管理

## 概要

`net use` コマンドは、コンピュータを共有リソースに接続したり、共有リソースから切断したりするために使用されます。また、現在のネットワーク接続に関する情報を表示します。

ネットワークドライブの割り当て（マウント）だけでなく、認証情報（ユーザー名・パスワード）を指定した一時的なセッションの確立にも利用されます。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `New-PSDrive` | PowerShell版。ドライブとしてだけでなく、レジストリや証明書ストアもマウント可能。 | PS環境 |
| `pushd` | UNCパス（`\\server\share`）を一時的にドライブ割り当てして移動し、`popd` で解除可能。 | 一時的な作業 |
| `net share` | **自分の**フォルダを他者に公開する。 | 自端末の共有作成 |

## よく連携されるコマンド

### シナリオ例: 認証情報を指定した管理者共有へのアクセス (運用・攻撃共通)

* **目的**: 現在のログインユーザーとは別の権限（管理者等）を使用して、リモートサーバーの C ドライブにアクセスする。
* **連携コマンド**: `net use`, `dir`, `copy`
* **解説**: 特定のユーザー名とパスワードをコマンドライン上で指定して接続を確立します。
* **コマンド例**:
    ```cmd
    REM ユーザー "admin" としてサーバー "SRV01" の C ドライブを Z ドライブに割り当て
    net use z: \\SRV01\C$ /user:admin P@ssw0rd123!
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| (引数なし) | 現在確立されているすべてのネットワーク接続を一覧表示する。 |
| `devicename` | 割り当てるドライブ文字（例: `Z:`）またはプリンタポートを指定。 |
| `\\computer\share` | 接続先の UNC パスを指定。 |
| ⭐ `/user:[domain\]user` | 接続に使用するユーザー名を指定。 |
| `password` | ユーザー名に対応するパスワードを指定。 `*` を使うと対話的に入力可能。 |
| ⭐ `/delete` | 指定したネットワーク接続を解除する。 |
| **接続設定** | |
| ⭐ `/persistent:yes|no` | 再起動後も接続を維持するか (`yes`) 、今回限りか (`no`) を指定。 |
| `/savecred` | 指定した認証情報を Windows 資格情報マネージャーに保存する。 |
| `/home` | ユーザーをホームディレクトリに接続する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ログオン時に、全社員共通の共有フォルダを「Sドライブ」として自動的に割り当てるスクリプトを作成する。
* **例**:
    ```cmd
    REM 既存の接続を解除してから、再起動後も有効な設定でマウント
    net use s: /delete /y
    net use s: \\FileServer\SharedData /persistent:yes
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査において、不審な外部サーバーや内部の他端末へ「現在進行系で」張られているセッションがないか確認する。
* **例**:
    ```cmd
    REM 現在の全接続を表示し、身に覚えのないサーバーへのマウントがないかチェック
    net use
    ```

### 3. レッドチーム視点 (偵察と横展開)

* **タスク 1: Authenticated Session (認証済みセッションの確立)**
    * **目的**: 奪取した認証情報を使用して、リモートホストへのアクセス権を得る。
    * **解説**: IPC$ (Inter-Process Communication) 共有に接続することで、直接ファイルを操作しなくても、リモートでの列挙やコマンド実行の準備が整います。
    * **コマンド例**:
        ```cmd
        net use \\192.168.1.10\IPC$ /user:EXAMPLE\j.doe Password123
        ```

* **タスク 2: Mapping Admin Shares (管理共有のマウント)**
    * **目的**: 管理者権限（Local Admin 等）がある場合、C$ や ADMIN$ に接続してペイロードの設置や機密データの窃取を行う。
    * **コマンド例**:
        ```cmd
        REM ターゲットの C ドライブをローカルの K ドライブとしてマウント
        net use k: \\TargetServer\C$ /user:Administrator
        ```

* **タスク 3: Persistence (永続化の補助)**
    * **目的**: `/persistent:yes` を利用して、システム再起動後も攻撃者が用意した共有フォルダ（ツールキット設置場所など）へのパスを維持させる。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 指定したユーザーに権限がない、またはパスワードが間違っている。
    * **解決策**: 認証情報の再確認。または UAC やリモート制限（LocalAccountTokenFilterPolicy）を確認。

2.  **System error 1219 has occurred.**
    * **原因**: 同一サーバーに対して、複数のユーザー名で同時に接続しようとした（Windows の仕様制限）。
    * **解決策**: 既存の接続を `net use \\Server /delete` で解除してから再接続してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **認証情報の露出 (Credential Exposure - T1552.001):**
    * **悪用シナリオ**: 攻撃者は侵入した端末内のバッチファイルやスクリプトを検索し、`net use ... /user:admin password` のような平文パスワードが含まれる行を特定して特権を奪取します。
* **横展開 (Lateral Movement - T1021.002):**
    * 攻撃者は `net use` を使ってリモートマシンの `C$\Windows\Temp` 等にアクセスし、そこにマルウェアをコピーして `psexec` や `wmiexec` で実行する流れを多用します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Lateral Movement`, `Data Exfiltration`
* **特性**: 外部ツールを使わずに「標準的なネットワーク接続」を装えるため、多くのセキュリティ製品の監視をすり抜ける可能性があります。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **パスワードの対話的入力**: スクリプトにはパスワードを記述せず、`*` を使用して入力を促すか、Group Policy Preferences (GPP) の利用を避ける（過去の脆弱性に関連）。
    * **管理共有の制限**: 必要に応じて、レジストリ設定により `C$` などの管理共有を無効化する。
* **Detection (検知)**:
    * **イベントID 4624 / 4648**: 明示的な資格情報を使用したログオン。
    * **コマンドラインロギング**: 引数にパスワードが含まれている不審な `net use` プロセスを SIEM で検知する。

## 注意点・補足

* **パスワードの `*`**: パスワード部分を `*` にすると、画面上に文字を表示せずにパスワードを入力でき、セキュリティが向上します。
* **IPC$ 接続**: ドライブ文字を割り当てずに、パスのみを指定（例: `net use \\server\ipc$`）することで、バックグラウンドでの認証セッションのみを確立できます。
