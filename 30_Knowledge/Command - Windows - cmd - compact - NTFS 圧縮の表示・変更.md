# Command - Windows - cmd - compact - NTFS 圧縮の表示・変更

## 概要

`compact` は、NTFS パーティション上のファイルやディレクトリの圧縮状態を表示、または変更するためのコマンドです。
ファイル単位だけでなく、ディレクトリに圧縮属性を付与することで、今後その中に作成されるファイルを自動的に圧縮するように設定することも可能です。

(出自: `Windows 標準搭載 - 外部コマンド (compact.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `expand` | `.cab` や `.ms_` などの配布用圧縮ファイルを展開する。 | インストーラ解析 |
| `compress` | 古い DOS 形式の圧縮を作成する（現在はほぼ使われない）。 | - |
| `cipher` | 圧縮ではなく**暗号化**を行う。圧縮と暗号化は排他（同時適用不可）。 | セキュリティ重視 |
| `PowerShell (Archive)` | `Compress-Archive` 等。`.zip` 形式を作成する。 | ファイル配布用 |

## よく連携されるコマンド

### シナリオ例: ログ出力ディレクトリの自動圧縮設定 (運用管理)

* **目的**: 肥大化しやすいログ出力先フォルダに圧縮属性を付け、ディスク消費を抑える。
* **連携コマンド**: `mkdir`, `compact`
* **解説**: フォルダ自体に属性を付与するのがポイントです。
* **コマンド例**:
    ```cmd
    REM フォルダを作成し、その中の新規ファイルが自動圧縮されるよう設定
    mkdir C:\Logs\Archive
    compact /c /s:C:\Logs\Archive
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| (引数なし) | 現在のディレクトリ内のファイル圧縮状態（圧縮率など）を表示する。 |
| ⭐ `/C` | 指定したファイルまたはディレクトリを圧縮する。 |
| ⭐ `/U` | 指定したファイルまたはディレクトリの圧縮を解除する。 |
| `/S[:dir]` | 指定したディレクトリとそのすべてのサブディレクトリに対して処理を行う。 |
| `/A` | 隠しファイルやシステムファイルも含めて表示・処理する。 |
| `/I` | エラーが発生しても処理を続行する。 |
| `/F` | すでに圧縮済みのファイルに対しても強制的に圧縮をやり直す。 |
| `/EXE` | 実行ファイル (exe, dll) 用の高圧縮アルゴリズム (LZX等) を使用する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 仮想マシンのテンプレート作成時に、OS の不要なファイルを圧縮してイメージサイズを軽量化する。
* **例**:
    ```cmd
    REM 実行ファイルに特化した圧縮を適用してサイズを削る
    compact /c /exe /s:C:\Windows\System32\Drivers
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査において、攻撃者がツールを隠蔽したり、盗み出したデータを圧縮してステージング（一時置き）している形跡がないか監査する。
* **注意**: `compact` は OS 標準機能であるため、圧縮されていても正規のファイルアクセスで透過的に読み取れてしまいます。しかし、圧縮率が異常に高いファイルなどは、ランダムなデータ（暗号化されたデータ）ではなく、反復的なデータ（テキストログ等）である可能性を示唆します。

### 3. レッドチーム視点 (偵察とデータ持ち出し)

* **タスク 1: Data Staging (持ち出しデータの準備)**
    * **目的**: ネットワーク経由でデータを持ち出す前に、送信データ量を減らして検知リスクを下げる。
    * **解説**: 外部の `.zip` ツールを導入できない環境でも、`compact` なら標準で圧縮が可能です。
    * **コマンド例**:
        ```cmd
        REM 収集したドキュメントを一括圧縮
        compact /c /s:C:\Users\Target\Documents\Stolen
        ```

* **タスク 2: Environment Enumeration (ディスク状況の把握)**
    * **目的**: ターゲットがディスク容量に困っていないか、あるいは特定のフォルダが重要視（圧縮して長期保存）されているかを確認する。

## エラーメッセージとトラブルシューティング

1.  **圧縮属性を付与できない**
    * **原因**: 対象が FAT32 や exFAT ボリュームである、または `cipher` によって暗号化されている。
    * **解決策**: ボリュームが NTFS であること、および暗号化属性が解除されていることを確認してください。

2.  **圧縮されているのにサイズが変わらない**
    * **原因**: すでに圧縮済みのファイル（JPG, ZIP, MP4等）や、ファイルサイズが非常に小さいファイル。
    * **解決策**: 圧縮のオーバーヘッドがあるため、テキストや実行ファイル以外では効果が薄いです。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **アンチ・フォレンジック**: 
    * 攻撃者が `compact` を利用してツールを圧縮状態で保持することで、単純なファイルサイズベースのシグネチャ検知を回避しようとする試みがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Data Compression`
* **特性**: インストール不要で動作するため、攻撃の最終段階でのデータ圧縮（Data Exfiltration T1002）によく利用されます。
* **参照**: [LOLBAS - compact.exe](https://lolbas-project.github.io/lolbas/Binaries/Compact/)

## 注意点・補足

* **透過的アクセス**: `compact` で圧縮されたファイルは、アプリケーションからは「普通のファイル」として読み書きできます（OS が背後で動的に展開・圧縮します）。そのため、ユーザーが意識することなく利用できます。
* **CPU負荷**: アクセスのたびに展開・圧縮が発生するため、頻繁にアクセスされるデータベースファイルなどへの適用は慎重に行うべきです。
