# Command - Windows - cmd - goto - 実行の制御（ラベルへのジャンプ）

## 概要

`goto` コマンドは、バッチ プログラム内の指定されたラベルが付けられた行に、Windows コマンド インタープリターをジャンプさせます。これを使用することで、条件分岐（`if` との組み合わせ）や無限ループ、特定の処理のスキップなど、スクリプトに論理的な構造を持たせることができます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `call` | ラベル（サブルーチン）を呼び出し、処理後に**戻ってくる**。 | 関数的な処理 |
| `exit /b` | 現在のバッチを終了する。`goto :eof` と似た挙動をする。 | 終了処理 |
| `choice` | ユーザーの入力を待ち、その結果で `goto` 先を分岐させる。 | インタラクティブ |

## よく連携されるコマンド

### シナリオ例: エラーハンドリングとリトライ処理 (運用・自動化)

* **目的**: ネットワーク接続が不安定な場合に、成功するまで特定の処理を繰り返す。
* **連携コマンド**: `if`, `ping`, `goto`
* **解説**: 処理の成否を `%ERRORLEVEL%` で判断し、失敗時のみラベルへ戻ります。
* **コマンド例**:
    ```cmd
    @echo off
    :retry_target
    echo Checking connection...
    ping -n 1 192.168.1.1 >nul
    if %ERRORLEVEL% neq 0 (
        echo Connection failed. Retrying in 5 seconds...
        timeout /t 5 >nul
        goto retry_target
    )
    echo Connection established!
    ```

## スイッチ/引数説明

**※重要: ラベルは行の先頭にコロン `:` を付けて定義します（例: `:MyLabel`）。**

| 引数 | 説明 |
| :--- | :--- |
| `label` | ジャンプ先のラベル名を指定する。 |
| ⭐ `:eof` | **【重要】** 特殊なラベル。実行をバッチファイルの末尾に移し、スクリプトを終了させる。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 簡易的な対話型メニューを作成し、管理者が実行したい操作（バックアップ、リストア、終了など）を選択できるようにする。
* **例**:
    ```cmd
    @echo off
    echo 1: Backup / 2: Restore / 3: Exit
    set /p choice="Select number: "
    if "%choice%"=="1" goto backup_proc
    if "%choice%"=="2" goto restore_proc
    goto :eof

    :backup_proc
    echo Running backup...
    goto :eof

    :restore_proc
    echo Running restore...
    ```

### 2. ブルーチーム視点 (フォレンジック・調査)

* **タスク**: 調査対象の OS バージョンを自動判定し、それぞれのバージョンに適した調査コマンドセクションへジャンプする。
* **解説**: 環境に応じた条件分岐を `goto` で整理し、1 つのスクリプトで多環境に対応させます。

### 3. レッドチーム視点 (難読化と防御回避 - T1027)

* **タスク 1: Spaghetti Code (スパゲッティコード化)**
    * **目的**: バッチファイルの中身を読み取りにくくし、静的解析を妨害する。
    * **解説**: 意味のないラベルと `goto` を大量に挟み込み、実行順序をわざと複雑に飛び回らせることで、解析者がロジックを追う意欲を削ぎます。
* **タスク 2: Persistence (永続化プロセスの監視)**
    * **目的**: 自分が実行したペイロードが終了してしまった場合、`goto` ループを使用して即座に再起動させる「ウォッチドッグ」を作成する。
    * **コマンド例**:
        ```cmd
        :watchdog
        tasklist | find "malware.exe" >nul
        if %ERRORLEVEL% neq 0 start malware.exe
        timeout /t 60 >nul
        goto watchdog
        ```

## エラーメッセージとトラブルシューティング

1.  **ラベルが見つかりません (The system cannot find the batch label specified)**
    * **原因**: `goto` で指定した名前のラベルがファイル内に存在しない、または綴りが間違っている。
    * **解決策**: ラベル定義（例: `:MyLabel`）が正しいか、コロンが抜けていないか確認してください。

2.  **無限ループによるフリーズ**
    * **原因**: 終了条件のない `goto` ループが高速で回っている。
    * **解決策**: ループ内に `timeout` や `pause` を入れて実行速度を制御するか、必ず抜けるための `if` 条件を追加してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **制御フローの乗っ取り**:
    * 攻撃者がバッチファイル内の環境変数を操作できる場合、ジャンプ先のラベル名を書き換えることで、本来実行されるべきではない「管理者向けメンテナンス処理」などへ実行を誘導される可能性があります。

## 注意点・補足

* **ラベルの文字数**: 有効なラベル名は最初の 8 文字までです。それ以降は無視されるため、似た名前の長いラベル（`:Setup_Server_A` と `:Setup_Server_B` など）は混同される可能性があります。
* **goto :eof**: これは「End Of File」の略です。バッチのどこからでも「最後へ飛んで終わる」ことができるため、サブルーチン（`call` 先）から戻る際や、エラー時にスクリプトを中断する際に多用されます。
