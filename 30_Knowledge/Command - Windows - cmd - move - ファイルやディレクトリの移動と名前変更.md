# Command - Windows - cmd - move - ファイルやディレクトリの移動と名前変更

## 概要

`move` コマンドは、1つまたは複数のファイルを別の場所に移動するために使用されます。また、ディレクトリ（フォルダ）の名前を変更する際にも利用されます。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `copy` | `copy` は元のファイルを残すが、`move` は移動後に元ファイルを削除する。 | 目的による |
| `ren` (rename) | `ren` は同一ディレクトリ内での名前変更のみ。`move` は別ディレクトリへの移動と同時に名前変更が可能。 | 同一内なら `ren` |
| `robocopy /move` | `robocopy` の `/move` は移動後にディレクトリごと削除するオプション。大量のファイルやエラー再試行が必要な場合に適している。 | 大規模移動は `robocopy` |

## よく連携されるコマンド

`move` は、条件分岐やループ処理の中で、処理済みのファイルを「完了フォルダ」へ移動させる際によく使われます。

### シナリオ例: 処理済みログのアーカイブ (運用自動化)

* **目的**: 読み取りが完了した古いログファイルをアーカイブ用ディレクトリに退避させる。
* **連携コマンド**: `for`, `if`
* **解説**: 特定の拡張子のファイルをループで回し、存在を確認してから移動します。
* **コマンド例**:
    ```cmd
    REM .logファイルをすべてArchiveフォルダへ移動（存在しない場合は作成）
    if not exist Archive mkdir Archive
    for %f in (*.log) do move "%f" Archive\
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **上書き制御** | |
| ⭐ `/Y` | 移動先に同名のファイルがある場合、確認メッセージを表示せずに上書きする。 |
| `/-Y` | 上書きする前に確認メッセージを表示する（デフォルトの挙動）。 |
| **引数** | |
| `[ソース]` | 移動したいファイルまたはディレクトリのパス。 |
| `[ターゲット]` | 移動先のディレクトリパス、または新しいファイル名。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルの世代管理（リネーム）
* **組み合わせ**: `%DATE%` 環境変数
* **解説**: 現在の設定ファイルを、日付を付与した名前に変更してバックアップとします。
* **例**:
    ```cmd
    REM config.txt を config_20260222.txt にリネームしてBackupへ移動
    move C:\App\config.txt C:\App\Backup\config_%DATE:/=%.txt
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なファイルの隔離（Quarantine）
* **組み合わせ**: `attrib`
* **解説**: 調査対象のファイルを一般ユーザーが触れない隔離用ディレクトリに移動し、読み取り専用属性を付与します。
* **例**:
    ```cmd
    REM 不審な.exeを隔離フォルダへ移動
    mkdir C:\Quarantine
    move C:\Users\Public\Downloads\suspicious.exe C:\Quarantine\
    attrib +R C:\Quarantine\suspicious.exe
    ```

### 3. レッドチーム視点

* **タスク**: ペイロードの配置と偽装
* **組み合わせ**: `type` (またはダウンロードコマンド)
* **解説**: `Temp` フォルダにダウンロードしたバイナリを、永続化のために正規のプログラム名に偽装して移動します。
* **例**:
    ```cmd
    REM ツールをシステムディレクトリ風の場所に移動して名前を偽装
    move %TEMP%\nc.exe C:\Windows\Tasks\taskhost_provider.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `プロセスはファイルにアクセスできません。別のプロセスが使用中です。`
    * **考えられる原因**: 移動しようとしているファイルが他のアプリケーションで開かれている。
    * **解決策**: `tasklist` で該当ファイルを使用しているプロセスを特定して終了させるか、使用が終わるまで待機する。

2.  **エラーメッセージ例 2**: `指定されたドライブ間にディレクトリを移動することはできません。`
    * **考えられる原因**: `move` コマンドで**ディレクトリ**を別のドライブ（例: CからDへ）に移動しようとした。※ファイル単位なら可能。
    * **解決策**: ディレクトリごとの場合は `robocopy` を使用するか、`xcopy` でコピーした後に元を `rd /s /q` で削除する。

## 環境変数と設定ファイル

### 影響を与える環境変数

* **`%COPYCMD%`**: この変数に `/Y` を設定しておくと、`move` コマンドのデフォルト挙動を常に「上書き確認なし」に設定できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **DLL Hijacking**: 脆弱なアプリケーションの検索パス内にある正規のDLLを `move` で退避させ、攻撃者の悪意あるDLLを同名で配置する。
    * **Persistence (永続化)**: スタートアップフォルダへのショートカットや実行ファイルの移動による自動実行。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `File Move`
* **悪用例**:
    * 保護されたディレクトリへの書き込み権限がある場合、システムファイルを置き換えるために使用。
    * `move` を使って、検知されにくい隠しディレクトリへデータをステージング（集約）する。
* **参照**: [LOLBAS - move.exe](https://lolbas-project.github.io/lolbas/Binaries/Move/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * `C:\Windows` や `C:\Program Files` などの重要ディレクトリへの書き込み権限を制限する（最小権限の原則）。
* **Detection (検知)**:
    * コマンドライン監査（イベントID 4688）を有効にし、`move` コマンドの引数に不審なパス（`Temp` から `System32` への移動など）が含まれていないか監視する。
    * EDRによる「ファイルのリネーム」イベントの相関分析。

## 注意点・補足

* **ワイルドカードの挙動**: `move *.txt C:\Target\` のようにワイルドカードを使用できますが、移動先に同名のファイルがあると、`/Y` を指定していない限り停止します。
* **PowerShellとの比較**: PowerShellの `Move-Item` は別ドライブ間でのディレクトリ移動も透過的に処理できるため、スクリプト作成時はPowerShellの方が柔軟です。
