# Command - Windows - cmd - echo - メッセージの表示と設定

## 概要

`echo` コマンドは、メッセージを表示したり、コマンドのエコー機能（実行したコマンドそのものを画面に表示するかどうか）をオン/オフにしたりするために使用されます。また、リダイレクト記号 (`>` や `>>`) と組み合わせることで、ファイルへの書き込みや新規作成を行う際の主要な手段となります。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `Write-Output` | PowerShell 版。文字列だけでなくオブジェクトをパイプラインに渡す。 | PS 環境 |
| `printf` | Linux/Unix の標準。書式指定（フォーマット）がより柔軟。 | Linux 環境 |
| `type` | ファイルの内容を表示する。`echo` は指定した文字列を表示する。 | ファイル閲覧 |

## よく連携されるコマンド

### シナリオ例: バッチ実行ログの記録 (運用・自動化)

* **目的**: スクリプトの実行開始時刻や結果をログファイルに追記していく。
* **連携コマンド**: `echo`, `date`, `time`
* **解説**: `>>`（追記リダイレクト）を使用して、既存のログを消さずに末尾に書き込みます。
* **コマンド例**:
    ```cmd
    REM ログファイルに実行時のタイムスタンプを追記
    echo [%DATE% %TIME%] Backup Process Started >> C:\Logs\backup.log
    ```

## スイッチ/オプション説明

| 使用法 | 説明 |
| :--- | :--- |
| `echo on` | コマンド プロンプトに、入力したコマンド自体を表示するように設定（デバッグ用）。 |
| `echo off` | コマンドの表示をオフにする。バッチファイルの冒頭で `@echo off` と書くのが通例。 |
| ⭐ `echo [message]` | 指定したメッセージを画面に表示する。 |
| ⭐ `echo.` | **【重要】** 空白行を 1 行表示する。(`echo` の直後にピリオド) |
| `echo %VARIABLE%` | 指定した環境変数の値を表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイル（例：hosts ファイル）に新しいエントリを 1 行追加する。
* **例**:
    ```cmd
    REM 管理者権限で実行し、hosts ファイルに IP とホスト名を追記
    echo 192.168.1.100 dev-server >> C:\Windows\System32\drivers\etc\hosts
    ```

### 2. ブルーチーム視点 (環境調査)

* **タスク**: 現在のユーザープロファイル、実行パス、OS バージョンなどの重要な環境変数を一括で確認する。
* **例**:
    ```cmd
    REM PATH 変数やユーザー名を確認
    echo USER:%USERNAME% / PATH:%PATH%
    ```

### 3. レッドチーム視点 (ペイロード作成と偵察)

* **タスク 1: On-the-fly Scripting (スクリプトの即時作成 - T1059.003)**
    * **目的**: ターゲット端末上で、外部からファイルを転送せずに、その場で悪意あるバッチファイルや VBScript を作成する。
    * **解説**: 1 行ずつ `echo` でファイルに書き込み、最終的にそれを実行します。
    * **コマンド例**:
        ```cmd
        echo @echo off > payload.bat
        echo whoami >> payload.bat
        echo net user /domain >> payload.bat
        payload.bat
        ```

* **タスク 2: Information Exfiltration Prep (データのラベル付け)**
    * **目的**: 収集した複数のコマンド結果を 1 つのテキストにまとめる際、区切り線や見出しを付けて解析しやすくする。
    * **コマンド例**:
        ```cmd
        echo === IP CONFIG === >> loot.txt & ipconfig >> loot.txt
        echo === USERS === >> loot.txt & net user >> loot.txt
        ```

* **タスク 3: Command Injection Verification (インジェクションの動作確認)**
    * **目的**: 脆弱性がある箇所で OS コマンドが実行できるかを、`echo` で文字を出させて目視確認する。

## エラーメッセージとトラブルシューティング

1.  **ECHO は <ON> です。 / <OFF> です。**
    * **原因**: 引数なしで `echo` とだけ打った場合、現在のエコー設定が表示される。
    * **解決策**: 空白行を出したい場合は、`echo.`（ピリオドを付ける）を使用してください。

2.  **特殊文字（&, |, <, >）が表示できない**
    * **原因**: これらはコマンドプロンプトの制御記号として解釈されるため。
    * **解決策**: エスケープ記号 `^` を使用してください。（例：`echo ^<hello^>`）

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の漏洩**:
    * バッチファイル内で `echo` を多用していると、実行時にパスワードや API キーなどが画面（またはリダイレクトされたログ）に表示されてしまうリスクがあります。
* **偽装メッセージ (Social Engineering)**:
    * `echo System Update Error: Please call extension 123...` のように表示させ、ユーザーを騙すスクリプトに悪用されます。

### LOLBAS における利用例

* **機能**: `AWL Bypass`, `Execute`
* **特性**: `echo` 自体はバイナリではありませんが、`cmd.exe /c echo ...` を介して別のバイナリやスクリプトを生成・実行するフローの起点として LOLBAS 手法に深く組み込まれます。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * **コマンドラインロギング**: 短時間に連続して `echo ... >> filename` を実行し、スクリプトファイルを生成しているプロセスを監視します。これはドロッパー（Dropper）の典型的な挙動です。
    * **イベントID 4688**: 生成された実行ファイルが直後に実行されていないか相関分析を行います。

## 注意点・補足

* **@echo off**: バッチファイルの先頭に `@` を付けることで、`echo off` というコマンド自体が表示されるのも防ぐことができます。
* **リダイレクトの挙動**: `> file` は上書き、`>> file` は追記です。間違えると重要なログを消してしまうため、運用時は特に注意が必要です。
