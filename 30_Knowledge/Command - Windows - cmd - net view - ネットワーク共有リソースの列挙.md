# Command - Windows - cmd - net view - ネットワーク共有リソースの列挙

## 概要

`net view` コマンドは、指定したコンピュータが公開している共有リソースの一覧、または現在のネットワーク（ドメインやワークグループ）内に存在するコンピュータの一覧を表示するために使用されます。
SMB (Server Message Block) プロトコルを利用して、ネットワーク上の「目に見える」資産を特定するための診断ツールです。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `net share` | **自分自身**のコンピュータが公開している共有フォルダを管理する。 | 自端末確認 |
| `Get-SmbShare` | PowerShell版。リモートコンピュータの共有情報をオブジェクトとして詳細に取得可能。 | PS環境 |
| `nmap -p 445` | 外部ツール。SMBポートが開いているホストを高速にスキャンする。 | 広範囲スキャン |
| `arp -a` | 通信が発生したIPとMACのアドレス対応を表示。コンピュータ名の解決は行わない。 | L2偵察 |

## よく連携されるコマンド

### シナリオ例: 共有フォルダの探索からマウントまで (管理者・攻撃者共通)

* **目的**: ターゲットサーバーにどのような共有があるかを確認し、特定のフォルダをドライブとして割り当てる。
* **連携コマンド**: `net view`, `net use`, `dir`
* **解説**: `net view` で共有名を確認し、`net use` で接続を確立します。
* **コマンド例**:
    ```cmd
    REM ターゲットサーバー "FS01" の共有フォルダを確認
    net view \\FS01

    REM 確認した共有名 "Backup" を Z ドライブにマウント
    net use z: \\FS01\Backup
    ```



## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | 現在のドメインまたはネットワーク内のコンピュータ一覧を表示する。 |
| `\\computername` | 指定した特定のコンピュータが公開している共有リソース（フォルダ、プリンタ）を表示する。 |
| ⭐ `/domain[:name]` | 指定したドメイン（省略時は現在のドメイン）内のコンピュータを一覧表示する。 |
| `/all` | Windows 以外のホストを含む、すべての共有を表示しようとする（一部環境のみ）。 |
| `/cache` | 指定したコンピュータのリソースのオフラインクライアント設定を表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しく構築したファイルサーバーの共有設定が、ネットワーク越しに正しく見えているか確認する。
* **例**:
    ```cmd
    REM 自サーバーがネットワークからどう見えているかセルフチェック
    net view \\%COMPUTERNAME%
    ```

### 2. ブルーチーム視点

* **タスク**: 意図しない「公開共有」や「匿名アクセス可能なリソース」が放置されていないか巡回監査する。
* **例**:
    ```cmd
    REM ネットワーク上の全端末をリストアップし、不審な端末が紛れていないか確認
    net view /domain
    ```

### 3. レッドチーム視点 (偵察と列挙)

* **タスク 1: Network Host Enumeration (ホストの列挙)**
    * **目的**: ネットワーク内の「生きている」Windows マシンのリストを取得し、攻撃対象（ターゲット）を選定する。
    * **解説**: Active Directory 環境では、コンピュータオブジェクトの一覧が返ってくるため、サーバーの命名規則から役割（例: `SQL`, `MAIL`, `DC`）を推測できます。
    * **コマンド例**:
        ```cmd
        net view /domain
        ```
    * **実務ログ例**:
        ```text
        サーバー名             注釈
        -------------------------------------------------------------------------------
        \\DC01                 Primary Domain Controller
        \\FILE-SRV-01          File Server
        \\SQL-PROD-01          Database Server
        \\WIN10-CLIENT-05      Marketing PC
        コマンドは正常に終了しました。
        ```

* **タスク 2: Share Enumeration (共有リソースの列挙)**
    * **目的**: ターゲットサーバー内に、アクセス権の不備（読み取り可能）がある共有フォルダを探す。
    * **解説**: 特に `Backup`, `Configs`, `Deploy`, `Public` といった名前の共有は、機密情報が含まれている可能性が高いため優先的に調査します。
    * **コマンド例**:
        ```cmd
        net view \\FILE-SRV-01
        ```
    * **実務ログ例**:
        ```text
        \\FILE-SRV-01 の共有リソース
        共有名  タイプ  注釈
        -------------------------------------------------------------------------------
        Backup  Disk    Nightly Backup Source
        Users   Disk    User Profiles
        Public  Disk    Temporary File Exchange
        コマンドは正常に終了しました。
        ```

* **タスク 3: 匿名/ゲストアクセスの確認 (Null Session)**
    * **目的**: 認証情報を持っていない状態で、どこまで情報を引き出せるか確認する。
    * **解説**: 最新の OS では制限されていますが、レガシーな環境や設定ミスがある場合、ユーザー名なしで共有一覧を取得できることがあります。
    * **コマンド例**:
        ```cmd
        REM ユーザー名・パスワードを空でセッションを張り、net view を試行
        net use \\TargetIP\IPC$ "" /u:""
        net view \\TargetIP
        ```

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: ターゲットの共有一覧を閲覧する権限がない。
    * **解決策**: 有効なドメインユーザーでログインするか、`net use` で認証を確立してから再試行してください。

2.  **System error 6118 has occurred. (The list of servers for this workgroup is not currently available)**
    * **原因**: Computer Browser サービスが停止している、またはファイアウォールでブロックされている。
    * **解決策**: ターゲットの IP アドレスを直接指定して `net view \\IP` を試すか、サービスの状態を確認してください。

## 環境変数と設定ファイル

* **SMB (TCP 445 / UDP 137, 138 / TCP 139)**: `net view` の動作にはこれらのポートが開放されている必要があります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **情報漏洩 (Network Service Scanning - T1046):**
    * **悪用シナリオ**: 攻撃者は侵入後、まず `net view /domain` で組織全体のサーバーリストを把握し、次に各サーバーに対して `net view \\Name` を実行して、機密データ（給与情報、個人情報、ネットワーク構成図等）が格納されていそうな場所を特定します。
* **パスワード情報の露出**:
    * 共有フォルダ内に `unattend.xml` や `web.config`、スクリプトファイル等が置かれている場合、`net view` でそれを見つけ、中身を読み取って特権アカウントのパスワードを窃取する（Looting）流れが非常に一般的です。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: Windows 標準の管理ツールであるため、ウイルス対策ソフトに検知されることなく、正当な「管理業務」を装ってネットワーク探索を行うことができます。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **アクセス権の最小化**: 「Everyone」や「Authenticated Users」に対する共有の参照権限を厳格に管理する。
    * **SMB 署名の強制**: 中間者攻撃（MITM）による共有アクセスの乗っ取りを防止するため、SMB 署名を必須にする。
    * **Computer Browser サービスの無効化**: 現代の環境（AD）ではブラウジング機能は必須ではないため、必要に応じて無効化してネットワーク情報の露出を抑える。
* **Detection (検知)**:
    * **不審な共有列挙の監視**: 単一の端末から短時間に多数のサーバーに対して `net view`（SMB Tree Connect）が行われている場合、内部偵察の兆候として検知します。
    * **イベントID 5140**: 共有オブジェクトへのアクセス。不自然なユーザーによるアクセスを監視。

## 注意点・補足

* **名前解決**: `net view` は NetBIOS または DNS 名を使用します。IP アドレスを直接指定して `net view \\192.168.1.10` と打つことも可能ですが、環境によってはエラーになる場合があります。
* **レガシー性**: `net view` は古いブラウジングプロトコルに依存している部分があり、最新の Windows 10/11 ネットワーク環境（ネットワーク探索が無効な環境）では期待通りに動作しないことが増えています。その場合は PowerShell の `Get-SmbShare` 等の使用を検討してください。
