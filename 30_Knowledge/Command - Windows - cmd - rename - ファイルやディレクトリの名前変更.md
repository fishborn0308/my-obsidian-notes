# Command - Windows - cmd - rename - ファイルやディレクトリの名前変更

## 概要

`ren` (または `rename`) コマンドは、ファイルやディレクトリの名前を変更するために使用されます。

このコマンドの最大の特徴は、**「移動」は行わず「名前の変更」のみを行う**点にあります。そのため、新しいパス（別のディレクトリ）を指定することはできません。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `move` | `move` は別ディレクトリへの移動が可能。`ren` は同一ディレクトリ内のみ。 | 移動を伴うなら `move` |
| `New-Item` (PS) | PowerShellのコマンドレット。名前変更には `Rename-Item` を使用する。 | PS環境なら `Rename-Item` |

## よく連携されるコマンド

`ren` はワイルドカード (`*`, `?`) をサポートしており、特定のパターンを持つファイル群を一括でリネームする際に重宝されます。

### シナリオ例: 拡張子の一括変更 (インフラ・運用)

* **目的**: 誤った拡張子で生成された大量のログファイルを一括で修正する。
* **連携コマンド**: `dir`
* **解説**: ワイルドカードを使用して、特定の拡張子を持つすべてのファイルを対象にします。
* **コマンド例**:
    ```cmd
    REM すべての .txt ファイルを .log に変更
    ren *.txt *.log
    ```

## スイッチ/オプション説明

`ren` コマンドには特別なスイッチ（`/s` など）は存在しません。非常にシンプルな構文です。

| 引数 | 説明 |
| :--- | :--- |
| `[ドライブ:][パス]ファイル名1` | 名前を変更する元のファイルまたはディレクトリを指定します。 |
| `ファイル名2` | 新しい名前を指定します。**ここにパス（フォルダ名）を含めることはできません。** |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 一時ファイルの正式採用
* **組み合わせ**: `if exist`
* **解説**: スクリプト等で生成した一時的な設定ファイルを、既存ファイルと入れ替える前にリネームします。
* **例**:
    ```cmd
    REM 古い設定を .old にし、新設定を正式採用する
    if exist config.conf.new (
        ren config.conf config.conf.old
        ren config.conf.new config.conf
    )
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠ファイルのタイムスタンプ保持を意識したリネーム
* **組み合わせ**: `%DATE%`, `%TIME%`
* **解説**: ファイル内容を変更（上書き）せずに、名前を変えることで時系列順に整理します。
* **例**:
    ```cmd
    REM 侵害調査中に見つけた怪しいログを保存用にリネーム
    ren "unusual.log" "investigation_%DATE:/=-%_%TIME::=-%.log"
    ```

### 3. レッドチーム視点

* **タスク**: 拡張子偽装によるペイロードの隠蔽
* **組み合わせ**: `assoc`, `ftype`
* **解説**: 実行ファイルを一見無害な拡張子（`.txt` や `.jpg`）に変更して転送し、実行直前に `.exe` や `.bat` に戻します。
* **例**:
    ```cmd
    REM 転送済みの画像ファイル(偽)をバッチファイルに戻して実行
    ren backup_img.jpg exploit.bat
    start exploit.bat
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `重複するファイル名があるか、ファイルが見つかりませんでした。`
    * **考えられる原因**:
        * 指定した「新しい名前」が既に存在している。
        * 「元の名前」のファイルが存在しない。
    * **解決策**: 移動先に同名ファイルがないか `dir` で確認する。

2.  **エラーメッセージ例 2**: `指定されたパスが見つかりません。` (新しい名前側にパスを含めた場合)
    * **考えられる原因**: `ren old.txt C:\NewFolder\new.txt` のように、新しい名前側にディレクトリパスを含めてしまった。
    * **解決策**: `ren` は名前のみを変更するコマンドです。移動が必要な場合は `move` を使用してください。

## 環境変数と設定ファイル

`ren` 自体に影響を与える固有の環境変数はありませんが、`%COMSPEC%` (cmd.exe) の内部コマンドとして動作します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **Evasion (回避)**: アンチウイルス(AV)やEDRが監視している特定の拡張子を一時的に変更し、検知を回避する。
    * **DLL Search Order Hijacking**: 正規のDLLをリネームして無効化し、攻撃者の用意した悪意あるDLLを正規の名前にリネームして配置する。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `ren` は `cmd.exe` の内部コマンドであるため、独立したバイナリとしての LOLBAS 登録はありませんが、攻撃チェーンにおける「偽装」のステップで必ずと言っていいほど登場します。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * 重要なシステムファイルの書き込み・変更権限を制限する（ACLの適切化）。
* **Detection (検知)**:
    * **イベントID 4663**: ファイルのリネーム（削除と作成のセットとして記録される場合もあります）を監視する。
    * 特に、`.exe` や `.dll` などの実行可能ファイルが短時間でリネームされている挙動をSIEMで相関分析する。

## 注意点・補足

* **ディレクトリ名の変更**: ファイルだけでなく、ディレクトリの名前も変更可能です（ただし、そのディレクトリ内のサブディレクトリへ「移動」させることはできません）。
* **ワイルドカードのテクニック**: `ren file????.txt new_????.txt` のように `?` を使うと、特定の位置の文字を維持したままリネームが可能です。
