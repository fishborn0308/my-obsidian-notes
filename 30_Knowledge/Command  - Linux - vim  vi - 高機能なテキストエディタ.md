# Command  - Linux - vim/ vi - 高機能なテキストエディタ

## 概要

`vim` (Vi IMproved) は、Unix系OSの標準エディタである `vi` を大幅に機能拡張した、高機能なテキストエディタです。`vim` は「ノーマルモード」「挿入モード」「コマンドラインモード」といった複数の**モード**を切り替えて操作するのが最大の特徴です。マウスを使わずにキーボードだけで高速なテキスト編集が可能であり、特にサーバー管理やプログラミングにおいて、その強力な機能と普遍性から必須のツールとされています。

現在、多くのシステムでは `vi` コマンドを実行すると、実際には `vim` が起動します。

(出自: `vim` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`nano`** | モードの概念がなく、常に文字を入力できるシンプルなエディタ。初心者でも直感的です。`vim` は習熟が必要ですが、遥かに高速で複雑な編集が可能です。 | `nano` (初心者) / `vim` (熟練者) |
| **`emacs`** | `vim` と並ぶ高機能エディタの代表格。`vim` とは設計思想が異なり、Ctrl/Altキーを多用します。「エディタの形をしたOS」とも呼ばれます。 | - |
| **`sed`** | **非対話的**なストリームエディタ。スクリプト内でテキストを自動処理する場合に使われます。`vim` は**対話的**にファイルを編集します。 | - |

## よく連携されるコマンド

`vim` は、`git` や `crontab -e` などのコマンドからデフォルトのエディタとして呼び出される形で連携します。

### シナリオ例: Gitのコミットメッセージを編集する (インフラ構築・運用視点)

* **目的**: `git commit` を実行した際に `vim` を起動し、規約に沿ったコミットメッセージを記述する。
* **連携コマンド**: `git`
* **解説**: 環境変数 `EDITOR` または `VISUAL` に `vim` を設定しておくと、`git commit` などのコマンドがエディタを必要とするときに自動的に `vim` が起動します。
* **コマンド例**:

    ```bash
    # .bashrc などでデフォルトエディタを設定
    export EDITOR=vim

    # git commit を実行すると vim が起動する
    git commit
    ```

## ⌨️ ショートカットまとめ (ノーマルモード)

`vim` の操作は、**ノーマルモード**でコマンド（ショートカット）を実行するのが基本です。どのモードにいるか分からなくなったら、まず `Esc` キーを押してノーマルモードに戻りましょう。

### 移動 (Navigation)

| キー | 機能 |
| :--- | :--- |
| ⭐ `h` `j` `k` `l` | 左、下、上、右 (カーソルキーの代わり) |
| `w` / `b` | 次の単語 / 前の単語 の先頭へ |
| `^` (or `0`) / `$` | 行頭 / 行末 へ |
| ⭐ `gg` / `G` | ファイルの先頭 / ファイルの末尾 へ |
| `<N>G` | `N` 行目へジャンプ (例: `10G` は10行目へ) |

### 編集 (Editing)

| キー | 機能 |
| :--- | :--- |
| ⭐ `i` / `a` | カーソル位置の前 / 後 から**挿入モード**へ |
| `o` / `O` | 現在行の下 / 上 に新しい行を挿入して**挿入モード**へ |
| ⭐ `x` / `dw` | 1文字削除 / 1単語削除 |
| ⭐ `dd` | 1行削除 (カット) |
| `yy` | 1行コピー (ヤンク) |
| `p` / `P` | カーソル位置の後 / 前 に貼り付け (ペースト) |
| ⭐ `u` / `Ctrl + r`| 元に戻す (Undo) / やり直し (Redo) |

### ビジュアルモード (Visual Mode - 範囲選択)

| キー | 機能 |
| :--- | :--- |
| `v` | 文字単位のビジュアルモード開始 |
| `V` | 行単位のビジュアルモード開始 |
| `Ctrl + v` | 矩形選択（ブロック選択）モード開始 |

### コマンドラインモード (Command-line Mode) (`:`から開始)

| コマンド | 機能 |
| :--- | :--- |
| ⭐ `:w` | 保存 (write) |
| ⭐ `:q` | 終了 (quit) |
| ⭐ `:wq` | 保存して終了 |
| ⭐ `:q!` | 変更を破棄して強制終了 |
| ⭐ `:%s/old/new/g` | ファイル全体 (`%`) で `old` を `new` に**一括置換** |
| `:set number` | 行番号を表示 |
| `/pattern` | `pattern` を前方検索 |
| `n` / `N` | 次 / 前 の検索結果へ移動 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイル内の特定のディレクティブを検索し、値を変更する。
* **組み合わせ**: (ノーマルモード) `/pattern` → `n` → `i`
* **解説**: `/listen` で特定のディレクティブを検索し、`n` で目的の箇所に移動。`i` で挿入モードに入り編集後、`Esc` でノーマルモードに戻り、`:wq` で保存して終了します。
* **例**:

    ```bash
    # Nginxの設定ファイルをvimで開く
    sudo vim /etc/nginx/nginx.conf
    ```

### 2. ブルーチーム視点

* **タスク**: ログファイルや不審なスクリプトを、誤って変更するリスクなしに調査する。
* **組み合わせ**: `vim -R <filename>` (または `view <filename>`)
* **解説**: `-R` オプションでファイルを読み取り専用で安全に開きます。`:set number` で行番号を表示させ、`/` や `?` でIOC（IPアドレス、不審な関数名など）を検索します。
* **例**:

    ```bash
    # 認証ログを読み取り専用で開く
    vim -R /var/log/auth.log
    ```

### 3. レッドチーム視点

* **タスク**: 最小構成のサーバー上で、攻撃用のスクリプトやペイロードをその場で作成・編集する。
* **組み合わせ**: `vim /tmp/payload.py` → `i` → (ペースト) → `<Esc>` → `:wq`
* **解説**: GUI環境がなく、`nano` すら入っていないような最小構成のサーバーでは、プリインストールされている `vi`/`vim` を使ってペイロードを修正・調整する能力が必須となります。
* **例**:

    ```bash
    # リバースシェルスクリプトを/tmpに作成
    vim /tmp/reverse_shell.sh
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **現象**: **`vim` を終了できない！**
    * **考えられる原因**: 挿入モードのまま `:q` を入力している、またはファイルを変更したのに `:w` (保存) をしていない。
    * **解決策**:
        1. まず **`Esc` キーを数回押し**、確実にノーマルモードに戻ります。
        2. 変更を保存して終了する場合: **`:wq`** と入力して `<Enter>`。
        3. 変更を破棄して強制終了する場合: **`:q!`** と入力して `<Enter>`。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* ⭐ **`~/.vimrc`**: `vim` の個人用の設定ファイル。`set number` や `syntax on` などを記述してカスタマイズします。
* **`/etc/vim/vimrc`**: システム全体に適用される `vim` の設定ファイルです。

### 影響を与える環境変数

* ⭐ **`EDITOR`**, `VISUAL`: `crontab -e` などのコマンドが、どのエディタを起動するかの決定に使います。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `vim` は非常に多機能なため、過去には脆弱性が発見されたことがあります（例: CVE-2019-12735 Modelines脆弱性）。
* **悪用シナリオ**: 攻撃者は、システムに侵入した後、`vim` を使って設定ファイルを改ざんし、バックドアを仕掛けたり、ログを改変して侵入の痕跡を消したりします。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo vim` を実行できる権限を持っている場合、`vim` の内部からシェルを起動 (`:shell`) することで、簡単に root 権限を奪取できます。
* **参照**: `https://gtfobins.github.io/gtfobins/vim/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定で、ユーザーに `vim` のようなエディタの `sudo` 権限を安易に与えない。`sudoedit` を使うなど、より安全な方法を検討する。
* **Detection (検知)**: FIMツールで、`.vimrc` や `.viminfo` の変更を監視する。`vim` からの不審なプロセス生成がないかEDRで監視する。

## 注意点・補足

* **`.vimrc` の活用**: 自分好みの設定を `~/.vimrc` に記述していくことで、`vim` は非常に強力で使いやすいエディタになります。行番号の常時表示 (`set number`) やシンタックスハイライトの有効化 (`syntax on`) は多くのユーザーが設定しています。
