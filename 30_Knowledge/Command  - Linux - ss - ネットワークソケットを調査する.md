# Command  - Linux - ss - ネットワークソケットを調査する

## 概要

`ss` (socket statistics) は、`iproute2` パッケージに含まれる、システムの**ネットワークソケット（接続）**に関する情報を表示するためのモダンなコマンドラインツールです。実行中のTCP/UDP接続、待ち受け（LISTEN）状態のポート、UNIXソケットなどを詳細に調査できます。

`ss` は、旧来の `netstat` コマンドの代替として開発され、カーネルのnetlinkソケットから直接情報を取得するため、特に接続数が多い高負荷なサーバーにおいて**`netstat` よりも遥かに高速に**動作します。

(出自: `iproute2` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`netstat`** | **旧世代のコマンド**。`ss` は `netstat` の全ての機能を、より高速かつ高機能に提供します。多くのオプション (`-tulpn`など) に互換性があります。 | `ss` |
| **`lsof -i`** | 開かれているネットワークソケットを、プロセスの詳細情報と共に表示します。`ss -p` と似ていますが、`lsof` はよりファイルディスクリプタの観点から情報を提供します。 | `ss` (ネットワーク特化) |

## よく連携されるコマンド

`ss` は `grep` とも連携できますが、それ自体が強力なフィルタ機能を持っているため、単体で完結することが多いです。

### シナリオ例: 特定の条件で接続をフィルタリングする (インフラ構築・運用視点)

* **目的**: WebサーバーへのアクティブなHTTPS接続の状況を確認する。
* **連携コマンド**: `ss` (単体)
* **解説**: `ss` の強力なフィルタ構文を使い、`ESTABLISHED` 状態のTCP接続のうち、宛先ポートが443 (HTTPS) のものだけを表示します。`grep` を使うよりも正確で効率的です。
* **コマンド例**:

		```bash
    # 宛先ポートが443の確立済みTCP接続を表示

    ss -t state established dport = :443

    ```


## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示対象** | |
| ⭐ `-t` | **TCP** ソケットを表示します。 |
| ⭐ `-u` | **UDP** ソケットを表示します。 |
| ⭐ `-l` | **LISTEN** (待ち受け) 状態のソケットのみを表示します。 |
| ⭐ `-a` | LISTEN状態とそうでないものを含め、**全ての**ソケットを表示します。 |
| **表示形式** | |
| ⭐ `-n` | ホスト名やサービス名を解決せず、**数値** (IPアドレス、ポート番号) のまま表示します。 |
| ⭐ `-p` | ソケットを使用しているプロセスの**名前とPID**を表示します。(`root`権限が必要です) |
| `-e` | ソケットに関する**拡張・詳細情報**（UID, iノードなど）を表示します。 |
| `-o` | TCPタイマー情報を表示します。 |
| **フィルタ構文** | |
| `state <state>`| `established`, `listening`, `closed` など、TCPの状態でフィルタします。 |
| `dport <port>`| 宛先ポート (destination port) でフィルタします。 |
| `sport <port>`| 送信元ポート (source port) でフィルタします。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーでどのサービスがどのポートで待ち受けているかを確認する。
* **組み合わせ**: `sudo ss -tulpn`
* **解説**: 「TCP/UDPのLISTEN状態の、PID付きの数値表示」を意味し、`netstat -tulpn` と同じ感覚で使えます。サービス起動後の確認やポート競合のトラブルシューティングに不可欠です。
* **例**:

		```bash

    sudo ss -tulpn

    # State    Recv-Q   Send-Q     Local Address:Port      Peer Address:Port   Process
    # LISTEN   0        128            0.0.0.0:22             0.0.0.0:* users:(("sshd",pid=1234,fd=3))
    # LISTEN   0        511          127.0.0.1:3306           0.0.0.0:* users:(("mysqld",pid=5678,fd=30))
    ```


### 2. ブルーチーム視点

* **タスク**: 侵害されたホスト上で、不審な外部接続がないか調査する。
* **組み合わせ**: `sudo ss -anp`
* **解説**: **ライブフォレンジックの基本**。全ての接続を数値とプロセス情報付きで表示します。`ESTABLISHED` 状態のリストの中から、見慣れない外部IPアドレスへの接続を探し、関連するプロセス名を特定することで、悪意のあるプロセスを突き止めます。
* **例**:

		```bash
    # 全ての確立済みTCP接続をプロセス情報付きで表示

    sudo ss -t -p state established

    ```


### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、システムのネットワーク活動を完全にマッピングする。
* **組み合わせ**: `ss -anp`
* **解説**: `LISTEN` 状態のポート（特に `127.0.0.1` でのみリッスンしている内部サービス）を特定したり、`ESTABLISHED` 状態の接続から、このサーバーがどの他の内部サーバーと通信しているかを把握したりして、ラテラルムーブメント（横展開）の標的を探します。
* **例**:

		```bash
    # 待ち受けポートと確立済みの接続をリストアップ

    ss -anp

    ```


## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **現象**: **`-p` オプションを付けたのにPID/プログラム名が表示されない。**
		* **考えられる原因**: `root` 権限なしで実行しています。他のユーザーが所有するプロセスの情報を閲覧するには特権が必要です。
		* **解決策**: `sudo ss ...` のように `sudo` を付けて実行してください。

## 環境変数と設定ファイル

* `ss` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ss` 自体ではなく、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: `ss` の出力は、システムのネットワーク構成、稼働中のサービス、外部との通信相手など、攻撃者にとって非常に価値の高い偵察情報です。

### GTFOBins / LOLBAS における利用例

`ss` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を適用し、不要なユーザーに `ss` の実行権限を与えない。
* **Detection (検知)**: `auditd` で `ss` コマンドの実行を監視し、不審な偵察活動がないかを確認する。EDR製品は通常、`ss` のような偵察コマンドの実行を検知・記録します。

## 注意点・補足

* **`netstat` からの移行**: `netstat` に慣れている場合でも、`ss` に移行することを強く推奨します。基本的なオプションは互換性があるため、`netstat -tulpn` を `ss -tulpn` に置き換えることから始めるのが簡単です。
