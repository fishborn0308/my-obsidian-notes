# Command  - Linux - df - ファイルシステムのディスク空き容量を表示する

## 概要

`df` (disk free) は、マウントされている全てのファイルシステムについて、その**ディスク容量の合計、使用量、空き容量、使用率**を報告するコマンドです。サーバーのディスクが一杯になっていないかを監視したり、どのパーティションに空きがあるかを確認したりするための、最も基本的なシステム管理コマンドの1つです。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`du`** | (disk usage) **ファイルやディレクトリ単位**で、それらがどれくらいのディスク容量を消費しているかを計算します。`df` は**ファイルシステム全体**の使用状況を表示します。 | - |
| **`lsblk`** | ブロックデバイスをツリー形式で一覧表示し、物理的な構成を確認するのに便利ですが、使用率は表示しません。 | - |
| **`fdisk -l`**| ディスクのパーティションテーブル情報を表示しますが、使用状況は分かりません。 | - |

## よく連携されるコマンド

`df` は、ディスク容量の問題を発見した後に、原因を特定するための `du` や `lsof` と連携して使われます。

### シナリオ例: ディスク容量逼迫の原因を特定する (インフラ構築・運用視点)

* **目的**: `df` でディスク使用率が高いことを確認した後、`du` を使ってどのディレクトリが容量を消費しているかを特定する。
* **連携コマンド**: `df`, `du`, `sort`, `head`
* **解説**: まず `df -h` で全体像を把握し、問題のファイルシステム（マウントポイント）を特定します。次にそのマウントポイントに移動し、`du` と `sort` を組み合わせることで、容量を圧迫している犯人を見つけ出します。
* **コマンド例**:

```bash
# 1. まずはファイルシステム全体の状況を確認
df -h

# 2. /var の使用率が高いことが判明したと仮定し、/var に移動
cd /var

# 3. /var 直下で最も容量を消費しているディレクトリトップ5を表示
sudo du -sh * | sort -rh | head -n 5
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `-h`, `--human-readable` | サイズを人間が読みやすい単位 (K, M, G) で表示します。 |
| `-H`, `--si` | `1024` の累乗 (KiB, MiB) ではなく、`1000` の累乗 (KB, MB) でサイズを表示します。 |
| `-T`, `--print-type` | ファイルシステムの種類 (`ext4`, `xfs`, `nfs`など) を表示します。 |
| **情報種別** | |
| ⭐ `-i`, `--inodes` | ブロックの使用状況の代わりに、**iノード**の使用状況を表示します。 |
| **フィルタリング** | |
| `-x`, `--exclude-type`| 指定した種類のファイルシステム（`tmpfs`など）を結果から除外します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 全マウントポイントのディスク使用状況を素早く確認する。
* **組み合わせ**: `df -h`
* **解説**: 事実上の標準的な使い方。人間が読みやすい形式で、サーバーの定期的なヘルスチェックや、ディスク容量に関するアラートへの対応を行います。
* **例**:

```bash
# 読みやすい形式でディスク使用状況を確認
df -h
```

### 2. ブルーチーム視点

* **タスク**: インシデントレスポンスの初期調査で、ファイルシステムの構成と状態を把握する。
* **組み合わせ**: `df -hT`
* **解説**: ファイルシステムの種類 (`-T`) も併せて表示することで、ホストの構成をより詳細に把握します。異常な使用率のパーティションや、意図しないネットワーク共有 (`nfs`など) がマウントされていないかを確認します。
* **例**:

```bash
# ファイルシステムタイプも併せてディスク状況を確認
df -hT
```

### 3. レッドチーム視点

* **タスク**: 侵入したホストの偵察の一環として、ディスク構成と空き容量を確認する。
* **組み合わせ**: `df -h`
* **解説**: 攻撃ツールや窃取したデータを一時的に保存するのに十分なスペースがあるかを確認します。また、`/mnt/backup` や `/home` のようなデータが保存されていそうなパーティションを探し、次の探索対象とします。
* **例**:

```bash
# 偵察のためディスク情報を表示
df -h
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`du` で計算した使用量と `df` の使用量が一致しない。**
		* **考えられる原因**: あるプロセスがファイルを開いている状態で、そのファイルが削除された場合に発生します。`du` は削除されたファイルをカウントしませんが、`df` はプロセスがファイルハンドルを掴んでいる限り、ディスク上のブロックがまだ確保されていることを報告します。
		* **解決策**: `sudo lsof | grep deleted` コマンドを実行して、削除済みのファイルを開き続けているプロセスを特定し、そのプロセスを再起動または終了させることで、ディスクスペースが解放されます。

2. **現象**: **iノード不足でファイルが作成できない。**
		* **考えられる原因**: `df -h` では空き容量があるように見えるのに、「No space left on device」エラーが出る場合、ディスクブロックは余っていても、ファイルのメタデータを管理するiノードを使い果たしている可能性があります。
		* **解決策**: `df -i` でiノードの使用率を確認します。大量の小さなファイル（セッションファイル、キャッシュなど）が原因であることが多いため、`find` コマンドなどで不要なファイルを削除します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `df` 自体は読み取り専用のコマンドであり、脆弱性を持つ可能性は低いです。
* **悪用シナリオ**: 攻撃者は意図的に巨大なファイルを生成したり、ログを溢れさせたりしてファイルシステムを一杯にし、サービスを停止させる（**サービス妨害攻撃, DoS**）ことがあります。`df` は、このような攻撃の**症状を観測する**ためのツールです。

### GTFOBins / LOLBAS における利用例

`df` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ユーザーごとにディスククォータを設定する。`logrotate` でログファイルのサイズを適切に管理する。
* **Detection (検知)**: ZabbixやPrometheusなどの監視ツールで、ファイルシステムの使用率を常時監視し、しきい値を超えた場合にアラートを発報するように設定する。

## 注意点・補足

* **予約ブロック**: `ext4` などのファイルシステムは、デフォルトで全容量の5%程度を `root` ユーザー専用の予約領域として確保します。`df` の `Avail` は、この予約領域を差し引いた、一般ユーザーが利用可能な容量を示します。
