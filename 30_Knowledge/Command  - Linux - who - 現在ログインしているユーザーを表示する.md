# Command  - Linux - who - 現在ログインしているユーザーを表示する

## 概要

`who` は、現在システムにログインしているユーザーのセッション情報を一覧表示するコマンドです。「今、誰がこのサーバーを使っているか？」を素早く確認するための、最も基本的なユーティリティの1つです。このコマンドは、主に `/var/run/utmp` ファイルを読み取って情報を表示します。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`w`** | `who` の**高機能版**。`who` が表示する情報に加えて、システムの稼働時間、ロードアベレージ、そして各ユーザーが**何を実行しているか (WHAT)** まで表示します。 | `w` (詳細情報) |
| **`users`** | 現在ログインしているユーザーの**名前**だけをスペース区切りで一行で表示します。 | - |
| **`last`** | **過去の**ログイン履歴を表示します。`who` は**現在の**アクティブなセッションを表示します。 | - |

## よく連携されるコマンド

`who` は、出力を `wc -l` にパイプすることで、現在のログインセッション数を数えるのによく使われます。

### シナリオ例: 現在のログインセッション数をカウントする (インフラ構築・運用視点)

* **目的**: 現在システムにいくつのアクティブなログインセッションがあるかを素早くカウントする。
* **連携コマンド**: `wc`
* **解説**: `who` の出力は1行1セッションなので、その行数を `wc -l` で数えることで、現在のセッション数が分かります。
* **コマンド例**:

		```bash
		# 現在のログインセッション数を表示

		who | wc -l

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `(引数なし)` | ログインユーザー名、端末、ログイン日時、接続元を表示します。 |
| `-b`, `--boot` | システムの最終起動日時を表示します。 |
| `-H`, `--heading` | 各カラムのヘッダー（見出し）を表示します。 |
| `-q`, `--count` | ログインしているユーザーの数と、ユーザー名の一覧のみを簡潔に表示します。 |
| **自分自身** | |
| `am i` | (または `am I`) このコマンドを実行している**自分自身**のセッション情報のみを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 重要な作業の前に、他にログインしている管理者がいないか確認する。
* **組み合わせ**: `who`
* **解説**: システムの再起動や、重要なサービスの再起動を行う前に `who` を実行します。もし他の管理者がログインしていれば、声をかけてから作業を開始することで、意図しない作業の衝突を防ぎます。
* **例**:

		```bash
		# 現在ログインしている全ユーザーを確認

		who -H

		```

## 2. ブルーチーム視点

* **タスク**: **ライブレスポンスの基本**。インシデント発生時に、攻撃者がログインしていないかを確認する。
* **組み合わせ**: `who`
* **解説**: 不正アクセスの疑いがあるサーバーで `who` を実行します。
	* **ユーザー名**: 見慣れないユーザーや、本来SSHログインしないはずのサービスアカウントがいないか。
	* **接続元**: 許可されていないIPアドレスや、不審なホスト名からの接続がないか。
		これらの情報は、攻撃者が現在アクティブに活動していることを示す強力な証拠となります。
* **例**:

		```bash
		# ライブ調査で現在の全セッションを監査

		who

		```

## 3. レッドチーム視点

* **タスク**: **侵入後の偵察 (Reconnaissance)**。
* **組み合わせ**: `who`
* **解説**: 攻撃者は侵入後、`w` や `who` を実行して、他にどのようなユーザー（特に管理者）がログインしているかを把握します。もし管理者がアクティブであれば、自身の活動をより隠密に行うか、管理者がログアウトするのを待ちます。また、接続元の情報から、組織のネットワーク構成（踏み台サーバーの存在など）を推測します。
* **例**:

		```bash
		# システムの活動状況を偵察

		who

		```

## 環境変数と設定ファイル

* `who` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/var/run/utmp`**: `who` が情報を読み取る、現在のログインセッションを記録したバイナリファイル。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `who` 自体ではなく、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: `who` コマンドの出力は、**誰が、どこから**システムにアクセスしているかを明らかにします。これは、攻撃者が次の行動を計画するための非常に価値の高い偵察情報です。

### GTFOBins / LOLBAS における利用例

`who` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `utmp` ファイルは、root権限を持つ攻撃者によって改ざん（utmp spoofing）され、`who` の結果から自身の存在を隠蔽される可能性があります。`auditd` で `who` コマンドの実行自体を監視することも有効です。

## 注意点・補足

* **`who am i`**: 自分がどのターミナル (`pts/N`) を使って、どのIPアドレスから、いつログインしたかを確認するのに便利なイディオムです。
