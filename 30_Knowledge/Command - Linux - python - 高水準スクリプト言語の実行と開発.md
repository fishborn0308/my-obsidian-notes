# Command - Linux - python - 高水準スクリプト言語の実行と開発

## 概要

`python`（および `python3`）は、汎用性の高いインタプリタ型プログラミング言語です。読みやすい構文と膨大な標準ライブラリにより、プロトタイプ開発から大規模システム、AI モデルの構築まで幅広く使われます。Linux 環境では、シェルスクリプトで書くには複雑すぎるタスクを自動化するための強力な武器になります。

## 🛠 `python` vs `bash`

| 特徴 | bash (Shell Script) | python |
| :--- | :--- | :--- |
| **得意分野** | シンプルなコマンド連携、ファイル操作。 | 複雑なロジック、データ処理、API連携。 |
| **可読性** | 短いものは良いが、長くなると難解。 | 構造化されており、長期メンテナンスに向く。 |
| **速度** | プロセス起動のオーバーヘッドがある。 | 実行速度自体は速いが、起動にわずかな時間がかかる。 |
| **依存関係** | ほぼ標準コマンドのみで完結。 | 外部ライブラリ（pip）が必要になることが多い。 |

## 主要な実行方法

| 形式 | コマンド例 | 説明 |
| :--- | :--- | :--- |
| ⭐️ インタラクティブ | `python3` | 対話型シェルを起動して、1行ずつコードを試す。 |
| ⭐️ スクリプト実行 | `python3 script.py` | 保存されたファイルを実行する。 |
| ⭐️ ワンライナー | `python3 -c "print(1+1)"` | ターミナルから直接コードを1行実行する。 |
| ⭐️ モジュール実行 | `python3 -m venv venv` | インストールされたモジュールをアプリとして実行する。 |

## 💡 モダンな Python 運用の三則 (2026 Edition)

1.  **システム Python を汚さない**: `sudo pip install` は厳禁。システムの動作を壊す恐れがあります。
2.  **仮想環境の徹底**: プロジェクトごとに `python3 -m venv` を使い、ライブラリを隔離します。
3.  **Python 2 は歴史の闇へ**: 2026年現在、使うべきは `python3` 一択です。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: 大量のリソース監視と通知。
* **解説**: AWS や GCP の SDK (Boto3等) を使い、クラウド上のサーバーの状態をチェックして Slack に通知する、といった高度な自動化を実現します。
* **ツール**: **Ansible**（Python で書かれた構成管理ツール）。

### 2. ブルーチーム視点

* **タスク**: 膨大なログの解析と異常検知。
* **解説**: `pandas` や `re` (正規表現) モジュールを使い、数ギガバイトのアクセスログから特定の攻撃パターンを高速に抽出します。
* **例**:
    ```python
    # 特定の攻撃文字列を含む行を抽出する簡易スクリプト
    import re
    with open('access.log') as f:
        for line in f:
            if re.search(r'union.*select', line, re.I):
                print(f"Detected potential SQLi: {line}")
    ```

### 3. レッドチーム視点

* **タスク**: カスタムエクスプロイト（攻撃コード）の作成。
* **解説**: 既存のツールでは対応できない独自の脆弱性を突くために、`requests` や `pwntools` ライブラリを使用して、ターゲットに合わせた精密な攻撃スクリプトを作成します。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **サプライチェーン攻撃 (Typosquatting)**: `pip install` する際に、有名なライブラリと名前が似た悪意あるパッケージ（例: `requesst` などの綴り間違い）をインストールしてしまうリスク。
* **機密情報のハードコード**: スクリプト内に API キーやパスワードを直書きしてしまうミス。

### 対応策・緩和策 (ブルーチーム視点)

* **`requirements.txt` の固定**: ライブラリのバージョンをハッシュ値まで含めて固定し、意図しない更新を防ぎます。
* **コードスキャン**: `bandit` などの静的解析ツールを使い、コード内のセキュリティ上の不備を自動でチェックします。
