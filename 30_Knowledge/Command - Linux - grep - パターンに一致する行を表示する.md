# Command - Linux - grep - パターンに一致する行を表示する

## 概要
`grep` は、ファイルまたは標準入力から特定の文字列パターン（正規表現）を検索し、一致した行を標準出力に表示します。パイプ（`|`）を使って他のコマンドの出力をフィルタリングするのが最も一般的な使い方です。
(出自: Unix のエディタ `ed` のコマンド `g/re/p` が由来)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `egrep` | `grep -E` と同等。拡張正規表現（`|`, `+`, `?` など）が使える。 | 複雑なパターン検索 |
| `fgrep` | `grep -F` と同等。正規表現を解釈せず、単なる「固定文字列」として爆速で検索する。 | 単純な文字列検索 |
| `ripgrep (rg)` | Rust 製のモダンな代替。デフォルトで再帰検索を行い、恐ろしく速い。 | 開発中のコード検索 |
| `awk / sed` | 検索だけでなく、列の抽出や置換など「加工」まで行う場合に使う。 | テキスト編集・集計 |

## よく連携されるコマンド
### シナリオ例: 特定のプロセスが動いているか確認 (運用視点)
* **目的**: システム内で `nginx` が実行中かどうかを調べる。
* **連携**: `ps aux`
* **解説**: 全プロセスリストから `nginx` を含む行だけを抜き出します。
* **コマンド例**:
    ```bash
    # 実行中の nginx プロセスを確認 (自分自身の grep は除外)
    ps aux | grep nginx | grep -v grep
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **表示のカスタマイズ** | |
| ⭐️ `-i`, `--ignore-case` | 大文字と小文字を区別せずに検索する。 |
| ⭐️ `-v`, `--invert-match` | 一致しなかった行（不一致行）を表示する。 |
| ⭐️ `-n`, `--line-number` | 行番号を付けて表示する。 |
| `-c`, `--count` | 一致した「行数」のみを表示する。 |
| **検索範囲の制御** | |
| ⭐️ `-r`, `-R`, `--recursive` | ディレクトリ内を再帰的に検索する。 |
| ⭐️ `-l`, `--files-with-matches` | 一致した行ではなく、一致した「ファイル名」だけを表示する。 |
| ⭐️ `-w`, `--word-regexp` | 単語全体が一致するものだけを検索する（`test` で `testing` に反応しない）。 |
| **前後のコンテキスト** | |
| `-A <n>`, `-B <n>`, `-C <n>` | 一致した行の「後(After)」「前(Before)」「前後(Context)」の n 行も表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: エラーログから深刻なメッセージを特定。
* **解説**: `error` や `critical` を抽出し、さらにその前後の状況を確認します。
* **例**:
    ```bash
    # エラーメッセージとその前後3行を表示
    grep -Ci 3 "error" /var/log/syslog
    ```

### 2. ブルーチーム視点
* **タスク**: SSH への不正アクセス試行の調査。
* **解説**: ログイン失敗（Failed password）のログから、攻撃元の IP アドレスをリストアップします。
* **例**:
    ```bash
    # ログイン失敗のログからユニークなIPアドレスを抽出
    grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq
    ```

### 3. レッドチーム視点
* **タスク**: 設定ファイルやバックアップからパスワードを探索。
* **解説**: `/etc` などのディレクトリから、`pass` や `key` を含むファイルを再帰的に探し出します。
* **例**:
    ```bash
    # /etc 配下のファイルから "password" を含むものを再帰的に検索（エラーは捨てる）
    grep -ri "password" /etc 2>/dev/null
    ```

## 正規表現（Regex）のキホン
* `.`: 任意の1文字
* `*`: 直前の文字の0回以上の繰り返し
* `^`: 行の先頭
* `$`: 行の末尾
* `[a-z]`: 指定した範囲のいずれかの文字

## エラーメッセージとトラブルシューティング

1.  **「Binary file ... matches」**:
    * **原因**: 検索対象がテキストではなくバイナリファイルだった。
    * **解決策**: バイナリもテキストとして扱いたい場合は `-a` オプションを付ける。

2.  **「grep: ...: Is a directory」**:
    * **原因**: ディレクトリを指定したが、再帰オプション（`-r`）を忘れている。
    * **解決策**: `grep -r "pattern" directory/` とする。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **情報漏洩**: `history` コマンドと `grep` を組み合わせることで、過去に誰かが実行したコマンド（平文のパスワードが含まれる場合がある）を簡単に盗み見ることができます。
* **Log Injection**: 攻撃者がログに「偽の検索ヒット」を誘発するような文字列を書き込み、管理者を誤誘導（Social Engineering）させることがあります。

### GTFOBins における利用例
`grep` に `sudo` 権限がある場合、本来読み取れないファイル（`/etc/shadow` 等）の内容を検索・表示することが可能です。
* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo grep を使って /etc/shadow から root のハッシュを読み取る
    sudo grep "root" /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/grep/`

## 注意点・補足
* **ダブルクォートの推奨**: 検索パターンにスペースや特殊文字が含まれる場合を考慮し、常にパターンは `"..."` で囲む習慣をつけましょう。
* **終了ステータス**: 
    * `0`: 一致あり
    * `1`: 一致なし
    * `2`: エラー発生
    この特性を利用して、スクリプトの `if` 文で「特定の文字列があるかないか」を判定するのによく使われます。
