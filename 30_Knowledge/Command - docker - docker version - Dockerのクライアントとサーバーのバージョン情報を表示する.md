# Command - docker - docker version - Dockerのクライアントとサーバーのバージョン情報を表示する

## 概要

`docker version` は、インストールされているDockerの**クライアント**（CLIツール）と、**サーバー**（Dockerデーモン）それぞれの詳細なバージョン情報を表示するコマンドです。単なるバージョン番号だけでなく、APIバージョン、Go言語のバージョン、OS/アーキテクチャなど、コンポーネントごとの詳細な情報を提供します。互換性の確認や、バグ報告、脆弱性情報を調査する際に、正確なバージョンを把握するために使用されます。

(出自: `docker` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`docker info`** | Dockerデーモンの**設定やリソース状況**（イメージ数、コンテナ数など）を詳細に表示します。`version` は**バージョン情報**に特化しています。 | `docker info` (全体像) |
| **`docker --version`** | **クライアントのバージョン番号のみ**を一行でシンプルに表示します。 | - |

## よく連携されるコマンド

`docker version` は、その出力をスクリプトで解析し、特定のバージョンであることを確認するテストなどで利用されます。

### シナリオ例: スクリプトでDocker Serverのバージョンを特定する (インフラ構築・運用視点)

* **目的**: Ansibleやシェルスクリプトで、Dockerホストが特定のバージョン以上であることを確認する。
* **連携コマンド**: `--format` (Go template)
* **解説**: `--format` オプションとGo言語のテンプレート構文を使い、JSON形式の出力からサーバーのバージョン番号だけをピンポイントで抜き出します。
* **コマンド例**:

		```bash
		# Docker Serverのバージョン番号だけを取得

		DOCKER_VERSION=$(docker version --format '{{.Server.Version}}')

		echo "Docker Server version is: $DOCKER_VERSION"

		```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](../linux/troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `Client: ... Server: Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?`
		* **考えられる原因**: Dockerクライアントは動作していますが、Dockerデーモンと通信できていません。デーモンが起動していないか、実行ユーザーにDockerソケットへのアクセス権限がありません。
		* **解決策**: `sudo systemctl status docker` でデーモンの状態を確認・起動します。`sudo` を付けてコマンドを実行するか、現在のユーザーを `docker` グループに追加します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **バージョンの陳腐化**。
* **悪用シナリオ**: 古いバージョンのDocker Engineを使い続けることは、重大なセキュリティリスクです。過去にはコンテナエスケープを可能にする深刻な脆弱性が何度も発見されています。`docker version` は、このリスクを評価するための最初のステップです。

### GTFOBins / LOLBAS における利用例

`docker version` は `docker` コマンドの一部として扱われます。`sudo docker` の権限があれば、任意の操作が可能です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: Docker Engineを常に最新の安定版にアップデートする運用を徹底する。
* **Detection (検知)**: `auditd` で `docker version` の実行を監視し、不審なユーザーによる偵察活動がないかを確認する。

## 注意点・補足

* **クライアントとサーバー**: `docker version` の出力が `Client` と `Server` に分かれていることは、Dockerがクライアント・サーバー型のアーキテクチャであることを明確に示しています。
