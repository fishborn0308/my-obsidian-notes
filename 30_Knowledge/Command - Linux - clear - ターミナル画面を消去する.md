# Command - Linux - clear - ターミナル画面を消去する

## 概要
`clear` は、現在のターミナルの表示内容を消去し、プロンプトを画面の最上行に移動させるコマンドです。実際にはデータを削除するのではなく、ターミナルの「スクロールバックバッファ」へ内容を押し上げる、または画面をリセットする制御文字を送信することで実現されています。
(出自: `ncurses` パッケージに含まれる。環境変数 `TERM` を参照して動作する)



## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `Ctrl+L` | シェルのショートカットキー。実行中の入力文字列を保持したまま画面をクリアできる。 | 作業中の視認性向上 |
| `reset` | 画面だけでなく、エコーバック設定や特殊文字の設定などターミナル自体を初期化する。 | ターミナルの表示が崩れた時 |
| `tput clear` | 低レベルな端末制御コマンド。スクリプト内で特定の端末種別に依存せずクリアしたい場合に便利。 | シェルスクリプト内 |
| `printf "\033c"` | エスケープシーケンスを直接送信。スクロールバックバッファも完全に消去できる場合がある。 | 完全な隠蔽が必要な時 |

## よく連携されるコマンド
### シナリオ例: ログのリアルタイム監視と画面リフレッシュ (運用視点)
* **目的**: 常に最新のシステム状態だけを画面に表示させ、古い情報を混同しないようにする。
* **連携コマンド**: `watch`
* **解説**: `watch` コマンドは内部的に `clear` と同様の処理を行い、定期的（デフォルト2秒）に画面を更新します。
* **コマンド例**:
    ```bash
    # 2秒ごとに画面をクリアして実行結果を更新表示
    watch -n 2 "df -h && free -m"
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐️ `(引数なし)` | 標準的な画面クリア。 |
| `-V` | バージョン情報を表示。 |
| `-x` | スクロールバックバッファを維持したまま画面をクリアする（最近の実装のデフォルト）。 |
| `-T <type>` | 指定した端末種別（TERM）として動作させる。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 手順書作成やデモンストレーション前の画面整理。
* **組み合わせ**: `clear` 実行後の作業開始。
* **解説**: 過去の試行錯誤（エラーログ等）が画面に残っていると、手順のスクリーンショットを撮る際にノイズとなり、混乱を招くため事前にクリアします。
* **例**:
    ```bash
    # 画面をきれいにしてから本番のコマンドを実行
    clear && ./deploy_script.sh
    ```

### 2. ブルーチーム視点
* **タスク**: フォレンジック調査時や共有端末での機密情報露出の抑制。
* **組み合わせ**: `alias clear='clear && history -c'`（非推奨だが一時的な対策）
* **解説**: パスワードや機密ファイルの内容を `cat` した後に画面に残しておくと、背後からの覗き見（ショルダーサーフィン）により情報が漏洩します。
* **例**:
    ```bash
    # 機密情報を見た後に即座に隠す
    cat secret_config.txt
    # (確認後)
    clear
    ```

### 3. レッドチーム視点
* **タスク**: ターゲット上での作業痕跡の「視覚的」な隠蔽。
* **組み合わせ**: `clear`
* **解説**: 侵入後、複数のコマンドを実行して画面が埋まった際、管理者がコンソールを覗き見た時に不審な文字列が目に入らないよう、こまめに画面をクリアします。ただし、`history` や `auditd` には残るため、あくまで対人（目視）の隠蔽です。
* **例**:
    ```bash
    # 攻撃コマンド実行後に実行
    python3 -c '...'
    clear
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `terminals database is inaccessible`
    * **考えられる原因**: 環境変数 `TERM` が設定されていない、または `ncurses` のデータベースファイルが不足している。
    * **解決策**: `export TERM=xterm` または `export TERM=linux` を実行する。

2.  **画面がクリアされない**:
    * **考えられる原因**: 非常に長いスクロールバック設定を持つターミナルエミュレータを使用している。
    * **解決策**: マウスホイールで上にスクロールすると古い内容が見えてしまう。完全に消したい場合はターミナル側の「Clear Scrollback」機能を使用するか、`reset` を試みる。

## 環境変数と設定ファイル

### 影響を与える環境変数
* **`TERM`**: 使用している端末の種類（xterm, vt100 等）。`clear` はこれを見てどのエスケープシーケンスを送るか決定します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: それ自体に脆弱性はほぼありませんが、**「消したつもり」による油断**が脆弱性となります。
* **悪用シナリオ**: 攻撃者が `clear` コマンドをエイリアスやパスの優先順位で上書きし、実行時に裏で `history` を盗み出すなどの工作が考えられます。

### GTFOBins における利用例
`clear` は単機能なため、特権昇格などの目的で GTFOBins に登録されることはまずありません。

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: 画面をクリアしても、物理的な端末のメモリやシェルの履歴には情報が残っていることを教育し、機密情報の扱いに注意させる。
* **Detection (検知)**: 不自然な頻度で `clear` が実行されている場合、何かを隠そうとしている（目視回避）可能性を考慮する。

## 注意点・補足
* **スクロールバック**: 大抵のモダンなターミナルでは、`clear` しても上にスクロールすれば内容が見えます。完全に消したい場合は `alias cls='printf "\033c"'` などの工夫が必要です。
* **リモート作業**: SSH経由でも有効ですが、接続が不安定で画面が崩れた場合は `clear` よりも `reset` の方が効果的です。
