# Command - Windows - cmd - hostname - コンピューター名の表示

## 概要

`hostname` コマンドは、現在のコンピューターの「ホスト名（ネットワーク上の名前）」を表示します。

これは DNS や NetBIOS で他から参照される名前であり、システムの識別において最も基本的な情報です。Windows 版の `hostname` は Linux 版と異なり、名前の表示機能に特化しており、このコマンド単体で名前を変更する機能は持っていません。

(出自: `Windows 標準搭載 - 外部コマンド (hostname.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `echo %COMPUTERNAME%` | 環境変数から取得。高速だが、変数が書き換えられている可能性が微かにある。 | 簡易確認 |
| `whoami` | 「ユーザー名」を表示。`hostname` は「マシン名」のみを表示。 | 権限確認 |
| `net config workstation` | コンピューター名に加え、ドメイン名やソフトウェアバージョンも表示。 | 詳細調査 |
| `wmic computersystem get name` | WMI 経由で取得。スクリプト等でのプロパティ取得に利用。 | プログラム利用 |

## よく連携されるコマンド

### シナリオ例: インシデント発生時の証拠保存（エビデンス収集）

* **目的**: 収集したログやスクリーンショットが「どの端末のものか」を確実に記録する。
* **連携コマンド**: `hostname`, `ipconfig`, `date`, `time`
* **解説**: 調査結果のファイル名の先頭にホスト名を付与したり、ログの冒頭に刻印したりします。
* **コマンド例**:
    ```cmd
    REM 調査結果ファイルにホスト名と日付を記録
    hostname > evidence.txt & ipconfig /all >> evidence.txt & date /t >> evidence.txt
    ```

## スイッチ/オプション説明

Windows の `hostname` コマンドには、実質的なオプションは存在しません。

| 引数 | 説明 |
| :--- | :--- |
| (なし) | 現在のホスト名を表示する。 |
| `/?` | ヘルプを表示する（唯一の有効なスイッチ）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートデスクトップ（RDP）や SSH で多数のサーバーに同時接続している際、今操作しているウィンドウが「開発機」か「本番機」かを即座に判別する。
* **コツ**: 迷ったらまず `hostname` を打つ習慣をつけることで、誤操作（オペミス）を防止します。

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: マルウェアの実行ログやネットワークトラフィックを解析する際、ホスト名を確認して、命名規則（例: `WS-Sales-01` など）から、その端末の役割や所属部署を推測する。

### 3. レッドチーム視点 (偵察と列挙 - T1082)

* **タスク 1: System Information Discovery (初期偵察)**
    * **目的**: 侵入成功直後（Initial Access）、自分がどの端末にいるのかを把握する。
    * **解説**: ホスト名に `DC` (Domain Controller), `SQL`, `WEB`, `FILE`, `BACKUP` といった文字列が含まれていれば、その端末が高い価値を持つターゲット（HVT）であることを示唆します。
* **タスク 2: Environment Awareness (命名規則の把握)**
    * **目的**: 複数の端末で `hostname` を実行し、その組織の「命名ルール」を特定する。
    * **解説**: ルールが分かれば、まだ見ぬ他のサーバー名（例: `SRV-APP-01` があるなら `SRV-APP-02` もあるはず）を予測して攻撃範囲を広げられます。

## エラーメッセージとトラブルシューティング

1.  **'hostname' は、内部コマンドまたは外部コマンド...として認識されていません。**
    * **原因**: 実行パス (`PATH`) が通っていないか、稀に `System32` 内の `hostname.exe` が削除・改ざんされている。
    * **解決策**: `C:\Windows\System32\hostname.exe` を直接指定して実行できるか確認。

2.  **期待した名前と違う**
    * **原因**: コンピューター名を変更した直後で、再起動が完了していない可能性がある。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**: ホスト名自体に「プロジェクト名」や「顧客名」を含めている場合、外部にその名前が漏れるだけで組織の内部活動が推測されるリスクがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: 非常に無害なコマンドに見えるため、どのような監視環境下でも実行を制限されることはまずありません。

## 注意点・補足

* **名前の変更**: Windows でホスト名を変更するには、`sysdm.cpl` (GUI) を使うか、PowerShell の `Rename-Computer`、または `netdom` コマンドを使用する必要があります。`hostname` コマンドでは変更できません。
