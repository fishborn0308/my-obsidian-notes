# Command - Windows - cmd - endlocal - 環境変数のローカライズの終了

## 概要

`endlocal` コマンドは、バッチファイル内で `setlocal` によって開始された環境変数のローカライズ（局所化）を終了させます。

このコマンドが実行されると、`setlocal` 以降に行われたすべての変数変更や設定（遅延展開の有効化など）が破棄され、環境変数は `setlocal` 実行直前の状態に復元されます。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `setlocal` | ローカライズを開始する。 | セットで使用 |
| `exit /b` | バッチファイルを終了する。明示的な `endlocal` がなくても、バッチ終了時に自動で `endlocal` が処理される。 | 構造化には `endlocal` |

## よく連携されるコマンド

### シナリオ例: 計算結果のみを親スコープに引き継ぐ (スクリプト技法)

* **目的**: ローカルスコープ内で複雑な計算や処理を行い、その**結果の変数だけ**を保持してスコープを抜ける。
* **連携コマンド**: `setlocal`, `set`
* **解説**: `endlocal` と `set` を同一行に `&` で記述することで、変数が破棄される直前の値を展開し、親スコープの変数に代入し直すことができます。
* **コマンド例**:
    ```cmd
    @echo off
    setlocal
    set /a RESULT=100 + 200
    REM ローカル変数 RESULT を親スコープに「投げる」テクニック
    endlocal & set FINAL_RESULT=%RESULT%
    
    echo 結果は %FINAL_RESULT% です
    ```

## スイッチ/オプション説明

`endlocal` コマンドにスイッチやオプションはありません。単体で実行します。

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サブルーチン（関数）の作成
* **組み合わせ**: `call`, `:label`
* **解説**: バッチ内でラベルを関数のように呼び出す際、呼び出し側の変数を汚さないように `setlocal` で入り、`endlocal` で抜けるのが作法です。
* **例**:
    ```cmd
    @echo off
    set GLOBAL_VAR=A
    call :MyFunction
    echo Global: %GLOBAL_VAR%
    pause
    exit /b

    :MyFunction
    setlocal
    set GLOBAL_VAR=B
    echo Local: %GLOBAL_VAR%
    endlocal
    exit /b
    ```

### 2. ブルーチーム視点

* **タスク**: 解析環境のクリーンアップ
* **組み合わせ**: `set PATH`
* **解説**: 調査用スクリプトを実行する際、調査対象の環境変数を一時的に書き換えてツールを動かした後、確実に元のクリーンな状態に戻すために使用します。
* **例**:
    ```cmd
    @echo off
    setlocal
    set PATH=%PATH%;C:\ForensicTools
    REM ここで各種解析コマンドを実行
    analyze.exe --output C:\Evidence
    endlocal
    REM 調査環境のPATH設定はここでリセットされる
    ```

### 3. レッドチーム視点

* **タスク**: 痕跡を残さない環境操作（Anti-Forensicsの一部）
* **組み合わせ**: `set`
* **解説**: スクリプト内で一時的にプロンプトの表示形式を変えたり、攻撃用ツールのパスを通したりしても、スクリプト終了後に `endlocal` が走ることで、管理者が後で `set` コマンドを叩いても攻撃者の痕跡（追加されたパスなど）が見えなくなります。
* **例**:
    ```cmd
    @echo off
    setlocal
    set PROMPT=System Error: 
    REM 悪意ある処理を隠蔽しつつ実行
    endlocal
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: (メッセージは出ないが、変数が期待通りに残らない)
    * **考えられる原因**: `endlocal` を実行したため、その中での変更がすべてリセットされた。
    * **解決策**: 値を残したい場合は、前述の `endlocal & set VAR=%VAR%` テクニックを使用する。

2.  **エラーメッセージ例 2**: `対応する SETLOCAL コマンドがありません。`
    * **考えられる原因**: `setlocal` を実行していないのに `endlocal` を実行した。
    * **解決策**: スクリプトの構造を見直し、必ずペアになるように配置する。

## 環境変数と設定ファイル

`endlocal` は環境変数を「復元」するコマンドであり、それ自体が特定の環境変数に依存することはありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **隠蔽**: 攻撃者が対話型シェルではなく、複雑なバッチファイルを介して攻撃を行う場合、`setlocal/endlocal` を使うことで、システムの環境変数リストを汚さずに（＝目立たずに）作業を完結させます。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * バッチファイルの実行プロセスを監視し、短時間に大量の環境変数変更とリセットが行われている場合、難読化されたダウンローダーや偵察スクリプトの可能性があります。
    * **Transcriptロギング**（PowerShellの場合）や、コマンドライン引数の詳細ログを確認し、`endlocal & set` のような特殊な記述がないかチェックします。

## 注意点・補足

* **自動実行**: バッチファイルの最後に到達すると、明示的に記述していなくても `endlocal` が自動的に実行されます。
* **ネスト（入れ子）**: `setlocal` を複数回呼び出した場合（最大32回）、`endlocal` は最後に実行された `setlocal` の階層のみを終了させます。
