# Command  - Linux - dpkg - Debianパッケージ管理システム (低レベル)

## 概要

`dpkg` (Debian Package) は、DebianやUbuntuなどのDebianベースのLinuxディストリベーションにおける、中核的なパッケージ管理ツールです。`.deb` 形式のパッケージファイルのインストール、削除、情報表示などを直接行います。`dpkg` はあくまでローカルにあるパッケージファイルを操作するための低レベルなツールであり、パッケージの依存関係を**自動的には解決しません**。

(出自: `dpkg` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`apt` / `apt-get`** | `apt` は、リポジトリからパッケージをダウンロードし、**依存関係を自動的に解決・インストールする**高レベルなフロントエンドです。普段使いは `apt` が推奨されます。 | `apt` |
| **`rpm`** | RHEL/CentOS系のディストリビューションで使われる低レベルなパッケージ管理ツール。`rpm` が `.rpm` ファイルを扱うのに対し、`dpkg` は `.deb` ファイルを扱います。 | - |

## よく連携されるコマンド

`dpkg` は、依存関係の問題を解決するために `apt` と連携することが非常に多いです。

### シナリオ例: 依存関係エラーを解決しながらパッケージをインストールする (インフラ構築・運用視点)

* **目的**: ダウンロードした `.deb` ファイルをインストールする際に発生した依存関係のエラーを、`apt` を使って修正・解決する。
* **連携コマンド**: `dpkg`, `apt`
* **解説**: `dpkg -i` でインストールを試み、依存関係エラーで中断された後、`apt -f install` を実行するのが定石です。`apt` は、中途半半端にインストールされたパッケージが必要とする依存関係をリポジトリから自動的に探し出し、全てインストールしてくれます。
* **コマンド例**:

```bash
# 1. Google Chromeの .deb パッケージをインストールしようとする
sudo dpkg -i google-chrome-stable_current_amd64.deb
# -> dpkg: dependency problems prevent configuration of ...

# 2. apt を使って、依存関係の問題を自動修復
sudo apt -f install
# -> ... (依存パッケージがインストールされ、設定が完了する)
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **パッケージ操作** | |
| ⭐ `-i`, `--install <.deb>` | パッケージをインストールします。 |
| ⭐ `-r`, `--remove <pkg>` | パッケージを削除します（**設定ファイルは残ります**）。 |
| `-P`, `--purge <pkg>` | パッケージを**設定ファイルも含めて**完全に削除します。 |
| **情報表示 (Query)** | |
| ⭐ `-l`, `--list [pattern]` | インストール済みのパッケージを一覧表示します。 |
| ⭐ `-S`, `--search <pattern>` | 指定したファイルパスが、どのパッケージに属しているかを検索します。 |
| `-L`, `--listfiles <pkg>` | 指定したパッケージによってインストールされたファイルの一覧を表示します。 |
| `-s`, `--status <pkg>` | パッケージの状態（インストール済みか、バージョンなど）を詳細に表示します。 |
| **その他** | |
| `--verify [package]` | パッケージの整合性を検証します。ファイルの欠損や変更がないかを確認します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 公式リポジトリにない `.deb` ファイルをインストールする。
* **組み合わせ**: `sudo dpkg -i <package_file.deb>`
* **解説**: サードパーティ製ソフトウェアなど、公式サイトからダウンロードしたパッケージをサーバーにインストールする際の基本的な操作です。
* **例**:

```bash
# ダウンロードした監視エージェントをインストール
wget [https://example.com/agent_1.2.3_amd64.deb](https://example.com/agent_1.2.3_amd64.deb)
sudo dpkg -i agent_1.2.3_amd64.deb
```

### 2. ブルーチーム視点

* **タスク**: 不審なファイルが、正規のパッケージによってインストールされたものか特定する。
* **組み合わせ**: `dpkg -S <file_path>`
* **解説**: フォレンジック調査において、不審なファイルを発見した場合にこのコマンドを実行します。もしどのパッケージにも属していない (`no path found matching pattern`) となれば、攻撃者が手動で配置したマルウェアである可能性が極めて高いと判断できます。
* **例**:

```bash
# /tmp/kworker という不審なファイルを発見した場合
dpkg -S /tmp/kworker
```

### 3. レッドチーム視点

* **タスク**: 侵入したシステムにインストールされているソフトウェアの一覧を取得し、脆弱性を探す。
* **組み合わせ**: `dpkg -l`
* **解説**: 権限昇格の足がかりを探すため、インストール済みのパッケージ一覧から、既知の脆弱性 (CVE) を持つ古いバージョンのサービス（例: Sudo, Kernel, Exim）がないかを探します。
* **例**:

```bash
# インストール済みの全パッケージをリストし、grepでsudoのバージョンを確認
dpkg -l | grep "sudo"
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `dpkg: error: dependency problems prevent configuration of <package>: ...`
		* **考えられる原因**: 最もよくあるエラー。インストールしようとしているパッケージが必要とする他のパッケージ（依存関係）がシステムにインストールされていません。
		* **解決策**: `sudo apt --fix-broken install` を実行します。`apt` が不足している依存関係を自動的に解決してインストールしてくれます。

## 環境変数と設定ファイル

* `dpkg` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](OS%20%20-%20Linux%20-%20environment_and_config%20-%20環境変数と設定ファイル.md) を参照。

### 関連する設定ファイル

* **`/var/lib/dpkg/`**: `dpkg` の状態データベース。インストール済みのパッケージ情報などが格納されています。
* **`/var/lib/dpkg/status`**: インストール済みパッケージの状態、バージョン、説明などが記述された重要なファイル。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `dpkg` 自体ではなく、**悪意のある `.deb` パッケージのインストール**がリスクとなります。
* **悪用シナリオ**: `.deb` パッケージ内のメンテナスクリプト (`preinst`, `postinst` など) は、root権限で任意のシェルコマンドを実行できるため、攻撃者にとって格好の永続化・権限昇格ツールとなります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo dpkg` を実行できる権限を持っている場合、悪意のある `postinst` スクリプトを含んだ `.deb` パッケージを作成し、それを `sudo dpkg -i` でインストールすることで、簡単にroot権限のコマンドを実行できます。
* **参照**: `https://gtfobins.github.io/gtfobins/dpkg/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、ユーザーに `dpkg` の `sudo` 権限を安易に与えない。公式リポジトリ以外の `.deb` パッケージはインストール前に内容物（特にメンテナスクリプト）を `dpkg-deb -e` などで展開・確認する。
* **Detection (検知)**: `dpkg` の実行ログ (`/var/log/dpkg.log`) や `auditd` で、意図しないパッケージのインストールがないか監視する。

## 注意点・補足

* **`apt` を優先する**: 依存関係の問題を避けるため、日常的なパッケージのインストール・アンインストールには、**常に `apt` を使用してください**。`dpkg` は、`apt` では解決できない特定の問題に対処するための、より低レベルなツールと位置づけられます。
