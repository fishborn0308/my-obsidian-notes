# Command - Windows - cmd - help - コマンド ヘルプの表示

## 概要

`help` コマンドは、Windows の特定のコマンドに関する詳細な説明、構文、およびオプション（スイッチ）を表示します。引数なしで実行した場合は、システムで利用可能な主要な標準コマンドの一覧と簡単な説明が表示されます。

(出自: `Windows 標準搭載 - 外部コマンド (help.exe) / 一部は cmd.exe 内蔵`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **`[command] /?`** | ほぼすべてのコマンドで有効なスイッチ。`help` コマンドよりも汎用性が高い。 | 最優先 |
| `man` | Linux (bash) のマニュアル表示コマンド。 | (Linux環境) |
| `Get-Help` | PowerShell 版。オンライン更新や例示 (`-Examples`) が充実。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: 使える武器（コマンド）の棚卸し (偵察)

* **目的**: ターゲット端末にどのような管理用コマンドが備わっているか、ざっくりと把握する。
* **連携コマンド**: `help`, `findstr`
* **解説**: 標準的なコマンド一覧を出し、特定のキーワードで絞り込みます。
* **コマンド例**:
    ```cmd
    REM ネットワーク関連のコマンドがあるか確認
    help | findstr /i "network tcp ip"
    ```

## スイッチ/引数説明

| 引数 | 説明 |
| :--- | :--- |
| (なし) | 利用可能な主要コマンドの一覧を表示する。 |
| `[command]` | 指定した特定のコマンドに関するヘルプ情報を表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新人オペレーターが、複雑な `robocopy` や `netsh` のオプションを忘れてしまった際、公式の定義をその場で確認する。
* **例**:
    ```cmd
    REM netsh コマンドの構文を確認
    help netsh
    ```

### 2. ブルーチーム視点 (環境把握)

* **タスク**: 制限されたシェル環境（Restricted Shell）において、実行が許可されているコマンドがどれくらいあるかを `help` の出力から推測する。

### 3. レッドチーム視点 (偵察と武器の選定 - T1082)

* **タスク: Living off the Land Capabilities (標準機能の調査)**
    * **目的**: 外部ツールを持ち込まずに攻撃を完結させるため、その OS バージョンで `bitsadmin` や `certutil` といった「悪用可能な標準機能」が利用可能か、またどのような構文かを公式マニュアルで再確認する。
    * **解説**: 隔離環境（Air-gapped）での攻撃において、`help` はオフラインのチートシートとして機能します。

## エラーメッセージとトラブルシューティング

1.  **このコマンドは、ヘルプ ユーティリティでサポートされていません。**
    * **原因**: `help` は主に標準の外部コマンドと一部の内蔵コマンドを対象としています。サードパーティ製ツールや一部の特殊なコマンドには対応していません。
    * **解決策**: `[command] /?` を試してください。こちらの方が成功率が高いです。

2.  **画面が流れて読めない**
    * **解決策**: `more` コマンドにパイプして、1 ページずつ表示させます。
    * **例**: `help | more`

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**: `help` 自体に脆弱性はありませんが、攻撃者に「このシステムで何ができるか」を教えるガイドブックになり得ます。
* **UAC Bypass のヒント**: 一部のコマンドのヘルプには「管理者権限が必要」といった注意書きがあり、攻撃者が権限昇格が必要なポイントを特定する材料になります。

## 注意点・補足

* **オンライン vs オフライン**: `help` コマンドが表示する内容はローカルに保存されているファイルに基づいています。最新の Windows 機能については Microsoft Learn (公式ドキュメント) の方が詳しい場合があります。
* **/? との違い**: `dir /?` は `dir.exe` (または cmd.exe) 自体が持つヘルプ機能ですが、`help dir` は `help.exe` が `dir` の情報を呼び出して表示する仕組みです。
