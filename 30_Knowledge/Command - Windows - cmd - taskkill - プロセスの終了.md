# Command - Windows - cmd - taskkill - プロセスの終了

## 概要

`taskkill` コマンドは、1 つ以上のプロセスを終了させるために使用されます。プロセス ID (PID) またはイメージ名（ファイル名）を指定して実行します。通常の終了要請だけでなく、応答のないプロセスを「強制終了」させるオプションも備えています。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `Stop-Process` | PowerShellコマンドレット。オブジェクトをパイプラインで渡して終了できる。 | `Stop-Process` (PS) |
| `tskill` | `taskkill` の軽量・簡易版。古いWindowsとの互換性用だが、現在は `taskkill` が推奨。 | `taskkill` |
| タスク マネージャー | GUI上でマウス操作により終了させる。 | GUI環境ならこちら |

## よく連携されるコマンド

### シナリオ例: 特定の不審なプロセスをフィルタリングして一括終了 (ブルーチーム)

* **目的**: 特定のユーザー権限で動いている、メモリを異常消費しているプロセスなどを一括で停止させる。
* **連携コマンド**: `tasklist` (確認用)
* **解説**: `/FI` (フィルター) スイッチを使用して、条件に合致するプロセスのみを対象にします。
* **コマンド例**:
    ```cmd
    REM 応答なし(NOT RESPONDING)の状態になっているプロセスをすべて強制終了
    taskkill /F /FI "STATUS eq NOT RESPONDING"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **対象指定** | |
| ⭐ `/PID <ProcessID>` | 終了させるプロセスの PID を指定する。 |
| ⭐ `/IM <ImageName>` | 終了させるプロセスのイメージ名（例: `notepad.exe`）を指定する。ワイルドカード `*` も使用可能。 |
| **実行オプション** | |
| ⭐ `/F` | プロセスを「強制的に」終了させる。 |
| ⭐ `/T` | 指定したプロセスだけでなく、そのプロセスが起動した「子プロセス」もすべて終了させる（ツリー終了）。 |
| **リモート・権限** | |
| `/S <System>` | リモートコンピューターを指定する。 |
| `/U <UserName>` | 実行するユーザーコンテキストを指定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: フリーズしたエクスプローラーの再起動
* **組み合わせ**: `/F`, `/IM`
* **解説**: 画面が固まった際、一度強制終了させてから再起動させる定番の手順です。
* **例**:
    ```cmd
    REM エクスプローラーを強制終了し、再起動する
    taskkill /F /IM explorer.exe & start explorer.exe
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアのプロセスツリー一括排除
* **組み合わせ**: `/F`, `/T`, `/PID`
* **解説**: 1つのプロセスを消しても別のプロセスが検知して復活させる「永続化」を防ぐため、子プロセスを含めて一括で強制終了します。
* **例**:
    ```cmd
    REM PID 1234 のプロセスとその子プロセスをすべて強制終了
    taskkill /F /T /PID 1234
    ```

### 3. レッドチーム視点

* **タスク**: 競合するプロセスの停止、または簡易的な防御無効化
* **組み合わせ**: `/F`, `/IM`
* **解説**:
    1. 自身のツール（例: リバースシェルのリスナー）とポートが重複している既存プロセスを止める。
    2. (権限がある場合) セキュリティ監視ツールのプロセスを停止させて検知を免れる。
* **例**:
    ```cmd
    REM 特定の監視エージェントを強制停止（※管理者権限が必要）
    taskkill /F /IM "SecurityAgent.exe"
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `エラー: プロセス "..." (PID ...) を終了できませんでした。原因: アクセスが拒否されました。`
    * **考えられる原因**: 終了させようとしているプロセスが自分より高い権限（SYSTEM権限など）で動作している。
    * **解決策**: コマンドプロンプトを「管理者として実行」して再試行する。

2.  **エラーメッセージ例 2**: `エラー: プロセス "..." が見つかりませんでした。`
    * **考えられる原因**: 指定したPIDやイメージ名が既に終了している、または綴りが間違っている。
    * **解決策**: `tasklist` で現在の正確な PID/名称を確認する。

## 環境変数と設定ファイル

`taskkill` 自体に影響を与える固有の変数はありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    * **Evasion (回避)**: 攻撃者が管理者権限を奪取した後、AV (Anti-Virus) や EDR (Endpoint Detection and Response) のプロセスを `taskkill /F` で停止させ、その後の攻撃活動を隠蔽する。
    * **Denial of Service (DoS)**: 重要なシステムプロセスを終了させることで、システムを意図的にクラッシュさせる。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Standard Command`
* **解説**: 攻撃の「妨害排除」ステップで頻繁に利用される。
* **参照**: [LOLBAS - taskkill.exe](https://lolbas-project.github.io/lolbas/Binaries/Taskkill/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * セキュリティ製品自体の「自己保護（Self-Protection）機能」を有効にし、管理者であってもプロセスを簡単に殺せないように設定する。
* **Detection (検知)**:
    * **イベントID 4688**: `taskkill` の実行ログを監視し、引数にセキュリティ製品の名前や `/F` スイッチが含まれている場合にアラートを出す。
    * 特に `SYSTEM` 権限のプロセスを対象にした `taskkill` は厳重に監視する。

## 注意点・補足

* **`/F` のリスク**: 強制終了はメモリ上のデータを保存せずにプロセスを閉じるため、データの破損を招く恐れがあります。通常はまず `/F` なしで試し、終了しない場合にのみ `/F` を使うのが推奨されます。
* **ワイルドカード**: `taskkill /IM note*` とすると、`notepad.exe` も `notepad++.exe` もすべて終了対象になります。意図しないプロセスを殺さないよう注意が必要です。
