# Command  - Linux - file - ファイルの種類を判別する

## 概要

`file` コマンドは、引数で与えられたファイルの内容を調べ、そのファイルがどのような種類（テキストファイル、実行ファイル、画像、圧縮ファイルなど）であるかを判別して表示します。ファイル名や拡張子に頼るのではなく、ファイル先頭の特定のバイトシーケンス（**マジックナンバー**）や、テキストファイルの場合はその言語構造などを分析して判断します。拡張子を偽装したマルウェアの特定など、セキュリティやフォレンジック調査において極めて重要なツールです。

(出自: `file` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **拡張子の確認** | Linux/Unixでは拡張子に意味はなく、単なる名前の一部です。`file` は拡張子を無視してファイルの内容から種類を判断するため、より信頼性が高いです。 | `file` |
| **`ls -F`** | ファイルの種類に応じて末尾に記号（`/`など）を付与しますが、`file` は `ELF 64-bit executable` のように、より詳細で具体的な種類を報告します。 | `file` (詳細特定) |
| **`stat`** | ファイルのタイムスタンプやiノード情報などの**メタデータ**を表示します。`file` はファイルの中身に基づいた**種類**を表示します。 | - |

## よく連携されるコマンド

`file` はワイルドカード (`*`) と組み合わせて、ディレクトリ内の全ファイルの種別を一括で確認する際によく使われます。

### シナリオ例: アップロードされたファイルの安全確認 (ブルーチーム視点)

* **目的**: Webサーバーのアップロードディレクトリに置かれたファイルが、拡張子通りのものであるかを確認し、偽装された実行ファイルがないかを調査する。
* **連携コマンド**: `ls`, `file`
* **解説**: `file *` を実行することで、ディレクトリ内の全ファイルの種類を一括で確認できます。これにより、`.jpg` という拡張子なのに中身が `PHP script` であるといった偽装を簡単に見破ることができます。
* **コマンド例**:

```bash
# Webサーバーのアップロードディレクトリに移動
cd /var/www/uploads/

# 全てのファイルの種類を確認
file *
# -> avatar.jpg: JPEG image data, ... (正常)
# -> shell.php.jpg: PHP script, ASCII text (危険！)
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `-b`, `--brief` | ファイル名を表示せず、ファイルの種類のみを出力します。 |
| `-i`, `--mime-type` | `text/plain`, `image/jpeg` のようなMIMEタイプ形式で結果を表示します。 |
| **特殊ファイル/圧縮ファイル** | |
| `-s`, `--special-files`| ブロックデバイスやキャラクタデバイス（`/dev/sda`など）の内容も読み取って判別しようとします。 |
| `-z`, `--uncompress` | 圧縮ファイル（`.gz`など）の中身を読み取り、圧縮されている元のファイルの種類を判別しようと試みます。 |
| **シンボリックリンク** | |
| `-L`, `--dereference`| シンボリックリンクを追跡し、リンク先の実体のファイルの種類を表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 拡張子がない、あるいは見慣れないファイルが何であるかを確認する。
* **組み合わせ**: `file <filename>`
* **解説**: 設定ファイルだと思って `cat` で開いたら文字化けした、といった場合に `file` を実行します。`data` や `binary` と表示されれば、それがテキストファイルではないことが分かり、`strings` や `xxd` で中身を確認する、といった次のステップに進めます。
* **例**:

```bash
# ディレクトリ内の全ファイルの種類をまとめて確認
file *
```

### 2. ブルーチーム視点

* **タスク**: ファイルの拡張子を信用せず、その正体を突き止める。
* **組み合わせ**: `file <suspicious_file>`
* **解説**: **フォレンジック調査における最重要コマンドの一つ**。ユーザーがダウンロードした `invoice.pdf` というファイルがマルウェアではないかと疑われる場合に `file` を実行します。もし結果が `PDF document, ...` ではなく `ELF 64-bit LSB executable, ...` と表示されれば、それはPDF文書に偽装したLinuxの実行ファイル（マルウェア）であると断定できます。
* **例**:

```bash
# GIF画像に偽装したPHPスクリプトを見破る
file a.php.gif
# -> a.php.gif: PHP script, ASCII text
```

### 3. レッドチーム視点

* **タスク**: 侵入したホストで価値のある情報を効率的に探す。
* **組み合わせ**: `file *`
* **解説**: ユーザーのホームディレクトリなどから窃取した大量のファイルの中から、`SQLite 3.x database`, `Zip archive data`, `GPG encrypted data` などの出力を得ることで、パスワードが保存されていそうなデータベースファイルや、重要な情報が含まれていそうなアーカイブ・暗号化ファイルに当たりをつけ次の調査対象を絞り込みます。
* **例**:

```bash
# ホームディレクトリ内のファイルを調査
file /home/user/*
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **ファイルのタイプが `data` と表示される。**
		* **考えられる原因**: `file` コマンドが、既知のマジックナンバーやテキストのパターンに一致するものを見つけられませんでした。これは、独自フォーマットのバイナリデータ、暗号化されたデータ、あるいは単にランダムなデータであることを示唆しています。
		* **解決策**: `strings` コマンドでファイル内に含まれる可読文字列を探したり、`xxd` や `hexdump` でバイナリの内容を直接確認したりして、さらなる手がかりを探します。

## 環境変数と設定ファイル

* `file` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](OS%20%20-%20Linux%20-%20environment_and_config%20-%20環境変数と設定ファイル.md) を参照。

### 関連する設定ファイル

* **`/usr/share/misc/magic.mgc`**: `file` コマンドがファイルの種類を決定するために使用する、コンパイル済みのデータベースファイル（マジックファイル）。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `file` 自体は読み取り専用ツールですが、過去には特別に細工されたファイルを処理する際のライブラリに脆弱性が発見されたことがあります。
* **悪用シナリオ**: 攻撃者はファイルの拡張子を偽装して、ユーザーやシステム管理者にマルウェアを実行させようとします。`file` はこの偽装を看破するための防御ツールです。

### GTFOBins / LOLBAS における利用例

`file` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: Webサーバーのファイルアップロード機能など、外部からファイルを受け取る箇所では、アップロードされたファイルに対して `file` コマンドを実行し、拡張子と中身が一致しない場合は警告を発するかアップロードを拒否する。

## 注意点・補足

* **マジックナンバー**: ファイルの種類の多くは、ファイル先頭の数バイト（マジックナンバー）を見れば判別できます。（例: `JPEG`は`FF D8 FF`、`ELF`実行ファイルは`7F 45 4C 46`で始まる）`file` コマンドはこの仕組みを主に利用しています。
