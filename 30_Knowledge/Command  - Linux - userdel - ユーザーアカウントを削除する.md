# Command  - Linux - userdel - ユーザーアカウントを削除する

## 概要

`userdel` (user delete) は、システムから既存のユーザーアカウントを削除するためのコマンドです。このコマンドは、`/etc/passwd` や `/etc/shadow` などの関連ファイルから、指定されたユーザーのエントリを削除します。オプションを指定することで、ユーザーのホームディレクトリやメールスプールも同時に削除できます。

このコマンドの実行には `root` 権限が必要です。

(出自: `shadow-utils` または `passwd` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`passwd -l`** | ユーザーアカウントを**ロック**します。アカウントとデータは削除せず、一時的にログインを無効化したい場合に使います。`userdel` は恒久的な削除です。 | `passwd -l` (一時的な無効化) |
| **`useradd`** | 新しいユーザーを**作成**します。 | - |
| **`usermod`** | 既存のユーザーの属性を**変更**します。 | - |

## よく連携されるコマンド

`userdel` を実行する前に、そのユーザーが所有するファイルがないかを確認するために `find` コマンドと連携することがあります。

### シナリオ例: ユーザー削除前の残存ファイル確認 (インフラ構築・運用視点)

* **目的**: ユーザーのホームディレクトリ以外に、そのユーザーが所有するファイルがシステム上に残っていないかを確認する。
* **連携コマンド**: `find`
* **解説**: ユーザーアカウントを削除する前に `find` を実行することで、意図しない場所に重要なファイルが残ってしまうのを防ぎ、クリーンな削除を保証します。
* **コマンド例**:

    ```bash
    # ユーザー 'olduser' が所有するファイルをシステム全体から検索
    sudo find / -user olduser -ls 2>/dev/null

    # (確認後、安全であればユーザーをホームディレクトリごと削除)
    sudo userdel -r olduser
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **削除オプション** | |
| ⭐ `-r`, `--remove` | **最も重要なオプション**。ユーザーの**ホームディレクトリ**と**メールスプール**も一緒に削除します。このオプションを付けないと、アカウント情報だけが削除され、ホームディレクトリは残ります。 |
| **動作制御** | |
| `-f`, `--force` | ユーザーが現在ログイン中であっても、強制的にアカウントを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 退職した従業員のアカウントと、そのホームディレクトリ内の全データを完全に削除する。
* **組み合わせ**: `sudo userdel -r <username>`
* **解説**: ユーザーのアカウント情報と関連データをシステムから完全に抹消する際の、最も基本的な使い方です。
* **例**:

    ```bash
    # 退職したユーザー 'old-employee' のアカウントとホームディレクトリを削除
    sudo userdel -r old-employee
    ```

### 2. ブルーチーム視点

* **タスク**: 攻撃者によって作成された不正なアカウントを削除し、システムを修復する。
* **組み合わせ**: `sudo userdel -r <rogue_account>`
* **解説**: **インシデント対応の最終段階**。調査の結果、攻撃者が永続化のために作成したバックドアアカウントを発見した場合、関連する全ての証拠保全が完了した後、このコマンドで不正なアカウントをシステムから完全に除去します。
* **例**:

    ```bash
    # 攻撃者が作成した 'hacker' アカウントをホームディレクトリごと削除
    sudo userdel -r hacker
    ```

### 3. レッドチーム視点

* **タスク**: **破壊活動 (Sabotage)** または**サービス妨害 (DoS)**。
* **組み合わせ**: `sudo userdel <critical_user>`
* **解説**: 攻撃者がroot権限を奪取した後、システムの正常な運用を妨害するために、アプリケーションの実行に必要なサービスアカウント（`nginx`, `mysql`など）や、他の管理者アカウントを意図的に削除します。
* **例**:

    ```bash
    # アプリケーションのサービスアカウントを削除してサービスを妨害
    sudo userdel application_user
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **エラーメッセージ例 1**: `userdel: user <username> is currently logged in`
    * **考えられる原因**: 削除しようとしているユーザーが、まだシステムにログインしているセッションを持っています。
    * **解決策**:
        * `who` や `w` コマンドでユーザーのセッションを確認します。
        * `pkill -u <username>` でそのユーザーのプロセスを全て終了させます。
        * その後、再度 `userdel` を実行するか、`-f` (`--force`) オプションを付けて強制的に削除します。

## 環境変数と設定ファイル

* `userdel` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/passwd`, `/etc/shadow`, `/etc/group`**: `userdel` はこれらのファイルからユーザーのエントリを削除します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `userdel` 自体ではなく、その**破壊的な性質**がリスクとなります。
* **悪用シナリオ**:
  * **サービス妨害**: レッドチームのシナリオで述べたように、重要なサービスアカウントを削除することで、アプリケーションを機能不全に陥らせます。
  * **証拠の破壊**: 攻撃者が作成したアカウントをブルーチームが調査する前に、別の攻撃者がそのアカウントを `userdel` で削除し、痕跡を消す可能性があります。

### GTFOBins / LOLBAS における利用例

`userdel` はOSの構成を変更するコマンドであり、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要なサービスアカウントの削除権限を厳格に管理する。重要なデータのバックアップを定期的に取得する。
* **Detection (検知)**: `/etc/passwd` などのファイルの変更をFIMツールで監視する。`auditd` で `userdel` コマンドの実行を監視し、意図しないアカウントの削除がないかを確認する。

## 注意点・補足

* **孤立したファイル (Orphaned Files)**: `-r` オプションを**付けずに**ユーザーを削除した場合、そのユーザーがホームディレクトリ以外に所有していたファイルは、所有者が数値のUIDで表示される「孤立した」状態になります。`find / -nouser` などのコマンドで定期的に見つけ出し、適切に処理する必要があります。
