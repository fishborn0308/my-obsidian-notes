# Command  - Linux - htop - 対話的なプロセスビューア

## 概要

`htop` は、Linuxシステムで実行中のプロセスをリアルタイムで一覧表示し、管理するための対話的なシステムモニタです。旧来の `top` コマンドを大幅に改良したもので、色付けされた見やすい表示、マウスやカーソルキーによるスクロールと操作、プロセスの検索、フィルタリング、ツリー表示、シグナル送信（kill）などを簡単に行うことができます。サーバーのパフォーマンス問題を診断する際に、「何がCPUやメモリを消費しているのか」を特定するための非常に強力なツールです。

(出自: `htop` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`top`** | `htop` の元になった古典的なプロセスビューア。`htop` は `top` の**使いやすさと機能性を大幅に向上させた上位互換**と考えることができます。 | `htop` |
| **`ps`** | コマンド実行時点のプロセスの**スナップショット**を静的に表示します。`htop` は**リアルタイム**で更新され続ける動的なビューを提供します。 | - |
| **`glances`, `atop`**| `htop` 同様に高機能ですが、ディスクI/OやネットワークI/Oなど、プロセス情報に加えてより広範なシステムメトリクスを一つの画面で表示します。 | - |

## よく連携されるコマンド

`htop` は、`lsof` や `strace` といった、より詳細なプロセス分析ツールと対話的に連携できます。

### シナリオ例: 不審なプロセスの詳細を調査する (ブルーチーム視点)

* **目的**: `htop` で発見した不審なプロセスが、どのようなファイルを開き、どのようなネットワーク接続を行っているかをその場で調査する。
* **連携コマンド**: `lsof`, `strace`
* **解説**: `htop` の画面で不審なプロセスを選択し、`l` キー (lsof) または `s` キー (strace) を押すことで、`htop` を終了することなく、直接それらのコマンドを呼び出して詳細な動的分析を開始できます。
* **コマンド例**:

```bash
# 1. htopを起動
sudo htop

# 2. リストから不審なプロセス (例: PID 12345) をカーソルキーで選択

# 3. 'l' キーを押す -> lsof -p 12345 が実行され、開いているファイルが表示される
# 4. 's' キーを押す -> strace -p 12345 が実行され、システムコールが追跡される
```

## オプション説明 (`htop` 内部のキー操作)

| オプション | 説明 |
| :--- | :--- |
| **ナビゲーションと表示** | |
| ⭐ `F5` / `t` | プロセスの親子関係を**ツリー形式**で表示/非表示を切り替えます。 |
| ⭐ `F6` / `<` `>` | `CPU%`, `MEM%` など、ソートするカラムを選択します。 |
| `P`, `M`, `T` | (大文字) それぞれ `CPU%`, `MEM%`, `TIME+` でソートします。 |
| `u` | 特定の**ユーザー**が実行しているプロセスのみをフィルタリング表示します。 |
| **検索とフィルタ** | |
| ⭐ `F3` / `/` | コマンド名でプロセスを検索（インクリメンタルサーチ）します。 |
| `F4` / `\` | コマンド名で表示するプロセスをフィルタリングします。 |
| **プロセス操作** | |
| ⭐ `F9` / `k` | 選択したプロセスにシグナル（`SIGTERM`, `SIGKILL`など）を送信して終了させます。 |
| `F2` / `S` | 表示するメーター、カラム、配色などをカスタマイズします。 |
| `F10` / `q` | `htop` を終了します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 「サーバーが重い」の原因となっているプロセスを特定し、対処する。
* **組み合わせ**: (対話モード) `P` (CPUソート) -> `M` (メモリソート) -> `F9` (Kill)
* **解説**: `htop` を起動し、`P` や `M` キーでリソースを異常に消費しているプロセスを特定します。暴走していると判断した場合、`F9` で安全に停止 (`SIGTERM`) または強制終了 (`SIGKILL`) させ、サービスを復旧させます。
* **例**:

```bash
# サーバーのパフォーマンス問題を調査するためにhtopを起動
htop
```

### 2. ブルーチーム視点

* **タスク**: 侵害されたホスト上で不審なプロセスの活動をライブで調査する。
* **組み合わせ**: (対話モード) `F5` (ツリー表示) -> `l` (lsof) -> `s` (strace)
* **解説**: `F5` でWebサーバーから不審なシェルが起動していないかなどの親子関係を確認します。不審なプロセスを発見したら、カーソルを合わせて `l` (lsof) や `s` (strace) を押し、そのプロセスが開いているファイル、ネットワーク接続、システムコールを詳細に調査します。
* **例**:

```bash
# ライブフォレンジック調査のためにhtopを管理者権限で起動
sudo htop
```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察と、自身の活動の防御回避。
* **組み合わせ**: `htop`
* **解説**:
	* **偵察**: 攻撃者は侵入後 `htop` を実行し、どのようなサービスやセキュリティ製品（EDRエージェントなど）が動いているか、また、どのユーザーがログインしているかを把握します。
	* **防御回避**: 自身のマルウェアやリバースシェルがCPUを異常に消費していないかを `htop` で確認し、管理者に発見されるリスクを低減させます。

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `bash: htop: command not found`
		* **考えられる原因**: `htop` は多くのシステムでデフォルトではインストールされていません。
		* **解決策**: システムのパッケージマネージャを使ってインストールしてください。（Debian/Ubuntu: `sudo apt install htop`, RHEL/CentOS: `sudo yum install htop`）

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`~/.config/htop/htoprc`**: `F2` (Setup) でカスタマイズした表示設定（カラム、配色など）が保存されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `htop` 自体ではなく、その**表示内容**が**情報漏洩**に繋がります。
* **悪用シナリオ**: `htop` は実行中の全てのプロセスと、その**コマンドライン引数**を表示します。もし、`python my_app.py --password 'MyS3cr3t'` のように、引数にパスワードなどの機密情報が平文で渡されている場合、`htop` を実行できる全てのユーザーにその情報が閲覧可能になってしまいます。

### GTFOBins / LOLBAS における利用例

`htop` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: アプリケーションにパスワードなどの機密情報を渡す際は、コマンドライン引数ではなく、環境変数や設定ファイル、シークレット管理ツールを使用する。
* **Detection (検知)**: -

## 注意点・補足

* **設定の保存**: `F2` (Setup) でカスタマイズした設定は、`F10` (Quit) で終了する際に自動的に `~/.config/htop/htoprc` ファイルに保存され、次回の起動時にも引き継がれます。
