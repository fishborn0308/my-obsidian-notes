# Command - Windows - cmd - date - システム日付の表示と設定

## 概要

`date` コマンドは、システムの現在の日付を表示、または新しい日付に設定するために使用されます。
引数なしで実行すると現在の設定が表示され、新しい日付の入力を求められます。単に表示のみを行いたい場合は `/T` スイッチを使用します。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `time` | システムの「時刻」を表示・設定する。 | セットで利用 |
| `Get-Date` | PowerShell 版。日付の加算・減算やフォーマット指定が強力。 | スクリプト処理 |
| `w32tm` | Windows Time サービスの管理（NTP 同期設定など）。 | 時刻同期管理 |

## よく連携されるコマンド

### シナリオ例: ログファイル名への日付付与 (運用自動化)

* **目的**: 毎日実行されるバックアップやログ収集において、ファイル名に実行時の日付を自動で含める。
* **連携コマンド**: `set`, `date`
* **解説**: 環境変数 `%DATE%` から必要な部分を切り出して変数に格納します。
* **コマンド例**:
    ```cmd
    REM 日付（YYYYMMDD形式）を変数に格納してファイル名に利用
    set TODAY=%DATE:/=%
    echo "Backup started" > backup_%TODAY%.log
    ```

## スイッチ/オプション説明

**※日付の「設定」には管理者権限が必要です。表示のみなら一般権限で可能です。**

| 使用法 / スイッチ | 説明 |
| :--- | :--- |
| `date` | 現在の日付を表示し、新しい日付の入力を促す。変更しない場合は Enter。 |
| ⭐ `/T` | **【重要】** 新しい日付の入力を求めず、現在の日付のみを表示する。 |
| `mm-dd-yy` | 指定した形式（地域設定に依存）で直接日付を変更する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 閉鎖環境（NTP サーバーがない環境）での手動時刻合わせ。
* **例**:
    ```cmd
    REM システム日付を 2026年2月21日に設定
    date 2026-02-21
    ```

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: 侵害調査において、ファイルシステムのタイムスタンプ（作成日時・更新日時）とシステムログの整合性を確認する。
* **注意**: 攻撃者が `date` コマンドを使ってシステム時刻を一時的に操作し、ログの発生順序を攪乱させていないか、イベントログ（Event ID 4616: システム時刻が変更されました）を精査します。

### 3. レッドチーム視点 (防御回避とエクスプロイト)

* **タスク 1: Timestomping (タイムスタンプ改ざんの準備 - T1070.006)**
    * **目的**: 設置したマルウェアの作成日時を、正規のシステムファイルと同じ日時に見せかけてカモフラージュする。
    * **解説**: システムの日付を過去に変更した状態でファイルを配置・作成し、その後に現在時刻に戻すことで、不審な「最近作成されたファイル」として検知されるのを回避します。
* **タスク 2: Certificate/Trial Expiration Bypass (有効期限の回避)**
    * **目的**: 有効期限が切れたデジタル証明書や、試用期間が終了したソフトウェアを強制的に動作させる。
    * **コマンド例**:
        ```cmd
        REM 期限が切れる前の日付に巻き戻す
        date 2024-01-01
        ```

## エラーメッセージとトラブルシューティング

1.  **A required privilege is not held by the client.**
    * **原因**: 管理者権限がない状態で日付を変更しようとした。
    * **解決策**: 「管理者として実行」したプロンプトで実行してください。

2.  **無効な日付を入力しました**
    * **原因**: Windows の地域設定（ロケール）で許可されていない形式（例: YY/MM/DD と MM-DD-YY の混同）で入力した。
    * **解決策**: プロンプトに表示される形式（通常は YYYY/MM/DD 等）に従ってください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **ログの攪乱**:
    * 攻撃者がシステム時刻を大幅に変更すると、SIEM などの相関分析ツールがイベントを正しく並べられなくなり、攻撃の全体像の把握が遅れるリスクがあります。
* **認証エラーの誘発**:
    * Active Directory 環境では、ドメインコントローラとクライアントの時刻差が 5 分以上あると Kerberos 認証に失敗します。攻撃者が意図的に `date` や `time` を操作してサービスの可用性を損なう（DoS）可能性があります。

## 注意点・補足

* **%DATE% 変数の罠**: `echo %DATE%` で返ってくる文字列の形式は、コントロールパネルの地域設定によって異なります（スラッシュ区切り、ハイフン区切り、曜日の有無など）。バッチファイルで日付をパースする場合は、環境に依存しない PowerShell の使用が推奨されます。
* **NTP による自動修正**: Windows Time サービスが稼働している場合、手動で変更した日付はしばらくすると NTP サーバーの値によって自動的に修正されます。
