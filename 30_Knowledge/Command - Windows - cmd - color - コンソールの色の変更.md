# Command - Windows - cmd - color - コンソールの色の変更

## 概要

`color` コマンドは、コマンドプロンプト ウィンドウの背景色と前景（文字）色を変更します。

引数なしで実行すると、コマンドプロンプトを起動したときのデフォルトの色（通常は黒背景に白文字）に戻ります。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| **コンソールのプロパティ** | GUI で恒久的な色設定が可能。 | 個人の好み |
| `$Host.UI.RawUI` | PowerShell でより細かく（背景色、文字色を個別に）操作する。 | PS 環境 |
| `ANSI エスケープコード` | モダンなターミナル (Windows Terminal等) で、テキストの一部だけ色を変える。 | 表現力の重視 |

## よく連携されるコマンド

### シナリオ例: エラー発生時に画面を赤くして警告する

* **目的**: バッチファイルの処理が失敗した際、視覚的に即座に異常を知らせる。
* **連携コマンド**: `if`, `color`, `echo`
* **解説**: 終了ステータス (`ERRORLEVEL`) を判定し、異常があれば背景を赤にします。
* **コマンド例**:
    ```cmd
    @echo off
    REM 失敗する可能性のある処理
    command_that_might_fail
    if %ERRORLEVEL% neq 0 (
        color 4F
        echo [ERROR] 致命的なエラーが発生しました！
        pause
    )
    ```

## スイッチ/オプション説明

書式: `color [属性]`

属性は **2 桁の 16 進数**で指定します。1 桁目が「背景色」、2 桁目が「前景色（文字色）」です。

### カラーコード対応表

| 値 | 色 | 値 | 色 |
| :--- | :--- | :--- | :--- |
| **0** | 黒 | **8** | 灰色 |
| **1** | 青 | **9** | 明るい青 |
| **2** | 緑 | **A** | 明るい緑 |
| **3** | 水色 | **B** | 明るい水色 |
| **4** | 赤 | **C** | 明るい赤 |
| **5** | 紫 | **D** | 明るい紫 |
| **6** | 黄色 | **E** | 明るい黄色 |
| **7** | 白 | **F** | 明るい白 |

* **例**: `color 0A` (黒背景に明るい緑 = いわゆるハッカー風)
* **例**: `color F0` (白背景に黒文字)

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 踏み台サーバーや本番サーバーにログインした際、操作ミスを防ぐために背景色を自動で変更する。
* **例**:
    ```cmd
    REM 本番環境用のバッチファイル冒頭に記述
    color 4F
    echo [WARNING] 現在、本番環境 (PROD) を操作しています。
    ```

### 2. ブルーチーム視点

* **タスク**: 監視端末において、正常時は青、警戒時は黄色、インシデント発生時は赤と画面の色を動的に変えるダッシュボードスクリプトを作成する。

### 3. レッドチーム視点 (心理的効果とデモ)

* **タスク 1: Social Engineering (権威性の演出)**
    * **目的**: ターゲットの前で「高度な作業をしている」ように見せかけ、信頼を得る（または恐怖を与える）。
    * **解説**: 黒背景に緑文字 (`0A`) は、映画やドラマの影響で「プロのハッカー」というステレオタイプな印象を与えやすいため、デモの際の見栄えとして多用されます。
* **タスク 2: 作業状況の識別**
    * **目的**: 複数のリバースシェルやリモートセッションを同時に開いている際、どのウィンドウがどのターゲットに対応するかを色で識別する。

## エラーメッセージとトラブルシューティング

1.  **背景と前景を同じ色にしようとした**
    * **現象**: `color 77` のように指定しても色が変更されない。
    * **原因**: 背景色と前景色を同じにすると文字が見えなくなるため、`cmd.exe` が設定を拒否します。
    * **解決策**: 異なる 2 つの値を指定してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Social Engineering**:
    * 偽のシステムアップデート画面や「スキャン中」の画面をバッチファイルで作成し、`color` を使ってそれっぽく見せることで、ユーザーにパスワードを入力させるといったフィッシング攻撃に悪用されることがあります。
* **証拠の隠蔽（簡易的）**:
    * 背景色と前景色を極端に近い色（例：白と明るい白）にすることで、画面を一見空に見せかけ、背後でコマンドを実行し続けるといった小細工に使われることがあります（ただし、ドラッグして反転させれば見えてしまいます）。

## 注意点・補足

* **リセット**: 引数なしの `color` を実行すれば、いつでもデフォルトの色に戻ります。
* **レジストリ**: コマンドプロンプトの既定の色は `HKEY_CURRENT_USER\Software\Microsoft\Command Processor` の `DefaultColor` で管理されています。
