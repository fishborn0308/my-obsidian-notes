# Command - Windows - cmd - cipher - EFS暗号化の管理と空き領域の消去

## 概要

`cipher` は、NTFS ボリューム上のディレクトリやファイルの暗号化を表示・変更するためのコマンドです。Windows の標準機能である **EFS (Encrypting File System)** を制御します。また、削除済みのデータが復元されないよう、ディスクの空き領域を物理的に上書き（ワイプ）する機能も備えています。

(出自: `Windows 標準搭載 - 外部コマンド (cipher.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `manage-bde` | **BitLocker** (ドライブ全体の暗号化) を管理する。 | ドライブ全体保護 |
| `sdelete` (Sysinternals) | 指定したファイルそのものを復元不可能な形で削除するツール。 | 特定ファイルの抹消 |
| `Get-Acl` / `icacls` | アクセス権を管理。暗号化は「鍵」による保護、ACLは「権限」による保護。 | 権限管理 |

## よく連携されるコマンド

### シナリオ例: 機密データフォルダの保護とバックアップ (運用・セキュリティ)

* **目的**: 盗難や紛失に備え、特定の重要データフォルダをユーザー固有の鍵で暗号化する。
* **連携コマンド**: `mkdir`, `cipher`, `robocopy`
* **解説**: フォルダに暗号化属性を付与すると、以降その中に保存されるファイルは自動的に暗号化されます。
* **コマンド例**:
    ```cmd
    REM フォルダを暗号化属性に設定
    cipher /e /s:"C:\SensitiveData"
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| (引数なし) | 現在のディレクトリ内のファイルが暗号化されているか（U=非暗号化, E=暗号化）を表示。 |
| ⭐ `/E` | 指定したディレクトリまたはファイルを暗号化する。 |
| ⭐ `/D` | 指定したディレクトリまたはファイルの暗号化を解除する。 |
| ⭐ `/W:directory` | **【アンチ・フォレンジック】** 指定したボリュームの空き領域からデータを完全に消去（3回上書き）する。 |
| `/K` | 現在のユーザー用に新しい証明書と鍵を作成する。 |
| `/S:directory` | 指定したディレクトリとそのすべてのサブディレクトリに対して操作を行う。 |
| `/C` | 暗号化エラーが発生しても処理を続行し、最後に概要を表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: PCの返却や廃棄前に、以前使用していたデータの痕跡をディスクから完全に消去する。
* **例**:
    ```cmd
    REM Cドライブの空き領域をすべて上書き消去（時間がかかるため注意）
    cipher /w:C:
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査において、攻撃者がツールを削除した後に `cipher /w` を実行した形跡がないか、イベントログや MFT (Master File Table) の不自然な更新を確認する。
* **注意**: `cipher /w` はファイルそのものを消すのではなく「空き領域」を消すため、実行中にディスク I/O がスパイクします。

### 3. レッドチーム視点 (アンチ・フォレンジックと偵察)

* **タスク 1: Anti-Forensics (証拠隠滅 - T1070.004)**
    * **目的**: 攻撃に使用したツールやログを削除した後、フォレンジック調査による復元を阻止する。
    * **解説**: ファイルを通常削除 (`del`) した後、`cipher /w` を実行することで、クラスタ上のデータ残渣を 0x00、0xFF、乱数の順で上書きします。
    * **コマンド例**:
        ```cmd
        del C:\Windows\Temp\exploit.exe
        cipher /w:C:\Windows\Temp
        ```

* **タスク 2: Encryption Enumeration (機密情報の探索)**
    * **目的**: システム内で既に暗号化されているファイルを特定し、そこが重要なデータの保管場所であると判断する。
    * **解説**: `cipher /s:C:\ /qu` などを実行して `E` 属性がついたファイルをリストアップし、ターゲットを絞り込みます。

## エラーメッセージとトラブルシューティング

1.  **The system cannot find the file specified.**
    * **原因**: パスの指定ミス、またはシステムファイルなどアクセス権がないファイルを操作しようとした。
    * **解決策**: パスを引用符で囲む、または管理者権限で実行する。

2.  **暗号化できない (FAT32/exFATボリューム)**
    * **原因**: `cipher` (EFS) は NTFS ボリューム限定の機能です。
    * **解決策**: ボリュームを NTFS に変換するか、別の暗号化ツールを検討してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **鍵の紛失**: EFS で暗号化したユーザーの証明書（鍵）をバックアップせずにユーザープロファイルを削除すると、データは**二度と復元できなくなります**。
* **情報の露出**: `cipher /w` はファイル名は消去しますが、MFT 内のメタデータ（かつてそこに何のファイルがあったかという名前情報）までは完全に消去しきれない場合があります。

## 注意点・補足

* **BitLocker との併用**: 一般的にはドライブ全体の BitLocker を推奨しますが、マルチユーザー環境で特定のユーザーからのみデータを隠したい場合に `cipher` (EFS) が有効です。
* **パフォーマンス**: 暗号化されたファイルへのアクセスは、CPU によるリアルタイムな復号処理を伴うため、極端に大量のファイルを暗号化するとパフォーマンスに影響が出る場合があります。
