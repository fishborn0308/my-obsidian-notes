# Command - Linux - pr - テキストファイルを印刷用に整形（改ページ・段組み）する

## 概要

`pr` は、テキストファイルをページ単位に分割し、ヘッダー（日付、ファイル名、ページ番号）を付与して出力します。また、複数のファイルを「マルチカラム（段組み）」で横に並べて表示する機能が非常に強力です。

## 🛠 テキスト表示ツールの比較

| コマンド | 特徴 | 用途 |
| :--- | :--- | :--- |
| `cat` | 内容をそのまま流し込む。 | 結合、単純な表示。 |
| **`pr`** | **ヘッダー付与、段組み、改ページ。** | 報告書作成、ファイル比較、印刷用。 |
| `column` | 列をきれいに揃える（テーブル化）。 | CSV風データの可視化。 |
| `more` / `less` | 画面単位でスクロール。 | 閲覧用（ページャー）。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-k`, `--columns=K` | **最頻用。** 出力を K 列の段組みにする。 |
| ⭐️ `-m`, `--merge` | **最頻用。** 複数のファイルを 1 行ずつ横に並べて表示する。 |
| ⭐️ `-t`, `--omit-header` | ヘッダーとフッターを表示しない（純粋な整形のみ）。 |
| ⭐️ `-h <string>` | ヘッダーの中央に表示する文字列を指定（デフォルトはファイル名）。 |
| ⭐️ `-n` | 行番号を付加する。 |
| `-l <N>` | 1 ページの行数を指定する（デフォルトは 66 行）。 |
| `-w <N>` | 1 ページの横幅（文字数）を指定する。 |

## コマンド例

```bash
# ファイルにヘッダーとページ番号を付けて表示
pr file.txt

# 2 つのファイルを左右に並べて比較する（diff より見やすい場合がある）
pr -m -t file1.txt file2.txt

# 1 つのファイルを 3 段組みにして、スペースを節約して表示
ls /usr/bin | pr -3 -t
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 簡易的なシステム監査レポートの作成。
- **解説**: 複数の設定ファイルやステータスを一つのファイルにまとめ、日付とタイトル入りのヘッダーを付けて出力することで、そのまま「証跡」として保存可能な形式に整えます。

### 2. ブルーチーム視点

- **タスク**: ログファイルの「横並び」調査。
- **解説**: 二つの異なるサーバーのアクセスログを時刻順に並べ、`-m` オプションで横に並べて表示することで、攻撃者の並行した動きを視覚的に追跡しやすくします。

### 3. レッドチーム視点

- **タスク**: データの難読化・偽装（稀なケース）。
- **解説**: 重要な機密データを `pr` の多段組みで整形し、一見すると何らかの無害なリストのように見せかけるなど、視覚的なカモフラージュに利用される可能性があります。

---

## 💡 プロの知恵：pr -m は簡易 diff

`diff` コマンドの結果が複雑すぎてわかりにくい時、`pr -m -t file1 file2` を使うと、二つのファイルが完全に左右対称に並びます。変更箇所を自分の目でじっくり見比べたい時に、これほど便利な方法はありません。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: `pr` 自体に脆弱性はほとんどありませんが、ヘッダーに「実行時の日付」や「ファイルパス」が自動で付与されるため、レポートを外部に公開する際に、予期せぬ内部情報（絶対パスなど）が混入しないよう注意が必要です。

### 対応策・緩和策

- **`-t` オプションの活用**: パス情報の漏洩を防ぎたい場合や、スクリプトで後処理をする場合は、余計なヘッダーを出さない `-t` を指定するのが定石です。
