# Command - Windows - cmd - chcp - コードページの表示・変更

## 概要

`chcp` (Change Code Page) は、コンソール ウィンドウで使用されるアクティブなコードページを表示、または変更するためのコマンドです。Windows では地域設定により標準の文字コードが異なりますが（日本語環境では通常 932: Shift-JIS）、これを変更することで UTF-8 などの他言語セットを正しく表示・処理できるようになります。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `[Console]::OutputEncoding` | PowerShell で文字コードを操作する際の方式。 | PS環境 |
| **レジストリ設定** | `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\CodePage` で恒久的な変更が可能だが、リスクが高い。 | 一時的なら `chcp` |

## よく連携されるコマンド

### シナリオ例: UTF-8 出力のツールを正しく表示する (実務・ペンテスト)

* **目的**: 海外製ツールや Python スクリプトなど、UTF-8 (65001) で出力されるテキストが日本語環境 (932) で文字化けするのを防ぐ。
* **連携コマンド**: `chcp`, `type`, `python`
* **解説**: ツール実行前にコードページを 65001 に切り替えます。
* **コマンド例**:
    ```cmd
    REM コードページを UTF-8 に変更
    chcp 65001
    REM 以降、UTF-8 のログ出力などが正しく表示される
    type utf8_log.txt
    ```



## 主要なコードページ番号

| 番号 | 文字エンコーディング | 備考 |
| :--- | :--- | :--- |
| **932** | ANSI/OEM 日本語 (Shift-JIS) | 日本語 Windows のデフォルト。 |
| ⭐ **65001** | UTF-8 | Unicode 対応。現代の多くのツールで推奨される。 |
| **437** | OEM 米国 (US-ASCII) | 英語環境のデフォルト。 |
| **1200** | UTF-16 | Unicode (LE) |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバー構築スクリプト内で、マルチバイト文字を含む設定ファイルを処理する前に、確実に Shift-JIS 環境であることを保証する。
* **例**:
    ```cmd
    REM 日本語環境に戻してから処理を開始
    chcp 932 > nul
    ```

### 2. ブルーチーム視点

* **タスク**: 多言語混在環境から収集された不審なログファイルを解析する際、文字化けによって重要な IP アドレスやファイル名を見逃さないように表示を切り替える。

### 3. レッドチーム視点 (環境適応と偵察)

* **タスク 1: UTF-8 ベースの攻撃ツールの実行**
    * **目的**: Go 言語や Rust で書かれた現代的なバイナリツール（出力が UTF-8 前提）を Windows 上で実行し、結果を正しく読み取る。
    * **コマンド例**:
        ```cmd
        chcp 65001
        .\modern_recon_tool.exe
        ```

* **タスク 2: システム言語の推測 (Enumeration)**
    * **目的**: `chcp` を引数なしで実行し、現在のコードページからターゲット端末の地域設定（言語圏）を推測する。
    * **解説**: `932` なら日本、`936` なら中国（簡体字）、`949` なら韓国といった具合に、地理的所在の裏付けに使用します。
    * **コマンド例**:
        ```cmd
        chcp
        ```

## エラーメッセージとトラブルシューティング

1.  **フォントによる制限**
    * **現象**: `chcp 65001` に切り替えても文字化けが直らない。
    * **原因**: コンソールのフォント（例: ラスターフォント）が Unicode に対応していない。
    * **解決策**: コンソールのプロパティから「MS ゴシック」や「Lucida Console」などの TrueType フォントに変更してください。

2.  **Invalid code page**
    * **原因**: 指定したコードページ番号が存在しない、または現在のシステムでサポートされていない。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **回避（Evasion）**: 
    * 攻撃者が特殊なコードページを利用して、セキュリティ製品のシグネチャ検知（文字列ベースの検知）を回避しようとする試みがあります。
    * 一部のログ収集ツールが特定のコードページ以外を正しくパースできない場合、その隙を突いて痕跡を隠蔽される可能性があります。

## 注意点・補足

* **スコープ**: `chcp` による変更は、**そのコマンドプロンプト セッション内でのみ有効**です。ウィンドウを閉じると元のデフォルト設定に戻ります。
* **リダイレクトへの影響**: `chcp 65001` を実行した状態で出力をファイルにリダイレクトすると、そのファイルは UTF-8 (BOMなし) で保存されます。
