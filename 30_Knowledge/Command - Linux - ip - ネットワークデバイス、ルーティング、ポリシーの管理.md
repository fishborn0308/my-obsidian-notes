# Command - Linux - ip - ネットワークデバイス、ルーティング、ポリシーの管理

## 概要

`ip` コマンドは、現代の Linux におけるネットワーク設定の標準ツールです（`iproute2` スイート）。従来の `ifconfig`, `route`, `arp` といった古いツールの機能をすべて一つに統合し、さらに VLAN, トンネル, ポリシーベースルーティングといった高度な設定までカバーします。

基本構造は **`ip [オプション] <オブジェクト> <サブコマンド>`** という形式をとります。

## 1. 主要オブジェクト（操作対象）サマリー

`ip` コマンドで最もよく使われるオブジェクトの一覧です。これらを使い分けることで、ネットワークの異なる層を操作します。

| オブジェクト | 略称 | 役割 | 置き換わる古いコマンド |
| :--- | :--- | :--- | :--- |
| **`address`** | `addr`, `a` | IP アドレスの表示・設定。 | `ifconfig` |
| **`link`** | `l` | インターフェース（L2層）の表示・状態変更。 | `ifconfig` (up/down/mtu等) |
| **`route`** | `r` | ルーティングテーブルの管理。 | `route`, `netstat -r` |
| **`neigh`** | `n` | ARP キャッシュ（隣接ノード）の管理。 | `arp` |
| **`rule`** | | ルーティングポリシー（特定の条件での経路制御）。 | (なし) |
| **`tunnel`** | | IP トンネルの設定。 | `iptunnel` |

## 2. 共通サブコマンド（アクション）

各オブジェクトに対して、以下のようなアクションを組み合わせて実行します。

* **`show` / `list`**: 情報の表示（デフォルトのアクション）。
* **`add`**: 新しい設定（IPや経路）の追加。
* **`del` / `delete`**: 設定の削除。
* **`set`**: 属性の変更（状態を UP にするなど）。
* **`flush`**: 設定の一括削除（全アドレスの消去など）。

## 3. 実践：よく使われるコマンド集

### アドレス管理 (`ip addr`)

```bash
# 全インターフェースのアドレスを表示
ip a

# eth0 に IP アドレスを追加
sudo ip addr add 192.168.1.10/24 dev eth0

# eth0 から IP アドレスを削除
sudo ip addr del 192.168.1.10/24 dev eth0
````

### リンク状態管理 (`ip link`)

Bash

```
# インターフェースを有効化 / 無効化
sudo ip link set eth0 up
sudo ip link set eth0 down

# MAC アドレスを偽装（変更）する
sudo ip link set eth0 address 00:11:22:33:44:55
```

### ルーティング管理 (`ip route`)

Bash

```
# ルーティングテーブルを表示
ip r

# デフォルトゲートウェイを設定
sudo ip route add default via 192.168.1.1
```

## 4. 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 一時的なネットワークセグメントの追加。
- **解説**: 同一 NIC に複数の IP を振る際、`ifconfig` のようなエイリアス名（eth0:1）を意識せず、単純に追加（add）するだけで管理できるのが `ip` の強みです。

### 2. ブルーチーム視点

- **タスク**: ネットワーク内の不審な隣接機器の特定。
- **解説**: `ip neigh` を使って ARP テーブルを確認し、ゲートウェイの MAC アドレスが偽装されていないか（ARPスプーフィング攻撃の有無）をチェックします。

### 3. レッドチーム視点

- **タスク**: ネットワークの隠蔽とピボッティング。
- **解説**: `ip rule` を使って特定の通信だけを別のインターフェースやトンネルに逃がす「ポリシーベースルーティング」を構築し、検知を回避しながら内部ネットワークを探索します。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **設定ミスによる DoS**: `ip addr flush` などを不用意に実行すると、リモート接続中の全 IP が消え、即座に通信不能（セルフ DoS）になります。
- **ルーティングの乗っ取り**: 攻撃者が `ip route` 権限を得ると、すべてのトラフィックを攻撃者のサーバー経由（Man-in-the-Middle）に変更されるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)

- **Sudo 制限**: 一般ユーザーには `ip` の参照（show）のみを許可し、設定変更（add, set, del）は厳格に制限します。
- **Audit**: ネットワーク設定の変更イベントを `auditd` で監視し、予期せぬ経路変更がないか記録します。

## 知っておくと便利なヒント

- **色付き表示**: `-c` オプションを付けると、出力がカラーになり格段に読みやすくなります (`ip -c a`)。
- **JSON 出力**: `-j` オプションを付けると、結果を JSON 形式で出力できます。`jq` コマンドと組み合わせることで、自動化スクリプトが非常に書きやすくなります。
    - 例: `ip -j a | jq '.[].addr_info[] | select(.family == "inet") | .local'`
