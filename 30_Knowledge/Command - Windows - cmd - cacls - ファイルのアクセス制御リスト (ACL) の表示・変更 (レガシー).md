# Command - Windows - cmd - cacls - ファイルのアクセス制御リスト (ACL) の表示・変更 (レガシー)

## 概要

`cacls` (Control Access Control Lists) は、NTFS ファイルシステムのアクセス制御リストを表示または変更するためのレガシーなコマンドラインツールです。

ユーザーやグループに対して、ファイルやディレクトリへの「読み取り」「書き込み」「変更」「フルコントロール」といった権限を割り当てます。

(出自: `Windows 標準搭載 - 現在は非推奨 (Deprecated)`)

> [!WARNING]
> `cacls` は現在、**非推奨**です。現代のシステム管理やペンテストでは、より正確に継承を扱える **`icacls`** の使用が強く推奨されます。

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `icacls` | `cacls` の後継。整合性レベル、継承、所有権をより正確に管理可能。 | `icacls` |
| `xcacls` | リソースキットに含まれていた拡張版。現在は `icacls` に統合。 | - |
| `takeown` | ファイルの所有権を強制的に取得する。 | - |
| `Get-Acl` / `Set-Acl` | PowerShell でオブジェクトとして ACL を操作する。 | PowerShell 環境 |

## よく連携されるコマンド

### シナリオ例: 脆弱な権限設定の探索 (レッドチーム視点)

* **目的**: 誰でも変更可能な実行ファイルやスクリプトを探し、権限昇格の足掛かりにする。
* **連携コマンド**: `for`, `cacls`, `findstr`
* **解説**: ディレクトリ内のファイルに対して `cacls` を一括実行し、`Everyone` や `Authenticated Users` に `(C)` (Change) や `(F)` (Full Control) が付与されているものを探します。
* **コマンド例**:
    ```cmd
    REM カレントディレクトリの全ファイルの権限を表示し、Everyoneに対する書き込み/変更権限を抽出
    for %f in (*) do cacls "%f" | findstr "Everyone:(C) Everyone:(F)"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示・基本** | |
| (引数なし) | 指定したファイルの ACL を表示する。 |
| ⭐ `/E` | ACL を置き換えるのではなく、**編集 (Edit)** する。これを忘れると既存の権限が全て消えるため注意。 |
| **権限付与・変更** | |
| `/G user:perm` | 指定したユーザーに権限を**付与 (Grant)** する。 |
| `/P user:perm` | 指定したユーザーの権限を**置換 (Replace)** する。 |
| `/R user` | 指定したユーザーの権限を**取り消す (Revoke)** (`/E` と併用必須)。 |
| `/D user` | 指定したユーザーのアクセスを**拒否 (Deny)** する。 |
| **権限の種類 (perm)** | |
| `R` | 読み取り (Read) |
| `W` | 書き込み (Write) |
| `C` | 変更 (Change) - 書き込みを含む |
| `F` | フル コントロール (Full Control) |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点 (レガシー環境)

* **タスク**: 特定の共有フォルダに対し、新しく追加されたグループに読み取り権限を追加する。
* **組み合わせ**: `/E /G "Group名":R`
* **解説**: 既存の権限を維持したまま (`/E`)、追加で付与します。
* **例**:
    ```cmd
    REM 既存の権限を維持しつつ、Developersグループに読み取り権限を追加
    cacls D:\Shared\Project /E /G "Developers":R
    ```

### 2. ブルーチーム視点

* **タスク**: セキュリティ監査で、重要ファイルの ACL が勝手に変更されていないか確認。
* **組み合わせ**: `cacls C:\Windows\System32\config\SAM`
* **解説**: 本来管理者以外アクセスできないはずのファイルに、不要な権限が付与されていないかチェックします。

### 3. レッドチーム視点

* **タスク**: 防御側が調査できないよう、ログファイルやツールフォルダへのアクセスをシステム全体で拒否する (Defense Evasion)。
* **組み合わせ**: `/E /D Everyone`
* **解説**: ファイルを削除するのではなく「見えなくする/アクセス不能にする」ことで、調査を妨害します。
* **例**:
    ```cmd
    REM Everyone（自分含む全員）からのアクセスを拒否してロックする（悪用厳禁）
    cacls C:\Payloads /E /D Everyone
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `Access is denied.`
    * **考えられる原因**: 管理者権限がない、または他のプロセスがファイルをロックしている。
    * **解決策**: 管理者として実行する。所有権がない場合は `takeown` で所有権を取得してから再試行する。

2.  **エラーメッセージ例 2**: `The system cannot find the file specified.`
    * **考えられる原因**: パスが間違っているか、ファイル名にスペースが含まれているのに引用符 `"` で囲んでいない。
    * **解決策**: パスを `"` で囲む。

## 環境変数と設定ファイル

`cacls` は直接 NTFS のメタデータを操作するため、特有の設定ファイルはありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: **Privilege Escalation (権限昇格)**。サービスのバイナリや、管理者権限で動作するバッチファイルの ACL が弱く（例: `Everyone:(F)`）、攻撃者がその中身を書き換えられる場合、容易に SYSTEM 権限を奪取されます。OSCP の試験でも「書き込み可能なパスの探索」は定石です。

### LOLBAS における利用例

* **機能**: `Defense Evasion` (アクセス権の操作による検知回避)
* **参照**: [LOLBAS - cacls.exe](https://lolbas-project.github.io/lolbas/Binaries/Cacls/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則 (PoLP) を適用し、不要な書き込み権限を削除する。
* **Detection (検知)**: **イベントID 4670**（オブジェクトへのアクセス許可が変更されました）を監視する。

## 注意点・補足

* **非推奨の理由**: `cacls` は「継承（Inheritance）」の処理が不完全であり、現代の複雑な NTFS 構造（上位フォルダからの継承設定など）を壊してしまうリスクがあります。
* **次なるステップ**: 同様の操作をより安全かつ高度に行うには **`icacls`** の学習が必須です。
