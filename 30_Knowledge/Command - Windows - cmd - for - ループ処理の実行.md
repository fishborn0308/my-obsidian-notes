# Command - Windows - cmd - for - ループ処理の実行

## 概要

`for` コマンドは、指定されたセット（ファイル群、数値範囲、コマンドの出力結果など）に対して、指定したコマンドを繰り返し実行します。

単なるファイル操作にとどまらず、`/f` スイッチを用いることでテキストファイルやコマンド出力を 1 行ずつ解析し、特定の要素を抽出して変数に格納するといったプログラミング的な処理が可能です。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ForEach-Object` | PowerShell 版。オブジェクトをそのまま扱えるため、プロパティ操作に強い。 | PS 環境 |
| `while` | 条件が真の間繰り返す (Bash 等)。CMD では `if` と `goto` で代用する。 | (Linux環境) |
| `powershell -c "1..254 | % { ... }"` | 数値範囲のループ。CMD の `for /l` より記述が短い。 | ワンライナー |

## よく連携されるコマンド

### シナリオ例: 生存ホストの列挙 (Ping Sweep)

* **目的**: 特定のサブネット（例: 192.168.1.0/24）内で、稼働しているホストを高速に特定する。
* **連携コマンド**: `for /l`, `ping`, `find`
* **解説**: `/l` スイッチで数値を生成し、各 IP に対して Ping を 1 回ずつ送信、応答があったものだけを表示します。
* **コマンド例**:
    ```cmd
    REM 192.168.1.1 から 254 までスキャン
    for /l %i in (1,1,254) do @ping -n 1 -w 100 192.168.1.%i | find "Reply from"
    ```

## スイッチ/オプション説明

**※重要: コマンドラインでは `%i`、バッチファイル内では `%%i` と記述する必要があります。**

| スイッチ | 説明 |
| :--- | :--- |
| (なし) | 指定したファイルセットに対してループする。 |
| ⭐ `/d` | **ディレクトリのみ**を対象にループする。 |
| ⭐ `/r` | **再帰的**にディレクトリを辿り、指定したファイルを探してループする。 |
| ⭐ `/l` | **数値範囲**を指定してループする。`(開始,ステップ,終了)`。 |
| ⭐ `/f` | **ファイルの内容やコマンドの出力**をパース（解析）してループする。 |

### /f スイッチの詳細パラメータ (Options)

`for /f "options" %i in (...) do ...`

* `tokens=n,m`: 各行の何番目の単語を抽出するか。
* `delims=xxx`: 区切り文字の指定（デフォルトはスペースとタブ）。
* `skip=n`: 最初の n 行をスキップする。

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディレクトリ内のすべての `.log` ファイルを、末尾に日付を付けて `.bak` に一括リネームする。
* **例**:
    ```cmd
    REM カレントディレクトリの全ログをバックアップ
    for %f in (*.log) do copy "%f" "%f.bak"
    ```

### 2. ブルーチーム視点 (監査・調査)

* **タスク**: 特定のディレクトリツリーに含まれる全実行ファイル (.exe) のハッシュを一括で計算し、リスト化する。
* **例**:
    ```cmd
    REM /r で全サブディレクトリを走査し CertUtil でハッシュ化
    for /r C:\Tools %i in (*.exe) do @certutil -hashfile "%i" SHA256 >> hashes.txt
    ```

### 3. レッドチーム視点 (偵察・列挙・攻撃)

* **タスク 1: User Enumeration via SID (SID からのユーザー名列挙)**
    * **目的**: SID の末尾（RID）をループさせ、有効なユーザー名を網羅的に取得する。
    * **コマンド例**:
        ```cmd
        for /l %i in (1000,1,1050) do @wmic useraccount where "sid like '%-%i'" get name 2>nul
        ```

* **タスク 2: Local File Inclusion (LFI) 後の機密情報一括読み取り**
    * **目的**: 取得したファイルリストのテキストを 1 行ずつ読み込み、中身を表示またはダウンロードする。
    * **コマンド例**:
        ```cmd
        REM files.txt に書かれたパスを順に type で表示
        for /f "delims=" %i in (files.txt) do @type "%i"
        ```

* **タスク 3: SMB Brute Force (簡易的なパスワード試行)**
    * **目的**: ユーザーリストに対して、特定のパスワードで `net use` を試行し、ログインできるか確認する。

## エラーメッセージとトラブルシューティング

1.  **%i の指定が正しくありません (%%i was unexpected at this time)**
    * **原因**: コマンドラインで `%%i` を使った、あるいはバッチファイルで `%i` を使った。
    * **解決策**: コマンドプロンプト直打ちは `%` 1つ。バッチファイル（.bat/.cmd）内は `%%` 2つにする。

2.  **tokens で指定した変数が参照できない**
    * **原因**: `tokens=1,2` とした場合、1番目は `%i`、2番目はアルファベット順で次の `%j` に自動的に割り当てられる。
    * **解決策**: 続く文字（%j, %k...）を意識してコマンドを記述する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **コマンドインジェクションの増幅**:
    * `for` ループ内にユーザー入力を不適切に組み込むと、単一のインジェクションでシステム全体の破壊や、ネットワーク全体への攻撃（ワームのような挙動）を自動実行される恐れがあります。

### LOLBAS における利用例

* **機能**: `Execute`, `Reconnaissance`
* **特性**: `for` 自体はシェル機能ですが、他の LOLBAS バイナリと組み合わせることで、「全ドライブからのデータ探索」や「シグネチャベースの検知を回避する逐次実行」などを自動化する基盤となります。

## 注意点・補足

* **遅延環境変数の展開**: ループ内で `set` した変数を同じループ内で使いたい場合は、`cmd /v:on` を起動し、`!var!` 形式（遅延展開）を使用する必要があります。
* **ファイル名にスペース**: ループ対象にスペースが含まれる可能性がある場合は、必ず `"%i"` のように引用符で囲んでください。
