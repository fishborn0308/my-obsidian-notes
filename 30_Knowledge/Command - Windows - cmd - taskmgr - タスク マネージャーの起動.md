# Command - Windows - cmd - taskmgr - タスク マネージャーの起動

## 概要

`taskmgr` は、実行中のプロセス、サービスの監視、リソースの使用状況（CPU、メモリ等）、およびスタートアップ プログラムの管理を行うための GUI ツールです。

ハングアップしたアプリケーションの強制終了や、不審なプロセスの特定、簡易的なパフォーマンス診断に利用されます。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `tasklist` | **CLI版。** 現在実行中のプロセス一覧をテキストで出力。スクリプトに最適。 | `tasklist` (自動化) |
| `taskkill` | **CLI版。** プロセスIDや名前を指定して強制終了させる。 | `taskkill` (自動化) |
| `resmon` / `perfmon` | より詳細なリソース（ネットワーク接続先やディスクI/O）の調査。 | `resmon` (詳細調査) |

## よく連携されるコマンド

### シナリオ例: デスクトップがフリーズした際の復旧 (インフラ運用)

* **目的**: `explorer.exe` (シェル) がクラッシュして画面が真っ白になった際、再起動なしに復旧させる。
* **連携コマンド**: `explorer.exe`
* **解説**: `Ctrl+Shift+Esc` またはコマンドから `taskmgr` を起動し、「新しいタスクを実行」からシェルを再起動します。
* **コマンド例**:
    ```cmd
    REM コマンドプロンプトからタスクマネージャーを呼び出す
    taskmgr
    ```

## 主要なタブと機能説明

| タブ名称 | 説明 |
| :--- | :--- |
| **プロセス** | CPU、メモリ、ディスク、ネットワークの消費量をアプリごとに表示。 |
| ⭐ **パフォーマンス** | リソース使用率のグラフ表示。ここから `resmon` も起動可能。 |
| ⭐ **スタートアップ** | OS起動時に自動実行されるアプリの有効/無効を切り替える（旧msconfigの機能）。 |
| **ユーザー** | ログイン中の各ユーザーが消費しているリソースを表示。 |
| ⭐ **詳細** | 旧来の「プロセス」タブ。PID (プロセスID) や実行パスの確認に必須。 |
| **サービス** | システム上で動作している各サービスの状態確認と開始/停止。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リソース枯渇の原因プロセスの特定
* **解説**: CPU 100% 状態などの際に、どのプロセスが「暴走」しているかを瞬時にソートして特定し、右クリックから「ダンプ ファイルの作成」を行って後でデバッグ解析します。

### 2. ブルーチーム視点

* **タスク**: 不審なプロセスの初期調査
* **解説**: 「詳細」タブで「コマンド ライン」や「イメージ パス名」の列を表示させ、`svchost.exe` が不自然な場所（例: `C:\Users\Public`）から実行されていないか目視で確認します。
* **例**: `taskmgr` > 詳細タブ > 列の選択 > 「コマンド ライン」にチェック。

### 3. レッドチーム視点 (OSCP / Evasion)

* **タスク**: 防御製品 (AV/EDR) の特定
* **解説**: 侵入後、どのようなセキュリティ製品（例: `MsMpEng.exe`, `cb.exe`, `SentinelOne.exe`）が動いているかを確認し、攻撃手法を調整します。
* **例**: `taskmgr` の詳細タブでベンダー名やプロセス名を確認。

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `タスク マネージャーは管理者によって無効にされています。`
    * **考えられる原因**:
        - マルウェアが自身の停止を防ぐためにレジストリを書き換えた。
        - 組織のグループポリシー (GPO) で制限されている。
    * **解決策**: レジストリの `DisableTaskMgr` 値を確認・修正するか、`tasklist` 等の CLI ツールで代用する。

2.  **現象**: 「詳細モード」が表示されず、簡易表示しか出ない。
    * **解決策**: ウィンドウ下部の「詳細」をクリックして拡張表示に切り替える。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Process Ghosting / Hollowing**: 攻撃者が正規プロセス（例: `notepad.exe`）を偽装して実行している場合、`taskmgr` 上では正常に見えますが、メモリの中身は悪意あるコードに置き換わっていることがあります。これを見抜くには `resmon` や `procexp` (Sysinternals) が必要です。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    - **イベントID 4688**: `taskmgr.exe` の起動ログを監視。管理者がいないはずの時間帯の起動は、攻撃者の偵察活動の可能性があります。

## 注意点・補足

* **ショートカット**: `Ctrl + Shift + Esc` は、`Ctrl + Alt + Del` を経由せずに直接 `taskmgr` を開くことができる最速の方法です。
* **管理者権限**: 一般ユーザーでも起動可能ですが、他ユーザーのプロセス終了や詳細なシステム情報の閲覧には管理者権限への昇格（UAC承認）が必要です。
