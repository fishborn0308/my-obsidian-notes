# Command  - Linux - zip - ファイルやディレクトリを圧縮・アーカイブする

## 概要

`zip` コマンドは、ファイルやディレクトリを `.zip` 形式のアーカイブファイルに圧縮・格納するためのユーティリティです。`.zip` 形式はWindows, macOS, Linux間で広く互換性があり、ファイルの配布やバックアップに頻繁に利用されます。`zip` は単なるアーカイブ機能だけでなく、圧縮機能やパスワードによる暗号化機能も内蔵しています。対になるコマンドとして、展開(解凍)には `unzip` を使用します。

(出自: `zip` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`tar`** | `tar` はアーカイブに特化し、圧縮は `gzip` (`-z`) などと連携します。Linux/Unixのパーミッション保持に優れています。`zip` はWindowsとの互換性が高いです。 | `tar` (Linux環境) |
| **`gzip`, `bzip2`** | これらは**単一のファイル**を圧縮するツールです。`zip` は複数のファイルやディレクトリを直接扱えます。 | - |
| **`7z`** | より高い圧縮率を誇る `7-Zip` 形式などを扱います。互換性重視なら `zip`、圧縮率重視なら `7z` を検討します。 | `7z` (高圧縮) |

## よく連携されるコマンド

`zip` は `find` コマンドと連携して、特定の条件に合うファイルだけをアーカイブする際によく使われます。

### シナリオ例: 特定の拡張子を持つファイルのみをアーカイブする (インフラ構築・運用視点)

* **目的**: プロジェクトディレクトリ内にあるソースコード(`.c`, `.h` ファイル)だけを検索し、1つのzipファイルにまとめる。
* **連携コマンド**: `find`, `xargs`
* **解説**: `find` コマンドで対象ファイルをリストアップし、その結果を `xargs` に渡して `zip` コマンドで処理します。
* **コマンド例**:

```bash
# ファイル名にスペースが含まれていても安全に処理するため -print0 と -0 を使用
find . -type f \( -name "*.c" -o -name "*.h" \) -print0 | xargs -0 zip source_codes.zip
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ファイル/ディレクトリ操作** | |
| ⭐ `-r` | (recursive) ディレクトリを再帰的に圧縮対象に含めます。ディレクトリ圧縮時の必須オプションです。 |
| `-x <pattern>` | 指定したパターンに一致するファイルやディレクトリを圧縮対象から除外します。 |
| **暗号化** | |
| `-e` | (encrypt) パスワードを設定してアーカイブを暗号化します。 |
| **出力制御** | |
| `-[0-9]` | 圧縮レベルを指定します。`-0`(無圧縮)から `-9`(最高圧縮)まで。 |
| `-q` | (quiet) 実行中のメッセージを抑制し、静かに出力します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Webサーバーのドキュメントルートをバックアップのために1つのファイルにまとめる。
* **組み合わせ**: `zip -r <archive_name>.zip <directory_to_compress>`
* **解説**: `-r` オプションを使い、ディレクトリ全体を再帰的に圧縮する最も基本的な使い方です。
* **例**:

```bash
# /var/www/html ディレクトリ全体を web_backup.zip として圧縮
zip -r web_backup_$(date +%F).zip /var/www/html/
```

### 2. ブルーチーム視点

* **タスク**: 証拠ファイルを、関係者に安全に送付するためにパスワード付きで暗号化する。
* **組み合わせ**: `zip -e evidence.zip <files...>`
* **解説**: `-e` (`encrypt`) オプションを使い、証拠ファイルをパスワード付きで暗号化します。データの機密性を確保し、意図しない開封を防ぎます。
* **例**:

```bash
# 収集したログファイルと不審なバイナリをパスワード付きで圧縮
zip -e case001_evidence.zip /var/log/messages /tmp/suspicious_binary
```

### 3. レッドチーム視点

* **タスク**: 窃取したデータをパスワード付きで圧縮し、隠蔽・持ち出しの準備をする。
* **組み合わせ**: `zip -r -e exfil.zip /path/to/stolen_data`
* **解説**: 窃取した機密情報を1つのファイルにまとめ、暗号化してデータ持ち出し (Exfiltration) の準備をします。暗号化することで、ネットワーク監視 (DLPなど) による検知を回避しやすくなります。
* **例**:

```bash
# ユーザーのホームディレクトリから.sshと.bash_historyをパスワード付きで圧縮
zip -r -q -e secret.zip /home/user/.ssh /home/user/.bash_history
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **ディレクトリを圧縮しようとしたが、空のzipファイルが作成される。**
		* **考えられる原因**: `-r` (recursive) オプションを付け忘れています。
		* **解決策**: `zip -r archive.name.zip directory_name` のように、必ず `-r` オプションを追加してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **Zip Slip / Path Traversal** 脆弱性。アーカイブ内のファイル名に `../../..` のようなパスを含めることで、展開時に想定外のディレクトリにファイルを書き込ませる攻撃。
* **悪用シナリオ**: 攻撃者は、Webシェルの設置などを狙い、アプリケーションのファイルアップロード機能などを悪用して、細工したzipファイルを展開させようとします。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (間接的)
* **解説**: `zip` で作成したアーカイブを `sudo unzip` で展開する際に、SUIDビットを付与したバイナリを含めることで、権限昇格に繋がる可能性があります。
* **参照**: `https://gtfobins.github.io/gtfobins/zip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 信頼できないソースからのzipファイルは安易に展開しない。ファイルアップロード機能では、ファイル名に `../` が含まれていないか厳しくチェックする。
* **Detection (検知)**: `unzip -l` で展開前に内容を確認する。

## 注意点・補足

* **パスワードの強度**: `zip` の標準的な暗号化 (ZipCrypto) は比較的強度が低く、総当たり攻撃に弱いとされています。機密性の高いデータを扱う場合は、AES-256暗号化に対応した `7z` を使うか、`gpg` などで暗号化してからzipで圧縮することを推奨します。
