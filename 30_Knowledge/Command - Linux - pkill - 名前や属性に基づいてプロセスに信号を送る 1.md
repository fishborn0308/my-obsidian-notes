# Command - Linux - pkill - 名前や属性に基づいてプロセスに信号を送る

## 概要

`pkill` は、指定したパターン（正規表現）に一致するプロセスに対して、シグナル（デフォルトは `SIGTERM`）を送信します。`kill` コマンドが特定の PID を狙い撃ちする「拳銃」なら、`pkill` は条件に合うものを一掃する「散弾銃」のような特性を持っています。

## 🛠 プロセス終了ツールの三つ巴

| コマンド | ターゲット指定 | 特徴 |
| :--- | :--- | :--- |
| `kill` | **PID (数字)** | 最も正確。個別のプロセスを確実に操作する際に使用。 |
| **`pkill`** | **名前 / パターン** | 柔軟性が高い。正規表現や実行ユーザーでの指定が可能。 |
| `killall` | **名前 (完全一致)** | 指定した名前と完全に一致するプロセスをすべて殺す。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-f`, `--full` | 引数を含む**コマンドライン全体**をマッチ対象にする。 |
| ⭐️ `-u <user>` | 特定のユーザーが実行しているプロセスのみを狙う。 |
| ⭐️ `-t <tty>` | 特定のターミナル（pts/1 など）に関連付けられたプロセスを対象にする。 |
| ⭐️ `-9`, `--signal KILL` | 容赦ない強制終了 (`SIGKILL`) を送る。 |
| ⭐️ `-v`, `--inverse` | 条件に**一致しない**プロセスを対象にする（逆転指定）。 |

## 実戦コマンド例

```bash
# 「chrome」という名前を含むプロセスをすべて終了
pkill chrome

# スクリプトのフルパスを指定して終了させる
pkill -f /home/user/scripts/monitor.py

# 特定のユーザー (alice) のプロセスをすべて強制終了
sudo pkill -u alice -9

# 設定ファイルを再読み込みさせる (HUPシグナルを nginx に送信)
sudo pkill -HUP nginx
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 固まった SSH セッションの強制切断。
- **解説**: ネットワークトラブル等でゴースト化したセッションを、端末番号指定で一気に掃除します。
- **例**: `sudo pkill -t pts/2`

### 2. ブルーチーム視点

- **タスク**: 不審なリバースシェルの即時遮断。
- **解説**: 侵入者が `nc` (Netcat) や `python` でコネクションを張っている疑いがある場合、そのプロセス名やコマンドライン引数のパターンを指定して即座に通信を断ち切ります。
- **例**: `sudo pkill -f "python -c.*import socket"`

### 3. レッドチーム視点

- **タスク**: 監視エージェントの無効化。
- **解説**: 自分の活動をログに残す `auditd` や、セキュリティスキャンを行うプロセスを `pkill` で密かに停止させます。管理者が気づかないよう、定期的に実行するスクリプトを仕込むこともあります。

## ⚠️ 防御の知恵：予行演習の徹底

`pkill` は強力すぎて、間違ったプロセスを巻き込む「誤爆」が致命的です（例：`pkill -f .` と打つと、ほぼすべてのプロセスが対象になります）。

実行前に必ず **`pgrep -fl <パターン>`** を実行し、どのプロセスが対象になるかを確認する癖をつけましょう。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **自己 DoS (Denial of Service)**: root 権限で不用意なパターン指定を行うと、システム維持に必要なプロセスまで殺してしまい、システムがハングアップします。

### 対応策・緩和策

- **sudoers による制限**: `/etc/sudoers` で、特定のユーザーが `pkill` で root 権限のプロセスを殺せないよう制限をかけることが推奨されます。
