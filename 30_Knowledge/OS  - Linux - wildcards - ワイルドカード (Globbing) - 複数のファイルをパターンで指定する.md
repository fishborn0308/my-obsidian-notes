# OS  - Linux - wildcards - ワイルドカード (Globbing) - 複数のファイルをパターンで指定する

## 概要

ワイルドカード（またはグロブ）は、複数のファイル名やディレクトリ名を、少数の文字を使った**パターン**で一度に指定するためのシェル機能です。`ls`, `cp`, `mv`, `rm` といった多くのコマンドと組み合わせて使用します。

重要なのは、`ls *.txt` と入力した場合、`ls` コマンドが `*` という文字を受け取るのではなく、**コマンドが実行される前にシェルが `*` を展開**し、カレントディレクトリにある `.txt` で終わる全てのファイル名のリスト (`a.txt b.txt c.txt ...`) に置き換えてから `ls` コマンドに渡しているという点です。

(出自: `bash`, `zsh` などのシェルの基本機能)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **正規表現 (Regex)** | ワイルドカードは**ファイル名**のマッチングに使われます。正規表現は主に**テキストの中身**のパターンマッチングに使われます (`grep`など)。構文も似て非なるものであるため注意が必要です。 | - |

## よく連携されるコマンド

ワイルドカードは、ファイル操作コマンドと連携してその真価を発揮します。

### シナリオ例: 複数のログファイルをまとめてバックアップする (インフラ構築・運用視点)

* **目的**: `/var/log` ディレクトリにある、`.log` で終わる全てのファイルをバックアップディレクトリにコピーする。
* **連携コマンド**: `cp`
* **解説**: `*` ワイルドカードを使って、パターンに一致する全てのファイルを一度に `cp` コマンドの引数として渡します。
* **コマンド例**:

```bash
# /var/log/ 内の全 .log ファイルを /mnt/backups/ にコピー
sudo cp /var/log/*.log /mnt/backups/
```

## オプション説明 (ワイルドカード文字)

| オプション | 説明 |
| :--- | :--- |
| **主要なワイルドカード** | |
| ⭐ `*` | **任意の0文字以上の文字列**にマッチします。最もよく使われます。（例: `*.log`） |
| `?` | **任意の1文字**にマッチします。（例: `image_?.jpg`） |
| `[...]` | 角括弧内の**いずれか1文字**にマッチします。（例: `file[abc].txt`） |
| **拡張グロブ (Bash)** | (`shopt -s extglob` で有効化) |
| `!(pattern-list)` | パターンのいずれか**以外**の全てにマッチします。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディレクトリ内の `.iso` と `.zip` ファイル**以外**の全ての一時ファイルを削除する。
* **組み合わせ**: `rm !(*.iso|*.zip)` (拡張グロブ)
* **解説**: `!` を使った否定マッチングで、特定のファイルを除外して一括操作を行います。
* **例**:

```bash
# 拡張グロブを有効化
shopt -s extglob
# .iso と .zip 以外を削除
rm -v !(*.iso|*.zip)
```

### 2. ブルーチーム視点

* **タスク**: `/tmp` ディレクトリに隠された不審なファイルやディレクトリを調査する。
* **組み合わせ**: `ls -ld /tmp/.*`
* **解説**: `.*` パターンで**全ての隠しファイル・ディレクトリ**をリストアップします。攻撃者はしばしば `/tmp` に `.x` のような隠しディレクトリを作成してツールを隠すため、このパターンでの調査は不可欠です。
* **例**:

```bash
# /tmp 配下の隠しファイル・ディレクトリを詳細表示
ls -ld /tmp/.*
```

### 3. レッドチーム視点

* **タスク**: **情報収集**のため、システム上の全ユーザーの `.ssh` ディレクトリをまとめてアーカイブする。
* **組み合わせ**: `tar czf loot.tar.gz /home/*/.ssh`
* **解説**: `*` を使って複数のユーザーのホームディレクトリを横断的に探索し、秘密鍵、設定ファイル、コマンド履歴などを効率的に収集します。
* **例**:

```bash
# 全ユーザーの .bash_history を収集
cat /home/*/.bash_history > all_history.txt
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `bash: /bin/rm: Argument list too long`
		* **考えられる原因**: ワイルドカードが数千・数万のファイルに展開され、コマンドラインの引数の長さの上限を超えてしまいました。
		* **解決策**: `find` コマンドと `-exec` オプションや `xargs` を使います。

```bash
# 大量の .tmp ファイルを安全に削除する
find . -name "*.tmp" -print0 | xargs -0 rm
```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **意図しないファイルのマッチ**。とくに `rm` コマンドとの組み合わせは非常に危険です。
* **悪用シナリオ**: `rm -rf $MY_VAR/*` のようなスクリプトで、変数 `$MY_VAR` が空だった場合、`rm -rf /*` と解釈されてシステム全体が削除される大惨事を引き起こします。

### GTFOBins / LOLBAS における利用例

ワイルドカードはシェルの機能であり、GTFOBinsにはリストされていません。しかし、リストされている多くのコマンドが悪用される際に、ワイルドカードと組み合わせて使われます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: スクリプト内で変数を使ってパスを指定する場合は、`rm -rf "${MY_VAR:?}"/*` のように、変数が未定義の場合にエラーで終了させる防御策を講じる。
* **Detection (検知)**: `auditd` で `rm` や `chown` の実行を監視し、引数に `*` が含まれている場合や、異常に多数のファイルが一度に操作されている場合にアラートを出す。

## 注意点・補足

* **隠しファイル**: デフォルトでは、`*` や `?` は `.` で始まる隠しファイルにはマッチしません。隠しファイルも含めたい場合は、`.*` のように明示的に指定するか、`bash` の場合は `shopt -s dotglob` を実行します。
