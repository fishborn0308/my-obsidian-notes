# Command - Windows - cmd - w32tm - Windows タイム サービスの設定と診断

## 概要

`w32tm` は、Windows Time サービス (W32Time) の構成、監視、およびトラブルシューティングを行うための包括的なツールです。

NTP (Network Time Protocol) サーバーとの同期状態の確認、手動同期の強制、同期先の設定など、システムの時刻管理に関するすべての詳細な操作を行います。

(出自: Windows XP 以降標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `time` / `date` | 時刻・日付の表示と手動設定のみ。同期の管理は不可。 | `time` (簡易確認) |
| `net time` | リモートマシンの時刻確認や簡単な同期。古い方式であり現在は `w32tm` が推奨される。 | `w32tm` |
| `sc query w32time` | Windows Time サービス自体の実行状態（起動/停止）を確認する。 | `sc` |

## よく連携されるコマンド

### シナリオ例: 同期エラー発生時の再同期フロー (インフラ運用)

* **目的**: 時刻がずれて認証エラーが出ている端末を、ドメインコントローラーに強制的に再同期させる。
* **連携コマンド**: `net start`, `ipconfig`
* **解説**: サービスが動いていることを確認してから、再同期を命じます。
* **コマンド例**:
    ```cmd
    REM サービスを起動（停止している場合）
    net start w32time
    REM 時刻の再同期を強制実行
    w32tm /resync
    ```

## スイッチ/オプション説明

`w32tm` はサブコマンド形式で動作します。

| サブコマンド | 説明 |
| :--- | :--- |
| ⭐ `/query /status` | 現在の同期状態（同期先、最終同期時刻、精度など）を表示する。 |
| ⭐ `/resync` | ターゲットとなるサーバーに対して即座に再同期を試みる。 |
| ⭐ `/config` | NTP サーバーの指定や、同期の間隔などの設定を変更する。 |
| `/monitor` | ドメインコントローラーなどの時刻のズレを一覧表示する。 |
| `/stripchart` | 指定したホストとの時刻の差をリアルタイムでグラフ（テキスト）表示する。 |
| `/tz` | 現在のタイムゾーン設定を表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 外部 NTP サーバー（例: ntp.nict.jp）との同期設定
* **解説**: ドメイン内の最上位サーバー（PDC エミュレーター）などで、外部の正確な時刻ソースを参照するように設定します。
* **例**:
    ```cmd
    REM 外部NTPサーバーを指定し、設定を即時反映（要管理者権限）
    w32tm /config /manualpeerlist:"ntp.nict.jp,0x8" /syncfromflags:manual /reliable:yes /update
    ```

### 2. ブルーチーム視点

* **タスク**: 時刻同期の異常によるログの不整合調査
* **解説**: 複数のサーバーでログの時刻がバラバラな場合、どのサーバーがどの NTP サーバーを参照しているかを一括調査し、信頼できるソースに統一します。
* **例**: `w32tm /query /source` で現在のソースを素早く確認。

### 3. レッドチーム視点

* **タスク**: 認証プロトコルの脆弱性調査
* **解説**: ターゲットの時刻同期設定が不適切（認証なしの NTP 参照など）であれば、偽の NTP 応答を送ることで時刻を操作し、Kerberos チケットの有効期限を狂わせる等の攻撃の可能性を検討します。

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `w32tm /resync` を実行すると「サービスが開始されていません」と出る。
    * **原因**: Windows Time サービスが「無効」または「停止」している。
    * **解決策**: `sc config w32time start= auto` を実行し、`net start w32time` で開始する。

2.  **現象**: `w32tm /resync` を実行すると「時刻データがないため、コンピューターは再同期しませんでした」と出る。
    * **原因**: ネットワークの問題で NTP サーバーに到達できない、または UDP 123 ポートが遮断されている。
    * **解決策**: ファイアウォール設定を確認し、`w32tm /stripchart /computer:<Target>` で疎通を確認する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Replay Attack**: システム時刻を過去に戻すことで、すでに無効化されたはずのセッションやパスワードを一時的に有効に見せかける。
    - **Log Obfuscation**: 時刻を大幅に進めたり遅らせたりすることで、攻撃の痕跡をタイムライン上で見つけにくくする。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - ドメイン環境では、クライアントが外部 NTP を直接参照するのを禁止し、必ずドメインコントローラーを参照するように GPO で強制する。
* **Detection (検知)**:
    - **イベントID 4616**: システム時刻の変更を監視。
    - **イベントID 35 (W32Time)**: 時刻同期のソースが予期せぬものに変更された場合に警告。

## 注意点・補足

* **`0x8` の意味**: 手動設定 (`/manualpeerlist`) 時の `0x8` フラグは、クライアントとして NTP 要求を送信することを意味します。
* **ADの基本ルール**: ドメインに参加している端末は、デフォルトで「自分の所属するドメインの DC」を自動的に参照します。基本的には手動設定は不要で、`/resync` だけで事足ります。
