# Command - Windows - cmd - exit - インタープリターまたはスクリプトの終了

## 概要

`exit` コマンドは、現在の `cmd.exe` インスタンス（コマンド プロンプト ウィンドウ）を終了させます。バッチファイル内で使用する場合は、呼び出し元のプロセスに終了コード（Exit Code / Errorlevel）を返しながら、スクリプトの実行を停止させることができます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `exit /b` | **バッチファイルのみ**を終了し、呼び出し元の CMD ウィンドウは閉じない。 | サブルーチン用 |
| `goto :eof` | バッチファイルの末尾へジャンプして暗黙的に終了する。 | スクリプト末尾 |
| `taskkill /im cmd.exe` | 外部から CMD プロセスを強制終了する。 | 強制終了時 |

## よく連携されるコマンド

### シナリオ例: 条件分岐によるエラー終了 (自動化・運用)

* **目的**: 重要なファイルが見つからない場合に、エラーコード `1` を返してスクリプトを安全に停止させる。
* **連携コマンド**: `if`, `exit /b`
* **解説**: `%ERRORLEVEL%` を通じて呼び出し元（親スクリプトやジョブ管理システム）に異常を伝えます。
* **コマンド例**:
    ```cmd
    @echo off
    if not exist "C:\config\settings.ini" (
        echo [ERROR] Config file not found.
        exit /b 1
    )
    echo Success.
    exit /b 0
    ```



## スイッチ/オプション説明

| スイッチ/引数 | 説明 |
| :--- | :--- |
| ⭐ `/b` | **【重要】** 現在のコマンド プロンプトを終了せずに、現在の「バッチ スクリプト」のみを終了する。 |
| `exitCode` | 呼び出し元に返す数値を指定する。通常、`0` は成功、`1` 以上はエラーを意味する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 多数の構築用バッチを順番に実行する親スクリプトにおいて、子スクリプトの失敗を検知して停止する。
* **例**:
    ```cmd
    call setup_db.bat
    if %ERRORLEVEL% neq 0 (
        echo DB Setup Failed.
        exit /b %ERRORLEVEL%
    )
    ```

### 2. ブルーチーム視点

* **タスク**: インシデント調査スクリプトの最後に、収集完了を示す特定の終了コードを返し、SIEM 等のオーケストレーションツールが次のアクション（ログの自動回収など）へ移れるようにする。

### 3. レッドチーム視点 (隠蔽と自動化)

* **タスク 1: Cleanup and Close (セッションの終了と痕跡隠蔽)**
    * **目的**: リモートシェルや対話型セッションでの作業終了後、ウィンドウを確実に閉じてプロセスを消去する。
    * **解説**: 痕跡を残さないよう、作業の最後に `exit` を実行してセッションを切断します。

* **タスク 2: Conditional Payload Execution (条件付きペイロード実行)**
    * **目的**: 前の攻撃ステップが成功した (`0`) か失敗した (`1`) かを判定し、次のエクスプロイトを実行するか決める。
    * **コマンド例**:
        ```cmd
        REM ペイロード1を実行し、成功した場合のみペイロード2を実行
        cmd /c "payload1.exe & if !ERRORLEVEL! equ 0 payload2.exe"
        ```

## エラーメッセージとトラブルシューティング

1.  **ウィンドウが勝手に閉じてしまう**
    * **原因**: バッチファイル内で `/b` なしの `exit` を使用している。
    * **解決策**: 現在の CMD プロセスを維持したい場合は、必ず `exit /b` を使用してください。

2.  **終了コードが反映されない**
    * **原因**: `setlocal` / `endlocal` のネスト内で `exit /b` を使う際、変数の展開タイミングによって期待したコードが返らないことがある。
    * **解決策**: `exit /b %MY_ERROR%` のように、明示的に変数を指定して終了します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **エラーコードによる情報漏洩**: 
    * 独自に設計されたアプリケーションやスクリプトが、詳細すぎるエラーコードを `exit` で返している場合、攻撃者がその数値を分析してシステムの内部状態（「ユーザーは存在するがパスワードが違う」など）を推測される可能性があります。

### LOLBAS における利用例

* **機能**: なし (内部制御コマンドのため、単体での悪用は稀)
* **特性**: プロセスのライフサイクル管理に使用される基本機能です。

## 注意点・補足

* **exit /b vs goto :eof**: 両者ともバッチを終了させますが、`exit /b [code]` は明示的な終了コードを返せるのに対し、`goto :eof` は直前のコマンドの終了コードが維持されます。
* **対話型シェルでの挙動**: 通常のコマンドプロンプトで `exit /b` を打っても何も起きません（バッチ実行中ではないため）。
