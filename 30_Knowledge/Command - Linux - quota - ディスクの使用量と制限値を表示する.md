# Command - Linux - quota - ディスクの使用量と制限値を表示する

## 概要

`quota` は、現在のユーザー（または指定したユーザー）に課せられているディスク容量の制限値と、現在の使用量を表示します。マルチユーザー環境において、特定のユーザーがストレージを独占しないように管理するために使用されます。

## 🛠 クォータの「二つの壁」

クォータを理解する上で最も重要なのが、**Soft Limit** と **Hard Limit** の違いです。

| 制限の種類 | 説明 | 挙動 |
| :--- | :--- | :--- |
| **Soft Limit** | 「警告」のライン。 | 超えても書き込みはできるが、警告が出る。猶予期間（Grace Period）が始まります。 |
| **Hard Limit** | 「絶対」のライン。 | **1バイトたりともこれを超えて書き込むことはできません。** |
| **Grace Period** | 猶予期間。 | Soft Limit を超えた状態でこの期間が過ぎると、Soft Limit が Hard Limit 扱いになります。 |

## 🛠 関連コマンドファミリー

| コマンド | 役割 |
| :--- | :--- |
| `quota` | 自分の（または特定のユーザーの）制限状況を見る。 |
| `edquota` | エディタを開いてユーザーの制限値を編集する（root用）。 |
| `repquota` | ファイルシステム全体のクォータ報告書を表示する。 |
| `quotacheck` | ファイルシステムをスキャンして現在の使用量を再計算する。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-u`, `--user` | 指定したユーザーのクォータを表示する（デフォルト）。 |
| ⭐️ `-g`, `--group` | グループ単位のクォータを表示する。 |
| ⭐️ `-v`, `--verbose` | 制限が設定されていないファイルシステムの情報も表示する。 |
| ⭐️ `-s`, `--human-readable` | 容量を KB, MB, GB などの読みやすい単位で表示する。 |

## コマンド例

```bash
# 自分の現在のディスク使用量と制限を確認
quota -s

# 管理者が特定のユーザー (alice) の状況を確認
sudo quota -u alice
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 共有サーバー（ファイルサーバー、メールサーバー）の安定運用。
- **解説**: ログファイルの肥大化や、一人のユーザーによる大量のデータアップロードでディスクが 100% になるのを防ぎます。ディスクが満杯になると、システムサービスが起動できなくなる等の致命的な障害に繋がります。

### 2. ブルーチーム視点

- **タスク**: サービス拒否攻撃（DoS）の未然防止。
- **解説**: 攻撃者が一時ディレクトリ（`/tmp` 等）に膨大なファイルを生成してシステムを麻痺させようとする攻撃に対し、クォータを設定しておくことで被害を最小限に食い止めます。また、`repquota` を定期的にチェックし、急激に使用量が増えたユーザーを「異常な動き」として検知します。

### 3. レッドチーム視点

- **タスク**: 環境の制約確認と「音を立てない」活動。
- **解説**: 侵入後、大きなツールをダウンロードしたり、大量のデータを持ち出そう（Exfiltration）としたりする前に `quota` を確認します。もし Hard Limit に近い状態であれば、書き込みエラーによる「異常終了ログ」を残して管理者に気づかれるのを避けるため、活動を慎重にする必要があります。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **クォータ未設定の罠**: クォータが設定されていないシステムでは、一般ユーザーが root の領域すら圧迫できる場合があり、システム全体のハングアップを招く脆弱性となり得ます。

### 対応策・緩和策 (ブルーチーム視点)

- **`/home` と `/var` の分離**: クォータの設定と併せて、パーティション自体を物理的に分けておくことで、ユーザーデータの増大がシステムログの記録（`/var/log`）を妨げないように設計します。
