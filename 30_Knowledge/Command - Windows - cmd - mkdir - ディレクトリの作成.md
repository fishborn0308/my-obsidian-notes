# Command - Windows - cmd - mkdir - ディレクトリの作成

## 概要

`mkdir` (または `md`) コマンドは、新しいディレクトリ（フォルダ）を作成するために使用されます。

Windowsのコマンドプロンプトにおける `mkdir` は、指定されたパスの中間に存在しないディレクトリがある場合、それらも自動的に作成する特性（Linuxの `mkdir -p` 相当の挙動）を持っています。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `md` | `mkdir` の短縮形。機能的な違いはない。 | `md` (入力短縮) |
| `New-Item` | PowerShellのコマンドレット。ディレクトリ以外（ファイル、シンボリックリンク等）も作成可能。 | `New-Item` (PS環境) |

## よく連携されるコマンド

`mkdir` は、環境構築やデータ整理の初期段階で他のコマンドと組み合わせて使用されます。

### シナリオ例: 作業環境の初期化と移動 (インフラ構築)

* **目的**: 新しいプロジェクト用のディレクトリを作成し、即座にその中へ移動する。
* **連携コマンド**: `cd`
* **解説**: ディレクトリ作成後、`&&`（正常終了時のみ実行）を使用してカレントディレクトリを移動します。
* **コマンド例**:
    ```cmd
    REM ディレクトリを作成して移動
    mkdir C:\Labs\Project01 && cd C:\Labs\Project01
    ```

## スイッチ/オプション説明

Windowsの `mkdir` コマンドには、コマンドヘルプ（`mkdir /?`）上で明示されている機能スイッチは存在しませんが、パスの指定方法により挙動が変わります。

| 表記 | 説明 |
| :--- | :--- |
| **パス指定** | |
| `mkdir <Name>` | カレントディレクトリに指定した名前のフォルダを作成する。 |
| `mkdir <Path\To\Dir>` | **中間ディレクトリの自動作成**: `Path` や `To` が存在しない場合、それらも含めて一括作成する。 |
| `"双引用符"` | スペースを含むディレクトリ名を作成する場合は、パスを `"` で囲む必要がある。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 階層構造を持つログ保存先の一括作成
* **組み合わせ**: 直接パス指定
* **解説**: Windowsの `mkdir` は、Linuxの `-p` オプションを付けた状態がデフォルト挙動であるため、深い階層も一行で作成可能です。
* **例**:
    ```cmd
    REM 年/月/日の階層を一度に作成
    mkdir C:\Logs\App01\2026\02\22
    ```

### 2. ブルーチーム視点

* **タスク**: 調査用の一時フォルダ作成とアクセス権設定の準備
* **組み合わせ**: `icacls`
* **解説**: 証拠保全（Forensics）などの際、分析用ディレクトリを作成し、即座に権限を制限するシナリオ。
* **例**:
    ```cmd
    REM 分析用フォルダを作成し、特定のユーザー以外アクセス禁止にする準備
    mkdir C:\Evidence\Case001
    icacls C:\Evidence\Case001 /inheritance:r /grant:r Administrators:(OI)(CI)F
    ```

### 3. レッドチーム視点

* **タスク**: 隠蔽性の高い場所へのステージングディレクトリ作成
* **組み合わせ**: `attrib`
* **解説**: 侵入後、ツールを配置するための場所を作成し、隠し属性を付与して目立たなくさせる。
* **例**:
    ```cmd
    REM システムのタスクフォルダ配下に偽装用ディレクトリを作成し、隠しファイル属性を付与
    mkdir C:\Windows\Tasks\.internal_data
    attrib +h +s C:\Windows\Tasks\.internal_data
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `サブディレクトリまたはファイル ... は既に存在します。`
    * **考えられる原因**: 作成しようとしたディレクトリ名と同じ名前のディレクトリ、またはファイルが既にその場所に存在する。
    * **解決策**: 既存のファイル/フォルダを確認するか、別の名前を指定する。

2.  **エラーメッセージ例 2**: `アクセスが拒否されました。`
    * **考えられる原因**: 実行ユーザーに、指定した親ディレクトリ内での書き込み権限がない。
    * **解決策**: コマンドプロンプトを「管理者として実行」するか、権限のあるディレクトリ（`%TEMP%` など）を使用する。

## 環境変数と設定ファイル

### 影響を与える環境変数

* **`%CD%`**: `mkdir` で相対パスを指定した場合、この変数（現在のディレクトリ）を基点に作成されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `mkdir` 自体に直接的なリモートコード実行などの脆弱性は稀ですが、ディレクトリトラバーサル攻撃と組み合わされることがあります。
* **悪用シナリオ**: Webアプリケーションの脆弱性を突き、`mkdir` を介して公開ディレクトリ内に書き込み可能なディレクトリを作成し、そこにWebシェルを配置する足場として利用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `mkdir` は直接的なLOLBAS（ファイルのダウンロードや実行）には分類されませんが、**攻撃の準備フェーズ（Staging）**において必須のステップとなります。
* **参照**: [LOLBAS Project (Not explicitly listed as primary binary)](https://lolbas-project.github.io/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * 重要なシステムディレクトリ（`C:\Windows`, `C:\Program Files` 等）の書き込み権限を制限する。
    * `AppLocker` や `Windows Defender Application Control (WDAC)` でスクリプト実行を制限する。
* **Detection (検知)**:
    * **イベントID 4656 / 4663**: ファイルシステムオブジェクトへのアクセス・作成を監視する。
    * 不審な場所（例: `C:\Windows\Tasks` や `C:\Users\Public`）での予期せぬディレクトリ作成をEDR/SIEMでアラート化する。

## 注意点・補足

* **PowerShellとの比較**: PowerShellの `mkdir` は実際には `New-Item -ItemType Directory` のラッパー（関数）であり、オブジェクトを返します。対してCMDの `mkdir` は単なるコマンド実行であり、戻り値はテキスト出力のみです。
* **パスの最大長**: WindowsのAPI制約により、デフォルトでは約260文字（MAX_PATH）を超える深さのディレクトリ作成には注意が必要です。
