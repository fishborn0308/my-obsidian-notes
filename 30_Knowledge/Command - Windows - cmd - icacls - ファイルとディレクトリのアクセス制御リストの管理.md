# Command - Windows - cmd - icacls - ファイルとディレクトリのアクセス制御リストの管理

## 概要

`icacls` (Integrity Control Access Control List) は、ファイルやディレクトリのアクセス制御リスト (ACL) を表示、変更、バックアップ、および復元するための強力なツールです。

NTFS ファイルシステム上の詳細なアクセス許可（読み取り、書き込み、フルコントロールなど）や、継承の挙動、所有者の変更などをコマンドラインから一括で制御できます。

(出自: `Windows 標準搭載 - 外部コマンド (icacls.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `cacls` | `icacls` の旧バージョン。現在は非推奨。 | `icacls` を使用 |
| `takeown` | ファイルの「所有権」を取得することに特化したツール。 | 所有権取得 |
| `Get-Acl` / `Set-Acl` | PowerShell 版。オブジェクト指向で、複雑な権限の複製などに強い。 | PS 環境 |
| `attrib` | 読み取り専用や隠し属性など、基本的な「ファイル属性」を操作する。 | 簡易属性操作 |

## よく連携されるコマンド

### シナリオ例: 書き込み可能なシステムディレクトリの探索 (レッドチーム視点)

* **目的**: 一般ユーザーに書き込み権限が付与されている不適切なディレクトリを探し、DLL 読み込み順序の乗っ取り (DLL Hijacking) などの足がかりにする。
* **連携コマンド**: `icacls`, `findstr`
* **解説**: 特定のユーザーグループ（Users や Authenticated Users）に対して (W) 権限や (M) 権限があるパスを抽出します。
* **コマンド例**:
    ```cmd
    REM C:\Program Files 配下で、Users グループに書き込み権限がある箇所を探す
    icacls "C:\Program Files\*" /t /c /l | findstr "(I)(M)"
    ```

## 主要なスイッチと権限表記

### 実行スイッチ

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/grant` | 指定したユーザーに権限を付与する（既存の権限は維持）。 |
| ⭐ `/deny` | 指定したユーザーの権限を明示的に拒否する。 |
| ⭐ `/remove` | 指定したユーザーの権限エントリを ACL から削除する。 |
| `/t` | 現在のディレクトリとそのすべてのサブディレクトリに対して再帰的に実行。 |
| `/c` | エラーが発生しても処理を継続する。 |
| `/inheritance:e|d|r` | 継承を有効(e)、無効(d)、または削除(r)する。 |

### 権限の略称 (Permissions)

| 略称 | 意味 |
| :--- | :--- |
| **F** | フル コントロール (Full Access) |
| **M** | 変更 (Modify Access) |
| **RX** | 読み取りと実行 (Read and Execute Access) |
| **R** | 読み取りのみ (Read-only Access) |
| **W** | 書き込みのみ (Write-only Access) |
| **(I)** | 権限が親ディレクトリから**継承**されていることを示す。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しい共有フォルダにおいて、特定のグループに「読み取り/実行」権限だけを与え、かつサブフォルダにもその設定を継承させる。
* **例**:
    ```cmd
    REM Engineers グループに読み取り/実行権限を付与（継承あり）
    icacls "D:\SharedData" /grant Engineers:(OI)(CI)RX
    ```

### 2. ブルーチーム視点 (セキュリティ監査)

* **タスク**: 機密情報が格納されたフォルダ（例: `C:\Confidential`）に、管理者以外の不要なユーザーがアクセスできるようになっていないか確認する。
* **例**:
    ```cmd
    REM フォルダの権限一覧を表示
    icacls "C:\Confidential"
    ```

### 3. レッドチーム視点 (権限昇格と隠蔽)

* **タスク 1: Misconfiguration Hunting (設定不備の探索 - T1083)**
    * **目的**: サービス実行ファイルのパスに対して、一般ユーザーが変更権限 (M) を持っていないか確認する。もし持っていれば、バイナリを入れ替えることでシステム権限を奪取できます。
* **タスク 2: Persistence (永続化のための権限改ざん)**
    * **目的**: 設置した Web シェルやツールを管理者から削除されないよう、自分以外のすべてのユーザー（SYSTEM を含む）に対して「拒否」を設定する。
    * **コマンド例**:
        ```cmd
        REM Everyone からのすべてのアクセスを拒否する（悪用厳禁）
        icacls "C:\temp\backdoor.exe" /deny Everyone:F
        ```

## エラーメッセージとトラブルシューティング

1.  **アクセスが拒否されました (Access is denied)**
    * **原因**: 自身の権限が不足している、またはファイルの所有者が自分ではない。
    * **解決策**: まず `takeown /f [ファイル名]` で所有権を取得し、その後 `icacls` で権限を自分に付与します。

2.  **パラメーターが間違っています**
    * **原因**: ユーザー名や権限の記述ミス（コロン `:` の位置など）。
    * **解決策**: `icacls user:F` のように、ユーザー名と権限をコロンで繋いでいるか確認してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Weak Permissions (弱い権限設定)**:
    * `C:\` 直下やサードパーティ製アプリのインストールフォルダに Users:F が付与されていると、攻撃者は容易にシステムの一部を書き換え、権限昇格を達成します。
* **DLL Side-Loading**:
    * 権限設定が緩いディレクトリに悪意ある DLL を置くことで、正規の実行ファイルにその DLL を読み込ませる攻撃の温床となります。

### 対応策・緩和策 (ブルーチーム視点)

* **最小権限の原則 (PoLP)**: ユーザーには業務に必要な最低限の権限（例: R だけ）を付与し、書き込み権限 (W, M, F) は厳格に管理する。
* **継承の管理**: 意図しない権限が親から引き継がれていないか定期的に監査する。

## 注意点・補足

* **拒否の優先順位**: Windows では「許可」よりも「拒否」が優先されます。あるユーザーに F が許可されていても、同時に F が拒否されていればアクセスはできません。
* **順序の重要性**: ACL にはエントリの順序があり、通常は「明示的な拒否」→「明示的な許可」→「継承された拒否」→「継承された許可」の順で評価されます。
