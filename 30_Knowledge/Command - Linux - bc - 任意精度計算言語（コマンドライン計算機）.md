# Command - Linux - bc - 任意精度計算言語（コマンドライン計算機）

## 概要

`bc` は、対話的な計算だけでなく、シェルスクリプト内での数値計算をサポートするコマンドライン・インタプリタです。Bash 標準の算術展開 `$(( ))` では扱えない**小数の計算**や、高度な数学関数を使用できるのが最大の特徴です。

(出自: `bc` パッケージに含まれる。POSIX 準拠)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `$(( expression ))` | シェル標準の算術演算。整数のみ。高速。 | 単純な整数の加減乗除 |
| `expr` | 古い形式。整数のみ。現在は `$(( ))` が推奨される。 | 互換性維持のみ |
| `python3 -c` | 非常に多機能。ただし、起動が重く、Python未導入環境では使えない。 | 複雑な計算・解析 |
| `bc` | 小数点以下を扱える。進数変換が容易。標準で入っていることが多い。 | スクリプト内での小数・進数変換 |

## よく連携されるコマンド

### シナリオ例: パイプを利用したワンライナー計算 (運用視点)

* **目的**: 対話モードに入らずに、他のコマンドの出力を即座に計算する。
* **連携コマンド**: `echo`, `printf`
* **解説**: 標準入力から計算式を渡すことで、即座に結果を標準出力に返します。
* **コマンド例**:
    ```bash
    # 小数点第2位まで計算
    echo "scale=2; 10 / 3" | bc
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **動作設定** | |
| ⭐️ `-l`, `--mathlib` | 標準数学ライブラリをロードする。デフォルトで `scale`（小数点以下桁数）が 20 になり、sin/cos/log 等が使用可能。 |
| ⭐️ `-q`, `--quiet` | 起動時の著作権表示等のバナーを非表示にする。 |
| `-s`, `--standard` | POSIX bc 拡張機能に対する警告を有効にする。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ディスクやメモリの使用率を小数点付きのパーセンテージで算出する。
* **組み合わせ**: `df` や `free` コマンドとの連携。
* **解説**: 整数の切り捨てを防ぐため、`scale` で精度を指定して計算します。
* **例**:
    ```bash
    # メモリ使用率を計算
    free | grep Mem | awk '{print "scale=2; "$3"/"$2"*100"}' | bc
    ```

### 2. ブルーチーム視点

* **タスク**: ログに含まれる 16 進数のオフセットや UID の 10 進数変換。
* **組み合わせ**: `ibase` (入力進数), `obase` (出力進数)
* **解説**: `auditd` やバイナリログに含まれる値を解析する際、進数を切り替えて計算します。
* **例**:
    ```bash
    # 16進数(input)を10進数(output)へ変換 (16進数は大文字で入力)
    echo "ibase=16; DEADBEEF" | bc
    ```

### 3. レッドチーム視点

* **タスク**: バッファオーバーフロー攻撃時のオフセット計算やシェルコードのサイズ計算。
* **組み合わせ**: `scale=0` (整数演算)
* **解説**: アドレス計算など、スクリプト内で正確な 16 進数演算が必要な際に使用します。
* **例**:
    ```bash
    # 2つの16進アドレスの差分を10進数で得る
    echo "ibase=16; FFFFFFFF - FFFF0000" | bc
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `(standard_in) 1: syntax error`
    * **考えられる原因**: スペースの欠如や、16進数変換時の小文字使用 (`ibase=16` 時は A-F は大文字必須)。
    * **解決策**: 式をダブルクォートで囲み、16進数は大文字で記述する。

2.  **結果が 0 になる**: `1/3` が `0` になる
    * **考えられる原因**: デフォルトの `scale` が 0（整数のみ）に設定されている。
    * **解決策**: `scale=2;` のように明示的に小数点以下の桁数を指定するか、`-l` オプションを付ける。

## 環境変数と設定ファイル

### 影響を与える環境変数

* **`BC_ENV_VAR`**: 起動時に読み込むファイルを指定できる。
* **`BC_LINE_LENGTH`**: 出力が長い場合の折り返し文字数を指定。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: それ自体にバグは少ないが、**ファイルの読み書き**機能（特定のバージョンや実装による）を悪用される可能性がある。

### GTFOBins における利用例

`bc` はファイル読み込みや実行を直接行う機能は標準ではありませんが、一部の実装や `sudo` の設定次第では、設定ファイルを読み込ませることで機密情報を漏洩させる可能性があります。

* **機能**: `File Read`
* **コマンド例**:
    ```bash
    # sudo bc でファイルを読み込ませる（エラーメッセージ経由で中身を確認）
    sudo bc -q /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/bc/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 特権環境で `bc` を実行させる必要はほぼ無いため、`/etc/sudoers` での許可を避ける。
* **Detection (検知)**: 異常に長い計算式や、`/etc/shadow` 等を引数に取る `bc` の実行を監視する。

## 注意点・補足

* **進数変換の罠**: `obase` を先に指定すると、その後の `ibase` の数値も `obase` の進数で解釈されます。
    * ❌ `obase=10; ibase=16` (10進数の16として解釈される)
    * ⭕️ `ibase=16; obase=10` (16進数モードにしてから、10進数出力を指定)
* **数学ライブラリ**: `$ bc -l` で起動すると、$e$（指数関数）や $s(x)$（サイン関数）などの数学関数が利用可能です。
