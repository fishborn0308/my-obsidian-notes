# Command - Linux - ldd - 共有ライブラリの依存関係を表示する

## 概要

`ldd` は、実行ファイルや共有ライブラリが必要とする共有オブジェクト（Shared Objects, .so）の一覧を表示します。プログラムが起動できない原因（ライブラリ不足）の特定や、どのバージョンのライブラリがリンクされているかの確認に非常に有用です。

## 類似・関連コマンド

| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `nm` | オブジェクトファイル内のシンボル（関数名や変数名）をリストアップする。 | 関数レベルの調査 |
| `readelf` | ELF 形式（Linuxの標準バイナリ形式）の詳細なヘッダー情報を表示。 | **信頼できないバイナリの安全な調査** |
| `objdump` | バイナリの逆アセンブルやセクション情報の表示。 | 低レイヤーの解析 |
| `strace` | 実行中にどのライブラリを「実際に」探そうとしたかを追跡する。 | 動的なトラブル解決 |

## よく使われるオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `(引数なし)` | 基本的な依存関係と、解決されたメモリパスを表示する。 |
| ⭐️ `-v`, `--verbose` | 各ライブラリのバージョン情報など、より詳細な情報を表示する。 |
| ⭐️ `-u`, `--unused` | リンクはされているが、実際には使用されていないライブラリを表示する。 |
| `-d`, `--data-relocs` | データリロケーションを実行し、不足しているオブジェクトを報告する。 |
| `-r`, `--function-relocs` | データと関数の両方のリロケーションを実行し、不足を報告する。 |

## 出力項目の読み解き

```text
linux-vdso.so.1 (0x00007ffca35f7000)
libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f8e8c400000)
/lib64/ld-linux-x86-64.so.2 (0x00007f8e8c7fc000)
````

- **`libc.so.6`**: プログラムが要求しているライブラリ名。
- **`=> /lib/...`**: 実際にシステムで見つかったライブラリの絶対パス。
- **`(0x0000...)`**: そのライブラリがロードされる予定の仮想メモリ上のアドレス。
- **`not found`**: ⭐️ **重要**。ライブラリが見つからず、このままでは実行できない状態。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 「error while loading shared libraries」の解決。
- **解説**: 他の環境から持ってきたバイナリが動かない際、`ldd` で `not found` になっているライブラリを特定し、不足しているパッケージをインストールするか `LD_LIBRARY_PATH` を設定します。

### 2. ブルーチーム視点

- **タスク**: ライブラリ乗っ取り（Library Hijacking）の検知。
- **解説**: `LD_PRELOAD` 環境変数などを使って、悪意あるライブラリが優先的に読み込まれるように細工されていないか、意図しないパスのライブラリが参照されていないかを確認します。

### 3. レッドチーム視点

- **タスク**: 権限昇格の糸口探し（Shared Object Injection）。
- **解説**: `SUID` 権限がついたバイナリが `ldd` で `not found` になっているライブラリを探している場合、そこに自作の悪意あるライブラリを置くことで、root 権限を奪取できる可能性があります。

## ⚠️ セキュリティ上の重大な注意点

### 脆弱性とリスク

- **任意コード実行**: ⭐️ **超重要**。一部の OS や古いバージョンの `ldd` は、情報を得るために **対象のプログラムを実際に（一部）実行** します。そのため、**「信頼できない出所不明のバイナリ」に対して `ldd` を実行すると、その瞬間にマルウェアに感染する恐れがあります。**

### 対応策・緩和策 (ブルーチーム視点)

- **readelf の使用**: 信頼できないバイナリを調べる際は、実行を伴わない `readelf -d <file>` を使用して、安全に依存関係（NEEDED セクション）を確認してください。

## 注意点・補足

- **静的リンクバイナリ**: 全ての機能を自前で持っている「静的リンク（Static Link）」されたバイナリに対して実行すると、`not a dynamic executable` と表示されます。
