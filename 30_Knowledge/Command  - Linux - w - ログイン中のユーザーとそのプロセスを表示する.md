# Command  - Linux - w - ログイン中のユーザーとそのプロセスを表示する

## 概要

`w` は、現在システムにログインしているユーザーの一覧と、各ユーザーが**何を実行しているか**を詳細に表示するコマンドです。出力の1行目には `uptime` コマンドと全く同じ情報（現在時刻、稼働時間、ユーザー数、ロードアベレージ）が表示され、2行目以降に各ユーザーの情報が表示されます。

「誰がどこから接続して、今何をしているのか？」をリアルタイムで把握するための、基本的なシステム監視・調査ツールです。

(出自: `procps-ng` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`who`** | 現在ログインしているユーザーの一覧を、よりシンプルな形式で表示します。**ユーザーが何を実行しているか**は表示しません。 | `w` (詳細情報) |
| **`users`** | 現在ログインしているユーザーの**名前**だけを一行で表示します。 | - |
| **`uptime`**| `w` コマンドの**ヘッダー部分だけ**を表示します。ユーザーの詳細は表示しません。 | - |

## よく連携されるコマンド

`w` は単体で完結する情報表示コマンドですが、その出力に含まれるユーザー名やIPアドレスを元に、`grep` や `journalctl` でさらに詳細なログを調査する、という流れで使われます。

### シナリオ例: 不審なユーザーの活動を調査する (ブルーチーム視点)

* **目的**: `w` コマンドで不審なログインセッションを発見した後、そのユーザーの行動履歴を追跡する。
* **連携コマンド**: `grep`, `history`
* **解説**: `w` で不審なユーザー (`attacker`) とその接続元IPを特定します。次に、そのユーザーのコマンド履歴 (`.bash_history`) を確認し、どのような操作を行ったかを調査します。
* **コマンド例**:

		```bash
		# 1. 現在のログイン状況を確認

		w

		# 2. 不審なユーザー 'attacker' の活動を確認した場合、そのユーザーのコマンド履歴を調査

		sudo cat /home/attacker/.bash_history

		```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示制御** | |
| ⭐ `-h`, `--no-header` | ヘッダー情報（`uptime`相当の部分）を表示しません。 |
| ⭐ `-s`, `--short` | 短い形式で表示します。`LOGIN@`, `JCPU`, `PCPU` のカラムが省略されます。 |
| `-f`, `--from` | `FROM` (接続元) カラムの表示/非表示を切り替えます。 |
| **フィルタリング** | |
| `<user>` | 指定したユーザーの情報のみを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 重要な作業の前に、他にログインしている管理者がいないか確認する。
* **組み合わせ**: `w`
* **解説**: システムの再起動や、重要なサービスの再起動を行う前に `w` を実行します。もし他の管理者がログインして作業中 (`WHAT` カラムに `vi` や `docker` などが表示されている) であれば、声をかけてから作業を開始することで、意図しない作業の衝突を防ぎます。
* **例**:

		```bash
		# 現在ログインしている全ユーザーとその活動を確認

		w

		```

## 2. ブルーチーム視点

* **タスク**: ライブレスポンスで、攻撃者がログインしていないか、何をしているかを確認する。
* **組み合わせ**: `w`
* **解説**: 不正アクセスの疑いがあるサーバーで `w` を実行し、**`USER`** (不審なユーザー)、**`FROM`** (不審なIP)、**`WHAT`** (不審なコマンド) の3点を確認します。これらは攻撃者がアクティブに活動していることを示す強力な証拠となります。
* **例**:

		```bash
		# ライブ調査で現在の全セッションを監査

		w

		```

## 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、他にどのようなユーザーがログインしているか把握する。
* **組み合わせ**: `w`
* **解説**: 攻撃者は侵入後、`w` を実行して、他にどのようなユーザー（特に管理者）がログインしているか、また彼らがどのような作業をしているかを把握します。もし管理者がアクティブであれば、自身の活動をより隠密に行うか、管理者がログアウトするのを待ちます。
* **例**:

		```bash
		# システムの活動状況を偵察

		w

		```

## 環境変数と設定ファイル

* `w` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/var/run/utmp`**: `w` が情報を読み取る、現在のログインセッションを記録したバイナリファイル。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `w` 自体ではなく、その出力が**深刻な情報漏洩**に繋がります。
* **悪用シナリオ**: `w` コマンドの出力は、**誰が、どこから、何をしているか**というシステムの活動状況をリアルタイムで明らかにします。これは、攻撃者が次の行動を計画するための非常に価値の高い偵察情報です。コマンドライン引数にパスワードなどが含まれていた場合、それも `WHAT` カラムに表示されてしまう可能性があります。

### GTFOBins / LOLBAS における利用例

`w` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードなどの機密情報をコマンドライン引数で渡さない。
* **Detection (検知)**: `/var/run/utmp` ファイルは、root権限を持つ攻撃者によって改ざん（utmp spoofing）され、`w` の結果から自身の存在を隠蔽される可能性があります。`auditd` で `w` コマンドの実行自体を監視することも有効です。

## 注意点・補足

* **アイドル時間の意味**: `IDLE` カラムの時間が長いユーザーは、ターミナルを開いたまま放置している可能性があります。セキュリティポリシーによっては、一定時間アイドル状態のセッションを自動的に切断する設定が推奨されます。
