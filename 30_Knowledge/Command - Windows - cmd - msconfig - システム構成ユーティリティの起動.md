# Command - Windows - cmd - msconfig - システム構成ユーティリティの起動

## 概要

`msconfig` は、Windowsの起動時の問題（スタートアップ エラーなど）を特定して解決するためのシステム構成ユーティリティです。
スタートアップ プログラムの有効化/無効化、ブート オプションの変更、サービスの管理などを一元的に行えます。
(出自: Windows標準搭載)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `bcdedit` | ブート構成データ (BCD) を**コマンドラインから**詳細に編集する。 | `bcdedit` (CLI操作) |
| `services.msc` | サービスの管理に特化した管理コンソール。詳細な設定が可能。 | `services.msc` |
| `taskmgr` | Windows 10以降、スタートアップ項目の管理はタスクマネージャーに移行している。 | `taskmgr` (起動項目) |

## よく連携されるコマンド

`msconfig` はGUIツールであるため、他のCLIコマンドで情報を得た後に、具体的な修正を行うために呼び出されることが多いです。

### シナリオ例: セーフモードでの再起動準備 (トラブルシューティング)

* **目的**: ドライバー競合やマルウェア感染が疑われる際に、最小構成で起動する設定を行う。
* **連携コマンド**: `bcdedit` (確認用)
* **解説**: `msconfig` を起動して「ブート」タブから「セーフ ブート」にチェックを入れます。
* **コマンド例**:
    ```cmd
    REM システム構成を起動してGUIで設定を行う
    msconfig
    ```

## スイッチ/オプション説明

`msconfig` は主に引数なしで実行されますが、以下の「タブ」構成を理解することが実質的なオプション理解となります。

| タブ名称 | 説明 |
| :--- | :--- |
| **全般** | 通常スタートアップ、診断スタートアップ（最小構成）、選択スタートアップの切り替え。 |
| **ブート** | セーフモードの指定、タイムアウト設定、OSの起動オプションの変更。 |
| **サービス** | Microsoft以外のサービスを隠して、不審なサードパーティ製サービスを特定・停止できる。 |
| **スタートアップ** | Windows 10以降はタスクマネージャーへのリンクが表示される。 |
| **ツール** | イベントビューアーやレジストリエディターなどの関連ツールを素早く起動できる。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: クリーンブートによるアプリケーション競合の特定
* **解説**: サードパーティ製の全サービスを停止し、スタートアッププログラムも無効にした状態で再起動し、問題がOS標準機能に起因するかを確認します。
* **例**: 「全般」タブで「選択スタートアップ」を選び、「システム サービスを読み込む」以外のチェックを外す。

### 2. ブルーチーム視点

* **タスク**: サービスを利用した永続化（Persistence）の調査
* **解説**: 「サービス」タブで「Microsoft のサービスをすべて隠す」にチェックを入れ、ベンダー名が不明、または不審な実行パスを持つサービスがないかを目視で確認します。
* **例**: `msconfig` > 「サービス」タブ > 「Microsoft のサービスをすべて隠す」を有効化。

### 3. レッドチーム視点

* **タスク**: 防御機能の無効化（Evasion）と環境調査
* **解説**: RDPなどでGUI操作が可能な場合、セーフモードでの起動を仕込んだり、セキュリティ製品のサービスをスタートアップから除外して無効化を試みます（※管理者権限が必要）。
* **例**: RDP経由で `msconfig` を実行し、ウイルス対策ソフトのサービスを「無効」に設定して再起動を待つ。

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `システム構成で、後で復元するために元の構成を保存できません。`
    * **考えられる原因**: ユーザーに管理者権限がない、またはレジストリへの書き込み制限がある。
    * **解決策**: 「管理者として実行」したコマンドプロンプトから `msconfig` を起動する。

## 環境変数と設定ファイル

`msconfig` で設定した内容は、主に以下のレジストリキーに反映されます。
* **ブート設定**: `HKEY_LOCAL_MACHINE\BCD00000000`
* **サービス**: `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services`

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    * **Persistence**: 攻撃者が自身の実行ファイルをサービスとして登録した後、`msconfig` の設定を改ざんして、再起動後も常に実行されるように仕向ける。
    * **Anti-Forensics**: セーフモードで起動させることで、通常モードで動作している監視ツールやEDRのプロセスが立ち上がらない状態にし、その隙にログ消去などを行う。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 
    * UAC（ユーザーアカウント制御）を最高レベルに設定し、`msconfig` のようなシステム設定変更時に必ず警告・認証を求めるようにする。
* **Detection (検知)**: 
    * **イベントID 4697**: サービスがインストールされたログを監視。
    * BCD（ブート構成）の変更を監視し、予期せぬセーフモード設定が行われていないかチェックする。

## 注意点・補足

* **設定の永続性**: `msconfig` で行った変更は、再度 `msconfig` で戻さない限り有効なままです。トラブルシューティング後に「通常スタートアップ」に戻すのを忘れると、一部の機能が動かないままになるリスクがあります。
* **Windows Serverでの挙動**: サーバーOSでも利用可能ですが、リモートデスクトップ経由での変更はネットワークサービスを止めてしまうリスクがあるため、慎重な操作が求められます。
