# Command  - Linux - groupdel - 既存のユーザーグループを削除する

## 概要

`groupdel` (group delete) は、システムから既存のユーザーグループを削除するためのコマンドです。プロジェクトが終了して不要になったグループや、誤って作成したグループを整理する際に使用します。このコマンドは、`/etc/group` や `/etc/gshadow` といった関連ファイルから、指定されたグループのエントリを削除します。

**注**: あるユーザーの**プライマリグループ**になっているグループは、そのままでは削除できません。

(出自: `shadow-utils` または `passwd` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`groupadd`** | 新しいグループを**作成**します。 | - |
| **`groupmod`** | 既存のグループの名前やGIDを**変更**します。 | - |
| **`userdel`** | ユーザーを**削除**します。ユーザーを削除しても、そのユーザーが所属していたグループは削除されません。 | - |

## よく連携されるコマンド

`groupdel` は、グループを削除する前に、そのグループに所属するユーザーやファイルがもう存在しないことを確認するために `getent` や `find` と連携して使われます。

### シナリオ例: グループを安全に削除する前の確認作業 (インフラ構築・運用視点)

* **目的**: グループを削除する前に、そのグループをプライマリグループとして使用しているユーザーがいないか、また、そのグループが所有するファイルが存在しないかを確認する。
* **連携コマンド**: `getent`, `find`
* **解説**: 意図しない権限の問題を防ぐため、グループ削除前には十分な確認が必要です。`getent group` でグループの存在とメンバーを確認し、`find` で関連ファイルがないかを調査します。
* **コマンド例**:

```bash
# 1. 削除したいグループ 'old-project' のメンバーを確認
getent group old-project

# 2. 'old-project' グループが所有するファイルがないかシステム全体を検索
sudo find / -group old-project -ls 2>/dev/null

# 3. (問題がなければ) グループを削除
sudo groupdel old-project
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `<group_name>` | 削除するグループの名前を指定します。 |
| **その他** | |
| `-f`, `--force` | エラーが発生した場合でも、強制的にコマンドを終了します。プライマリグループの削除など、根本的な問題を解決するわけではないため、通常の使用は推奨されません。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: プロジェクト終了に伴い、不要になったグループをシステムから削除する。
* **組み合わせ**: `sudo groupdel <group_name>`
* **解説**: 関連するユーザーの所属グループを変更した後、このコマンドで不要になったグループを削除し、システムを整理整頓します。
* **例**:

```bash
# 不要になった 'web-staging' グループを削除
sudo groupdel web-staging
```

### 2. ブルーチーム視点

* **タスク**: 攻撃者が作成した不正なグループを削除する。
* **組み合わせ**: `sudo groupdel <malicious_group>`
* **解説**: インシデント対応の修復 (Remediation) フェーズで使用します。調査の結果、攻撃者が永続化のために作成したとみられる不審なグループを発見した場合、関連するファイルやプロセスの無力化が完了した後、このコマンドで悪意のあるグループを削除し、システムを正常な状態に戻します。
* **例**:

```bash
# 攻撃者が作成した 'adm1n_grp' を削除
sudo groupdel adm1n_grp
```

### 3. レッドチーム視点

* **タスク**: **破壊活動 (Sabotage)** の一環として、正規のグループを削除する。
* **組み合わせ**: `sudo groupdel <critical_group>`
* **解説**: 攻撃者がroot権限を奪取した後、システムの正常なアクセス制御を妨害するために、正規のアプリケーションが必要とするグループ（例: `docker`, `www-data`）を意図的に削除します。これにより、サービスが起動しなくなったり、ファイルアクセスに失敗したりする可能性があります。

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `groupdel: cannot remove the primary group of user '<username>'`
		* **考えられる原因**: 削除しようとしているグループが、少なくとも一人のユーザーのプライマリグループとして設定されています。
		* **解決策**:
				1. `usermod -g <new_primary_group> <username>` を実行して、そのユーザーのプライマリグループを別の既存のグループに変更します。
				2. プライマリグループとして使用しているユーザーがいなくなったら、再度 `groupdel` を実行します。

## 環境変数と設定ファイル

* `groupdel` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/group`**, **`/etc/gshadow`**: `groupdel` はこれらのファイルからグループのエントリを削除します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `groupdel` 自体ではなく、**意図しないグループの削除**がサービス停止やセキュリティリスクに繋がります。
* **悪用シナリオ**: 攻撃者が、システムの正常運用に必要なグループを削除することで、サービス妨害を引き起こします。

### GTFOBins / LOLBAS における利用例

`groupdel` はOSの構成を変更するコマンドであり、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則を適用し、不要なユーザーに `groupdel` を実行できる `sudo` 権限を与えない。
* **Detection (検知)**: `/etc/group` ファイルへの変更をファイル整合性監視(FIM)ツールで監視する。`auditd` で `groupdel` コマンドの実行を監視する。

## 注意点・補足

* **孤立したファイル**: グループを削除しても、そのグループが所有していたファイルのグループ所有権は、削除されたグループの**GID（数値）**として残ります。`find / -nogroup` などのコマンドで該当するファイルを探し出し、`chgrp` で新しい適切なグループに所有者を変更することが推奨されます。
