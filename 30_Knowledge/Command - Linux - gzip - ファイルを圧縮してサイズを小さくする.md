# Command - Linux - gzip - ファイルを圧縮してサイズを小さくする

## 概要
`gzip` は、Lempel-Ziv (LZ77) アルゴリズム（DEFLATE）を用いてファイルを圧縮します。圧縮されたファイルには通常 `.gz` という拡張子が付き、元のファイルは（デフォルト設定では）削除されます。
(出自: Jean-loup Gailly と Mark Adler によって開発)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `gunzip` | `gzip -d` と同じ。圧縮ファイルを元に戻す。 | 展開・解凍 |
| `zgrep / zless` | 圧縮されたままのファイルに対して検索や閲覧を行う。 | **ログのクイック調査** |
| `bzip2` | `gzip` より圧縮率が高いが、処理が遅い。`.bz2`。 | 高圧縮重視 |
| `xz` | 現代最強クラスの圧縮率。さらに遅いが非常に小さくなる。 | 配布用アーカイブ |
| `zip` | 複数ファイルを一つにまとめる（アーカイブ）機能も持つ。 | Windowsとのやり取り |

## よく連携されるコマンド
### シナリオ例: 巨大なテキストログのバックアップ (運用視点)
* **目的**: 数GBある `access.log` を圧縮して、別サーバーに転送しやすくする。
* **解説**: 標準出力をリダイレクトすることで、元のファイルを残したまま圧縮版を作成することも可能です。
* **コマンド例**:
    ```bash
    # access.log を圧縮し、元のファイルは削除する (デフォルト)
    gzip access.log
    # 元のファイルを残したまま、最高圧縮率で圧縮ファイルを作成
    gzip -9 -c access.log > access.log.gz
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-c`, `--stdout` | 結果を標準出力に書き出す。元のファイルは変更されない。 |
| ⭐️ `-d`, `--decompress` | 圧縮ファイルを元に戻す（`gunzip` と同等）。 |
| ⭐️ `-r`, `--recursive` | ディレクトリ内の全てのファイルを再帰的に圧縮する。 |
| ⭐️ `-1` 〜 `-9` | 圧縮率を指定。`-1` は最速（低圧縮）、`-9` は最高圧縮（低速）。デフォルトは `-6`。 |
| ⭐️ `-l`, `--list` | 圧縮ファイルの中身（圧縮前のサイズ等）を表示する。 |
| `-k`, `--keep` | 圧縮後も元のファイルを削除せずに残す（新しいバージョンのみ）。 |
| `-v`, `--verbose` | 圧縮率などの統計情報を表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: ログローテーションの自動化。
* **解説**: `logrotate` 設定内で `compress` オプションを指定すると、内部的に `gzip` が呼ばれ、古いログが自動で `.gz` 化されます。

### 2. ブルーチーム視点
* **タスク**: 証拠保全（フォレンジック）データの保護。
* **解説**: 収集したメモリダンプやディスクイメージを圧縮して、ハッシュ値とともに保存します。
* **例**:
    ```bash
    # メモリダンプを圧縮して保存
    gzip -c mem_dump.raw > mem_dump.raw.gz
    ```

### 3. レッドチーム視点
* **タスク**: データの持ち出し（Exfiltration）の効率化。
* **解説**: 大量の機密文書を盗み出す際、そのまま送るとネットワーク監視（IDS/IPS）に検知されやすく、時間もかかります。`gzip` で固めることで、通信量を減らし、中身を直接見られないようにします（パスワード保護はないため、`zip` や `7z` と併用されることが多い）。

## エラーメッセージとトラブルシューティング

1.  **「gzip: filename already has .gz suffix -- unchanged」**:
    * **原因**: 既に圧縮済みのファイルを重ねて圧縮しようとした。
    * **解決策**: 二重圧縮は効率が悪いためスキップされます。必要なら拡張子を変えるか、別のツールを検討してください。

2.  **「gzip: stdout: No space left on device」**:
    * **原因**: 圧縮プロセス中にディスク容量がいっぱいになった。
    * **解決策**: 不要なファイルを削除するか、別のパーティションにリダイレクトして保存してください。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `gzip` はデータを圧縮しますが、**暗号化はしません**。`.gz` ファイルを盗まれれば、誰でも `gunzip` で中身を読めます。機密情報には必ず暗号化ツール（`gpg` 等）を併用してください。
* **メタデータの保持**: 圧縮ファイルには元のファイル名やタイムスタンプが含まれることがあります。これが調査のヒント（いつ作成されたファイルか等）になることがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **Integrity**: 圧縮ファイルが改ざんされていないか、`sha256sum` 等でハッシュ値を計算・比較する習慣をつけます。
* **Detection**: 短時間に大量のファイルが `gzip` されている場合、ランサムウェアによる暗号化の前段階（データの持ち出し準備）である可能性を考慮します。

## 注意点・補足
* **`tar` との親友関係**: `gzip` は単一のファイルしか圧縮できません。フォルダごと固めたい場合は、まず `tar` で一つにまとめてから `gzip` する（`.tar.gz`）のが標準的な作法です。
    * 便利なショートカット: `tar -zcvf archive.tar.gz folder/`
