# Command - Windows - cmd - getmac - MAC アドレスの取得

## 概要

`getmac` は、コンピューター上のすべてのネットワーク アダプターのメディア アクセス制御 (MAC) アドレスと、各アドレスに関連付けられているネットワーク プロトコルの一覧を返します。

`ipconfig /all` よりも MAC アドレスの抽出に特化しており、ローカルだけでなくリモート システムの情報も取得可能です。

(出自: `Windows 標準搭載 - 外部コマンド (getmac.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ipconfig /all` | IP アドレス、DNS、ゲートウェイを含む統合情報。 | ネットワーク全般 |
| `wmic nic get ...` | WMI を経由してアダプターの詳細（ステータス等）を取得。 | 高度な列挙 |
| `arp -a` | 通信相手（他端末）の MAC アドレスを確認する。 | 近隣端末調査 |
| `Get-NetAdapter` | PowerShell 版。インターフェース速度や状態も詳しく表示。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: 資産管理用 CSV の作成 (運用)

* **目的**: ネットワーク内のコンピューター名と MAC アドレスを紐づけたリストを自動生成する。
* **連携コマンド**: `hostname`, `getmac`
* **解説**: `/fo csv` スイッチで出力形式を整形し、リダイレクトで保存します。
* **コマンド例**:
    ```cmd
    REM ホスト名と MAC アドレスを 1 行にまとめて出力（簡易版）
    echo %COMPUTERNAME% > mac_list.txt & getmac /fo csv /nh >> mac_list.txt
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/v` | **詳細表示**。接続名やアダプターの種類（物理、仮想）を表示する。 |
| ⭐ `/fo {TABLE|LIST|CSV}` | 出力形式を指定。CSV はスクリプトでの再利用に最適。 |
| `/nh` | 出力時にカラムヘッダー（見出し）を非表示にする。 |
| `/s [Computer]` | **リモート システム**を指定して実行。 |
| `/u [Domain\User]` | リモート実行時に使用するユーザー資格情報を指定。 |
| `/p [Password]` | 指定したユーザーのパスワード。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: DHCP サーバーの予約リストを作成するために、対象サーバーの NIC の物理アドレスを正確に把握する。
* **例**:
    ```cmd
    REM ネットワーク接続名を含めて詳細に表示
    getmac /v /fo list
    ```

### 2. ブルーチーム視点 (不正接続調査)

* **タスク**: ネットワーク監視ログに現れた不審な MAC アドレスが、社内のどの端末に該当するかを特定する。
* **ポイント**: 取得した MAC アドレスの上位 3 バイト（OUI）を照合し、メーカー（Dell, HP, Apple 等）を確認することで、デバイスの種類を絞り込みます。

### 3. レッドチーム視点 (Anti-VM / サンドボックス検知)

* **タスク: Virtual Environment Detection (仮想環境の特定)**
    * **目的**: 自分のペイロードが「解析用サンドボックス」や「仮想マシン」の上で動いているかを判定し、解析を妨害または終了する。
    * **解説**: MAC アドレスの特定のプレフィックス（例: VMware なら `00-05-69`、VirtualBox なら `08-00-27`）が含まれているかを確認します。
    * **コマンド例**:
        ```cmd
        REM VMware 関連の MAC アドレスがないか確認
        getmac /v | findstr /i "00-05-69 00-0C-29 00-50-56"
        ```

## エラーメッセージとトラブルシューティング

1.  **RPC サーバーを利用できません (The RPC server is unavailable)**
    * **原因**: リモート実行 (`/s`) 時に、ターゲット側で RPC サービスが停止している、またはファイアウォールでブロックされている。
    * **解決策**: ポート 135 (RPC) および WMI 関連の通信を許可する。

2.  **情報が表示されない (No adapters found)**
    * **原因**: ネットワーク アダプターがすべて無効化されている、またはドライバが正常に動作していない。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **MAC スプーフィング (MAC Spoofing)**:
    * 攻撃者は `getmac` で正規のアドレスを調査した後、自身の端末の MAC アドレスを偽装（スプーフィング）することで、MAC アドレス制限のある Wi-Fi やポートセキュリティをバイパスしようとします。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * 短時間に大量のリモート端末に対して `getmac` を実行している不審な管理者権限プロセス（横展開の偵察）を監視。
* **Prevention (予防)**:
    * MAC アドレス制限だけに頼らず、802.1X 認証などのより強固なネットワーク認証を導入する。

## 注意点・補足

* **VPN アダプター**: 物理的な NIC 以外にも、VPN クライアントが作成する仮想アダプターの MAC アドレスも表示されます。
* **非対話実行**: バッチファイルでリストを作成する際は、`/nh` を付けてヘッダーを消すと、後の `findstr` や `awk` 等での処理が楽になります。
