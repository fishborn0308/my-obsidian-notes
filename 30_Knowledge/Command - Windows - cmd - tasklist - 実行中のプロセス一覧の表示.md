# Command - Windows - cmd - tasklist - 実行中のプロセス一覧の表示

## 概要

`tasklist` コマンドは、ローカルまたはリモート コンピューター上で実行されているアプリケーションとサービスの一覧を表示します。プロセスID (PID) やメモリ使用量を確認できるほか、フィルタリング機能を使用して特定の条件に合致するプロセスを抽出することが可能です。
(出自: Windows標準搭載)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `taskmgr` | GUIツール（タスク マネージャー）。リアルタイムなリソース監視に適している。 | `taskmgr` |
| `Get-Process` | PowerShellコマンドレット。プロセスをオブジェクトとして取得でき、後続の処理が容易。 | `Get-Process` (PS) |
| `wmic process list` | プロセスの実行パスやコマンドライン引数まで詳細に取得できるが、廃止予定。 | `Get-CimInstance` (PS) |

## よく連携されるコマンド

### シナリオ例: 特定のサービスをホストしているプロセスの特定 (調査)

* **目的**: 複数の `svchost.exe` のうち、特定のサービス（例: Windows Update）を動かしているプロセスを特定する。
* **連携コマンド**: `findstr`
* **解説**: `/svc` スイッチでプロセスとサービスの対応を表示し、`findstr` で絞り込みます。
* **コマンド例**:
    ```cmd
    REM "wuauserv" (Windows Update) を含むプロセスを表示
    tasklist /svc | findstr "wuauserv"
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **表示内容の拡張** | |
| ⭐ `/v` | 実行中のプロセスの「詳細情報」を表示。ステータスやユーザー名が含まれる。 |
| ⭐ `/svc` | 各プロセスでホストされているサービスを表示。 |
| `/m [Module]` | 指定したDLLモジュールをロードしているプロセスを一覧表示。 |
| **出力フォーマット** | |
| `/fo <Format>` | `TABLE` (表), `LIST` (リスト), `CSV` (カンマ区切り) から選択。 |
| **フィルタリング** | |
| `/fi "<Filter>"` | 特定の条件（例: `MEMUSAGE gt 50000`）でプロセスを絞り込む。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: メモリを大量消費しているプロセスの特定
* **組み合わせ**: `/fi`
* **解説**: システムのパフォーマンス低下時に、50MB以上のメモリを使用しているプロセスをリストアップします。
* **例**:
    ```cmd
    REM メモリ使用量が 50,000 KB を超えるプロセスを表示
    tasklist /fi "MEMUSAGE gt 50000"
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なプロセスの所有者確認
* **組み合わせ**: `/v`
* **解説**: 実行されているプロセスの実行ユーザーを確認し、一般ユーザー権限で動くべきでないプロセス（例: `lsass.exe` が一般ユーザーで動いている等）がないか調査します。
* **例**:
    ```cmd
    REM 詳細情報を表示し、プロセス所有者を確認
    tasklist /v
    ```

### 3. レッドチーム視点

* **タスク**: セキュリティソフト (AV/EDR) の列挙とプロセスインジェクションの対象探し
* **組み合わせ**: `/m`, `/fi`
* **解説**: 
    1. 導入されているセキュリティ製品のプロセス名（例: `MsMpEng.exe` 等）を探す。
    2. インジェクション対象として安定したプロセス（例: `explorer.exe`, `notepad.exe`）の PID を特定する。
* **例**:
    ```cmd
    REM 特定のDLL（例: wininet.dll）をロードしているプロセスを探す
    tasklist /m wininet.dll
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `エラー: プロセスが見つかりません。`
    * **考えられる原因**: フィルタ条件に合致するプロセスが存在しない、または指定したPID/名前が間違っている。
    * **解決策**: 条件を緩めて再試行する。

2.  **エラーメッセージ例 2**: `エラー: アクセスが拒否されました。` (リモート実行時)
    * **考えられる原因**: リモートコンピューターの管理共有やRPCへのアクセス権限がない。
    * **解決策**: 適切な管理者権限を持つユーザーで `/u` スイッチを使用する。

## 環境変数と設定ファイル

`tasklist` 自体に影響を与える固有の変数はありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    * **Enumeration**: 攻撃者が侵入後、EDRやアンチウイルスが動作しているか、サンドボックス環境でないか（仮想化関連のプロセスがないか）を確認するために多用されます。
    * **Process Injection**: `tasklist` で取得した PID は、`Reflective DLL Injection` や `Process Hollowing` などの攻撃におけるターゲット指定に利用されます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Information Requirements`
* **解説**: 攻撃の偵察フェーズにおいて、システム環境を把握するための標準的な手段として利用されます。
* **参照**: [LOLBAS - tasklist.exe](https://lolbas-project.github.io/lolbas/Binaries/Tasklist/)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**: 
    * 短時間に `tasklist /svc` や `tasklist /v` が繰り返し実行される挙動を監視する。
    * プロセス名やPIDを列挙した直後に、メモリへの不審な書き込み（インジェクションの兆候）がないか相関分析を行う。

## 注意点・補足

* **情報の鮮度**: `tasklist` は実行した瞬間のスナップショットです。リアルタイムで消え去るプロセスを捕まえるには、ループさせるか、イベントログ（プロセス作成イベント 4688）を確認する必要があります。
* **コマンドライン引数の欠如**: `tasklist` では、プロセスが起動された際の引数（例: `powershell.exe -enc ...` のエンコードされた中身）は見えません。それが必要な場合は `wmic` または PowerShell の `Get-CimInstance` が必要です。
