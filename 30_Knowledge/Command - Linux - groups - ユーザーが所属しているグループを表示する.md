# Command - Linux - groups - ユーザーが所属しているグループを表示する

## 概要

`groups` は、現在のユーザー、または引数で指定されたユーザーが所属している全てのグループ名を一覧表示します。Linux のアクセス制御は「どのユーザーがどのグループに属しているか」に依存するため、権限トラブルの切り分けにおいて非常に重要な役割を果たします。

(出自: `coreutils` パッケージに含まれる)

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `id` | ユーザーID(UID)やグループID(GID)を数値と名前の両方で詳しく表示する。 | **詳細な権限確認** |
| `whoami` | 現在の有効なユーザー名のみを表示する。 | 実行ユーザーの確認 |
| `getent group` | システム上の全グループ、または特定のグループのメンバー一覧を表示する。 | グループ側からの逆引き |
| `groups` | 所属グループ名だけをスペース区切りでサッと確認する。 | **簡易的な所属確認** |

## よく連携されるコマンド

### シナリオ例: 新しいグループ権限が反映されたかの確認 (運用視点)

* **目的**: `usermod` で自分を `docker` グループに追加した後、正しく反映されているか確認する。
* **解説**: 所属グループの一覧に目的のグループ名があるかチェックします。
* **コマンド例**:
    ```bash
    # 自分の所属グループを確認
    groups
    # 結果例: user_name adm cdrom sudo dip plugdev lpadmin docker
    ```

## オプションと引数

`groups` は非常にシンプルなコマンドで、主な引数は「ユーザー名」です。

| 用法 | 説明 |
| :--- | :--- |
| ⭐️ `groups` | 現在ログインしているユーザーの所属グループを表示する。 |
| ⭐️ `groups <username>` | 指定した `<username>` の所属グループを表示する。 |
| `--help` | ヘルプを表示する。 |
| `--version` | バージョン情報を表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 権限不足エラー（Permission Denied）の調査。
* **解説**: 共有ディレクトリに書き込めないユーザーに対し、適切なグループ（例: `www-data` や `devel`）に属しているかを確認させます。
* **例**:
    ```bash
    # ユーザー 'bob' の所属を確認
    groups bob
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なユーザー権限の監査。
* **解説**: 侵入者が作成した不審なユーザーが、管理者グループ（`sudo`, `wheel`）や機密データにアクセス可能なグループに属していないかチェックします。
* **例**:
    ```bash
    # 全ユーザーの所属グループを調査し、sudo権限持ちを特定
    grep -E '^sudo|^wheel' /etc/group
    # あるいは個別に確認
    groups suspect_user
    ```

### 3. レッドチーム視点

* **タスク**: 権限昇格（Privilege Escalation）の足掛かり探し。
* **解説**: 侵入に成功した直後、自分がどのグループに属しているかを確認します。例えば `lxd`, `docker`, `disk`, `video` などのグループに属していれば、既知の手法で root 権限を奪取できる可能性が高まります。
* **例**:
    ```bash
    # 現在の権限を把握
    groups
    ```

## エラーメッセージとトラブルシューティング

1.  **「グループに追加したのに groups コマンドの結果に出てこない」**:
    * **原因**: Linux では、所属グループの変更は**次回のログイン（セッションの開始）時**に反映されます。
    * **解決策**: 一度ログアウトして再ログインするか、`newgrp <groupname>` コマンドを使用して現在のシェルで一時的に反映させます。

2.  **「groups: 'username': no such user」**:
    * **原因**: 指定したユーザー名がシステムに存在しない。
    * **解決策**: `id <username>` や `/etc/passwd` を確認する。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **情報漏洩**: 所属しているグループ名は、そのユーザーの役割（DB管理者、Web開発者など）を如実に表します。攻撃者にとって、どのユーザーをターゲットにすべきかの重要な判断材料になります。

### 対応策・緩和策 (ブルーチーム視点)

* **最小権限の原則**: ユーザーには必要なグループ権限のみを付与し、不要になったら即座に削除（`gpasswd -d` 等）する運用を徹底します。特に `sudo` や `docker` などの強力なグループへの所属は厳格に管理します。

## 注意点・補足

* **プライマリ vs セカンダリ**: 最初に表示されるグループが通常「プライマリグループ（メイン）」であり、それ以降が「セカンダリグループ（サブ）」です。
* **表示形式**: `groups` は名前だけを表示しますが、スクリプト等で GID（数値）が必要な場合は `id -G` を使用します。
