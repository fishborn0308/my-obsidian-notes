# Command - Linux - ls [Blue Team Edition] - 異常検知と堅牢化の監査術

## 概要

ブルーチームにおける `ls` は、システムの「ベースライン（正常な状態）」からの逸脱を検知するための**プライマリ・監査ツール**です。パーミッションの不備を修正し、攻撃者が設置した隠しファイルや、改ざんされたタイムスタンプの矛盾を暴きます。

---

## 1. パーミッションの「健全性」を監査する (Hardening)

### 🛡 不適切な所有権と権限の特定

攻撃者に悪用される「書き込み可能（Writable）」な設定や、意図しない SUID 設定を定期的に監査します。

* **監査ポイント**: `/etc` や `/usr/bin` などの重要ディレクトリで、一般ユーザーに書き込み権限が与えられていないか。
* **コマンド**:
    ```bash
    # /etc 配下で、所有者以外に書き込み権限(w)があるファイルをリストアップ
    ls -lR /etc | grep '^..w..w'
    ```

### 🚫 スティッキービットの確認

`/tmp` などの共有ディレクトリで、他人のファイルを勝手に消せない設定（スティッキービット）が正しく機能しているか確認します。

* **コマンド**:
    ```bash
    # パーミッションの末尾が 't' になっているか確認
    ls -ld /tmp
    # 出力例: drwxrwxrwt ... (末尾の t が重要)
    ```

---

## 2. 潜伏（Persistence）の痕跡を暴く (Forensics)

### 🕵️‍♂️ 異常なドットファイルの調査

攻撃者は `...`（ドット3つ）や `.. `（ドット2つ+スペース）といった、一見するとディレクトリに見える巧妙な名前の隠しファイルを作成します。

* **テクニック**: `ls -a` で、標準的なファイル（`.`, `..`, `.bashrc` 等）以外の「見慣れないドットファイル」がないか精査します。
* **コマンド**:
    ```bash
    # /dev/shm（メモリ上の共有領域）など、マルウェアが置かれやすい場所をスキャン
    ls -la /dev/shm
    ```

### 🕒 タイムスタンプの「三元論」による分析

`ls` は3種類の時刻を表示できます。これらを比較することで、攻撃者が `touch` で `mtime` だけを偽装した痕跡を見抜ける可能性があります。

| 時刻タイプ | オプション | 意味 | 攻撃者による偽装 |
| :--- | :--- | :--- | :--- |
| **mtime** | (標準) | 内容の最終修正時刻。 | **容易** (`touch -m`) |
| **atime** | `-u` | 最終アクセス（閲覧）時刻。 | **容易** (`touch -a`) |
| **ctime** | `-c` | 属性（権限・所有者）の最終変更時刻。 | **困難** (システム時刻変更が必要) |

* **コマンド**:
    ```bash
    # ctime（属性変更時刻）を表示。mtime と大きく乖離していれば改ざんの疑い。
    ls -lc --full-time <filename>
    ```

---

## 3. インシデント発生時の初動調査 (Incident Response)

### 🚨 大量ファイル作成・変更の特定

攻撃者が自動化ツールでファイルを展開したり、大量のデータを持ち出そうとしたりした際の痕跡を時刻順に追います。

* **コマンド**:
    ```bash
    # 直近の変更を、秒単位の精度で新しい順に表示
    ls -lat --full-time /var/www/html | head -n 20
    ```

---

## 4. 実行ログの監視 (SIEM / EDR 視点)

### 📈 異常な `ls` 実行パターンの検知

通常のユーザーやアプリケーションが実行しないような `ls` の使い方は、偵察行動としてアラートの対象にします。

* **検知ルール例 (SIEM/Auditd)**:
    * **再帰的スキャン**: `ls -R /` や `ls -R /home` の実行。
    * **隠しファイル一括表示**: 短時間での複数ディレクトリに対する `ls -la`。
    * **権限チェック**: `ls -l /etc/shadow` などの重要ファイルへのアクセス試行。

## 💡 ブルーチームの防衛コンボ

| 目的 | コマンド例 | 防衛効果 |
| :--- | :--- | :--- |
| **ベースライン比較** | `ls -laR /etc > baseline.txt` | 定期的に取得し、`diff` で予期せぬ変更（改ざん）を即座に特定する。 |
| **バイナリ保護確認** | `ls -l /bin /sbin /usr/bin` | 重要な実行ファイルに `s` (SUID) が増えていないか、サイズが変わっていないか確認。 |
| **Webシェル検知** | `ls -lt /var/www/html` | 新しく作成された PHP や ASPX ファイルがないか、更新日時でソートして監視。 |

> [!TIP]
> 堅牢化の究極の手段は、ディレクトリインデックス（ブラウザでのファイル一覧表示）を Web サーバー設定で無効にすることです。`ls` という「窓」を外部から閉ざすことが、最も効果的な防御になります。
