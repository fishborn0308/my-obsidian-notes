# Command - Windows - cmd - pause - バッチ処理の一時停止とユーザー待機

## 概要

`pause` コマンドは、バッチファイルの処理を一時的に停止し、ユーザーがキーボードのキーを押すまで待機させるために使用されます。
実行されると、標準出力に「続行するには何かキーを押してください . . .」というメッセージが表示されます。
(出自: Windows標準搭載 / MS-DOS由来)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `timeout` | 指定した秒数だけ待機する。キー入力でスキップも可能。 | 自動進行なら `timeout` |
| `choice` | 特定のキー入力のみを受け付け、結果を ERRORLEVEL に返す。 | 選択分岐なら `choice` |
| `set /p` | ユーザーに任意の文字列を入力させる。 | データ入力なら `set /p` |

## よく連携されるコマンド

### シナリオ例: 処理結果の確認 (インフラ運用)

* **目的**: バッチファイル実行後、ウィンドウがすぐに閉じてしまうのを防ぎ、出力されたログやエラーを確認する。
* **連携コマンド**: `echo`, `exit`
* **解説**: スクリプトの最後に配置することで、実行結果を目視確認するまでプロセスを維持します。
* **コマンド例**:
    ```cmd
    @echo off
    REM ネットワーク設定のバックアップ処理
    ipconfig /all > network_config.txt
    echo 設定を保存しました。内容を確認してください。
    pause
    exit
    ```

## スイッチ/オプション説明

`pause` コマンドにはスイッチやオプションはありません。

| 構文 | 説明 |
| :--- | :--- |
| `pause` | メッセージを表示してキー入力を待つ。 |
| `pause > nul` | **メッセージを表示せず**にキー入力を待つ（ステルス一時停止）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 危険な操作（削除など）の直前確認
* **解説**: ファイル削除やサービス停止など、取り消しのつかない処理の前にユーザーの「覚悟」を促します。
* **例**:
    ```cmd
    @echo off
    echo 【警告】これから一時ファイルをすべて削除します。
    echo 中断する場合は Ctrl+C を押してください。
    pause
    del /q C:\Temp\*.*
    ```

### 2. ブルーチーム視点

* **タスク**: インシデント調査スクリプトの段階的実行
* **解説**: 大量の情報を出力する調査スクリプトで、セクションごとに `pause` を挟み、アナリストが画面上の情報を読み取る時間を確保します。
* **例**:
    ```cmd
    @echo off
    echo === 実行中の不審なプロセス一覧 ===
    tasklist /v | findstr /i "unauthorized"
    pause
    echo === ネットワーク接続状況 ===
    netstat -ano
    pause
    ```

### 3. レッドチーム視点

* **タスク**: アンチ・サンドボックス (Anti-Sandbox / Human Interaction)
* **解説**: 自動解析環境（サンドボックス）は人間によるキー入力をシミュレートしない場合が多いため、`pause` を挟むことでその後の悪意あるコード（ペイロード）の実行を阻止し、解析を回避します。
* **例**:
    ```cmd
    @echo off
    rem 人間が操作している場合のみ先に進む
    pause > nul
    rem ペイロードのデコードと実行
    certutil -decode encoded.txt payload.exe
    start payload.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `pause` を入れているのにウィンドウが閉じてしまう。
    * **考えられる原因**: `pause` に到達する前の行で致命的なエラーが発生している、または `exit` コマンドが `pause` より前に実行されている。
    * **解決策**: エラーが発生しそうな箇所に `pause` を細かく挟んでデバッグする。

2.  **現象**: リダイレクトした際に止まらなくなる。
    * **原因**: `cmd < script.bat` のように入力をリダイレクトして実行している場合、`pause` は入力を待ちません。

## 環境変数と設定ファイル

`pause` に直接影響を与える環境変数はありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    - **Social Engineering**: バッチファイルを実行させたターゲットに対し、「システム修復中... 完了するには何かキーを押してください」と表示させ、その間にバックグラウンドでデータを窃取する時間を稼ぐ。
    - **Anti-Analysis**: 前述の通り、動的解析環境をストップさせるために使用。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Delay / Interaction`
* **解説**: `pause` 自体は `cmd.exe` の内部コマンドですが、攻撃フローの制御において「人間の介入を確認する」ステップとして利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**: 
    - スクリプトブロックロギング等で、`pause > nul` が含まれる不審なバッチファイルが一時フォルダ（`Temp`）から実行されていないか監視する。
    - サンドボックス環境では、`pause` を自動的にスキップする、あるいはキー入力をシミュレートする設定を検討する。

## 注意点・補足

* **メッセージのカスタマイズ**: 標準のメッセージが味気ない場合は、`echo` と `pause > nul` を組み合わせます。
    ```cmd
    echo [OK] を押して次のフェーズへ進んでください...
    pause > nul
    ```
* **Ctrl+C による中断**: `pause` で待機中に `Ctrl+C` を押すと、「バッチ ジョブを終了しますか (Y/N)?」というプロンプトが表示され、実行自体を安全に中止できます。
