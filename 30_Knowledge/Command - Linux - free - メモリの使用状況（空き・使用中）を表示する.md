# Command - Linux - free - メモリの使用状況（空き・使用中）を表示する

## 概要

`free` は、`/proc/meminfo` ファイルを読み取り、システムの物理メモリおよびスワップ領域の合計、使用済み、空き、共有、バッファ、キャッシュ、および利用可能な容量を表示します。現代の Linux において、単なる「free」列よりも「available」列が重要であることを教えてくれるツールです。

(出自: `procps` または `procps-ng` パッケージに含まれる)

## 類似・関連コマンド

| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `top / htop` | プロセスごとのメモリ消費量もリアルタイムで表示する。 | プロセス特定の調査 |
| `vmstat` | メモリだけでなく、CPU、IO、スワップの動態を時系列で表示する。 | ボトルネックの継続調査 |
| `cat /proc/meminfo` | `free` の元データ。最も詳細なメモリ情報が得られる。 | カーネルレベルの深い調査 |
| `free` | システム全体のメモリ概要を瞬時に把握する。 | **クイックチェック** |

## よく連携されるコマンド

### シナリオ例: メモリ消費の監視 (運用・自動化視点)

* **目的**: メモリの空きが少なくなった際にログに記録、または警告を出す。
* **連携**: `awk`, `watch`
* **解説**: 定期的に実行して特定の列（例: available）を抽出します。
* **コマンド例**:
    ```bash
    # 5秒ごとにメモリ状況を監視（変化をハイライト表示）
    watch -d free -h
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-h`, `--human` | 1G, 512M のように、人間が読みやすい単位（1024単位）で表示する。 |
| `-b`, `-k`, `-m`, `-g` | 単位を固定する（バイト, KB, MB, GB）。 |
| ⭐️ `-s <n>`, `--seconds <n>` | `n` 秒おきに繰り返し表示する。 |
| `-c <n>`, `--count <n>` | `n` 回表示して終了する。 |
| ⭐️ `-t`, `--total` | 物理メモリとスワップの合計行を表示する。 |
| `-w`, `--wide` | キャッシュとバッファを分けて表示する（モダンな表示形式）。 |

## 出力項目の読み方 (重要)

* **total**: 装着されている全メモリ。
* **used**: 使用中のメモリ (total - free - buff/cache)。
* **free**: **全く使われていない**メモリ。Linuxは空きメモリをキャッシュに回すため、通常はここが少なくなります。
* **buff/cache**: ディスクI/Oを高速化するために一時的に使われているメモリ。
* **available**: ⭐️ **実質的な空き容量。** アプリケーションが新しくメモリを必要としたとき、キャッシュを解放するなどして即座に割り当て可能な予測値。

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スワップ（Swap）の発生確認。
* **解説**: 物理メモリが足りなくなると低速なディスク領域（Swap）が使われ始めます。`free` で `swap` の `used` が増えている場合は、メモリ増設やチューニングが必要です。
* **例**:
    ```bash
    # 物理メモリとスワップの状況を分かりやすく表示
    free -h -t
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なプロセスのメモリ占有による DoS 兆候の検知。
* **解説**: 攻撃者がメモリリークを起こすペイロードを送り込んだり、マイニングツールを動かしたりしている場合、`available` が急激に減少します。
* **例**:
    ```bash
    # 利用可能メモリが 500MB を切っているかチェック
    free -m | awk '/^Mem:/ {if($7 < 500) print "Warning: Low Memory!"}'
    ```

### 3. レッドチーム視点

* **タスク**: ターゲット環境のスペック調査とエクスプロイトの安定性評価。
* **解説**: メモリが極端に少ない環境で巨大なバイナリやスキャナを動かすと、OOM Killer に殺されたり、システムをクラッシュさせたりして発覚のリスクが高まります。
* **例**:
    ```bash
    # 侵入後の基本偵察（メモリ容量の確認）
    free -g
    ```

## エラーメッセージとトラブルシューティング

1.  **「available 列がない」**:
    * **原因**: 古いバージョンの `free` (procps 3.3.10未満) を使用している。
    * **解決策**: `MemFree + Cached` を目安にするか、最新のパッケージに更新する。

2.  **「free が 0 に近くて焦る」**:
    * **原因**: Linux の正常な仕様。メモリを余らせておくのはもったいないので、カーネルがキャッシュとして活用しています。
    * **解決策**: `available` を見て、そこが十分であれば問題ありません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Denial of Service (DoS)**: 攻撃者が意図的にメモリを消費し切ることで、重要なサービス（SSHやDBなど）を OOM Killer に終了させ、サービス停止に追い込みます。

### GTFOBins における利用例

`free` はシステム情報を表示するだけでファイル操作やコマンド実行ができないため、GTFOBins には登録されていません。しかし、特権で実行することで通常は見えない共有メモリの情報を得られる可能性があります。

## 注意点・補足

* **単位の違い**: `-h` は 1024 単位 (GiB)、`--si` は 1000 単位 (GB) です。商用ストレージなどの表記と合わせる場合は注意してください。
