# Command - Linux - alias - コマンドの別名（ショートカット）を作成・管理する

## 概要
`alias` コマンドは、長いコマンド文字列を短い名前（別名）に置き換えるためのシェル内蔵コマンド（builtin）です。
(出自: `Bash / Zsh 等のシェル内蔵機能`)

## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `function` | 複雑なロジックや引数の処理が必要な場合は関数を使用する。aliasは単純な置換に限定される。 | 引数を使うなら `function` |
| `symlink (ln -s)` | 実行ファイル自体に別名をつけ、パスを通す。システム全体やスクリプトからも参照可能。 | 永続的なバイナリ別名は `ln` |
| `scripts` | 処理が長大になる場合は独立したスクリプトファイルにする。 | 汎用性は `script` |

## よく連携されるコマンド
`alias` 自体は単体で定義に使われますが、設定を反映させるために以下のコマンドと併用されます。

### シナリオ例: 設定の即時反映 (共通)
* **目的**: `.bashrc` 等に記述した alias 設定を、ログアウトせずに現在のセッションに反映させる。
* **連携コマンド**: `source` または `.`
* **解説**: 設定ファイルを再読み込みすることで、定義した alias を有効化します。
* **コマンド例**:
    ```bash
    # 設定ファイルを編集後、反映
    source ~/.bashrc
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **表示・設定** | |
| ⭐️ `alias` (引数なし) | 現在定義されているすべての alias を一覧表示する。 |
| `alias name='command'` | 新しい alias を定義する。 |
| **解除** | |
| `unalias name` | 指定した alias を削除する。 |
| `unalias -a` | すべての alias を現在のセッションから削除する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 誤操作によるファイル消失の防止と、ログ確認の効率化。
* **組み合わせ**: `alias rm='rm -i'`, `alias la='ls -A'`
* **解説**: 破壊的なコマンドに確認オプションを付加したり、頻用するディレクトリ移動を短縮します。
* **例**:
    ```bash
    # 安全策としてのエイリアス
    alias rm='rm -i'
    alias cp='cp -i'
    alias mv='mv -i'

    # ログ確認の短縮
    alias journal-err='journalctl -p 3 -xb'
    ```

### 2. ブルーチーム視点
* **タスク**: 調査時における環境の「汚染」確認と、不審なエイリアスの検知。
* **組み合わせ**: `alias`
* **解説**: 攻撃者が既存コマンド（`ls`, `ps`, `netstat` 等）を alias で上書きし、自身のプロセスやファイルを隠蔽していないか確認します。
* **例**:
    ```bash
    # 現在のセッションで上書きされているコマンドがないか目視確認
    alias
    
    # alias を無視してオリジナルのコマンドを実行（エスケープ）
    \ls
    command ls
    /bin/ls
    ```

### 3. レッドチーム視点
* **タスク**: 永続化（Persistence）および認証情報の窃取。
* **組み合わせ**: `alias sudo='...'`
* **解説**: `sudo` を alias で偽装し、ユーザーが入力したパスワードを外部へ送信したりファイルに記録したりする「Sudo Password Sniffing」に悪用されます。
* **例**:
    ```bash
    # パスワードを盗む alias の例（概念）
    alias sudo='read -sp "[sudo] password for $USER: " pass; echo "$USER : $pass" >> /tmp/.pass; echo; sleep 1; echo "Sorry, try again."; /usr/bin/sudo'
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `-bash: alias: name: not found`
    * **考えられる原因**: `alias` コマンドの引数に `=` が含まれていない、またはスペースが入っている。
    * **解決策**: `alias name='command'` のように、イコールの前後にスペースを入れずに記述する。

2.  **エラーメッセージ例 2**: `alias を設定したのに、スクリプト内で動かない`
    * **考えられる原因**: デフォルトでは、非対話型シェル（スクリプト実行時）では alias は展開されない。
    * **解決策**: スクリプト内で `shopt -s expand_aliases` を設定するか、`function` を使用する。

## 環境変数と設定ファイル

### 影響を与える環境変数
* **`PATH`**: alias は PATH にあるコマンドより優先して評価されます。

### 関連する設定ファイル
* **`~/.bashrc` / `~/.zshrc`**: ユーザー個別の alias 定義場所。
* **`/etc/bashrc`**: システム全体の alias 定義場所。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: alias 自体にバイナリ脆弱性はありませんが、**ポストエクスプロイト（権限昇格後の維持）**において極めて強力な武器になります。
* **悪用シナリオ**: 
    * `ls` を alias し、特定のバックドアファイルを表示させないようにする。
    * `ssh` を alias し、接続先 IP アドレスを密かに中間者攻撃（MITM）サーバーへ誘導する。

### GTFOBins / LOLBAS における利用例
`alias` はシェル組み込みのため直接的なバイナリ登録はありませんが、**Persistence** の文脈で頻繁に言及されます。
* **参照**: `https://gtfobins.github.io/` (シェル機能全般の悪用を確認してください)

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: `~/.bashrc` などのドットファイルの書き込み権限を厳格に管理する。
* **Detection (検知)**: 
    * ログインスクリプトのハッシュ値監視（EDRや整合性監視ツール）。
    * 調査時には `command <cmd>` や絶対パスを使用して、alias をバイパスして実行する。

## 注意点・補足
* **展開の順序**: シェルは、1. 別名(alias)、2. キーワード、3. 関数、4. 組み込みコマンド、5. 外部コマンド(PATH) の順で解釈します。
* **パフォーマンス**: 大量の alias を定義しても、現代のシステムではパフォーマンスへの影響は無視できるレベルです。
