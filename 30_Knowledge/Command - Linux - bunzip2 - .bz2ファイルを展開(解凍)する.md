# `bunzip2` - .bz2ファイルを展開（解凍）する

## 概要

`bunzip2` は、`bzip2` アルゴリズムで圧縮されたファイル（通常は `.bz2` の拡張子を持つ）を展開（解凍）するためのコマンドです。`bzip2` は `gzip` よりも高い圧縮率を特徴とし、大きなファイルの配布やバックアップによく利用されます。`bunzip2` コマンドは、実際には `bzip2 -d` コマンドへのシンボリックリンクまたはエイリアスであることがほとんどで、機能は完全に同じです。

デフォルトでは、展開に成功すると元の圧縮ファイル (`.bz2`) は削除されます。

(出自: `bzip2` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`gunzip`** | `gzip` で圧縮されたファイル (`.gz`) を展開します。圧縮アルゴリズムが異なり、互換性はありません。 | - |
| **`unzip`** | 複数のファイルをまとめるアーカイバ機能も持つ `.zip` 形式のアーカイブを展開します。 | - |
| **`tar`** | `.tar.bz2` (または `.tbz2`) 形式のアーカイブを展開するには `tar xjf <file>` を使います。`tar` はアーカイブの展開と解凍を一度に行えるため、tarアーカイブに対しては `tar` の使用が推奨されます。 | `tar` |

## よく連携されるコマンド

`bunzip2` は、展開した結果を他のコマンドに渡して処理するために、パイプと `-c` オプションを組み合わせて使われることがよくあります。

### シナリオ例: 圧縮ログをディスクに書き出さずに調査する (ブルーチーム視点)

* **目的**: 圧縮された巨大なログファイルをディスクに展開することなく、中身から特定のキーワードを検索する。
* **連携コマンド**: `grep`
* **解説**: `-c` (`--stdout`) オプションで展開結果を標準出力に送り、それをパイプで `grep` に渡します。これにより、ディスクI/Oを減らして調査を高速化し、元ファイルも変更しないため証拠保全の観点からも安全です。
* **コマンド例**:

```bash
# 圧縮された認証ログの中から、特定のIPアドレスからのログイン失敗記録を調査

bunzip2 -c /mnt/evidence/auth.log.bz2 | grep "Failed password for root from 198.51.100.10"

```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `-k`, `--keep` | 展開後も、元の圧縮ファイル (`.bz2`) を**削除せずに残します (keep)**。 |
| ⭐ `-c`, `--stdout` | 展開結果をファイルに書き出す代わりに、**標準出力 (stdout)** に出力します。 |
| **その他** | |
| `-v`, `--verbose` | 実行中の詳細な情報を表示します。 |
| `-f`, `--force` | 同名の出力ファイルがすでに存在する場合でも、警告なしで**強制的に上書き**します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 圧縮保管されている過去のログファイルを展開して内容を確認する。
* **組み合わせ**: `bunzip2 <filename>.bz2`
* **解説**: ダウンロードしたソフトウェアのソースコードや、`logrotate` で圧縮されたデータベースのダンプファイルなどを展開する最も基本的な使い方です。
* **例**:

```bash
# 過去のアクセスログを展開する

bunzip2 /var/log/httpd/access_log-20250914.bz2

# -> access_log-20250914 が生成され、元の .bz2 ファイルは削除される
```

### 2. ブルーチーム視点

* **タスク**: 証拠として保全した圧縮ログファイルの内容を、元ファイルを変更せずに調査する。
* **組み合わせ**: `bunzip2 -ck <evidence_file>.bz2 | less`
* **解説**: `-c` で標準出力に、`-k` で元ファイルを保持しながら展開し、その結果を `less` で閲覧します。これにより、ディスク上に展開後の一時ファイルを残さず、かつ安全に内容を確認できます。
* **例**:

```bash
# 証拠ファイルを元ファイルのまま、内容をページャで確認

bunzip2 -ck /mnt/evidence/huge_log.bz2 | less

```

### 3. レッドチーム視点

* **タスク**: 侵入したホスト上で、持ち込んだ圧縮済みのツールキットを展開する。
* **組み合わせ**: `bunzip2 <tool_archive>.bz2`
* **解説**: 攻撃者は、ネットワークの検知を回避したり、転送量を減らしたりするためにツールを圧縮しておくことがあります。ホスト上で `bunzip2` を使ってこれらのツールを展開し、攻撃の準備を整えます。
* **例**:

```bash
# 持ち込んだワードリストを展開する

bunzip2 wordlist.txt.bz2

```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `bunzip2: <filename>: not a bzip2 file.`
		* **考えられる原因**: 指定したファイルが `bzip2` 形式で圧縮されていません。ファイルが破損しているか、`gzip` (`.gz`) など別の形式のファイルである可能性があります。
		* **解決策**: `file <filename>` コマンドを実行して、ファイルの形式を正しく確認してください。

## 環境変数と設定ファイル

* `bunzip2` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **伸張爆弾 (Decompression Bomb)**。非常に高い圧縮率を利用し、展開すると極端に巨大なファイル（数GB〜数TB）になるように細工された悪意のある圧縮ファイルです。
* **悪用シナリオ**: 伸張爆弾をサーバ上で展開させてしまうと、ディスクスペースを使い果たし、**サービス妨害 (DoS) 攻撃**を引き起こす可能性があります。

### GTFOBins / LOLBAS における利用例

`bunzip2` はファイルの展開に特化しており、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 信頼できないソースから入手した圧縮ファイルは、隔離されたサンドボックス環境で安全に展開・調査する。
* **Detection (検知)**: ファイルアップロード機能を持つアプリケーションでは、アップロードされた圧縮ファイルのサイズや内容を検証する仕組みを導入する。

## 注意点・補足

* **`.tar.bz2` ファイルの展開**: `bunzip2` はあくまで圧縮を解くだけのコマンドです。複数のファイルが `tar` でまとめられた後に `bzip2` で圧縮されたアーカイブ (`.tar.bz2`, `.tbz2`) の場合は、`tar xjf archive.tar.bz2` のように `tar` コマンドを使うのが一般的です。
