# Command  - Linux - awk - 高機能なテキスト処理・パターン抽出言語

## 概要

`awk` は、入力されたテキスト(ファイルや標準入力)を一行ずつスキャンし、各行をさらに区切り文字(フィールド)で分割し、指定したパターンに一致する行に対してさまざまな処理(アクション)を実行する非常に強力なプログラミング言語です。単なるテキストフィルタリングに留まらず、フィールドの抽出・並べ替え、数値計算、条件分岐、書式付き出力など、高度なデータ整形やレポート作成が可能です。

(出自: ほとんどのLinuxディストリビューションに標準でインストールされている `gawk` (GNU Awk) として提供される)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`grep`** | `grep` は一致する行全体を出力することに特化しています。`awk` は行をフィールドに分割し、特定のフィールドを抽出・加工できます。 | `grep` (単純な検索) |
| **`sed`** | `sed` は行単位での置換や削除といった編集作業を得意とします。`awk` は列（フィールド）単位でのデータ処理能力に長けています。 | `sed` (行単位の置換) |
| **`cut`** | `cut` は単純にフィールドを切り出すことに特化しています。`awk` は条件に応じて出力するフィールドを変えるなど、より複雑な列操作が可能です。 | `cut` (単純な列抽出) |

## よく連携されるコマンド

`awk` は他のコマンドの出力を整形・分析するパイプラインで使われることが非常に多いです。

### シナリオ例: ウェブログから特定ステータスコードのアクセス元IPを抽出・集計 (ブルーチーム視点)

* **目的**: Apacheのアクセスログから、ステータスコードが404 (Not Found) だったリクエストのアクセス元IPアドレスを抽出し、出現回数順にソートしてトップ10を表示する。
* **連携コマンド**: `sort`, `uniq`, `head`
* **解説**: `awk` を使ってログの1番目のフィールド(IP)と9番目のフィールド(ステータスコード)を評価し、404の場合にIPを出力します。その結果を `sort` でまとめ、`uniq -c` で出現回数を数え、最後に再度 `sort` と `head` でトップ10を整形します。
* **コマンド例**:

```bash
# $9がステータスコード, $1がIPアドレス (ログ形式による)

awk '$9 == 404 {print $1}' /var/log/apache2/access.log | sort | uniq -c | sort -nr | head -n 10

```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **主要オプション** | |
| ⭐️ `-F <区切り文字>` | 入力フィールドの区切り文字を指定します。`-F','` のように指定し、CSVファイルの処理などで多用されます。 |
| `-f <スクリプトファイル>` | `awk` のスクリプトをコマンドラインではなく、ファイルから読み込みます。 |
| `-v var=value` | `awk` スクリプトが実行される前に、`awk` 内部の変数を定義します。シェル変数との連携に不可欠です。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: `/etc/passwd` からユーザー名とUIDの一覧を抽出する。
* **組み合わせ**: `awk -F':' '{print $1, $3}' <file>`
* **解説**: `-F` オプションで区切り文字をコロン `:` に指定し、ユーザー名(1番目のフィールド)とUID(3番目のフィールド)を抽出します。
* **例**:

```bash
# ユーザー名とUIDをタブ区切りで表示

awk -F':' '{print "User: " $1 "\tUID: " $3}' /etc/passwd

```

### 2. ブルーチーム視点

* **タスク**: 特定の不審なIPアドレスからのアクセスがログファイル内に何回出現するかをカウントする。
* **組み合わせ**: `awk -v ip="..." '$1 == ip {count++} END {print count}' <logfile>`
* **解説**: `-v` オプションで調査対象のIPアドレスをシェルから受け取り、ログファイル内でそのIPアドレスが出現する回数をカウントして、最後に `END` ブロックで集計結果を出力します。
* **例**:

```bash
# access.logからIP "198.51.100.10" のアクセス回数を数える

awk -v ip="198.51.100.10" '$1 == ip {count++} END {print ip, "found", count, "times."}' access.log

```

### 3. レッドチーム視点

* **タスク**: `nmap` のスキャン結果から、特定のポートが開いているホストのIPアドレスだけをリストアップする。
* **組み合わせ**: `nmap ... | awk '/Up$/ && /open/ {print $2}'`
* **解説**: NmapのGrepable形式 (`-oG`) の出力をパイプで受け取り、`awk` でホストが "Up" であり、かつポートが "open" である行から、2番目のフィールド(IPアドレス)のみを抽出します。
* **例**:

```bash
# 192.168.1.0/24 ネットワークでポート80が開いているホストをリストアップ

nmap -p80 -oG - 192.168.1.0/24 | awk '/Up$/ && / 80\/open\// {print $2}'

```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `awk: syntax error at source line 1 context is >>> ... <<<`
		* **考えられる原因**: `awk` スクリプト内の構文エラー。`{}` や `()` の対応が取れていない、クォーテーション `'` が閉じていない、などが一般的です。
		* **解決策**: エラーメッセージが指し示している箇所を確認し、構文を修正します。

2. **現象**: **期待した出力が得られない**
		* **考えられる原因**: フィールド区切り文字の間違い (`-F`)、パターンの不一致、フィールド番号 (`$N`) の間違いなどが考えられます。
		* **解決策**: まずは `{print $0, "NF="NF}` のように行全体とフィールド数を出力させ、意図通りに行とフィールドが分割されているかを確認します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `awk` 自体に脆弱性があることは稀ですが、外部からの信頼できない入力を処理するスクリプトは、インジェクション攻撃の対象となり得ます。
* **悪用シナリオ**: 攻撃者がシステム侵入後、ログファイルからパスワードの痕跡を探したり、設定ファイルを解析して次の攻撃のヒントを得るために、`grep` や `sed` と同様に `awk` を頻繁に利用します。

### GTFOBins / LOLBAS における利用例

* **機能**: **Command Execution** (コマンド実行), **Privilege Escalation** (権限昇格)
* **解説**: `sudo` 権限で `awk` を実行できる場合、`awk` の `system()` 関数を使って任意のコマンドをroot権限で実行できます。これにより、簡単にrootシェルを奪取することが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/awk/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、ユーザーに `awk` のような高機能なコマンドの `sudo` 権限を安易に与えない。
* **Detection (検知)**: `system()` 関数の呼び出しなど、`awk` による不審なプロセス生成を監査ログやEDRで監視します。

## 注意点・補足

* **クォートの使い分け**: `awk` スクリプトは**シングルクォート (`'`)** で囲むのが基本です。これは、`$1` などの `awk` の変数がシェルによって先に展開されてしまうのを防ぐためです。
* **BEGINとEND**: `BEGIN` ブロックは入力ファイルを読み込む前に一度だけ実行され、ヘッダーの出力や変数の初期化に使います。`END` ブロックは全ての行の処理が終わった後に一度だけ実行され、合計値の表示や集計レポートの作成に非常に便利です。
