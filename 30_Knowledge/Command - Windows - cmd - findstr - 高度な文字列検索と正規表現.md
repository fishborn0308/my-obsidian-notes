# Command - Windows - cmd - findstr - 高度な文字列検索と正規表現

## 概要

`findstr` は、ファイルやコマンドの出力から、特定の文字列や正規表現パターンに一致する行を検索・抽出します。

単純な文字列検索を行う `find` の上位互換であり、複数のファイルに対する再帰検索や、大文字小文字を区別しない検索、正規表現によるパターンマッチングなど、実務およびペンテストの列挙（Enumeration）フェーズにおいて極めて強力な機能を提供します。

(出自: `Windows 標準搭載 - 外部コマンド (findstr.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `find` | シンプルな文字列検索のみ。正規表現不可。 | 簡易フィルタ |
| `grep` | Linux の標準。正規表現の機能がより豊富（拡張正規表現など）。 | (Linux環境) |
| `Select-String` | PowerShell 版。マッチした箇所を「オブジェクト」として扱う。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: 稼働中の特定ポートのプロセス特定 (調査)

* **目的**: ネットワーク接続一覧から、特定のポート番号（例: 4445）に関わる行だけを素早く抜き出す。
* **連携コマンド**: `netstat`, `findstr`
* **解説**: `netstat -ano` の結果から、正規表現を用いて特定のポート番号をフィルタリングします。
* **コマンド例**:
    ```cmd
    REM 4445 ポートを使用しているプロセスの行を抽出
    netstat -ano | findstr /r ":4445 "
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `/s` | 現在のディレクトリとすべてのサブディレクトリからファイルを検索する。 |
| ⭐ `/i` | 検索時に大文字と小文字を区別しない。 |
| ⭐ `/v` | 一致**しない**行のみを表示する（除外フィルタ）。 |
| ⭐ `/r` | 検索文字列を**正規表現**として扱う。 |
| `/n` | 一致した行の前に**行番号**を表示する。 |
| `/m` | 一致したファイルが含まれる**ファイル名のみ**を表示する。 |
| `/c:"string"` | スペースを含む文字列を 1 つの検索語として扱う。 |
| `/g:file` | 検索に使用するパターンが記述されたファイルを指定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 大量のログファイル群の中から、"Error" または "Critical" を含む行を再帰的に探し出し、行番号と共に表示する。
* **例**:
    ```cmd
    REM .log ファイルの中からエラーメッセージを検索
    findstr /s /i /n "error critical" C:\Logs\*.log
    ```

### 2. ブルーチーム視点

* **タスク**: ホストファイル（hosts）に、不審なドメインのリダイレクト（例: google.com を含む行）が追加されていないか確認する。
* **例**:
    ```cmd
    findstr /i "google.com" C:\Windows\System32\drivers\etc\hosts
    ```

### 3. レッドチーム視点

* **タスク**: 設定ファイルや無人インストール用ファイル（unattend.xml）の中から、"password" という単語を含む行を検索し、認証情報を奪取する。
* **例**:
    ```cmd
    REM configファイル内からパスワードの記述を列挙
    findstr /s /i /p "password pwd credential" C:\*.xml C:\*.config C:\*.txt
    ```

## エラーメッセージとトラブルシューティング

1.  **引数が多すぎます / 構文エラー**
    * **原因**: 検索文字列にスペースが含まれているが、`/c:` スイッチで囲っていない。
    * **解決策**: `findstr /c:"word A word B" filename` のように指定。

2.  **正規表現が期待通りに動かない**
    * **原因**: `findstr` の正規表現は一部のメタ文字（`+` や `?` など）をサポートしていない。
    * **解決策**: 以下の基本メタ文字の範囲で構成するか、PowerShell の `Select-String` を検討する。

## サポートされる主な正規表現

| 記号 | 意味 |
| :--- | :--- |
| `.` | 任意の 1 文字。 |
| `*` | 直前の文字の 0 回以上の繰り返し。 |
| `^` | 行の先頭。 |
| `$` | 行の末尾。 |
| `[class]` | クラス内の任意の 1 文字（例: `[0-9]`）。 |
| `[^class]` | クラスに含まれない任意の 1 文字。 |

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の探索 (T1083):** 攻撃者は `findstr` を自動化したスクリプトで、接続されている全ドライブから証明書（`.pfx`）、秘密鍵（`id_rsa`）、設定ファイル（`web.config`）などを瞬時に検索・特定します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Credentials`, `Reconnaissance`
* **特性**: Windows 標準ツールであるため、検索行動そのものを EDR が「悪意ある」と断定しにくい点に強みがあります。
* **参照**: [LOLBAS - findstr.exe](https://lolbas-project.github.io/lolbas/Binaries/Findstr/)

## 注意点・補足

* **OR 検索**: `findstr "A B" file` と記述すると、A または B を含む行がヒットします。
* **AND 検索**: `findstr "A" file | findstr "B"` とパイプで繋ぐことで、A と B 両方を含む行に絞り込めます。
