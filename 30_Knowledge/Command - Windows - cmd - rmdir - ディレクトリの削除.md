# Command - Windows - cmd - rmdir - ディレクトリの削除

## 概要

`rd` (または `rmdir`) コマンドは、空のディレクトリ、または指定したオプションによって中身を含むディレクトリツリー全体を削除するために使用されます。

(出自: Windows標準搭載 / MS-DOS由来)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `rmdir` | `rd` のフルスペル形式。機能的な違いはない。 | `rd` (入力短縮) |
| `del` (erase) | **ファイル**を削除するためのコマンド。ディレクトリ自体は削除できない。 | ファイル削除なら `del` |
| `Remove-Item` | PowerShell版。ファイルとディレクトリを区別せず、より強力なフィルタリングが可能。 | `Remove-Item` (PS) |

## よく連携されるコマンド

`rd` は、作業用の一時ディレクトリを作成して利用した後の、クリーンアップ処理で頻繁に利用されます。

### シナリオ例: 一時作業環境の完全削除 (インフラ構築)

* **目的**: スクリプトの実行中に作成した一時的なフォルダを、処理終了時に中身ごと完全に削除する。
* **連携コマンド**: `mkdir`, `cd`, `del`
* **解説**: `/s` オプションでサブディレクトリとファイルを含め、`/q` オプションで確認メッセージをスキップします。
* **コマンド例**:
    ```cmd
    REM 一時フォルダを中身ごと静かに削除
    rd /s /q C:\Temp\Work_20260222
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **削除モード** | |
| ⭐ `/s` | 指定されたディレクトリに加えて、その中のすべてのサブディレクトリとファイルを削除する（ツリー削除）。 |
| ⭐ `/q` | **Quietモード。** `/s` で削除する際に、確認のメッセージ (`よろしいですか (Y/N)?`) を表示しない。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 古いバックアップディレクトリのローテーション
* **組み合わせ**: `if exist`
* **解説**: 新しいバックアップを作成する前に、前回作成した古いディレクトリが存在すれば削除します。
* **例**:
    ```cmd
    REM 古いバックアップフォルダがあれば削除
    if exist C:\Backup\Old_Site (
        rd /s /q C:\Backup\Old_Site
    )
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアによって作成されたステージング領域の排除
* **組み合わせ**: `taskkill`
* **解説**: 不審なプロセスを停止させた後、そのプロセスが利用していた一時ディレクトリ（例: `C:\Users\Public\Temp` など）を中身ごと完全に削除して無害化します。
* **例**:
    ```cmd
    REM プロセス停止後の残存ファイルをツリーごと削除
    rd /s /q C:\ProgramData\Suspicious_App
    ```

### 3. レッドチーム視点 (Anti-Forensics)

* **タスク**: 侵入の痕跡（アーティファクト）のクリーンアップ
* **組み合わせ**: `cd ..`
* **解説**: 攻撃に使用したツールや、収集したデータの展開先フォルダを削除します。**「カレントディレクトリは削除できない」**ため、一度上の階層に移動してから実行するのが定石です。
* **例**:
    ```cmd
    REM ツールフォルダを削除して痕跡を消す
    cd C:\Windows\Temp
    rd /s /q .attacker_tools
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `ディレクトリが空ではありません。`
    * **考えられる原因**: `/s` スイッチを付けずに、中にファイルやフォルダが含まれるディレクトリを削除しようとした。
    * **解決策**: ディレクトリごと消して良い場合は `/s` オプションを追加する。

2.  **エラーメッセージ例 2**: `別のプロセスがファイルを使用しているため、プロセスはファイルにアクセスできません。`
    * **考えられる原因**: 削除対象のディレクトリ内のファイルが他のアプリで開かれている、または自分自身のプロンプトがそのディレクトリ内にいる。
    * **解決策**: `cd ..` などで移動するか、`tasklist` 等でファイルを開いているプロセスを特定し終了させる。

## 環境変数と設定ファイル

`rd` 自体に直接影響を与える環境変数はありませんが、削除対象のパス指定に `%TEMP%` などの環境変数が多用されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Anti-Forensics**: 攻撃者がフォレンジック調査を妨害するために、ログが格納されているディレクトリや、プリフェッチファイル、自身の活動痕跡を `rd /s /q` で一気に消去する。
    - **Denial of Service (DoS)**: 管理者権限を奪取した攻撃者が、アプリケーションの重要なライブラリディレクトリを削除することでサービスを停止させる。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Artifact Removal`
* **参照**: [LOLBAS - rd.exe (Standard command)](https://lolbas-project.github.io/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - 重要なシステムディレクトリへの `Delete` 権限を一般ユーザーから剥奪する。
* **Detection (検知)**:
    - **イベントID 4660**: オブジェクトが削除された。
    - **イベントID 4663**: ファイルシステムオブジェクトへのアクセス（Deleteフラグの監視）。
    - 短時間に大量のディレクトリやファイルが `rd /s /q` で削除される挙動を EDR でアラート化する。

## 注意点・補足

* **ネットワークパス**: `rd` は `\\Server\Share\Dir` のような UNC パスに対しても有効ですが、適切な権限が必要です。
* **PowerShellとの挙動差**: PowerShell の `rd` は `Remove-Item` のエイリアスであり、ディレクトリが空でない場合に `/s` を忘れても「再帰的に削除しますか？」と親切に聞いてくれますが、CMD の `rd` は即座にエラーになります。
