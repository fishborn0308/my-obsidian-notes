# Command - Linux - exit - 現在のシェルセッションやスクリプトを終了する

## 概要
`exit` は、現在実行中のシェル（Bash, Zsh, Dash等）や実行中のシェルスクリプトを終了させます。終了時に「終了ステータス（Exit Status）」という数値を親プロセスに返すことができ、これが後続処理の成否を判断する材料となります。
(出自: シェルの組み込みコマンド（Built-in）)

## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `logout` | ログインシェル（SSHログイン直後のシェル等）のみを終了させる。 | リモート接続の切断 |
| `Ctrl+D` | EOF（End of File）を送信し、対話型シェルを終了させるショートカット。 | 素早いセッション終了 |
| `break` | ループ処理（for, while等）を抜けるためのコマンド。シェル自体は終了しない。 | ループ制御 |
| `return` | 関数（function）から抜けるためのコマンド。スクリプト全体は終了しない。 | 関数内の制御 |

## よく連携されるコマンド
### シナリオ例: 前のコマンドの成否に応じた条件分岐 (運用・自動化視点)
* **目的**: バックアップが成功した時だけ、古いファイルを削除する。
* **連携**: `&&` (成功時実行) または `||` (失敗時実行)
* **解説**: 前のコマンドが `exit 0` を返すと「成功」と見なされます。
* **コマンド例**:
    ```bash
    # backup.sh が正常終了(0)した時のみ削除を実行
    ./backup.sh && rm old_data.tar.gz
    ```

## 引数と終了ステータス
`exit [n]` の形式で使用します。`n` は 0 から 255 の整数です。

| 戻り値 (n) | 意味 |
| :--- | :--- |
| ⭐️ `0` | **成功 (Success)**。正常に処理が完了したことを示す。 |
| ⭐️ `1` | **一般的なエラー (General Error)**。何らかの理由で失敗した。 |
| `2` | 誤った使用法（コマンドライン引数のミスなど）。 |
| `126` | 実行権限がないファイルを叩こうとした。 |
| `127` | コマンドが見つからない。 |
| `130` | `Ctrl+C` などによる強制中断。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: シェルスクリプト内でのエラーハンドリング。
* **解説**: 必須のディレクトリ作成に失敗した場合など、致命的なエラーが発生した際にスクリプトを中断させ、異常を知らせます。
* **例**:
    ```bash
    # ディレクトリ作成に失敗したら、エラーコード 1 で終了
    mkdir /important_dir || { echo "Failed to create dir"; exit 1; }
    ```

### 2. ブルーチーム視点
* **タスク**: セッション放置（Abandoned Terminals）による不正操作の防止。
* **解説**: 作業終了後に必ず `exit` を行う習慣（または `TMOUT` 変数による自動 exit）を徹底することで、離席中の端末を第三者に操作されるリスクを低減します。
* **設定例 (`/etc/profile` 等)**:
    ```bash
    # 600秒(10分)操作がない場合に自動的に exit する
    readonly TMOUT=600
    export TMOUT
    ```

### 3. レッドチーム視点
* **タスク**: ブラインド・ペイロード（出力が見えない状況）の実行結果確認。
* **解説**: コマンドの実行結果を画面で見られない場合、`exit` に特定の数値を指定し、それを後続の条件（例: `sleep` の実行など）に繋げることで、内部の状況を間接的に推測します。

## エラーメッセージとトラブルシューティング

1.  **「exit と打っても終わらない」**:
    * **原因**: 多重にシェルが起動している（例: `bash` の中で `bash` を起動した）。または、フォアグラウンドで別のプロセスが動いている。
    * **解決策**: `exit` を繰り返すか、`Ctrl+C` で実行中のプロセスを止めてから `exit` する。

2.  **「終了ステータスを確認したい」**:
    * **解決策**: 直前のコマンドの戻り値は環境変数 `$?` に格納されます。`echo $?` で確認できます。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **不適切な戻り値設定**: スクリプトがエラーで停止したのに `exit 0`（成功）を返してしまうと、不完全な状態のまま後続の自動化処理（デプロイ等）が進んでしまい、システムが不安定になる、あるいはセキュリティ設定が適用されないまま公開されるといったリスクが生じます。

### GTFOBins における利用例
`exit` はシェルの内部機能であるため、GTFOBins に独立したバイナリとして登録されることはありません。しかし、特権を得たシェルを閉じて痕跡を消すための「最後の仕上げ」として必ず使われます。

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: `TMOUT` を設定し、無通信セッションを自動終了させる。
* **Detection (検知)**: `/var/log/auth.log` や `last` コマンドで、不自然な時間帯のセッション終了や、長時間開きっぱなしのセッションがないか監視する。

## 注意点・補足
* **スクリプト外での実行**: 現在のターミナルで直接 `exit` を打つと、ウィンドウ（または SSH セッション）が閉じます。
* **数値の範囲**: 255 を超える数値を指定すると、256 で割った余りがステータスとして返されます（例: `exit 256` は `0` になる）。
