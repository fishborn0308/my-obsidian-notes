# Command - Windows - cmd - net session - リモートセッションの表示と管理

## 概要

`net session`（または `net sessions`）コマンドは、ローカルコンピュータとネットワーク上のクライアントとの間のセッションを表示、または切断するために使用されます。
主にファイルサーバーとして機能している端末で、現在「誰が」「どのコンピュータから」SMB 接続を行っているかをリアルタイムで確認するためのツールです。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `netstat` | L4 レベル（TCP/UDP ポート）の接続を表示。ユーザー名は不明。 | 通信全般の確認 |
| `openfiles` | セッションだけでなく、具体的に**どのファイル**を開いているかを表示。 | ファイルロック調査 |
| `Get-SmbSession` | PowerShell版。フィルタリングやオブジェクト操作に長けている。 | PS環境 |
| `fsmgmt.msc` | GUIによる「共有フォルダ」管理。視覚的にセッションを切断可能。 | - |

## よく連携されるコマンド

### シナリオ例: 不審なアクセスの緊急切断 (ブルーチーム視点)

* **目的**: 侵害が疑われる端末からのセッションを強制的に切断し、さらなる被害（データ窃取）を防ぐ。
* **連携コマンド**: `net session`, `taskkill`
* **解説**: `net session` で対象を特定し、`/delete` スイッチで接続を強制解除します。
* **コマンド例**:
    ```cmd
    REM 接続中のセッション一覧を表示
    net session

    REM 端末 "ATTACKER-PC" からのセッションをすべて強制切断
    net session \\ATTACKER-PC /delete
    ```



## スイッチ/オプション説明

**※実行には「管理者権限」が必要です。一般ユーザーでは実行できません。**

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | 現在コンピュータに接続されているすべてのセッションを表示する。 |
| `\\computername` | 指定したコンピュータからのセッション情報（ユーザー名、開いているファイル数など）を表示する。 |
| **セッション操作** | |
| ⭐ `/delete` | **【重要】** 指定したコンピュータとのセッションを切断し、開いているファイルをすべて閉じる。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーのメンテナンス（再起動など）の前に、現在誰かがファイルを編集していないか確認する。
* **例**:
    ```cmd
    REM アクティブなセッションがあるか確認。アイドル時間もチェック可能。
    net session
    ```

### 2. ブルーチーム視点 (インシデントレスポンス)

* **タスク**: ドメイン管理者のアカウントが、本来ログインすべきでないサーバーや一般端末に接続していないか監視する。
* **例**:
    ```cmd
    REM 特定の特権ユーザー（例: Administrator）がどこから繋いでいるかを目視確認
    net session | findstr /i "Administrator"
    ```

### 3. レッドチーム視点 (偵察と横展開)

* **タスク 1: High-Value Target (HVT) の所在特定**
    * **目的**: ドメイン管理者が現在どのサーバーにログイン（セッション確立）しているかを特定する。
    * **解説**: ファイルサーバーやドメインコントローラで `net session` を実行することで、管理者が現在活動している「熱い（Hot）」端末を特定でき、その端末へ向けて横展開（Lateral Movement）を行う指標になります。
    * **コマンド例**:
        ```cmd
        net session
        ```
    * **実務ログ例**:
        ```text
        コンピュータ           ユーザー名            クライアント タイプ  受付け   アイドル
        -------------------------------------------------------------------------------
        \\192.168.1.50         ADMIN_JOE             Windows              0        00:00:15
        \\192.168.1.102        USER_SARAH            Windows              12       01:30:00
        コマンドは正常に終了しました。
        REM 考察: 192.168.1.50 から管理者権限を持つユーザーが接続中。この端末を狙う価値がある。
        ```

* **タスク 2: 接続端末の列挙**
    * **目的**: 組織内の端末の IP アドレスと、それを使用しているユーザーの紐付けリストを作成する。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 管理者権限がない。
    * **解決策**: 「管理者として実行」したプロンプトで実行してください。

2.  **There are no entries in the list.**
    * **原因**: 現在、リモートからの SMB 接続が存在しない。
    * **解決策**: 正常な動作です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **偵察 (Information Disclosure - T1049):**
    * 攻撃者がサーバーの管理者権限を得た場合、`net session` を使用して、ネットワーク内の他の誰がこのサーバーを利用しているかを把握します。これにより、攻撃の影響範囲や、次なるターゲット（特権ユーザーがログインしている端末）を正確に特定されます。
* **セッション切断による妨害 (DoS - T1531):**
    * 攻撃者が嫌がらせや作業妨害のために、正規ユーザーのセッションを `/delete` で片っ端から切断する可能性があります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察)
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **最小権限**: 一般ユーザーにはサーバーのローカル管理者権限を与えない。これにより、一般ユーザーが `net session` で他者の接続状況を覗き見ることはできなくなります。
* **Detection (検知)**:
    * **不審なコマンド実行の監視**: 管理者以外（または普段実行しない管理者）による `net session` の実行を EDR/SIEM で監視する。
    * **イベントID 5140**: ネットワーク共有オブジェクトへのアクセス。これとログオンイベントを組み合わせることで、誰がどこからアクセスしたかを記録できます。

## 注意点・補足

* **アイドル時間**: 表示される「アイドル」時間は、そのセッションで最後に通信が発生してからの経過時間です。これが長いユーザーは、接続したまま放置している可能性が高いです。
* **プロトコル**: 基本的に SMB (TCP 445) 経由のセッションのみが表示されます。HTTP や RDP のセッションは表示されません（RDP は `qwinsta` コマンド等を使用します）。
