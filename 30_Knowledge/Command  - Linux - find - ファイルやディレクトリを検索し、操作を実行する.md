# Command  - Linux - find - ファイルやディレクトリを検索し、操作を実行する

## 概要

`find` は、指定したディレクトリ階層以下を探索し、様々な条件（名前、種類、更新日時、サイズ、パーミッションなど）に一致するファイルやディレクトリを見つけ出すための、非常に強力で多機能なコマンドです。単に検索するだけでなく、見つけ出した各ファイルに対して、削除、コマンド実行、情報表示などの任意の**アクション**を実行できるのが最大の特徴です。

(出自: `findutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`locate` / `mlocate`** | `locate` は事前に作成されたデータベースを検索するため**非常に高速**ですが、結果が最新でない場合があります。`find` は**リアルタイムのファイルシステム**を検索するため正確ですが、時間がかかります。 | `locate` (高速) / `find` (正確・高機能) |
| **`grep`** | `grep` は**ファイルの中身**をパターンで検索します。`find` は**ファイルの属性（メタデータ）**で検索します。 | - |
| **`which`** | 実行可能ファイルを環境変数 `PATH` の中から探します。`find` は任意の場所からあらゆる種類のファイルを検索できます。 | - |

## よく連携されるコマンド

`find` は、検索結果に対して一括で処理を行うために `xargs` や `-exec` オプションと連携することが非常に多いです。

### シナリオ例: 特定の条件に一致するファイルを一括で削除する (インフラ構築・運用視点)

* **目的**: プロジェクトディレクトリ内で、`*.tmp` という名前の一時ファイルを全て検索し、安全に削除する。
* **連携コマンド**: `find`, `xargs`, `rm`
* **解説**: `find` で条件に合うファイルを検索し、その結果を `-print0`（NULL文字区切り）で出力します。それをパイプで `xargs -0` に渡すことで、ファイル名にスペースが含まれていても安全に `rm` コマンドで削除できます。
* **コマンド例**:

```bash
# このコマンドで、カレントディレクトリ以下の .tmp ファイルを安全に一括削除する
find . -type f -name "*.tmp" -print0 | xargs -0 rm
```

## オプション説明 (`find` のテストとアクション)

| オプション | 説明 |
| :--- | :--- |
| **テスト (条件式)** | |
| ⭐ `-name "pattern"` | ファイル名が `pattern` に一致（ワイルドカード使用可）。 |
| `-iname "pattern"`| `-name` と同じだが、大文字小文字を区別しません。 |
| ⭐ `-type [f\|d\|l]` | ファイルの種類を指定 (`f`:ファイル, `d`:ディレクトリ, `l`:シンボリックリンク)。 |
| ⭐ `-mtime [-n\|n\|+n]`| **n日前**に更新されたファイル (`-n`: n日以内, `n`: ちょうどn日前, `+n`: n日より前)。 |
| `-size [+n\|-n][c\|k\|M\|G]`| **nサイズ**より大きい/小さいファイル。単位は `c`(バイト), `k`(KB), `M`(MB), `G`(GB)。 |
| `-perm /mode` | `mode` で指定した権限の**いずれか**を持つ。（例: `-perm /4000` (SUIDビットを持つ)） |
| `-user <name>` | 指定したユーザーが所有するファイル。 |
| **アクション** | |
| ⭐ `-exec cmd {} \;`| 見つかったファイルごとにコマンド `cmd` を実行します。`{}` がファイル名に置き換わります。 |
| `-exec cmd {} +` | 可能な限り多くのファイルをまとめてコマンド `cmd` に渡します。より効率的です。 |
| ⭐ `-delete`| 見つかったファイルを削除します。**非常に強力なので使用には細心の注意が必要**。 |
| `-ls` | `ls -l` に似た形式で詳細情報を出力します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 30日より前に更新された古いログファイルを定期的にクリーンアップする。
* **組み合わせ**: `find <path> -type f -name "*.log" -mtime +30 -delete`
* **解説**: `/var/log` 以下にある、30日より前に更新された `.log` ファイルを検索し、直接削除します。これを `cron` に登録して自動化するのが一般的です。
* **例**:

```bash
# 7日より古いバックアップファイル (.gz) を検索して削除
find /mnt/backups -name "*.gz" -mtime +7 -delete
```

### 2. ブルーチーム視点

* **タスク**: システムに不適切なパーミッションが設定されたファイルがないか監査する。
* **組み合わせ**: `find / -type f -perm /4000 -ls 2>/dev/null`
* **解説**: システム全体から SUID ビットが設定されたファイルを探し、詳細情報を表示します。SUIDビットが設定された実行ファイルは、権限昇格攻撃の足がかりになる可能性があるため、定期的な洗い出しが不可欠です。
* **例**:

```bash
# Webサーバーの公開ディレクトリで、過去24時間以内に変更されたファイルを探す (改ざん調査)
find /var/www/html -mtime -1 -ls
```

### 3. レッドチーム視点

* **タスク**: 侵入したシステムで、機密情報が含まれていそうな設定ファイルを探し出す。
* **組み合わせ**: `find / -type f -name "*config*" -ls 2>/dev/null`
* **解説**: システム侵入後の情報収集フェーズで、データベース接続情報やパスワードなどが平文で書かれている設定ファイルを探し出すために使用します。
* **例**:

```bash
# ホームディレクトリ以下で、SSHの秘密鍵ファイルを探す
find /home -name "id_rsa" 2>/dev/null
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `find: ‘/proc/1234/fd’: Permission denied`
		* **考えられる原因**: 検索を実行しているユーザーに、特定のディレクトリへのアクセス権限がありません。
		* **解決策**: これはエラーではなく警告に近いものです。検索結果と混ざって見にくいため、`2>/dev/null` をコマンドの末尾に付けて、標準エラー出力を破棄するのが一般的です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `find` 自体ではなく、**不適切なSUIDビットの設定**や、`-exec` オプションの不安全な使用がリスクとなります。
* **悪用シナリオ**: 攻撃者がシステム侵入後に行う**情報収集**および**権限昇格**のための最重要ツールの1つです。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もし `find` にSUIDビットが設定されているか、ユーザーが `sudo find` を実行できる権限を持っている場合、`-exec` アクションを利用して任意のコマンドを root 権限で実行できます。
* **参照**: `https://gtfobins.github.io/gtfobins/find/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、ユーザーに `find` のような強力なコマンドの `sudo` 権限を安易に与えない。
* **Detection (検知)**: `auditd` を利用し、`find` による不審な `-exec` の実行を監視する。`find / -perm /4000` でSUIDが設定されたファイルを定期的に監査する。

## 注意点・補足

* **式の組み合わせ**: `-a` (AND, デフォルト)、`-o` (OR)、`!` (NOT) といった論理演算子を `\( ... \)` でグループ化することで、複雑な検索条件を組み立てることができます。
