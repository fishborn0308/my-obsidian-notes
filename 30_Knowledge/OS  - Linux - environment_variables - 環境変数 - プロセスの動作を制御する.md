# OS  - Linux - environment_variables - 環境変数 (Environment Variables) - プロセスの動作を制御する

## 概要

環境変数とは、OS上で実行されるプロセスが共通して利用できる、**キーと値のペアで構成される変数**です。プロセスが起動する際に、親プロセス（通常はシェル）から環境変数が引き継がれ、アプリケーションはこれらの値を参照して自身の動作を動的に変更します。例えば、`PATH` 環境変数はコマンドの検索パスを定義し、`HTTP_PROXY` はプロキシサーバーのアドレスを指定します。

(出自: シェルおよびOSの基本機能)

## 類似概念との差異

| 概念 | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **シェル変数** | **現在のシェルセッション内**でのみ有効。`export` されていない変数は子プロセスに引き継がれません。 | 環境変数 (プロセス間共有) |

## よく連携されるコマンド

環境変数は、それを設定・表示・利用する多くのコマンドと連携します。

### シナリオ例: 特定のコマンドに対してのみ一時的に環境変数を設定する (インフラ構築・運用視点)

* **目的**: 普段はプロキシを使わないが、特定の `curl` コマンドだけプロキシを経由させたい。
* **連携コマンド**: `curl`
* **解説**: コマンドの直前に `<VAR>=<value>` を置くことで、そのコマンドの実行中だけ環境変数が設定されます。現在のシェル環境には影響しません。
* **コマンド例**:

```bash
# このcurlコマンドの実行中だけHTTPS_PROXYが設定される
HTTPS_PROXY=http://my-proxy:8080 curl [https://example.com](https://example.com)
```

## 主要なコマンドと環境変数

### 操作コマンド

| コマンド | 説明 |
| :--- | :--- |
| ⭐ `export VAR="value"` | シェル変数を環境変数として設定（エクスポート）します。 |
| ⭐ `env` | 現在設定されている環境変数の一覧を表示します。 |
| `printenv [VAR]`| 全て、または指定した環境変数の値を表示します。 |
| `unset VAR` | 変数 `VAR` を削除します。 |

### 主要な環境変数

| 環境変数 | 説明 |
| :--- | :--- |
| ⭐ `PATH` | コマンド実行時に、シェルが実行ファイルを探すディレクトリのリスト（コロン区切り）。 |
| `HOME` | 現在のユーザーのホームディレクトリのパス。 |
| `USER` | 現在のログインユーザー名。 |
| `SHELL` | ユーザーのデフォルトシェルのパス。 |
| `EDITOR`/`VISUAL` | `crontab -e` などが使用するデフォルトのテキストエディタ。 |
| `LD_PRELOAD`| **(セキュリティ上要注意)** プログラム実行前に、指定した共有ライブラリを強制的に読み込ませる。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Javaアプリケーションのために `JAVA_HOME` を永続的に設定する。
* **組み合わせ**: `export <VAR>="<value>"` を `~/.bashrc` に追記
* **解説**: ログインするたびに特定の環境変数が自動的に設定されるように、シェルの起動ファイルに `export` コマンドを記述します。
* **例**:

```bash
# ~/.bashrc に追記
export JAVA_HOME=/opt/jdk-17
export PATH=$PATH:$JAVA_HOME/bin

# 編集後、現在のセッションに即時反映
source ~/.bashrc
```

### 2. ブルーチーム視点

* **タスク**: 実行中の不審なプロセスの環境変数を調査する。
* **組み合わせ**: `cat /proc/<PID>/environ | tr '\0' '\n'`
* **解説**: 実行中の特定のプロセス（PIDで指定）がどのような環境変数を持って動作しているかを調査します。
* **例**:

```bash
# 不審なプロセスのPIDを特定後、その環境変数をリスト表示する
cat /proc/12345/environ | tr '\0' '\n' | less
```

### 3. レッドチーム視点

* **タスク**: 侵入したシステムで機密情報を探索する。
* **組み合わせ**: `env` または `export -p`
* **解説**: 侵入後の情報収集の一環として、現在設定されている環境変数の一覧を取得します。`DB_PASSWORD`, `API_KEY`, `AWS_SECRET_ACCESS_KEY` のような名前の環境変数に、認証情報が平文で設定されていないかを探します。
* **例**:

```bash
# 環境変数を一覧表示し、"PASS" や "KEY" といった文字列でフィルタリング
env | grep -i "PASS\|KEY\|SECRET"
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **`.bashrc` に `export` を書いたのに、スクリプト実行時に変数が認識されない。**
		* **考えられる原因**: `cron` などから実行されるスクリプトは、ログインシェルを起動しないため `.bashrc` や `.profile` を読み込みません。
		* **解決策**: スクリプト内で必要な環境変数を直接定義するか、変数を定義した別の設定ファイルを `source` コマンドで読み込むようにしてください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **シークレット（機密情報）の漏洩**。環境変数にパスワードやAPIキーなどの機密情報を平文で保存するのは非常に危険です。
* **悪用シナリオ**: 攻撃者は侵入後、`/proc/<PID>/environ` を読んだり、`env` コマンドを実行したりして、平文で保存された認証情報を窃取します。また、`LD_PRELOAD` 環境変数を悪用して共有ライブラリをハイジャックし、コマンド実行を乗っ取ります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**, **Hijacking**
* **解説**: `sudo` の設定で `env_keep` オプションが不適切に設定されていると、ユーザーが `sudo` 実行時に `LD_PRELOAD` などの危険な環境変数を保持でき、権限昇格に繋がります。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: パスワードなどの機密情報は環境変数に保存せず、HashiCorp Vaultのようなシークレット管理ツールや、アプリケーションの機能を使って管理する。`sudoers` の `secure_path` を設定し、`sudo` 実行時の `PATH` を安全なものに限定する。
* **Detection (検知)**: `auditd` で `setenv` システムコールの実行を監視する。`.bashrc` などの起動ファイルの変更をFIMツールで監視する。

## 注意点・補足

* **変数の展開**: シェルスクリプト内やコマンドラインで変数を参照する際は `$` を付けます（例: `echo $HOME`）。`export PATH="$PATH:/new/path"` のように、ダブルクォートで囲むと変数が展開され、シングルクォートでは展開されません。
