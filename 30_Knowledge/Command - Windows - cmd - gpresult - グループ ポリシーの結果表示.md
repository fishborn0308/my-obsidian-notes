# Command - Windows - cmd - gpresult - グループ ポリシーの結果表示

## 概要

`gpresult` は、特定のユーザーまたはコンピューターに対して適用されているグループ ポリシー設定の構成情報（RSoP: Resultant Set of Policy）を表示します。

Active Directory 環境において、どの GPO（Group Policy Object）が優先的に適用され、どの設定が拒否されているかを詳細に分析するために使用されます。

(出自: `Windows 標準搭載 - 外部コマンド (gpresult.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `rsop.msc` | GUI 版。視覚的に分かりやすいが、最近の Windows では非推奨。 | - |
| `gpupdate` | ポリシーを強制的に「更新」するコマンド。`gpresult` は「確認」用。 | セット利用 |
| `Get-GPResultantSetOfPolicy` | PowerShell 版。XML や HTML での結果出力が容易。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: ポリシー適用の競合調査 (運用・トラブルシューティング)

* **目的**: 設定したはずのセキュリティポリシーが、他の GPO によって上書き（拒否）されていないかを確認する。
* **連携コマンド**: `gpresult`, `findstr`
* **解説**: `/r` スイッチで概要を表示し、適用された GPO の一覧から優先順位を確認します。
* **コマンド例**:
    ```cmd
    REM 適用されている GPO の名前だけを抽出して確認
    gpresult /r | findstr "適用されたグループ ポリシー オブジェクト"
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/r` | **要約表示**。適用されている GPO の一覧、OS のバージョン、所属グループなどを表示。 |
| ⭐ `/v` | **詳細表示 (Verbose)**。具体的なポリシー設定値まで含めた詳細情報を表示。 |
| ⭐ `/h [filename].html` | **HTML レポート作成**。ブラウザで閲覧可能な非常に見やすいレポートを生成。 |
| `/scope {user | computer}` | 表示する対象を「ユーザー」または「コンピューター」に限定する。 |
| `/user [TargetUser]` | 指定したリモート ユーザーのポリシー結果を表示。 |
| `/z` | `/v` よりもさらに詳細な（全プロパティを含む）情報を表示。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しく作成した OU（組織単位）に適用した GPO が、クライアント PC に正しく反映されているかエビデンスを取得する。
* **例**:
    ```cmd
    REM HTML 形式でレポートを出力し、内容を監査用エビデンスとする
    gpresult /h C:\Reports\GPO_Report.html /f
    ```

### 2. ブルーチーム視点 (コンプライアンス監査)

* **タスク**: 組織のセキュリティ基準（パスワードの長さ、スクリーンセーバーのロック等）が、すべてのエンドポイントで強制されているか実態を調査する。
* **例**:
    ```cmd
    REM 特定のセキュリティ設定が適用されているか詳細モードで確認
    gpresult /v | findstr /i "Password Minimum"
    ```

### 3. レッドチーム視点 (偵察と環境把握 - T1615)

* **タスク 1: Defensive Softness Identification (防御の穴の探索)**
    * **目的**: 適用されている GPO を調査し、AppLocker が有効か、PowerShell の実行ポリシーが緩和されていないか、特定の管理者グループに誰が含まれているかを確認する。
* **タスク 2: Local Group Enumeration (特権グループの特定)**
    * **目的**: GPO によって強制的にローカル管理者に追加されているドメインユーザーやグループを特定し、横展開のターゲットにする。
* **タスク 3: Persistence Discovery (永続化の痕跡調査)**
    * **目的**: 攻撃者が GPO を改ざんしてスタートアップスクリプトを仕込んでいないか、不審な GPO 名がないかを確認する。

## エラーメッセージとトラブルシューティング

1.  **ERROR: アクセスが拒否されました (Access Denied)**
    * **原因**: コンピューター構成のポリシーを表示するには、管理者権限（管理者として実行）が必要です。
    * **解決策**: 管理者特権でプロンプトを開き直してください。

2.  **INFO: ユーザーに RSOP データがありません**
    * **原因**: 対象ユーザーが一度もそのコンピューターにサインインしていない、またはポリシーが一度も適用されていない。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**:
    * 稀に、GPO の「基本設定 (Group Policy Preferences)」の中に、サービスアカウントのパスワードが暗号化された状態で残っている（または過去に残っていた）ことがあり、`gpresult` の詳細出力はその探索のヒントになります。

### LOLBAS における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: Windows 標準の信頼されたツールであり、ポリシーを確認する行為自体は「正常な管理業務」と区別がつきにくいため、ステルス性の高い偵察が可能です。

## 注意点・補足

* **実行速度**: `/v` や `/z` を指定すると、情報の収集に数十秒から数分かかる場合があります。
* **リモート実行**: `/s [コンピューター名]` を使用してリモートから取得することも可能ですが、ターゲット側で RPC サービス等の開放が必要です。
