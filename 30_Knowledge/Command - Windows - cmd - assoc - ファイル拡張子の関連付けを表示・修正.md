# Command - Windows - cmd - assoc - ファイル拡張子の関連付けを表示・修正

## 概要

`assoc` (Association) コマンドは、ファイル拡張子（例: `.txt`）とファイル型（例: `txtfile`）の紐付けを表示、または変更するためのコマンドです。

Windowsが「どの拡張子を、どのグループとして扱うか」を定義するために使用されます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ftype` | `assoc` が「拡張子 → ファイル型」を定義するのに対し、`ftype` は「ファイル型 → 実行プログラム」を定義する。両者を組み合わせて動作が決まる。 | セットで使用 |
| `New-ItemProperty` (PS) | PowerShellでレジストリ (`HKCR`) を直接操作して関連付けを変更する。 | オブジェクト操作ならPS |

## よく連携されるコマンド

`assoc` は単体では「型名」を出すだけなので、詳細な実行パスを知るために `ftype` と連携することが一般的です。

### シナリオ例: 特定の拡張子が実行するプログラムの特定 (調査視点)

* **目的**: 不審な拡張子が、裏でどのバイナリを呼び出しているかを確認する。
* **連携コマンド**: `assoc`, `ftype`, `findstr`
* **解説**: まず `assoc` で拡張子の型名を調べ、その型名をキーに `ftype` でコマンドライン引数を確認します。
* **コマンド例**:
		```cmd
    REM .py 拡張子の型を確認し、その型に紐づく実行パスを表示する
    assoc .py && ftype Python.File
    ```


## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **引数なし** | 現在定義されているすべてのファイル拡張子の関連付けを表示する。 |
| `.ext` | 指定した拡張子（例: `.txt`）の現在の関連付けを表示する。 |
| `.ext=fileType` | 指定した拡張子を新しいファイル型（`fileType`）に関連付ける。 |
| `.ext=` | 指定した拡張子の関連付けを削除する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 特定の独自拡張子をテキストエディタで開くように一括設定する。
* **組み合わせ**: `assoc .log=txtfile`
* **解説**: ログファイルの拡張子が特殊な場合、OSに `txtfile`（標準のテキスト型）として認識させることで、ダブルクリック時にメモ帳などで開けるようにします。
* **例**:
		```cmd
    REM .myconf 拡張子を標準的なテキストファイル型として定義
    assoc .myconf=txtfile
    ```


### 2. ブルーチーム視点

* **タスク**: ファイル関連付けハイジャックによる不正な永続化や検疫回避の調査。
* **組み合わせ**: `assoc | findstr /V "expected_values"`
* **解説**: 本来テキストファイルであるはずの `.txt` や、ヘルプファイル `.chm` などが、不自然なファイル型に書き換えられていないかを監査します。
* **例**:
		```cmd
    REM 現在のすべての関連付けをログに出力し、ベースラインと比較する
    assoc > assoc_baseline.txt
    ```


### 3. レッドチーム視点

* **タスク**: 特定の拡張子をトリガーにしたコード実行（Persistence / 永続化）。
* **組み合わせ**: `assoc .txt=exefile` (極端な例)
* **解説**: ユーザーが日常的に開く拡張子の関連付けを変更し、自作のペイロードを介して本来のアプリを起動させることで、検知を遅らせつつコードを実行します。OSCPの試験範囲でも、標的のマシンでどのようにファイルが処理されるかを理解することは重要です。
* **例**:
		```cmd
    REM 標的環境で .php がどのように処理されるか設定を確認（Webシェル実行の前提確認）
    assoc .php
    ```


## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `Access is denied.` (アクセスが拒否されました。)
		* **考えられる原因**: `assoc` による関連付けの変更は、レジストリの `HKEY_CLASSES_ROOT` を操作するため、管理者権限が必要です。
		* **解決策**: コマンドプロンプトを「管理者として実行」してください。

2.  **エラーメッセージ例 2**: `File association not found for extension .xyz`
		* **考えられる原因**: その拡張子はまだシステムに登録されていません。
		* **解決策**: 新しく `assoc .xyz=typename` を実行して作成してください。

## 環境変数と設定ファイル

### 関連するレジストリ/設定ファイル

* **`HKEY_CLASSES_ROOT` (`HKCR`)**: `assoc` コマンドが参照・書き換えを行うメインの場所です。実際には `HKEY_LOCAL_MACHINE\Software\Classes` と `HKEY_CURRENT_USER\Software\Classes` のマージビューです。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: ファイル関連付けの書き換え。
* **悪用シナリオ**: 攻撃者が `ftype` で既存の型（例：`txtfile`）のオープンコマンドを悪意のあるスクリプトに変更し、`assoc` でその拡張子を確実に踏ませることで、ユーザーがドキュメントを開いた瞬間にマルウェアを実行させます。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Configuration` (システム設定の変更)
* **参照**: `assoc` 自体はバイナリの実行機能はありませんが、攻撃の準備段階（環境設定）で頻繁に利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 一般ユーザーからレジストリ書き込み権限を剥奪する。AppLocker等でスクリプトの実行を制限する。
* **Detection (検知)**: レジストリキー `HKCR` 以下の不審な変更を監視する（イベントID 4657: レジストリ値が変更されました）。

## 注意点・補足

* **PowerShellとの比較**: PowerShellでは `Get-ItemProperty` 等でより詳細なレジストリ操作が可能ですが、単純な確認であれば `assoc` の方が速く、出力も簡潔です。
* **優先順位**: Windowsの「設定」アプリ（GUI）でユーザーが手動で設定した「既定のアプリ」は、一部のレジストリ設定よりも優先される場合があります。
