# Command  - Linux - yum - RPM系パッケージ管理コマンド (旧)

## 概要

`yum` (Yellowdog Updater, Modified) は、Red Hat Enterprise Linux (RHEL) やCentOSなどのRPMベースのLinuxディストリビューションで使用される、高水準のパッケージ管理コマンドです。インターネット上のリポジトリからソフトウェアパッケージを検索し、依存関係を自動的に解決しながら、インストール、アップデート、削除といった操作を行います。

**注**: RHEL/CentOS 8以降のモダンなシステムでは、後継の `dnf` に置き換えられており、`yum` コマンドは `dnf` へのシンボリックリンクとなっていることがほとんどです。

(出自: `yum` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`dnf`** | `yum` の正式な後継。より高速で堅牢な依存関係解決エンジンを持ち、メモリ消費量も少ないです。コマンド構文はほぼ同じで互換性が保たれています。 | `dnf` |
| **`rpm`** | 低水準のパッケージ管理ツールで、`.rpm` ファイルを直接扱います。依存関係を自動で解決しません。 | `yum`/`dnf` |
| **`apt`** | Debian/Ubuntu系のパッケージ管理コマンド。機能的な役割は `yum` と同じですが、扱うパッケージ形式が `.deb` である点が異なります。 | - |

## よく連携されるコマンド

`yum` はシステムのパッケージ状態を変更するため `sudo` との連携が必須であり、その後にサービスを管理する `systemctl` と連携するのが一連のフローです。

### シナリオ例: サービスのインストールと有効化 (インフラ構築・運用視点)

* **目的**: Webサーバー (`httpd`) をインストールし、即座にサービスを起動、さらにOS起動時に自動で立ち上がるように設定する。
* **連携コマンド**: `sudo`, `systemctl`
* **解説**: `yum install` でパッケージを導入した後、`systemctl` を使ってサービスの管理(起動、自動起動設定)を行います。
* **コマンド例**:

```bash
# Apache httpdをインストール
sudo yum install -y httpd

# サービスを起動し、かつ自動起動も有効化する
sudo systemctl enable --now httpd

# サービスの状態を確認
sudo systemctl status httpd
```

## オプション説明 (`yum` のサブコマンド)

| オプション | 説明 |
| :--- | :--- |
| **パッケージリスト操作** | |
| ⭐ `install <pkg>` | 新しいパッケージをインストールします。 |
| ⭐ `update [<pkg>]` | 指定したパッケージ、または全システムをアップデートします。 |
| `remove <pkg>` | パッケージを削除します (`erase` も同義)。 |
| **情報表示** | |
| ⭐ `list [<keyword>]` | パッケージを一覧表示します。`--installed` や `--available` などの絞り込みが可能です。 |
| `search <keyword>` | パッケージ名や説明文からキーワードに一致するパッケージを検索します。 |
| `info <pkg>` | パッケージの詳細情報を表示します。 |
| `provides <file>` | 指定されたファイルを提供するパッケージを検索します (`whatprovides` も同義)。 |
| **その他** | |
| `history` | `yum` で行われたトランザクション(インストール、削除など)の履歴を表示します。 |
| `repolist` | 有効になっているリポジトリの一覧を表示します。 |
| `-y`, `--assumeyes` | すべての確認プロンプトに対して自動的に "yes" と回答します。スクリプト実行時に必須です。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システム全体のパッケージを最新の状態に保つ。
* **組み合わせ**: `sudo yum update -y`
* **解説**: パッケージリストを最新の状態に更新し、対話なしでシステム全体のパッケージをアップグレードする、最も基本的なサーバーメンテナンスコマンドです。
* **例**:

```bash
# 定期メンテナンスやセキュリティパッチの適用
sudo yum update -y
```

### 2. ブルーチーム視点

* **タスク**: セキュリティアップデートのみを適用して、システムの脆弱性を迅速に解消する。
* **組み合わせ**: `sudo yum update --security`
* **解説**: 利用可能なアップデートのうち、セキュリティ勧告が出ているものだけを適用します。機能変更を伴うメジャーアップデートを避け、セキュリティリスクのみを低減できます。
* **例**:

```bash
# まずセキュリティアップデートの有無を確認
sudo yum list updates --security

# 確認後、セキュリティアップデートのみを適用
sudo yum update --security -y
```

### 3. レッドチーム視点

* **タスク**: 侵入したシステムにインストールされているソフトウェアの一覧を取得し、脆弱性を探す。
* **組み合わせ**: `yum list installed`
* **解説**: 取得したソフトウェアリストとそのバージョンを元に、既知の脆弱性 (CVE) を持つソフトウェアが存在しないかを検索し、権限昇格やさらなる攻撃の足がかりを探します。
* **例**:

```bash
# インストール済みパッケージ一覧をファイルに出力して、攻撃者マシンに持ち帰り分析する
yum list installed > installed_pkgs.txt
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `No package <package_name> available.`
		* **考えられる原因**: パッケージ名が間違っているか、そのパッケージを含むリポジトリ(例: EPEL)が有効になっていません。
		* **解決策**: `yum search` でパッケージ名を再確認します。`sudo yum install epel-release` でEPELリポジトリを有効化するなど、必要なリポジトリを追加します。

2. **エラーメッセージ例 2**: `Could not resolve host: mirror.centos.org`
		* **考えられる原因**: サーバーのDNS設定に問題があるか、ネットワークに接続できていません。
		* **解決策**: `ping 8.8.8.8` や `cat /etc/resolv.conf` などでネットワーク接続とDNS設定を確認します。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/yum.conf`**: `yum` のグローバルな設定ファイル。
* **`/etc/yum.repos.d/`**: リポジトリ設定ファイル (`.repo`) を格納するディレクトリ。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `yum` 自体にも過去には脆弱性が存在しました。信頼できないリポジトリメタデータを処理する際の不備などです。
* **悪用シナリオ**: 攻撃者がリポジトリサーバーを侵害したり、中間者攻撃を仕掛けたりすることで、正規のパッケージに見せかけたマルウェア入りのRPMを配布する可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo yum` を実行できる権限を持っている場合、悪意のあるRPMパッケージを `localinstall` でインストールさせることで、RPMのインストールスクリプトをroot権限で実行できます。
* **参照**: `https://gtfobins.github.io/gtfobins/yum/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、`yum` のような高権限な操作が可能なコマンドを一般ユーザーに許可しない。リポジトリ設定でGPG署名検証 (`gpgcheck=1`) を常に有効にする。
* **Detection (検知)**: `yum history` を定期的に監査し、意図しないパッケージのインストールがないか確認する。

## 注意点・補足

* **EPELリポジトリ**: Enterprise Linux用の拡張パッケージ (EPEL) リポジトリは、標準リポジトリに含まれない多くの便利なパッケージを提供しており、`sudo yum install epel-release` で簡単に追加できます。
