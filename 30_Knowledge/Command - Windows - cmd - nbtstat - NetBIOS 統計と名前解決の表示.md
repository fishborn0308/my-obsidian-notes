# Command - Windows - cmd - nbtstat - NetBIOS 統計と名前解決の表示

## 概要

`nbtstat` は、TCP/IP 上の NetBIOS (NetBT) 統計、ローカルおよびリモート コンピュータの NetBIOS 名テーブル、および NetBIOS 名キャッシュを表示するためのコマンドです。

名前解決のトラブルシューティングや、IP アドレスからコンピュータ名、ログインユーザー、ドメイン名を特定する際に使用されます。

(出自: Windows標準搭載)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `arp -a` | IP から **MAC アドレス** を解決する。 | レイヤ2調査なら `arp` |
| `nslookup` | **DNS** 経由で名前解決を行う。 | インターネット/DNSなら `nslookup` |
| `ping -a` | IP からホスト名を解決しようとするが、NetBIOS 情報は取得できない。 | 簡易確認なら `ping -a` |

## よく連携されるコマンド

### シナリオ例: 同一セグメント内のホスト情報の詳細列挙 (偵察)

* **目的**: ターゲット IP の背後にあるコンピュータ名、ドメイン名、現在ログインしているユーザーを特定する。
* **連携コマンド**: `for /L`, `echo`
* **解説**: 特定の IP 範囲に対してループを回し、応答のあるホストの NetBIOS 情報を取得します。
* **コマンド例**:
    ```cmd
    REM 192.168.1.1から20まで、NetBIOS名をスキャン
    for /L %i in (1,1,20) do @nbtstat -A 192.168.1.%i
    ```

## スイッチ/オプション説明

`nbtstat` のスイッチは**大文字と小文字を区別する**点に注意してください。

| スイッチ | 説明 |
| :--- | :--- |
| **リモート調査** | |
| ⭐ `-a <Name>` | 指定した **コンピュータ名** を持つリモートマシンの名前テーブルを表示する。 |
| ⭐ `-A <IP>` | 指定した **IP アドレス** を持つリモートマシンの名前テーブルを表示する。 |
| **ローカル・キャッシュ** | |
| ⭐ `-c` | リモート名の **キャッシュ** 内容を表示する。 |
| `-n` | ローカルマシンの NetBIOS 名テーブルを表示する。 |
| `-r` | Windows ネットワーク名解決（ブロードキャストおよび WINS）の統計を表示。 |
| ⭐ `-R` | 名前キャッシュの内容を消去し、LMHOSTS ファイルから再ロードする（リセット用）。 |
| **統計情報** | |
| `-s` | NetBIOS セッションの統計情報を表示し、宛先 IP を名前で解決しようとする。 |
| `-S` | NetBIOS セッションの統計情報を IP アドレスのまま表示する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: コンピュータ名変更後の名前解決トラブル解消
* **解説**: サーバー名を変更したのに古い名前で参照される場合、ローカルのキャッシュをクリアして強制的に再解決させます。
* **例**:
    ```cmd
    REM NetBIOSキャッシュをクリアして再読み込み
    nbtstat -R
    ```

### 2. ブルーチーム視点

* **タスク**: 不審な NetBIOS ブロードキャストの送信元特定
* **解説**: ネットワーク内で不自然な名前解決要求を投げている端末の IP が判明した際、その「正体（コンピュータ名とユーザー）」を突き止めます。
* **例**:
    ```cmd
    REM 調査対象IP 10.0.0.50 の詳細情報を取得
    nbtstat -A 10.0.0.50
    ```

### 3. レッドチーム視点 (OSCP等の初期偵察)

* **タスク**: ターゲットの役割（DC、ファイルサーバー等）とドメイン名の特定
* **解説**: 取得した名前テーブルの「16進数サフィックス」を見ることで、そのマシンの役割を判断できます。
    - `<00>`: Workstation Service（一般的なPC）
    - `<20>`: File Server Service（共有がある）
    - `<1B>`: Domain Master Browser
    - `<03>`: ログインしているユーザー名
* **例**:
    ```cmd
    REM ターゲットの役割を分析
    nbtstat -A 192.168.10.101
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `ホストが見つかりません。`
    * **考えられる原因**:
        - ターゲットが NetBIOS を無効にしている（現代の Windows ではデフォルトで制限されている場合がある）。
        - ファイアウォール (UDP 137番ポート) で遮断されている。
    * **解決策**: ターゲットの NetBIOS over TCP/IP 設定を確認する。

2.  **エラーメッセージ例 2**: `名前を解決できませんでした。`
    * **考えられる原因**: 指定した名前がネットワーク上に存在しない、またはキャッシュが古すぎる。
    * **解決策**: `nbtstat -R` を試すか、IP アドレス (`-A`) での指定に切り替える。

## 環境変数と設定ファイル

* **LMHOSTS ファイル**: `%SystemRoot%\System32\Drivers\Etc\LMHOSTS`
    - NetBIOS 名と IP アドレスの静的な対応表。`nbtstat -R` はこのファイルを再読み込みします。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**:
    - **Information Disclosure**: NetBIOS は認証なしでコンピュータ名やログインユーザー名を返してしまうため、攻撃者に組織内の命名規則やアカウント名を把握されるきっかけになります。
    - **LLMNR/NBT-NS Poisoning**: 攻撃者が `Responder` などのツールを使い、`nbtstat` が探している名前に「私がそのサーバーです」と偽の応答を返すことで、認証ハッシュを窃取する（Relay 攻撃）。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    - 現代的なネットワークでは、DHCP 設定や GPO で **NetBIOS over TCP/IP を無効化** することを強く推奨します。
    - LLMNR も同時に無効化し、名前解決を DNS に一本化します。
* **Detection (検知)**:
    - UDP 137, 138 ポートの異常なスキャンを IDS/IPS で監視する。

## 注意点・補足

* **大文字・小文字**: `-a` (name) と `-A` (IP) は、他の Windows コマンド（多くは区別しない）と異なり厳密です。
* **IPv6 非対応**: `nbtstat` は IPv4 環境の NetBIOS を対象としています。IPv6 では別の仕組み（mDNS 等）が利用されます。
