# Command - Windows - cmd - fsutil - ファイルシステムの管理と操作

## 概要

`fsutil` は、FAT および NTFS ファイルシステムに関する高度なタスクを実行するためのコマンドライン ユーティリティです。

ディスク クォータの管理、ハードリンクの作成、USN ジャーナルのクエリ、スパースファイルの操作など、エクスプローラーや通常の `dir` コマンドでは不可能なファイルシステムの内部メタデータへのアクセスと操作を可能にします。

(出自: `Windows 標準搭載 - 外部コマンド (fsutil.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `chkdsk` | ファイルシステムの整合性チェックと修復。 | 修復・診断 |
| `diskpart` | ディスクのパーティションやボリュームレベルの管理。 | ボリューム管理 |
| `mklink` | シンボリックリンクやハードリンクの作成（内蔵コマンド）。 | リンク作成 |
| `cipher` | ファイルの暗号化や空き領域の消去に特化。 | データ抹消 |

## よく連携されるコマンド

### シナリオ例: ファイル操作履歴の調査 (ブルーチーム視点)

* **目的**: 特定のファイルがいつ作成、削除、または変更されたかを、NTFS の USN ジャーナルから特定する。
* **連携コマンド**: `fsutil usn`, `findstr`
* **解説**: ファイルが削除されていても、ジャーナルにはその操作記録（メタデータ）が残っている場合があります。
* **コマンド例**:
    ```cmd
    REM Cドライブの USN ジャーナルから、ファイル変更履歴を表示
    fsutil usn readjournal C:
    ```

## スイッチ/サブコマンド説明

**※実行には必ず「管理者権限」が必要です。**

| サブコマンド | 説明 |
| :--- | :--- |
| **ファイル・リンク操作** | |
| ⭐ `file createnew` | 指定したサイズの新しいファイルを瞬時に作成する。 |
| `hardlink create` | 既存のファイルに対するハードリンクを作成する。 |
| ⭐ `sparse` | スパースファイル（大部分がゼロで構成される巨大なファイル）を制御する。 |
| **ジャーナル・メタデータ** | |
| ⭐ `usn` | Update Sequence Number (USN) ジャーナルの管理と読み取り。 |
| `objectid` | オブジェクト ID（ファイルを追跡するための不変 ID）の管理。 |
| **ボリューム・クォータ** | |
| `volume diskfree` | 指定したボリュームの空き容量を詳細に表示。 |
| `quota` | NTFS ディスク クォータ（ユーザーごとの容量制限）を管理。 |
| `dirty` | ボリュームの「ダーティ ビット」（次回起動時に chkdsk が必要か）を管理。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのストレージ容量テストのために、10GB の空ファイルを一瞬で作成する。
* **例**:
    ```cmd
    REM 10GB (10737418240 バイト) のファイルを生成
    fsutil file createnew C:\temp\test_10GB.dat 10737418240
    ```

### 2. ブルーチーム視点 (デジタルフォレンジック)

* **タスク**: 攻撃者がツールをダウンロードして削除した可能性がある場合、USN ジャーナルを調べて証拠（ファイル名やタイムスタンプ）を回収する。
* **例**:
    ```cmd
    REM ログファイルに関連するジャーナル記録を抽出
    fsutil usn readjournal C: | findstr /i "secret.txt"
    ```

### 3. レッドチーム視点 (リソース消費と隠蔽)

* **タスク 1: Storage Exhaustion (リソース枯渇攻撃 - T1499)**
    * **目的**: ターゲットサーバーの空きディスク容量を巨大なファイルで埋め尽くし、システムログの記録停止やサービスの異常終了を誘発する。
    * **解説**: `createnew` は実際にディスクを消費しますが、`sparse` を併用すると「見かけ上のサイズは巨大だが実占有量は小さい」ファイルも作成可能です。
* **タスク 2: Defense Evasion (検知回避のためのハードリンク)**
    * **目的**: 重要なファイルを直接触るのではなく、別の場所にハードリンクを作成して操作することで、特定のファイルパスを監視する単純なツールを回避する試み。

## エラーメッセージとトラブルシューティング

1.  **アクセスが拒否されました (Access is denied)**
    * **原因**: 管理者権限のあるプロンプトで実行していない。`fsutil` はシステム深層を触るため必須。

2.  **この操作には NTFS が必要です**
    * **原因**: FAT32 ボリュームに対して、NTFS 専用のサブコマンド（`usn`, `quota` 等）を実行した。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **USN ジャーナルの消去 (T1070.003)**:
    * 攻撃者は痕跡を消すために `fsutil usn deletejournal` を実行することがあります。これは非常に強力な証拠隠滅手法です。
* **シンボリックリンク/ハードリンクの悪用**:
    * 権限の弱いディレクトリから重要なシステムファイルへリンクを張り、特権プロセスにそのリンク先を操作させることで権限昇格を狙う手法（リンク攻撃）に関連します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Artifact Removal` (痕跡消去), `Denial of Service`
* **参照**: [LOLBAS - fsutil.exe](https://lolbas-project.github.io/lolbas/Binaries/Fsutil/)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * **イベントID 4688**: `fsutil.exe` の実行、特に `usn deletejournal` や `file createnew` の引数を監視。
    * **ジャーナル監視**: ジャーナル自体が削除されたことを検知するのは困難な場合が多いため、`fsutil` の実行そのものを SIEM でアラート化すべきです。

## 注意点・補足

* **即時性**: `file createnew` で作成されたファイルは中身がゼロで埋められませんが、ファイルシステム上では即座に指定サイズとして確保されます。
* **スパースファイル**: `fsutil sparse setflag` を使うと、1TB のファイルを数 KB の実容量で作成するといったことが可能になり、ディスククォータの制限をテストする際などに役立ちます。
