# Command  - Linux - chkconfig - SysVinitサービスの自動起動を管理する (RHEL系/旧)

## 概要

`chkconfig` は、**SysVinit** を採用した旧世代のRed Hat系Linuxディストリビューション（RHEL 6, CentOS 6以前など）において、システムサービスの自動起動設定を管理するためのコマンドです。OSの起動レベル（ランレベル）ごとに、どのサービスを起動 (`S*`)・停止 (`K*`) するかを、`/etc/rc.d/rc<runlevel>.d/` ディレクトリ配下のシンボリックリンクを操作することで制御します。

**⚠️ 重要**: このコマンドは**レガシー（旧式）**です。現在主流の `systemd` を採用したシステム (RHEL/CentOS 7以降など) では、**`systemctl enable/disable`** を使用してください。

(出自: `chkconfig` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`systemctl`** | `systemd` 環境におけるモダンな後継コマンド。サービスの起動/停止、自動起動設定、ログ確認などを統一的に扱え、依存関係管理も強力。 | `systemctl` |
| **`update-rc.d`** | `chkconfig` のDebian/Ubuntu系における同等コマンド。機能的な役割は同じだが、コマンドの構文が異なる。 | - |
| **`service`** | **今すぐ**サービスを起動/停止するためのコマンド。`chkconfig` は**OS起動時**の自動起動を設定するコマンドであり、役割が異なる。 | `systemctl` |

## よく連携されるコマンド

`chkconfig`は、`/etc/init.d/` 配下のスクリプトを管理対象に追加する際に、ファイルのパーミッションを設定するコマンドなどと連携します。

### シナリオ例: 新しいinitスクリプトをサービスとして登録する (インフラ構築・運用視点)

* **目的**: カスタムアプリケーションの起動スクリプトを `/etc/init.d/` に配置し、`chkconfig` で管理できるようにして、OS起動時に自動実行させる。
* **連携コマンド**: `cp`, `chmod`, `chkconfig`
* **解説**: まず起動スクリプトを所定の場所にコピーし、実行権限を付与します。次に `chkconfig --add` でサービスとして登録し、`chkconfig on` で自動起動を有効化します。
* **コマンド例**:

```bash
# このコマンドシーケンスでカスタムサービスを登録・有効化する
sudo cp myapp-script /etc/init.d/myapp
sudo chmod +x /etc/init.d/myapp
sudo chconfig --add myapp
sudo chkconfig myapp on
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **サービス管理** | |
| ⭐ `--list [<service>]` | 全て、または指定したサービスの自動起動設定を、ランレベルごとに一覧表示します。 |
| ⭐ `<service> on` | サービスをデフォルトのランレベル（通常 2,3,4,5）で有効化します。 |
| ⭐ `<service> off`| サービスを全てのランレベルで無効化します。 |
| **詳細設定** | |
| `--add <service>`| `/etc/init.d/` にある新しいサービスを `chkconfig` の管理対象に追加します。 |
| `--del <service>` | サービスを管理対象から削除します（スクリプトファイル自体は削除しません）。 |
| `--level <levels>`| 特定のランレベル (`<levels>`) に対してのみ、`on` または `off` を設定する、より詳細な制御。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Apache (`httpd`) サービスをOS起動時に自動で立ち上がるように設定する。
* **組み合わせ**: `chkconfig <service_name> on`
* **解説**: サーバが再起動してもWebサービスが自動で開始されるように、サービスの自動起動を有効化します。
* **例**:

```bash
# httpdサービスの現在の設定を確認
chkconfig --list httpd

# httpdサービスを自動起動ONに設定
sudo chkconfig httpd on
```

### 2. ブルーチーム視点

* **タスク**: システムに不審な永続化設定がないか監査する。
* **組み合わせ**: `chkconfig --list`
* **解説**: システムで自動起動が有効になっているサービスを全てリストアップします。リストの中に、標準的でない、あるいは明らかに不審な名前のサービスが `on` になっているのを発見した場合、それは攻撃者が設置した永続化の仕組みである可能性が極めて高いです。
* **例**:

```bash
# 自動起動が有効なサービスをリストアップし、不審なものがないか確認
chkconfig --list | grep 'on'
```

### 3. レッドチーム視点

* **タスク**: レガシーシステム上で永続化を確立する。
* **組み合わせ**: `chkconfig --add <backdoor_svc> && chkconfig <backdoor_svc> on`
* **解説**: 攻撃者がroot権限を奪取した後、`/etc/init.d/` ディレクトリにリバースシェルなどを起動する悪意のあるスクリプトを設置し、このコマンドでサービスとして登録・有効化します。これにより、システムが再起動しても自身のバックドアが自動的に実行されるように仕掛けます。
* **例**:

```bash
# 悪意のあるスクリプトをサービスとして登録し、有効化する
sudo chkconfig --add evil-svc
sudo chkconfig evil-svc on
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `error reading information on service <service_name>: No such file or directory`
		* **考えられる原因**: `/etc/init.d/` ディレクトリに、指定したサービス名の起動スクリプトが存在しません。
		* **解決策**: サービス名が正しいか、パッケージが正しくインストールされているかを確認してください。

2. **エラーメッセージ例 2**: `service <service_name> does not support chkconfig`
		* **考えられる原因**: `/etc/init.d/` にある起動スクリプトの冒頭に、`chkconfig` が必要とする特別なコメント行（例: `# chkconfig: 2345 80 20`）が記述されていません。
		* **解決策**: スクリプトを編集し、適切な `chkconfig` ヘッダを追加する必要があります。

## 環境変数と設定ファイル

* `chkconfig` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/init.d/`**: サービスの起動・停止などを制御する実際のスクリプトが置かれるディレクトリ。
* **`/etc/rc.d/rc[0-6].d/`**: 各ランレベルに対応するディレクトリ。中には `/etc/init.d/` にあるスクリプトへのシンボリックリンクが置かれます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `chkconfig` 自体ではなく、それが管理するSysVinitの仕組みが、攻撃者による**永続化**の確立に悪用されます。
* **悪用シナリオ**: 攻撃者がroot権限を奪取すると、`/etc/init.d/` に悪意のあるスクリプトを設置し、`chkconfig` でそれをサービスとして有効化することで、永続的なバックドアを作成します。

### GTFOBins / LOLBAS における利用例

`chkconfig` は、OSの構成を変更するコマンドであり、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。しかし、**永続化 (Persistence)** の文脈で、悪意のあるスクリプトを登録するために悪用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `/etc/init.d/` ディレクトリ配下のスクリプトのパーミッションを厳格に管理し、不要な書き込み権限を許可しない。
* **Detection (検知)**: `chkconfig --list` の出力を定期的に取得し、ベースラインと比較して意図しないサービスの追加がないかを監視する。ファイル整合性監視 (FIM) ツールで `/etc/init.d/` や `/etc/rc.d/` 配下の変更を検知する。

## 注意点・補足

* **ランレベル (Runlevel)**: SysVinitは、システムの動作モードを0から6の「ランレベル」で定義します（例: `3`はテキストベースのマルチユーザーモード、`5`はグラフィカルモード）。`chkconfig` は、どのランレベルでどのサービスを起動するかを設定しています。
