# OS  - Linux - top, htop, free - システムリソース監視ツール

## 概要

`top`, `htop`, `free` は、Linuxシステムの**リソース使用状況**を把握するための基本的なコマンド群です。それぞれに異なる役割と特徴があり、組み合わせて使用することで、パフォーマンス問題の診断を効率的に行うことができます。

* **`free`**: **メモリ**の全体的な使用状況の**スナップショット**を素早く表示します。
* **`top`**: CPUやメモリの使用状況を、**プロセスごと**に**リアルタイム**で表示する古典的なツールです。
* **`htop`**: `top` のモダンな後継。色付けされた表示、スクロール、対話的な操作が特徴で、**より直感的で高機能**です。

(出自: `procps-ng` (`top`, `free`), `htop` パッケージ)

## 類似コマンドと差異

| 特徴 | `free` | `top` | `htop` (推奨) |
| :--- | :--- | :--- | :--- |
| **主目的** | **メモリ全体**の概要把握 | **プロセスごと**のリソース監視 | **プロセスごと**のリソース監視 (高機能) |
| **インターフェイス**| 静的（一回表示して終了） | リアルタイム（自動更新）、キー操作 | **対話的**（カーソルキー、マウス操作可） |
| **表示** | テキストベースのテーブル | テキストベース（単色） | **色付き**、**グラフィカルなメーター** |
| **使いやすさ**| 非常に簡単 | 学習が必要 | **直感的で簡単** |
| **デフォルト** | ほぼ全てのシステムに導入済み | ほぼ全てのシステムに導入済み | インストールが必要な場合がある |

## よく連携されるコマンド

これらのコマンドは、問題を発見した後に原因を特定・対処するため、`kill` や `strace` といったコマンドと連携します。

### シナリオ例: 「サーバーが遅い」の原因調査 (インフラ構築・運用視点)

* **目的**: システムのパフォーマンス低下の原因を特定し、対処する。
* **連携コマンド**: `free`, `htop`, `kill`
* **解説**: まず `free -h` でメモリ全体像を把握します。もしメモリが逼迫していれば `htop` を起動し、`M`キーでメモリ消費順にソートして原因プロセスを特定します。CPU負荷が高い場合は `P`キーでソートします。暴走しているプロセスは `F9`キーで停止させます。
* **コマンド例**:

		```bash
    # このコマンドシーケンスでパフォーマンス問題を診断する
    # 1. まずはメモリの全体状況を確認

    free -h

    # 2. 次にhtopで詳細なプロセス状況を確認

    htop

    # 3. (htop内で) 'P' や 'M' を押してソートし、原因プロセスを特定

    # 4. (htop内で) 'k' または 'F9' を押して、必要であればプロセスを停止
    ```


## オプション説明 (各コマンド)

| コマンド | オプション | 説明 |
| :--- | :--- |:---|
| ⭐ **`free`** | `-h` | 人間が読みやすい単位 (K, M, G) で表示します。 |
| ⭐ **`top`** | `P` (大文字) | **CPU使用率**でソートします。 |
| | `M` (大文字) | **メモリ使用率**でソートします。 |
| | `k` | プロセスをkillします。 |
| ⭐ **`htop`** | `F6` | ソートするカラムを選択します。 |
| | `F9` / `k` | プロセスにシグナルを送信（kill）します。 |
| | `F5` | プロセスの親子関係をツリー形式で表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーのパフォーマンス低下の一次対応。
* **組み合わせ**: 上記「よく連携されるコマンド」のシナリオ例を参照。
* **解説**: `free` で全体像を掴み、`htop` で犯人を特定し、`kill` で対処する、という流れはトラブルシューティングの基本です。
* **例**:

		```bash
    # まずはメモリ状況を確認

    free -h

    # 次に詳細をhtopで確認

    htop

    ```


### 2. ブルーチーム視点

* **タスク**: ライブレスポンス中に不審なプロセスの活動を発見する。
* **組み合わせ**: `sudo htop`
* **解説**: `htop` を実行し、CPUを異常に消費している偽装名のプロセス（クリプトマイナーなど）や、サービスアカウントから起動された予期せぬシェル (`/bin/sh`) がいないかを `F5` (ツリー表示) で確認します。
* **例**:

		```bash
    # ライブフォレンジック調査のためにhtopを管理者権限で起動

    sudo htop

    ```


### 3. レッドチーム視点

* **タスク**: 侵入後の偵察と、自身の活動の防御回避。
* **組み合わせ**: `top`, `htop`
* **解説**: 攻撃者は `top` や `htop` を実行し、システムでどのようなサービスやセキュリティ製品（EDRエージェントなど）が動いているかを把握します。また、自身のマルウェアがCPUを過剰に消費して発見されることがないよう、リソース消費を低く抑えます。
* **例**:

		```bash
    # システムで稼働中のプロセスを偵察

    top -b -n 1

    ```


## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **エラーメッセージ例 1**: `bash: htop: command not found`
		* **考えられる原因**: `htop` は多くのシステムでデフォルトではインストールされていません。
		* **解決策**: システムのパッケージマネージャを使ってインストールしてください。（Debian/Ubuntu: `sudo apt install htop`, RHEL/CentOS: `sudo yum install htop`）

## 環境変数と設定ファイル

* `free`, `top` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`~/.config/htop/htoprc`**: `htop` の `F2` (Setup) でカスタマイズした表示設定が保存されます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: これらのツール自体ではなく、その**表示内容**が**情報漏洩**に繋がります。
* **悪用シナリオ**: `top` や `htop` は実行中の全てのプロセスと、その**コマンドライン引数**を表示します。もし、引数にパスワードなどの機密情報が平文で渡されている場合、これらのコマンドを実行できる全てのユーザーにその情報が閲覧可能になってしまいます。

### GTFOBins / LOLBAS における利用例

これらのコマンドは情報表示が主目的であり、直接的な権限昇格機能はないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: アプリケーションにパスワードなどの機密情報を渡す際は、コマンドライン引数ではなく、環境変数や設定ファイル、シークレット管理ツールを使用する。
* **Detection (検知)**: EDRや`auditd`で、`top` や `htop` の実行を監視し、不審なユーザーによる偵察活動がないかを確認する。

## 注意点・補足

* **`free` の `available`**: メモリの空き状況を判断する際は、`free` カラムではなく、解放可能なキャッシュを含む `available` カラムを見るのが最も重要です。
