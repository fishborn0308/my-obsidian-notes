# Command - Linux - cat - ファイルの内容を表示・連結する

## 概要

`cat` は、1つまたは複数のファイルの内容を標準出力（通常は画面）に連続して出力するコマンドです。本来の目的は「ファイルの連結（concatenate）」ですが、単一ファイルの内容を素早く確認するために最も頻繁に使用されます。

(出自: `coreutils` パッケージに含まれる。Unix初期から存在する最古参コマンドの一つ)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `less / more` | 長大なファイルを表示する際、スクロールや検索が可能。 | 巨大なファイルの閲覧 |
| `tac` | `cat` の逆（catのスペルを逆にしたもの）。ファイルの末尾行から順に表示する。 | 最新のログを先に読みたい時 |
| `head / tail` | ファイルの先頭、または末尾の数行だけを表示する。 | ログの断片確認 |
| `bat` | `cat` のクローンだが、シンタックスハイライトやGit統合機能がある。 | 開発・コード閲覧 |

## よく連携されるコマンド

### シナリオ例: 複数ファイルの一括結合 (運用視点)

* **目的**: 分割されたログファイルや設定ファイルを1つのファイルにまとめる。
* **連携コマンド**: リダイレクト (`>`)
* **解説**: 複数の引数を取り、順番に結合して出力する本来の機能を利用します。
* **コマンド例**:
    ```bash
    # log1, log2, log3 を結合して all_logs.txt を作成
    cat log1.txt log2.txt log3.txt > all_logs.txt
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示カスタマイズ** | |
| ⭐️ `-n`, `--number` | 全ての行に番号を付けて表示する。 |
| ⭐️ `-b`, `--number-nonblank` | 空行を除いて行番号を付ける。 |
| ⭐️ `-A`, `--show-all` | タブ (`^I`) や改行記号 (`$`)、制御文字を可視化して表示する。 |
| `-s`, `--squeeze-blank` | 連続する空行を1行にまとめて表示する。 |
| `-e` | 改行文字の末尾に `$` を表示する（`-vE` と同等）。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイル内の見えない「タブ」や「行末スペース」の確認。
* **組み合わせ**: `-A`
* **解説**: 設定ファイル（特に行末のスペースに厳しいYAMLやスクリプト）で、不具合の原因となる制御文字が混入していないか調査します。
* **例**:
    ```bash
    # 目に見えない文字を可視化して表示
    cat -A /etc/nginx/nginx.conf
    ```

### 2. ブルーチーム視点

* **タスク**: ログインログ等の短い証拠データの素早い確認。
* **組み合わせ**: `cat` + `grep`
* **解説**: ログファイル全体を一度に読み出し、必要なパターンのみを抽出します。
* **例**:
    ```bash
    # 失敗したログイン試行のみを素早く表示
    cat /var/log/auth.log | grep "Failed password"
    ```

### 3. レッドチーム視点

* **タスク**: `/etc/passwd` 等の機密情報の取得、およびファイル作成。
* **組み合わせ**: `cat << EOF` (ヒアドキュメント)
* **解説**: エディタ（vi等）が使えない制限されたシェル環境で、複数行のペイロードやスクリプトをファイルとして書き出す際によく使われます。
* **例**:
    ```bash
    # ターゲット上でスクリプトを生成
    cat << EOF > /tmp/exploit.sh
    #!/bin/bash
    id
    uname -a
    EOF
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `cat: <filename>: Permission denied`
    * **考えられる原因**: 実行ユーザーにそのファイルを読み取る権限がない。
    * **解決策**: `sudo` を使用する、またはファイルパーミッションを確認する。

2.  **エラーメッセージ例 2**: `cat: <directory_name>: Is a directory`
    * **考えられる原因**: ファイルではなくディレクトリを指定した。
    * **解決策**: 適切なファイル名を指定する。

## 環境変数と設定ファイル

* `cat` 自体には設定ファイルはありませんが、出力先のターミナルの環境変数（`LANG`等）が文字化けに影響することがあります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: それ自体にバグは稀ですが、**バイナリファイルを `cat` することによるターミナルの混乱**に注意が必要です。
* **悪用シナリオ**: 攻撃者が細工したエスケープシーケンスを含むファイルを作成し、管理者がそれを `cat` した瞬間にターミナルの設定を変更したり、特定のキー入力をエミュレートさせたりする（Terminal Escape Sequence Injection）。

### GTFOBins における利用例

`cat` に `sudo` 権限がある場合、本来読み取れない特権ファイル（`/etc/shadow` や `root/.ssh/id_rsa`）を読み取ることができます。OSCP等のペンテストで最初に確認すべき項目の一つです。

* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo を使って特権ファイルを読み取る
    sudo cat /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/cat/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要な機密ファイル（`/etc/shadow` 等）の読み取り権限を最小化する。
* **Detection (検知)**: `/etc/shadow` やプライベートキーへの `cat` 実行を `auditd` で監視し、不自然なアクセスをアラート対象にする。

## 注意点・補足

* **UUOC (Useless Use of Cat)**: `cat file | grep pattern` と書くのは、実は非効率です（`grep pattern file` で十分）。プロの現場では「不必要な cat」を避けることが推奨されます。
* **巨大ファイル**: 数GBあるファイルを `cat` すると、ターミナルの表示が止まらなくなります。その場合は `Ctrl+C` で中断するか、最初から `less` を使いましょう。
