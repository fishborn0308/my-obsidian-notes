# Command - Linux - fuser - ファイルやソケットを使用しているプロセスを特定する

## 概要
`fuser` は、引数として指定したファイル、ディレクトリ、またはネットワークソケットを現在使用しているプロセスの PID（プロセスID）を表示します。デフォルトでは標準出力に PID を、標準エラー出力にアクセスの種類（読み取り、書き込み等）を表示します。
(出自: `psmisc` パッケージに含まれる)

## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `lsof` | 「List Open Files」。より多機能で詳細だが、出力が膨大。特定のファイルから逆引きするには `fuser` が速い。 | 網羅的な調査 |
| `ss / netstat` | ネットワークソケットの情報に特化。 | 接続状況の確認 |
| `kill` | PIDを指定してプロセスを終了させる。 | 個別のプロセス終了 |
| `fuser` | 「対象（ファイル/ポート）」を指定して、それを使っている者を特定・終了させる。 | **原因の特定と即時排除** |

## よく連携されるコマンド
### シナリオ例: 使用中のポートを強制解放してサービス再起動 (運用視点)
* **目的**: Webサーバーなどが異常終了し、ポート 80 が掴みっぱなしで再起動できない時に、犯人を強制終了させる。
* **解説**: `-k` オプションで特定したプロセスを即座に Kill します。
* **コマンド例**:
    ```bash
    # TCP 80番ポートを使っているプロセスを特定し、SIGKILL(9)で終了させる
    sudo fuser -k -n tcp 80
    ```

## オプション説明
※詳細な情報の表示やプロセスの操作には **root 権限** が推奨されます。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-v`, `--verbose` | 詳細表示モード。ユーザー名、PID、アクセスの種類などを表形式で表示する。 |
| ⭐️ `-k`, `--kill` | 指定したファイルを使用しているプロセスを終了させる（デフォルトは SIGTERM）。 |
| ⭐️ `-n <space>`, `--namespace` | 名前空間を指定する（`file` (デフォルト), `tcp`, `udp`）。 |
| ⭐️ `-m`, `--mount` | 指定したファイルがマウントされているファイルシステム全体を使用しているプロセスを列挙する。 |
| `-i`, `--interactive` | プロセスを終了させる前にユーザーに確認を求める（`-k` と併用）。 |
| `-SIGNAL` | 指定したシグナルを送る（例: `-HUP`, `-9`）。 |

### アクセス種別の記号 (ACCESS 列)
* `c`: カレントディレクトリとして使用中
* `e`: 実行ファイルとして使用中
* `f`: オープン中のファイル（デフォルトでは表示されない場合あり）
* `m`: 共有ライブラリとしてマッピング中
* `r`: ルートディレクトリとして使用中

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 「Device is busy」でアンマウントできない原因の調査。
* **解説**: `/mnt/usb` などのディレクトリを誰かが開いている（あるいは `cd` している）場合に特定します。
* **例**:
    ```bash
    # マウントポイントを使用しているプロセスを表示
    sudo fuser -v /mnt/usb
    ```

### 2. ブルーチーム視点
* **タスク**: 不審なポート待ち受け（バックドア）のプロセス特定。
* **解説**: `netstat` 等で見つけた怪しいポート番号を `fuser` に渡し、その実体（どのバイナリか）を突き止めます。
* **例**:
    ```bash
    # TCP 4444番ポートを使用しているプロセスの詳細を確認
    sudo fuser -v -n tcp 4444
    ```

### 3. レッドチーム視点
* **タスク**: セキュリティソフトやログ監視プロセスの特定と妨害。
* **解説**: 特定の設定ファイルやログファイルを常に読み書きしている監視プロセスを特定し、`-k` で終了させて活動を隠蔽しようと試みます。
* **例**:
    ```bash
    # ログファイルを監視しているプロセスを特定
    fuser -v /var/log/auth.log
    ```

## エラーメッセージとトラブルシューティング

1.  **「PID しか表示されない」**:
    * **原因**: デフォルトの出力形式。
    * **解決策**: `-v` オプションを付けて人間が読みやすい形式にする。

2.  **「何も表示されない」**:
    * **原因**: そのファイルは誰にも使われていないか、権限不足で情報を取得できていない。
    * **解決策**: `sudo` を付けて実行する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **Denial of Service (DoS)**: 攻撃者が `sudo fuser` を実行できる場合、システムにとって重要なプロセス（データベースや SSH デーモン）が使用しているポートを指定して `-k` を実行することで、サービスを簡単に停止させることができます。

### GTFOBins における利用例
`fuser` が `sudo` で実行可能な場合、任意のプロセスを終了させることができるため、セキュリティ対策ソフトを無効化するなどの悪用が可能です。
* **機能**: `Sudo`
* **コマンド例**:
    ```bash
    # sudo fuser を使って root 権限のプロセスを終了させる
    sudo fuser -k /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/fuser/`

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: 一般ユーザーに `sudo fuser` を許可しない。
* **Detection (検知)**: 重要なシステムファイルや特権ポートに対して `fuser -k` が実行された形跡をログから検知する。

## 注意点・補足
* **環境差**: `fuser` は Linux 固有の機能（`/proc`）に強く依存しているため、BSD 系や macOS では `lsof` を使うのが一般的です。
* **慎重な Kill**: `-k` を使う際は、まず `-v` で対象を確認し、さらに `-i` で確認を入れるのが安全です。
