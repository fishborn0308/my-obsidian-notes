# Command  - Linux - telnet - テキストベースのリモート接続プロトコル (旧)

## 概要

`telnet` は、ネットワーク上の別のコンピュータと対話的なテキストベースの通信を行うための、古典的なプロトコルおよびクライアントコマンドです。元々は、リモートのマシンにログインし、シェルを操作するために広く利用されていました。

しかし、`telnet` は通信内容（ユーザー名、パスワード、操作コマンドなど）を**一切暗号化せず、平文で送信します**。そのため、リモートログインの目的で利用することは**極めて危険であり、現在では完全に時代遅れ**です。現代における `telnet` の主な用途は、特定のホストのTCPポートが開いているかを**手動でテストする**ための、シンプルな接続確認ツールとしての役割に限られます。

(出自: `telnet` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`ssh`** | **`telnet` の安全な後継**。全ての通信を暗号化します。リモートログインには**常に `ssh` を使用してください**。 | `ssh` |
| ⭐ **`netcat` (`nc`)**| `telnet` と同様にポート接続テストができますが、サーバーとして動作したり、UDPを使ったりと、遥かに高機能でスクリプトとの親和性も高いです。 | `nc` |
| **`curl`** | HTTP/HTTPSなどのアプリケーション層プロトコルのテストに特化しています。`telnet` はより低レベルなTCP接続の確立を確認します。 | - |

## よく連携されるコマンド

`telnet` は単体で完結する対話的なツールのため、他のコマンドとパイプで連携することはほとんどありません。

### シナリオ例: SMTPサーバーの動作を手動でテストする (インフラ構築・運用視点)

* **目的**: メールサーバーのSMTPポート(25)が応答し、基本的なコマンドを受け付けるかを手動で確認する。
* **連携コマンド**: `telnet` (単体)
* **解説**: `telnet` でSMTPサーバーに接続すると、SMTPプロトコルのコマンド (`HELO`, `MAIL FROM`など) を直接入力し、サーバーの応答をリアルタイムで確認できます。
* **コマンド例**:

		```bash
		# メールサーバーの25番ポートに接続

		telnet mail.example.com 25

		# (接続後、SMTPコマンドを対話的に入力)
		# HELO mydomain.com
		# MAIL FROM: <test@mydomain.com>
		```

## オプション説明 (`telnet` の引数)

| オプション | 説明 |
| :--- | :--- |
| **接続先指定** | |
| ⭐ `<host>` | 接続先のホスト名またはIPアドレス。 |
| ⭐ `[port]` | 接続先のTCPポート番号。省略した場合は、デフォルトのtelnetポートである23番に接続します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ファイアウォール設定後、特定のTCPポートが外部から到達可能かを確認する。
* **組み合わせ**: `telnet <server_ip> <port>`
* **解説**: `telnet 192.168.1.100 80` を実行し、`Connected to ...` と表示されれば、ポートは開いています。`Connection refused` はポートが閉じており、`Connection timed out` はファイアウォールなどでブロックされていることを示唆します。
* **例**:

		```bash
		# Webサーバーのポート80が開いているか確認

		telnet web-server.example.com 80

		```

## 2. ブルーチーム視点

* **タスク**: **バナーグラビング**。不審なホストのオープンポートに接続し、どのようなサービスが応答するか調査する。
* **組み合わせ**: `telnet <suspicious_host> <port>`
* **解説**: ログに記録された不審なIPとポートの組み合わせに対し、`telnet` で接続を試みます。`SSH-2.0-OpenSSH_8.2p1` や `220 ProFTPD ...` のような応答バナーから、そこで稼働しているソフトウェアの情報を収集します。
* **例**:

		```bash
		# 不審なIPの8080番ポートを調査

		telnet 198.51.100.10 8080

		```

## 3. レッドチーム視点

* **タスク**: レガシーシステムの偵察と侵害。
* **組み合わせ**: `telnet <target_ip> 23`
* **解説**: ポートスキャンの結果、ポート23番（telnet）が開いている古い機器（ネットワーク機器、組込みシステムなど）を発見した場合、`telnet` で接続を試み、デフォルトパスワードや推測しやすいパスワードでログインを試みます。
* **例**:

		```bash
		# 古いルーターにtelnetで接続

		telnet 192.168.1.254

		```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](OS%20%20-%20Linux%20-%20troubleshooting_common_errors%20-%20Linux共通エラー対応ガイド.md) を参照。

1. **エラーメッセージ例 1**: `Connection refused`
		* **考えられる原因**: ネットワークは到達可能ですが、宛先ホストの指定ポートでリッスンしているプロセスが存在しません。
		* **解決策**: 宛先ホストでサービスが起動しているか、ポート番号が正しいかを確認してください。

2. **エラーメッセージ例 2**: `Connection timed out` or `No route to host`
		* **考えられる原因**: ネットワーク経路上の問題、またはファイアウォールによってパケットがブロックされています。
		* **解決策**: ネットワーク経路を `traceroute` や `mtr` で確認し、途中や宛先のファイアウォール設定を見直してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **平文通信**。`telnet` プロトコルは、**ユーザー名、パスワード、全ての通信内容を平文でネットワークに送信します**。
* **悪用シナリオ**: 共有ネットワーク上で `telnet` を利用すれば、悪意のある第三者によって簡単に通信が盗聴され、認証情報が窃取されてしまいます。

### GTFOBins / LOLBAS における利用例

* **機能**: **Sudo**, **Privilege Escalation**
* **解説**: もしユーザーが `sudo telnet` を実行できる権限を持っている場合、`telnet` の対話モードに入り、`!` (外部コマンド実行) を使って `sh` を実行することで、root権限のシェルを奪取できます。
* **参照**: `https://gtfobins.github.io/gtfobins/telnet/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **リモートログイン目的での `telnet` の使用を完全に禁止し、`ssh` に移行する**。`telnetd` サーバーは決して稼働させない。
* **Detection (検知)**: ネットワーク監視ツールで、ポート23への通信を監視し、アラートを発報する。`auditd` で `telnet` コマンドの実行を監視する。

## 注意点・補足

* **コマンドの終了方法**: `telnet` で接続した後、セッションの切り方が分からなくなった場合は、`Ctrl + ]` を押して `telnet>` プロンプトを出し、`quit` と入力してエンターキーを押してください。
