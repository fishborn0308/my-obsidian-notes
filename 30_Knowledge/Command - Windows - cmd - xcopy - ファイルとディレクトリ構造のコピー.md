# Command - Windows - cmd - xcopy - ファイルとディレクトリ構造のコピー

## 概要

`xcopy` (Extended Copy) は、ファイルだけでなく、ディレクトリ構造（サブディレクトリ）を再帰的にコピーするためのコマンドです。`copy` コマンドよりも高機能で、特定の属性を持つファイルのみを対象にしたり、ディレクトリ構造を維持したまま複製したりすることが可能です。

(出自: `Windows 標準搭載 - 外部コマンド (現在は robocopy への移行が推奨されている)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `copy` | ディレクトリ構造を扱えない。単一ファイルの複製用。 | 単一ファイル |
| `robocopy` | `xcopy` の進化版。ネットワーク断線への耐性、ミラーリング、ログ出力が強力。 | 大規模同期・移行 |

## よく連携されるコマンド

### シナリオ例: 特定拡張子の設定ファイルのみを構造を維持して収集 (レッドチーム視点 / Looting)

* **目的**: ターゲット端末内の各ユーザーディレクトリから、`.conf` や `.config` ファイルだけを、元のフォルダ構造を保ったまま一括で「戦利品」として取得する。
* **連携コマンド**: `xcopy`, `cd`
* **解説**: `/S` でサブフォルダを探索し、`/I` でコピー先がフォルダであることを明示します。
* **コマンド例**:
    ```cmd
    REM ユーザーディレクトリ内のすべての .config ファイルを収集用フォルダにコピー
    xcopy C:\Users\*.config C:\Temp\Loot\ /S /I /Y
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/S` | 空でないディレクトリとサブディレクトリをコピーする。 |
| ⭐ `/E` | 空のディレクトリも含めて、サブディレクトリをすべてコピーする。 |
| ⭐ `/H` | 隠しファイルやシステムファイルもコピー対象に含める。 |
| `/I` | コピー先が存在せず、複数のファイルをコピーする場合、コピー先をディレクトリとみなす。 |
| `/Y` | 既存のファイルを上書きする際、確認メッセージを表示しない。 |
| `/K` | 属性をコピーする。通常の `xcopy` は読み取り専用属性をリセットするが、これを付けると維持される。 |
| `/O` | 所有権と ACL (アクセス権) 情報をコピーする。 |
| `/X` | 監査設定をコピーする（`/O` の機能を含む）。 |
| `/T` | ファイルはコピーせず、ディレクトリ構造（ディレクトリのみ）を作成する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションの更新時に、古いバージョンのディレクトリを丸ごとバックアップする。
* **例**:
    ```cmd
    REM 空フォルダも含め、隠しファイルも含めてまるごとバックアップ
    xcopy C:\inetpub\wwwroot C:\Backup\wwwroot_v1 /E /H /I /Y
    ```

### 2. ブルーチーム視点

* **タスク**: 侵害調査において、証拠となりうるログディレクトリを、アクセス権（ACL）を維持したまま保全用ドライブにコピーする。
* **例**:
    ```cmd
    REM 権限情報(ACL)を維持してログフォルダをコピー
    xcopy C:\Windows\System32\winevt\Logs D:\Forensics\Logs /E /O /I
    ```

### 3. レッドチーム視点

* **タスク**: 偵察フェーズで、ディレクトリ構造だけを把握するために中身なしのツリーを作成する。
* **例**:
    ```cmd
    REM ファイルはコピーせず、フォルダ構成だけを複製（構造の偵察）
    xcopy C:\SensitiveData C:\Temp\Structure /T /E
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `Does <Path> specify a file name or directory name on the target? (F = file, D = directory)`
    * **考えられる原因**: コピー先に新しい名前を指定したが、その名前のディレクトリが存在しないため、ファイルかディレクトリか判断できない。
    * **解決策**: スイッチ `/I` を追加するか、末尾に `\` を付けることでディレクトリであることを明示します。

2.  **エラーメッセージ例 2**: `Access denied`
    * **考えられる原因**: システムファイルや他のプロセスが使用中のファイルをコピーしようとした。
    * **解決策**: 管理者権限で実行する。それでもダメな場合は、`robocopy` の `/B` (バックアップモード) を試すのが一般的です。

## セキュリティに関する考慮事項

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Copy` (ファイルの複製)
* **悪用シナリオ**: 攻撃者が `xcopy` を使用して、検出を避けるためにバイナリを `ADS` (代替データストリーム) 内外へ移動させたり、機密データフォルダ全体をステージング領域へ一括コピーしたりします。
* **参照**: [LOLBAS - xcopy.exe](https://lolbas-project.github.io/lolbas/Binaries/Xcopy/)

## 注意点・補足

* **非推奨**: 大規模なコピーやネットワーク経由のコピーでは、エラーハンドリングが貧弱なため、現代の Windows では **`robocopy`** を使うのがベストプラクティスです。
* **パスの長さ**: 256文字を超える長いパス（Long Path）の扱いには弱いため、深い階層のコピーには注意が必要です。
