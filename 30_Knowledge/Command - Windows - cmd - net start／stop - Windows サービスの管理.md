# Command - Windows - cmd - net start / stop - Windows サービスの管理

## 概要

`net start` および `net stop` コマンドは、Windows サービスの開始、停止、および現在実行中のサービスの一覧表示を行うために使用されます。

Windows サービスはシステムのバックグラウンドで動作するプログラムであり、このコマンドを通じて、ネットワーク機能、セキュリティ機能、アプリケーション固有の機能などを制御します。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `sc` | サービスの開始・停止だけでなく、構成の変更、削除、詳細な状態確認が可能。 | 詳細な管理 |
| `Get-Service` | PowerShell 版。オブジェクトとして取得でき、ステータスによる抽出などが容易。 | PS 環境 |
| `services.msc` | GUI ツール。視覚的に依存関係や実行アカウントを確認・設定できる。 | - |

## よく連携されるコマンド

### シナリオ例: サービスの再起動による設定反映 (運用・トラブルシューティング)

* **目的**: 設定を変更したサービス（例: Web サーバーや印刷サービス）を再起動して設定を有効にする。
* **連携コマンド**: `net stop`, `net start`
* **解説**: 一度停止してから開始することで、再起動を実現します。
* **コマンド例**:
    ```cmd
    REM 印刷スプーラーサービスを再起動する
    net stop "Print Spooler"
    net start "Print Spooler"
    ```

## スイッチ/オプション説明

**※サービスの操作（開始・停止）には「管理者権限」が必要です。**

| コマンド / オプション | 説明 |
| :--- | :--- |
| **net start** | |
| (引数なし) | 現在実行されているすべてのサービスを一覧表示する。 |
| `service` | 指定したサービスを開始する。 |
| **net stop** | |
| `service` | 指定したサービスを停止する。 |
| **その他（関連）** | |
| `net pause` | サービスを一時停止する（対応しているサービスのみ）。 |
| `net continue` | 一時停止しているサービスを再開する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーの負荷が高い際、一時的に不要なサービスを停止してリソースを確保する。
* **例**:
    ```cmd
    REM 特定のアプリケーションサービスを停止
    net stop "MyApplicationService"
    ```

### 2. ブルーチーム視点 (フォレンジック・調査)

* **タスク**: 実行中のサービス一覧を取得し、通常とは異なる名称や、予期しない実行状態のサービスがないか監査する。
* **例**:
    ```cmd
    REM 実行中のサービスをリスト化して確認
    net start
    ```

### 3. レッドチーム視点 (防御回避と永続化)

* **タスク 1: Impair Defenses (防御機能の無効化 - T1562.001)**
    * **目的**: 攻撃活動を検知されないよう、セキュリティソフトやロギングサービスを停止させる。
    * **解説**: 管理者権限（またはサービス制御権限）を奪取した後、EDR やウイルス対策ソフトのサービスを狙います。
    * **コマンド例**:
        ```cmd
        REM 標的のセキュリティに関連するサービスを停止
        net stop "WinDefend"
        net stop "MpsSvc" (Windows Firewall)
        ```

* **タスク 2: Persistence (永続化の確認 - T1543.003)**
    * **目的**: 自分が作成した悪意あるサービスが、正しく「実行中」になっているか確認する。
    * **解説**: 新しく登録したバックドアサービスが起動しているかを `net start` の一覧から目視確認します。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 管理者権限が不足している。サービスの開始・停止には特権が必要です。
    * **解決策**: 「管理者として実行」したプロンプトから実行してください。

2.  **The service name is invalid.**
    * **原因**: サービス名のスペルミス、またはスペースを含むサービス名を引用符 `"` で囲んでいない。
    * **解決策**: 正確なサービス名（表示名ではなくサービスキー名）を確認し、引用符を使用してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **サービスの悪用 (T1543.003):**
    * 攻撃者は、システムの起動時に自動実行されるサービスとしてマルウェアを登録し、`net start` でその動作を即座に開始させます。
* **Defense Evasion:**
    * セキュリティログを収集するサービス（例: `EventLog`）を停止させることで、攻撃の痕跡を残さないように工作されるリスクがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Defense Evasion` (防御回避), `Persistence` (永続化)
* **特性**: Windows 標準の管理ツールであるため、実行自体がブロックされることは稀ですが、重要サービスの停止は EDR 等で強力に監視される対象となります。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **サービス ACL の強化**: 一般ユーザーや特定のサービスアカウントが、重要サービスを停止・構成変更できないようアクセス権を厳格に制限する。
* **Detection (検知)**:
    * **イベントID 7036**: サービスの状態が変更されました。
    * **イベントID 4688**: コマンドライン引数に `stop` とセキュリティサービス名が含まれるプロセスの生成を監視する。

## 注意点・補足

* **依存関係**: 一部のサービスは他のサービスに依存しています。`net stop` を実行した際、関連する他のサービスも停止するかどうかの確認を求められる場合があります。
* **`sc` との最大の違い**: `net start / stop` は、依存関係にあるサービスをある程度自動的にハンドリングしようとしますが、`sc` はより低レイヤーで直接的な制御を行います。
