# Command - Linux - ip tunnel - IPトンネルの構成と管理

## 概要

`ip tunnel` オブジェクトは、IP パケットの中に別の IP パケットを包み込む（カプセル化）ことで、異なるネットワーク間を仮想的に直結する「トンネル」を管理します。これは、プライベートネットワーク同士をインターネット経由で接続したり、IPv4 ネットワーク上で IPv6 通信を通したりするために使用されます。

## 主要なトンネル形式（モード）

| モード | 説明 | 主な用途 |
| :--- | :--- | :--- |
| **`ipip`** | IPv4 の中に IPv4 を入れる最もシンプルな形式。 | シンプルな拠点間接続 |
| **`gre`** | 汎用的なカプセル化。IP 以外のプロトコルも運べる。 | 高機能な拠点間接続、ルーター間 |
| **`sit`** | IPv4 の中に IPv6 を入れる（IPv6 over IPv4）。 | IPv6 への移行期対応 |
| **`vti`** | IPsec と組み合わせて使われる仮想トンネル。 | セキュアな VPN 構築 |

## 主要サブコマンド

| サブコマンド | 略称 | 説明 |
| :--- | :--- | :--- |
| **`show`** | `sh`, `s` | 設定されているトンネルの一覧を表示する（デフォルト）。 |
| **`add`** | `a` | 新しいトンネルデバイスを作成する。 |
| **`del`** | `d` | 既存のトンネルを削除する。 |
| **`change`** | `c` | 既存のトンネルの設定を変更する。 |

## コマンド例と構文

### 1. トンネルの作成 (add)

作成時には「自分の出口（local）」と「相手の入り口（remote）」のグローバル IP を指定します。

```bash
# 'tun0' という名前の IPIP トンネルを作成
# 自分のIP: 203.0.113.1, 相手のIP: 198.51.100.2
sudo ip tunnel add tun0 mode ipip local 203.0.113.1 remote 198.51.100.2

# トンネルデバイスを有効化
sudo ip link set tun0 up

# トンネルにプライベート IP を割り当て
sudo ip addr add 10.0.0.1/30 dev tun0
````

### 2. トンネルの確認 (show)

Bash

```
# 設定されている全トンネルを確認
ip tunnel show
```

### 3. トンネルの削除 (del)

Bash

```
# 'tun0' を削除
sudo ip tunnel del tun0
```

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: ハイブリッドクラウドの構築。
- **解説**: オンプレミスのサーバーと AWS/GCP などのクラウド環境をプライベート IP で直接通信させたい場合、GRE や IPIP トンネルを使って仮想的な専用線を作ります。

### 2. ブルーチーム視点

- **タスク**: トンネルを悪用した隠密通信の検知。
- **解説**: ファイアウォールで IP プロトコル 4 (IPIP) や 47 (GRE) が許可されている場合、攻撃者がこれを使って内部ネットワークのトラフィックを外部へ密かに転送（トンネリング）していないか監視します。

### 3. レッドチーム視点

- **タスク**: ネットワーク制限の回避とデータ持ち出し。
- **解説**: 通常の HTTP/HTTPS 通信が厳しく監視されている環境で、許可されている特定の IP プロトコルを使ってトンネルを掘り、検知を回避しながら C2（制御）通信やデータの窃取を行います。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **暗号化の欠如**: ⭐️ **重要**。`ip tunnel` 自体には**暗号化機能がありません**。カプセル化されたパケットはインターネット上を平文で流れるため、途中で盗聴されるリスクがあります。
- **MTU の問題**: カプセル化によるヘッダーの付加（通常 20〜24 バイト程度）により、パケットサイズが MTU を超えて断片化（フラグメンテーション）が発生し、通信速度が低下することがあります。

### 対応策・緩和策 (ブルーチーム視点)

- **IPsec の併用**: 秘匿性が必要な場合は、必ず IPsec などの暗号化技術と組み合わせてトンネルを保護します。
- **フィルタリング**: 必要な相手以外とのトンネルプロトコル（GRE, IPIP 等）の通信をファイアウォールで遮断します。

## 注意点・補足

- **現代の手法**: `iproute2` の新しいバージョンでは、`ip link add name tun0 type gre ...` のように、`ip link` コマンドでトンネルを作成することが推奨されています。`ip tunnel` は古い実装への互換性のために残されています。
