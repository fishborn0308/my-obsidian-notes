# Command - Linux - more - ファイルの内容を一画面ずつ表示する（元祖）

## 概要

`more` は、テキストファイルを一画面（ページ）ごとに区切って表示する基本的なプログラムです。ファイル全体を一度に表示せず、ユーザーの入力に合わせて読み進めていくため、長いテキストの閲覧に適しています。

(出自: BSD UNIX 以来の伝統的なツール)

## ⚡️ `more` vs `less`

現代の Linux では `more` も多機能化していますが、本質的な違いは以下の通りです。

| 機能 | `more` | `less` |
| :--- | :--- | :--- |
| **スクロール** | 基本は前方のみ（一部後方も可）。 | 前後自由にスクロール可能。 |
| **読み込み** | 表示前にファイル全体を読み込む場合がある。 | 必要な部分だけ読み込む（爆速）。 |
| **検索** | 前方検索のみ。 | 前後両方の検索が可能。 |
| **終了** | ファイルの末尾に達すると自動で終了する。 | 明示的に `q` を押すまで終了しない。 |
| **格言** | (特になし) | **"less is more"** (less は more 以上である) |

## 基本的な操作（キーバインド）

`more` の実行中に使用できる主要なキーです。

| キー | アクション |
| :--- | :--- |
| ⭐️ `Space` | 1 画面分、先に進む。 |
| ⭐️ `Enter` | 1 行ずつ、先に進む。 |
| ⭐️ `b` | 1 画面分、後ろに戻る（※環境により使えない場合あり）。 |
| ⭐️ `/` | 前方検索。 |
| ⭐️ `v` | 現在の行でエディタ（vi等）を起動する。 |
| `q` | 終了する。 |

## 主要なオプション

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-d` | 画面下部に「[Press space to continue, 'q' to quit.]」といったガイドを表示。 |
| ⭐️ `-c` | 画面をスクロールさせず、上から書き換えるように表示する（描画が速い）。 |
| ⭐️ `-p` | スクロールではなく、画面をクリアしてから次ページを表示する。 |
| `-s` | 連続する空行を 1 行にまとめて表示する。 |
| `+<num>` | 指定した行数目から表示を開始する。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

* **タスク**: 非常に古い UNIX システムや、最小構成のコンテナでのログ確認。
* **解説**: `less` すらインストールされていない環境では、`more` が唯一の救いになります。基本操作が共通しているため、焦らずに使えます。

### 2. ブルーチーム視点

* **タスク**: スクリプト内の出力をページ分けする。
* **解説**: 管理用スクリプトの出力が長い場合、パイプで `more` に渡すことで、作業者が情報を読み飛ばすのを防ぎます。
* **例**:
    ```bash
    cat audit_report.txt | more
    ```

### 3. レッドチーム視点

* **タスク**: 特権環境でのエディタ起動（権限昇格の予備調査）。
* **解説**: `more` の中から `v` キーでエディタを起動できる機能を利用します。もし `sudo more` が許可されていれば、そこからエディタを介して root シェルを奪取できる可能性があります。

## セキュリティに関する考慮事項

### 脆弱性とリスク

* **Shell Escape**: `less` と同様、`more` の中から外部コマンドやエディタを呼び出せるため、高権限で実行されている場合は注意が必要です。

### 対応策・緩和策 (ブルーチーム視点)

* **環境変数の制限**: `less` と同様に `MORESECURE=1` のような直接的な制限設定は `more` にはありませんが、そもそも `less` が使える環境なら、より高機能で安全な `less` の使用を推奨します。
