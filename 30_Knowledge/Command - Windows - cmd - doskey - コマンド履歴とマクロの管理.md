# Command - Windows - cmd - doskey - コマンド履歴とマクロの管理

## 概要

`doskey` は、コマンド プロンプトで入力したコマンドの履歴を呼び出したり、編集したりする機能を提供します。また、複雑で長いコマンドに短い名前（マクロ/エイリアス）を付けて実行する機能も備えています。対話的な操作性を向上させるためのユーティリティです。

(出自: `Windows 標準搭載 - 外部コマンド (doskey.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `Get-History` | PowerShell 版。セッション内の履歴をオブジェクトとして取得。 | PS 環境 |
| `Set-Alias` | PowerShell でマクロ（エイリアス）を作成する。 | PS 環境 |
| `alias` | Linux (bash/zsh) におけるエイリアス作成コマンド。 | Linux 環境 |
| **F7 キー** | 実行履歴をグラフィカルなリスト形式で表示する（doskey の機能の一部）。 | - |

## よく連携されるコマンド

### シナリオ例: 複雑なネットワーク調査コマンドの短縮化 (運用効率化)

* **目的**: 頻繁に実行する「リスニングポートとプロセスIDの確認」を短い単語で実行できるようにする。
* **連携コマンド**: `doskey`, `netstat`, `findstr`
* **解説**: マクロ機能を使用して、特定の文字列をコマンド全体に置き換えます。
* **コマンド例**:
    ```cmd
    REM "lp" と打つだけでリスニングポートを表示するマクロを作成
    doskey lp=netstat -ano ^| findstr LISTENING
    ```



## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| ⭐ `/history` | 現在のセッションで実行された全コマンドの履歴を表示する。 |
| ⭐ `/macros` | 定義されているすべてのマクロを表示する。 |
| `/reinstall` | 新しい doskey のコピーをインストールする（履歴バッファのクリアに利用可能）。 |
| `/listsize=size` | 履歴バッファに保持するコマンド数を設定する。 |
| `macroname=[text]` | 指定した名前で新しいマクロを作成する（text を空にすると削除）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 作業ログを保存するために、そのセッションで実行した全コマンドをテキストファイルに出力する。
* **例**:
    ```cmd
    REM 実行履歴をテキストに書き出す
    doskey /history > work_log_20260222.txt
    ```

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: 侵入された可能性のある端末で、攻撃者がどのようなコマンドを実行したかを確認する。
* **解説**: 攻撃者がウィンドウを閉じる前であれば、`doskey /history` で活動内容を把握できます。
* **注意**: ただし、攻撃者が後述の履歴消去を行っている場合、ここからは何も得られません。

### 3. レッドチーム視点 (証拠隠滅と効率化)

* **タスク 1: Indicator Removal (履歴の消去 - T1070.003)**
    * **目的**: 作業終了後、自分が実行した不審なコマンド（`whoami`, `net user` 等）を管理者に辿らせないようにする。
    * **解説**: `doskey /reinstall` を実行すると、現在のコマンド履歴バッファが即座にクリアされます。
    * **コマンド例**:
        ```cmd
        REM コマンド履歴をすべて消去する
        doskey /reinstall
        ```

* **タスク 2: Command Aliasing (コマンドのすり替え - T1546.011)**
    * **目的**: 管理者が実行する正規のコマンドを、攻撃者の意図する挙動にすり替える。
    * **解説**: `doskey dir=C:\temp\malware.exe` のように設定すると、管理者が `dir` と打つたびにマルウェアが実行されるようになります。

## エラーメッセージとトラブルシューティング

1.  **マクロが動作しない**
    * **原因**: `doskey` マクロは現在の CMD プロセス内でのみ有効です。新しいウィンドウを開くと設定は消えています。
    * **解決策**: 永続化したい場合は、マクロ定義をバッチファイルに書き、レジストリの `AutoRun` に登録する必要があります。

2.  **パイプやリダイレクトがマクロ内で使えない**
    * **原因**: マクロ内で `|` や `>` を使う場合は、エスケープ記号 `^` を付けるか、特別なコード `$B` (pipe), `$G` (greater than) を使う必要があります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **コマンド実行履歴の窃取**: 
    * 攻撃者が `doskey /history` を実行することで、前のユーザーが打ったパスワード（`net use \\server /user:admin password` など）を平文で取得されるリスクがあります。
* **フィッシング/偽装**:
    * `doskey help=echo System update in progress, please wait...` のように定義し、ユーザーに偽の情報を信じ込ませる。

* **機能**: `Artifact Removal` (痕跡の消去)
* **参照**: [LOLBAS - doskey.exe](https://lolbas-project.github.io/lolbas/Binaries/Doskey/)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * **イベントID 4688**: `doskey.exe /reinstall` や `doskey /history` の実行を監視する。特に `/reinstall` は証拠隠滅の強い予兆です。
* **Prevention (予防)**:
    * 重要なサーバーでは、セッション終了時に自動的に履歴をクリアするか、あるいはより詳細なログ（PowerShell の Script Block Logging 等）を優先的に使用する。

## 注意点・補足

* **特殊記号のマッピング**:
    * `$T`: コマンドの区切り（複数のコマンドを 1 つのマクロに。`&` に相当）
    * `$1` 〜 `$9`: 引数
    * `$*`: すべての引数
* **履歴の最大数**: デフォルトでは 50 個まで保存されます。
