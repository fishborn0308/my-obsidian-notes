# Command - Linux - parted - パーティションの作成・削除・リサイズを行う

## 概要

`parted` (Partition Editor) は、ハードディスクのパーティションテーブルを操作するためのプログラムです。MBR (msdos) だけでなく GPT にも完全対応しており、2TB を超える現代のストレージ管理には欠かせません。実行には **root 権限 (sudo)** が必須です。

## 🛠 ディスク管理ツールの比較

| コマンド | 得意なこと | 苦手なこと |
| :--- | :--- | :--- |
| `fdisk` | 古いMBR形式、対話的操作。 | 2TB以上のディスク、GPT（古い版）。 |
| **`parted`** | **GPT対応、スクリプト利用、リサイズ。** | 対話モードが少し独特。 |
| `gdisk` | GPT専用。fdiskに近い操作感。 | MBR形式。 |
| `lsblk` | 現在の構成を確認する（表示のみ）。 | 変更はできない。 |

## 主要なモードとオプション

`parted` は引数なしで起動すると「対話モード」になり、引数を与えると直接実行されます。

| オプション / コマンド | 説明 |
| :--- | :--- |
| ⭐️ `-l`, `--list` | **最頻用。** すべてのデバイスのパーティション情報を表示する。 |
| ⭐️ `print` | (対話モード内) 現在選択中のディスク情報を表示する。 |
| ⭐️ `mklabel <type>` | パーティションテーブルを作成（gpt または msdos）。**※データ全消去。** |
| ⭐️ `mkpart` | 新しいパーティションを作成する。 |
| ⭐️ `resizepart` | パーティションのサイズを変更する。 |
| ⭐️ `rm <num>` | 指定した番号のパーティションを削除する。 |

## コマンド例

```bash
# システム全体のディスク構成を一覧表示
sudo parted -l

# /dev/sdb に新しい GPT ラベルを作成（一括実行例）
sudo parted /dev/sdb mklabel gpt

# /dev/sdb の先頭から 10GB のパーティションを作成
sudo parted /dev/sdb mkpart primary ext4 0% 10GB
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 新設サーバーのデータ用ディスク初期化。
- **解説**: クラウドや物理サーバーに追加した未フォーマットのディスクに対し、`parted` で GPT ラベルを貼り、単一の巨大なパーティションを作成して OS に認識させます。

### 2. ブルーチーム視点

- **タスク**: ディスクフォレンジック前の構成確認。
- **解説**: 調査対象のディスクにどのような隠しパーティションがあるか、あるいはパーティション外に未割り当ての怪しい領域（データの隠し場所）がないかを精査します。

### 3. レッドチーム視点

- **タスク**: データの永続的な破壊、または隠蔽。
- **解説**: 撤退時に `mklabel` を実行してパーティション情報を吹き飛ばし、データの復旧を困難にします（※クイックフォーマットに近い状態になるため）。また、パーティションの隙間にデータを隠す手法の検討にも使われます。

## ⚠️ 重要な注意点：取り扱い注意！

`parted` は `fdisk` と違い、対話モードであっても **「コマンドを打った瞬間にディスクに書き込まれる」** ことが多いです（fdisk のように最後に 'w' を押すまで確定しない、という猶予がない場合があります）。

**実行前には必ずバックアップを取り、対象のデバイス名（`/dev/sdb` 等）を三回確認してください。**

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **物理アクセスの脅威**: root 権限を持つ者が `parted` を使える場合、システムパーティションを破壊して OS を起動不能にしたり、別パーティションに隠された機密データを削除したりすることが容易に可能です。

### 対応策・緩和策 (ブルーチーム視点)

- **sudo ログの監視**: `parted` や `fdisk` といった破壊的変更が可能なコマンドの実行履歴を `auditd` 等で厳重に記録・監視します。
