# Command - Windows - cmd - fc - ファイルの比較と差分表示

## 概要

`fc` (File Compare) は、2 つのファイルまたはファイル セットを比較し、それらの間の違いを表示します。
テキスト ファイルの行ごとの比較に加え、実行ファイルやデータ ファイルのバイナリ比較（16進数表示）もサポートしています。特にテキスト比較では、不一致が発生した後に再び一致する箇所を探す「再同期」機能により、差分のみを効率的に抽出できます。

(出自: `Windows 標準搭載 - 外部コマンド (fc.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `comp` | バイナリが「完全に同一か」のみを高速に判定する。差分の詳細は出さない。 | 整合性判定 |
| `diff` | Linux の標準。差分をパッチ形式などで出力可能。 | (Linux環境) |
| `Compare-Object` | PowerShell 版。オブジェクト単位で比較。CSV や配列の比較に強い。 | PS環境 |
| `CertUtil -hashfile` | ハッシュ値で比較。内容は一切見ず、改ざんの有無だけを保証する。 | 改ざん検知 |

## よく連携されるコマンド

### シナリオ例: アップデート前後の設定ファイル確認 (運用・保守)

* **目的**: ソフトウェアの更新後に、設定ファイル (`.ini` や `.conf`) のどの項目が書き換わったかを特定する。
* **連携コマンド**: `fc`, `findstr`
* **解説**: `/n` スイッチで追加・変更された行番号を特定します。
* **コマンド例**:
    ```cmd
    REM 新旧の設定ファイルを比較し、行番号を表示
    fc /n config.ini.old config.ini.new
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/b` | **バイナリ比較を実行**。不一致箇所をアドレスと 16 進数で表示する。EXE や DLL に必須。 |
| ⭐ `/l` | **テキスト比較を実行** (デフォルト)。行単位で不一致を表示し、再同期を試みる。 |
| `/c` | 英字の大文字と小文字を区別せずに比較する。 |
| `/n` | テキスト比較時に、各行の**行番号を表示**する。 |
| `/t` | タブをスペースに変換せずにそのまま比較する。 |
| `/w` | 連続する空白（スペースやタブ）を 1 つの空白として圧縮して比較する。 |
| `/lb[n]` | 不一致が続く場合に、処理を打ち切る行数を指定する（デフォルトは 100 行）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: クラスター環境やミラーリング環境において、2 台のサーバーにある設定ファイルが同期されているか（あるいは意図した差異があるか）を確認する。
* **例**:
    ```cmd
    REM 空白の違いを無視して内容の「意味」が同じか比較
    fc /w \\SRV01\C$\config\web.config \\SRV02\C$\config\web.config
    ```

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: 侵害が疑われる端末で、`hosts` ファイルや `System32` 内の重要な DLL が、正規のバックアップとどう異なっているかをバイト単位で特定する。
* **例**:
    ```cmd
    REM 署名済みの DLL と現在の DLL をバイナリ比較
    fc /b C:\Windows\System32\kernel32.dll D:\Backup\kernel32.dll
    ```

### 3. レッドチーム視点 (リサーチとバイパス)

* **タスク 1: Patch Diffing (パッチ解析)**
    * **目的**: セキュリティパッチが適用される「前」と「後」の実行ファイルを比較し、どの関数やコードが修正されたかを特定することで、脆弱性の箇所（エクスプロイトの標的）を逆算する。
* **タスク 2: Finding Embedded Secrets (埋め込み情報の探索)**
    * **目的**: ソフトウェアのバージョンアップ前後で、実行ファイル内に埋め込まれたハードコード済みのパスワードや API キーの変更を追跡する。
* **タスク 3: Detection Evasion (検知回避の検証)**
    * **目的**: 難読化ツールをかけた前後のペイロードを比較し、シグネチャ（特徴的なパターン）が十分に変化しているかを確認する。

## エラーメッセージとトラブルシューティング

1.  **Resync Failed. Files are too different.**
    * **原因**: テキスト比較中、あまりにも多くの差異があり、どこから再び一致するかの手がかりが見失われた。
    * **解決策**: `/lb` スイッチでバッファを増やすか、ファイルが実行ファイルであるなら `/b`（バイナリ比較）に切り替えてください。

2.  **FC: cannot open FILE - No such file or directory**
    * **原因**: パスが間違っているか、ファイル名にスペースが含まれているのに引用符 `"` で囲んでいない。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**:
    * 攻撃者が `fc` を使って、暗号化前のバックアップと暗号化後のファイルを比較することで、暗号アルゴリズムの挙動や、特定のパターン（既知の平文攻撃など）を推測する手がかりにする可能性があります。

### LOLBAS における利用例

* **機能**: `Reconnaissance` (偵察)
* **特性**: 外部ツールを持ち込まずに、システムの微細な変更（改ざんの痕跡やパッチの有無）を詳細に調査できるため、ステルス性の高い調査活動に利用されます。

## 注意点・補足

* **バイナリ vs テキスト**: 実行ファイル（.exe, .dll, .sys）に対してデフォルトのテキスト比較を行うと、文字化けした大量のデータが表示されます。バイナリデータには必ず `/b` を指定してください。
* **Errorlevel**:
    * `0`: 2 つのファイルは同一。
    * `1`: 2 つのファイルは異なる。
    * `2`: ファイルが見つからない、またはエラー。
    * この戻り値を利用して、バッチファイル内で `if errorlevel 1 echo 変更検知!` のような自動監視が可能です。
