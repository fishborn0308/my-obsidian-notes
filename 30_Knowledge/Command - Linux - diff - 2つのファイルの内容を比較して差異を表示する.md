# Command - Linux - diff - 2つのファイルの内容を比較して差異を表示する

## 概要
`diff` は、2つのファイルを1行ずつ比較し、それらの違いをリストアップします。どの行を変更、削除、あるいは追加すれば2つのファイルが同じになるかという「編集指示」を出力するのが本来の役割です。
(出自: `diffutils` パッケージに含まれる)



## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `cmp` | バイト単位で比較。最初の異なる箇所のみ報告。バイナリ比較に強い。 | バイナリの同一性確認 |
| `sdiff` | 2つのファイルを左右に並べて（Side-by-side）表示する。 | 直感的な比較 |
| `vimdiff` | Vimのエディタ上で差異を色分け表示し、編集も可能。 | 対話的な修正 |
| `git diff` | Git管理下のファイルの変更点を確認する。 | 開発中のコード管理 |

## よく連携されるコマンド
### シナリオ例: 修正用パッチファイルの作成と適用 (運用・開発視点)
* **目的**: 設定ファイルの変更箇所を「パッチ」として保存し、別のサーバーにも同じ変更を適用する。
* **連携コマンド**: `patch`
* **解説**: `diff` の出力をファイルに保存し、`patch` コマンドで読み込ませます。
* **コマンド例**:
    ```bash
    # 1. 差異をパッチファイルとして保存 (Unified形式)
    diff -u old_config.conf new_config.conf > change.patch
    # 2. (別の場所で) パッチを適用
    patch old_config.conf < change.patch
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐️ `-u`, `--unified` | **Unified形式**で出力。前後3行のコンテキストを含み、最も一般的で読みやすい。 |
| ⭐️ `-y`, `--side-by-side` | 画面を左右に分割して比較を表示する。 |
| `-c`, `--context` | Context形式で出力（古い形式のパッチ等で使用）。 |
| **比較オプション** | |
| ⭐️ `-r`, `--recursive` | ディレクトリ内のファイルを再帰的に比較する。 |
| ⭐️ `-q`, `--brief` | 詳細な差異は表示せず、ファイルが「異なるか、同じか」だけを表示する。 |
| `-i`, `--ignore-case` | 英字の大文字小文字を区別せずに比較する。 |
| `-w`, `--ignore-all-space` | 空白の数の違いを無視して比較する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 本番環境とバックアップ設定ファイルの差異確認。
* **組み合わせ**: `diff -u`
* **解説**: 誤って書き換えてしまった箇所がないか、デフォルト設定からどこを変えたかを一目で把握します。
* **例**:
    ```bash
    # 現在の設定とバックアップを比較
    diff -u /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak
    ```

### 2. ブルーチーム視点
* **タスク**: システムファイルの改ざん調査（整合性確認）。
* **組み合わせ**: `diff -rq`
* **解説**: 正常な時のバックアップディレクトリと現在の `/etc` や `/bin` を再帰的に比較し、変更されたファイルや追加されたファイル（バックドア等）を特定します。
* **例**:
    ```bash
    # 正常なバックアップと現在の /etc を比較して、異なるファイル名だけを表示
    sudo diff -rq /mnt/safe_backup/etc /etc
    ```

### 3. レッドチーム視点
* **タスク**: 侵害後の内部偵察（以前の攻撃者による変更や、機密情報の探索）。
* **組み合わせ**: `diff`
* **解説**: `/etc/passwd` のバックアップ（`/etc/passwd-`）と現在のファイルを比較し、新しく追加された不審なユーザーがないか、あるいは自身の作業痕跡が残っていないかを確認します。
* **例**:
    ```bash
    # パスワードファイルの変更履歴を確認
    diff /etc/passwd- /etc/passwd
    ```

## エラーメッセージとトラブルシューティング

1.  **出力に `^M` が含まれる**:
    * **考えられる原因**: 比較対象の一方が Windows 形式（CRLF）、もう一方が Linux 形式（LF）の改行コードを使用している。
    * **解決策**: `dos2unix` で改行コードを統一してから比較するか、`diff --strip-trailing-cr` を使用する。

2.  **バイナリファイルの比較で「Binary files ... differ」としか出ない**:
    * **考えられる原因**: `diff` はテキスト用なので、バイナリの中身を解析できない。
    * **解決策**: 詳細なバイナリ比較が必要なら `cmp -l` や `hexdump` を併用する。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: それ自体に脆弱性は少ないですが、**機密情報の露出**に注意が必要です。
* **悪用シナリオ**: 攻撃者が `sudo diff` を利用できる場合、本来アクセスできない特権ファイル（`/etc/shadow` 等）と、自分が用意した「既知のハッシュが書かれたファイル」を比較させることで、エラーメッセージや差異の有無からパスワードを推測したり、内容を実質的に読み取ったりすることが可能です。

### GTFOBins における利用例
`diff` が `sudo` で実行可能な場合、任意のファイルを読み取ることができます。
* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo diff を使って /etc/shadow の内容を表示させる（/dev/null と比較）
    sudo diff /etc/shadow /dev/null
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/diff/`

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: 一般ユーザーに `sudo diff` を許可しない。
* **Detection (検知)**: `/etc/shadow` やプライベートキーなどの重要ファイルに対して `diff` が実行されているログを `auditd` で監視する。

## 注意点・補足
* **戻り値**: 
    * `0`: 差異なし
    * `1`: 差異あり
    * `2`: エラー発生
    この特性を利用して、シェルスクリプト内の条件分岐によく使われます。
