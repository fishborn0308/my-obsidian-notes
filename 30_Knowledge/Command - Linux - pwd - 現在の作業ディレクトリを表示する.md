# Command - Linux - pwd - 現在の作業ディレクトリを表示する

## 概要

`pwd` は、ユーザーが現在作業しているディレクトリのフルパス（ルート `/` からの絶対パス）を標準出力に書き出します。

## 🛠 関連コマンド

| コマンド | 役割 |
| :--- | :--- |
| `cd` | 作業ディレクトリを移動する。 |
| `ls` | 現在のディレクトリ内のファイルを表示する。 |
| `basename` | パスの末尾（ファイル名や現在のフォルダ名のみ）を取り出す。 |
| `dirname` | パスの親ディレクトリ部分を取り出す。 |

## 主要なオプション

普段は引数なしで使われますが、シンボリックリンク（ショートカット）を扱っている場合に以下のオプションが重要になります。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-L`, `--logical` | **デフォルト。** シンボリックリンクを辿った後の「論理的な」パスを表示する。 |
| ⭐️ `-P`, `--physical` | シンボリックリンクを解決し、**物理的な実体**のパスを表示する。 |

## コマンド例

```bash
# 基本的な使用方法
pwd
# 出力例: /home/user/documents

# シンボリックリンクの実体を確認する
# 例: /var/www/html が /mnt/storage/data へのリンクだった場合
cd /var/www/html
pwd -P
# 出力例: /mnt/storage/data
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 設定ファイル内のパス指定。
- **解説**: 構成管理ツール（Ansible 等）やスクリプトを作成する際、相対パスで書くと実行場所によってエラーになることがあります。`pwd` を使って絶対パスを確認し、確実なパスを指定する習慣をつけます。

### 2. ブルーチーム視点

- **タスク**: ログ解析やフォレンジック。
- **解説**: 不審な実行ファイルが見つかった際、そのプロセスの作業ディレクトリ（CWD）を特定することは極めて重要です。`pwd` そのものではありませんが、`/proc/<PID>/cwd` を確認することで、攻撃者がどのディレクトリで活動していたかを特定します。

### 3. レッドチーム視点

- **タスク**: 内部情報の収集（Environment Enumeration）。
- **解説**: 侵入に成功した直後、自分がどのディレクトリに放り込まれたのか、そこが Web 公開領域なのか、あるいは隔離された一時フォルダなのかを `pwd` で即座に判断し、次の行動（権限昇格や横展開）の足がかりにします。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: シェルのプロンプトに常に `pwd` の結果（フルパス）を表示させている場合、スクリーンショットや画面共有を通じて、ユーザー名や隠しディレクトリの名前といった内部構造が漏洩するリスクがあります。

### 対応策・緩和策

- **プロンプトのカスタマイズ**: `PS1` 環境変数を調整し、プロンプトにはパスの末尾だけを表示する（`\W`）か、必要時のみ `pwd` を叩く運用にします。
