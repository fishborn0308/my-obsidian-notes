# Command  - Linux - wc - ファイルの行数、単語数、バイト数を数える

## 概要

`wc` (word count) は、指定したファイルまたは標準入力から受け取ったテキストデータの**行数 (lines)**、**単語数 (words)**、そして**バイト数 (bytes)** を数えるためのコマンドです。「このログファイルは何行あるか？」「コマンドの出力結果は何件あるか？」といった問いに答えるための、シンプルで非常に便利なユーティリティです。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`ls -l`** | ファイルの**バイト数**を表示しますが、行数や単語数は表示しません。 | - |
| **`grep -c`** | パターンに**一致した行の数**を数えます。`wc -l` は**全ての行**の数を数えます。 | - |

## よく連携されるコマンド

`wc` は、他のコマンドの出力をパイプ (`|`) で受け取り、その結果が何行あるかを数えるために頻繁に連携されます。

### シナリオ例: 実行中のプロセス数を数える (インフラ構築・運用視点)

* **目的**: システムで実行中の `nginx` プロセスの数を正確に数える。
* **連携コマンド**: `ps`, `grep`
* **解説**: `ps aux` で全プロセスのリストを出力し、`grep` で `nginx` を含む行をフィルタリングします。最後にその結果を `wc -l` に渡すことで、プロセスの数をカウントできます。
* **コマンド例**:

    ```bash
    # 実行中の nginx プロセスの数を数える
    ps aux | grep '[n]ginx' | wc -l
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **カウント対象指定** | |
| ⭐ `-l`, `--lines` | **行数**のみをカウントして表示します。**最もよく使われるオプション**です。 |
| `-w`, `--words` | **単語数**のみをカウントして表示します。 |
| `-c`, `--bytes` | **バイト数**のみをカウントして表示します。 |
| `-m`, `--chars` | **文字数**のみをカウントして表示します。マルチバイト文字を含むファイルでは `-c` と結果が異なります。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 特定のディレクトリにあるファイルの数を数える。
* **組み合わせ**: `ls -1 | wc -l`
* **解説**: `ls -1` でファイル名を1行に1つずつ表示させ、その行数を `wc -l` で数えることで、ディレクトリ内のアイテム数をカウントします。
* **例**:

    ```bash
    # /etc/systemd/system/ ディレクトリにあるユニットファイルの数を数える
    ls -1 /etc/systemd/system/ | wc -l
    ```

### 2. ブルーチーム視点

* **タスク**: ログファイル内の特定のイベント発生回数を定量的に把握する。
* **組み合わせ**: `grep <pattern> <logfile> | wc -l`
* **解説**: 認証ログに、ブルートフォース攻撃の兆候であるログイン失敗 (`Failed password`) が何回記録されているかを数えます。この回数が短時間に急増していれば、攻撃を受けていると判断できます。
* **例**:

    ```bash
    # 直近の認証ログから "Failed password" が含まれる行数をカウント
    grep "Failed password" /var/log/auth.log | wc -l
    ```

### 3. レッドチーム視点

* **タスク**: パスワード攻撃に使用するワードリストの規模を確認する。
* **組み合わせ**: `wc -l <wordlist>`
* **解説**: ブルートフォース攻撃を仕掛ける前に、使用する辞書ファイル（ワードリスト）に何件のパスワード候補が含まれているかを `wc -l` で確認します。これにより、攻撃にかかるおおよその時間を推測します。
* **例**:

    ```bash
    # rockyou.txt の行数を数える
    wc -l /usr/share/wordlists/rockyou.txt
    ```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `wc` は読み取り専用のデータ処理ツールであり、直接的なセキュリティリスクは低いです。
* **悪用シナリオ**: `wc` は、セキュリティの**攻撃**と**防御**の両方の文脈で利用されます。
  * **防御側**: ログエントリの数を数えることで、ブルートフォース攻撃などの異常を定量的に検知します。
  * **攻撃側**: 偵察段階で収集したデータ（ユーザーリストなど）の規模を把握し、攻撃計画を立てるために利用します。

### GTFOBins / LOLBAS における利用例

`wc` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: -
* **Detection (検知)**: `wc` は非常にありふれたコマンドのため、単体での検知は困難です。しかし、`grep /etc/shadow | wc -l` のような、機密ファイルへのアクセスと組み合わせた一連のコマンド実行を `auditd` で監視することは有効です。

## 注意点・補足

* **複数ファイルの指定**: `wc -l file1.txt file2.txt` のように複数のファイルを引数に取ることができます。その場合、各ファイルごとの行数と、最後に行数の**合計**が表示されます。
* **バイト数 vs 文字数**: 日本語のようなマルチバイト文字を含むファイルを扱う場合、`-c` (バイト数) と `-m` (文字数) の結果は異なります。
