# Command - Windows - cmd - ftp - ファイル転送プロトコル クライアント

## 概要

`ftp` は、FTP (File Transfer Protocol) を使用して、リモート サーバーとの間でファイルを転送するための対話型コマンドライン ツールです。

古くから存在するプロトコルであるため、暗号化（SSL/TLS）をサポートしていない「プレーンテキスト」での通信が基本となります。現代のインフラではセキュリティ上の理由から制限されることが多いですが、レガシーシステムの保守や、特定の制限環境下での「足場」として利用されます。

(出自: `Windows 標準搭載 - 外部コマンド (ftp.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `scp` / `sftp` | SSH を使用した暗号化転送。現代の標準。 | セキュア転送 |
| `curl` | 多彩なプロトコルに対応したデータ転送ツール（Windows 10 以降標準）。 | Web/API 連携 |
| `bitsadmin` | Windows のバックグラウンド転送。ステルス性が高い。 | ペイロード取得 |
| `Invoke-WebRequest` | PowerShell 版。HTTP/HTTPS 経由のダウンロード。 | PS 環境 |

## よく連携されるコマンド

### シナリオ例: 非対話型（スクリプト実行）による自動バックアップ

* **目的**: ユーザーの入力を待たずに、あらかじめ用意した指示書（テキストファイル）に従ってファイルを自動転送する。
* **連携コマンド**: `echo` (指示書作成), `ftp -s`
* **解説**: `-s` スイッチを使用して、ログインから転送、ログアウトまでの一連の動作を自動化します。
* **コマンド例**:
    ```cmd
    REM 転送用スクリプトの作成
    echo open ftp.example.com > ftp_script.txt
    echo myuser >> ftp_script.txt
    echo mypassword >> ftp_script.txt
    echo put backup.zip >> ftp_script.txt
    echo quit >> ftp_script.txt

    REM スクリプトを指定して実行
    ftp -s:ftp_script.txt
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `-s:filename` | 指定したテキストファイル内のコマンドを順に実行する（自動化に必須）。 |
| ⭐ `-i` | 複数ファイルを転送する際、対話型の確認（プロンプト）をオフにする。 |
| `-n` | 最初の接続時に自動ログインを試行しない。 |
| `-v` | サーバーの応答をすべて表示する（詳細出力）。 |
| `-g` | ファイル名のワイルドカード（グロブ）を無効にする。 |
| `-A` | 匿名（Anonymous）としてログインを試行する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: セキュリティが分離されたセグメントにある古い計測機器やサーバーから、定期的にログファイルを回収する。
* **解説**: 複雑なエージェントを導入できないレガシー環境において、標準機能だけで完結できる点が強みです。

### 2. ブルーチーム視点 (トラフィック分析)

* **タスク**: ネットワーク内で大量の FTP 通信が発生していないか監視し、内部不正によるデータ持ち出しを検知する。
* **重要ポイント**:
    * **平文の窃取**: パケットキャプチャ（Wireshark 等）で、ユーザー名とパスワードがそのまま流れていることを確認し、リスクを報告する。
    * **Event ID 4688**: `ftp.exe` の起動、特に `-s` スイッチで外部ファイルを参照している挙動を精査する。

### 3. レッドチーム視点 (ファイル取得と持ち出し - T1048)

* **タスク 1: Living off the Land (標準ツールの悪用)**
    * **目的**: ブラウザや PowerShell が監視されている環境で、`ftp.exe` を使って C2 サーバーから攻撃ツールをダウンロードする。
* **タスク 2: Data Exfiltration (データの外部送出)**
    * **目的**: 奪取した機密データを、攻撃者の用意した FTP サーバーへ「匿名（Anonymous）」または「奪取した資格情報」でアップロードする。
* **タスク 3: Shell Escape (対話モードからのコマンド実行)**
    * **解説**: FTP プロンプト内で `!` を入力することで、一時的にローカルのシェル（CMD）に戻ることができます（環境による）。

## エラーメッセージとトラブルシューティング

1.  **Connection closed by remote host.**
    * **原因**: サーバー側で接続が拒否された、またはファイアウォール（ポート 21）で遮断されている。
    * **解決策**: ネットワーク経路とサーバー側のホワイトリストを確認する。

2.  **500 Invalid PORT Command (アクティブモードの問題)**
    * **原因**: クライアント側のファイアウォールが原因で、サーバーからのデータ接続（ポート 20）を受け取れない。
    * **解決策**: Windows 標準の `ftp.exe` は「パッシブモード（PASV）」への切り替えが困難な場合があります。ネットワーク設定を調整するか、他のクライアントを検討します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Cleartext Protocol**: 通信全体が暗号化されないため、同一ネットワーク内の第三者がログイン情報や転送データの内容を容易に傍受可能です。
* **Anonymous FTP**: 適切に設定されていない FTP サーバーは、誰でも（Anonymous ユーザーとして）ファイルを読み書きできる「公開ゴミ捨て場」になり、マルウェアの配布元として悪用されることがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Download`, `Execute`
* **参照**: [LOLBAS - ftp.exe](https://lolbas-project.github.io/lolbas/Binaries/Ftp/)

### 対応策・緩和策 (ブルーチーム視点)

* **プロトコルの禁止**: 原則として FTP サービスを無効化し、SFTP や HTTPS への移行を推奨します。
* **ログ監視**: `-s` スイッチで読み込まれるスクリプトファイル（一時ファイル）の内容を監視し、不審な外部ドメインへの接続がないかチェックします。

## 注意点・補足

* **バイナリモード**: 画像や実行ファイルを転送する際は、FTP プロンプトで必ず `binary` コマンドを打ってください（デフォルトの `ascii` モードではファイルが壊れます）。
* **対話型コマンド**: プロンプト内では `ls`, `cd`, `get`, `put`, `mget`, `mput`, `quit` など、Unix ライクなコマンドが使用可能です。
