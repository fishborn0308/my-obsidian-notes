# Command - Windows - cmd - net user - ユーザーアカウントの管理

## 概要

`net user` コマンドは、ローカルコンピュータまたは Active Directory ドメイン上のユーザーアカウントの追加、削除、および情報の照会を行うために使用されます。
アカウントの有効/無効の切り替えやパスワードリセットなど、アカウント管理におけるほぼ全ての基本操作をカバーします。

(出自: `Windows 標準搭載 - net.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `Get-LocalUser` | PowerShell版。オブジェクトとして返るため、条件抽出（例：無効アカウントのみ）が容易。 | PS環境 |
| `Get-ADUser` | ADモジュール。AD環境でより詳細なプロパティ（最終ログイン時間等）を取得可能。 | AD管理ツール導入済環境 |
| `lusrmgr.msc` | GUIによる管理。直感的だがリモートやスクリプトには不向き。 | - |

## よく連携されるコマンド

### シナリオ例: 特定ドメインユーザーの詳細情報取得 (レッドチーム視点)

* **目的**: ドメイン内に存在する特定のユーザー（例: 管理者候補）のアカウント詳細（所属グループや最終ログインなど）を確認する。
* **連携コマンド**: `net user`, `findstr`
* **解説**: `/domain` スイッチを付けることで、ドメインコントローラへ問い合わせを行います。
* **コマンド例**:
    ```cmd
    REM ドメインユーザー "j.smith" の詳細情報を表示
    net user j.smith /domain
    ```



## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本表示** | |
| (引数なし) | ローカルの全ユーザーアカウントを一覧表示する。 |
| `username` | 特定のユーザーアカウントの詳細情報を表示する。 |
| ⭐ `/domain` | **【重要】** ローカルではなく Active Directory ドメインに対して操作を行う。 |
| **アカウント操作** | |
| `/add` | 新しいユーザーアカウントを作成する。 |
| `/delete` | 既存のユーザーアカウントを削除する。 |
| `password` | 指定したユーザーのパスワードを設定・変更する。 `*` を使うと対話的に入力可能。 |
| **プロパティ設定** | |
| `/active:yes|no` | アカウントを有効 (`yes`) または無効 (`no`) にする。 |
| `/expires:date|never` | アカウントの有効期限を設定する。 |
| `/passwordchg:yes|no` | ユーザーによるパスワード変更を許可 (`yes`) または禁止 (`no`) する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 踏み台サーバーなどの構築時に、パスワード期限のないローカル作業アカウントを作成する。
* **例**:
    ```cmd
    REM 新規ユーザーをパスワード指定・期限なしで作成
    net user ops_admin P@ssw0rd123! /add /expires:never
    ```

### 2. ブルーチーム視点

* **タスク**: 退職者や不審なアカウントがまだ「有効」になっていないか、ポリシーに違反していないか調査する。
* **例**:
    ```cmd
    REM 特定のユーザーがいつ最後にパスワードを変更したかを確認
    net user guest_user | findstr "最終パスワード変更"
    ```

### 3. レッドチーム視点 (偵察と権限昇格)

* **タスク 1: Domain User Enumeration (ドメインユーザーの列挙)**
    * **目的**: ターゲットドメイン内の全有効ユーザーリストを取得し、その後の「パスワードスプレー攻撃」や「フィッシング」のターゲットリストを作成する。
    * **解説**: 低権限シェルを取得した後、最初に行う偵察（Reconnaissance）の一つです。AD環境では、デフォルトで一般ユーザーにもこの読み取り権限が与えられていることが多いです。
    * **コマンド例**:
        ```cmd
        REM 全ドメインユーザーを表示（出力が多い場合はファイルへリダイレクト）
        net user /domain
        ```
    * **実務ログ例**:
        ```text
        \\DC01.EXAMPLE.COM のユーザー アカウント
        -------------------------------------------------------------------------------
        Administrator            Guest                    krbtgt                   
        j.doe                    s.smith                  sql_service              
        web_admin                vpn_user                 backup_svc               
        コマンドは正常に終了しました。
        ```

* **タスク 2: ターゲットアカウントの詳細分析 (High-Value Target の特定)**
    * **目的**: 権限昇格の標的となるアカウント（管理者、サービスアカウントなど）を絞り込む。
    * **解説**: 列挙したリストから、`admin` や `svc` といったキーワードを含むアカウントの詳細を確認します。「最終ログイン」や「パスワードの最終設定日」を見ることで、そのアカウントが現在も活発に使われているか（＝パスワードが有効か）を判断できます。
    * **コマンド例**:
        ```cmd
        REM サービスアカウント "sql_service" の詳細と所属グループを確認
        net user sql_service /domain
        ```
    * **実務ログ例**:
        ```text
        ユーザー名                           sql_service
        フル ネーム                         SQL Service Account
        コメント                             Password is: SQLServer2025!  <-- 設定ミスによる情報の露出
        アカウント有効                       Yes
        最終パスワード変更                   2025/12/01 10:20:15
        所属しているグローバル グループ       *Domain Admins  <-- 非常に高い権限を持っていることを確認
        ```

* **タスク 3: パスワードポリシーの推測とスプレー攻撃の準備**
    * **目的**: `net user` の結果から組織のパスワード命名規則を推測し、効率的な攻撃を行う。
    * **解説**: アカウントの「最終パスワード変更日」が全ユーザーで同時期であれば、定期的な一括変更ポリシーがあることが分かります。また、`/domain` を付けた際に応答するドメインコントローラの名称から、ネットワークの地理的・組織的な構造を推測します。



* **タスク 4: 権限昇格後の永続化 (Persistence)**
    * **目的**: 管理者権限を奪取した後、将来の再侵入のための裏口を作成する。
    * **解説**: 目立たない名前（例：`DefaultAccount_` や `Svc_HealthCheck`）で新しいユーザーを作成し、パスワード期限を無期限（`/expires:never`）に設定します。
    * **コマンド例**:
        ```cmd
        REM パスワード期限なしのバックドアアカウントを作成
        net user backup_admin P@ssw0rd123! /add /expires:never /active:yes
        ```
    * **実務ログ例**:
        ```text
        コマンドは正常に終了しました。
        REM この後、net localgroup administrators backup_admin /add を組み合わせて特権を付与する。
        ```

* **タスク 5: パスワードハッシュの窃取（前段階）**
    * **目的**: どのユーザーがローカルにログオンしているかを確認し、ハッシュ取得の優先順位をつける。
    * **解説**: `net user` でローカルユーザーを確認し、管理者アカウントが複数存在する場合、それぞれのディレクトリを探索して、機密情報が含まれていないか、あるいはメモリ上にハッシュが残っていないか（Mimikatz等の使用検討）を判断します。

## エラーメッセージとトラブルシューティング

1.  **System error 5 has occurred. Access is denied.**
    * **原因**: 管理者権限（昇格）がない。
    * **解決策**: 「管理者として実行」したプロンプトで実行してください。

2.  **The password does not meet the password policy requirements.**
    * **原因**: 設定しようとしたパスワードが、複雑さの要件や最小文字数を満たしていない。
    * **解決策**: より複雑な（大文字小文字記号数字を含む）パスワードを試すか、`net accounts` でポリシーを確認してください。

## 環境変数と設定ファイル

* **SAM (Security Accounts Manager)**: ローカルユーザー情報は `C:\Windows\System32\config\SAM` に格納されています。
* **NTDS.dit**: ドメインユーザー情報はドメインコントローラのこのデータベースファイルに格納されています。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **アカウントの列挙 (Account Enumeration - T1087.001/.002):**
    * **低権限からの情報収集**: デフォルト設定の Active Directory 環境では、一般ユーザー権限であっても `net user /domain` を実行してドメイン内の全ユーザーリストを取得できる場合が多いです。これは攻撃者にとって、パスワードスプレー攻撃（共通の簡単なパスワードを全ユーザーに試す攻撃）の標的リストを作成するための「宝の山」となります。
    * **機密情報の露出**: アカウントの「コメント」欄や「説明」欄に、以前のパスワードや、サーバーの用途、緊急連絡先などの機密情報が不用意に記載されているケースがあり、偵察フェーズでの重要なヒントになります。

* **認証情報の露出 (Credential Exposure):**
    * **コマンドライン履歴の汚染**: `net user username password /add` のように実行すると、引数に含まれるパスワードがコマンドプロンプトの履歴（`doskey /history`）や、EDR/SIEM が収集するプロセス監視ログ（Event ID 4688 等）に平文で記録されます。
    * ** shoulder hacking / 画面共有**: リアルタイムで画面を共有している際や、背後から覗き見られた際にパスワードが露出します。

* **永続化 (Persistence - T1136.001):**
    * **バックドアアカウントの作成**: 攻撃者は管理者権限を奪取した後、将来の再侵入のために `net user` で目立たない名前のユーザー（例: `svc_admin` や `DefaultAccount_`）を作成し、`net localgroup administrators` で特権を付与します。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Reconnaissance` (偵察), `Persistence` (永続化)
* **特性**: `net.exe` は MS 署名済みの正当な管理ツールであるため、実行自体をブロックすることは困難です。攻撃者は「検出されにくい方法」として、外部ツールを使わずにこのコマンドを多用します。
* **参照**: [LOLBAS - net.exe](https://lolbas-project.github.io/lolbas/Binaries/Net/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **パスワード入力の隠蔽**: パスワードを設定・変更する際は、`net user username *` のようにアスタリスクを使用し、対話的に入力を求めるように徹底します。これによりログへの平文記録を防げます。
    * **最小権限の原則**: 一般ユーザーによる不必要なドメイン情報の列挙を制限するため、AD のアクセス制御リスト (ACL) を見直し、機密性の高いプロパティへの読み取り権限を絞り込みます。

* **Detection (検知)**:
    * **重要なイベント ID の監視**: 
        * **4720**: ユーザー アカウントが作成されました。
        * **4722**: ユーザー アカウントが有効になりました（無効だったアカウントの有効化は攻撃の兆候）。
        * **4724**: ユーザーによるパスワード変更の試行。
        * **4738**: ユーザー アカウントが変更されました（特にパスワード設定や有効期限の変更）。
    * **コマンドライン引数の分析**: `/add`, `/domain`, `/active:yes` といったキーワードが含まれるプロセス生成イベントを SIEM で相関分析し、正規の管理者業務以外の時間帯や端末からの実行をアラート化します。
