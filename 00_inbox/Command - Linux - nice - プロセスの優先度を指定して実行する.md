# Command - Linux - nice - プロセスの優先度を指定して実行する

## 概要
`nice` は、プログラムを起動する際にそのプロセスの「スケジューリング優先度（Niceness）」を設定します。Linux カーネルのスケジューラはこの値を参照し、CPU 時間をどのプロセスに多く割り当てるかを決定します。
名前の通り、値を大きくするほど他のプロセスに対して「ナイス（謙虚）」になり、CPU を譲るようになります。



## 🛠 `nice` vs `renice`
| コマンド | 役割 | タイミング |
| :--- | :--- | :--- |
| **`nice`** | 優先度を決めて**新しく起動**する。 | 実行時 |
| `renice` | **既に動いている**プロセスの優先度を変える。 | 実行中 |

## 📊 優先度（Nice値）のルール
Nice 値は **-20 から 19** までの範囲で指定します。

* **19 (最弱)**: 最も謙虚。他のプロセスが暇な時だけ CPU を使う。
* **0 (標準)**: デフォルトの優先度。
* **-20 (最強)**: 最も傲慢。最優先で CPU 時間を奪い取る。

> [!IMPORTANT]
> **一般ユーザーは優先度を下げる（値を大きくする）ことしかできません。** 優先度を上げる（値をマイナスにする）には **root 権限 (sudo)** が必須です。

## 主要なオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-n <value>` | 指定した Nice 値を加算して実行する（デフォルトは +10）。 |
| ⭐️ `--help` | ヘルプを表示する。 |

## コマンド例
```bash
# 非常に重い圧縮作業を、他の邪魔をしないように「謙虚に」実行
nice -n 19 tar czf backup.tar.gz /large/data

# 重要な計算処理を「最優先」で実行 (sudoが必要)
sudo nice -n -20 ./critical_compute_job
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: バックアップやログ集計などのバックグラウンド処理。
    
- **解説**: サービス稼働中のサーバーで重い処理を走らせる際、ウェブサーバー（nginx等）のレスポンスを落とさないよう、バッチ処理に `nice -n 19` を指定するのは運用の鉄則です。
    

### 2. ブルーチーム視点

- **タスク**: セキュリティスキャンやウイルス検査の実行。
    
- **解説**: 脆弱性スキャナやフルスキャンは CPU を激しく消費します。これが原因で本番サービスがダウンする（意図しない自己 DoS）のを防ぐため、スキャンプロセスを `nice` で実行するよう設定を確認します。
    

### 3. レッドチーム視点

- **タスク**: ステルス性の維持（リソース監視の回避）。
    
- **解説**: 侵入後、パスワードクラックやデータマイニングを行う際、CPU 使用率が 100% に張り付くと管理者に気づかれます。あえて `nice -n 19` で実行し、システムの負荷を上げすぎずに「細く長く」処理を続けることで検知を遅らせます。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **リソース枯渇 (DoS)**: 悪意あるユーザー（または設定ミスのスクリプト）が非常に高い優先度 (`-20`) で無限ループなどを実行すると、他の重要なプロセス（SSH ログインなど）が CPU を割り当てられず、システムが操作不能になる恐れがあります。
    

### 対応策・緩和策 (ブルーチーム視点)

- **`limits.conf` の活用**: `/etc/security/limits.conf` で、一般ユーザーが設定できる Nice 値の範囲を制限できます。
    
- **監視**: `top` や `htop` コマンドで `NI` 列を監視し、異常に高い優先度で動いている不審なプロセスがないかチェックします。
    

## 注意点・補足

- **I/O 優先度**: `nice` はあくまで CPU の優先度です。ディスク読み書きの優先度を調整したい場合は、**`ionice`** コマンドを併用します。
