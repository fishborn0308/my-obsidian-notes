# OS  - Linux - passwd_shadow_group_gshadow - `/etc/passwd`, `shadow`, `group`, `gshadow` - ユーザー・グループ情報ファイル

## 概要

Linuxシステムにおけるユーザーとグループの情報は、主に4つのテキストファイルによって管理されています。これらのファイルは、ユーザー認証、アクセス権の制御、アカウント管理の基礎となる、極めて重要なデータベースです。

* **`/etc/passwd`**: ユーザーアカウントの基本情報を格納します。
* **`/etc/shadow`**: 暗号化されたユーザーパスワードと、パスワードの有効期限ポリシーを格納します。
* **`/etc/group`**: グループの基本情報を格納します。
* **`/etc/gshadow`**: 暗号化されたグループパスワードと、グループ管理者情報を格納します。

**⚠️重要**: これらのファイルを直接編集することは非常に危険です。管理には `useradd`, `passwd` などの専用コマンドを使用してください。

(出自: `shadow-utils` または `passwd` パッケージの管理対象)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`getent`** | `getent passwd <user>` のように、NSS (Name Service Switch) を通じてユーザーやグループの情報を表示します。LDAPなどが構成されている環境でも、ローカルファイルとネットワーク上の情報を透過的に扱えます。 | `getent` (情報表示) |
| **`vipw`, `vigr`**| `/etc/passwd` と `/etc/group` を安全に編集するための専用コマンド。ファイルをロックし、破損を防ぎます。 | `vipw`, `vigr` (手動編集時) |

## ファイルフォーマットの詳細

各ファイルはコロン (`:`) で区切られたフィールドで構成されています。

### ⭐ `/etc/passwd` (全ユーザーが読み取り可能)

`username:x:UID:GID:GECOS:home_directory:login_shell`

* `username`: ログイン名
* `x`: パスワードが `/etc/shadow` にあることを示す
* `UID`: ユーザーID (`0` は `root`)
* `GID`: プライマリグループID
* `GECOS`: ユーザーのフルネームなど
* `home_directory`: ホームディレクトリ
* `login_shell`: ログインシェル (`/sbin/nologin` はログイン不可)

---

### ⭐ `/etc/shadow` (rootのみ読み取り可能)

`username:password_hash:last_change:min:max:warn:inactive:expire`

* `password_hash`: `$`で区切られたハッシュ化パスワード
* `last_change`: パスワード最終変更日
* `min`/`max`: パスワードの最小/最大有効日数

---

### `/etc/group` (全ユーザーが読み取り可能)

`group_name:x:GID:members_list`

* `group_name`: グループ名
* `x`: グループパスワードが `/etc/gshadow` にあることを示す
* `GID`: グループID
* `members_list`: 所属するユーザーのリスト（カンマ区切り）

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ユーザーがログインできない原因を調査する。
* **組み合わせ**: `cat /etc/passwd`
* **解説**: `/etc/passwd` を確認し、対象ユーザーのログインシェルが `/sbin/nologin` になっていないか、ホームディレクトリのパスが正しいかなどを確認します。
* **例**:

```bash
# ユーザー 'user1' の定義を確認
grep '^user1' /etc/passwd
```

### 2. ブルーチーム視点

* **タスク**: 不審なアカウントが作成されていないか監査する。
* **組み合わせ**: `cat /etc/passwd`, `cat /etc/shadow`
* **解説**: **ファイル整合性監視 (FIM)** の最重要対象。`/etc/passwd` に見慣れないユーザー（特にUIDが `0` のユーザー）が追加されていないか、`/etc/shadow` のパスワードハッシュが意図せず変更されていないかなどをベースラインと比較して監査します。
* **例**:

```bash
# UIDが0のユーザーアカウントを全てリストアップする
awk -F: '($3 == 0) { print $1 }' /etc/passwd
```

### 3. レッドチーム視点

* **タスク**: 権限昇格と永続化の足がかりを探す。
* **組み合わせ**: `cat /etc/passwd`, `cat /etc/shadow`
* **解説**:
	* **偵察**: `cat /etc/passwd` でシステムに存在するユーザーをリストアップします。
	* **権限昇格**: `/etc/shadow` が何らかのミスで読み取り可能になっていれば、パスワードハッシュを窃取し、オフラインでのクラッキングを試みます。root権限を奪取した場合、`/etc/passwd` に `hacker:x:0:0...` のような行を追加して、自分専用のrootアカウントを作成します。
* **例**:

```bash
# 全ユーザーのリストを取得
cat /etc/passwd
```

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **不適切なパーミッション設定**が最大の脆弱性です。
* **悪用シナリオ**: `/etc/shadow` が一般ユーザーから読み取り可能になっている場合、攻撃者は全ユーザーのパスワードハッシュを窃取できます。また、これらのファイルが書き込み可能になっている場合、攻撃者はアカウントを追加・変更して、システムを完全に掌握できます。

### GTFOBins / LOLBAS における利用例

これらのファイルは攻撃対象であり、GTFOBinsにはリストされていません。しかし、GTFOBinsにリストされている多くのコマンド (`vi`, `sed`, `chown`など) は、`sudo` 権限と組み合わせることで、これらのファイルを不正に読み書きし権限昇格を行うために利用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: ファイルのパーミッションを常に標準的な安全な値 (`passwd`/`group`は`644`, `shadow`/`gshadow`は`000`または`640`) に維持する。
* **Detection (検知)**: FIMツール (`AIDE`など) を使い、これらのファイルへの全ての変更を監視し、意図しない変更があれば即座にアラートを発報するように設定する。

## 注意点・補足

* **NSS (Name Service Switch)**: LDAPやSSSDなどを利用している環境では、ユーザーやグループの情報はネットワークから取得されることもあります。その場合、これらのローカルファイルだけを見てもシステム全体のユーザー情報を把握できないことがあります。挙動は `/etc/nsswitch.conf` ファイルで制御されます。
