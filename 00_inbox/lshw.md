# `lshw` - ハードウェア構成を詳細に一覧表示する

## 概要

`lshw` (list hardware) は、マシンのハードウェア構成に関する非常に詳細な情報を抽出・表示するためのコマンドラインユーティリティです。CPU、メモリ、マザーボード、PCIデバイス、USBデバイス、ネットワークカード、ディスクなど、システムに搭載されている各コンポーネントのモデル名、ベンダー、ドライバー、設定などを網羅的にリストアップします。

この情報は、`/proc` や `/sys` ファイルシステム、DMIテーブルなど、複数の情報源から収集されます。**正確で詳細な情報を得るためには、root権限で実行する必要があります。**

(出自: `lshw` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`dmidecode`** | BIOS/UEFIの**DMIテーブル**から情報を読み取ることに特化しています。`lshw` はDMI情報に加えて、カーネルが認識している現在のデバイス状態など、より広範な情報源からデータを収集します。 | `lshw` (全体像) |
| **`lspci`, `lsusb`**| それぞれ**PCIデバイス**と**USBデバイス**の情報表示に特化しています。`lshw` はこれらの情報を包含する、システム全体のハードウェア情報を提供します。 | - |

## よく連携されるコマンド

`lshw` の出力は詳細で長大になることが多いため、`grep` や `less` と連携して、目的の情報を効率的に探します。

### シナリオ例: ネットワークカードのドライバと設定を確認する (インフラ構築・運用視点)

* **目的**: サーバーに搭載されているネットワークカードの詳細情報（モデル名、ドライバ、MACアドレスなど）をまとめて確認する。
* **連携コマンド**: `grep`
* **解説**: `lshw -C network` でネットワークデバイスに絞り込んだ情報を、`grep` を使ってさらにフィルタリングすることもできますが、`-C network` 自体で十分詳細な情報が得られます。
* **コマンド例**:

```bash
# ネットワークカードに関する情報のみを抽出して表示
sudo lshw -C network
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **表示形式** | |
| ⭐ `-short` | ハードウェアのプロファイルを、簡潔なサマリー形式で表示します。 |
| `-businfo`| PCIアドレスやUSBパスなど、バス情報を表示します。 |
| `-html`, `-xml`, `-json`| 出力形式をそれぞれHTML, XML, JSON形式に変換します。 |
| **フィルタリング** | |
| ⭐ `-C <class>`, `--class <class>`| 特定の**クラス**（カテゴリ）のハードウェア情報のみを表示します。（例: `system`, `processor`, `memory`, `network`, `storage`, `disk`） |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: システムのハードウェアインベントリ（棚卸し情報）を素早く確認する。
* **組み合わせ**: `sudo lshw -short`
* **解説**: CPUのモデル、メモリの総量、ディスクの数と種類、ネットワークカードのモデルなどを一覧で把握でき、資産管理や構成管理の基礎情報として利用します。
* **例**:

```bash
# システム全体のハードウェア概要をリスト表示
sudo lshw -short

# ネットワークカードの詳細情報だけを確認する
sudo lshw -C network
```

### 2. ブルーチーム視点

* **タスク**: 調査対象ホストのハードウェアベースラインを詳細に記録する。
* **組み合わせ**: `sudo lshw -json > baseline.json`
* **解説**: インシデント発生時に、対象システムの正確なハードウェアプロファイル（メーカー、モデル、ファームウェアバージョン）をJSON形式で保存します。このベースラインと比較し、意図しないハードウェアの追加や変更がないかを確認します。
* **例**:

```bash
# 現在のハードウェア構成をJSONファイルとして保存
sudo lshw -json > hardware-$(hostname)-$(date +%F).json
```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、標的システムの物理的・仮想的な環境を詳細に把握する。
* **組み合わせ**: `sudo lshw`
* **解説**: `lshw` の出力から、`product` や `vendor` に `VMware`, `KVM`, `QEMU` などの文字列があれば、仮想マシン上であることが確定し、VMエスケープ脆弱性の利用を検討します。また、NICやマザーボードの正確なモデルを特定し、そのデバイスドライバに存在する脆弱性（ローカルでの権限昇格など）を探します。
* **例**:

```bash
# 仮想環境かどうかを判定
sudo lshw -C system
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **警告メッセージ**: `WARNING: you should run this program as super-user.`
    * **考えられる原因**: 一般ユーザーで `lshw` を実行しました。
    * **解決策**: `lshw` は `root` 権限で実行することで、DMIテーブルなどからより完全で詳細な情報を取得できます。`sudo lshw` として実行してください。

2. **現象**: **デバイス情報に `UNCLAIMED` と表示される。**
    * **考えられる原因**: カーネルがそのハードウェアを認識しているものの、対応する適切なデバイスドライバがロードされていない状態です。
    * **解決策**: デバイスのベンダーとモデルを特定し、対応するドライバをインストールまたは `modprobe` でロードする必要があります。

## 環境変数と設定ファイル

* `lshw` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `lshw` 自体は読み取り専用ツールですが、その出力が**情報漏洩**に繋がります。
* **悪用シナリオ**: `lshw` は、システムのハードウェア、ファームウェア、シリアル番号に関する極めて詳細な情報を出力します。これらの情報は、攻撃者がハードウェアやファームウェア固有の脆弱性を探し出し、標的を絞った高度な攻撃を計画するための重要な偵察情報となります。

### GTFOBins / LOLBAS における利用例

`lshw` は情報表示のみを行うコマンドであり、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、不要なユーザーに `sudo` での `lshw` 実行権限を与えない。
* **Detection (検知)**: `auditd` などで `lshw` コマンドの実行を監視し、不審な偵察活動がないかを確認する。

## 注意点・補足

* **インストール**: `lshw` は多くのディストリビューションで標準インストールされていません。使用するには、`sudo apt install lshw` や `sudo yum install lshw` などでパッケージをインストールする必要があります。
* **出力形式の活用**: `-html` や `-json` オプションを使うと、結果を機械処理しやすい形式で出力できます。資産管理システムと連携し、サーバーインベントリを自動的に収集・更新するようなスクリプトで非常に役立ちます。
