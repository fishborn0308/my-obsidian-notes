# Command - Windows - cmd - cls - 画面の消去

## 概要

`cls` (Clear Screen) は、コマンドプロンプト ウィンドウに表示されているすべてのテキストを消去し、カーソルを左上（ホームポジション）に移動させるためのコマンドです。画面上の表示は消えますが、それまでに設定した環境変数や実行したコマンドの履歴がリセットされることはありません。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `clear` (Linux/PS) | Linux/Unix や PowerShell における画面消去コマンド。 | 各環境の標準 |
| `exit` | ウィンドウそのものを閉じる。 | 終了時 |
| **スクロール** | マウス等で上にスクロールすれば過去の履歴が見えるが、`cls` はバッファを（見かけ上）クリアする。 | - |

## よく連携されるコマンド

### シナリオ例: ループ処理におけるステータスのリアルタイム更新

* **目的**: バッチファイルなどで、常に最新のステータスのみを固定位置に表示させたい場合。
* **連携コマンド**: `cls`, `timeout`, `echo`
* **解説**: 画面を一度消してから新しい情報を出力することで、更新されているように見せます。
* **コマンド例**:
    ```cmd
    @echo off
    :loop
    cls
    echo 現在時刻: %TIME%
    echo サーバー監視中...
    timeout /t 5 > nul
    goto loop
    ```



## スイッチ/オプション説明

`cls` にはオプションや引数は存在しません。

| 使用法 | 説明 |
| :--- | :--- |
| `cls` | 画面全体を消去する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: デモンストレーションやプレゼンテーションの際、前のコマンドの出力が残っていると紛らわしいため、新しい手順に移る前に画面を清掃する。

### 2. ブルーチーム視点

* **タスク**: フォレンジック調査の過程で、機密性の高いパスワードや個人情報が画面に表示されたまま離席する必要がある場合、`cls` を実行して画面から情報を物理的に隠蔽する（ただし、スクロールバックバッファの設定によっては戻せてしまうため注意が必要）。

### 3. レッドチーム視点 (隠蔽と整理)

* **タスク 1: 画面上の痕跡消去 (Cleaning Traces)**
    * **目的**: ターゲット端末を操作中に、誰かが画面を覗き見たり、戻ってきたりした際に、不審なコマンドの実行結果（例: `whoami`, `net user /domain` 等）を見られないようにする。
    * **解説**: 作業の区切りごとに `cls` を挟むことで、「今何をしているか」を即座に隠します。
* **タスク 2: ロギング時の整形**
    * **目的**: 画面キャプチャをエビデンスとして保存する際、不要な情報を排除してターゲットとなる出力だけを際立たせる。

## 注意点・補足

* **バッファの保持**: 多くの現代的なターミナルエミュレータでは、`cls` を実行しても完全にデータがメモリから消えるわけではなく、スクロールアップすれば過去の内容が見える場合があります。セキュリティ上の「完全な消去」を目的とする場合は、ウィンドウを閉じるのが最も確実です。
* **履歴の保持**: `doskey /history` や矢印キー（↑）によるコマンド履歴は、`cls` を実行しても維持されます。
