# Command - Linux - lshw - ハードウェアの構成情報を詳細に表示する

## 概要
`lshw` は、マシンのハードウェア構成に関する詳細な情報を抽出する小さなツールです。メモリ構成、ファームウェアのバージョン、メインボードの構成、CPU の種類や速度、キャッシュ構成、バス速度などを報告します。
(出自: 単体の `lshw` パッケージ。多くのディストリビューションで標準、または要インストール)



## 🛠 ハードウェア調査ツールの使い分け
| コマンド | 調査対象 | 特徴 |
| :--- | :--- | :--- |
| **`lshw`** | **ハードウェア全般** | 木構造で全てを網羅する「全部入り」。 |
| `lspci` | PCI デバイス | グラボ、NIC、RAIDカード等に特化。 |
| `lsusb` | USB デバイス | 外付けデバイス、キーボード等の調査。 |
| `lsblk` | ブロックデバイス | ディスクとパーティションの構造を確認。 |
| `dmidecode` | BIOS/DMI 情報 | ハードウェアのシリアル番号や物理スロット情報。 |

## 主要なオプション
※詳細な情報を得るには、ほぼ全てのケースで **root 権限 (sudo)** が必要です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-short` | **推奨。** ハードウェアのパス、クラス、詳細を一行ずつ簡潔に表示。 |
| ⭐️ `-html` | ハードウェア構成を HTML 形式で出力（ブラウザで見るのに最適）。 |
| ⭐️ `-json` | JSON 形式で出力（プログラムや自動化スクリプトでの利用に最適）。 |
| ⭐️ `-C <CLASS>` | 特定のクラス（`memory`, `network`, `cpu`, `storage`等）に絞って表示。 |
| `-sanitize` | シリアル番号などの機密情報を隠して出力。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: サーバーの資産管理（インベントリ）作成。
* **解説**: 物理サーバーが何枚のメモリを積んでいて、空きスロットがいくつあるか、NIC の正確な型番は何かを、データセンターへ行かずに把握します。
* **例**:
    ```bash
    # ネットワークデバイスの型番とドライバ情報を特定
    sudo lshw -C network
    ```

### 2. ブルーチーム視点
* **タスク**: 物理的なバックドア（不審なデバイス）の検知。
* **解説**: 定期的に `lshw` のスナップショットを取り、身に覚えのない PCI デバイスや USB コントローラが「生えて」いないか比較・監視します。

### 3. レッドチーム視点
* **タスク**: 環境のフィンガープリント（特定）。
* **解説**: 侵入した環境が「本物の物理マシン」か「ハニーポット（罠の仮想環境）」か、あるいは「クラウド上のインスタンス」かを、マザーボードのベンダー名や仮想ドライバの有無から判断します。
* **例**:
    ```bash
    # システム全体を簡潔に表示し、仮想化の兆候（VMware, QEMU等）を探す
    sudo lshw -short
    ```

## エラーメッセージとトラブルシューティング

1.  **「lshw: command not found」**:
    * **原因**: パッケージがインストールされていない。
    * **解決策**: `sudo apt install lshw` (Debian/Ubuntu) や `sudo dnf install lshw` (RHEL/Fedora) でインストール。

2.  **「WARNING: you should run this program as super-user」**:
    * **状況**: 一般ユーザーで実行したため、多くの情報が `(unclaimed)` や `(unknown)` になっている。
    * **解決策**: `sudo lshw` を使用する。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出 (Information Disclosure)**: 攻撃者にマシンの正確な型番やシリアル番号が知られると、そのハードウェア固有の脆弱性（例: メモリの Rowhammer 脆弱性や特定の CPU バグ）を突いた攻撃を計画されるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **権限管理**: 一般ユーザーによる詳細なハードウェア情報の閲覧を制限したい場合は、バイナリの実行権限を絞ることも検討しますが、通常は `sudo` の制限で十分です。
* **ログ管理**: ハードウェア構成の変更（NIC の交換やメモリ増設）はシステムの重要なイベントとして記録し、意図しない物理的な変更がないかチェックします。

## 注意点・補足
* **出力の長さ**: オプションなしの `lshw` は非常に長いテキストを出力します。まずは `-short` で全体を把握し、必要な箇所を `-C` で深掘りするのが効率的です。
