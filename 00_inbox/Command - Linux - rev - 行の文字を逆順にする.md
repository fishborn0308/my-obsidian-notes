# Command - Linux - rev - 行の文字を逆順にする

## 概要
`rev` は、指定されたファイル（または標準入力）の各行について、文字の順番を左右反転させて出力します。単体では「鏡文字」を作るだけの遊びのように見えますが、他のコマンドと組み合わせることで、通常のテキスト処理コマンドでは難しい操作を容易にします。



## 🛠 `rev` の真価：リバース・エンジニアリング的思考
多くのテキスト処理コマンド（`cut` など）は「左から数える」のは得意ですが、「右から（末尾から）数える」のが苦手です。そこで以下のプロセスが使われます。
1. **`rev`** で左右をひっくり返す。
2. 他のコマンドで「左から」処理する（実質的に末尾を処理していることになる）。
3. もう一度 **`rev`** で元に戻す。

## 主要なオプション
`rev` は非常にシンプルな設計のため、オプションはほとんどありません。

| オプション | 説明 |
| :--- | :--- |
| `-V`, `--version` | バージョン情報を表示する。 |
| `-h`, `--help` | ヘルプを表示する。 |

## コマンド例
```bash
# 文字列を反転させる
echo "Linux" | rev
# 出力: xuniL

# 【実戦】ファイルパスの「拡張子だけ」を取り出す
# 1. パスを反転 -> 2. 最初の "." までを切り出す -> 3. 再度反転
echo "/home/user/data.archive.tar.gz" | rev | cut -d. -f1 | rev
# 出力: gz
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: ログファイル内の「末尾の識別子」に基づいた並べ替えや抽出。
    
- **解説**: ログの行末にランダムな ID やステータスコードが付与されている場合、`rev` を使って一時的に行頭に持ってくることで、標準的な `sort` や `uniq` をかけやすくします。
    

### 2. ブルーチーム視点

- **タスク**: 難読化された通信やスクリプトの解析。
    
- **解説**: 攻撃者は検知を避けるために、あえて文字列を逆順にしてプログラム内に隠すことがあります。不自然な文字列を見つけた際、`rev` を通してみることで、本来のコマンド（例: `atob` や `eval` 等）が浮かび上がることがあります。
    

### 3. レッドチーム視点

- **タスク**: ペイロードの簡易的な難読化。
    
- **解説**: セキュリティ製品のシグネチャ検知（特定のキーワードのスキャン）を回避するために、スクリプトの一部を逆順にして保存しておき、実行直前に `rev` で戻して実行する、といった「原始的だが有効な」回避策の検討に使用します。

## 💡 プロの知恵：鏡の国のテキスト

`rev` は文字を反転させますが、行の順番（1行目と最終行）を入れ替えるのは **`tac`** (cat の逆) コマンドです。

- 左右反転 ↔️ **`rev`**
    
- 上下反転 ↕️ **`tac`**
    
    これらを組み合わせると、テキストを完全に「真後ろから」読むことができます。
