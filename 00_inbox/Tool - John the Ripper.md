# John the Ripper
John the Ripper（JtR）は、多様なハッシュ形式に対応したオープンソースのパスワード解析ツールであり、ペネトレーションテストにおけるオフライン攻撃の要となるツールです。以下に主要な機能を体系的にまとめます。

## 1. 準備：ハッシュの抽出と変換（The *2john ユーティリティ）

ハッシュを John が解析可能な形式に変換するための補助ツール群です。

- **Linuxハッシュの統合 (unshadow):**
    
    - `/etc/passwd` と `/etc/shadow` を統合し、JtRが解読できる形式を作成します。
        
    - コマンド: `unshadow /etc/passwd /etc/shadow > hashes.txt`
        
- **SSH秘密鍵 (ssh2john):**
    
    - SSHの `id_rsa` などの秘密鍵からパスフレーズを抽出します。
        
    - コマンド: `/usr/share/john/ssh2john.py id_rsa > id_rsa.hash`
        
- **KeePassデータベース (keepass2john):**
    
    - `.kdbx` ファイルからマスターパスワードのハッシュを抽出します。
        
- **その他:** Windowsの `pwdump` 形式、Kerberosの `unafs` 形式などもサポートされています。
    

## 2. 解析モードの種類

JtRは効率性を高めるために、複数の解析モードを段階的に適用します。

|**解析モード**|**動作ロジック**|**ペンテストにおける価値**|
|---|---|---|
|**Single Crack**|ユーザー名、フルネーム（GECOS）、ホームディレクトリ名に基づいた解析を実施。|初期偵察で驚くほど効果的。ユーザー名に関連した脆弱なパスワードを即座に特定。|
|**Wordlist**|辞書ファイルを使用し、マングリングルールを適用して解析。|既知の漏洩リスト（rockyou.txt等）を用いた高速な攻撃が可能。|
|**Incremental**|全ての組み合わせを試行するブルートフォース攻撃。|最終手段。文字数や種類が少ない場合に有効だが、膨大な時間を要する。|
|**External**|外部のスクリプトやプログラムを呼び出して解析。|標準で対応していない独自のハッシュ関数や解析ロジックを試行可能。|

## 3. マングリングルールとカスタムルールの構築

辞書攻撃の成功率を飛躍的に高める「ルール」設定は、JtRの最も強力な機能の一つです。

- **基本コマンド例:**
    
    - `$` : 文字を末尾に追加（例: `$1` は末尾に "1" を追加）。
        
    - `^` : 文字を先頭に追加（例: `^!` は先頭に "!" を追加）。
        
    - `l` : 全て小文字化、`u` : 全て大文字化。
        
    - `c` : 先頭のみ大文字化。
        
- **カスタムルールのテスト:**
    
    - 設定ファイル（`john.conf`）に記述したルールが単語にどう作用するかを確認できます。
        
    - コマンド: `john --rules=Custom --wordlist=list.txt --stdout`。
        

## 4. 実行と結果の管理

- **解析の実行:**
    
    - 自動モード: `john <hash_file>`
        
    - 形式指定: `john --format=nt <hash_file>`（NTLMの場合など）。
        
- **結果の表示:**
    
    - 解析済みパスワードは `john.pot` ファイルに保存されますが、`--show` オプションで見やすく表示されます。
        
    - コマンド: `john --show <hash_file>`
        
- **特定のターゲットに限定:**
    
    - `--users=0` や `--users=root` で、root（UID 0）のアカウントのみを対象に解析結果を確認できます。
        

## 5. 高度なパフォーマンス制御

- **分散処理:**
    
    - `--node=1/2` や `--node=2/2` を使用して、複数のノードやプロセスに負荷を分散させることが可能です。
        
- **セッション管理:**
    
    - `--session=name` でセッション名を指定し、中断した解析を後から再開できます。
        

JtRは単なるクラッカーではなく、Jumbo版であれば数百種類のハッシュ形式（WordPress、MySQL、LDAP、ネットワークトラフィックキャプチャ等）をサポートする、極めて多機能なツールです。
