# Command - Linux - bind - Readline のキーバインドと変数の設定・管理

## 概要
`bind` は、Bash 等のシェルが入力待ちの際に使用するライブラリ「Readline」の挙動を制御します。キーボードの特定のキーを押した際に実行される機能（関数）を定義したり、マクロを設定したりすることができます。
(出自: `Bash / Zsh 等のシェル内蔵機能`)



## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `stty` | 端末（TTY）の低レベルな設定（通信速度、Ctrl+C の動作等）を管理。 | 端末全体の制御 |
| `bind` | シェル上での編集機能（Ctrl+A で行頭へ移動等）の高レベルな設定を管理。 | キー操作のカスタマイズ |
| `alias` | コマンドの文字列を置き換える。`bind` は「キー入力そのもの」に機能を割り当てる。 | コマンド名の短縮 |

## よく連携されるコマンド
### シナリオ例: 作業効率化のためのカスタムマクロ (運用視点)
* **目的**: 頻繁に打つコマンドや定型句を、1つのキーに割り当てる。
* **連携コマンド**: `grep`, `cat` (設定の確認)
* **解説**: 特定のキー（例: F1キー）を押したときに、特定の文字列が入力されるようにします。
* **コマンド例**:
    ```bash
    # F12キーを押したときに " | grep -i " と自動入力されるように設定
    bind '"\e[24~":" | grep -i "'
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **表示・確認** | |
| ⭐️ `-p` | 現在有効なすべてのキーバインドと関数名を、再利用可能な形式で表示。 |
| ⭐️ `-P` | 現在のキーバインドと関数名を、人間が読みやすい形式で一覧表示。 |
| `-v` | Readline の変数（設定値）と値を表示。 |
| **設定・削除** | |
| `-f <filename>` | 指定したファイル（通常は `.inputrc` 形式）からキーバインドを読み込む。 |
| `-q <function>` | 指定した関数（例: `beginning-of-line`）がどのキーに割り当てられているか確認。 |
| `-u <function>` | 指定した関数に割り当てられているキーをすべて解除する。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 大量ログ確認時の検索効率向上。
* **組み合わせ**: `history-search-backward` の割り当て
* **解説**: デフォルトでは上下キーは単なる履歴移動ですが、これを「入力途中の文字列で履歴を検索する」機能に変更します。
* **例**:
    ```bash
    # 上下キーで「コマンドの先頭一致」による履歴検索を可能にする
    bind '"\e[A":history-search-backward'
    bind '"\e[B":history-search-forward'
    ```

### 2. ブルーチーム視点
* **タスク**: 巧妙に隠されたバックドアや永続化の痕跡調査。
* **組み合わせ**: `bind -p` および `.inputrc` の確認
* **解説**: 攻撃者が特定のキー（例: Enterキー）に、自身のスクリプトを実行するマクロを仕込んでいないか調査します。
* **例**:
    ```bash
    # 怪しいマクロが定義されていないかチェック
    bind -p | grep '"'
    # ユーザーのホームディレクトリにある設定ファイルを確認
    cat ~/.inputrc
    ```

### 3. レッドチーム視点
* **タスク**: ポストエクスプロイト（侵入後）の永続化とコマンドハイジャック。
* **組み合わせ**: `bind` によるマクロ定義
* **解説**: `\C-m`（Enterキー）の入力をフックし、本来のコマンド実行の直前に自身のペイロードを実行させる手法です。
* **例**:
    ```bash
    # Enterキーが押されるたびに、バックグラウンドで隠しスクリプトを実行させる
    # (\C-m は Enter, \C-e は行末移動, ; はコマンド区切り)
    bind '"\C-m": "\C-e; /tmp/.hidden_script.sh &\C-m"'
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `bash: bind: warning: line edit not enabled`
    * **考えられる原因**: シェルが対話モードでない（スクリプト内など）、または Readline が無効になっている。
    * **解決策**: 対話型シェル (`bash -i`) で実行するか、設定が反映されるべき環境を確認する。

2.  **キーコードが分からない**:
    * **解決策**: `Ctrl+V` を押した後に、調べたいキー（例: F1）を押すと、そのキーのエスケープシーケンス（例: `^[OP`）が表示されます。設定時は `^[ ` を `\e` に置き換えて記述します。

## 環境変数と設定ファイル

### 関連する設定ファイル
* **`~/.inputrc`**: ユーザー個別の Readline 設定ファイル。`bind` コマンドでの設定を永続化したい場合はここに記述します。
* **`/etc/inputrc`**: システム全体の Readline 設定ファイル。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: それ自体に脆弱性はありませんが、**「入力の信頼性」を損なう**悪用が可能です。
* **悪用シナリオ**: 
    * `Ctrl+L` (画面クリア) を `rm -rf / --no-preserve-root` にバインドし、管理者が画面を掃除しようとした瞬間にシステムを破壊する。
    * ユーザーがパスワードを入力する場面で、特定のキーにデータを外部へ送るマクロを仕込む。

### GTFOBins における利用例
`bind` はシェル内蔵コマンド（builtin）であるため、バイナリとしての GTFOBins 登録はありませんが、**Escaping Restricted Shells (rbash回避)** において、`bind` を使って新しいシェルを起動する機能をキーに割り当てる手法が知られています。

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: システム全体の設定ファイル `/etc/inputrc` を root 所有にし、書き込み権限を制限する。
* **Detection (検知)**: ユーザーの `.bash_history` に `bind` コマンドが含まれていないか、あるいは `.inputrc` に予期せぬマクロが追加されていないかを監視する。

## 注意点・補足
* **DNSのBINDとの混同**: 前述の通り、ネットワーク構築において「BIND」は DNS サーバを指しますが、コマンドラインで `bind` と打つとこのシェル機能が動きます。DNSサーバの管理には通常 `named` や `rndc` コマンドを使用します。
