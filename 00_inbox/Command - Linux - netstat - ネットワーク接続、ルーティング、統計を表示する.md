# Command - Linux - netstat - ネットワーク接続、ルーティング、統計を表示する

## 概要
`netstat` は、ホストが現在どのようなネットワーク接続を持っているか、どのポートで待ち受けているか、カーネル内にどのようなルーティングテーブルがあるかを表示します。トラブルシューティングからセキュリティ監査まで、ネットワークに関わるあらゆる場面で活躍します。
(※現代の推奨は `iproute2` パッケージの `ss` コマンドですが、`netstat` も現役です)



## 🛠 `netstat` vs `ss`
| 項目 | `netstat` | `ss` |
| :--- | :--- | :--- |
| **速度** | 普通 ( `/proc` を走査) | **爆速** (カーネルから直接情報を取得) |
| **情報の詳細度** | 標準的 | 非常に詳細 (TCP内部情報など) |
| **普及率** | 非常に高い (レガシー環境含む) | 現代の Linux 標準 |
| **推奨度** | 互換性重視なら | **2026年の標準** |

## 主要なオプション
これらは組み合わせて使うのが一般的です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-t`, `--tcp` | TCP 接続のみを表示する。 |
| ⭐️ `-u`, `--udp` | UDP 接続のみを表示する。 |
| ⭐️ `-l`, `--listening` | 接続待ち（Listen）状態のソケットのみを表示する。 |
| ⭐️ `-p`, `--program` | その接続を使っている**プロセス ID と名前**を表示する (sudo推奨)。 |
| ⭐️ `-n`, `--numeric` | ホスト名やサービス名に変換せず、**数字（IP、ポート）のまま**表示する。 |
| ⭐️ `-a`, `--all` | 全ての状態（接続中 + 待ち受け）を表示する。 |
| `-r`, `--route` | ルーティングテーブル（経路情報）を表示する。 |
| `-i`, `--interfaces` | ネットワークインターフェースの統計を表示する。 |

## 💡 最強の鉄板コンボ: `netstat -tunlp`
「このサーバー、今何が動いてるの？」という時に必ず叩く呪文です。
```bash
sudo netstat -tunlp
````

- **t**cp / **u**dp で、**n**umeric(数字)表記し、**l**isten(待ち受け)中の、**p**rogram(プロセス)を表示せよ、という意味になります。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: アプリケーションが起動しているのに繋がらない問題の調査。
    
- **解説**: アプリが意図したポートで `LISTEN` しているか、あるいは `127.0.0.1`（自分自身のみ）にバインドされていて外部から拒否されていないかを確認します。
    
- **例**:
    
    Bash
    
    ```
    # 80番ポートが正しく LISTEN されているか確認
    netstat -an | grep :80
    ```
    

### 2. ブルーチーム視点

- **タスク**: 不審なバックドアの検知。
    
- **解説**: 覚えのない高番号ポート（例: 4444, 31337）が外部の不審な IP と `ESTABLISHED`（接続確立）になっていないか、あるいは `nc` や `bash` がポートを開いていないかを監視します。
    

### 3. レッドチーム視点

- **タスク**: ローカル限定サービスの特定。
    
- **解説**: 外部からは見えないが、サーバー内部（localhost）だけで動作している管理パネルやデータベース、脆弱な旧バージョンサービスを特定し、ポートフォワーディング等の次なる攻撃に繋げます。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **情報の露出**: 動作中のサービスやバージョン、接続元の IP アドレスが丸見えになるため、攻撃者にとっての「攻略マップ」になります。
    

### 対応策・緩和策 (ブルーチーム視点)

- **不要なサービスの停止**: `netstat` で見つかった「使っていないが空いているポート」は、サービスを停止するかファイアウォールで即座に閉じます。
    
- **`ss` への移行**: 攻撃者は `/proc` の走査を監視することがありますが、`ss` はより隠密に（カーネル空間の netlink を通じて）情報を取得するため、フォレンジック的な観点からも `ss` の習得を推奨します。
    

## 注意点・補足

- **名前解決の罠**: `-n` を付けないと、IP をホスト名に逆引きしようとして表示が極端に遅くなることがあります。基本は `-n` を付けるのがプロの作法です。
