# ネットワークスキャン実務統合ガイド

## 1. ネットワークスイープ (Host Discovery)

ネットワーク内の生存ホスト（IP）を特定します。

| **手法** | **コマンド例**                                                                        | **特徴**                                              |
| -------- | ------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| **Nmap** | `nmap -sn 192.168.1.0/24`  | 高速・高機能。ARP/ICMPを自動使い分け。|
| **Bash** | `for i in {1..254}; do ping -c 1 -W 1 192.168.1.$i >/dev/null && echo "$i up" & done` | 追加ツール不要。バックグラウンド処理（`&`）で高速化。 |

Nmapでのネットワークスイープ (Host Discovery)は他にも`-PS<ポート番号>`を使う方法もある

`nmap -PS445 192.168.1.0/24`

## 2. ポートスキャン (Port Scanning)

どのサービスが待ち受けているかを確認します。

|**手法**|**コマンド例**|**特徴**|
|---|---|---|
|**Nmap**|`sudo nmap -sS -p 1-1024 192.168.1.50`|ステルス性が高い（SYNスキャン）。|
|**Netcat**|`nc -zv 192.168.1.50 22 80 443`|単一ポートの疎通確認に最適。|
|**Bash**|`(echo > /dev/tcp/192.168.1.50/80) && echo "Open"`|`/dev/tcp` デバイスを利用した極限の軽量調査。|

よく使われる Nmapコマンド

- フルポートスキャン（全部入り）

	```
	nmap -sT -sV -sC -p- -T4 -oA full_scan_results <Target_IP_or_Range>
	```

- 特定のプロトコル特化

	```
	nmap -p*<Protcols>* --script=*<Protcols> --open -oA <Protcols>_scan_result
	```

## 3. OSフィンガープリンティング (OS Detection)

ターゲットのOSを特定します。

- **能動的手法 (Nmap):**
    
    `sudo nmap -O 192.168.1.50`
    
    - TCP/IPスタックの応答を解析。
        
- **受動的手法 (Bash/Curl):**
    
    `curl -I http://192.168.1.50`
    
    - HTTPヘッダーの `Server:` や `X-Powered-By:` からOSを推測。
        
- **TTL解析手法）**

	- 代表的な初期TTL値
	
	|**OS種別**|**初期TTL値**|**特徴**|
	|---|---|---|
	|**Linux / Unix**|**64**|Androidや組み込みデバイスも多くはこれに該当。|
	|**Windows**|**128**|9x系から最新の11/Serverまで共通。|
	|**Solaris / AIX**|**255**|主にレガシーな商用UNIXや一部のNW機器。|
	
	- 実践例
	
	- **コマンド (ping):**
	    
	    `ping -c 1 192.168.1.50`
	    
	- **結果の読み方:**
	    
	    `64 bytes from ...: icmp_seq=1 ttl=64 time=0.052 ms` → **Linuxの可能性大**
	    
	    `64 bytes from ...: icmp_seq=1 ttl=127 time=0.821 ms` → **Windowsの可能性大**（ルーターを1つ経由して128から1減っている）

## 4. バージョンスキャン (Service/Version Detection)

具体的なアプリケーション名とバージョンを特定します。

- **詳細スキャン (Nmap):**
    
    `nmap -sV 192.168.1.50`
    
- **バナーグラビング (Netcat):**
    

	- 接続直後にサーバーが吐き出す「挨拶文（バナー）」を直接読み取る。

	`nc -vn 192.168.1.50 22`

	 - SMTPとのやりとりをする際に、改行コードをCRLFに変更して実施

	`nc -vnC 192.168.1.50`

## 5. 脆弱性スキャン (Vulnerability Scanning)

既知の脆弱性や設定不備を網羅的に調査します。

### nmap
- **スクリプト実行 (Nmap NSE):**
    
    `nmap --script vuln 192.168.1.50`
    
- **個別診断 (Nmap NSE):**
    
    `nmap -p 443 --script ssl-enum-ciphers 192.168.1.50`
    
    - 古い暗号化スイート（TLS 1.0等）の使用をチェック。

### SSH

```
sudo nmap -n --open -F --script=ssh-auth-methods 10.130.10.10 -sV
```

### HTTP

- `-p*http* `は、ファイル内の名前が「http」を含むすべてのサービスを検索し、それらのポートをスキャンする
	- `reverse-index`スクリプトを使用すると、サブネット内の「HTTPを持つ全ホスト」や「SSHを持つ全ホスト」を、各ホストのセクションを個別に調べる手間なく簡単に確認できます。これは特に、特定サービス（例：80/443番ポートのHTTP）のみに焦点を当てたいフォローアップスキャンで有用
	- `--max-retries 0` を指定すると、プローブの再送信が無効化され、スキャンが若干高速化されます

```
sudo nmap -n -p*http* --open 10.130.10.0/24 --script reverse-index --max-retries 0
```

- HTTP関連のNmapスクリプトは130以上存在する。特に有用なのは下記の4つ。
	- http-headers: WebサーバーからのHTTPレスポンスヘッダーを取得・表示します。
	- http-methods: 許可されているHTTPメソッド（GET、POST、PUT、DELETEなど）を列挙します。
	- http-robots.txt: サイトのrobots.txtファイルの内容を取得します。
	- http-auth-finder: 認証が必要なウェブリソースを検出します。 

```
sudo nmap 10.130.10.0/24 -n -p80,443 -PS80,443 --script http-headers,http-methods,http-robots.txt,http-auth-finder -oA /tmp/webservers
```

## 運用のためのベストプラクティス

1. **出力の習慣化:**
    
    Nmap実行時は常に `-oA`（全形式保存）を付け、後で `grep`（`.gnmap`）や `Metasploit`（`.xml`）で利用できるようにする。
    
2. **権限の使い分け:**
    
    `-sS`（SYNスキャン）は `sudo` が必要。権限がない場合は自動的に `-sT`（コネクトスキャン）に切り替わるが、ログに残りやすい点に注意。
    
3. **効率化:**
    
    広大なネットワークでは、まず Bash や Nmap のリストスキャン（`-sL`）で対象を絞り込んでから、詳細なバージョンスキャン（`-sV`）へ移行する。
