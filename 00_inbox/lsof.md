# `lsof` - 開かれているファイルとプロセスを一覧表示する

## 概要

`lsof` (list open files) は、システム上で現在**開かれているファイル**に関する情報を一覧表示する、非常に強力な診断・調査ツールです。UnixライクなOSでは、「全てがファイルである」という哲学に基づき、通常のファイルやディレクトリだけでなく、**ネットワークソケット**、パイプ、デバイスなどもファイルとして扱われます。

`lsof` を使うことで、「どのプロセスがこのファイルを開いているのか？」「どのプロセスがこのポートを使用しているのか？」といった問いに答えることができます。

(出自: `lsof` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`netstat` / `ss`**| **ネットワークソケット**の表示に特化しています。`lsof -i` は同様の機能を提供しますが、`lsof` はネットワーク以外のファイルも網羅的に調査できる、より汎用的なツールです。 | `ss` (ネットワーク特化) |
| **`fuser`** | 特定のファイルやファイルシステムを**どのプロセスが使用しているか**を特定するのに特化しています。`lsof` はより詳細な情報（ファイルディスクリプタ番号など）を表示します。 | - |
| **/proc/&lt;PID&gt;/fd/**| `lsof` が情報を取得している大元のデータソース。`lsof` はこれを全プロセスに対して行い、見やすく整形して表示します。 | `lsof` |

## よく連携されるコマンド

`lsof` は、ディスク容量のトラブルシューティングや、特定の接続のフィルタリングのために `grep` とよく連携されます。

### シナリオ例: 削除済みファイルが占有するディスク容量を解放する (インフラ構築・運用視点)

* **目的**: `df` と `du` の結果が合わない原因である、「削除したはずの巨大なファイル」をどのプロセスが掴んでいるかを特定し、解放する。
* **連携コマンド**: `grep`, `kill`
* **解説**: `lsof` の全出力の中から、`grep` を使って `(deleted)` という文字列を含む行をフィルタリングします。これにより、ファイルハンドルを掴み続けているプロセスを特定し、そのプロセスを再起動または `kill` することでディスクスペースを解放します。
* **コマンド例**:

```bash
# 削除済みのファイルを開き続けているプロセスを検索
sudo lsof | grep deleted
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **フィルタリング (最重要)** | |
| ⭐ `-i [<condition>]` | **ネットワークソケット**（インターネットファイル）を表示します。<br> `-i :<port>`: 特定のポート番号。<br> `-i TCP` or `UDP`: プロトコル。<br> `-i @<host>`: 接続先ホスト。 |
| ⭐ `-p <PID>` | 指定した**プロセスID (PID)** が開いているファイルのみを表示します。 |
| `-c <command>` | 指定した**コマンド名**で始まるプロセスが開いているファイルのみを表示します。 |
| ⭐ `-u <user>` | 指定した**ユーザー**が開いているファイルのみを表示します。 |
| `+D <directory>`| 指定したディレクトリ内（サブディレクトリも含む）で開かれているファイルを再帰的に検索します。 |
| `<filename>` | 引数としてファイルパスを直接指定すると、そのファイルをどのプロセスが開いているかを表示します。 |
| **出力制御** | |
| ⭐ `-n` | ホスト名をIPアドレスに変換（DNSルックアップ）しません。高速化に有効。 |
| ⭐ `-P` | ポート番号をサービス名 (`http`など) に変換しません。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 「Port already in use」エラーの原因となっているプロセスを特定する。
* **組み合わせ**: `sudo lsof -i :<port_number>`
* **解説**: 指定したポート番号をどのプロセスがリッスンしているかを特定します。Webサーバーが起動できない、といった問題の解決に不可欠です。
* **例**:

```bash
# ポート8080をLISTENしているプロセスを特定
sudo lsof -i :8080
```

### 2. ブルーチーム視点

* **タスク**: 侵害されたホスト上の全てのネットワーク接続を、名前解決せずありのまま調査する。
* **組み合わせ**: `sudo lsof -i -n -P`
* **解説**: **ライブフォレンジックの基本**。`ESTABLISHED` 状態のコネクションの中から、見慣れないIPアドレスやポートへの不審なアウトバウンド通信を発見し、マルウェアのC2通信などを特定します。
* **例**:

```bash
# 全てのネットワーク接続をプロセス情報付きで表示
sudo lsof -i -n -P
```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、ホストのネットワーク活動や開かれているファイルから情報を収集する。
* **組み合わせ**: `lsof -i -n -P`
* **解説**: `LISTEN` 状態のサービスを探して攻撃可能な内部サービスを特定したり、`ESTABLISHED` 状態の接続から、このサーバーがどのデータベースサーバーやAPIサーバーと通信しているかを把握したりします。
* **例**:

```bash
# ユーザー 'webapp' が開いているファイルをリストアップし、設定ファイルなどを探す
lsof -u webapp
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **コマンドの実行に非常に時間がかかる。**
    * **考えられる原因**: オプションなしで実行すると、システム上の全てのプロセスが開いている全てのファイルをリストアップしようとするためです。また、DNSやポート名の解決にも時間がかかります。
    * **解決策**: `-p`, `-c`, `-u`, `-i` などのフィルタオプションで対象を絞り込み、`-n` と `-P` を付けて名前解決をスキップしてください。

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `lsof` 自体ではなく、その**出力内容**が**情報漏洩**に繋がります。
* **悪用シナリオ**: `lsof` の出力は、システム上で何が動いているかの詳細なスナップショットです。プロセス名、ユーザー名、開いているファイル、ネットワーク接続先など、システムの内部構成に関する機密情報が豊富に含まれており、攻撃者の偵察に悪用されます。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo lsof` を実行できる権限を持っている場合、`lsof` の `!` コマンド（外部コマンド実行）を悪用して、`root` シェルを起動することが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/lsof/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定を厳格にし、`lsof` のような強力な情報収集ツールに `sudo` 権限を安易に与えない。
* **Detection (検知)**: `auditd` で `lsof` コマンドの実行を監視し、特に重要なプロセス (`-p`) やユーザー (`-u`) に対する不審な調査がないかを確認する。

## 注意点・補足

* **インストール**: `lsof` は多くのディストリ-ビューションで標準インストールされていません。使用するには、`sudo apt install lsof` や `sudo yum install lsof` などでパッケージをインストールする必要があります。
