# Command - Linux - mount - ファイルシステムをディレクトリツリーに接続する

## 概要
`mount` は、特定のストレージデバイス上のファイルシステムを、指定したディレクトリ（マウントポイント）に結びつけ、ユーザーがその内容にアクセスできるようにします。物理ディスクだけでなく、ISO ファイルやネットワーク上の共有フォルダ（NFS, CIFS）も同様に扱います。
(出自: `util-linux` パッケージに含まれる)



## 🛠 類似・関連コマンド
| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `umount` | マウントを解除する（「取り出し」に相当）。 | デバイスの安全な取り外し |
| `findmnt` | 現在のマウント状態を木構造で見やすく表示する。 | **マウント状況の確認（2026年推奨）** |
| `lsblk` | ブロックデバイスの一覧と、現在のマウントポイントを表示。 | 物理接続の確認 |
| `fdisk -l` | パーティション構成を確認する。 | マウント前の調査 |

## 主要なオプション
※実行には原則として **root 権限 (sudo)** が必要です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-t <type>` | ファイルシステムの種類（ext4, xfs, nfs, iso9660等）を指定。 |
| ⭐️ `-o <options>` | マウントの詳細設定（ro, rw, noexec, nosuid等）を指定。 |
| ⭐️ `-a`, `--all` | `/etc/fstab` に記載されているすべてのデバイスをマウントする。 |
| ⭐️ `-r`, `--read-only` | 読み取り専用モードでマウントする（`-o ro` と同等）。 |
| `-v`, `--verbose` | 実行経過を詳細に表示する。 |
| `--bind` | 既にあるディレクトリを別の場所にもマウントする。 |

## ⚙️ 重要なマウントオプション (`-o`)
セキュリティ堅牢化において非常に重要なパラメータです。

* **`ro` / `rw`**: 読み取り専用 / 読み書き可能。
* **`noexec`**: その領域内の実行ファイルを「実行禁止」にする。
* **`nosuid`**: SUID/SGID ビットを無視する（権限昇格の防止）。
* **`nodev`**: デバイスファイル（/dev/..）を解釈しない。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 外部ストレージやネットワーク共有の接続。
* **解説**: 新しく追加したディスクや、バックアップサーバーの NFS 領域をシステムに認識させます。
* **例**:
    ```bash
    # /dev/sdb1 を /mnt/backup に読み書き可能でマウント
    sudo mount -t ext4 /dev/sdb1 /mnt/backup
    ```

### 2. ブルーチーム視点
* **タスク**: システムの要塞化（Hardening）。
* **解説**: `/tmp` や `/var/tmp` など、誰でも書き込める場所に `noexec, nosuid, nodev` を設定し、攻撃者がそこに置いたツールを実行できないようにします。
* **例**:
    ```bash
    # 現在のマウントに noexec 等を動的に追加して再マウント
    sudo mount -o remount,noexec,nosuid /tmp
    ```

### 3. レッドチーム視点
* **タスク**: 隠されたデータの探索と権限昇格。
* **解説**: `/etc/fstab` を確認し、現在はマウントされていないが機密データが入っていそうなパーティションがないか探します。また、不注意に `rw` でマウントされている外部メディアを探し、そこに悪意あるバイナリを仕込みます。

## エラーメッセージとトラブルシューティング

1.  **「mount: /mnt: target is busy.」**:
    * **原因**: 他のプロセスがそのマウントポイントを使用中（またはシェルがそのディレクトリにいる）。
    * **解決策**: `lsof +D /mnt` で犯人を特定し、終了させる。

2.  **「mount: /dev/sdb1: can't read superblock」**:
    * **原因**: ファイルシステムが壊れているか、指定したタイプ（`-t`）が間違っている。
    * **解決策**: `lsblk -f` でタイプを確認し、必要なら `fsck` で修復を試みる。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **Mount Point Shadowing**: 既にデータがあるディレクトリに別のデバイスをマウントすると、元のデータが「隠され」ます。攻撃者はこれを利用し、重要ファイルを隠蔽することがあります。
* **Insecure fstab**: `/etc/fstab` に `user` オプション（一般ユーザーがマウント可能）が不用意に設定されていると、悪意あるデバイスの接続を許すリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **監査**: 定期的に `findmnt` を実行し、予期せぬマウントや、`noexec` が外れている重要ディレクトリがないかチェックします。
* **不必要なマウントの制限**: USB 自動マウント機能（automount）をサーバー環境では無効化します。

## 注意点・補足
* **永続化**: `mount` コマンドの効果は再起動すると消えます。起動時に自動マウントさせるには、**`/etc/fstab`** への追記が必須です。
