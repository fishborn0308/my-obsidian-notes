# Command - Linux - id - ユーザーとグループの識別情報を表示する

## 概要
`id` は、現在のユーザー、または指定したユーザーの「実効（Effective）」および「実（Real）」のユーザー ID (UID) とグループ ID (GID) を表示します。また、所属している全ての補助グループや、SELinux が有効な環境ではセキュリティコンテキストも出力します。
(出自: `coreutils` パッケージに含まれる)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `whoami` | 現在の「名前（文字列）」だけを簡潔に表示する。 | スクリプト内での名前取得 |
| `groups` | 所属している「グループ名」のみを一覧表示する。 | 所属確認の簡易版 |
| `getent passwd` | `/etc/passwd` からユーザーの登録情報を取得する。 | ユーザー定義の確認 |
| `id` | 数値 ID、名前、グループ、セキュリティ属性を網羅的に表示。 | **権限の最終確認** |

## よく連携されるコマンド
### シナリオ例: 特定の UID を持つユーザーの存在確認 (運用視点)
* **目的**: ユーザー名ではなく、UID (例: 1000) から誰が割り当てられているか逆引きする。
* **解説**: `id` 自体は名前に対応しますが、他のツールと組み合わせて管理状況を調べます。
* **コマンド例**:
    ```bash
    # 現在のユーザーの UID だけを抽出する
    id -u
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `(引数なし)` | UID, GID, 所属グループ、SELinux コンテキストを全て表示する。 |
| ⭐️ `-u`, `--user` | ユーザー ID (UID) のみを表示する。 |
| ⭐️ `-g`, `--group` | 有効なグループ ID (GID) のみを表示する。 |
| ⭐️ `-G`, `--groups` | 所属している全てのグループ ID を表示する。 |
| ⭐️ `-n`, `--name` | ID（数値）ではなく、名前（文字列）を表示する（`-u`, `-g`, `-G` と併用）。 |
| `-a` | (互換用) 他のオプションを無視し、全情報を表示する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: スクリプト内での root 権限チェック。
* **解説**: 管理者権限が必要なスクリプトで、実行ユーザーが root (UID=0) かどうかを判定します。
* **例**:
    ```bash
    if [ $(id -u) -ne 0 ]; then
        echo "このスクリプトは root 権限で実行してください。"
        exit 1
    fi
    ```

### 2. ブルーチーム視点
* **タスク**: 権限昇格攻撃の検知。
* **解説**: `sudo` や脆弱性を利用して権限が書き換えられたプロセスにおいて、`id` の結果が意図せず `uid=0(root)` になっていないか、不審な所属グループが増えていないかを監査します。

### 3. レッドチーム視点
* **タスク**: 侵入成功後の初期偵察（Post-Exploitation）。
* **解説**: シェルを取得した直後、最初に `id` を実行します。自分が `www-data` なのか、`backup` なのか、あるいは最初から `root` なのかを知ることで、次の攻撃ステップ（権限昇格の手法）を決定します。
* **例**:
    ```bash
    # 現在のコンテキスト（SELinuxなど）も含めて把握
    id
    ```

## 出力の読み方（例）
`uid=1000(alice) gid=1000(alice) groups=1000(alice),27(sudo),46(plugdev)`
* **uid**: ユーザーの数値IDと名前。
* **gid**: ログイン時のメイングループ（プライマリグループ）。
* **groups**: サブグループを含めた全ての所属。ここに `sudo` や `docker` があれば、特権操作が可能です。

## エラーメッセージとトラブルシューティング

1.  **「id: 'username': no such user」**:
    * **原因**: 指定したユーザーが存在しない、あるいは LDAP/AD 等のネットワーク認証が一時的に切れている。
    * **解決策**: `/etc/passwd` を確認するか、ネットワーク接続をチェック。

2.  **「グループに追加したのに id コマンドに反映されない」**:
    * **原因**: グループ設定の変更は、セッション開始時（ログイン時）に読み込まれるため。
    * **解決策**: 一度ログアウトしてログインし直すか、`newgrp` コマンドを使用する。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `id` コマンドが実行できることは、システムのユーザー構成やグループ設計という「攻略マップ」の一部を攻撃者に提供することになります。

### 対応策・緩和策 (ブルーチーム視点)
* **最小権限**: 不要なユーザーを `sudo` や `lxd`, `docker` といった強力なグループに所属させないよう、定期的に `id` 相当の情報（`/etc/group` 等）を監査します。

## 注意点・補足
* **実効(Effective) vs 実(Real)**:
    SUID が設定されたプログラムを実行中などは、「本来の自分」と「現在行使できる権限」が異なる場合があります。`id` は通常、現在行使可能な「実効 ID」を表示します。
