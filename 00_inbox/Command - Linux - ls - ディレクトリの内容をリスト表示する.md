# Command - Linux - ls - ディレクトリの内容をリスト表示する

## 概要
`ls` は、指定したディレクトリに含まれるファイルやサブディレクトリの一覧を表示します。単に名前を出すだけでなく、パーミッション、所有者、サイズ、更新日時といった「メタデータ」を詳細に確認するための、システム調査の第一歩となるコマンドです。



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `dir` | Windows 由来。`ls` とほぼ同じだがオプション体系が異なる。 | (あまり使われない) |
| `vdir` | `ls -l` と同等の出力をデフォルトで行う。 | 長いリスト用 |
| `exa` / `lsd` | 現代的な `ls` の代替。アイコン表示やカラーリングが強力。 | **2026年のモダン環境** |
| `tree` | ディレクトリ構造をツリー形式で視覚的に表示する。 | 階層構造の把握 |

## 主要なオプション
`ls` の真価はオプションの組み合わせにあります。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-l` | **最頻用。** ロングフォーマットで詳細情報（権限、所有者等）を表示。 |
| ⭐️ `-a` | **最頻用。** `.` で始まる隠しファイル（ドットファイル）も含めて全て表示。 |
| ⭐️ `-h` | **推奨。** サイズを「KB, MB, GB」など人間が読みやすい形式で表示。 |
| ⭐️ `-t` | 更新日時が新しい順に並べ替えて表示（トラブル調査で重宝）。 |
| ⭐️ `-R` | サブディレクトリ内も含めて再帰的に表示する。 |
| `-S` | ファイルサイズが大きい順に並べ替える。 |
| `-F` | ファイル名の末尾に種類（`/`:フォルダ, `*`:実行ファイル等）を付与。 |
| `--color` | 種類ごとに色分けして表示する（最近の環境ではエイリアスで標準設定）。 |

## 💡 出力結果の読み解き (`ls -l`)
`-l` オプションを付けた際の先頭10文字は、そのファイルの「身分証」です。

| 文字目 | 意味 | 内容例 |
| :--- | :--- | :--- |
| 1 | **種別** | `-`:ファイル, `d`:ディレクトリ, `l`:リンク |
| 2-4 | **所有権限** | `rwx`: 読み(4)+書き(2)+実行(1) = 7 |
| 5-7 | **グループ権限** | `r-x`: 読み(4)+実行(1) = 5 |
| 8-10 | **その他権限** | `r--`: 読み(4)のみ = 4 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: ディスク容量を圧迫している巨大ログの特定。
* **解説**: サイズ順に並べ、読みやすい単位で表示します。
* **例**:
    ```bash
    # サイズが大きい順に、人間が読みやすい形式で表示
    ls -lhS /var/log/
    ```

### 2. ブルーチーム視点
* **タスク**: 不審な実行ファイルの検知。
* **解説**: 全員に実行権限 (`rwxrwxrwx`) がついているような、ガバガバな権限設定のファイルや、不自然な隠しファイルがないか確認します。
* **例**:
    ```bash
    # 隠しファイルを含め、更新日時順に詳細表示
    ls -lat
    ```

### 3. レッドチーム視点
* **タスク**: 権限昇格（Privilege Escalation）の足がかり探し。
* **解説**: `ls -l` で表示される権限の中に `s` (SUID) が含まれるファイルを探します。これは実行時に所有者（多くは root）の権限で動くため、攻撃の標的になります。
* **例**:
    ```bash
    # SUID が設定されたファイルを確認する (一部抜粋)
    ls -l /usr/bin/sudo
    ```

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: ⭐️ **重要**。
公開ディレクトリ（Webサーバーの公開フォルダなど）で `ls` が機能していると、中にあるファイル名がすべて丸見えになり、脆弱なスクリプトやバックアップファイルを探し出される原因になります。

### 対応策・緩和策 (ブルーチーム視点)
* **Directory Indexing の無効化**: Web サーバー（Nginx, Apache）の設定で、ファイル一覧を表示させないようにします。
* **エイリアスの活用**: `alias ls='ls --color=auto'` などを設定し、不審な属性（実行権限やリンク）を色で直感的に判別できるようにします。

## 注意点・補足
* **`ll` コマンドの正体**: 多くの環境で `ll` は `ls -l` へのエイリアス（別名）として設定されています。
* **空ディレクトリの削除**: `ls` で何も表示されなくても、`ls -a` を打つと `.` や `..`、あるいは隠し設定ファイルが残っていることがあります。
