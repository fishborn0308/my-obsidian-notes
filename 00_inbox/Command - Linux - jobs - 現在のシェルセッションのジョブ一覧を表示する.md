# Command - Linux - jobs - 現在のシェルセッションのジョブ一覧を表示する

## 概要
`jobs` は、現在のシェルセッションで実行中のバックグラウンドジョブや、一時停止中のジョブの状態を表示します。OS 全体のプロセスを管理する `ps` とは異なり、「自分が今開いているターミナル」に関連付けられたタスクのみを対象とするのが特徴です。
(出自: シェルの組み込みコマンド (Built-in))



## 類似・関連コマンド
| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `ps` | システム全体のプロセスを表示する。 | 全プロセスの監視 |
| `fg` | バックグラウンドのジョブをフォアグラウンド（最前面）に戻す。 | 停止した対話処理の再開 |
| `bg` | 一時停止中のジョブをバックグラウンドで再開させる。 | 長い処理の並行実行 |
| `kill` | ジョブ番号を指定してプロセスを終了させる（`kill %1` など）。 | 特定ジョブの強制終了 |

## よく連携されるコマンド
### シナリオ例: バックグラウンド処理の完了確認 (運用視点)
* **目的**: 巨大なファイルの圧縮（`gzip`）をバックグラウンドで開始し、終わったかどうかを定期的に確認する。
* **解説**: ジョブ番号を使って、特定の処理が「Running」から「Done」に変わるのを見届けます。
* **コマンド例**:
    ```bash
    # 巨大なログをバックグラウンドで圧縮
    gzip access.log &
    # 状態を確認
    jobs
    ```

## 主要なオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `(引数なし)` | ジョブ番号、状態（Running/Stopped/Done）、実行コマンドを表示する。 |
| ⭐️ `-l` | ジョブ番号に加えて、プロセス ID (PID) も表示する。 |
| ⭐️ `-n` | 前回の通知以降に状態が変化したジョブのみを表示する。 |
| `-p` | ジョブの PID のみを表示する。 |
| `-r` | 実行中 (Running) のジョブのみを表示する。 |
| `-s` | 一時停止中 (Stopped) のジョブのみを表示する。 |

## 出力項目の読み解き
* **`[1]`**: ジョブ番号。`fg %1` や `kill %1` のように指定して操作します。
* **`+` (プラス)**: 「カレントジョブ」。引数なしで `fg` などを打った際に対象となるジョブ。
* **`-` (マイナス)**: 「バックアップジョブ」。カレントジョブが終了した次に昇格するジョブ。
* **`Running / Stopped / Done`**: ジョブの現在のステータス。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 複数タスクの並行処理。
* **解説**: エディタ（`vim`）で設定ファイルを開きながら、別のコマンドを試したい時、`Ctrl-Z` でエディタを一時停止して `jobs` で確認し、作業が終わったら `fg` で戻る、という流れるような作業を実現します。

### 2. ブルーチーム視点
* **タスク**: セッション内の不審な動きの検知。
* **解説**: ログイン中のユーザーが裏でこっそりスキャナーやバックドアを動かしていないか、`jobs -l` でプロセス ID を特定し、`lsof -p <PID>` などで詳細を調査します。

### 3. レッドチーム視点
* **タスク**: 持続的なシェル操作。
* **解説**: 取得したシェルの中で、リバースシェルのコネクションを維持しつつ、別の探索コマンドを実行するためにジョブ制御を駆使します。また、`disown` コマンドと組み合わせて、ログアウト後もプロセスを残す処理の前段階として `jobs` を使います。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報漏洩**: `jobs` の結果には実行中のコマンドライン引数が含まれます。機密情報を含むコマンドをバックグラウンド実行している場合、同じシェルを共有している（あるいは画面を覗き見ている）他者に内容を知られるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **セッションの監視**: 共有サーバーなどで、ユーザーがバックグラウンドに怪しいプロセスを隠していないか、管理者権限で `ps` や `htop` を使って全体監視を行います。

## 注意点・補足
* **シェルの制約**: `jobs` はシェルの組み込みコマンドであるため、シェルを閉じると（ログアウトすると）その管理下のジョブ情報も消滅します（プロセス自体は HUP シグナルを受けて終了するか、`nohup` 等により継続します）。
* **`%` 記法**: ジョブを操作する際は、PID の代わりに `%1`（ジョブ番号1）のような記法が使えることを覚えておくと非常に効率的です。
