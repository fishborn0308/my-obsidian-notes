# Command  - Linux - update-rc.d - SysVinitサービスの自動起動を管理する (Debian系/旧)

## 概要

`update-rc.d` は、**SysVinit** を採用した旧世代のDebian系Linuxディストリビューション（Debian 7, Ubuntu 14.04以前など）において、サービスの自動起動設定を管理するためのコマンドです。OSの起動レベル（ランレベル）ごとに、どのサービスを起動・停止するかを、`/etc/rc<runlevel>.d/` ディレクトリ配下にある起動スクリプトへのシンボリックリンクを操作することで制御します。

**⚠️ 重要**: このコマンドは**レガシー（旧式）**です。現在主流の `systemd` を採用したシステム (Debian 8+, Ubuntu 16.04+など) では、**`systemctl enable/disable`** を使用してください。

(出自: `sysv-rc` または `init-system-helpers` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`systemctl`** | `systemd` 環境におけるモダンな後継コマンド。サービスの起動/停止、自動起動設定などを統一的に扱え、依存関係管理も強力です。 | `systemctl` |
| **`chkconfig`** | `update-rc.d` の**Red Hat/CentOS系**における同等コマンド。機能的な役割は同じですが、コマンドの構文が異なります。 | - |
| **`service`** | **今すぐ**サービスを起動/停止するためのコマンド。`update-rc.d` は**OS起動時**の自動起動を設定するコマンドであり、役割が異なります。 | `systemctl` |

## よく連携されるコマンド

`update-rc.d` は、`/etc/init.d/` 配下に配置したカスタムスクリプトをサービスとして登録する際に、`chmod` などと連携して使われます。

### シナリオ例: 新しいinitスクリプトをサービスとして登録する (インフラ構築・運用視点)

* **目的**: カスタムアプリケーションの起動スクリプトをサービスとして登録し、OS起動時に自動実行させる。
* **連携コマンド**: `cp`, `chmod`, `update-rc.d`
* **解説**: まず起動スクリプトを所定の場所にコピーし、実行権限を付与します。次に `update-rc.d ... defaults` でサービスとして登録・有効化します。
* **コマンド例**:

    ```bash
    # このコマンドシーケンスでカスタムサービスを登録・有効化する
    sudo cp myapp-script /etc/init.d/myapp
    sudo chmod +x /etc/init.d/myapp
    sudo update-rc.d myapp defaults
    ```

## オプション説明 (`update-rc.d` のアクション)

| オプション | 説明 |
| :--- | :--- |
| **サービス管理** | |
| ⭐ `<service> defaults` | **最も基本的な使い方**。サービスをデフォルトのランレベル（通常 2,3,4,5）で、標準的な優先度で有効化します。 |
| ⭐ `<service> remove` | サービスの自動起動設定（シンボリックリンク）を全て削除し、無効化します。 |
| **詳細設定** | |
| `<service> enable` | 特定のランレベルでサービスを有効化します。（より詳細な制御） |
| `<service> disable`| 特定のランレベルでサービスを無効化します。 |
| `-f` | (`remove`と併用) `/etc/init.d`にスクリプトが残っていても強制的にシンボリックリンクを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Apache (`apache2`) サービスをOS起動時に自動で立ち上がるように設定する (Ubuntu 14.04)。
* **組み合わせ**: `sudo update-rc.d <service_name> defaults`
* **解説**: サーバーが再起動してもWebサービスが自動で開始されるように、サービスの自動起動を有効化します。
* **例**:

    ```bash
    # apache2サービスを自動起動ONに設定
    sudo update-rc.d apache2 defaults

    # 自動起動をOFFにする
    sudo update-rc.d apache2 remove
    ```

### 2. ブルーチーム視点

* **タスク**: システムに不審な永続化設定がないか監査する。
* **組み合わせ**: `ls /etc/rc*.d/S*`
* **解説**: システムで自動起動が有効になっているサービス (`S`で始まるシンボリックリンク) をリストアップします。リストの中に、標準的でない、あるいは明らかに不審な名前のサービスがないかを確認します。
* **例**:

    ```bash
    # ランレベル3で起動するサービスをリストアップし、不審なものがないか確認
    ls -l /etc/rc3.d/S*
    ```

### 3. レッドチーム視点

* **タスク**: レガシーシステム上で永続化を確立する。
* **組み合わせ**: `update-rc.d <backdoor_svc> defaults`
* **解説**: 攻撃者がroot権限を奪取した後、`/etc/init.d/` ディレクトリにリバースシェルなどを起動する悪意のあるスクリプトを設置し、このコマンドでサービスとして有効化します。これにより、システムが再起動してもバックドアが自動的に実行されるように仕掛けます。
* **例**:

    ```bash
    # 悪意のあるスクリプトをサービスとして登録し、有効化する
    sudo chmod +x /etc/init.d/evil-svc
    sudo update-rc.d evil-svc defaults
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `update-rc.d: error: initscript does not exist: /etc/init.d/<service_name>`
    * **考えられる原因**: `/etc/init.d/` ディレクトリに、指定したサービス名の起動スクリプトが存在しません。
    * **解決策**: サービス名が正しいか、パッケージが正しくインストールされているかを確認してください。

## 環境変数と設定ファイル

* `update-rc.d` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/init.d/`**: サービスの起動・停止などを制御する実際のスクリプトが置かれるディレクトリ。
* **`/etc/rc[0-6].d/`**: 各ランレベルに対応するディレクトリ。中には `/etc/init.d/` にあるスクリプトへのシンボリックリンクが置かれます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `update-rc.d` 自体ではなく、それが管理するSysVinitの仕組みが、攻撃者による**永続化**の確立に悪用されます。
* **悪用シナリオ**: 攻撃者がroot権限を奪取すると、`/etc/init.d/` に悪意のあるスクリプトを設置し、`update-rc.d` でそれをサービスとして有効化することで、永続的なバックドアを作成します。

### GTFOBins / LOLBAS における利用例

`update-rc.d` はOSの構成を変更するコマンドであり、直接的なコード実行や権限昇格の機能を持たないため、GTFOBinsにはリストされていません。しかし、**永続化 (Persistence)** の文脈で、悪意のあるスクリプトを登録するために悪用されます。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `/etc/init.d/` ディレクトリ配下のスクリプトのパーミッションを厳格に管理し、不要な書き込み権限を許可しない。
* **Detection (検知)**: `ls /etc/rc*.d` の出力を定期的に取得し、ベースラインと比較して意図しないサービスの追加がないかを監視する。ファイル整合性監視 (FIM) ツールで `/etc/init.d/` や `/etc/rc.d/` 配下の変更を検知する。

## 注意点・補足

* **LSBヘッダ**: `update-rc.d` が正しく動作するためには、`/etc/init.d/` にある起動スクリプトの冒頭に、依存関係などを示す特別なコメントブロック（LSBヘッダ）が記述されている必要があります。
