# Command - Linux - install - ファイルをコピーし、属性を一度に設定する

## 概要
`install` は、ファイルを指定した場所にコピーすると同時に、そのファイルのパーミッション（モード）、所有者、グループを設定します。また、必要に応じてコピー先のディレクトリを自動作成する機能も備えています。
(出自: `coreutils` パッケージに含まれる)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `cp` | 単純なコピー。属性の変更は別コマンドが必要。 | 一般的なコピー |
| `chmod` | 既存のファイルの権限を変更する。 | 権限のみの変更 |
| `chown` | 既存のファイルの所有者を変更する。 | 所有者のみの変更 |
| `mkdir -p` | ディレクトリを再帰的に作成する。 | フォルダ作成のみ |
| `install` | **コピー + 権限設定 + 所有者設定 + フォルダ作成** を一気に完結。 | **スクリプト・バイナリの配布** |

## よく使われるオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-d`, `--directory` | 指定された全ての引数をディレクトリとして作成する（`mkdir -p` 相当）。 |
| ⭐️ `-m <MODE>`, `--mode` | コピー後のパーミッションを設定する（例: `755`, `644`）。 |
| ⭐️ `-o <USER>`, `--owner` | 所有者を指定する（root 権限が必要）。 |
| ⭐️ `-g <GROUP>`, `--group` | グループを指定する（root 権限が必要）。 |
| ⭐️ `-s`, `--strip` | コピー時にバイナリからデバッグ用のシンボル情報を削り、サイズを小さくする。 |
| `-p`, `--preserve-timestamps` | 元のファイルのアクセス・修正時刻を保持する。 |
| `-D` | コピー先のパスに含まれる親ディレクトリが存在しない場合、自動作成する。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 自作スクリプトを `/usr/local/bin` に安全に配置する。
* **解説**: 権限設定を忘れて「実行できない！」となるミスを防ぎ、確実に実行可能な状態で設置します。
* **例**:
    ```bash
    # backup.sh を実行可能権限(755)で root 所有として配置
    sudo install -m 755 -o root -g root backup.sh /usr/local/bin/
    ```

### 2. ブルーチーム視点
* **タスク**: 最小権限原則に基づく設定ファイルの設置。
* **解説**: 設定ファイルをコピーする際、デフォルトの緩いパーミッションが適用されるのを防ぎ、最初から `600`（所有者のみ読み書き）などで厳格に設置します。
* **例**:
    ```bash
    # 機密情報を書き込んだ設定ファイルを安全な権限で設置
    sudo install -m 600 config.conf /etc/myapp/config.conf
    ```

### 3. レッドチーム視点
* **タスク**: ペイロード（攻撃用コード）の展開。
* **解説**: 侵入後、自分のツールを `/tmp` などに配置する際、`chmod +x` を打つ手間（とコマンド履歴）を省くために `install` で一気に実行権限を付けて展開することがあります。

## エラーメッセージとトラブルシューティング

1.  **「install: cannot change ownership of ...: Operation not permitted」**:
    * **原因**: `-o` や `-g` で自分以外の所有者を指定しようとしたが、root 権限がない。
    * **解決策**: `sudo` を付けて実行してください。

2.  **「install: omitting directory 'folder_name'」**:
    * **原因**: ディレクトリをコピーしようとしたが、`-d` オプションが適切に使われていない。
    * **解決策**: 単一のファイルをコピーするのか、ディレクトリを作成（`-d`）するのかを明確に指定してください。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **Setuid 属性の誤設定**: `-m 4755` のように誤って Setuid 属性を付けて設置してしまうと、一般ユーザーが root 権限でそのファイルを実行できてしまう「権限昇格」の脆弱性を生む可能性があります。

### 対応策・緩和策 (ブルーチーム視点)
* **監査**: `Makefile` 内の `install` コマンドに不審なパーミッション設定がないかレビューします。
* **監視**: システムディレクトリ（`/usr/bin` 等）への `install` コマンドによる書き込みを `auditd` で記録し、不正なバイナリ置換がないか監視します。

## 注意点・補足
* **上書き動作**: コピー先に同名のファイルがある場合、**警告なしで上書き**されます（`cp` と同様）。
* **ディレクトリの一括作成**: `install -d /path/to/a /path/to/b` と打つだけで、複数の深い階層のディレクトリを一度に安全に作れるため、環境構築スクリプトで重宝します。
