# Command - Linux - chgrp - ファイルやディレクトリの所有グループを変更する

## 概要
`chgrp` は、ファイルやディレクトリの所有グループを変更するためのコマンドです。所有ユーザー（Owner）はそのままに、特定のグループ（Group）のメンバーに対してアクセス権を付与・変更したい場合に使用します。
(出自: `coreutils` パッケージに含まれる)



## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `chown` | 所有ユーザーとグループを両方変更可能。現在の主流。 | 総合的な所有権変更 |
| `newgrp` | 現在のセッションのプライマリグループを一時的に変更する。 | 一時的なグループ切り替え |
| `chgrp` | **所有グループのみ**を明示的に変更する。スクリプト等で意図せぬ所有者変更を防ぐのに安全。 | グループのみの変更 |

## よく連携されるコマンド
### シナリオ例: プロジェクト共有ディレクトリのセットアップ (運用視点)
* **目的**: 複数のユーザーが所属するプロジェクトグループに対して、共通の作業ディレクトリへのアクセス権を与える。
* **連携コマンド**: `chmod`, `ls -l`
* **解説**: ディレクトリを作成後、所有グループを変更し、グループに対して書き込み権限を付与します。
* **コマンド例**:
    ```bash
    # ディレクトリの所有グループを 'developers' に変更
    sudo chgrp -R developers /opt/project_data
    # グループに対して書き込み権限を付与
    sudo chmod -R g+w /opt/project_data
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐️ `-R`, `--recursive` | ディレクトリとその中身を再帰的に変更する。 |
| ⭐️ `-v`, `--verbose` | 変更があったファイル名を表示する詳細モード。 |
| `-c`, `--changes` | 実際に所有グループが変わった場合のみ表示する。 |
| **リンク・参照** | |
| `-h`, `--no-dereference` | シンボリックリンク自体の所有グループを変更する（リンク先は変えない）。 |
| `--reference=<RFILE>` | 指定したファイル（RFILE）と同じグループ設定をコピーする。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: Webコンテンツ公開時のグループ権限修正。
* **組み合わせ**: `chgrp -R www-data`
* **解説**: 開発者がアップロードしたファイルの所有ユーザーを維持したまま、Webサーバ（Nginx/Apache）が読み取れるようグループだけを統一します。
* **例**:
    ```bash
    # ファイルの所有者はそのままで、グループだけWebサーバ用に変更
    sudo chgrp -R www-data /var/www/html/assets
    ```

### 2. ブルーチーム視点
* **タスク**: 機密ファイル（ログ等）の不適切なグループ所有権の監査。
* **組み合わせ**: `find` との連携
* **解説**: `/var/log` や `/etc` 配下で、特権グループ（`sudo`, `wheel`, `shadow` 等）が所有すべきファイルが、一般ユーザーのグループになっていないか調査します。
* **例**:
    ```bash
    # /etc/shadow のグループが shadow 以外になっていないか確認
    ls -l /etc/shadow | awk '$4 != "shadow" {print "Alert: " $0}'
    ```

### 3. レッドチーム視点
* **タスク**: `sudo chgrp` を悪用した機密情報の窃取。
* **組み合わせ**: `sudo chgrp <my_group> /etc/shadow`
* **解説**: `sudoers` 設定の不備で `chgrp` が許可されている場合、重要ファイルのグループを自分が所属するグループに変更します。その後 `chmod g+r` を組み合わせることで、本来読めないファイルを読み取ります。
* **例**:
    ```bash
    # shadowファイルのグループを自分のメイングループに変更
    sudo chgrp $(id -gn) /etc/shadow
    # グループに読み取り権限を付与して中身を盗む
    chmod g+r /etc/shadow
    cat /etc/shadow
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `chgrp: changing group of '...': Operation not permitted`
    * **考えられる原因**: 一般ユーザーは、自分が所属していないグループに所有グループを変更することはできません（root以外）。
    * **解決策**: `sudo` を使用するか、自分が所属しているグループを指定する。

2.  **エラーメッセージ例 2**: `chgrp: invalid group: 'dev-team'`
    * **考えられる原因**: 指定したグループ名がシステムに存在しない。
    * **解決策**: `/etc/group` を確認するか、GID（数値）で直接指定する。

## 環境変数と設定ファイル
* `chgrp` 自体に設定ファイルはありませんが、グループ情報は `/etc/group` および `/etc/gshadow` に依存します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **悪用シナリオ**: 攻撃者が `docker` グループや `lxd` グループの所有権を操作できる場合、それらの特権を利用してコンテナ経由でホストの root 権限を奪取（Privilege Escalation）する足掛かりにされます。

### GTFOBins における利用例
`chgrp` が `sudo` で実行可能な場合、特権ファイルのグループを操作して内容を読み取ることが可能です。
* **機能**: `Sudo`
* **コマンド例**:
    ```bash
    # 自身のグループに shadow ファイルの所有権を移す
    sudo chgrp $(id -gn) /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/chgrp/`

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: `sudoers` で `chgrp` を許可する際は、対象となるディレクトリやグループを厳格に制限する。
* **Detection (検知)**: システムの重要ディレクトリにおけるグループ所有権の変更を `auditd` で常時監視する。

## 注意点・補足
* **数値指定**: グループ名だけでなく GID (例: `1001`) での指定も可能です。
* **シンボリックリンク**: デフォルトではリンク先のファイルのグループが変わります。リンク自体を変更したい場合は `-h` オプションが必須です。
