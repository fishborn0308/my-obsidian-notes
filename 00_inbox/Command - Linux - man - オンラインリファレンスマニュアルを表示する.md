# Command - Linux - man - オンラインリファレンスマニュアルを表示する

## 概要
`man` は、システムにインストールされているコマンド、設定ファイル、ライブラリ関数などのマニュアルを表示します。`less` をベースにしたページャーで表示されるため、閲覧中に検索やスクロールが可能です。



## 📖 マニュアルの「セクション」
`man` は膨大な情報を整理するため、以下のセクションに分かれています。同名のコマンドと関数（例: `passwd`）がある場合、セクション番号を指定して呼び分ける必要があります。

| セクション | 内容 | 例 |
| :--- | :--- | :--- |
| **1** | **ユーザーコマンド** | `ls`, `man`, `cat` |
| **2** | システムコール | `open`, `read`, `fork` |
| **3** | Cライブラリ関数 | `printf`, `malloc` |
| **5** | **設定ファイルの形式** | `/etc/passwd`, `/etc/hosts` |
| **8** | **管理者用コマンド** | `iptables`, `fdisk`, `reboot` |

## 類似・関連コマンド
| コマンド | 役割 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `info` | より詳細な解説とノード（リンク）機能を持つ。 | GNUツールの深い調査 |
| `tldr` | 難しい解説を省き、よく使う**「具体例」のみ**を表示する。 | **2026年のタイパ重視派** |
| `help` | シェル組み込みコマンド（`cd`, `exit`等）の簡易説明。 | シェル機能の確認 |
| `apropos` | キーワードから関連する man ページを検索する (`man -k`)。 | コマンド名忘れ対策 |

## 主要なオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `man [section] <name>` | 指定したセクションのマニュアルを開く。 |
| ⭐️ `-k <keyword>` | キーワードを含むマニュアルを一覧表示する（`apropos`）。 |
| ⭐️ `-f <name>` | そのコマンドが何をするものか、一行で表示する（`whatis`）。 |
| ⭐️ `-a`, `--all` | 一致するすべてのセクションのマニュアルを順に表示する。 |
| `-w`, `--path` | マニュアルファイルが保存されている場所（パス）を表示する。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 不慣れな設定ファイルの書き方を確認。
* **解説**: `/etc/fstab` の各カラムの意味が分からなくなった時、ネットを見るより `man 5 fstab` を引く方が、今使っている OS の正確な仕様に辿り着けます。
* **例**:
    ```bash
    # fstab 設定ファイルの仕様を確認
    man 5 fstab
    ```

### 2. ブルーチーム視点
* **タスク**: セキュリティ設定のオプション調査。
* **解説**: `ssh` などの重要サービスのコンフィグを堅牢化する際、どのようなパラメータがサポートされているかを `man 5 sshd_config` で精査し、デフォルト値を確認します。

### 3. レッドチーム視点
* **タスク**: 隠れた「危険なオプション」の探索。
* **解説**: 侵入後、あるコマンドが `sudo` 権限で使えることが判明した際、そのコマンドに「シェルエスケープ」や「ファイル書き出し」ができるマイナーなオプションがないか `man` で徹底的に調べ、権限昇格に繋げます。

## 💡 知っておくと便利なテクニック
* **`man man`**: `man` コマンド自体の使い方を `man` で調べる。基本中の基本です。
* **検索**: マニュアル表示中に `/keyword` と打つと、その単語を検索できます。`n` で次へ。
* **PDF化**: `man -t ls | ps2pdf - ls.pdf` のようにして、マニュアルを PDF として保存し、オフラインでゆっくり読むこともできます。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `man -k` を実行することで、そのサーバーにどのようなツール（スキャナー、コンパイラ、デバッガ等）がインストールされているかを、攻撃者に把握されるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **最小限のインストール**: 本番サーバーには不要な開発ツールやそのマニュアルをインストールしないことで、攻撃者にヒントを与えない「情報の要塞化」を行います。
