# Command - Linux - route - IP ルーティングテーブルを表示・操作する

## 概要
`route` は、カーネルの IP ルーティングテーブルを表示したり、新しいルートを追加・削除したりするために使用されます。パケットの「出口」を決定する重要な設定を司ります。



## 🛠 新旧コマンド比較
| 機能 | 伝統的コマンド (`net-tools`) | 現代的コマンド (`iproute2`) |
| :--- | :--- | :--- |
| **表示** | `route -n` | `ip route show` |
| **追加** | `route add ...` | `ip route add ...` |
| **削除** | `route del ...` | `ip route del ...` |
| **特徴** | 出力が直感的で読みやすい。 | 高機能で詳細な制御が可能。 |

## 主要なフラグと引数
| オプション / 引数 | 説明 |
| :--- | :--- |
| ⭐️ `-n` | **超重要。** 名前解決をせず IP アドレスを数字で表示する（高速かつ確実）。 |
| ⭐️ `add` | 新しい経路（ルート）を追加する。 |
| ⭐️ `del` | 既存の経路を削除する。 |
| ⭐️ `gw` | ゲートウェイ（次のルーター）の IP を指定する。 |
| ⭐️ `netmask` | ネットワークマスクを指定する。 |
| ⭐️ `dev` | 出力するネットワークインターフェース（eth0 など）を指定する。 |

## コマンド例
```bash
# 現在のルーティングテーブルを数字で表示
route -n

# デフォルトゲートウェイ（全ての外向き通信の出口）を設定
sudo route add default gw 192.168.1.1

# 特定のネットワーク (10.0.0.0/24) への経路を追加
sudo route add -net 10.0.0.0 netmask 255.255.255.0 gw 192.168.1.254 dev eth0

# 特定のホストへの経路を削除
sudo route del -host 192.168.1.50
````

---

## 📊 出力の読み方（Flags 欄）

- **U (Up)**: 経路が有効。
    
- **G (Gateway)**: ゲートウェイを使用する（別のルーターを経由する）。
    
- **H (Host)**: ネットワークではなく、特定の 1 台（ホスト）への経路。
    
- **! (Reject)**: 通信を拒否する経路。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 二重ネットワーク環境での通信制御。
    
- **解説**: 例えば「社内用 LAN」と「インターネット用 WAN」の 2 枚の NIC がある場合、社内宛てのパケットだけを特定のルーターに飛ばす「スタティックルート（静的経路）」の設定に `route` を使用します。
    

### 2. ブルーチーム視点

- **タスク**: 通信経路の異常検知。
    
- **解説**: 意図しないデフォルトゲートウェイが設定されていないか監視します。もし身に覚えのない `0.0.0.0` 宛ての経路が攻撃者のマシンを向いていれば、全ての通信を盗聴（スニッフィング）されている可能性があります。
    

### 3. レッドチーム視点

- **タスク**: 中間者攻撃 (MITM) とトラフィックのリダイレクト。
    
- **解説**: 侵入後、ターゲットマシンのルーティングテーブルを書き換えることで、本来のサーバーへ行くはずの通信を自分の待機しているマシンへバイパスさせ、認証情報などを奪取します。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **経路の毒入れ (Route Poisoning)**: 偽のルーティング情報を注入されると、正しい宛先にパケットが届かなくなったり、攻撃者の元へパケットが吸い込まれたりします。
    

### 対応策・緩和策 (ブルーチーム視点)

- **設定の永続化と保護**: `route` コマンドによる変更は再起動で消えてしまうため、`/etc/network/interfaces` や `Netplan` などの設定ファイルで正しく管理し、これらのファイルの書き込み権限を厳重に制限します。
