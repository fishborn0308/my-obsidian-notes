# Command - Linux - ip neigh - 近隣ノード（ARP/NDPキャッシュ）の管理

## 概要
`ip neigh` オブジェクトは、同一ネットワークセグメント内に存在する他のホスト（近隣ノード）との対応表を管理します。IP アドレスから MAC アドレスを特定するためのキャッシュを操作し、通信の疎通確認や、セキュリティ上の理由による固定エントリーの作成などを行います。



## 主要サブコマンド
| サブコマンド | 略称 | 説明 |
| :--- | :--- | :--- |
| **`show`** | `sh`, `s` | ARP/NDP キャッシュテーブルを表示する（デフォルト）。 |
| **`add`** | `a` | 新しい近隣ノードのエントリーを手動で追加する。 |
| **`del`** | `d` | キャッシュから特定のエントリーを削除する。 |
| **`replace`** | | 既存のエントリーを上書き・修正する。 |
| **`flush`** | `f` | 条件に合うキャッシュを一括で削除（フラッシュ）する。 |

## コマンド例と構文

### 1. キャッシュの確認 (show)
現在把握している「隣人」のリストを表示します。
```bash
# ARP テーブルを一覧表示
ip neigh show

# 特定のインターフェース（eth0）に関連する隣人のみ表示
ip neigh show dev eth0
````

### 2. エントリーの手動追加と削除 (add / del)

特定のホストを「常にこの MAC アドレスだ」と固定（Static）したい場合に使用します。

Bash

```
# 192.168.1.1 の MAC アドレスを固定で登録する
sudo ip neigh add 192.168.1.1 lladdr 00:11:22:33:44:55 dev eth0

# 指定した IP のキャッシュを削除する
sudo ip neigh del 192.168.1.1 dev eth0
```

### 3. キャッシュの全クリア (flush)

ネットワーク構成が変わった際、古い情報を一掃するために使います。

Bash

```
# eth0 の全 ARP キャッシュを消去
sudo ip neigh flush dev eth0
```

## エントリーの状態（State）の読み解き

`ip neigh` の出力の末尾には、その情報の「鮮度」を示す状態が表示されます。

- **`REACHABLE`**: 有効。最近通信が行われ、疎通が確認されている。
    
- **`STALE`**: 古い。情報は残っているが、再確認が必要な状態。
    
- **`DELAY`**: 遅延。再確認のためのパケット送信を待機中。
    
- **`PROBE`**: 調査中。現在 ARP リクエストを送信して確認している。
    
- **`FAILED`**: 失敗。解決できなかった。
    
- **`PERMANENT`**: ⭐️ **重要**。手動で設定された固定（Static）エントリー。
    

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 重複 IP の調査。
    
- **解説**: 通信が不安定な際、`ip neigh` を見て同一 IP に対して予期せぬ MAC アドレスが紐付いていないか、または頻繁に `REACHABLE` と `STALE` を繰り返していないか確認します。
    

### 2. ブルーチーム視点

- **タスク**: ARP スプーフィング攻撃の検知。
    
- **解説**: ゲートウェイ（ルーター）の MAC アドレスが、突然別のもの（攻撃者の端末）に書き換わっていないか監視します。
    
- **例**: `ip neigh show` を定期的に実行し、ゲートウェイの MAC アドレスを監視するスクリプトを運用。
    

### 3. レッドチーム視点

- **タスク**: Man-in-the-Middle（中間者攻撃）の下準備。
    
- **解説**: ターゲットの ARP テーブルを汚染（Poisoning）する前に、現在の正当な対応関係を把握し、攻撃後の「戻し」や「隠蔽」の計画を立てるために参照します。
    

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **ARP Poisoning**: 同一ネットワーク内の誰でも ARP 応答を偽装できるため、トラフィックを盗聴（Sniffing）されるリスクがあります。
    
- **情報の露出**: セグメント内に他にどのようなサーバーが存在するかが筒抜けになります。
    

### 対応策・緩和策 (ブルーチーム視点)

- **Static ARP**: 重要なサーバー間の通信には、`ip neigh add ... PERMANENT` で MAC アドレスを固定し、偽装を無効化します。
    
- **DAI (Dynamic ARP Inspection)**: スイッチ側で不正な ARP パケットを遮断する機能を有効にします。
