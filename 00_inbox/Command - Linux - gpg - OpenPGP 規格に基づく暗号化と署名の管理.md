# Command - Linux - gpg - OpenPGP 規格に基づく暗号化と署名の管理

## 概要
`gpg` は、データの暗号化とデジタル署名を行うためのツールです。公開鍵暗号方式（非対称鍵）を主軸とし、受信者の公開鍵で暗号化して本人だけが復号できるようにしたり、自身の秘密鍵で署名して「本人が作成したこと」を証明したりできます。
(出自: GNU プロジェクトによる PGP のフリーな代替実装)

## よく連携されるコマンド
### シナリオ例: 機密情報の安全な受け渡し (運用・開発視点)
* **目的**: パスワードリストや設計書を、平文のまま送らずに特定の相手にだけ読めるようにする。
* **解説**: 相手の公開鍵をインポートし、それを使ってファイルを暗号化します。
* **コマンド例**:
    ```bash
    # 1. 相手の公開鍵をインポート
    gpg --import friend_pubkey.asc
    # 2. 相手(friend@example.com)の鍵を使ってファイルを暗号化
    gpg -e -r friend@example.com secret.txt
    ```

## 主要なサブコマンドとオプション

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: スクリプト内でのバックアップファイルの暗号化。
* **解説**: パスフレーズを対話的に入力できない自動処理では、あらかじめ用意したパスフレーズファイルやエージェントを利用して暗号化します。
* **例**:
    ```bash
    # 共通鍵方式で、非対話的にファイルを暗号化
    gpg --batch --yes --passphrase-file ./pass.txt -c backup.tar.gz
    ```

### 2. ブルーチーム視点
* **タスク**: ダウンロードしたソフトウェアの改ざんチェック。
* **解説**: 配布元が提供している署名ファイル（`.sig` や `.asc`）を検証し、配布過程でマルウェアが混入されていないか確認します。
* **例**:
    ```bash
    # ダウンロードしたバイナリの署名を検証
    gpg --verify software.tar.gz.asc software.tar.gz
    ```

### 3. レッドチーム視点
* **タスク**: 秘密情報の窃取（エクスフィルトレーション）。
* **解説**: 侵入先のサーバーに `~/.gnupg` ディレクトリがある場合、秘密鍵（`secring.gpg` 等）を盗み出そうとします。鍵が盗まれると、そのユーザー宛の暗号化メールやファイルがすべて解読される恐れがあります。

## エラーメッセージとトラブルシューティング

1.  **「gpg: decryption failed: No secret key」**:
    * **原因**: 暗号化されたファイルを解くための秘密鍵が自分のキーリングに存在しない。
    * **解決策**: 適切な秘密鍵をインポートするか、正しい鍵ペアを使用しているか確認。

2.  **「gpg: decryption failed: Bad passphrase」**:
    * **原因**: 秘密鍵を使用するためのパスフレーズが間違っている。
    * **解決策**: 正しいパスフレーズを入力。忘れた場合は、失効証明書を使って鍵を無効化するしかない。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **秘密鍵の管理**: 秘密鍵が流出すると、過去の暗号化データも含めてすべて解読されるリスクがあります。秘密鍵には必ず強力なパスフレーズをかけるべきです。
* **Web of Trust の誤解**: 公開鍵が本人のものかどうかを「信頼の署名」で繋いでいく仕組みですが、信頼の起点を誤ると偽の鍵を信じ込まされる可能性があります。

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention**: 秘密鍵を物理的なセキュリティキー（YubiKey 等）に格納し、OS 上から直接盗み出せないようにする。
* **Detection**: `~/.gnupg` フォルダに対する予期せぬアクセスやコピーを監視する。

## 注意点・補足
* **失効証明書の作成**: 鍵を作成した直後に「失効証明書（Revocation Certificate）」を作っておきましょう。万が一秘密鍵を紛失したり盗まれたりしたときに、その鍵が無効であることを世界に知らせる唯一の手段です。
* **gpg-agent**: パスフレーズをメモリにキャッシュし、何度も入力を求められないようにするデーモンが背後で動いています。
