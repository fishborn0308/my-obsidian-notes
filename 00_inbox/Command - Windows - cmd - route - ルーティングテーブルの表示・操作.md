# Command - Windows - cmd - route - ルーティングテーブルの表示・操作

## 概要

`route` コマンドは、Windows の IP ルーティング テーブルの内容を表示および変更するために使用されます。
パケットがどのネットワーク インターフェイスやゲートウェイを経由して宛先に到達するかを制御します。ネットワーク設計におけるマルチホーム環境（複数のネットワークに接続された端末）の構築や、障害時の経路切り替えに欠かせません。
(出自: Windows標準搭載 / TCP/IP ユーティリティ)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `netsh interface ipv4 show route` | `netsh` はよりモダンな管理コマンド。`route` よりも詳細なインターフェイス情報と連携可能。 | `netsh` (詳細管理) |
| `ipconfig` | 自分の IP とデフォルトゲートウェイを確認するのみ。経路の詳細は見れない。 | `ipconfig` (簡易確認) |
| `tracert` | 実際にパケットが辿った経路を表示する。`route` は「これから通る予定の道」を設定する。 | `tracert` (経路確認) |

## よく連携されるコマンド

`route` で設定を変更した後は、必ず疎通確認コマンドとセットで検証を行います。

### シナリオ例: 特定の内部セグメントへの静的ルート追加 (インフラ構築)

* **目的**: デフォルトゲートウェイとは別のルーター経由でしか到達できない社内サブネットへの通信を有効にする。
* **連携コマンド**: `ping`, `tracert`
* **解説**: 経路を追加した後、実際にそのゲートウェイを通ってパケットが届くかを確認します。
* **コマンド例**:
    ```cmd
    REM 10.0.0.0/24 への通信を 192.168.1.254 経由に設定
    route add 10.0.0.0 mask 255.255.255.0 192.168.1.254
    tracert -d 10.0.0.1
    ```

## スイッチ/オプション説明

`route` は [オプション] [コマンド] [宛先] の形式で実行します。

| オプション / コマンド | 説明 |
| :--- | :--- |
| **主要コマンド** | |
| ⭐ `PRINT` | 現在のルーティング テーブルを表示する。 |
| ⭐ `ADD` | 新しいルート（経路）を追加する。 |
| ⭐ `DELETE` | 既存のルートを削除する。 |
| `CHANGE` | 既存のルートのゲートウェイやメトリックを変更する。 |
| **スイッチ** | |
| ⭐ `-f` | すべてのゲートウェイ エントリのルーティング テーブルを消去する（慎重に使用）。 |
| ⭐ `-p` | **Persistent (永続化)**。`ADD` と併用し、再起動後も設定を維持する。 |
| `-4` / `-6` | IPv4 または IPv6 の使用を強制する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 二重ルーター環境でのルーティング整理
* **組み合わせ**: `ADD -p`
* **解説**: 特定の業務サーバーへの通信だけ、高速な別回線（ルーター）を通したい場合に使用します。永続化オプションを忘れると再起動時に通信不可となるため注意が必要です。
* **例**:
    ```cmd
    REM 再起動後も有効な静的ルートの追加
    route -p add 172.16.0.0 mask 255.255.0.0 192.168.1.50
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なバックドア経路の調査
* **解説**: 攻撃者がデータを外部に持ち出すために、VPN やプロキシを介した「裏道」を作っていないか監査します。
* **例**:
    ```cmd
    REM 現在の経路一覧を表示し、見覚えのないゲートウェイがないか確認
    route print
    ```

### 3. レッドチーム視点 (OSCP / Pivoting)

* **タスク**: 侵入端末を踏み台にした内部ネットワークへのピボッティング準備
* **解説**: 侵入した Windows 端末が別のセグメント（例: 10.10.10.0/24）にも足をかけている場合、その端末上で適切なルートを確認・設定し、自分の攻撃端末からのパケットを中継させる足がかりにします。
* **例**:
    ```cmd
    REM インターフェイス一覧とメトリックを確認して最適な経路を判断
    route print -4
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `名前を解決できませんでした。` または `引数が足りません。`
    * **考えられる原因**: `mask` キーワードを忘れている、または IP アドレスの形式が不正。
    * **解決策**: `route add [宛先IP] mask [サブネットマスク] [ゲートウェイ]` の順序を確認する。

2.  **エラーメッセージ例 2**: `要求された操作には、昇格（管理者として実行）が必要です。`
    * **考えられる原因**: ルーティングテーブルの変更はシステム全体に影響するため、管理者権限が必須。
    * **解決策**: 管理者権限でコマンドプロンプトを起動する。

## 環境変数と設定ファイル

`route` 自体は環境変数を持ちませんが、`-p` で追加された永続的なルートはレジストリに保存されます。
* **レジストリパス**: `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\PersistentRoutes`

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    - **Man-in-the-Middle (MitM)**: 攻撃者が自身の端末をゲートウェイとして登録させ、通信を盗聴・改ざんする。
    - **Network Pivoting**: 侵害した端末にスタティックルートを追加し、外部から隔離された内部セグメント（DBサーバー等）への攻撃経路を確立する。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Information Requirements` / `Pivoting`
* **参照**: [LOLBAS - route.exe](https://lolbas-project.github.io/lolbas/Binaries/Route/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 
    - ホスト型のファイアウォールで、予期せぬインターフェイス間転送（IP Forwarding）を無効にする。
* **Detection (検知)**: 
    - **イベントID 4688**: `route` コマンドの実行、特に `add` や `-p` スイッチの使用を監視。
    - 定期的にルーティングテーブルをエクスポートし、ベースラインと比較する。

## 注意点・補足

* **メトリック (Metric)**: 同じ宛先に対して複数の経路がある場合、メトリックの値が小さい方が優先されます。
* **0.0.0.0**: `0.0.0.0` 宛のルートは「デフォルトゲートウェイ」を意味します。
