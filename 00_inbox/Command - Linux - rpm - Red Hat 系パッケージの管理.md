# Command - Linux - rpm - Red Hat 系パッケージの管理

## 概要
`rpm` は、`.rpm` 形式のパッケージファイルをインストール、アンインストール、クエリ（照会）、検証するための強力なパッケージ管理システムです。システムに導入されているすべてのソフトウェアの状態をデータベースで管理しており、ファイルの改ざん検知などにも利用されます。



## 🛠 `rpm` vs `dnf` / `yum`
| 特徴 | `rpm` | `dnf` / `yum` |
| :--- | :--- | :--- |
| **依存関係** | **解決しない。** 足りないものがあるとエラー。 | **自動で解決・インストールする。** |
| **入手先** | ローカルの `.rpm` ファイル。 | リモートのリポジトリ（インターネット等）。 |
| **役割** | 個別のパッケージ操作、詳細な照会。 | システム全体の更新、ソフトの検索と導入。 |

## 主要なモードとオプション
`rpm` は、最初の 1 文字で「何をするか」を指定します。

| モード (大文字) | 説明 |
| :--- | :--- |
| ⭐️ `-i`, `--install` | パッケージを新規インストールする。 |
| ⭐️ `-U`, `--upgrade` | パッケージをアップグレード（または新規導入）する。 |
| ⭐️ `-e`, `--erase` | パッケージをアンインストールする。 |
| ⭐️ `-q`, `--query` | インストール済みパッケージを照会する。 |
| ⭐️ `-V`, `--verify` | パッケージの整合性（改ざん等）を確認する。 |

### よく使われる組み合わせ
* **`-ivh`**: インストール状況をハッシュマーク (`#`) で表示しながら実行。
* **`-qa`**: インストール済みの全パッケージをリストアップ。
* **`-qf <path>`**: 指定したファイルが「どのパッケージ」から入ったものか調べる。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: オフライン環境でのパッケージ導入。
* **解説**: インターネットに接続できないセキュアなサーバーでは、USBメモリ等で持ち込んだ `.rpm` ファイルを `rpm -ivh` で直接叩き込みます。

### 2. ブルーチーム視点
* **タスク**: システムファイルの整合性チェック（改ざん検知）。
* **解説**: ⭐️ **重要機能。** `rpm -V` を使うと、インストール時と現在のファイルを比較し、サイズ、パーミッション、MD5ハッシュ等の変更をあぶり出します。
* **例**:
    ```bash
    # すべてのパッケージを検証し、変更があったファイルのみ表示
    # (S: サイズ, 5: MD5, T: 時間, P: 権限 の変更を意味する)
    rpm -Va
    ```

### 3. レッドチーム視点
* **タスク**: ターゲット環境の偵察（Enumeration）。
* **解説**: `rpm -qa` でインストール済みのソフトとそのバージョンを全て取得し、既知の脆弱性（CVE）がある古いパッケージが残っていないか、あるいはセキュリティ製品（EDR等）が導入されていないかを調査します。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **署名の未検証**: `rpm --nosignature` でインストールすると、攻撃者が用意したバックドア入りのパッケージを掴まされるリスクがあります。
* **依存関係の無視**: `--nodeps` で強制インストールすると、システムが不安定になり、予期せぬ挙動（DoS状態など）を引き起こす可能性があります。

### 対応策・緩和策 (ブルーチーム視点)
* **GPG 鍵による検証**: インストール前に必ず `rpm --import <GPG-KEY>` で鍵を取り込み、パッケージの署名が正しいことを確認します。
* **AIDE 等との連携**: `rpm -V` だけに頼らず、AIDE などのホスト型 IDS を併用してバイナリの健全性を二重にチェックします。
