# `less` - 高機能なファイルビューア（ページャ）

## 概要

`less` は、テキストファイルやコマンドの標準出力を、1画面ずつ対話的に表示するための「ページャ」と呼ばれるプログラムです。`cat` コマンドのようにファイル全体を一気に出力するのではなく、画面のサイズに合わせて表示を止め、ユーザーが自由に上下にスクロールしたり、内容を検索したりできます。巨大なログファイルや設定ファイルを安全に、かつ快適に閲覧するための標準的なツールです。

(出自: `less` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`more`** | `less` の前身となった古典的なページャ。`more` は前方にしかスクロールできませんが、`less` は**後方にも自由にスクロールできる**点が最大の違いです。 | `less` |
| **`cat`** | ファイルの**全体**を一度に表示します。大きなファイルには `less` を使うのが適切です。 | - |
| **`tail -f`**| ファイルの**末尾**をリアルタイムで監視するのに特化しています。`less` も `F` キーで同様の「追跡モード」に入ることができます。 | `tail -f` (監視) / `less` (閲覧) |

## よく連携されるコマンド

`less` は、他のコマンドの長い出力を受け取って、ページごとに閲覧するためにパイプと連携するのが最も一般的な使い方です。

### シナリオ例: `journalctl`の長い出力を閲覧する (インフラ構築・運用視点)

* **目的**: `systemd` のジャーナルログ全体を、スクロールや検索をしながら快適に確認する。
* **連携コマンド**: `journalctl`, パイプ (`|`)
* **解説**: `journalctl` は非常に多くのログを出力することがあります。その出力をパイプで `less` に渡すことで、ログの先頭から末尾まで自由に行き来したり、特定のキーワードで検索したりしながら、じっくりと内容を分析できます。
* **コマンド例**:

    ```bash
    # journalctlの全出力をlessで閲覧する
    journalctl | less
    ```

## オプション説明 (`less` 内部のキー操作)

| オプション | 説明 |
| :--- | :--- |
| **画面操作と移動** | |
| ⭐ `q` | `less` を**終了**します。 |
| `Space` / `f` | 1画面**下**へ進みます (forward)。 |
| `b` | 1画面**上**へ戻ります (backward)。 |
| `j` / `k` | 1行**下** / **上**へ進みます。 |
| `g` / `G` | ファイルの**先頭** / **末尾**に移動します。 |
| **検索とフィルタ** | |
| ⭐ `/pattern` | `pattern` を現在位置から**前方（下方向）に検索**します。 |
| `?pattern` | `pattern` を現在位置から**後方（上方向）に検索**します。 |
| `n` / `N` | 次 / 前 の検索結果へ移動します。 |
| `&pattern`| `pattern` に**一致する行のみをフィルタリング表示**します。 |
| **表示切替** | |
| ⭐ `F` (大文字) | ファイルの末尾をリアルタイムで監視する「追跡モード」を開始/終了します (`tail -f` と同じ)。 |
| `-S` (起動時オプション) | 長い行を折り返さずに、左右スクロールで閲覧できるようにします。 |
| `-N` (起動時オプション) | 各行の先頭に行番号を表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 巨大なログファイルや設定ファイルを安全に閲覧する。
* **組み合わせ**: `less <logfile>`
* **解説**: `/` キーで `/POST` や `/192.168.1.10` のようにキーワード検索をしたり、`G` キーで末尾に移動して最新のログを確認したりします。
* **例**:

    ```bash
    # Nginxのアクセスログを開く
    less /var/log/nginx/access.log
    ```

### 2. ブルーチーム視点

* **タスク**: 大量のログから、特定のIOC（Indicator of Compromise）が関与する行だけを効率的に調査する。
* **組み合わせ**: `less <evidence_file>` → `&<IOC>`
* **解説**: `&` キーによるフィルタリング機能を使います。`&198.51.100.10` と入力することで、ログ全体からそのIPアドレスが関与する行だけを抜き出し、攻撃者の行動を時系列で追跡できます。
* **例**:

    ```bash
    # 巨大な認証ログを開く
    less /var/log/auth.log
    # 内部で `&Failed password` と入力し、ログイン失敗の行だけを抽出する
    ```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察として、システム上の様々なファイルを安全かつ隠密に閲覧する。
* **組み合わせ**: `less /etc/passwd` や `less config.php`
* **解説**: `cat` と違って画面が流れず、ログにも残りにくいため、隠密な調査に適しています。`/password` や `?api_key` のように検索機能を使って、ファイル内から効率的に認証情報を探し出します。
* **例**:

    ```bash
    # Webアプリケーションの設定ファイルからパスワードを探す
    less /var/www/html/config.php
    # 内部で `/password` と入力して検索
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **バイナリファイルを開くと文字化けし、警告が出る。**
    * **考えられる原因**: `less` はテキストファイル用のビューアです。実行ファイルなどのバイナリファイルを開こうとすると警告を表示します。
    * **解決策**: `y` を押せば強制的に表示できますが、通常は `strings` や `xxd` といったバイナリ調査用のコマンドを使うべきです。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 影響を与える環境変数

* **`LESS`**: `less` のデフォルトオプションを指定できます。`export LESS="-R -N"` としておくと、常に色付きで行番号が表示されるようになり便利です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `less` 自体ではなく、それが持つ**外部コマンド実行機能**が悪用される可能性があります。
* **悪用シナリオ**: `less` は、表示中のファイルから外部のエディタを起動 (`v`) したり、外部コマンドを実行 (`!`) したりする機能を持っています。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo less <any_file>` のように、`sudo` で `less` を実行できる権限を持っている場合、ページャの画面で `!/bin/sh` と入力することで、**root権限のシェル**を起動できてしまいます。
* **参照**: `https://gtfobins.github.io/gtfobins/less/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定で、ユーザーに `less` のようなページャの `sudo` 権限を安易に与えない。`sudoedit` を使うなど、より安全な方法を検討する。
* **Detection (検知)**: `auditd` を使い、`less` からの不審なプロセス生成（`execve`）がないか監視する。

## 注意点・補足

* **`less +F <file>`**: このように起動すると、最初から `tail -f` と同じ追跡モードでファイルを開けます。`Ctrl+C` で追跡を中断し、通常の `less` の操作に戻れるため、`tail -f` よりも高機能です。
