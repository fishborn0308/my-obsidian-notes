# Command - Windows - cmd - shutdown - システムのシャットダウン・再起動

## 概要

`shutdown` コマンドは、ローカルまたはリモートの Windows コンピューターをシャットダウン、再起動、またはログオフさせるためのコマンドです。
指定した時間（秒）の待機や、実行中のアプリケーションを強制終了させるオプション、さらにシャットダウンの理由を記録する機能も備えています。
(出自: Windows標準搭載)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `psshutdown` | Sysinternals製。リモートデスクトップがオフでも強制的に実行できるなど、より多機能。 | `psshutdown` (高度な管理) |
| `Stop-Computer` | PowerShell版。複数のコンピューターに対して一括で実行しやすい。 | `Stop-Computer` (PS) |
| `logoff` | ユーザーセッションの終了（ログオフ）に特化したコマンド。 | `logoff` (ログオフのみ) |

## よく連携されるコマンド

### シナリオ例: メンテナンス後の即時再起動 (インフラ運用)

* **目的**: パッチ適用や設定変更後、すべてのアプリを強制終了して即座に再起動する。
* **連携コマンド**: `timeout`, `echo`
* **解説**: ユーザーに通知を表示し、数秒の猶予を与えてから再起動を実行します。
* **コマンド例**:
    ```cmd
    REM 警告を表示し、0秒後に強制再起動
    echo システムメンテナンスのため再起動します。 && shutdown /r /f /t 0
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| **動作指定** | |
| ⭐ `/s` | コンピューターをシャットダウンする。 |
| ⭐ `/r` | コンピューターを再起動する。 |
| ⭐ `/a` | 実行中のシャットダウン/再起動処理を **中止 (Abort)** する。 |
| `/l` | ログオフする（`/m` リモート指定とは併用不可）。 |
| **実行オプション** | |
| ⭐ `/f` | 実行中のアプリケーションを警告なしで強制終了する。 |
| ⭐ `/t <xxx>` | シャットダウンまでの待機時間を秒単位で指定（デフォルトは30秒、`/t 0` で即時）。 |
| `/m \\<Computer>` | 指定したリモートコンピューターを対象にする。 |
| `/c "<Comment>"` | シャットダウン時に表示するメッセージ（最大512文字）を指定。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: リモートサーバーの計画外フリーズ対応
* **解説**: 管理端末からリモートで強制再起動を試みます。
* **例**:
    ```cmd
    REM リモートサーバーを理由コード付きで強制再起動
    shutdown /m \\Server01 /r /f /t 0 /c "Emergency Restart by Admin"
    ```

### 2. ブルーチーム視点

* **タスク**: 不審なシャットダウン履歴の調査
* **解説**: 攻撃者によってシステムがいつ、どのような理由で落とされたかを確認します。イベントログの ID をキーに調査します。
* **例**:
    ```cmd
    REM イベントビューアーで以下のIDを確認 (wevtutil等)
    REM ID 1074: シャットダウンの開始
    REM ID 6006: システムの正常終了
    ```


### 3. レッドチーム視点 (OSCP / Persistence)

* **タスク**: 永続化（Persistence）トリガーの強制発動
* **解説**: レジストリの `Run` キーや `Startup` フォルダにペイロードを仕込んだ後、再起動を強制することで、自分のコードを管理者権限等で確実に実行させます。
* **例**:
    ```cmd
    REM ペイロード設置後、即座に再起動してコードを実行させる
    shutdown /r /f /t 0
    ```
* **タスク**: Denial of Service (DoS)
* **解説**: ターゲットの可用性を奪うために、ループ内で `shutdown` を実行し続けるなどの嫌がらせや、単純な業務妨害として利用されます。

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `アクセスが拒否されました。(5)`
    * **考えられる原因**: 管理者権限がない、またはリモートシャットダウンに必要な権限（`SeShutdownPrivilege`）が不足している。
    * **解決策**: 管理者として実行するか、`whoami /priv` で権限を確認する。

2.  **現象**: `シャットダウンは既にスケジュールされています。(1190)`
    * **原因**: 別のシャットダウン命令が既にキューに入っている。
    * **解決策**: 一度 `shutdown /a` でキャンセルしてから再実行する。

## 環境変数と設定ファイル

* **権限設定**: `ローカル セキュリティ ポリシー (secpol.msc)` -> `ユーザー権利の割り当て` -> `システムのシャットダウン` にて実行可能なユーザーを制限できます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    - **Persistence Activation**: インストールしたバックドアやルートキットを有効化するための再起動。
    - **Anti-Forensics**: フォレンジックツールによるメモリ調査を逃れるために、即座に電源を落としてメモリ内のデータを揮発させる。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Standard Command`
* **参照**: [LOLBAS - shutdown.exe](https://lolbas-project.github.io/lolbas/Binaries/Shutdown/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 
    - サーバー環境では、一般ユーザーから `SeShutdownPrivilege` を剥奪し、勝手に電源を落とせないようにする。
* **Detection (検知)**: 
    - **イベントID 1074**: どのプロセス（またはユーザー）がシャットダウンを要求したかを記録。不審なプロセス名（例: `revshell.exe`）からの要求を監視する。

## 注意点・補足

* **`/t 0` と `/f`**: `/t` で 0 を指定した場合、`/f` は暗黙的に適用されます。
* **GUI版**: `shutdown /i` を実行すると、GUI ベースの「リモート シャットダウン ダイアログ」が表示され、複数のコンピューターを視覚的に管理できます。
