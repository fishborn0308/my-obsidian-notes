# Command - Linux - nano - 小回りの効く、ユーザーフレンドリーなテキストエディタ

## 概要
`nano` は、ターミナルで動作するシンプルで直感的なテキストエディタです。`vi` や `emacs` のような独特の「モード」や複雑なコマンド体系を持たず、画面下部に常に主要なショートカットキーが表示されているため、誰でもすぐに使いこなすことができます。
(出自: `Pico` エディタの自由なクローンとして誕生)



## 🛠 エディタの比較
| 特徴 | `nano` | `vi` / `vim` | `emacs` |
| :--- | :--- | :--- | :--- |
| **学習コスト** | **極めて低い** | 高い（モード切替） | 非常に高い（独自の哲学） |
| **起動速度** | 爆速 | 爆速 | やや重い |
| **普及率** | ほぼ全ての Linux | **絶対的標準** | 選択肢として存在 |
| **主な用途** | クイックな設定変更 | 本格的なコーディング | 全環境の統合管理 |

## 主要な操作（ショートカットキー）
`nano` では、`Ctrl` キーを **`^`** と表記します。

| キー | アクション | 説明 |
| :--- | :--- | :--- |
| ⭐️ `Ctrl + O` | **保存 (Write Out)** | ファイルをディスクに書き込みます。 |
| ⭐️ `Ctrl + X` | **終了 (Exit)** | エディタを閉じます。変更があれば保存を確認されます。 |
| ⭐️ `Ctrl + K` | **カット (Cut)** | 現在の行を切り取ります。 |
| ⭐️ `Ctrl + U` | **貼り付け (Uncut)** | 切り取った行を貼り付けます。 |
| ⭐️ `Ctrl + W` | **検索 (Where Is)** | ファイル内の文字列を検索します。 |
| ⭐️ `Ctrl + C` | **位置確認** | カーソルの現在の行・列番号を表示します。 |

## よく使われるオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-w`, `--nowrap` | 長い行を自動で折り返さない。**設定ファイルの編集では必須。** |
| ⭐️ `-m`, `--mouse` | マウスによるクリックでカーソル移動やスクロールを可能にする。 |
| ⭐️ `-v`, `--view` | 読み取り専用モードで開く（間違えて書き換えるのを防ぐ）。 |
| `-c`, `--constantshow` | 常にカーソル位置（行・列）を表示し続ける。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: `/etc/hosts` や `/etc/fstab` の迅速な編集。
* **解説**: `vi` の操作に慣れていないメンバーが混在するチームでも、`nano` なら操作ミスによる事故を減らせます。また、SSH 経由でのちょっとしたスクリプト作成にも最適です。
* **例**:
    ```bash
    # 行を折り返さずに設定ファイルを開く
    sudo nano -w /etc/netplan/00-installer-config.yaml
    ```

### 2. ブルーチーム視点
* **タスク**: 設定ファイルの不用意な権限昇格の防止。
* **解説**: `nano` は `vi` と違い、エディタ内からシェルを呼び出す機能（シェルエスケープ）が標準では貧弱です。そのため、一時的な作業を一般ユーザーに許可する際、`sudo vi` よりも `sudo nano` を使わせる方がセキュリティリスクを抑えられる場合があります。

### 3. レッドチーム視点
* **タスク**: ペイロード（攻撃コード）の素早い作成と修正。
* **解説**: 侵入した環境で `vi` のキー設定が壊れていたり、複雑な操作を履歴に残したくない場合、`nano` は「目立たず、確実に、素早く」ファイルを編集するための信頼できるツールになります。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **一時ファイルの露出**: `nano` が異常終了した際などに作成される一時ファイル（`.save` 等）に、編集中の機密情報が残ってしまうリスクがあります。
* **Sudo の悪用**: `sudo nano` を許可している場合、設定ファイルを改ざんされる（例: `/etc/passwd` を書き換えて自分を root にする）リスクは他のエディタと同じです。

### 対応策・緩和策
* **環境変数の設定**: デフォルトのエディタを `nano` にしたい場合は、`export EDITOR=nano` を `.bashrc` に追記します。
* **履歴の管理**: 重要なファイルを編集した後は、`.nano_history` に残った記録に機密情報が含まれていないか確認しましょう。

## 注意点・補足
* **`-w` オプションの重要性**: `nano` はデフォルトで行を勝手に折り返すことがあり、これに気づかず設定ファイルを保存すると、1 行であるべき設定が 2 行に分断され、サービスが起動しなくなる致命的なミスに繋がることがあります。設定ファイルを触る際は、常に `-w` を付ける癖をつけましょう。
