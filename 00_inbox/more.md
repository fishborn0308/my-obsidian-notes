# `more` - ファイルを1画面ずつ表示するページャ (旧)

## 概要

`more` は、テキストファイルやコマンドの標準出力を、1画面ずつ表示するための古典的な「ページャ」プログラムです。ファイル全体を一度に表示する `cat` とは異なり、画面がいっぱいになると表示を停止し、ユーザーのキー入力を待ちます。これにより、巨大なファイルの内容を最初から順番に読むことができます。

**⚠️ 重要**: `more` は、後方スクロールができないなど機能が限定的であるため、**レガシー（旧式）**なコマンドと見なされています。特別な理由がない限り、後方スクロールや高機能な検索が可能な **`less` の使用が強く推奨されます**。

(出自: `util-linux` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`less`** | `more` の高機能な後継。`less` は**前方にも後方にも自由にスクロール**できますが、`more` は基本的に前方にしか進めません。「`less` is `more`」という言葉で知られます。 | `less` |
| **`cat`** | ファイルの**全体**を一度に標準出力に表示します。`more` は1画面ずつ表示を停止します。 | - |

## よく連携されるコマンド

`more` は、他のコマンドの長い出力を受け取って、ページごとに閲覧するためにパイプと連携するのが最も一般的な使い方です。

### シナリオ例: 長いプロセスリストを確認する (インフラ構築・運用視点)

* **目的**: `ps aux` のように大量の行を出力するコマンドの結果を、画面が流れてしまわないように確認する。
* **連携コマンド**: `ps`, パイプ (`|`)
* **解説**: `ps aux` の出力をパイ-プで `more` に渡すことで、1画面ずつ停止させながら、じっくりとプロセスリストを確認できます。
* **コマンド例**:

```bash
# プロセス一覧を1画面ずつ確認する
ps aux | more
```

## オプション説明 (`more` 内部のキー操作)

| オプション | 説明 |
| :--- | :--- |
| **画面操作** | |
| ⭐ `q` | `more` を**終了**します。 |
| ⭐ `Space` | 1画面**下**へ進みます。 |
| ⭐ `Enter` | 1行**下**へ進みます。 |
| **検索** | |
| `/pattern` | `pattern` という文字列を、現在位置から**前方（下方向）に検索**します。 |
| `n` | 次の検索結果へ移動します。 |
| **その他** | |
| `=` | 現在の行番号を表示します。 |
| `h` | ヘルプを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 設定ファイルの内容を順番に確認する。
* **組み合わせ**: `more <config_file>`
* **解説**: `less` が使えない最小構成の環境などで、設定ファイルの内容を最初から順番に読んでいく際に使用します。
* **例**:

```bash
# /etc/services ファイルの内容を閲覧する
more /etc/services
```

### 2. ブルーチーム視点

* **タスク**: `less` が利用できないような、非常に古いシステムやレスキュー環境でログファイルを確認する。
* **組み合わせ**: `more <logfile>`
* **解説**: 古い産業用制御システムや組込み機器のログを調査する必要があるが、利用可能なコマンドが `more` と `grep` に限られている場合などに、ログを読み進め、`/` 機能で不審なIPアドレスなどを検索します。
* **例**:

```bash
# 古いシステムのログを確認
more /var/log/syslog.1
```

### 3. レッドチーム視点

* **タスク**: 侵入したシステムが非常に古い、または制限された環境である場合に、偵察のために `more` を使用する。
* **組み合わせ**: `more /etc/passwd`
* **解説**: 攻撃者は、`less` や `cat` が使えない環境でも `more` が存在することを知っています。`more` を使って、設定ファイルやワールドリーダブルなログファイルを閲覧し、機密情報を探し出します。
* **例**:

```bash
# パスワードファイルの内容を閲覧
more /etc/passwd
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **現象**: **上の行に戻りたいが、スクロールできない。**
    * **考えられる原因**: それが `more` コマンドの仕様です。`more` は前方に進むことしかできません。
    * **解決策**: `q` で一度終了し、再度ファイルを開き直すか、最初から `less` コマンドを使用してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `more` 自体ではなく、それが持つ**外部コマンド実行機能**が悪用される可能性があります。
* **悪用シナリオ**: `more` は `less` と同様に、`!` コマンドで外部コマンドを実行する機能を持っています。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation** (権限昇格)
* **解説**: もしユーザーが `sudo more <any_file>` のように `sudo` で `more` を実行できる権限を持っている場合、ページャの画面で `!/bin/sh` と入力することで、**root権限のシェル**を起動できてしまいます。
* **参照**: `https://gtfobins.github.io/gtfobins/more/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `sudoers` の設定で、ユーザーに `more` のようなページャの `sudo` 権限を安易に与えない。`sudoedit` を使うなど、より安全な方法を検討する。
* **Detection (検知)**: `auditd` を使い、`more` からの不審なプロセス生成（`execve`）がないか監視する。

## 注意点・補足

* **`less` への移行**: `more` の全ての機能は `less` に含まれており、さらに `less` は後方スクロールや高機能なフィルタリングなど、多くの優れた機能を持っています。特別な理由がない限り、**常に `less` を使う習慣**をつけましょう。
