# `mkdir` - 新しいディレクトリを作成する

## 概要

`mkdir` (make directory) は、新しいディレクトリ（または複数のディレクトリ）を作成するための基本的なコマンドです。ファイルやプロジェクトを整理するための階層構造を作る際に使用します。

(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`touch`** | サイズが0の空の**ファイル**を作成します。`mkdir` は**ディレクトリ**を作成します。 | - |
| **`cp -r`** | 既存のディレクトリを**コピー**することで、新しいディレクトリを作成します。`mkdir` は空のディレクトリを新規に作成します。 | - |
| **`install -d`**| `mkdir -p` と同様に、親ディレクトリも含めてディレクトリを作成します。主にシェルスクリプトやMakefile内で、パーミッションや所有者を同時に設定したい場合に使われます。 | `install -d` (スクリプト内) |

## よく連携されるコマンド

`mkdir` は、ディレクトリを作成した直後にそのディレクトリに移動する `cd` や、パーミッションを設定する `chmod` とよく連携されます。

### シナリオ例: 作業ディレクトリを作成してすぐに移動する (インフラ構築・運用視点)

* **目的**: 新しいプロジェクト用のディレクトリを作成し、すぐにそのディレクトリを作業場所として使い始める。
* **連携コマンド**: `cd`
* **解説**: `&&` を使うことで、`mkdir` が成功した場合にのみ、続けて `cd` コマンドが実行されます。
* **コマンド例**:

```bash
# 'new-project' ディレクトリを作成し、成功したらその中に移動する
mkdir new-project && cd new-project
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **ディレクトリ作成** | |
| ⭐ `-p`, `--parents` | 指定したディレクトリの親ディレクトリが存在しない場合、それらもまとめて作成します。また、指定したディレクトリが既に存在していてもエラーになりません。 |
| `-m`, `--mode=<mode>`| ディレクトリのパーミッションを八進数 (`755`など) で指定して作成します。 |
| **出力制御** | |
| `-v`, `--verbose` | 作成された各ディレクトリに関するメッセージを表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 複数階層のディレクトリを1回のコマンドで確実に作成する。
* **組み合わせ**: `mkdir -p /path/to/nested/directory`
* **解説**: `-p` (`--parents`) オプションは、スクリプト内でのディレクトリ作成における事実上の標準です。スクリプトを何度実行しても「既に存在する」というエラーが出ないため、冪等性が保たれます。
* **例**:

```bash
# プロジェクト用のディレクトリ構造を一括で作成
mkdir -p my-project/{src,docs,tests,build}
```

### 2. ブルーチーム視点

* **タスク**: 証拠ファイルを隔離するための、所有者のみがアクセスできるディレクトリを作成する。
* **組み合わせ**: `mkdir -m 700 /path/to/quarantine`
* **解説**: `-m` (`--mode`) オプションを使い、特定のパーミッションを持つディレクトリを作成します。`700` (`rwx------`) に設定することで、所有者 (`root`など) のみが出入りできるディレクトリとなり、隔離したファイルを安全に保管できます。
* **例**:

```bash
# 隔離用ディレクトリを作成
sudo mkdir -m 700 /var/log/quarantine
```

### 3. レッドチーム視点

* **タスク**: **永続化 (Persistence)** と**隠蔽 (Evasion)** のために、隠しディレクトリを作成する。
* **組み合わせ**: `mkdir /path/to/.hidden_dir`
* **解説**: 攻撃者は侵入後、自身のツールや窃取したデータを保管するために、正規のディレクトリに見せかけた隠しディレクトリを作成します。`.` で始まるディレクトリは `ls` のデフォルト表示では現れないため、発見を困難にします。
* **例**:

```bash
# ユーザーのホームディレクトリに、正規のディレクトリに見せかけた隠しディレクトリを作成
mkdir -p ~/.config/systemd/updates
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `mkdir: cannot create directory ‘a/b/c’: No such file or directory`
    * **考えられる原因**: 親ディレクトリ (`a/b`) が存在しない状態で、その中に `c` を作成しようとしました。
    * **解決策**: `-p` オプションを使用してください (`mkdir -p a/b/c`)。

## 環境変数と設定ファイル

* `mkdir` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **不適切なパーミッション**。`-m 777` のように、誰でも書き込める (`world-writable`) パーミッションでディレクトリを作成することは、重大なセキュリティリスクとなります。
* **悪用シナリオ**: 誰でも書き込める `/tmp` のようなディレクトリでは、あるプロセスがファイルを作成する前に、攻撃者が同名のディレクトリを `mkdir` で作成してしまうことで、プロセスの動作を妨害したり意図しない場所にファイルを書き込ませたりする競合状態 (Race Condition) 攻撃の可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: もしユーザーが `sudo mkdir` を実行できる権限を持っている場合、`/etc/cron.d/` のような誰でも書き込めるcronディレクトリを作成し、そこにcronジョブを仕掛ける、といった攻撃が考えられます。
* **参照**: `https://gtfobins.github.io/gtfobins/mkdir/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: `umask` を適切に設定し、デフォルトで作成されるディレクトリのパーミッションを安全なものにする。スクリプト内で一時ディレクトリを作成する場合は `mktemp -d` を使用する。
* **Detection (検知)**: `/etc` などの重要なシステムディレクトリ配下での `mkdir` の実行を `auditd` で監視する。

## 注意点・補足

* **`umask` との関係**: `-m` オプションを指定しない場合、作成されるディレクトリのパーミッションは、システムの `umask` の設定値に影響を受けます。通常は `777` から `umask` 値を引いた権限（例: `umask`が`022`なら`755`）になります。
