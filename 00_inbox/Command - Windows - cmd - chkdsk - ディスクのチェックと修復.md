# Command - Windows - cmd - chkdsk - ディスクのチェックと修復

## 概要

`chkdsk` (Check Disk) は、指定したディスクボリュームのファイルシステム整合性を検査し、論理的なエラーを修復するコマンドです。また、物理的な不良セクタを特定し、そこから読み取り可能な情報を回復する機能も備えています。

(出自: `Windows 標準搭載 - 外部コマンド (chkdsk.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `sfc /scannow` | Windows の**システムファイル**の破損を修復する。 | OSの挙動不審時 |
| `dism` | Windows イメージの破損を修復する。`sfc` で直らない場合に有効。 | OS深部の修復 |
| `Repair-Volume` | PowerShell 版。オンラインでのスキャンや修復が可能。 | PS環境 |

## よく連携されるコマンド

### シナリオ例: システムドライブの整合性修復 (トラブルシューティング)

* **目的**: OSの動作が不安定、またはファイル読み込みエラーが発生した際に、Cドライブの論理エラーを修復する。
* **連携コマンド**: `chkdsk`, `shutdown`
* **解説**: 使用中のボリューム（特にCドラ）は即座に修復できないため、再起動時の実行予約を行います。
* **コマンド例**:
    ```cmd
    REM Cドライブのエラーを修復予約（再起動が必要）
    chkdsk C: /f
    ```



## スイッチ/オプション説明

**※実行には「管理者権限」が必要です。**

| スイッチ | 説明 |
| :--- | :--- |
| **引数なし** | 現在のドライブを読み取り専用モードでスキャンし、状態を表示する（修復はしない）。 |
| ⭐ `/F` | ディスクのエラーを修復する。ディスクがロックされている必要があります。 |
| ⭐ `/R` | 不良セクタを特定し、読み取り可能な情報を回復する（`/F` の機能を包含）。 |
| `/X` | 必要に応じて、ボリュームを強制的にアンマウントする。 |
| `/V` | スキャン中にすべてのファイルのパスと名前を表示する（詳細モード）。 |
| `/C` | (NTFSのみ) フォルダ構造内のサイクルチェックをスキップする。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーの定期メンテナンスや、バックアップ取得前にファイルシステムの健全性を確認する。
* **例**:
    ```cmd
    REM Dドライブ（データ用）を強制アンマウントして即座に修復
    chkdsk D: /f /x
    ```

### 2. ブルーチーム視点 (デジタルフォレンジックの前段)

* **タスク**: 証拠保全（イメージ取得）を行う前に、ファイルシステムが破損していないか確認する。
* **注意**: `/f` や `/r` を実行すると**メタデータが書き換わる**ため、証拠保全の厳密性が求められる場合は「読み取り専用（引数なし）」での確認に留めるのが鉄則です。

### 3. レッドチーム視点 (偵察と妨害)

* **タスク 1: 物理的環境の調査 (Enumeration)**
    * **目的**: ディスクの総容量、隠しセクタの存在、不良セクタの割合から、ターゲット端末の「古さ」や「メンテナンス状況」を推測する。
* **タスク 2: サービス拒否 (DoS) への悪用**
    * **目的**: 管理者権限を奪取後、`/r` スイッチで巨大なドライブのスキャンを開始し、ディスク I/O を占有してシステムを重くさせる、あるいは再起動を強要してダウンタイムを発生させる。

## エラーメッセージとトラブルシューティング

1.  **Access Denied as you do not have sufficient privileges.**
    * **原因**: 管理者権限がない。
    * **解決策**: 「管理者として実行」したプロンプトで実行。

2.  **Chkdsk cannot run because the volume is in use by another process.**
    * **原因**: 対象ドライブが使用中。
    * **解決策**: 次回システム再起動時に実行するか (`Y` を入力)、`/X` で強制解除を試みる。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **アンチ・フォレンジック**: 
    * `/r` による修復プロセスはディスク上の未使用領域や不良セクタを再スキャンするため、削除されたファイルの痕跡（スラックスペース）を上書きし、データ復元を困難にさせる目的で悪用される可能性があります。

### 対応策・緩和策 (ブルーチーム視点)

* **モニタリング**: 
    * 業務時間外に予期せぬ `chkdsk` が実行されていないか、イベントログ（Source: `Wininit` または `chkdsk`）を監視します。

## 注意点・補足

* **実行時間**: `/R` スイッチはディスク全域をセクタ単位でスキャンするため、大容量 HDD では数時間〜数日かかる場合があります。
* **SSD での利用**: SSD に対して `/R` を行うのは、寿命（書き換え回数）の観点からあまり推奨されません。論理エラーの `/F` 程度に留めるのが現代的な運用です。
