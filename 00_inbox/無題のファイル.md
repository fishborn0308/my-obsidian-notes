# 構築手順書（追加・強化編）

## 1. ターミナル環境の強化（zsh, fzf, zoxide）

標準のzshをより使いやすくします。ディレクトリ移動の `z` は、より高速でモダンな後継ツールの `zoxide` を使用し、エイリアスで `z` として動かします。

1. **ツールのインストール:**
```bash
sudo apt update
sudo apt install -y flameshot fzf zoxide zsh-autosuggestions zsh-syntax-highlighting

```


2. **`.zshrc` への設定追記:**
以下の内容を `~/.zshrc` の末尾に追加します。
```bash
# zoxide (z) の初期化
eval "$(zoxide init zsh --cmd z)"

# zsh拡張の読み込み
source /usr/share/zsh-autosuggestions/zsh-autosuggestions.zsh
source /usr/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh

# fzf の履歴検索を Ctrl+R で使いやすくする設定
source /usr/share/doc/fzf/examples/key-bindings.zsh

```



## 2. スクリーンショット（Flameshot）の設定

OSCPのレポートには「実行したコマンド」と「結果」の証拠が不可欠です。

1. **起動確認:**
ターミナルで `flameshot gui` を実行し、範囲選択ができるか確認します。
2. **ショートカット登録 (重要):**
Kaliの [Settings] > [Keyboard] > [Application Shortcuts] から、以下のショートカットを登録すると効率が劇的に上がります。
* **Command:** `flameshot gui`
* **Shortcut:** `Print` キー（または好みのキー）



## 3. HTTP Server の拡張（Updog）

Python標準の `http.server` は便利ですが、ファイルのアップロードができない欠点があります。OSCPではターゲットからファイルを回収することもあるため、アップロード対応の `updog` が便利です。

1. **pipx でインストール:**
```bash
pipx install updog

```


2. **使い方:**
```bash
# カレントディレクトリを80番ポートで公開（パスワード設定も可能）
updog -p 80

```

# 強化版：環境構築完了チェックリスト

### □ ターミナル・操作効率

* [ ] **z (zoxide):** 何度か `cd` で移動した後、 `z <ディレクトリ名の一部>` でジャンプできるか？
* [ ] **fzf:** `Ctrl + R` を押した際、過去のコマンド履歴が曖昧検索できるか？
* [ ] **zsh拡張:** コマンド入力中に薄く候補（autosuggestions）が表示され、右矢印キーで確定できるか？
* [ ] **zsh拡張:** 入力中のコマンドが正しい場合に緑色、間違っている場合に赤色（syntax-highlighting）になるか？

### □ スクリーンショット（証拠保全）

* [ ] **Flameshot:** 指定したショートカットキーで即座に範囲選択画面が起動するか？
* [ ] **Flameshot:** 選択範囲に矢印や枠線、モザイクをその場で入れられるか？
* [ ] **保存先:** キャプチャした画像が指定のディレクトリ（例：`~/oscp/screenshots`）にスムーズに保存できるか？

### □ ファイル転送

* [ ] **Updog:** `updog -p 80` で起動し、ブラウザからアクセスしてファイルのアップロード・ダウンロードができるか？
* [ ] **Python Alias:** （予備として） `alias pserv='python3 -m http.server 80'` が `.zshrc` に設定されているか？

### 活用のポイント

* **fzf + 履歴:** 以前打った複雑な `msfvenom` のペイロード作成コマンドなどを、数文字の入力で呼び出せるようになります。
* **z の学習:** `z` は一度訪れたディレクトリを学習するため、構築直後に `~/oscp/labs` や `/usr/share/wordlists` などによく使う場所へ一度 `cd` しておくと、次から数文字で移動できるようになります。
* **スナップショットの再取得:** これらの設定が完了し、動作確認ができたら、再度VMのスナップショットを更新（または新規作成）しておくことを強くお勧めします。
