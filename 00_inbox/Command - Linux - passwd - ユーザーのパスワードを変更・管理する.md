# Command - Linux - passwd - ユーザーのパスワードを変更・管理する

## 概要
`passwd` は、ユーザーのログインパスワードを変更します。一般ユーザーは自分のパスワードのみ変更できますが、root ユーザーはシステム上の全ユーザーのパスワードを変更でき、さらにアカウントのロックや有効期限の設定も可能です。



## 🛠 関連コマンド・ファイル
| 項目 | 役割 |
| :--- | :--- |
| `/etc/passwd` | ユーザー情報の基本名簿（パスワード自体はここにはない）。 |
| `/etc/shadow` | **ハッシュ化されたパスワード**と期限情報が格納される重要ファイル。 |
| `chage` | パスワードの有効期限や強制変更の設定に特化したコマンド。 |
| `openssl passwd` | スクリプト等で利用するハッシュ化された文字列を生成する。 |

## 主要なオプション（管理者向け）
※一般ユーザーは引数なしで実行するのが基本です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-l`, `--lock` | 指定したユーザーを**ロック**する（ログイン不可にする）。 |
| ⭐️ `-u`, `--unlock` | ユーザーのロックを解除する。 |
| ⭐️ `-d`, `--delete` | パスワードを削除し、**無効（空）**にする（非常に危険）。 |
| ⭐️ `-e`, `--expire` | 次回のログイン時に**パスワード変更を強制**する。 |
| ⭐️ `-S`, `--status` | アカウントの状態（ロック中か、最終変更日はいつか等）を表示する。 |
| `--stdin` | 標準入力からパスワードを受け取る（スクリプト用 ※非推奨の環境あり）。 |

## ⚙️ 仕組みの裏側
`passwd` コマンドを実行すると、以下のプロセスが発生します：
1.  入力された平文パスワードを強力なアルゴリズム（2026年現在は `yescrypt` や `argon2` が主流）でハッシュ化。
2.  **`/etc/shadow`** ファイルを更新。
> [!NOTE]
> `/etc/shadow` は root しか書き込めませんが、`passwd` バイナリには **Set-UID (SUID)** ビットが立っているため、一般ユーザーが実行しても「一時的に root 権限で」ファイルを更新できるようになっています。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 新規ユーザーへの初期パスワード割り当て。
* **解説**: セキュリティ上、初期パスワードのまま使い続けさせるのは厳禁です。
* **例**:
    ```bash
    # パスワードを設定し、初回ログイン時に変更を強制する
    sudo passwd newuser
    sudo passwd -e newuser
    ```

### 2. ブルーチーム視点
* **タスク**: 退職者や不審なアカウントの即時無効化。
* **解説**: 削除（`userdel`）する前に、まずは `passwd -l` でロックをかけ、セッションを遮断するのが鉄則です。
* **例**:
    ```bash
    # 攻撃に使われている可能性のあるアカウントを凍結
    sudo passwd -l suspect_user
    ```

### 3. レッドチーム視点
* **タスク**: 永続化（Persistence）のためのパスワード変更。
* **解説**: 権限昇格に成功した後、既存ユーザーのパスワードを自分の知っているものに変更して、いつでも再侵入できるようにします。ただし、`passwd` を叩くと `/etc/shadow` のタイムスタンプが変わるため、慎重な攻撃者は `sed` などでファイルを直接弄ることを好みます。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **Weak Passwords**: 短い、あるいは辞書に載っているようなパスワードは数秒でクラックされます。
* **SUID の悪用**: `passwd` バイナリ自体に脆弱性があると、一般ユーザーが root 権限を奪取する足がかりになります。

### 対応策・緩和策 (ブルーチーム視点)
* **PAM (Pluggable Authentication Modules)**: `/etc/pam.d/common-password` 等で、パスワードの最低文字数や複雑さの要件（大文字、数字、記号の混在）を強制します。
* **パスワードの有効期限**: 定期的な変更を促す設定を `chage` または `passwd -x` で行います。
