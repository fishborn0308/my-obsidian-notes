# Command - Linux - htop - インタラクティブなシステムモニター・プロセスビューアー

## 概要
`htop` は、システムの CPU、メモリ、スワップの使用状況をバーグラフで視覚的に表示し、実行中のプロセスを一覧表示するツールです。標準の `top` とは異なり、画面の縦横スクロールが可能で、キー入力やマウス操作によってプロセスの終了（Kill）や優先度の変更（Nice値）を直感的に行えます。
(出自: Hisham Muhammad によって開発された)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `top` | ほぼ全ての Unix 系に標準搭載されている。質素だがどこでも動く。 | 標準環境での確認 |
| `btop / bpytop` | さらにグラフィカルでモダン。ディスク I/O やネットワークも統合。 | **見た目と情報量重視** |
| `glances` | Web ブラウザ経由での監視も可能。システム全体を俯瞰。 | 統合モニタリング |
| `htop` | 操作性が抜群で、プロセス管理に特化。多くの環境で導入が容易。 | **日常のトラブル調査** |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: CPU 負荷の異常なスパイク（急上昇）の特定。
* **解説**: どのプロセスが、どのコアを占有しているかをバーグラフで確認します。
* **便利キー**: `P` (CPU順ソート)、`M` (メモリ順ソート)、`T` (実行時間順ソート)。
* **例**:
    ```bash
    # 特定のユーザーのプロセスだけを監視したい場合
    htop -u user_name
    ```

### 2. ブルーチーム視点
* **タスク**: 不審なプロセスの発見。
* **解説**: 見慣れない名前のプロセスや、一時ディレクトリ（`/tmp` 等）から実行されているプロセスがないか、ツリー表示 (`F5`) で親プロセスを確認しながら調査します。
* **便利キー**: `s` (プロセスが発行しているシステムコールを `strace` で追跡)、`l` (プロセスが開いているファイルを `lsof` で表示)。

### 3. レッドチーム視点
* **タスク**: ターゲット環境のリソース制限の把握。
* **解説**: 自分の実行したペイロードがどれだけ目立っているか、メモリを食いつぶしていないかを確認します。また、他のユーザーが実行しているコマンドライン引数（パスワードが混じっていないか等）を監視することもあります。

## エラーメッセージとトラブルシューティング

1.  **「htop: command not found」**:
    * **原因**: 標準ではインストールされていないことが多い。
    * **解決策**: `sudo apt install htop` (Debian系) や `sudo dnf install htop` (RedHat系) で導入。

2.  **「プロセスの優先度を変えられない / Kill できない」**:
    * **原因**: 自分以外のユーザー（特に root）のプロセスを操作しようとした。
    * **解決策**: `sudo htop` で起動する（※ただし操作には細心の注意が必要）。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `htop` の画面には、プロセスが実行された際のコマンドライン引数が表示されます。もしスクリプト等にパスワードを直接渡している場合、`htop` を見ている全ユーザーにそれが漏洩（Information Disclosure）するリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **Hidepid**: `/proc` ファイルシステムを `hidepid=2` オプションでマウントすることで、一般ユーザーが他人のプロセス情報を見られないように制限できます。
* **Sudo の制限**: 一般ユーザーに対して `sudo htop` を許可すると、そこから任意のプロセスを Kill したり、OS の動作を妨害したりできるため、権限付与には慎重さが求められます。

## 知っておくと便利なショートカット
* `u`: 指定したユーザーのプロセスのみを表示。
* `H`: ユーザースレッドの表示/非表示。
* `K`: カーネルスレッドの表示/非表示。
* `space`: 複数のプロセスを「マーク」して、一括で Kill 等の操作を行う。
* `/`: 検索（F4 と同じ）。
