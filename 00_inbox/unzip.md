# Command  - Linux - unzip - .zip形式のアーカイブを展開（解凍）する

## 概要

`unzip` は、ZIP形式のアーカイブファイル (`.zip`) に格納されているファイルを**展開（解凍）**したり、**内容を一覧表示**したり、**整合性をテスト**したりするためのユーティリティです。ZIP形式は、特にWindows環境で広く使われている標準的なアーカイブフォーマットであり、複数のファイルやディレクトリを1つのファイルにまとめることができます。

(出自: `unzip` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`tar`** | Linux/Unix環境における標準的な**アーカイバ**。パーミッションや所有者情報の保持に優れています。圧縮は `gzip` (`-z`) や `bzip2` (`-j`) と組み合わせて行います。 | `tar` (Linux環境) |
| **`gunzip` / `bunzip2`**| これらは**単一のファイル**を解凍するためのツールであり、複数のファイルを含むアーカイブは扱えません。 | - |
| **`7z`** | `zip` よりも高い圧縮率を誇る `7-Zip` 形式などを扱う高機能なアーカイバ。 | `7z` (高圧縮) |

## よく連携されるコマンド

`unzip` は、ダウンロードしたファイルを展開したり、`find` で見つけた複数のZIPファイルを一括で展開する際に使われます。

### シナリオ例: 複数のZIPファイルを一括で展開する (インフラ構築・運用視点)

* **目的**: 特定のディレクトリにある全ての `.zip` ファイルを、それぞれ同名のサブディレクトリに展開する。
* **連携コマンド**: `find`, `sh`
* **解説**: `find` で `.zip` ファイルを検索し、`-exec` オプションを使って見つかった各ファイルに対して `unzip` を実行するシェルコマンドを呼び出します。
* **コマンド例**:

    ```bash
    # カレントディレクトリ以下の全 .zip ファイルを、それぞれファイル名から拡張子を除いたディレクトリに展開
    find . -name "*.zip" -exec sh -c 'unzip -d "${0%.*}" "$0"' {} \;
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **情報表示とテスト** | |
| ⭐ `-l` | (list) アーカイブを**展開せずに、中身のファイル一覧のみを表示**します。**安全確認に非常に重要です**。 |
| ⭐ `-t` | (test) アーカイブファイルが破損していないか、**整合性をテスト**します。 |
| **展開操作** | |
| ⭐ `-d <dir>` | (directory) 展開先のディレクトリを指定します。デフォルトはカレントディレクトリです。 |
| `-o` | (overwrite) 同名のファイルが既に存在する場合、**確認なしで上書き**します。 |
| `-n` | (no overwrite) 同名のファイルが既に存在する場合、**上書きせずにスキップ**します。 |
| `-j` | (junk paths) アーカイブ内のディレクトリ構造を無視し、全てのファイルを**同じディレクトリに展開**します。 |
| **その他** | |
| `-q` | (quiet) 実行中のメッセージを抑制し、静かに出力します。 |
| `-P <password>`| パスワードで暗号化されたZIPアーカイブを展開する際に、パスワードを指定します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: アプリケーションのパッケージ (`.zip`) をサーバー上で展開してデプロイする。
* **組み合わせ**: `unzip <archive.zip> -d /path/to/destination`
* **解説**: ダウンロードしたZIPアーカイブを、`-d` オプションで指定したデプロイ先のディレクトリに展開します。
* **例**:

    ```bash
    # アプリケーションを /opt/apps ディレクトリに展開する
    unzip my-app-v1.2.zip -d /opt/apps/
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアの疑いがあるZIPファイルを、安全に内容物を確認する。
* **組み合わせ**: `unzip -l <suspicious.zip>`
* **解説**: **安全な調査の第一歩**。ZIPファイルを展開せずに `-l` で内容物を確認し、`.exe` や `.scr` のような実行ファイル、あるいは `../../...` のような不審なパスが含まれていないかを調べます。
* **例**:

    ```bash
    # フィッシングメールに添付されていた invoice.zip の中身を確認
    unzip -l invoice.zip
    ```

### 3. レッドチーム視点

* **タスク**: **Zip Slip / Path Traversal攻撃**を試みるための、悪意のあるZIPファイルを作成・利用する。
* **組み合わせ**: (悪意のあるZIPファイルの作成)
* **解説**: アーカイブ内のファイル名に `../../../../etc/cron.d/backdoor` のような相対パスを含んだZIPファイルを作成します。もしサーバー側のアプリケーションが、安全でない方法でZIPファイルを展開する脆弱性を持っている場合、このファイルをアップロード・展開させることで、Webルートの外にある重要な場所に悪意のあるファイルを書き込みシステムを乗っ取ります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **Zip Slip (Path Traversal)** や **伸張爆弾 (Decompression Bomb)**。
* **悪用シナリオ**: レッドチームのシナリオで述べたように、細工されたZIPアーカイブは、展開時に意図しないディレクトリにファイルを書き込んだり、ディスクスペースを使い果たしてサービス妨害を引き起こしたりする可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: もし `unzip` にSUIDビットが設定されている（重大な設定ミス）、または `sudo unzip` を実行できる権限がある場合、これを利用して権限昇格が可能です。攻撃者は、`root` が所有する `/etc/shadow` のようなファイルを含むZIPアーカイブを作成し、`sudo unzip` で展開することで、システム上の重要ファイルを上書きできます。
* **参照**: `https://gtfobins.github.io/gtfobins/unzip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **信頼できないソースからのZIPファイルを、特にroot権限で展開しない**。ファイルアップロード機能を持つアプリケーションでは、展開処理を安全に実装する（パスの検証など）。
* **Detection (検知)**: FIMツールで、Webルート外の予期せぬ場所へのファイル書き込みを監視する。

## 注意点・補足

* **安全な展開の習慣**: 信頼できないZIPファイルは、まず `unzip -l` で中身を確認し、問題がなければ念のため `mkdir tmp_dir && unzip file.zip -d tmp_dir` のように、専用の空のディレクトリに展開するのが安全な手順です。
