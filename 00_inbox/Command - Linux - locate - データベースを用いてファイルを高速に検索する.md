# Command - Linux - locate - データベースを用いてファイルを高速に検索する

## 概要
`locate` は、ファイル名の一部を指定するだけで、システム内の該当するパスを即座にリストアップします。高速さの秘密は、`/var/lib/mlocate/mlocate.db`（環境により異なる）というインデックスデータベースを参照することにあります。
(現代の Linux では、より高速な `plocate` パッケージが標準的に使われています)



## ⚡️ `locate` vs `find`
| 特徴 | `locate` | `find` |
| :--- | :--- | :--- |
| **速度** | **爆速** (データベース参照) | 低速 (リアルタイム走査) |
| **鮮度** | データベース更新まで反映されない | 常に最新の状態 |
| **柔軟性** | 名前検索がメイン | 権限、サイズ、時刻など多機能 |
| **負荷** | 実行時の負荷は極めて低い | ディスク I/O 負荷が高い |

## 主要なオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-i`, `--ignore-case` | 大文字・小文字を区別せずに検索する。 |
| ⭐️ `-r`, `--regexp` | 基本正規表現を使って検索する。 |
| ⭐️ `-n <N>` | 検索結果を最大 `N` 件に制限する。 |
| ⭐️ `-c`, `--count` | ヒットしたファイルの「数」だけを表示する。 |
| `-e`, `--existing` | DB にあっても、現時点で既に消去されたファイルは表示しない。 |
| `-b`, `--basename` | パス全体ではなく、ファイル名のみを対象にマッチングさせる。 |

## よく使われる連携コマンド
### シナリオ例: データベースの更新 (必須知識)
* **タスク**: 今作ったばかりのファイルが `locate` で見つからない。
* **解説**: `locate` は定期実行（通常は1日1回）される `updatedb` によって情報を更新します。手動で最新の状態にする必要があります。
* **コマンド例**:
    ```bash
    # データベースを手動で更新（root権限が必要）
    sudo updatedb
    ```

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 設定ファイルやログディレクトリの特定。
* **解説**: ソフトウェアごとにバラバラな設定ファイルの場所を、「パスの一部」から推測して一気に探し出します。
* **例**:
    ```bash
    # 'nginx' を含む設定ファイル (.conf) を探す
    locate -i nginx.conf
    ```

### 2. ブルーチーム視点
* **タスク**: 既知の不審なファイル名のスキャン。
* **解説**: IOC（侵害指標）に含まれる特定のマルウェア名や、不自然な拡張子（`.sh.txt` など）がシステム全体に散らばっていないか、全ディレクトリを高速にチェックします。

### 3. レッドチーム視点
* **タスク**: 機密情報の収集。
* **解説**: 侵入後、`locate password` や `locate backup`, `locate .ssh` といったキーワードで、価値のあるデータがどこにあるかを瞬時に把握します。`find` を打つよりもディスク負荷が低いため、監視システムに「大量のファイル読み込み」として検知されにくい利点があります。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報漏洩（Information Disclosure）**: ⭐️ **重要**。
`mlocate`（現代の標準）は、実行ユーザーが「読み取り権限を持たないディレクトリ」にあるファイルは表示しないように設計されています。しかし、古い実装や設定によっては、他のユーザーしか見られないファイル名まで見えてしまうリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **権限設定**: `/var/lib/mlocate/` のパーミッションが適切か、また `locate` バイナリに不審な `SGID` が設定されていないか確認します。
* **updatedb の除外設定**: `/etc/updatedb.conf` を編集し、機密情報を置くディレクトリやマウントポイントを検索対象から外す（`PRUNEPATHS`）設定を行います。

## エラーメッセージとトラブルシューティング

1.  **「locate: can not stat () `/var/lib/mlocate/mlocate.db': No such file or directory」**:
    * **原因**: データベースがまだ一度も作成されていない。
    * **解決策**: `sudo updatedb` を実行して作成する。

2.  **「検索結果が多すぎて画面が流れる」**:
    * **解決策**: `locate pattern | head -n 20` で絞り込むか、`-n` オプションを使いましょう。
