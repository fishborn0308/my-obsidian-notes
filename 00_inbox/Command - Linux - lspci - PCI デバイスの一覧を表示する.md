# Command - Linux - lspci - PCI デバイスの一覧を表示する

## 概要
`lspci` は、システム内の PCI バスと、そこに接続されているすべてのデバイス（ビデオカード、ネットワークアダプタ、USB コントローラなど）の詳細情報を表示します。デバイスのベンダー ID やデバイス ID を特定し、トラブルシューティングやドライバの適合確認に使用されます。
(出自: `pciutils` パッケージに含まれる)



## 🛠 調査ツールの比較
| コマンド | 調査対象 | 特徴 |
| :--- | :--- | :--- |
| **`lspci`** | **PCI デバイス** | 内部接続デバイス（GPU, NIC, NVMe等）に特化。 |
| `lsusb` | USB デバイス | 外付けデバイス、入力機器等。 |
| `lshw` | ハードウェア全般 | 全体を木構造で網羅する。 |
| `lsblk` | ブロックデバイス | ディスクとパーティションの構造。 |

## 主要なオプション
※詳細な情報を得るには、多くの場合 **root 権限 (sudo)** が推奨されます。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-v`, `-vv`, `-vvv` | 冗長表示。情報の詳細度を上げる（ドライバやメモリ範囲等）。 |
| ⭐️ `-k` | **推奨。** そのデバイスを使用中のカーネルドライバを表示する。 |
| ⭐️ `-nn` | ベンダー名に加えて、16進数のベンダー/デバイス ID を表示。 |
| ⭐️ `-s <slot>` | 特定のスロット（例: `00:02.0`）の情報のみを表示。 |
| `-t` | ツリー形式で表示し、デバイス間の親子関係（バス構造）を可視化。 |
| `-m` | マシン可読形式（スクリプト処理用）で出力。 |

## 💡 アドレス形式の読み解き (Slot)
`00:02.0` のような形式は、PCI バス上の「住所」を表します。
* **`00`**: バス番号 (Bus)
* **`02`**: デバイス番号 (Device/Slot)
* **`0`**: 機能番号 (Function)

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: GPU サーバーでの NVIDIA ドライバのセットアップ。
* **解説**: OS が物理的に GPU を認識しているか、また現在どのドライバ（`nouveau` か `nvidia` か）が割り当たっているかを確認します。
* **例**:
    ```bash
    # ビデオカードに絞り、使用中のドライバを確認
    lspci -k | grep -A 3 -i vga
    ```

### 2. ブルーチーム視点
* **タスク**: 物理的な不審デバイス（ハードウェア・バックドア）の検知。
* **解説**: サーバーの標準構成にはないはずの PCI デバイス（未知の NIC や暗号化アクセラレータ等）が追加されていないか、定期的にインベントリと比較します。

### 3. レッドチーム視点
* **タスク**: VM エスケープやサイドチャネル攻撃の準備。
* **解説**: 侵入した環境が仮想マシン（VM）である場合、PCI デバイスの一覧からハイパーバイザの種類（VMware, KVM, Xen 等）を特定し、その仮想化ソフトウェアに特有の脆弱性を探ります。
* **例**:
    ```bash
    # ベンダー ID を確認し、仮想化ベンダーを特定
    lspci -nn | grep -i virtual
    ```

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: 接続されているハードウェアの正確な型番とリビジョンが判明すると、そのチップセット特有のハードウェア脆弱性（例: Spectre/Meltdown 関連や特定の NIC のバグ）を標的にされるリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **構成管理の徹底**: サーバーの「あるべき姿」の PCI デバイスリストを保存しておき、変更があった場合にアラートを出す仕組みを構築します。
* **物理アクセスの制限**: `lspci` で検知できるような変更は、物理的な接触を伴うことが多いため、データセンターの入退室管理や筐体の施錠が根本的な防御となります。

## 注意点・補足
* **データベースの更新**: デバイス名が `Unknown device` と表示される場合は、PCI ID データベースが古いです。`sudo update-pciids` を実行して最新の名称リストに更新してください。
