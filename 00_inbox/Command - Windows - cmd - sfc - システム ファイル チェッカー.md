# Command - Windows - cmd - sfc - システム ファイル チェッカー

## 概要

`sfc` (System File Checker) は、Windows のすべての保護されたシステム ファイルのスキャンと整合性の確認を行い、破損または変更されたファイルを正しいバージョンに置き換える（修復する）ためのツールです。
OS の挙動が不安定、特定のシステム機能が動作しない、あるいはシステムバイナリの改ざんが疑われる場合に使用されます。
(出自: Windows標準搭載)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `DISM` | システムイメージ（修復用ソース）自体を修復する。`sfc` で直らない場合に併用。 | `DISM` (まず実行) |
| `chkdsk` | ファイルシステム（ディスク上の物理的・論理的構造）を修復する。 | `chkdsk` (HDD/SSD異常) |
| `sigcheck` | Sysinternals製。個別のファイルがデジタル署名されているか詳細に検証する。 | `sigcheck` (詳細検証) |

## よく連携されるコマンド

### シナリオ例: システム修復のフルコース (インフラ構築・運用)

* **目的**: OS の不具合を完全に解消するために、修復ソースを正常化してからシステムファイルを修復する。
* **連携コマンド**: `DISM`, `shutdown`
* **解説**: まず `DISM` で修復用の「元のイメージ」をインターネット経由で正常化し、その後に `sfc` を実行することで、修復の成功率を最大化します。
* **コマンド例**:
    ```cmd
    REM 1. 修復ソースの正常化
    dism /online /cleanup-image /restorehealth
    REM 2. システムファイルの修復
    sfc /scannow
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| **スキャン実行** | |
| ⭐ `/scannow` | **最重要。** すべての保護されたシステムファイルを即座にスキャンし、破損があれば修復する。 |
| `/verifyonly` | スキャンのみを行い、修復（書き換え）は行わない。 |
| **個別ファイル指定** | |
| `/scanfile=<file>` | 特定のファイルの整合性をスキャンし、問題があれば修復する。 |
| `/verifyfile=<file>` | 特定のファイルの整合性を確認のみ行う。 |
| **オフライン修復** | |
| `/offbootdir=<dir>` | オフライン修復時、ブート ディレクトリの場所を指定する。 |
| `/offwindir=<dir>` | オフライン修復時、Windows ディレクトリの場所を指定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: Windows Update 失敗後のシステム不整合の解消
* **解説**: アップデートが途中で止まった際などに、不完全な状態で書き込まれたシステムファイルを正常な状態に戻します。
* **例**:
    ```cmd
    sfc /scannow
    ```


### 2. ブルーチーム視点

* **タスク**: システムバイナリの改ざん（Rootkit等）の検知
* **解説**: マルウェアが `kernel32.dll` などを悪意あるものに置き換えていないかを確認します。`/verifyonly` を使用することで、証拠を上書きせずに不整合の有無だけを確認できます。
* **例**:
    ```cmd
    REM 修復せずに不整合のみをログ(CBS.log)に記録する
    sfc /verifyonly
    ```

### 3. レッドチーム視点

* **タスク**: バックドア設置後の整合性チェック回避の確認
* **解説**: システムファイルを直接書き換えるタイプのバックドアを設置した場合、`sfc` によってそれが「修復（削除）」されてしまうリスクがあります。攻撃者は、自分の変更が `sfc` で検知されるかを事前にテスト環境で確認します。
* **例**:
    ```cmd
    REM 自分の変更が Windows Resource Protection (WRP) に検知されるか確認
    sfc /verifyfile=C:\Windows\System32\target_dll.dll
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ

1.  **エラーメッセージ例 1**: `Windows リソース保護は、破損したファイルを発見しましたが、その一部を修正できませんでした。`
    * **考えられる原因**: 修復に必要なソースファイル自体が破損している、またはファイルがロックされている。
    * **解決策**: `DISM /Online /Cleanup-Image /RestoreHealth` を先に実行してから再度 `sfc` を行う。またはセーフモードで実行する。

2.  **エラーメッセージ例 2**: `Windows リソース保護は要求された操作を実行できませんでした。`
    * **考えられる原因**: `sfc` が依存するサービス（Windows Modules Installer）が停止している、または WinPE 環境でのパス指定ミス。
    * **解決策**: サービスの状態を確認するか、オフライン修復時は `/offbootdir` オプションを正しく指定する。

## 環境変数と設定ファイル

* **ログファイル**: `%WinDir%\Logs\CBS\CBS.log`
    - スキャンの詳細結果はこのファイルに記録されます。どのファイルが破損していたかを知るには、このログを `findstr` 等で解析する必要があります。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    - **Anti-Forensics**: 攻撃者が自身の活動によって破損させたシステムを `sfc` で修復し、システムを正常に見せかける（ただし `CBS.log` には痕跡が残る）。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Standard Command`
* **解説**: `sfc.exe` 自体は攻撃的な機能（ダウンロード等）を持ちませんが、システム環境の健全性を確認するための標準ツールとして、侵害後の環境維持（Persistenceの安定化確認）に利用されることがあります。

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**: 
    - **CBS.log の監視**: `sfc` が頻繁に実行されたり、特定の重要バイナリの修復が記録されている場合、侵害の試みがあった可能性を考慮します。
    - **イベントID 4688**: 一般ユーザー権限（管理者への昇格なし）で `sfc` が実行された場合のログ監視（通常は権限不足で失敗しますが、実行の意図が不審です）。

## 注意点・補足

* **実行時間**: ドライブの速度やファイル数によりますが、通常 5 分〜 15 分程度の時間がかかります。
* **管理者権限**: 実行には必ず「管理者として実行」したコマンドプロンプトが必要です。
