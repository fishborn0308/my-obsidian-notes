# Command - Linux - ifconfig - ネットワークインターフェースの設定・表示を行う

## 概要
`ifconfig` は、ネットワークインターフェース（LANカードや仮想デバイス）のステータスを表示したり、IP アドレスの割り当て、インターフェースの有効化・無効化などを行うためのツールです。
(出自: `net-tools` パッケージに含まれる。現在は非推奨(deprecated)とされているディストリビューションが多い)



## ⚠️ 現代の潮流： `ifconfig` vs `ip`
現代の主要な Linux ディストリビューションでは、`ifconfig` ではなく **`ip`** コマンド（`iproute2` パッケージ）の使用が推奨されています。
* `ifconfig`: 古い。表示が分かりやすい。設定できる範囲に限界がある。
* `ip`: 新しい。多機能。VLAN やポリシーベースルーティングなども扱える。

## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `ip addr` | `ifconfig` の現代的な代替。より詳細な情報を取得できる。 | **現代の標準** |
| `nmcli` | NetworkManager を操作する。設定の永続化に向く。 | デスクトップ・サーバー設定 |
| `ethtool` | リンク速度やデュプレックスなど、ハードウェア層の設定を行う。 | 物理層の調査 |
| `iwconfig` | 無線 LAN インターフェースの設定に特化したツール。 | Wi-Fi設定 |

## よく連携されるコマンド
### シナリオ例: 自分の IP アドレスだけを抜き出す (運用・自動化視点)
* **目的**: スクリプト内で自分の IPv4 アドレスを取得し、ログに記録したい。
* **解説**: `grep` と `awk` を使って、必要な行と列だけをフィルタリングします。
* **コマンド例**:
    ```bash
    # eth0 インターフェースの IP アドレスのみを表示
    ifconfig eth0 | grep 'inet ' | awk '{print $2}'
    ```

## 主要なオプションと操作
※設定変更には **root 権限** が必要です。

| オプション / 引数 | 説明 |
| :--- | :--- |
| ⭐️ `(引数なし)` | 有効な（UP状態の）全インターフェースを表示する。 |
| ⭐️ `-a` | 無効なものも含め、全てのインターフェースを表示する。 |
| ⭐️ `<if> up` | 指定したインターフェース（例: `eth0`）を有効化する。 |
| ⭐️ `<if> down` | 指定したインターフェースを無効化する。 |
| `<if> <IP>` | 指定したインターフェースに IP アドレスを即時設定する。 |
| `netmask <MASK>` | サブネットマスクを設定する（IP 設定と併用）。 |

## よく使われる業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: 一時的なネットワーク疎通テスト。
* **解説**: 新しいセグメントに接続した際、サクッと IP を振って疎通を確認します（再起動すると消える一時的な設定です）。
* **例**:
    ```bash
    # eth0 に 192.168.1.10 を一時的に割り当てる
    sudo ifconfig eth0 192.168.1.10 netmask 255.255.255.0
    ```

### 2. ブルーチーム視点
* **タスク**: 無差別モード（Promiscuous mode）の検知。
* **解説**: 盗聴（パケットスニッフィング）が行われているインターフェースは `PROMISC` フラグが立ちます。これを見抜くことで、不正なパケットキャプチャを防ぎます。
* **例**:
    ```bash
    # PROMISC フラグが立っていないか確認
    ifconfig | grep -i PROMISC
    ```

### 3. レッドチーム視点
* **タスク**: ネットワーク内の潜伏と MAC アドレス偽装（Spoofing）。
* **解説**: ターゲットネットワークの MAC アドレスフィルタリングを回避するために、自分の MAC アドレスを偽装します。
* **例**:
    ```bash
    # MAC アドレスを一時的に変更する
    sudo ifconfig eth0 down
    sudo ifconfig eth0 hw ether 00:11:22:33:44:55
    sudo ifconfig eth0 up
    ```

## エラーメッセージとトラブルシューティング

1.  **「ifconfig: command not found」**:
    * **原因**: `net-tools` がインストールされていない、またはパスが通っていない。
    * **解決策**: `sudo apt install net-tools` でインストールするか、代わりに `ip addr` を使う。

2.  **「SIOCSIFFLAGS: Operation not permitted」**:
    * **原因**: 権限不足。インターフェースの状態変更には特権が必要です。
    * **解決策**: `sudo` を付けて実行する。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報漏洩**: `ifconfig` の結果には、内部 IP、MAC アドレス、パケットの送受信統計が含まれます。攻撃者はこの情報を元にネットワーク構成を把握し、次なる攻撃（ARPスプーフィングなど）のターゲットを選定します。

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention**: MAC アドレス偽装を防ぐため、スイッチ側でポートセキュリティを設定する。
* **Monitoring**: 重要なサーバーで、インターフェースの状態（UP/DOWN）やフラグの変更をシステムログで監視する。

## 注意点・補足
* **エイリアス**: 一つの LAN カードに複数の IP を持たせる場合、`eth0:1` のように仮想的な名前を付けて管理します。
