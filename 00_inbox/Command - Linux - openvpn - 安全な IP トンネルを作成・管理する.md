# Command - Linux - openvpn - 安全な IP トンネルを作成・管理する

## 概要
`openvpn` は、SSL/TLS を利用したオープンソースの VPN ソフトウェアです。TCP または UDP ポート上で動作し、TUN（レイヤー3：IP）または TAP（レイヤー2：イーサネット）という仮想ネットワークデバイスを使用して、暗号化された通信路を確立します。



## 🛠 VPN 技術の比較
| 技術 | 速度 | セキュリティ | 設定の難易度 | 特徴 |
| :--- | :--- | :--- | :--- | :--- |
| **OpenVPN** | 普通 | **最高** | 高め | 柔軟性が非常に高く、ポート443(HTTPS)等に偽装可能。 |
| WireGuard | **極速** | 高 | 低 | 現代的でシンプル。Linuxカーネルに統合。 |
| IPsec/L2TP | 速い | 中〜高 | 高 | 標準的だが、ファイアウォールで遮断されやすい。 |

## 主要なオプションと引数
通常、設定ファイル (`.ovpn` または `.conf`) を指定して実行します。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `--config <file>` | 設定ファイルを読み込んで起動する。 |
| ⭐️ `--dev <tun/tap>` | 仮想デバイスの種類を指定（tun:ルーティング、tap:ブリッジ）。 |
| ⭐️ `--remote <host> [port]` | 接続先サーバーのアドレスとポートを指定。 |
| ⭐️ `--secret <key>` | 静的共有鍵（Static Key）モードで使用。 |
| ⭐️ `--verb <n>` | ログの出力レベルを指定（0:静か 〜 9:デバッグ）。 |
| `--daemon` | バックグラウンド（デーモン）として実行。 |

## 💡 接続の基本操作
```bash
# 設定ファイルを使って VPN に接続
sudo openvpn --config client.ovpn

# 実行中の接続を確認するには、仮想インターフェースを確認
ip addr show tun0
````

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: 拠点間接続（Site-to-Site VPN）の構築。
    
- **解説**: 本社と支店のネットワークを `openvpn` で繋ぎ、あたかも同じ LAN 内にいるかのように通信させます。また、クラウド環境（AWS/GCP）へのセキュアな管理用回線としても重宝します。
    

### 2. ブルーチーム視点

- **タスク**: リモートアクセス時の多要素認証 (MFA) 強制。
    
- **解説**: `auth-user-pass` と外部プラグインを組み合わせ、電子証明書だけでなく、ワンタイムパスワードがないと接続できない「鉄壁の入り口」を構築・監視します。不審な接続試行は `openvpn.log` から即座に検知します。
    

### 3. レッドチーム視点

- **タスク**: 検閲・監視の回避と C2 通信の隠蔽。
    
- **解説**: ターゲットの環境から外部への通信が厳しく制限されている場合、`openvpn` を TCP 443 ポート（HTTPS）で動作させることで、一般的なウェブ通信に紛れて外部の攻撃用サーバー（C2）とのコネクションを維持します。

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **脆弱な暗号スイート**: 古い設定で `BF-CBC` (Blowfish) などを使っている場合、現代の計算能力では解読されるリスクがあります。
    
- **特権昇格**: `openvpn` はネットワークデバイスを作成するため通常 root 権限で動きます。設定ファイルの読み込み処理等に脆弱性があると、システム全体の権限を奪われる可能性があります。
    

### 対応策・緩和策 (ブルーチーム視点)

- **`user` / `group` オプション**: 初期化後に権限を捨てる設定 (`user nobody`, `group nogroup`) を有効にします。
    
- **TLS-Auth / TLS-Crypt**: 証明書交換の前に共有鍵による認証を行うことで、未認証の相手からの DoS 攻撃やポートスキャンを完全に無視（ステルス化）できます。
    

## 注意点・補足

- **MTU の問題**: VPN 経由で一部のサイトが見られない、あるいは通信が極端に遅い場合、パケットサイズ（MTU）の不整合が原因であることが多いです。`mssfix` オプションで調整します。
