# Command - Windows - cmd - at - タスクスケジューラの管理（レガシー）

## 概要

`at` コマンドは、指定した時間や日付にコマンドやプログラムを実行するようにスケジュールするコマンドです。
現代のWindowsでは `schtasks` に置き換わっていますが、古いエディションのWindowsでは **SYSTEM権限** でタスクが実行される仕様があるため、権限昇格の文脈で頻繁に登場します。

(出自: `Windows 標準搭載 - スケジュール済みタスクサービスを利用`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `schtasks` | `at` の後継。より詳細なスケジュール設定（ログオン時、アイドル時など）が可能。現代のWindowsの標準。 | `schtasks` |
| `cron` (Linux) | Linuxにおける標準的なジョブスケジューラ。 | - |

## よく連携されるコマンド

### シナリオ例: SYSTEM権限でのシェル取得 (レッドチーム視点/OSCPレガシー環境)

* **目的**: 現在のユーザーよりも高い権限（SYSTEM）でインタラクティブなシェルを取得する。
* **連携コマンド**: `at`, `cmd.exe`
* **解説**: 現在時刻の1〜2分後に `cmd.exe` を起動するように設定します。古いWindows（XP/2003等）では `/interactive` スイッチにより、デスクトップにSYSTEM権限のコマンドプロンプトが表示されます。
* **コマンド例**:
    ```cmd
    REM 現在が10:00の場合、10:01にSYSTEM権限でCMDを起動させる
    at 10:01 /interactive cmd.exe
    ```

## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **引数なし** | 現在スケジュールされているタスクの一覧を表示する。 |
| `\\computername` | リモートコンピュータ上のタスクを指定する。 |
| `id` | 特定のタスク番号を指定（削除や詳細表示に使用）。 |
| `/delete` | スケジュールされたタスクを削除する。`id`指定がない場合は全削除。 |
| `/yes` | タスク削除時の確認プロンプトをスキップする。 |
| `time` | タスクを実行する時刻（24時間形式）。 |
| `/interactive` | タスク実行時にデスクトップと対話することを許可する（古いOSのみ有効）。 |
| `/every:date,...` | 毎週または毎月の指定した日に実行（例: `M,T,W,Th,F`）。 |
| `/next:date,...` | 次に来る指定日に実行。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 深夜帯に一度だけバックアップスクリプトを実行する。
* **組み合わせ**: `at 02:00 "C:\scripts\backup.bat"`
* **解説**: サーバーの負荷が低い時間帯にメンテナンスバッチを単発実行させます。
* **例**:
    ```cmd
    REM 深夜2時にクリーンアップバッチを実行
    at 02:00 C:\maintenance\cleanup.exe
    ```

### 2. ブルーチーム視点

* **タスク**: 攻撃者によって作成された不審なスケジュールタスクがないか調査する。
* **組み合わせ**: `at` (引数なし)
* **解説**: 意図しないバイナリ（特に `Temp` フォルダや `RECYCLER` 等から実行されているもの）が登録されていないかを確認します。
* **例**:
    ```cmd
    REM 定期的にスケジュールリストをチェックする
    at
    ```

### 3. レッドチーム視点

* **タスク**: 攻撃用ペイロードの実行（Persistence / 永続化）。
* **組み合わせ**: `at 12:00 /every:M,T,W,Th,F,S,Su C:\Users\Public\backdoor.exe`
* **解説**: 毎日決まった時間にバックドアを実行させることで、接続が切れても再確保できるようにします。
* **例**:
    ```cmd
    REM 毎日深夜0時にリバースシェルを試行するように設定
    at 00:00 /every:M,T,W,Th,F,S,Su C:\Windows\Temp\rev.exe
    ```

## エラーメッセージとトラブルシューティング

### よくあるエラーメッセージ
1.  **エラーメッセージ例 1**: `The AT command has been deprecated. Please use schtasks.exe instead.`
    * **考えられる原因**: Windows 8/Server 2012以降のOSで実行した。
    * **解決策**: メッセージの通り `schtasks` を使用するか、どうしても `at` を使いたい場合は古いバージョンのWindows環境であるか確認してください。

2.  **エラーメッセージ例 2**: `Access is denied.`
    * **考えられる原因**: タスクの作成・管理には管理者権限が必要です。
    * **解決策**: 管理者として実行したコマンドプロンプトで実行してください。

## 環境変数と設定ファイル

### 関連するレジストリ/設定ファイル

* **`C:\Windows\Tasks`**: スケジュールされたタスクファイル（`.job`）が保存されるディレクトリです。
* **`HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Schedule`**: タスクスケジューラサービスの設定を管理するレジストリです。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **権限昇格**: 冒頭に述べた通り、古いOSで `at` で作成されたタスクが `SYSTEM` ユーザーとして実行される振る舞いを利用した権限昇格が有名です。
* **LOLBAS**: `at.exe` 自体が信頼されたマイクロソフト製署名済みバイナリであるため、セキュリティソフトの目を掻い潜って悪意のあるコードをスケジュール実行するのに利用されます。

### LOLBAS における利用例

* **機能**: `Scheduled Task` (タスクの実行)
* **参照**: [LOLBAS - At.exe](https://lolbas-project.github.io/lolbas/Binaries/At/)

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最新のOSを使用し、`at` コマンドが機能しないようにする。`Task Scheduler` サービスへのアクセス権を制限する。
* **Detection (検知)**: `at.exe` または `schtasks.exe` のプロセス生成イベント（Event ID 4688）を監視し、引数に不審なパスが含まれていないか確認する。

## 注意点・補足

* **非推奨化**: Windows 10以降の `at` は、参照（表示）はできても新規作成が制限されているケースが多いです。OSCP試験で提供されるターゲットOSが Windows 7以前や Windows Server 2008以前であれば、この `at` のテクニックが刺さる可能性が高いです。
