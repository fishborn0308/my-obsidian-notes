# Command - Windows - cmd - eventvwr - イベント ビューアーの起動

## 概要

`eventvwr` は、Windows イベント ビューアー (Event Viewer) の GUI ツールを起動します。
システム、セキュリティ、アプリケーションなどの各ログを視覚的に閲覧・フィルタリングするために使用されます。トラブルシューティング、パフォーマンス監視、そしてセキュリティ監査において最も基本的なツールです。

(出自: `Windows 標準搭載 - eventvwr.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| ⭐ `wevtutil` | イベントログの表示、エクスポート、削除を行う強力な **CLI ツール**。 | 自動化・スクリプト |
| `Get-WinEvent` | PowerShell 版。オブジェクトとしてログを高度にフィルタリング可能。 | 解析の効率化 |
| `mmc eventvwr.msc` | Microsoft 管理コンソール経由で起動。実体は同じ。 | - |

## よく連携されるコマンド

### シナリオ例: 特定のログファイルを GUI で詳細解析 (ブルーチーム視点)

* **目的**: 保存されたログファイル (`.evtx`) を開き、特定の時間帯の不審な動きを目視で精査する。
* **連携コマンド**: `wevtutil` (エクスポート用), `eventvwr` (閲覧用)
* **解説**: CLI で抽出したログを GUI でじっくり確認するフローです。
* **コマンド例**:
    ```cmd
    REM 保存済みのログファイルを指定してイベントビューアーを起動
    eventvwr /l:C:\Backups\Security_Log_Snapshot.evtx
    ```



## スイッチ/オプション説明

| スイッチ / 引数 | 説明 |
| :--- | :--- |
| (なし) | 通常のイベント ビューアーを起動。 |
| ⭐ `/l:filename` | 指定したイベント ログ ファイル (`.evtx`) を開く。 |
| `/c:computername` | リモート コンピュータのイベント ビューアーに接続する。 |
| `/s` | 実行時にスプラッシュ スクリーンを表示しない。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーの再起動後に、サービスが正常に開始されなかった原因を「システム」ログから特定する。
* **手順**: `eventvwr` を開き、[Windows ログ] -> [システム] でソースが `Service Control Manager` のエラーを探す。

### 2. ブルーチーム視点 (インシデントレスポンス)

* **タスク**: 侵害が疑われる端末で、ログイン試行の失敗やログの消去が行われていないか確認する。
* **重要 Event ID**:
    * **4624**: ログオン成功 (不審な時間帯、不審なユーザー)
    * **4625**: ログオン失敗 (ブルートフォースの兆候)
    * ⭐ **1102**: セキュリティ ログが消去されました (攻撃者による証拠隠滅)
    * **4698**: スケジュール タスクが作成されました (永続化の兆候)

### 3. レッドチーム視点 (偵察と痕跡確認)

* **タスク 1: Trace Verification (足跡の確認)**
    * **目的**: 自分の実行したコマンドやツールが、どのログにどのような形式で記録されているかを確認し、今後の攻撃の「静音化」に役立てる。
* **タスク 2: Local Information Discovery (情報の列挙)**
    * **目的**: ログの中に残された「過去の管理者によるエラーログ」などを探し、そこに含まれるファイルパス、サーバー名、あるいは不注意に残されたパスワード等の情報を探る。

## エラーメッセージとトラブルシューティング

1.  **イベント ログ サービスが利用できません**
    * **原因**: `Windows Event Log` (eventlog) サービスが停止している。
    * **解決策**: `net start eventlog` でサービスを再起動する。

2.  **リモート接続に失敗する**
    * **原因**: ターゲット側で RPC サービスが動いていない、またはファイアウォールでブロックされている。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **ログの消去 (Defense Evasion - T1070.001)**: 
    * 攻撃者は管理者権限を得た後、`eventvwr`（または `wevtutil`）を使用してログを消去します。
* **UAC Bypass**:
    * 過去には `eventvwr.exe` が特定のレジストリを読み込む挙動を悪用し、権限昇格（UAC バイパス）に利用される手法が存在しました。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * **ログの転送 (Event Forwarding)**: ログがローカルで消去されてもいいように、即座に外部の SIEM や中央ログ管理サーバーに転送する設定を行う。
* **Detection (検知)**:
    * `Event ID 1102` (ログ消去) の発生をリアルタイムでアラート通知する。

## 注意点・補足

* **検索の遅さ**: ログが膨大な場合、GUI での検索やフィルタリングは非常に重くなります。その場合は `wevtutil` や PowerShell の `Get-WinEvent` を使用してフィルタリングした結果をエクスポートする方が効率的です。
