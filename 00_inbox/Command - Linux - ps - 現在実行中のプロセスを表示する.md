# Command - Linux - ps - 現在実行中のプロセスを表示する

## 概要
`ps` は、現在実行されているプロセスのスナップショットを表示します。`top` のようにリアルタイムで更新され続けるのではなく、**「叩いたその瞬間」**の記録を出すのが特徴です。そのため、スクリプトでの解析や、特定の瞬間を切り取る調査に最適です。



## 🛠 プロセスの見分け方
| コマンド | 特徴 | 用途 |
| :--- | :--- | :--- |
| **`ps`** | その瞬間のスナップショット。 | スクリプト、特定のPID特定。 |
| `top` / `htop` | リアルタイム更新。 | 負荷の継続監視、リソース泥棒の特定。 |
| `pgrep` | 名前から PID だけを抽出。 | 他のコマンドへの PID 渡し。 |
| `pstree` | プロセスの親子関係を木構造で表示。 | どのプロセスが誰から生まれたか確認。 |

## 💡 鉄板の呪文（オプション）
`ps` には「BSD スタイル（ハイフンなし）」と「UNIX/System V スタイル（ハイフンあり）」が混在しており、歴史的な理由で両方使われます。

| 呪文 | 説明 |
| :--- | :--- |
| ⭐️ `ps aux` | **最も一般的（BSD）。** 全ユーザーの全プロセスを詳細（CPU/メモリ使用率含む）に表示。 |
| ⭐️ `ps -ef` | **標準的（UNIX）。** 親プロセスID（PPID）を含む全ての情報を表示。 |
| ⭐️ `ps --forest` | 親子関係をアスキーアートのツリー形式で表示。 |
| ⭐️ `ps -u <user>` | 特定のユーザーが動かしているプロセスのみを表示。 |



## 📊 出力項目の読み方
* **PID**: プロセス ID（個別の識別番号）。
* **%CPU / %MEM**: CPU とメモリの占有率。
* **VSZ / RSS**: 仮想メモリサイズ / 物理メモリ使用量。
* **STAT**: プロセスの状態（`R`: 実行中, `S`: スリープ, `Z`: ゾンビ, `T`: 停止中）。
* **COMMAND**: 実行されているコマンド名。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: ゾンビプロセスの特定とクリーンアップ。
* **解説**: 終了したはずなのに親に見捨てられ、システムに残っている「ゾンビ (`Z`)」を探します。
* **例**:
    ```bash
    # ゾンビ状態のプロセスを検索
    ps aux | grep 'Z'
    ```

### 2. ブルーチーム視点
* **タスク**: 不審なプロセスの検知。
* **解説**: プロセス名が `/usr/sbin/apache2` なのに、実行ユーザーが `www-data` ではなく一般ユーザーだったり、実行パスが `/tmp` だったりする「なりすましプロセス」を暴き出します。
* **例**:
    ```bash
    # /tmp から実行されている不自然なプロセスをチェック
    ps aux | grep '/tmp/'
    ```

### 3. レッドチーム視点
* **タスク**: ターゲット環境の偵察と権限昇格のヒント。
* **解説**: root 権限で動いている脆弱そうな独自スクリプトや、特定のポートで待ち受けているバックエンドプロセスを特定します。また、引数にパスワードが直接書き込まれている（例: `mysql -pSECRET`）不注意なプロセスがないかスキャンします。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `ps` は（カーネルの設定によりますが）一般ユーザーでも他人のプロセス情報を見ることができます。これにより、他のユーザーが実行しているコマンドの引数（機密情報が含まれる場合がある）が漏洩するリスクがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **hidepid の設定**: `/proc` をマウントする際のオプションに `hidepid=2` を設定することで、一般ユーザーが自分以外のプロセス情報を見られないように制限できます。
