# Command - Windows - cmd - control - コントロール パネルの起動と設定

## 概要

`control` コマンドは、Windows のコントロール パネル（Control Panel）を起動したり、特定のコントロール パネル項目（.cpl ファイル）を直接開いたりするために使用されます。
マウス操作では何度もクリックが必要な深い設定画面を、コマンド一つで呼び出すことができます。

(出自: `Windows 標準搭載 - control.exe`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推換 (あれば) |
| :--- | :--- | :--- |
| `start ms-settings:<name>` | Windows 10/11 の「設定」アプリの項目を開く。 | モダンな設定 |
| `sysdm.cpl` | システムのプロパティを直接実行して開く。 | 速度重視 |
| `netplwiz` | ユーザーアカウント制御（詳細）を直接開く。 | ユーザー管理 |

## よく連携されるコマンド

### シナリオ例: ネットワークアダプタの不具合調査 (運用・保守)

* **目的**: ネットワークが繋がらない際、即座にアダプタの一覧画面を開いて無効/有効の切り替えを行う。
* **連携コマンド**: `ipconfig`, `control`
* **解説**: `ipconfig` で状態を確認した後、GUI で詳細なプロパティを触るために呼び出します。
* **コマンド例**:
    ```cmd
    REM ネットワーク接続の一覧画面を直接開く
    control netconnections
    ```



## よく使われる引数と項目名

`control [項目名]` の形式で実行します。

| 引数 / 項目名 | 開かれる設定画面 |
| :--- | :--- |
| (なし) | コントロール パネルのホーム画面 |
| `admintools` | 管理ツール (Windows Tools) |
| `desktop` | 個人用設定 (画面の解像度など) |
| `folders` | エクスプローラーのオプション (隠しファイル表示設定など) |
| `keyboard` | キーボードのプロパティ |
| `mouse` | マウスのプロパティ |
| ⭐ `netconnections` | ネットワーク接続 (アダプタの一覧) |
| `printers` | デバイスとプリンター |
| ⭐ `userpasswords` | ユーザー アカウント |
| ⭐ `userpasswords2` | ユーザー アカウント（詳細版。自動ログイン設定などで使用） |

### Canonical Names (正規名) を使用した指定
`/name` スイッチを使うことで、より詳細な項目を指定できます。
* **例**: `control /name Microsoft.BackupAndRestore` (バックアップと復元)

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新規サーバーのセットアップ時に、ファイアウォールやリモートデスクトップの設定を素早く呼び出す。
* **例**:
    ```cmd
    REM ファイアウォールの詳細設定を開く
    control firewall.cpl
    ```

### 2. ブルーチーム視点 (インシデントレスポンス)

* **タスク**: 侵害の疑いがある端末で、インストールされている不審なプログラムを GUI で目視確認・アンインストールする。
* **例**:
    ```cmd
    REM 「プログラムと機能」を一発で開く
    control appwiz.cpl
    ```

### 3. レッドチーム視点 (制限の回避)

* **タスク 1: Bypassing Restricted Shells (制限付きシェルの回避)**
    * **目的**: 特定のアプリケーションしか動かせない環境（KIOSKモードなど）から、システム設定へアクセスする。
    * **解説**: ファイル保存ダイアログの「パス入力欄」などに `control` と打ち込むことで、本来禁止されている設定画面を無理やり引き出す手法があります。
* **タスク 2: Persistence Enumeration (永続化の確認)**
    * **目的**: GUI の「タスク スケジューラ」などを呼び出し、不審な登録がないか確認する。
    * **コマンド例**:
        ```cmd
        control schedtasks
        ```

## エラーメッセージとトラブルシューティング

1.  **項目が見つからない**
    * **原因**: Windows のバージョンによって項目名が変更されている、または廃止されている。
    * **解決策**: `ms-settings:` スキーム（例: `start ms-settings:windowsupdate`）の使用を検討してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **権限昇格の足がかり**: 
    * 管理者権限を必要とする項目（`userpasswords2` など）を呼び出す際、UAC（ユーザーアカウント制御）のダイアログが表示されますが、ここからさらにエクスプローラーを管理者権限で立ち上げるなどのテクニックに繋げられることがあります。
* **Social Engineering**: 
    * 攻撃者が「設定を確認してください」と偽り、特定の `control` コマンドを実行させてセキュリティ機能を無効化させる（例: ファイアウォールをオフにする）誘導に悪用されるリスクがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Execute` (実行)
* **特性**: 攻撃者が `.cpl` ファイル（実体は DLL）を作成し、それを `control.exe` 経由で実行させることで、ウイルス対策ソフトの検知を回避（バイナリの偽装）しようとする場合があります。
* **参照**: [LOLBAS - control.exe](https://lolbas-project.github.io/lolbas/Binaries/Control/)

## 注意点・補足

* **移行の波**: 現在の Windows では、コントロール パネルの機能が徐々に「設定」アプリへ移行しています。そのため、`control` コマンドを実行しても「設定」アプリが開く項目が増えています。
* **ショートカット**: 頻繁に使う項目は、`control <name>` をバッチファイルにしておくと、マウス操作の何倍も速く作業が完了します。
