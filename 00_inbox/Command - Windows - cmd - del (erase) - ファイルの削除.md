# Command - Windows - cmd - del (erase) - ファイルの削除

## 概要

`del` および `erase` コマンドは、1 つ以上のファイルを削除します。
ディレクトリ構造を維持したまま中身のファイルだけを消去したり、属性（読み取り専用など）を無視して強制的に削除したりすることが可能です。なお、このコマンドで削除されたファイルは「ごみ箱」には入らず、ファイルシステム上から直接リンクが外されます。

(出自: `Windows 標準搭載 - cmd.exe 内蔵コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `rd` / `rmdir` | **ディレクトリ（フォルダ）**そのものを削除する。 | フォルダ削除 |
| `Remove-Item` | PowerShell 版。再帰的な削除や、フィルタリングがより強力。 | PS 環境 |
| `sdelete` | Sysinternals ツール。データを物理的に上書きして復元不能にする。 | 安全な抹消 |
| `wipe` | Linux 等のツール。`del` と違い、ファイル残渣を残さない。 | - |

## よく連携されるコマンド

### シナリオ例: 一時ファイルの全削除 (運用・クリーンアップ)

* **目的**: 特定の拡張子を持つ一時ファイル（.tmp）をサブディレクトリまで含めて一括清掃する。
* **連携コマンド**: `del`, `dir` (確認用)
* **解説**: ワイルドカード `*` とサブディレクトリ検索 `/s` を組み合わせます。
* **コマンド例**:
    ```cmd
    REM C:\Temp 内のすべての .tmp ファイルを、確認なしで強制削除
    del /s /q /f C:\Temp\*.tmp
    ```



## スイッチ/オプション説明

| スイッチ/オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| `names` | 削除するファイル名またはリストを指定（ワイルドカード可）。 |
| ⭐ `/p` | 削除する前に確認メッセージを表示する。 |
| ⭐ `/f` | 読み取り専用ファイルでも強制的に削除する。 |
| ⭐ `/s` | 指定したファイルをすべてのサブディレクトリから削除する。 |
| ⭐ `/q` | 静止モード。ワイルドカードでの削除時に確認を求めない。 |
| **属性指定** | |
| `/a[:attributes]` | 属性に基づいて削除するファイルを選択する（R:読み取り専用, S:システム, H:隠し, A:アーカイブ）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: スクリプトの実行終了後、作業用に生成した中間ファイルをすべて削除して環境をクリーンに保つ。
* **例**:
    ```cmd
    REM 読み取り専用属性が付いていても強制的に静かに削除
    del /f /q "%TEMP%\work_data_*"
    ```

### 2. ブルーチーム視点 (デジタルフォレンジック)

* **タスク**: 攻撃者がファイルを削除した形跡（MFT 内の削除フラグや、未割り当て領域に残ったデータ残渣）を調査する。
* **解説**: `del` はデータを「上書き」するわけではないため、物理ディスク上にデータが残っている可能性が高いです。Autopsy や FTK Imager などのツールでこれらを復元します。

### 3. レッドチーム視点 (証拠隠滅 - Anti-Forensics)

* **タスク 1: Indicator Removal (痕跡の消去 - T1070.004)**
    * **目的**: 設置したペイロード、スクリプト、収集したデータの残骸を削除して、ブルーチームによる追跡を困難にする。
    * **コマンド例**:
        ```cmd
        REM 実行した実行ファイルと、ステージング用フォルダ内のファイルをすべて削除
        del /f /q C:\Windows\Temp\malware.exe
        del /s /q C:\Users\Public\Documents\staged_data\*
        ```

* **タスク 2: Batch Script Self-Deletion (バッチファイルの自己削除)**
    * **目的**: 処理が終わったバッチファイル自体をディスクから消去する。
    * **コマンド例**:
        ```cmd
        REM スクリプトの最後に記述することで自分自身を消す
        (goto) 2>nul & del "%~f0"
        ```

## エラーメッセージとトラブルシューティング

1.  **Access is denied. (アクセスが拒否されました)**
    * **原因**: 権限が不足している、または他のプロセスがファイルを開いている（ロック状態）。
    * **解決策**: 管理者権限で実行する、または `handle` などのツールでロックしているプロセスを特定し終了させる。

2.  **Could Not Find [filename]**
    * **原因**: 指定したパスにファイルが存在しない。
    * **解決策**: `dir` コマンドでファイル名を確認。特にスペースを含むファイル名が引用符 `"` で囲まれているかチェック。

## 環境変数と設定ファイル

* **影響を与える変数**: `del` 自体に直接影響する変数は少ないが、`%TEMP%` や `%APPDATA%` などのパスと組み合わせて、ユーザー環境の清掃によく使われます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **不適切なワイルドカードの使用**:
    * 管理者権限で `del /s /q C:\*` のようなミス（または悪意ある操作）を行うと、OS の重要なファイルが削除され、システムが起動不能（DoS）になります。
* **アンチ・フォレンジックの限界**:
    * `del` は「目次」から消すだけで「中身」は消さないため、フォレンジック調査で復元されるリスクがあります。より確実な隠滅には `cipher /w` や Sysinternals の `sdelete` が使用されます。

* **機能**: `Artifact Removal`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**:
    * 重要なシステムディレクトリへの書き込み・削除権限を一般ユーザーから剥奪する。
* **Detection (検知)**:
    * **イベントID 4660**: オブジェクトが削除されました。
    * **イベントID 4663**: オブジェクトへのアクセス試行（削除フラグを監視）。
    * 短時間に大量のファイルが削除されている挙動（特にシステム領域や重要ドキュメント領域）を監視し、ワイパー（Wiper）マルウェアの活動を検知します。

## 注意点・補足

* **読み取り専用ファイルの削除**: `/f` スイッチを忘れると、読み取り専用ファイルに遭遇した時点で処理が止まったり、スキップされたりします。
* **スペースを含むパス**: パスにスペースが含まれる場合は、必ず `"C:\Program Files\App\data.log"` のように全体を引用符で囲んでください。
