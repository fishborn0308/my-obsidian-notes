# `nslookup` - DNSサーバーに問い合わせる (旧)

## 概要

`nslookup` (name server lookup) は、ドメイン名からIPアドレスを調べる（正引き）、またはIPアドレスからドメイン名を調べる（逆引き）など、DNS (Domain Name System) サーバーに問い合わせを行うための古典的なコマンドラインツールです。対話モードと非対話モードの2つのモードを持ち、様々な種類のDNSレコードを照会できます。

**⚠️ 重要**: `nslookup` は現在**レガシー（旧式）**なツールと見なされています。出力がより詳細で、スクリプトでの利用にも適した **`dig` コマンドの使用が強く推奨されます**。

(出自: `bind-utils` (RHEL系) or `dnsutils` (Debian系) パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`dig`** | `nslookup` のモダンな後継。`dig` は出力が非常に詳細で構造化されており、TTLなどの情報も表示するため、プロフェッショナルな現場では `dig` の方が圧倒的に優れています。 | `dig` |
| **`host`** | `nslookup` よりもさらにシンプルなコマンド。手早くIPアドレスを知りたい場合に便利です。 | - |

## よく連携されるコマンド

`nslookup` は対話的な利用が主ですが、非対話モードの出力を `grep` などで処理することもあります。

### シナリオ例: 特定のレコードタイプを問い合わせる (インフラ構築・運用視点)

* **目的**: ドメインのメールサーバー (MXレコード) を確認する。
* **連携コマンド**: `nslookup` (単体)
* **解説**: `-query=MX` (または `-type=MX`) オプションを使って、特定のDNSレコードを直接問い合わせます。
* **コマンド例**:

```bash
# google.com のMXレコード（メールサーバー）を問い合わせる
nslookup -query=MX google.com
```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **基本操作** | |
| ⭐ `<domain>` | 問い合わせたいドメイン名またはIPアドレス。 |
| `[server]` | 問い合わせ先のDNSサーバーを明示的に指定します。 |
| **クエリ指定** | |
| ⭐ `-query=<type>` | 問い合わせるレコードタイプを指定します。(`A`, `MX`, `NS`, `SOA`, `ANY`など) |
| **対話モードコマンド** | (`nslookup`を引数なしで起動) |
| `server <ip>` | 問い合わせ先のDNSサーバーを変更します。 |
| `set type=<record>`| 問い合わせるレコードタイプを変更します。 |
| `exit` | 対話モードを終了します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新しく設定したDNSのAレコードが正しく反映されているか確認する。
* **組み合わせ**: `nslookup <domain_name>`
* **解説**: ドメイン名の正引きを行い、意図したIPアドレスが返ってくるかを確認する最も基本的な使い方です。
* **例**:

```bash
# [www.example.com](https://www.example.com) のIPアドレスを確認
nslookup [www.example.com](https://www.example.com)
```

### 2. ブルーチーム視点

* **タスク**: ファイアウォールログで検知した不審なIPアドレスの素性を調査する。
* **組み合わせ**: `nslookup <suspicious_ip>`
* **解説**: 不審なIPアドレスの逆引き（PTRレコード問い合わせ）を行い、どのようなドメイン名が割り当てられているかを確認します。これにより、そのIPアドレスの所有組織などを推測する手がかりとなります。
* **例**:

```bash
# 8.8.8.8 の逆引きを実行
nslookup 8.8.8.8
# -> 8.8.8.8.in-addr.arpa name = dns.google.
```

### 3. レッドチーム視点

* **タスク**: 偵察の一環として、標的組織のメールサーバーやネームサーバーを特定する。
* **組み合わせ**: `nslookup -query=MX <target_domain>`
* **解説**: 標的ドメインのMXレコードを調べることで、`mail.example.com` のようなメールサーバーのホスト名を特定します。メールサーバーは、フィッシング攻撃や認証情報を狙ったブルートフォース攻撃の格好の標的となります。NSレコードを調べれば、ゾーン転送を試みるべきDNSサーバーも特定できます。
* **例**:

```bash
# 標的ドメインのネームサーバー (NSレコード) を確認
nslookup -query=NS example.com
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **メッセージ**: `** server can't find <domain>: NXDOMAIN`
    * **考えられる原因**: "Non-Existent Domain"。問い合わせたドメイン名は存在しません。
    * **解決策**: ドメイン名のタイプミスがないか確認してください。

2. **メッセージ**: `** server can't find <domain>: SERVFAIL`
    * **考えられる原因**: "Server Failure"。DNSサーバーが応答の生成に失敗しました。
    * **解決策**: 別のDNSサーバー (`nslookup <domain> 8.8.8.8`など) を指定して再度試み、問題の切り分けを行います。

## 環境変数と設定ファイル

* `nslookup` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* **`/etc/resolv.conf`**: `nslookup` が問い合わせ先サーバーを明示的に指定されなかった場合に、デフォルトで使用するDNSリゾルバが記述されています。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `nslookup` 自体ではなく、DNSプロトコルの仕組みやサーバーの設定不備が悪用されます。
* **悪用シナリオ**: `nslookup` は、攻撃者が**情報収集（偵察）**フェーズで、標的のインフラ（メールサーバー、DNSサーバーなど）をマッピングするために利用します。また、`ls -d` サブコマンドで**ゾーン転送**を試みることも可能です。

### GTFOBins / LOLBAS における利用例

`nslookup` はネットワーク越しの情報収集ツールであり、ローカルシステム上でのファイル操作や権限昇格には直接利用できないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 権威DNSサーバーの設定で、ゾーン転送 (`AXFR`) を許可するIPアドレスを、セカンダリDNSサーバーなどの信頼できるホストのみに厳格に制限する。
* **Detection (検知)**: 内部から外部への異常なDNSクエリ（特に `ANY` や `AXFR` タイプのリクエスト）を監視することで、偵察活動を検知できる可能性があります。

## 注意点・補足

* **`dig` への移行**: `nslookup` の出力は人間には分かりやすいですが、プログラムでの処理には向いていません。DNS関連の調査やスクリプティングには、より詳細で信頼性の高い出力を提供する `dig` コマンドの使用に慣れることを強くお勧めします。
