# Command - Linux - ls [Red Team Edition] - 隠れた脆弱性を暴くスキャン術

## 概要
レッドチームにおける `ls` は、単なるファイル一覧表示コマンドではありません。ターゲット環境の「情報の密度」と「設定の不備」を可視化するための**偵察（Reconnaissance）および権限昇格（PrivEsc）の最重要ツール**です。



---

## 1. 権限昇格の「糸口」を探す (Privilege Escalation)

### 🚨 SUID / SGID バイナリの特定
実行時に所有者の権限で動作する SUID ファイルは、権限昇格の最大の標的です。`ls -l` の結果から `s` ビットを探します。
* **テクニック**: `/usr/bin` や `/usr/sbin` をスキャンし、`GTFOBins` に掲載されているような脆弱なバイナリがないか確認します。
* **コマンド**:
    ```bash
    # SUID ビットが立っているファイルを一括確認 (パーミッションの第4文字が 's')
    ls -l /usr/bin | grep '^-..s'
    ```

### 🔓 書き込み可能なディレクトリの探索
「誰でも書き込める（World-writable）」ディレクトリは、悪意ある共有ライブラリの設置や、設定ファイルの差し替えに利用できます。
* **テクニック**: `/` や `/etc` で `drwxrwxrwx` (777) になっている場所を探し、そこを攻撃の起点（ランディングゾーン）にします。
* **コマンド**:
    ```bash
    # ディレクトリ自体の権限を詳細表示し、777 の場所を特定
    ls -ld /tmp /var/tmp /dev/shm
    ```

---

## 2. 「王国の鍵」を盗み出す (Sensitive Data Discovery)

### 🗝 隠しファイルと設定ファイルの奪取
ユーザーのホームディレクトリには、認証情報が詰まった「宝箱」が隠されています。
* **ターゲット**:
    * `.ssh/`: 非公開鍵（id_rsa）
    * `.bash_history`: 過去に実行したコマンド（パスワードが平文で残っていることも）
    * `.aws/`, `.azure/`: クラウドのアクセスキー
* **コマンド**:
    ```bash
    # ホームディレクトリの全ての隠しファイルを詳細表示
    ls -la ~/
    ```

### 📂 バックアップと一時ファイルの特定
管理者が一時的に作成したバックアップ（`.bak`, `.old`, `~`）には、現在の設定よりも緩い権限で機密データが残っていることが多々あります。
* **コマンド**:
    ```bash
    # 拡張子やパターンで不審なバックアップファイルを炙り出す
    ls -lR /etc | grep -E '\.(bak|old|swp|save)$'
    ```

---

## 3. 「管理者の足跡」を追う (Temporal Analysis)

### 🕒 最近の変更を追跡する
システム管理者が最近どのファイルを触ったかを知ることで、アクティブなサービスや進行中のメンテナンス作業を特定できます。
* **テクニック**: `-t` (時刻順) と `-u` (アクセス時刻) を使い分け、管理者が「最近見たファイル」と「最近変えたファイル」を把握します。
* **コマンド**:
    ```bash
    # 更新日時が新しい順に表示（最近の変更を特定）
    ls -lt /etc
    # アクセス日時（最後に読まれた時）が新しい順に表示
    ls -ltu /etc
    ```

---

## 4. 隠蔽工作とアンチフォレンジック (Stealth)

### 👻 タイムスタンプの違和感を確認
自分の活動を隠すために `touch -t` などでタイムスタンプを偽装した後、`ls --full-time` を使って、他のファイルと秒単位・ナノ秒単位で整合性が取れているか（浮いていないか）を最終確認します。
* **コマンド**:
    ```bash
    # 秒・ナノ秒まで表示して、偽装したタイムスタンプの微調整を行う
    ls -l --full-time
    ```

## 💡 レッドチームの必勝コンボ
| 目的 | コマンド例 | 理由 |
| :--- | :--- | :--- |
| **全貌把握** | `ls -laR /home` | 全ユーザーの隠しファイルとディレクトリ構造を再帰的にぶっこ抜く。 |
| **大物狙い** | `ls -lhS` | 巨大なデータベースのダンプやログを探し、持ち出し（Exfiltration）対象を決める。 |
| **隙探し** | `ls -l /etc/shadow` | 万が一、読み取り権限が漏れていれば、即座にハッシュクラックへ。 |

> [!WARNING]
> ターゲット環境に強力な EDR (Endpoint Detection and Response) が導入されている場合、`/` からの `ls -R`（再帰的な全スキャン）は異常なファイル探索行動として検知アラートが飛ぶリスクがあります。探索は段階的に、かつ慎重に行うのが定石です。
