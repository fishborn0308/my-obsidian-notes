# Command - Linux - pacman - 強力かつシンプルなパッケージマネージャー

## 概要
`pacman` は、Arch Linux とその派生ディストリビューションで使用される標準のパッケージ管理ツールです。バイナリパッケージの管理に特化しており、シンプルな圧縮ファイル（.tar.zst）を扱うため動作が非常に機敏です。システムの完全なアップグレード、新しいソフトの導入、不要なソフトの削除を、短いオプションの組み合わせで実行します。



## 🛠 パッケージマネージャー比較
| コマンド | OS (代表例) | 特徴 |
| :--- | :--- | :--- |
| **`pacman`** | Arch Linux, Manjaro, SteamOS | **爆速、ローリングリリース、シンプルなコマンド。** |
| `apt` | Debian, Ubuntu | 安定性重視、広大なユーザーベース。 |
| `dnf` | Fedora, RHEL | 堅実な依存関係解決、高機能。 |
| `yay` / `paru` | Arch Linux (AUR) | `pacman` の機能を継承し、ユーザー投稿パッケージ (AUR) も扱える。 |

## 主要なオプション（呪文）
`pacman` のオプションは、大文字の「操作」と小文字の「詳細設定」を組み合わせて使います。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-S` (Sync) | パッケージのインストール。 |
| ⭐️ `-Syu` | **最強の呪文。** リポジトリを同期し、システム全体を最新に更新する。 |
| ⭐️ `-R` (Remove) | パッケージの削除。 |
| ⭐️ `-Rs` | 依存関係も含めてきれいに削除する。 |
| ⭐️ `-Q` (Query) | インストール済みパッケージの検索・表示。 |
| ⭐️ `-U` (Upgrade) | ローカルのパッケージファイル（.pkg.tar.zst）を直接インストールする。 |
| `-Ss` | リポジトリ内のパッケージを検索する。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: サーバー構築時の必要最小限なパッケージ導入。
* **解説**: Arch は初期状態が「空」に近いため、`pacman -S` で必要なものだけを積み上げ、肥大化（Bloat）のない純粋なシステムを構築します。
* **例**:
    ```bash
    # 基本的な開発ツールを一括インストール
    sudo pacman -S base-devel git vim
    ```

### 2. ブルーチーム視点
* **タスク**: インストール済みファイルの改ざん検知。
* **解説**: `pacman` は各ファイルのチェックサムを保持しています。これを利用して、システムファイルが本来の状態から変更されていないか検証できます。
* **例**:
    ```bash
    # インストール済みパッケージのファイルを検証し、改ざんがないか確認
    pacman -Qk
    ```

### 3. レッドチーム視点
* **タスク**: ペネトレーションテスト用ツールの迅速な展開。
* **解説**: ターゲットが Arch Linux（または Steam Deck 等の派生）の場合、`pacman` を使って `nmap` や `wireshark` などの調査ツールを即座に導入できます。また、バイナリパッケージがシンプルなため、悪意あるファイルをパッケージ化して `pacman -U` で忍び込ませる手法の検討にも使われます。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **署名の検証**: `pacman` は PGP 署名による検証を行いますが、キーサーバーとの通信トラブルや期限切れにより、正当なパッケージが弾かれたり、最悪の場合「検証なし」でインストールしてしまうリスクがあります。
* **AUR (Arch User Repository) の罠**: `yay` などで AUR を使う際、中身を確認せずにインストールすると、悪意あるスクリプトが含まれている可能性があります。

### 対応策・緩和策 (ブルーチーム視点)
* **Mirrorlist の保護**: `/etc/pacman.d/mirrorlist` で、信頼できる高速なミラーサーバーのみをリストに載せます。
* **定期的な -Syu**: ローリングリリースであるため、セキュリティパッチも即座に配信されます。毎日、あるいは毎週の更新を習慣化します。

## 注意点・補足
* **部分的な更新の禁止**: `pacman -Sy` (同期のみ) をした後に `pacman -S <package>` を実行するのは避けてください。依存ライブラリのバージョン不整合でシステムが壊れる可能性があります。常に **`-Syu`** を心がけましょう。
