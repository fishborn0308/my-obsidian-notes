# `ip neigh` - 近隣キャッシュ (ARPテーブル) を管理する

## 概要

`ip neigh` (または `ip neighbour`) は、`ip` コマンドスイートの一部であり、**近隣キャッシュ (neighbor cache)** を表示・操作するためのコマンドです。このキャッシュは、同じローカルエリアネットワーク (LAN) 上にある他のデバイスの **L3アドレス（IPアドレス）** と **L2アドレス（MACアドレス）** の対応関係を記録しています。

* **IPv4**: このキャッシュは **ARP (Address Resolution Protocol) テーブル**として知られます。
* **IPv6**: このキャッシュは **NDP (Neighbor Discovery Protocol)** テーブルとして知られます。

`ip neigh` は、ネットワークの接続問題をトラブルシューティングしたり、ARPスプーフィングのようなセキュリティ攻撃の兆候を発見したりするために使用されます。

(出自: `iproute2` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| **`arp`** | **旧世代のコマンド**。ARPテーブルを表示・操作します。`ip neigh` はIPv4のARPとIPv6のNDPの両方を扱える、よりモダンで高機能な後継コマンドです。 | `ip neigh` |
| **`ip addr` / `ip link`**| `ip addr` は自身のIPアドレス、`ip link` は自身のMACアドレスを管理します。`ip neigh` は**他のホスト**のIP/MACアドレスの対応関係を管理します。 | - |

## よく連携されるコマンド

`ip neigh` は、ネットワークのトラブルシューティングや偵察において、`ping` や `watch` と連携して使われます。

### シナリオ例: ARPスプーフィング攻撃の兆候を監視する (ブルーチーム視点)

* **目的**: ゲートウェイのARPエントリが不正に書き換えられていないかをリアルタイムで監視する。
* **連携コマンド**: `watch`
* **解説**: `watch` コマンドは、指定したコマンドを一定間隔で繰り返し実行し、その結果を画面に表示し続けます。これにより、ARPテーブルの動的な変化を監視し、中間者攻撃の兆候を捉えることができます。
* **コマンド例**:

```bash
# 5秒ごとにゲートウェイ (192.168.1.1) のARPエントリを監視する
watch -n 5 ip neigh show 192.168.1.1
```

## オプション説明 (`ip neigh` のアクション)

| オプション | 説明 |
| :--- | :--- |
| **表示** | |
| ⭐ `show` (or `list`, `ls`) | (デフォルト) 近隣キャッシュ（ARPテーブル）を表示します。 |
| **キャッシュ操作** | |
| ⭐ `flush dev <dev>` | 指定したデバイスのARPキャッシュを全てクリア（フラッシュ）します。 |
| `add <ip> lladdr <mac> dev <dev> nud perm`| 静的な (permanent) ARPエントリを追加します。 |
| `del <ip> dev <dev>` | ARPエントリを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: L2接続のトラブルシューティングで、ARPキャッシュを強制的にクリアして再学習させる。
* **組み合わせ**: `sudo ip neigh flush dev <device>`
* **解説**: 「特定のサーバーに `ping` が通らない」といった問題の際に、ARPテーブルに古い情報が残っていることが原因の場合があります。このコマンドでキャッシュをクリアし、ARP要求を再送させることで問題を解決します。
* **例**:

```bash
# eth0 のARPキャッシュを強制的にクリア
sudo ip neigh flush dev eth0
```

### 2. ブルーチーム視点

* **タスク**: ARPスプーフィング攻撃（中間者攻撃）を検知する。
* **組み合わせ**: `ip neigh show`
* **解説**: ネットワーク内で不審な挙動が観測された際、`ip neigh show` を実行します。もし、**ゲートウェイのIPアドレスに対するMACアドレスが、本来のルーターのものとは異なる不審な値**に書き換わっている場合、ARPスプーフィング攻撃を強く示唆します。
* **例**:

```bash
# ゲートウェイ (192.168.1.1) のMACアドレスを確認
ip n show 192.168.1.1
```

### 3. レッドチーム視点

* **タスク**: パケットを送信せずに、同じネットワークセグメント上の生存ホストを偵察する。
* **組み合わせ**: `ip neigh show`
* **解説**: `nmap` のようなアクティブスキャンを行う前に、まず `ip neigh show` を実行します。ホストが最近通信した他のホストのIP/MACアドレスがキャッシュに残っているため、**パケットを一切送信することなく**、同じセグメント上に存在する他のホスト（潜在的な次の標的）を発見できます。
* **例**:

```bash
# 現在のARPキャッシュから内部ネットワークのホストをリストアップ
ip n
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `RTNETLINK answers: Operation not permitted`
    * **考えられる原因**: ARPテーブルの変更 (`add`, `del`, `flush`) を一般ユーザー権限で実行しようとしました。
    * **解決策**: コマンドの前に `sudo` を付けて実行してください。

## 環境変数と設定ファイル

* `ip neigh` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: ARPプロトコル自体に認証の仕組みがないため、**ARPスプーフィング（中間者攻撃）**に対して脆弱です。
* **悪用シナリオ**: 攻撃者はローカルネットワーク内でARP応答を偽装し、クライアントとゲートウェイ間の通信を自身のマシン経由にさせることで、通信を盗聴したり改ざんしたりします。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: もしユーザーが `sudo ip` を実行できる権限を持っている場合、`-f` オプションでスクリプトファイルを読み込ませる機能などを悪用して、任意のコマンドを実行することが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/ip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 重要なサーバーには、ゲートウェイの静的なARPエントリ (`ip neigh add ... nud perm`) を設定する。動的ARPインスペクション (DAI) などの機能をサポートする高機能なネットワークスイッチを導入する。
* **Detection (検知)**: `arpwatch` などのツールを導入し、ARPテーブルの変更を常時監視・通知する。

## 注意点・補足

* **キャッシュの状態 (State)**: `REACHABLE` (到達可能), `STALE` (古いキャッシュ、通信時に再確認される), `FAILED` (到達失敗) などの状態は、カーネルがARPエントリをどのように管理しているかを示しています。
