# Command - Linux - ip rule - ルーティングポリシーデータベース(RPDB)の管理

## 概要
`ip rule` オブジェクトは、ルーティングポリシーデータベース（RPDB）を操作します。通常、カーネルは「main」テーブルという一つの地図を見てパケットを送りますが、`ip rule` を使うと「特定の送信元からの通信はテーブルAを、それ以外はテーブルBを使う」といった高度な制御が可能になります。



## 主要サブコマンド
| サブコマンド | 略称 | 説明 |
| :--- | :--- | :--- |
| **`list`** | `ls`, `l` | 現在設定されているルールを優先度順に表示する（デフォルト）。 |
| **`add`** | `a` | 新しいルール（条件とアクション）を追加する。 |
| **`del`** | `d` | 既存のルールを削除する。 |
| **`flush`** | | ルールをすべて削除する（注意：通信不能になる可能性が高い）。 |

## コマンド例と構文

### 1. 現在のルールの確認 (list)
ルールは「優先度（数値が小さいほど先）」に従って上からチェックされます。
```bash
# ルールの一覧を表示
ip rule list
````

**標準的な出力例:**

- `0: from all lookup local` (自ホスト宛などの特殊通信)
    
- `32766: from all lookup main` (通常の `ip route` で設定するテーブル)
    
- `32767: from all lookup default` (通常は空の予備テーブル)
    

### 2. ソースルーティングの設定 (add)

「特定の送信元 IP」をトリガーにして、別のルートを通したい場合に使用します。

Bash

```
# 192.168.10.0/24 からの通信には、テーブル ID 100 の地図を使わせる
sudo ip rule add from 192.168.10.0/24 table 100

# eth1 から入ってきた通信には、テーブル ID 200 を使わせる
sudo ip rule add iif eth1 table 200
```

### 3. ルールの削除 (del)

追加した時と同じ条件を指定して削除します。

Bash

```
# 優先度を指定して削除（安全・確実）
sudo ip rule del priority 1000

# 条件を指定して削除
sudo ip rule del from 192.168.10.0/24
```

## 主要なセレクタ（条件）

- **`from <ADDR>`**: 送信元 IP アドレス。
    
- **`to <ADDR>`**: 宛先 IP アドレス。
    
- **`iif <NAME>`**: 入力インターフェース名（Input Interface）。
    
- **`oif <NAME>`**: 出力インターフェース名（Output Interface）。
    
- **`fwmark <MARK>`**: `iptables` 等で付けられたマーク値を条件にする。
    

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点

- **タスク**: マルチホーム（2回線）環境での戻りパケット制御。
    
- **解説**: 2つのプロバイダーと契約しているサーバーで、「回線Aから入ってきた通信の返信は、必ず回線Aから出す」といった挙動をさせるために必須のコマンドです。
    

### 2. ブルーチーム視点

- **タスク**: VPN トラフィックの強制（キルスイッチ）。
    
- **解説**: 特定のアプリケーションやプロセスの通信が、VPN を通らずに生（RAW）の回線から漏れる（リークする）のを防ぐために、VPN 以外の経路を `ip rule` で遮断します。
    

### 3. レッドチーム視点

- **タスク**: 検知回避のための隠し経路構築。
    
- **解説**: 管理者がチェックする「main」テーブルには手を触れず、自分だけが知っている別のテーブル（例: `table 1337`）にバックドア用のルートを書き込み、`ip rule` で自分の攻撃元 IP からの通信だけをその隠しルートへ誘導します。
    

## セキュリティに関する考慮事項

### 脆弱性とリスク

- **トラフィックの乗っ取り**: 攻撃者が `ip rule` を操作できると、特定のユーザーやサービスの通信だけをこっそり攻撃者所有の VPN や Proxy へ横流し（Redirect）させ、パスワードなどを窃取されるリスクがあります。
    

### 対応策・緩和策 (ブルーチーム視点)

- **RPDB の定期監査**: `ip rule list` を定期的に実行し、`priority` 32766 より上に不審な `lookup` が割り込んでいないか監視します。
    
- **権限管理**: `ip rule` はネットワークの根幹を揺るがすため、実行権限は極めて厳格に制限すべきです。
