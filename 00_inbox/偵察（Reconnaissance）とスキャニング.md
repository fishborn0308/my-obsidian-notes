

# 偵察（Reconnaissance）とスキャニング

## 1. スキャニングフェーズの全体目標

ターゲットシステムと直接対話し、環境の詳細（トポロジー、OS、サービス、脆弱性）を深く理解することで、**「具体的な侵入経路（アタックベクター）」を特定すること**が包括的な目的です。

### 1.1. 具体的な5つのゴール

1. **ネットワーク構成の可視化**
    
    - **ホストの特定**: サーバー、クライアントだけでなく、ルーター、スイッチ、FWなどのネットワーク機器を特定。
        
    - **トポロジーの把握**: デバイスの接続関係を明らかにし、**ネットワークマップ**を作成する。
        
2. **OS（オペレーティングシステム）の特定**
    
    - OSの種類やバージョンを判別し、そのOS特有の脆弱性を利用した攻撃手法（エクスプロイト）を最適化する。
        
3. **サービスとポートの調査**
    
    - **リスニングポートの列挙**: 侵入経路となり得るTCP/UDPポートの一覧を作成。
        
    - **詳細情報の取得**: アプリケーション名やプロトコル（HTTP, SSH等）の**詳細なバージョン**を特定する。
        
4. **潜在的な脆弱性のリスト化**
    
    - 特定したOS・サービス情報を既知の脆弱性と照合。ネットワークの応答挙動から不備を推測する。
        
5. **リスクの最小化**
    
    - システムの速度低下やクラッシュを避ける手法を選択する。※対話型スキャンには常に負荷リスクが伴う。

## 2. スキャンの種類とワークフロー

### 2.1. 主なスキャンの種類と目的

|**スキャン種別**|**内容と目的**|
|---|---|
|**ネットワークスイープ**|稼働しているホストとその**IPアドレス**を特定する。|
|**ポートスキャン**|どの**TCP/UDPポート**が開放（リスニング状態）されているか確認する。|
|**OSフィンガープリンティング**|パケットの応答特性から**OS（Windows, Linux等）**を特定する。能動的/受動的手法がある。|
|**バージョンスキャン**|稼働中の**サービスのバージョン**を特定。代替ポートでの稼働も判別可能。|
|**脆弱性スキャン**|設定ミスや未修正のバグなど、数千の**潜在的脆弱性**を網羅的に調査する。|

### 2.2. スキャンフェーズの標準的なワークフロー

1. **ターゲット特定**: ネットワークスイープで「攻撃対象」を抽出。
    
2. **構造把握**: ネットワークアーキテクチャや接続関係を解明。
    
3. **サービス特定**: ポートスキャンで「入口」を探し、バージョンスキャンで「中身」を確認。
    
4. **属性特定**: OSフィンガープリンティングでマシンの種類を特定。
    
5. **弱点診断**: 最終的に脆弱性スキャンを実施し、侵入の足がかりを決定。

## 3. 内部ネットワークスキャンの実行ワークフロー

各フェーズは前のステップの結果を基盤とし、規模に合わせてツールを使い分けます。

### 3.1. フェーズ別の詳細

1. **ネットワーク範囲の特定（マッピング）**
    
    - **手法**: IP範囲（/24等）の確認、逆DNSルックアップ、クライアント情報の活用。
        
2. **ホストおよびポートの検出**
    
    - **中規模（1万ホスト未満）**: **Nmap** を使用（詳細度と速度のバランス）。
        
    - **大規模**: **Masscan** を使用（高速処理、低オーバーヘッド）。
        
3. **稼働サービスの検出（認証不要スキャン）**
    
    - Nmapを用いて、認証なしでアクセス可能なサービスからセキュリティ弱点を特定。
        
4. **アプリケーション固有のスキャン（詳細分析）**
    
    - **Webサーバー等**: **EyeWitness** 等の専用ツールでスクリーンショットの取得や詳細調査を実施。

## 4. 大規模スキャンの課題と戦略

### 4.1. 現実的な制約（シミュレーション）

1,000台のホストに対して全TCP/UDPポート（約1.3億ポイント）を1秒1ポートでスキャンすると、**約4.15年**を要します。100並列でも15日以上かかるため、効率化が不可欠です。

### 4.2. 大規模スキャンを処理する5つの戦略

1. **ポートのサンプリング**
    
    - 全ポートではなく、主要な10～1,000ポートに絞る。※標準外ポートの見逃しリスクあり。
        
2. **ホストのサンプリング**
    
    - 全端末ではなく、代表的な構成のシステムをサンプルとして選択。※ユニークな構成を逃すリスクあり。
        
3. **スキャンシステムの分散配置**
    
    - 複数の場所から分散して実行。※アラートの増加や調整の難化を招く。
        
4. **非同期スキャンの活用**
    
    - **Masscan** 等のステートレスなツールを使用し、送信と受信を分離して高速化。
        
5. **分離型ネットワーク検出（二段階スキャン）**
    
    - 先にホスト検出（Ping等）を行い、反応があったホストのみポートスキャン。※偽陰性のリスクあり。
        

---

## 5. まとめ：手法選択のポイント

- **スピード重視**: 非同期スキャン、ポートサンプリング。
    
- **正確性重視**: 分散配置、全ポート調査。
    
- **現実的な最適解**: 重要なセグメントには全ポート、それ以外にはサンプリングといった手法を組み合わせる。
