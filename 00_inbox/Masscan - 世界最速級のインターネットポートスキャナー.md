# Masscan - 世界最速級のインターネットポートスキャナー

## 1. 概念説明
**Masscan**は、極めて高速に動作するTCPポートスキャナーです。Nmapと似た操作性を持ちつつ、内部的には独自のTCP/IPスタックを実装しており、非同期送信を行うことで、理論上は**毎秒2,500万パケット**という驚異的な速度でスキャンが可能です。

Nmapが特定のホストの詳細（OSやサービス）を深く調査するのに適しているのに対し、Masscanは「広大なネットワーク（インターネット全体など）から特定のポートが開いているIPを瞬時に炙り出す」という用途に特化しています。



## 2. コマンド概要とオプション解説

### 基本構文
`masscan [ターゲット指定] -p[ポート] [オプション]`

### 必須・重要オプション
| オプション | 解説 |
| :--- | :--- |
| `-p <port>` | スキャン対象ポート（例: `-p80,443`, `-p1-65535`, `-pU:53` (UDP)）。 |
| `--rate <number>` | 送信レート（パケット/秒）。デフォルトは100。1M（100万）などの指定が可能。 |
| `--banners` | バナーグラビング（HTTP ServerヘッダやSSHバージョン等）を有効化。 |
| `--exclude <file>` | スキャン対象から除外するIPリスト（政府機関、自社資産など）を指定。 |
| `--wait <seconds>` | 最後のパケット送信後の待機時間。デフォルトは10秒。 |

### ネットワーク構成オプション
| オプション | 解説 |
| :--- | :--- |
| `--adapter-ip <IP>` | スキャンに使用する自身のIPアドレスを指定。 |
| `--adapter-port <Port>` | スキャンに使用するソースポートの範囲を指定。 |
| `--router-mac <MAC>` | ローカルルーターのMACアドレスを指定（自動検出に失敗する場合）。 |

### 出力オプション
| オプション | 解説 |
| :--- | :--- |
| `-oX <file>` | XML形式で保存。 |
| `-oJ <file>` | JSON形式で保存（解析が容易なため推奨）。 |
| `-oL <file>` | リスト形式（単純なテキスト）で保存。 |
| `-oG <file>` | Grep可能な形式で保存。 |

## 4. セキュリティ上の留意点

- **帯域飽和のリスク**: `--rate` を高く設定しすぎると、自組織のネットワーク帯域を使い果たしたり、上流のルーターをクラッシュさせる（DoS状態）可能性があります。
    
- **偽装IPの使用**: Masscanは独自のスタックを使うため、OSのTCPスタックを通しません。そのため、OSが想定外の応答パケットに対しRSTを送ってしまうことがあります。これを防ぐには `iptables` で該当ポートのRSTをDROPする設定が推奨されます。
    
- **法的・倫理的配慮**: インターネット全体へのスキャンは、多くの組織で攻撃予兆として検知され、プロバイダーへの通報対象となります。

## 1. Masscan の概要（要約）

Masscanは、速度を極限まで追求したオープンソースのポートスキャナーです。Robert Graham氏によって開発され、Nmapと似た構文を持ちながらも、内部構造は大きく異なります。

- **圧倒的なスピード:** カスタムTCP/IPスタックを採用。高速な回線があれば、単一のマシンから毎秒1,000万パケットを送信でき、インターネット全体（全IPv4アドレス）を5分未満でスキャン可能です。
    
- **「速度」への特化:** Nmapとは異なり、スキャン前のホスト探索（Ping等）を行わず、即座にポートスキャンを開始します。
    
- **高い互換性:** Linux（推奨）、Windows、macOSで動作します。
    
- **出力の柔軟性:** 大規模なスキャン結果を効率的に保存するための独自バイナリ形式を備えています。

## 2. Masscan 実用マニュアル

### スキャン準備



### 基本構文

Masscanを使用する際は、ターゲット、ポートに加え、パケット送信速度（`--rate`）の指定が重要です。

Bash

```
masscan -p [ポート] [ターゲットIP/範囲] --rate=[速度]
```

### クイックスタート・例

**例1：特定のサブネットをスキャン（標準的な安全設定）**

10.x.x.xの範囲（1,600万アドレス）に対し、主要な3つのポートを1秒間に15,000パケットの速度でチェックします。

Bash

```
masscan -p 22,445,3389 10.0.0.0/8 --rate 15000
```

**例2：Web関連ポートの素早い確認**

Bash

```
masscan -p 80,8000-8100 10.0.0.0/8 --rate 10000
```

> [!IMPORTANT]
> 
> **TIP: レート制限の適用**
> 
> 本ドキュメントの著者によれば、**15,000**が一般的に安全な上限値とされています。これを超える速度はネットワーク機器やターゲットに過負荷をかける恐れがあるため、必ずテストを行ってください。

---

### 出力形式と管理

大規模なスキャンでは、効率のために**バイナリ形式（-oB）**での保存が推奨されます。

|**オプション**|**形式**|**特徴**|
|---|---|---|
|**-oB**|**バイナリ**|**推奨。** 高速かつ省容量。後で他形式へ変換可能。|
|**-oL**|リスト|ステータス、プロトコル、ポート、IP、時間を列挙。|
|**-oX**|XML|NmapのXML形式と互換。|
|**-oG**|Grep可能|`grep`や`cut`などのコマンドで解析しやすい。|
|**-oJ**|JSON|プログラマブルな解析に適した形式。|

---

### データの変換と抽出（逆引きリファレンス）

スキャン結果（バイナリファイル）から必要な情報を抽出する手順です。

#### 1. バイナリを他形式に変換する

バイナリファイルを読み込むには `--readscan` オプションを使用します。

- **XMLへ変換:**
    
    `masscan --readscan scan.bin -oX scan.xml`
    
- **Grep可能形式（.gnmap）へ変換:**
    
    `masscan --open --readscan scan.bin -oG scan.gnmap`
    

#### 2. 特定の情報だけを抽出する（Linuxコマンド活用）

`.gnmap` 形式に変換したファイルから、特定のデータを抽出する便利なワンライナーです。

- **ポート80が開いている全IPアドレスを取得:**
    
    Bash
    
    ```
    grep /open/ scan.gnmap | cut -d ' ' -f 2 | sort -uV > ip_list_80.txt
    ```
    
- **「IP:ポート」の形式で一覧を取得:**
    
    Bash
    
    ```
    grep /open/ scan.gnmap | cut -d/ -f 1 | cut -d ' ' -f 2,4 | sed -e 's/ /:/g' | sort -uV > host_port_list.txt
    ```
    
- **開いているポート番号の種類をすべてリストアップ:**
    
    Bash
    
    ```
    grep /open/ scan.gnmap | cut -d ' ' -f 4 | cut -d / -f 1 | sort -nk 1 | uniq > open_ports.txt
    ```
