# Command - Windows - cmd - robocopy - 堅牢なファイル複製と同期

## 概要

`robocopy` (Robust File Copy) は、Windows のディレクトリおよびファイルを同期するための非常に強力なコマンドラインツールです。ネットワークの瞬断に対する再試行機能、属性や所有権の維持、ミラーリング機能などを備えており、サーバー移行やバックアップのデファクトスタンダードです。

(出自: `Windows 標準搭載 - 外部コマンド`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `xcopy` | シンプルだが再試行機能がなく、大規模なコピーではエラーで止まることが多い。 | 小規模 |
| `copy` | 単一ファイルのコピー用。ディレクトリ構造は扱えない。 | 単一ファイル |

## よく連携されるコマンド

### シナリオ例: サーバー間のフォルダ完全同期 (インフラ運用視点)

* **目的**: ソースとターゲットを全く同じ状態にする（ミラーリング）。
* **連携コマンド**: `robocopy`, `taskschd.msc`
* **解説**: `/MIR` スイッチにより、コピー元にないファイルをコピー先から削除し、完全に一致させます。
* **コマンド例**:
    ```cmd
    REM DドライブのデータをEドライブにミラーリングし、ログを記録
    robocopy D:\Data E:\BackupData /MIR /R:3 /W:5 /LOG:C:\logs\backup.log
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/S` | 空でないサブディレクトリをコピーする。 |
| ⭐ `/E` | 空のディレクトリを含むサブディレクトリをコピーする。 |
| ⭐ `/MIR` | ミラーリング（`/E` と `/PURGE` の組み合わせ）。削除も同期される。 |
| `/ZB` | 再起動可能モードを使用し、アクセス拒否時はバックアップモードを使用。 |
| `/COPYALL` | 全ファイル情報（データ、属性、タイムスタンプ、ACL、所有者、監査）をコピー。 |
| `/R:n` | 失敗したコピーに対する再試行回数（デフォルトは100万回）。 |
| `/W:n` | 再試行間の待機時間（秒）。 |
| `/MT:n` | マルチスレッド実行（デフォルト8）。高速化に寄与。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: NTFSアクセス権（ACL）を維持したまま、ファイルサーバーのデータを新サーバーへ移行する。
* **例**:
    ```cmd
    REM 権限情報を保持しつつ、マルチスレッドで高速コピー
    robocopy \\OldServer\Share \\NewServer\Share /E /COPYALL /MT:32 /LOG:migration.log
    ```

### 2. ブルーチーム視点

* **タスク**: 証拠保全（フォレンジック）の際、タイムスタンプを書き換えずにファイルをコピーする。
* **例**:
    ```cmd
    REM ファイルの属性やタイムスタンプを完全に維持してコピー
    robocopy C:\Evidence D:\Collection /E /COPY:DAT
    ```

## 注意点・補足

* **`/MIR` の破壊力**: コピー先にしかないファイルは削除されるため、指定ミスをするとデータ消失に繋がります。実行前に `/L` (テスト実行モード) で動作を確認することを強く推奨します。
