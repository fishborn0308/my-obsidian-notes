# `docker network rm` - 1つ以上のDockerネットワークを削除する

## 概要

`docker network rm` (または `docker network remove`) は、指定した1つ以上のDockerネットワークを削除するためのコマンドです。プロジェクトが終了し、コンテナがすべて削除された後に残ったカスタムネットワークなどをクリーンアップし、`docker network ls` の一覧を整理するために使用します。デフォルトでは、**どのコンテナからも使用されていないネットワークのみ**を削除できます。

(出自: `docker` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`docker network prune`** | **未使用の**ネットワークを**一括で**削除します。`rm` は削除するネットワークを**個別に**指定します。 | `prune` (一括削除) |
| **`docker compose down`**| **Composeプロジェクト**に関連するネットワークを、コンテナなどと一緒に一括で削除します。 | - |

## よく連携されるコマンド

`docker network rm` は、`docker network ls` で削除対象を確認した後に実行するのが一般的なワークフローです。

### シナリオ例: 不要になったネットワークのクリーンアップ (インフラ構築・運用視点)

* **目的**: テスト用に作成したネットワーク `test-net` を削除する。
* **連携コマンド**: `docker network ls`, `docker ps`
* **解説**: まず `docker network ls` でネットワークが存在することを確認します。もし削除に失敗した場合は、そのネットワークに接続しているコンテナがないか `docker ps -a` などで確認し、先にコンテナを削除します。
* **コマンド例**:

    ```bash
    # 1. ネットワーク一覧を確認
    docker network ls

    # 2. 不要なネットワークを削除
    docker network rm test-net
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **動作制御** | |
| ⭐ `<network...>` | 削除するネットワークの名前またはIDを一つ以上、スペース区切りで指定します。 |
| `-f`, `--force` | 現在はほぼ効果がありません。古いバージョンとの互換性のために残されています。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 不要になったカスタムネットワークを個別に削除する。
* **組み合わせ**: `docker network rm <network_name>`
* **解説**: テスト環境を片付ける際に、コンテナを削除した後、最後にこのコマンドでネットワークも削除し、リソースを完全にクリーンアップします。
* **例**:

    ```bash

    # テストプロジェクトのコンテナを削除
    docker rm test-webapp test-db

    # 残ったネットワークを削除
    docker network rm test-project-net
    ```

### 2. ブルーチーム視点

* **タスク**: 攻撃者が作成した不審なネットワークを削除する（クリーンアップ）。
* **組み合わせ**: `docker network rm <suspicious_network_name>`
* **解説**: インシデント対応後、攻撃者が活動のために作成したとみられる不審なネットワークを、関連コンテナを全て削除した上で削除し、システムを正常な状態に戻します。
* **例**: -

### 3. レッドチーム視点

* **タスク**: **痕跡消去 (Anti-Forensics)**。
* **組み合わせ**: `docker network rm <attacker_network_name>`

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](../linux/troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `Error response from daemon: network <network_name> is in use by container <container_id>`
    * **考えられる原因**: **最もよくあるエラー**。削除しようとしているネットワークが、まだ存在するコンテナによって使用されています。
    * **解決策**: エラーメッセージに表示されたコンテナIDを `docker rm` で先に削除してください。（もしコンテナが実行中なら `docker stop` が先に必要です）

## 環境変数と設定ファイル

* `docker network rm` の動作に直接影響する環境変数や設定ファイルは通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](../linux/environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: **意図しないサービス妨害**。
* **悪用シナリオ**: 稼働中のアプリケーションが使用しているネットワークを誤って、あるいは意図的に削除しようとすると、Dockerデーモンはエラーを返して削除を防ぎます。しかし、攻撃者がコンテナを停止させる権限も持っている場合、コンテナを停止させてからネットワークを削除することで、サービスの復旧を困難にさせることができます。

### GTFOBins / LOLBAS における利用例

`docker network rm` は `docker` コマンドの一部として扱われます。`sudo docker` の権限があれば、任意のネットワーク操作が可能です。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: Dockerソケットへのアクセス権限を厳格に管理する。
* **Detection (検知)**: `auditd` で `docker network rm` の実行を監視し、計画外のネットワーク削除がないかを確認する。

## 注意点・補足

* **デフォルトネットワーク**: Dockerがデフォルトで作成する `bridge`, `host`, `none` ネットワークは、通常 `docker network rm` では削除できません。
