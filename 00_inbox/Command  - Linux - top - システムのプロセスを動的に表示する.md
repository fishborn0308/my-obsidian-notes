# Command  - Linux - top - システムのプロセスを動的に表示する

## 概要

`top` は、Linux/Unixシステムで実行中のプロセスを**リアルタイム**で監視し、リソース（CPU、メモリ）を多く消費している順に一覧表示するための、古典的で基本的なコマンドです。システムのパフォーマンス状態を動的に把握し、どのプロセスがシステムに負荷をかけているかを特定するために使用されます。

`htop` ほど直感的ではありませんが、ほぼ全てのLinuxシステムにデフォルトでインストールされているため、どのような環境でも確実に使えるという大きな利点があります。

(出自: `procps-ng` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`htop`** | `top` のモダンで高機能な後継。色付けされた表示、マウスやカーソルキーでのスクロール、直感的な操作が可能で、`top` よりも遥かに使いやすいです。 | `htop` (対話的監視) |
| **`ps`** | コマンド実行**時点**のプロセスの**スナップショット**を静的に表示します。`top` は**リアルタイム**で更新され続ける動的なビューを提供します。 | - |
| **`free`** | **メモリ**の全体的な使用状況の概要を表示します。`top` はプロセスごとの詳細なメモリ使用量も表示します。 | - |

## よく連携されるコマンド

`top` は対話的なツールですが、バッチモード (`-b`) を使うことで、その出力を他のコマンドに渡して処理できます。

### シナリオ例: 特定のユーザーのプロセス情報をログに記録する (インフラ構築・運用視点)

* **目的**: ユーザー `www-data` が実行しているプロセスのリソース使用状況のスナップショットを取得し、ログファイルに保存する。
* **連携コマンド**: `grep`, リダイレクション (`>`)
* **解説**: `top -b -n 1` で現在のプロセス情報のスナップショットを一度だけ出力し、その中から `grep` を使って特定のユーザーの行だけをフィルタリングしてファイルに保存します。
* **コマンド例**:

    ```bash
    # topの出力を一度だけバッチモードで取得し、www-dataユーザーのプロセスを抽出してログに保存
    top -b -n 1 | grep "www-data" > /var/log/www-data_processes.log
    ```

## オプション説明 (対話モードのキー操作)

`top` 実行中に以下のキーを押すことで、表示を操作できます。

| オプション | 説明 |
| :--- | :--- |
| **表示と終了** | |
| ⭐ `q` | `top` を**終了**します。 |
| ⭐ `P` (大文字) | **CPU使用率**でプロセスをソートします。 |
| ⭐ `M` (大文字) | **メモリ使用率**でプロセスをソートします。 |
| `1` (数字の1)| CPUコアごとの使用率表示をトグルで切り替えます。 |
| **プロセス操作** | |
| ⭐ `k` | プロセスを**kill**します。PIDとシグナル番号の入力を求められます。 |
| `r` | プロセスの**renice**（優先度を変更）します。 |
| `u` | 特定の**ユーザー**のプロセスのみをフィルタリング表示します。 |
| **その他** | |
| `h` | ヘルプ画面を表示します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 「サーバーが重い」の一次調査として、リソースを大量消費しているプロセスを特定する。
* **組み合わせ**: (対話モード) `P` (CPUソート) -> `M` (メモリソート)
* **解説**: `top` を起動し、`P` や `M` キーでリソースを異常に消費しているプロセスを特定します。暴走しているプロセスは `k` で停止を試みます。
* **例**:

    ```bash
    # サーバーのパフォーマンス問題を調査するためにtopを起動
    top
    ```

### 2. ブルーチーム視点

* **タスク**: ライブレスポンス中に不審なプロセスの活動を調査する。
* **組み合わせ**: `top` (対話モード)
* **解説**: `htop` が使えない環境で `top` を使用します。CPUを異常に消費している偽装名のプロセス（クリプトマイナーなど）や、サービスアカウントから起動された予期せぬプロセスがないかを確認します。
* **例**:

    ```bash
    # ユーザー 'www-data' のプロセスのみをフィルタリングして表示
    # topを起動後、'u' キーを押して 'www-data' と入力
    ```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察と、自身の活動の防御回避。
* **組み合わせ**: `top`
* **解説**:
  * **偵察**: 攻撃者は `top` を実行し、システムでどのようなサービスやセキュリティ製品（EDRエージェントなど）が動いているかを把握します。
  * **防御回避**: 自身のマルウェアがCPUを過剰に消費すると `top` の一覧の上位に表示され発見されやすくなるため、リソース消費を低く抑えます。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `top` 自体ではなく、その**表示内容**が**情報漏洩**に繋がります。
* **悪用シナリオ**: `top` は実行中の全てのプロセスと、その**コマンドライン引数**を表示します。もし、引数にパスワードなどの機密情報が平文で渡されている場合、`top` を実行できる全てのユーザーにその情報が閲覧可能になってしまいます。

### GTFOBins / LOLBAS における利用例

`top` は情報表示が主目的であり、直接的な権限昇格機能はないため、GTFOBinsにはリストされていません。

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: アプリケーションにパスワードなどの機密情報を渡す際は、コマンドライン引数ではなく、環境変数や設定ファイル、シークレット管理ツールを使用する。
* **Detection (検知)**: `auditd` で `top` コマンドの実行を監視し、不審なユーザーによる偵察活動がないかを確認する。

## 注意点・補足

* **バッチモード**: `top -b -n 1` のようにバッチモードで実行すると、現在のスナップショットを一度だけ標準出力に表示して終了します。この出力は `grep` に渡したり、ログとしてファイルに保存したりするのに便利です。
