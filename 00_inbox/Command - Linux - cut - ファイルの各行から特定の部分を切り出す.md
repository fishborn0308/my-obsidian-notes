# Command - Linux - cut - ファイルの各行から特定の部分を切り出す

## 概要
`cut` は、テキストファイルや標準入力の各行から、バイト単位、文字単位、あるいはフィールド（列）単位で特定の部分を抽出し、表示するコマンドです。CSVやタブ区切りのログ、コロン区切りの設定ファイルから特定のデータを抜き出すのに最適です。
(出自: `coreutils` パッケージに含まれる)

## 類似コマンドと差異
| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `awk` | 多機能でプログラム的な処理が可能。空白の数が変動する列の抽出に強い。 | 複雑な加工・集計 |
| `sed` | 正規表現を用いた置換や行の抽出に強い。 | ストリーム編集 |
| `cut` | 処理が軽量で高速。区切り文字が明確なファイルからの列抽出に特化。 | 単純な列の切り出し |

## よく連携されるコマンド
### シナリオ例: システム内ユーザー名のリストアップ (運用・偵察視点)
* **目的**: `/etc/passwd` ファイルからユーザー名だけを抽出してリスト化する。
* **連携コマンド**: `sort`, `uniq`
* **解説**: `:` を区切り文字として指定し、1番目のフィールドを抽出します。
* **コマンド例**:
    ```bash
    # システムユーザーの一覧を取得し、アルファベット順に並べる
    cut -d: -f1 /etc/passwd | sort
    ```

## オプション説明
| オプション | 説明 |
| :--- | :--- |
| **抽出単位 (いずれか1つ必須)** | |
| ⭐️ `-f`, `--fields=LIST` | 指定したフィールド（列）を抽出する。 |
| `-b`, `--bytes=LIST` | 指定したバイト位置を抽出する。 |
| `-c`, `--characters=LIST` | 指定した文字位置を抽出する。 |
| **詳細設定** | |
| ⭐️ `-d`, `--delimiter=DELIM` | フィールドの区切り文字を指定する（デフォルトはタブ）。 |
| `--complement` | 指定したフィールド「以外」を抽出する。 |
| `-s`, `--only-delimited` | 区切り文字を含まない行を表示しない。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点
* **タスク**: サーバーのIPアドレス（IPv4）のみを抽出。
* **組み合わせ**: `hostname -I` または `ip addr` との連携。
* **解説**: スペース区切りの出力から、最初のフィールドだけを抜き出します。
* **例**:
    ```bash
    # サーバーのプライマリIPアドレスを取得
    hostname -I | cut -d' ' -f1
    ```

### 2. ブルーチーム視点
* **タスク**: `last` コマンドの結果から、ログインしたユーザー名とIPアドレスのみを抽出。
* **組み合わせ**: `last` との連携。
* **解説**: 固定長またはスペース区切りの出力から必要な列を抜き出し、調査を簡略化します。
* **例**:
    ```bash
    # 直近のログインユーザー名と接続元IPを抽出
    last -i | cut -c1-9,23-38
    ```

### 3. レッドチーム視点
* **タスク**: 侵入後の内部偵察における、ログイン可能なシェルを持つユーザーの特定。
* **組み合わせ**: `grep` でのフィルタリング後。
* **解説**: `/etc/passwd` から、シェルが `/bin/bash` や `/bin/sh` になっているユーザー名だけを抜き出します。
* **例**:
    ```bash
    # 有効なシェルを持つユーザーのリスト作成（パスワード攻撃用）
    grep -E "/bash$|/sh$" /etc/passwd | cut -d: -f1 > users.txt
    ```

## エラーメッセージとトラブルシューティング

1.  **エラーメッセージ例 1**: `cut: fields are numbered from 1`
    * **考えられる原因**: オプション `-f0` のように、0番目のフィールドを指定した。
    * **解決策**: フィールド番号は 1 から始まるため、1以上の数値を指定する。

2.  **空白区切りで上手く切り出せない**:
    * **考えられる原因**: `cut` は「連続する空白」を1つの区切りとして扱わず、1つ1つの空白を区切りとして扱います。
    * **解決策**: 空白の数が一定でない場合は `awk '{print $n}'` を使用するか、`tr -s ' '` で空白を1つにまとめてから `cut` に渡す。

## 環境変数と設定ファイル
* `cut` は特定の環境変数や設定ファイルを持ちません。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例
* **脆弱性**: それ自体に脆弱性は稀ですが、**情報の抜き出し（Data Exfiltration）**において極めて効率的に動作します。
* **悪用シナリオ**: 攻撃者が Web シェル経由で OS コマンドを実行できる場合、`cut` を使って設定ファイルからデータベースのパスワードや API キーが記述されている行の、特定の列（値の部分）だけを隠密に抽出します。

### GTFOBins における利用例
`cut` に `sudo` 権限がある場合、本来読み取れないファイル（`/etc/shadow` 等）の一部を「切り出して表示」することで、実質的に内容を窃取できます。
* **機能**: `File Read`, `Sudo`
* **コマンド例**:
    ```bash
    # sudo cut を使って /etc/shadow の1行目（rootのハッシュ等）を読み取る
    sudo cut -d: -f1- /etc/shadow
    ```
* **参照**: `https://gtfobins.github.io/gtfobins/cut/`

### 対応策・緩和策 (ブルーチーム視点)
* **Prevention (予防)**: 重要な機密ファイルの権限を最小限にし、一般ユーザーが `sudo cut` を実行できないようにする。
* **Detection (検知)**: `/etc/shadow` や設定ファイルに対して `cut` コマンドが実行されているログを `auditd` で監視する。

## 注意点・補足
* **マルチバイト文字**: 古いバージョンの `cut` はバイト単位で処理を行うため、日本語などのマルチバイト文字を `-c` で指定すると文字化けする場合があります。現代の GNU 版では改善されています。
* **デリミタの指定**: スペースを区切り文字にする場合は `-d' '` （シングルクォートでスペースを囲む）と記述します。
