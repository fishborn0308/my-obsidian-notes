# Command - Windows - cmd - netstat - ネットワーク接続と統計情報の表示

## 概要

`netstat` (Network Statistics) は、アクティブな TCP 接続、コンピュータがリスンしているポート、イーサネット統計情報、IP ルーティング テーブル、IPv4/IPv6 統計情報などのネットワーク接続に関する詳細情報を表示するコマンドです。
システムの通信状態をリアルタイムで把握し、不審な外部通信や内部サービスの待機状態を確認するために使用されます。

(出自: `Windows 標準搭載 - TCP/IP ユーティリティ`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `ss` | Linux における `netstat` の後継。より高速で詳細。 | (Linux環境) |
| `Get-NetTCPConnection` | PowerShell 版。オブジェクトを返すため、「特定のプロセスが持つ接続のみ」を抽出する等のフィルタリングが容易。 | PS 環境 |
| `TCPView` | Sysinternals ツール。GUI でリアルタイムに通信とプロセスを紐付けて表示する。 | 視覚的な調査 |

## よく連携されるコマンド

### シナリオ例: 特定のポートを使用しているプロセスの特定 (ブルー/レッドチーム共通)

* **目的**: 意図しないポートが開いている際、どの実行ファイルがその原因（犯人）かを突き止める。
* **連携コマンド**: `netstat`, `findstr`, `tasklist`
* **解説**: `netstat -ano` で PID を取得し、`tasklist` でその PID に紐付く実行ファイル名を確認します。
* **コマンド例**:
    ```cmd
    REM 8080番ポートを使っているプロセスの PID を確認
    netstat -ano | findstr :8080

    REM 見つかった PID (例: 1234) からプログラム名を特定
    tasklist /FI "PID eq 1234"
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `-a` | すべての接続とリスニング ポート（待機状態）を表示する。 |
| ⭐ `-n` | アドレスとポート番号を数値形式（IP:Port）で表示する。名前解決を行わないため**動作が高速**。 |
| ⭐ `-o` | 各接続に関連付けられた所有者 **プロセス ID (PID)** を表示する。調査に必須。 |
| `-p proto` | 指定したプロトコル（TCP, UDP, ICMP等）の接続を表示する。 |
| `-r` | ルーティング テーブル（`route print` 相当）を表示する。 |
| `-e` | イーサネット統計情報を表示する（送受信バイト数、エラー数など）。 |
| `-s` | プロトコルごとの統計情報を表示する。 |
| `interval` | 指定した秒数ごとに再表示する（例: `netstat -n 5`）。`Ctrl+C` で停止。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点 (Docker/VMware)

* **タスク**: 新しく立ち上げた Web サーバーやコンテナが、外部からの接続を待ち受けているか確認する。
* **解説**: `0.0.0.0` (全インターフェース) か `127.0.0.1` (ローカルのみ) かを確認し、意図した公開範囲になっているかチェックします。
* **例**:
    ```cmd
    REM TCP で待機中(LISTENING)のポートのみを一覧表示
    netstat -an | findstr LISTENING
    ```

### 2. ブルーチーム視点

* **タスク**: C2 サーバーへの不審な通信（ビーコン）の検知。
* **解説**: 外部への `ESTABLISHED` (確立済み) な接続を確認し、不審な外国の IP アドレスや、標準的でないポート（例: 4444, 8080）への通信がないか調査します。
* **例**:
    ```cmd
    REM 現在確立されている外部への接続を PID 付きで表示
    netstat -ano | findstr ESTABLISHED
    ```

### 3. レッドチーム視点

* **タスク**: ローカル限定サービスの列挙 (Local Service Enumeration)。
* **解説**: ファイアウォールで外部には公開されていないが、内部で `127.0.0.1` でのみ動いているサービス（データベース管理ツール等）を見つけ出し、ポートフォワーディングの標的にします。
* **例**:
    ```cmd
    REM ローカルホスト(127.0.0.1)だけで待機しているポートを探索
    netstat -ano | findstr 127.0.0.1 | findstr LISTENING
    ```

## 実務ログ例

```cmd
C:\> netstat -ano | findstr :445
  TCP    0.0.0.0:445            0.0.0.0:0              LISTENING       4
  TCP    [::]:445               [::]:0                 LISTENING       4

C:\> netstat -ano | findstr ESTABLISHED
  TCP    192.168.1.15:52401     13.107.21.200:443      ESTABLISHED     8642
```

- `445` ポートが `LISTENING` 状態（SMBサービス稼働中）。
    
- `192.168.1.15` から外部の `13.107.21.200` に対し、HTTPS (`443`) 通信が確立されている。
    

## エラーメッセージとトラブルシューティング

1. **エラーメッセージ例 1**: `要求された操作には、昇格が必要です。`
    
    - **考えられる原因**: `-b` オプション（実行ファイル名を直接表示）など、一部のスイッチは管理者権限が必要です。
        
    - **解決策**: 管理者として実行するか、`-o` で PID を取得し `tasklist` で補完してください。
        

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

- **情報漏洩**: 稼働しているサービスや接続先 IP のリストは、ネットワークマップを作成するための重要な情報源となります。
    
- **隠蔽テクニック**: 高度なマルウェアは、フック技術を使用して `netstat` の結果から自身の通信を隠すことがあります。
    

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

- **機能**: `Reconnaissance` (偵察)
    
- **参照**: [LOLBAS - netstat.exe](https://www.google.com/search?q=https://lolbas-project.github.io/lolbas/Binaries/Netstat/)
    

## 注意点・補足

- **`-n` の重要性**: DNS の逆引きが遅い環境で `-n` を忘れると、結果が表示されるまで数分待たされることがあります。**「基本は `-an` または `-ano`」**と覚えておくのが実務的です。
    
- **IPv6**: IPv6 接続も同時に表示されるため、混同しないように注意が必要です。
