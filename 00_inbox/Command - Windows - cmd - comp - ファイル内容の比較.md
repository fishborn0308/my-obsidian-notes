# Command - Windows - cmd - comp - ファイル内容の比較

## 概要

`comp` コマンドは、2 つのファイルまたはファイル セットの内容をバイト単位で比較します。
ファイルサイズが異なる場合は比較を行わず、内容が異なる場合はその不一致箇所を（デフォルトでは 16 進数で）報告します。
非常に高速であり、大規模なバイナリファイルが同一であるかを即座に判断するのに適しています。

(出自: `Windows 標準搭載 - 外部コマンド (comp.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `fc` | (File Compare) テキストファイルの行単位の差分表示に強い。 | テキスト比較 |
| `Compare-Object` | PowerShell 版。プロパティ単位での詳細な比較が可能。 | PS 環境 |
| `diff` | Linux の標準。差分（patch形式）の出力に長けている。 | Linux 環境 |
| `CertUtil -hashfile` | ハッシュ値 (SHA256等) で比較。内容は表示しないが、完全一致を確認する。 | 改ざん検知 |

## よく連携されるコマンド

### シナリオ例: バックアップファイルと原本の整合性確認

* **目的**: 重要な構成ファイルをコピーした際、コピー中にデータが破損していないか確認する。
* **連携コマンド**: `copy`, `comp`
* **解説**: コピー直後に原本とコピー先を比較します。
* **コマンド例**:
    ```cmd
    REM 構成ファイルをバックアップし、即座に比較
    copy config.ini config.ini.bak
    comp config.ini config.ini.bak /l
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| `/a` | 不一致箇所を ASCII 文字で表示する。 |
| `/d` | 不一致箇所を 10 進数で表示する（デフォルトは 16 進数）。 |
| ⭐ `/l` | 不一致がある箇所の「行番号」を表示する。 |
| `/n=number` | 各ファイルの最初の数行だけを比較する。 |
| ⭐ `/c` | 文字の「大文字・小文字」を区別せずに比較する。 |
| `/off[line]` | オフライン属性が設定されているファイルをスキップしない。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 新旧 2 台のサーバーにある設定ファイル（例: `hosts` ファイルや `web.config`）が同一であることを保証する。
* **例**:
    ```cmd
    REM 文字の違いを行番号で特定する
    comp \\SRV-OLD\C$\Windows\System32\drivers\etc\hosts C:\temp\hosts /l
    ```

### 2. ブルーチーム視点 (フォレンジック)

* **タスク**: 侵害が疑われるシステムにおいて、標準のシステムバイナリ（例: `cmd.exe`）が、クリーンなバックアップと同一かどうかを確認し、バックドアの混入を検知する。
* **注意**: 高度な攻撃者はファイルサイズを維持したまま改ざんを行うことがありますが、`comp` はバイト単位でチェックするため、1 バイトでも異なれば不一致を報告します。

### 3. レッドチーム視点 (偵察とデータ整合性)

* **タスク 1: Target Identification (バージョンの特定)**
    * **目的**: ターゲットサーバー上の DLL や実行ファイルと、手元の既知のバージョンのファイルを比較し、パッチの適用状況を推測する。
    * **解説**: 特定の脆弱性があるバージョンのファイルとバイト単位で一致すれば、エクスプロイトが成功する確率が高まります。
* **タスク 2: Exfiltration Verification (データ持ち出しの検証)**
    * **目的**: 秘密裏に分割・結合して持ち出したデータが、原本と乖離していないかを確認する。

## エラーメッセージとトラブルシューティング

1.  **Files are different sizes.**
    * **原因**: `comp` はファイルサイズが異なる場合、中身を比較せずに終了します。
    * **解決策**: サイズが異なるテキストファイルの「内容」を比較したい場合は、`fc` コマンドを使用してください。

2.  **Compare more files (Y/N)?**
    * **原因**: 対話型コマンドのため、1 つの比較が終わると次の比較をするか聞かれます。
    * **解決策**: スクリプト内で自動化する場合は、`echo n | comp file1 file2` のように入力をパイプで渡します。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **機密情報の露出**: 
    * 攻撃者は `comp` を使用して、2 つの暗号化されたファイル（あるいはパスワードハッシュのリスト）を比較し、同一の平文から生成されたものがないかを推測する一助にすることがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Enumeration` (情報の列挙)
* **特性**: Windows 標準のコマンドであるため、外部の比較ツールをインストールできない制限された環境でも、ファイルの変更履歴を詳細に調査できます。

## 注意点・補足

* **非対話実行**: `comp` は実行後に「さらに比較しますか？」と聞いてくるため、自動化スクリプトに組み込む際は注意が必要です（前述の `echo n |` 等を使用）。
* **ワイルドカード**: `comp *.txt *.bak` のように、複数のファイルを一括で比較することも可能です。
