# Command  - Linux - sudo - 他のユーザーとしてコマンドを実行する

## 概要

`sudo` (superuser do) は、一般ユーザーが、許可された別のユーザー（デフォルトでは`root`）の権限で、特定のコマンドを実行できるようにするための仕組みです。`root`のパスワードを共有することなく、**自身のパスワード**を使って一時的に管理者権限を得ることができます。

`sudo` は、**どのユーザーが、どのホストで、どのコマンドを誰として実行できるか**を `/etc/sudoers` というファイルで詳細に設定でき全ての実行履歴がログとして記録されるため、セキュリティと監査の観点から`su` に代わるモダンな標準的権限管理手法となっています。

(出自: `sudo` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| ⭐ **`su`** | **相手のパスワード**（例: `root`のパスワード）を使って、そのユーザーに完全に切り替わります。`sudo` は**自身のパスワード**で、**コマンド単位**の権限昇格を行います。 | `sudo` |
| **rootでの直接ログイン**| サーバーに直接`root`としてログインする旧来の方法。誰が何をしたかの追跡が困難であり、非常に危険です。`sudo` は、`root`ログインを無効化するための安全な代替手段です。 | `sudo` |

## よく連携されるコマンド

`sudo` は、管理者権限が必要なほぼ全てのコマンドと連携して使用されます。また、`sudo -l` は自身の権限を確認するために重要なコマンドです。

### シナリオ例: `sudo` を付け忘れたコマンドを再実行する (インフラ構築・運用視点)

* **目的**: 権限不足で失敗した直前のコマンドを、`sudo` を付けて素早く再実行する。
* **連携コマンド**: `!!` (history expansion)
* **解説**: シェルの履歴展開機能 `!!` は直前のコマンドを表します。`sudo !!` と入力することで、`sudo <直前のコマンド>` として実行でき、コマンドを再入力する手間を省きます。
* **コマンド例**:

    ```bash
    # sudoを付け忘れ
    apt update
    # -> E: Could not open lock file ... - open (13: Permission denied)

    # sudo !! で再実行
    sudo !!
    # -> sudo apt update が実行される
    ```

## オプション説明

| オプション | 説明 |
| :--- | :--- |
| **実行モード** | |
| ⭐ `<command>` | 指定したコマンドを `sudoers` ポリシーに従って実行します。 |
| ⭐ `-l`, `--list` | 現在のユーザーが `sudo` で実行可能なコマンドの一覧を表示します。 |
| ⭐ `-i` | `root` ユーザーの環境変数を読み込む**ログインシェル**を起動します (`su -` と同義)。 |
| `-s` | `root` ユーザーの**非ログインシェル**を起動します (`su` と同義)。 |
| **ユーザー指定** | |
| `-u <user>` | `root` 以外の、指定したユーザーとしてコマンドを実行します。 |
| **その他** | |
| `-k` | 認証のキャッシュを無効化し、次回 `sudo` 実行時に必ずパスワードを要求します。 |
| `-E` | 現在のユーザーの環境変数を引き継いでコマンドを実行します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: パッケージのインストールやサービスの再起動など、管理者権限が必要な操作を行う。
* **組み合わせ**: `sudo <command>`
* **解説**: システムに変更を加えるための最も基本的な使い方です。
* **例**:

    ```bash
    # パッケージをアップデート
    sudo apt update && sudo apt upgrade -y

    # nginxサービスを再起動
    sudo systemctl restart nginx
    ```

### 2. ブルーチーム視点

* **タスク**: `sudo` の実行ログを分析し、不審な特権昇格がないか調査する。
* **組み合わせ**: `journalctl | grep "sudo"` または `cat /var/log/auth.log`
* **解説**: `sudo` の実行履歴は、誰が、いつ、どのディレクトリで何のコマンドを特権で実行したかを追跡するための重要な監査証跡です。認証の失敗や意図しないコマンドの実行がないかを確認します。
* **例**:

    ```bash
    # ジャーナルログからsudoの実行履歴を確認
    sudo journalctl | grep "sudo"
    ```

### 3. レッドチーム視点

* **タスク**: **権限昇格の足がかりを探す**ため、現在のユーザーに付与された`sudo`権限を調査する。
* **組み合わせ**: `sudo -l`
* **解説**: 攻撃者は侵入後、即座に `sudo -l` を実行します。もし `(ALL) ALL` が許可されていれば `sudo su` で即座にrootになれます。特定のコマンドのみが許可されている場合は、**GTFOBins** を参照し、そのコマンドを悪用して権限昇格を試みます。
* **例**:

    ```bash
    # 現在のユーザーがsudoで何を実行できるか確認
    sudo -l
    ```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `user is not in the sudoers file. This incident will be reported.`
    * **考えられる原因**: コマンドを実行したユーザーは、`/etc/sudoers` ファイルで `sudo` の使用が許可されていません。
    * **解決策**: 管理者に依頼して、`visudo` コマンドで `sudoers` ファイルにユーザーを追加（または `sudo` グループに追加）してもらう必要があります。

## 環境変数と設定ファイル

* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

### 関連する設定ファイル

* ⭐ **`/etc/sudoers`**: `sudo` のメイン設定ファイル。
* ⭐ **`/etc/sudoers.d/`**: このディレクトリに置かれたファイルも `sudoers` の設定として読み込まれます。ユーザーごとや役割ごとの設定を分割して管理するのに便利です。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `sudo` 自体の脆弱性（例: Baron Samedit - CVE-2021-3156）と、**`/etc/sudoers` の設定ミス**がリスクとなります。
* **悪用シナリオ**: 「最小権限の原則」に反した、過度に寛容なルール（例: `user ALL=(ALL) /usr/bin/find`）は、GTFOBins に記載されているような手法で権限昇格に悪用されます。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**
* **解説**: `sudo` は、GTFOBins にリストされているほぼ全てのコマンドの権限昇格ベクトルです。GTFOBinsは、あるコマンドに `sudo` 権限が与えられていた場合、どのようにしてそこからシェルを奪取できるかの具体的な手法をリストアップしています。
* **参照**: `https://gtfobins.github.io/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: **`visudo` を必ず使用する**。**最小権限の原則**を徹底し、`ALL` の使用は極力避け、ユーザーには本当に必要なコマンドのフルパスのみを許可する。
* **Detection (検知)**: `sudo` の実行ログをSIEMに連携し、不審な特権コマンドの実行（特に認証失敗）を監視する。FIMツールで `sudoers` ファイルの変更を監視する。

## 注意点・補足

* **`visudo` の使用**: `/etc/sudoers` ファイルを編集する際は、必ず `visudo` コマンドを使用してください。`visudo` は、保存時にファイルの構文チェックを行い、エラーがあれば保存を中止してくれます。
