# Command - Windows - cmd - expand - 圧縮ファイルの展開

## 概要

`expand` コマンドは、1 つ以上の圧縮された配布用ファイル（主に `.cab` 形式や、末尾がアンダースコア `_` で終わる `.ex_`, `.dl_` などの形式）を展開します。
Windows のインストールメディアや更新プログラム、古いドライバパッケージなどで使用されている形式を扱うための標準ツールです。

(出自: `Windows 標準搭載 - 外部コマンド (expand.exe)`)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `compact` | NTFS ファイルシステムレベルでの圧縮を管理する。 | ディスク節約 |
| `makecab` | ファイルを Cabinet (`.cab`) 形式に圧縮（作成）する。 | 圧縮作成 |
| `Expand-Archive` | PowerShell 版。`.zip` 形式の展開に使用する。 | ZIP展開 |
| `extrac32` | 内部的に Cabinet ファイルを展開するためのレガシーなツール。 | - |

## よく連携されるコマンド

### シナリオ例: CAB パッケージ内からの特定ファイル抽出 (運用・調査)

* **目的**: 巨大なアップデートパッケージ（.cab）の中から、特定の DLL だけを取り出してバージョンを確認する。
* **連携コマンド**: `expand`, `dir`, `fc`
* **解説**: `-f` スイッチを使用して、展開するファイルをフィルタリングします。
* **コマンド例**:
    ```cmd
    REM cabinet_file.cab 内のすべての .dll ファイルを C:\temp に展開
    expand cabinet_file.cab -f:*.dll C:\temp
    ```



## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `-d` | **【重要】** ファイルを展開せず、ソースファイルに含まれるファイルの一覧のみを表示する。 |
| `-f:files` | ソースから展開する特定のファイルを指定する（ワイルドカード可）。 |
| ⭐ `-r` | 展開後のファイル名を自動的にリネームする（例: `test.dl_` -> `test.dll`）。 |
| `-i` | ディレクトリ構造を無視して展開する。 |
| `source` | 展開元のファイル名を指定する。 |
| `destination` | 展開先のパスを指定する。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 古いドライバのバックアップ（.dl_ 等の形式）から、本来の DLL ファイルを復元して手動でインストールする。
* **例**:
    ```cmd
    REM ドライバーファイルをリネームしつつ展開
    expand driver.dl_ -r C:\Windows\System32\drivers
    ```

### 2. ブルーチーム視点 (フォレンジック/マルウェア解析)

* **タスク**: 不審なインストーラーや CAB ファイルの中身を、**実行（インストール）せずに**解体して中身を静的に調査する。
* **例**:
    ```cmd
    REM パッケージの中身をリストアップして不審なファイル名がないか確認
    expand -d suspicious.cab
    ```

### 3. レッドチーム視点 (偵察とペイロード準備)

* **タスク 1: Static Analysis of Installers (インストーラーの静的解析)**
    * **目的**: MSI や CAB 形式で配布されている商用ソフトウェアの中に、権限昇格のヒントになる設定ファイルや、脆弱な古い DLL が隠されていないか探る。
* **タスク 2: Payload Delivery (隠蔽されたペイロードの展開)**
    * **目的**: 攻撃ツールを `.cab` 形式に偽装して転送し、ターゲット端末上で `expand` を使って元の実行ファイルに戻す。
    * **解説**: `.zip` ほど一般的でない形式を使うことで、一部の簡易的な検知ルールを回避できる場合があります。
    * **コマンド例**:
        ```cmd
        REM 偽装されたデータから実行ファイルを復元
        expand malicious_data.cab -f:exploit.exe C:\Windows\Temp
        ```

## エラーメッセージとトラブルシューティング

1.  **Can't open input file: [filename]**
    * **原因**: 指定したソースファイルが存在しない、またはアクセス権がない。
    * **解決策**: ファイルパスを確認し、必要に応じて管理者権限で実行してください。

2.  **Destination is not a directory**
    * **原因**: 複数のファイルを展開しようとしているが、展開先に既存の「ファイル」を指定してしまっている。
    * **解決策**: 展開先をフォルダパスに変更するか、あらかじめフォルダを作成してください。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **Path Traversal (パストラバーサル)**:
    * 悪意を持って作成された CAB ファイルを展開する際、ファイル名に `..\..\` などの相対パスが含まれていると、予期しないシステムディレクトリにファイルが上書き配置されるリスクがあります。
* **マルウェアの展開**:
    * 攻撃者は、アンチウイルスによるスキャンを避けるため、悪意あるコードを Cabinet ファイル内に圧縮して隠し、実行の直前に `expand` で展開する手法をとることがあります。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `AWL Bypass`, `Compression`
* **特性**: Windows 標準の信頼されたバイナリであるため、これを使用してファイルを操作しても不自然なログとして残りにくい特性があります。
* **参照**: [LOLBAS - expand.exe](https://lolbas-project.github.io/lolbas/Binaries/Expand/)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**:
    * **イベントID 4688**: `expand.exe` が不審なディレクトリ（`%TEMP%`, `C:\Users\Public` 等）で実行されていないか監視。
    * **引数の監視**: `-r` スイッチで拡張子を `.dl_` から `.dll` へ変更しているような挙動を注視します。

## 注意点・補足

* **書式**: 多くの CMD コマンドが `/`（スラッシュ）をスイッチに使いますが、`expand` は伝統的に `-`（ハイフン）を使用します。
* **cabarc との違い**: より詳細な CAB 操作が必要な場合は、かつてのリソースキットに含まれていた `cabarc` などのツールが必要でしたが、現在は `expand` と `makecab` で事足りることがほとんどです。
