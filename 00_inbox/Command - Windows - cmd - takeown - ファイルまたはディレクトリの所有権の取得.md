# Command - Windows - cmd - takeown - ファイルまたはディレクトリの所有権の取得

## 概要

`takeown` は、管理者が以前にアクセスを拒否されたファイルの所有権を取得し、そのファイルへのアクセスを回復できるようにするコマンドです。
Windowsでは「所有者」であれば、自分自身のアクセス権（ACL）を自由に変更できるため、システム管理やフォレンジック、権限昇格のステップとして多用されます。
(出自: Windows標準搭載)



## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 (あれば) |
| :--- | :--- | :--- |
| `icacls` | アクセス制御リスト（ACL）を変更する。所有権の変更も可能だが、`takeown` の方が「強制的な奪取」に向いている。 | `icacls` (権限付与) |
| `subinacl` | 旧式のリソースキットツール。より高度な所有権・権限の一括置換が可能。 | `takeown` (標準利用) |

## よく連携されるコマンド

`takeown` で所有者になっただけでは、まだ「読み取り」や「書き込み」の権限が付与されていない場合があります。そのため、通常は `icacls` とセットで実行されます。

### シナリオ例: アクセス拒否されたフォルダの完全開放 (インフラ構築・保守)

* **目的**: 前任者が作成し、誰も開けなくなったフォルダを管理者が強制的に開く。
* **連携コマンド**: `icacls`
* **解説**: 1. 所有権を奪う → 2. 自分にフルコントロール権限を与える。
* **コマンド例**:
    ```cmd
    REM 1. フォルダとその中身すべての所有権を取得
    takeown /f "C:\Archive\Confidential" /r /d y
    
    REM 2. 自分（現在のユーザー）にフルコントロール権限を付与
    icacls "C:\Archive\Confidential" /grant %USERNAME%:F /t
    ```

## スイッチ/オプション説明

| スイッチ | 説明 |
| :--- | :--- |
| ⭐ `/f <FileName>` | 所有権を取得するファイルまたはディレクトリのパスを指定する。 |
| ⭐ `/r` | **Recursive (再帰)**。指定したディレクトリとその中のすべてのファイル、サブディレクトリを対象にする。 |
| ⭐ `/a` | 所有権を「現在のユーザー」ではなく「**Administrators グループ**」に付与する。 |
| ⭐ `/d <Y/N>` | ユーザーがディレクトリの一覧表示権限を持っていない場合の既定の応答を指定（Y: 取得する、N: スキップ）。 |

## よく使われる組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: 退職者のプロファイルデータの整理
* **解説**: 元ユーザーにしかアクセス権がなかったデータを、管理者が一括で引き継ぐ際に使用します。
* **例**: `takeown /f C:\Users\RetiredUser /r /a`

### 2. ブルーチーム視点

* **タスク**: マルウェアが保護しているディレクトリの解析準備
* **解説**: マルウェアが ACL を操作して管理者すら見られないようにしたフォルダの所有権を奪い返し、中身をスキャンできるようにします。

### 3. レッドチーム視点 (OSCP / PrivEsc)

* **タスク**: `SeTakeOwnershipPrivilege` を利用した権限昇格
* **解説**: `whoami /priv` でこの特権が有効な場合、本来読めないシステムファイル（例: `SAM`, `SYSTEM` ハイブや設定ファイル）の所有権を奪い、内容を窃取します。
* **例**:
    ```cmd
    REM 重要な設定ファイルの所有権を奪取
    takeown /f C:\Windows\System32\config\SAM
    ```

## エラーメッセージとトラブルシューティング

### よくあるトラブル

1.  **現象**: `エラー: アクセスが拒否されました。`
    * **考えられる原因**: 実行ユーザーに「ファイル所有権の取得」特権がない、またはコマンドプロンプトが管理者として実行されていない。
    * **解決策**: 管理者として実行する。

2.  **現象**: `/r` オプションで途中で止まってしまう。
    * **原因**: 非常に深い階層や、システムによってロックされているファイルが含まれている。
    * **解決策**: `/d y` オプションを付けて、プロンプトを自動回答させる。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **悪用シナリオ**: 
    - **Data Exfiltration**: 攻撃者が管理者権限（または特定の特権）を得た後、機密データフォルダの所有権を `takeown` で奪い、中身をすべて持ち出す。
    - **Denial of Service (DoS)**: 重要なシステムバイナリの所有権を奪い、ACL を書き換えて OS 自体を起動不能にする。

### LOLBAS (Living Off The Land Binaries and Scripts) における利用例

* **機能**: `Ownership change`
* **参照**: [LOLBAS - takeown.exe](https://lolbas-project.github.io/lolbas/Binaries/Takeown/)

### 対応策・緩和策 (ブルーチーム視点)

* **Detection (検知)**: 
    - **イベントID 4612**: オブジェクトの所有権が変更された。
    - 重要なシステムパス（`C:\Windows\System32` 等）に対して `takeown.exe` が実行されたことを EDR で検知する。

## 注意点・補足

* **元に戻せない**: 所有権を変更すると、元の所有者が誰だったかの情報は上書きされます。監査上の理由で元の状態を知る必要がある場合は、実行前に現在の所有者を確認しておく必要があります。
