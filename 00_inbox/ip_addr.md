# `ip addr` - ネットワークアドレスを管理する

## 概要

`ip addr` (または `ip address`) は、`ip` コマンドスイートの一部であり、ネットワークインターフェイスに割り当てられた**IPアドレス**を表示、追加、削除するためのコマンドです。旧世代の `ifconfig` コマンドの主要な代替となるもので、IPv4とIPv6の両方のアドレスを扱うことができます。システムのIPアドレスを確認したり、一時的に静的IPアドレスを設定したりする際に使用します。

(出自: `iproute2` パッケージに含まれる)

## 類似コマンドと差異

| 類似コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- |:---|
| **`ifconfig`** | **旧世代のコマンド**。`ip addr` は、より詳細な情報（スコープなど）を表示でき、単一のインターフェイスに複数のアドレスを割り当てる際の管理も柔軟です。 | `ip addr` |
| **`ip link`** | **ネットワークデバイス自体**（L2層）の状態（UP/DOWN、MACアドレスなど）を管理します。`ip addr` はそのデバイスに割り当てられる**IPアドレス**（L3層）を管理します。 | - |
| **`hostname -I`**| ホストに割り当てられている**全てのIPアドレス**をスペース区切りでシンプルに表示します。手早くIPの一覧だけが欲しい場合に便利です。 | - |

## よく連携されるコマンド

`ip addr` は、ネットワークのトラブルシューティングにおいて、`ping` や `ip route` と連携して使われます。

### シナリオ例: ネットワーク接続の基本調査 (インフラ構築・運用視点)

* **目的**: サーバーのネットワーク接続に問題がある場合に、基本的な設定を確認する。
* **連携コマンド**: `ip addr`, `ip link`, `ip route`, `ping`
* **解説**: まず `ip addr` でIPアドレスが正しく割り当てられているか、`ip link` でインターフェイスが `UP` 状態かを確認します。次に `ip route` でデフォルトゲートウェイを確認し、最後にそのゲートウェイに `ping` が通るかをテストする、という一連の流れで問題を切り分けます。
* **コマンド例**:

```bash
# このコマンドシーケンスで基本的なネットワーク調査を行う
# 1. IPアドレスとインターフェースの状態を確認
ip addr show dev eth0

# 2. デフォルトゲートウェイを確認
ip route show

# 3. ゲートウェイ (例: 192.168.1.1) への疎通を確認
ping -c 4 192.168.1.1
```

## オプション説明 (`ip addr` のアクション)

| オプション | 説明 |
| :--- | :--- |
| **表示** | |
| ⭐ `show` (or `list`, `ls`) | (デフォルト) アドレス情報を表示します。`ip addr show dev eth0` で特定のデバイスに絞れます。 |
| **設定** | |
| ⭐ `add <ip>/<prefix> dev <dev>` | デバイスに新しいIPアドレスを追加します。 |
| `del <ip>/<prefix> dev <dev>` | デバイスからIPアドレスを削除します。 |
| `flush dev <dev>` | デバイスから全てのIPアドレスを削除します。 |

## よく使われるオプション組み合わせと業務シナリオ

### 1. インフラ構築・運用視点

* **タスク**: サーバーに割り当てられている全てのIPアドレスを確認する。
* **組み合わせ**: `ip addr show` (省略形: `ip a`)
* **解説**: ネットワークトラブルの際に、まずこのコマンドを実行して、IPアドレスが正しく割り当てられているか、インターフェイスが `state UP` になっているかを確認します。
* **例**:

```bash
# eth0インターフェースに一時的にセカンダリIPアドレスを追加
sudo ip addr add 192.168.1.101/24 dev eth0
```

### 2. ブルーチーム視点

* **タスク**: ホストのネットワーク構成を把握し、異常がないか確認する。
* **組み合わせ**: `ip addr show`
* **解説**: インシデントレスポンスの初期調査で `ip addr show` を実行し、意図しないIPアドレスが設定されていないか、あるいはインターフェイスが `promiscuous` モード（盗聴モード）になっていないかなどを確認します。
* **例**:

```bash
# 全インターフェースの情報を表示し、<PROMISC> フラグがないか確認
ip addr
```

### 3. レッドチーム視点

* **タスク**: 侵入後の偵察で、ネットワーク環境をマッピングする。
* **組み合わせ**: `ip addr show`
* **解説**: 攻撃者は侵入後、`ip a` を実行して、そのホストがどのようなサブネットに属しているかを把握します。例えば、`10.10.0.5/16` というアドレスが見つかれば、`10.10.0.0/16` のネットワーク全体がラテラルムーブメント（横展開）の対象になりうると判断します。
* **例**:

```bash
# 接続されているネットワークセグメントをリストアップ
ip addr | grep "inet "
```

## エラーメッセージとトラブルシューティング

* 一般的なエラーは [Linux共通のトラブルシューティング](./troubleshooting_common_errors.md) を参照。

1. **エラーメッセージ例 1**: `RTNETLINK answers: Operation not permitted`
    * **考えられる原因**: IPアドレスの追加 (`add`) や削除 (`del`) など、設定を変更する操作を一般ユーザー権限で実行しようとしました。
    * **解決策**: コマンドの前に `sudo` を付けて実行してください。

## 環境変数と設定ファイル

* `ip addr` の動作に直接影響する環境変数は通常ありません。
* 一般的な環境変数・設定ファイルは [Linux共通の環境変数・設定ファイル](./environment_and_config.md) を参照。

## セキュリティに関する考慮事項

### 脆弱性と悪用事例

* **脆弱性**: `ip addr` 自体ではなく、**不適切なネットワーク設定**がリスクとなります。
* **悪用シナリオ**: 攻撃者がroot権限を奪取した場合、`ip addr add/del` を使ってIPアドレスを変更できます。これにより、IPアドレスの重複によるサービス妨害や、中間者攻撃の準備などを行う可能性があります。

### GTFOBins / LOLBAS における利用例

* **機能**: **Privilege Escalation**, **File Read/Write**
* **解説**: もしユーザーが `sudo ip` を実行できる権限を持っている場合、`-f` オプションでスクリプトファイルを読み込ませる機能などを悪用して、任意のコマンドを実行したり、ファイルを読み書きしたりすることが可能です。
* **参照**: `https://gtfobins.github.io/gtfobins/ip/`

### 対応策・緩和策 (ブルーチーム視点)

* **Prevention (予防)**: 最小権限の原則に従い、一般ユーザーに `ip` コマンドの `sudo` 権限を安易に与えない。
* **Detection (検知)**: `auditd` を使って、`netlink` 経由でのネットワーク構成の変更を監視する。定期的に `ip addr` の出力を取得し、ベースラインと比較して意図しない変更がないかを確認する。

## 注意点・補足

* **設定の永続化**: `ip addr` コマンドで行った設定変更は、**OSを再起動すると失われます**。設定を永続化させるには、各ディストリビューションのネットワーク管理設定ファイル（`/etc/netplan/`など）を編集する必要があります。
