# Command - Linux - lsof - 開いているファイルをリスト表示する

## 概要
`lsof` は、システム上で「開かれているファイル」の情報を一覧表示します。Linux においては、通常のファイルだけでなく、ディレクトリ、ネットワークソケット、パイプ、デバイスなども「ファイル」として扱われるため、システム全体の動向を把握するのに極めて強力なツールとなります。



## 🛠 調査ツールの比較
| コマンド | 調査の切り口 | 特徴 |
| :--- | :--- | :--- |
| **`lsof`** | **ファイル/ポート中心** | 「このファイルを誰が使っているか」に強い。 |
| `fuser` | ファイル中心 | 特定のファイルにアクセスしている PID を特定・殺す。 |
| `ss` / `netstat` | ネットワーク中心 | ネットワーク接続の状態に特化。 |
| `ps` | プロセス中心 | プロセスの実行状態や親子関係に特化。 |

## 主要なオプション
※詳細な情報を得るには、通常 **root 権限 (sudo)** が必要です。

| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-i` | **最頻用。** インターネット（ネットワーク）ソケットを表示。 |
| ⭐️ `-i :<PORT>` | 指定したポート（例: `-i :80`）を使用中のプロセスを表示。 |
| ⭐️ `-p <PID>` | 特定のプロセス ID が開いているファイルをすべて表示。 |
| ⭐️ `-u <USER>` | 特定のユーザーが開いているファイルをすべて表示。 |
| ⭐️ `+D <DIR>` | 指定したディレクトリ配下のファイルを開いているプロセスを再帰表示。 |
| `-t` | プロセス ID (PID) のみを表示（スクリプトでの利用に最適）。 |
| `-n` | ホスト名の解決をスキップし、IP アドレスで表示（動作が速くなる）。 |
| `-P` | ポート番号をサービス名（http 等）に変換せず、数字のまま表示。 |

## 💡 出力項目の読み解き
* **COMMAND**: 実行コマンド名。
* **PID**: プロセス ID。
* **USER**: 実行ユーザー。
* **FD (File Descriptor)**: ファイル記述子。
    * `cwd`: カレントディレクトリ、`txt`: プログラムテキスト（実行ファイル本体）。
    * `0u`: 標準入力、`1u`: 標準出力、`2u`: 標準エラー出力。
* **TYPE**: ファイルの種類（`REG`: 通常ファイル、`DIR`: ディレクトリ、`IPv4`: ネットワーク接続等）。
* **NODE**: i-node 番号。
* **NAME**: ファイル名、または接続先。

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: 「Device is busy」でディスクをアンマウントできない時の調査。
* **解説**: そのディスク内のファイルを「誰かが掴んでいる」ためにアンマウントが拒否されています。犯人を特定して対処します。
* **例**:
    ```bash
    # /mnt/data 配下を掴んでいるプロセスを特定
    sudo lsof +D /mnt/data
    ```

### 2. ブルーチーム視点
* **タスク**: 不審なアウトバウンド通信の検知。
* **解説**: ネットワーク接続の一覧から、意図しないポートで外部（C2サーバー等）と通信しているプロセスがないか、バイナリの場所を含めて特定します。
* **例**:
    ```bash
    # 確立されたすべてのネットワーク接続を IP 表示で確認
    sudo lsof -i -n -P | grep ESTABLISHED
    ```

### 3. レッドチーム視点
* **タスク**: 削除された機密データの復元（Live Forensics 回避）。
* **解説**: 攻撃者が証拠隠滅のためにファイルを `rm` しても、プロセスがそのファイルを掴み続けている限り、`/proc/<PID>/fd/` 経由で中身を読み取ることができます。
* **例**:
    ```bash
    # 削除されたがプロセスに掴まれている (deleted) ファイルを探す
    sudo lsof | grep deleted
    ```

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **情報の露出**: `lsof` の出力には、開いているファイルパスに含まれる機密情報や、接続先の内部 IP などが含まれます。一般ユーザーが不必要に実行できる状態は、偵察の足がかりになります。

### 対応策・緩和策 (ブルーチーム視点)
* **削除済みファイルの監視**: `(deleted)` ファイルが異常に多い場合、ログの隠蔽や、削除された実行ファイルがメモリ上で動き続けている（ファイルレスマルウェア）可能性があります。
* **特権の制限**: `lsof` 自体の実行を制限するよりも、プロセス情報が集約されている `/proc` ディレクトリのパーミッションを適切に管理（`hidepid` オプション等）することが根本的な対策になります。

## 注意点・補足
* **動作の重さ**: `-i` オプションなどで名前解決を伴う場合、ネットワーク環境によっては非常に動作が遅くなります。急いでいる時は常に **`-n -P`** をセットで使うのがプロの作法です。
