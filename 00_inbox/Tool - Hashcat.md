# Hashcat

Hashcatは、GPUを利用した高速なオフライン・パスワード解析ツールであり、OSCP試験や実戦での資格情報クラックにおいて中心的な役割を果たします。以下に主要な機能をまとめます。

### 1. パスワード攻撃モード (`-a`)

Hashcatは複数の攻撃手法をサポートしており、状況に応じて使い分けます。

- **Wordlistモード (`-a 0`):** 辞書ファイル（rockyou.txt等）を使用した攻撃。最も一般的です。
    
- **Bruteforce / Maskモード (`-a 3`):** 特定のパターン（マスク）に基づいた総当たり攻撃 。
    
- **Hybridモード (`-a 6` / `-a 7`):** 辞書ファイルにマスクを組み合わせる手法。例: `辞書単語 + 数字4桁` 。
    

### 2. 主要なハッシュモード (`-m`)

ターゲットから窃取したハッシュの種類に応じて、正しいモード番号を指定する必要があります。

|**ハッシュ種別**|**モード番号 (-m)**|**用途・例**|
|---|---|---|
|**MD5**|`0`|古いシステムやWebアプリのハッシュ。|
|**MD5crypt**|`500`|Linuxの古いパスワードハッシュ。|
|**NTLM**|`1000`|Windowsのローカルハッシュ（SAM）。|
|**NetNTLMv2**|`5600`|ネットワーク越しに窃取したWindowsハッシュ。|
|**Kerberos AS-REQ**|`7500`|Responder等で取得した事前認証ハッシュ。|
|**Kerberos AS-REP**|`18200`|AS-REP Roastingで取得したハッシュ。|

### 3. マスク攻撃の構文

マスクモード (`-a 3`) では、以下の記号を使用してパスワードの構造を指定します 。

- `?l` : 小文字 (a-z)
    
- `?u` : 大文字 (A-Z)
    
- `?d` : 数字 (0-9)
    
- `?s` : 記号
    
- **実行例:** `hashcat -m 1000 hashes.txt -a 3?u?l?l?d?d?d?d` (大文字1つ、小文字2つ、数字4つのパターンを解析) 。
    

### 4. 辞書攻撃とルール (`-r`)

辞書ファイルにマングリング（加工）ルールを適用して、解析の成功率を高めます。

- John the Ripperのルールと互換性がありますが、Hashcatではルールを別ファイル（`.rule`）に保存して指定する必要があります。
    
- **実行例:** `hashcat -a 0 -m 0 hashes.txt wordlist.txt -r custom.rule`。
    

### 5. 特定のターゲット解析例

- **KeePass:** `keepass2john` で抽出したハッシュをHashcatでクラック可能です。
    
- **Active Directory:** `InternalMonologue` 等で生成したNetNTLMv2ハッシュの解析に使用されます 。
    
- **Kerberos:** `Rubeus` 等で取得したハッシュを解析する際は、モード `18200` (AS-REP) や `13100` (Kerberoasting) が利用されます。
    

### 6. 運用上のヒント

- **ファイルのクリーンアップ:** 解析後にハッシュを削除する場合は `--remove` オプションを使用します 。
    
- **出力管理:** 解析結果を指定ファイルに保存するには `-o output.txt` を使用します 。
    
- **ユーザー名の処理:** ハッシュファイルにユーザー名が含まれている場合は `--username` フラグを指定します。
