# Command - Linux - last - 最近ログインしたユーザーのリストを表示する

## 概要
`last` は、システムに最後にログインしたユーザーの履歴を新しい順に表示します。ユーザー名、ログインした端末（tty/pts）、接続元の IP アドレスまたはホスト名、ログイン時刻、およびセッションの継続時間を一覧化します。また、システムの再起動（reboot）の履歴も記録されているのが特徴です。
(出典: `util-linux` パッケージ等)



## 類似・関連コマンド
| コマンド | 差異 / 使い分けのポイント | 推奨 |
| :--- | :--- | :--- |
| `lastb` | ログインに「失敗」した記録（`/var/log/btmp`）を表示する。 | **不正アクセス試行の調査** |
| `lastlog` | 各ユーザーが「最後にいつログインしたか」の最新1件のみを表示。 | 全ユーザーの生存確認 |
| `who` | 「今現在」ログインしているユーザーを表示。 | リアルタイム監視 |
| `w` | `who` に加え、何を実行しているかまで表示。 | ログイン者の動向調査 |

## よく連携されるコマンド
### シナリオ例: 特定のユーザーのログイン履歴を抽出 (運用・調査視点)
* **目的**: ユーザー `alice` が先週いつ作業していたかを調べる。
* **連携**: `grep`
* **解説**: `last` の全出力から特定の文字列をフィルタリングします。
* **コマンド例**:
    ```bash
    # alice の履歴だけを抽出
    last | grep "alice"
    ```

## 主要なオプション
| オプション | 説明 |
| :--- | :--- |
| ⭐️ `-n <N>` | 直近の `N` 行のみを表示する。 |
| ⭐️ `-f <FILE>` | 指定したファイル（過去の wtmp のバックアップ等）を読み込む。 |
| ⭐️ `-F`, `--fulltimes` | ログイン・ログアウトの時刻を年月日・秒まで詳細に表示する。 |
| ⭐️ `-i`, `--ip` | ホスト名の代わりに接続元の IP アドレスを表示する。 |
| `-a`, `--hostlast` | ホスト名（IP）の列を最後尾に表示する（長いホスト名で見切れるのを防ぐ）。 |
| `-R` | ホスト名（IP）を表示しない。 |

## 業務シナリオ別活用法

### 1. インフラ構築・運用視点
* **タスク**: サーバーの再起動履歴の確認。
* **解説**: `last reboot` を実行することで、過去の再起動のタイミングを把握し、ハードウェアトラブルや計画外のダウンタイムを特定します。
* **例**:
    ```bash
    # システムの再起動履歴のみを表示
    last reboot
    ```

### 2. ブルーチーム視点
* **タスク**: 異常な時間帯や場所からのログイン検知。
* **解説**: 深夜時間帯のログインや、海外の IP アドレスからの接続がないかを精査します。
* **例**:
    ```bash
    # 詳細な時刻と IP アドレスを表示して精査
    last -Fi
    ```

### 3. レッドチーム視点
* **タスク**: 管理者のアクティビティ監視。
* **解説**: 侵入後、管理者が頻繁にログインする時間帯を把握し、その時間を避けて活動することで検知リスクを下げます。また、自分のログイン痕跡が残っていることを確認し、後で `utmpdump` 等を用いて痕跡を消去（Log Cleaning）する計画を立てます。

## エラーメッセージとトラブルシューティング

1.  **「last: /var/log/wtmp: No such file or directory」**:
    * **原因**: ログファイルが存在しない、またはディレクトリが異なる。
    * **解決策**: OS ごとにログの場所が異なる場合があります。`/var/log/account/wtmp` など他の場所を探すか、`-f` で指定してください。

2.  **「still logged in」と表示される**:
    * **状況**: ユーザーがまだログアウトしていない、または異常終了してログアウトが記録されなかった。

## セキュリティに関する考慮事項

### 脆弱性とリスク
* **Log Tampering (ログ改ざん)**: `last` が参照する `/var/log/wtmp` はバイナリファイルですが、root 権限があれば改ざん可能です。攻撃者は侵入の証拠を消すために、このファイルから自分のエントリだけを削除することがあります。

### 対応策・緩和策 (ブルーチーム視点)
* **Immutable Logs**: ログをリモートの syslog サーバーや、書き換え不可能な SIEM に即座に転送する設定を導入します。
* **File Integrity Monitoring (FIM)**: `wtmp` ファイルの変更を監視し、予期せぬ改ざん（エントリの削除など）を検知します。

## 注意点・補足
* **ログの回転（ローテーション）**: `/var/log/wtmp` は定期的に `wtmp.1` などにローテーションされます。古い記録を見たい場合は `last -f /var/log/wtmp.1` のように指定する必要があります。
