# # SEC560：Day 3講義記録

## 1. 認証情報の窃取とResponderの仕組み

Windowsネットワーク特有のプロトコル（LLMNR/NetBIOS）の脆弱性を突き、ユーザーのハッシュを強制的に送らせる手法です。

### Responderによる攻撃プロセス

- **原理:** Windowsマシンが名前解決に失敗した際、ネットワーク全体に「このサーバーを知りませんか？」と叫ぶ（ブロードキャスト）習性を利用します。
    
- **攻撃:** Responderが「私がそのサーバーです」と嘘の応答を返し、接続してきたクライアントに認証（NTLMv2ハッシュの送信）を要求します。
    
- **実習のポイント:** `responder -I eth0` を実行し、Windows側で存在しない共有フォルダ（例：`\\no-such-server`）にアクセスしてハッシュをキャプチャします。
    

### 防御策（最優先事項）

- **GPOによる無効化:** LLMNR、NBT-NS、mDNSを停止する。
    
- **SMB署名の強制:** リレー攻撃（盗んだハッシュを別のサーバーに転送してログインすること）を物理的に防ぎます。

## 2. エクスプロイトの分類とMetasploit

脆弱性を突いて「シェル（操作権限）」を得るプロセスです。

### エクスプロイトの種類

1. **サーバーサイド:** 公開サービス（Web, DB等）を直接攻撃。ネットワークの疎通性が鍵。
    
2. **クライアントサイド:** ブラウザやPDF閲覧ソフトを攻撃。フィッシング等の誘導が必要だが、境界防御を突破しやすい。
    
3. **ローカル特権昇格 (LPE):** 一般ユーザーから管理者（root/SYSTEM）へ権限を上げる。
    

### Metasploit / Meterpreter の構成

- **Staged Payload (ステイジャー):** 最初に送り込む小さなコード。メモリを確保し、後から本体（Stage）を呼び出す。検知されやすいが柔軟。
    
- **Meterpreter:** 強力なポストエクスプロイト用シェル。プロセス移行（migrate）やキーロガー、スクリーンショット撮影など、OSと高度に対話可能。

## 3. パスワードクラッキング（Hashcat）

盗み出したハッシュを、オフラインで元の平文パスワードに戻す作業です。

### Hashcatの主要モードと戦略

- **モード (-m):** NTLMなら `1000`、NTLMv2なら `5600` など、アルゴリズムごとに指定。
    
- **ルールベース攻撃 (-r):** `Password` → `P4ssw0rd!` のように、辞書単語を自動加工して試行。
    
- **GPUの活用:** CPUに比べ、数千個のコアを持つGPU（NVIDIA等）は解読速度が桁違いに速いため、実務では必須です。
    

|**攻撃モード**|**名称**|**内容**|
|---|---|---|
|**-a 0**|Straight|辞書にある単語をそのまま試す。|
|**-a 3**|Brute-force|全ての組み合わせを試す（短いパスワード用）。|
|**-a 6**|Hybrid|辞書単語 + マスク（例：`Password123`）。|

## 4. 侵害前提 (Assume Breach) と C2フレームワーク

「外壁はいずれ突破される」という前提に立ち、内部ネットワークでの活動に焦点を当てる現代的なアプローチです。

### C2（Command & Control）の役割

攻撃者が侵入成功後、継続的に指示を出し、データを持ち出すための司令塔です。

- **ビーコン (Beacon):** 感染端末が定期的に「指示はありますか？」とサーバーに問い合わせる通信。検知を避けるため、間隔をランダム化（ジッター）させます。
    
- **Sliver:** 明日の実習で使用する最新のC2フレームワーク。Go言語で書かれ、高度な回避機能を備えています。
