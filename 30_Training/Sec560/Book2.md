**GIAC試験対策 SEC560 Book 2 総合まとめ**

### 1. スキャンと列挙 (Scanning and Enumeration)

効率的なスキャンは「広さから深さへ」の段階的アプローチが基本です。

- **ポートスキャンの挙動**: TCPの3ウェイハンドシェイクにおいて、SYN-ACKは**Open**、RST-ACKは**Closed**（ホスト生存）、無応答やICMPエラーは**Filtered**（遮断）を意味します。
- **バージョン・OSスキャン**:
    - **Nmap**: `-sV`オプションでバージョン情報を取得します。OSの特定（`-O`）は、近年では信頼性が低いため、パケットの**TTL値（Windowsのデフォルトは128、Linux/UNIXの多くは64）**や、オープンポートの傾向（Linuxは22、Windowsは135/445など）から推測する方が確実です。
    - **Netcat**: バナーグラビングに使用します。`nc -nv target port` や `echo "" | nc -nvw2 target port-range`（2秒のタイムアウト設定）で自動化が可能です。
- **視覚的スキャン（Webトリアージ）**:
    - **GoWitness**: Go言語で書かれたツールで、NmapやNessusのXMLを読み込み、Webサイトのスクリーンショットやヘッダー情報を取得し、SQLiteデータベースに保存します。
    - **EyeWitness**: Pythonおよび.NETで提供され、スクリーンショットの取得に加えて**デフォルト認証情報（既知のパスワードなど）の特定**に優れています。
- **ネットワーク機器の特定**: Cisco Discovery Protocol (CDP) や Link Layer Discovery Protocol (LLDP) は、OSバージョンやVLAN情報を漏洩するため、VLANホッピング攻撃の足がかりになります。

### 2. 脆弱性スキャンとNSE (Nmap Scripting Engine)

- **脆弱性の確認手法**: ソフトウェアのバージョン確認（誤検知が多い）、プロトコルバージョン、挙動の確認、設定の確認、エクスプロイトの実行などの方法があります。
- **スキャンの種類**: ペネトレーションテストでは**非認証スキャン**が最も一般的ですが、クライアントサイドソフトウェアの脆弱性は特定できません。
- **Nmap Scripting Engine (NSE)**: Lua言語で記述されており、Nmapの機能を大幅に拡張します。`-sC`オプションでデフォルトカテゴリのスクリプトを実行し、`--script=`で個別指定が可能です。カテゴリには、`auth`, `brute`, `vuln`, `safe` (ターゲットに影響を与えない), `intrusive` (ログを残す・クラッシュの恐れがある) などが存在します。

### 3. パスワード攻撃

パスワード攻撃は、現在でも最も成功率の高い侵入手法の一つです。

- **オンライン攻撃 vs オフライン攻撃**: パスワード推測（オンライン）は**アカウントロックアウトのリスク**が常に伴いますが、パスワードクラッキング（オフライン）はハッシュを取得した上で自身のハードウェアで行うため、そのリスクはありません。
- **スプレー攻撃とスタッフィング**:
    - **パスワードスプレー**: 1つのパスワード（季節＋年など）を多数のアカウントに試す手法で、ロックアウトを回避しやすいのが特徴です。
    - **クレデンシャルスタッフィング**: 外部の漏洩データ（DeHashedなどから取得）から得たIDとパスワードのペアを再利用して不正アクセスを試みます。
- **ロックアウト回避の原則**: Windowsのロックアウトポリシー（しきい値、観察ウィンドウ、期間）を考慮し、**「観察ウィンドウ（カウンターがリセットされる時間）あたり1回の推測に留めること」**が最も安全なアプローチです。
- **ツール**:
    - **pw-inspector**: パスワードポリシーに基づいてワードリストをフィルタリングします。`-m`（最小文字数）、`-M`（最大文字数）、`-c`（必須条件の数）、`-l/u/n/p/s`（小文字/大文字/数字/記号等）を指定して無駄な推測を減らします。
    - **Hydra**: 多様なプロトコルに対応するパスワード推測ツール。`-l` (単一ユーザ) / `-L` (ユーザリスト), `-p` (単一パスワード) / `-P` (パスワードリスト) を使用します。ドメインを指定する場合は `-m workgroup:{DOMAINNAME}` を付与します。

### 4. Azure (Entra ID) 環境への攻撃

- **Azureのエンドポイント列挙**: 非認証で有効なユーザーを列挙できる2つの主要なエンドポイントが存在します。
    - `GetCredentialType`: 有効なユーザーの場合は`0`、無効な場合は`1`を返します。ただし、リクエスト過多で**スロットリング**が発生すると偽陽性が出るため、`ThrottleStatus`が`2`でないことを確認する必要があります。
    - `OAuth Token`: パスワード推測も同時に可能。有効なユーザーでパスワードが間違っている場合は**50126**、ユーザーが存在しない場合は**50034**のエラーコードを返します。
- **レガシー認証によるMFAバイパス**: IMAP、POP3、Exchange ActiveSync (EAS) などの基本認証（Basic Auth）を使用するレガシープロトコルはMFAをサポートしていないため、これらが有効な環境ではMFAを回避して認証できる可能性があります。
- **Entra Smart Lockout**: Azure上の自動ブルートフォース保護機能。既定で全テナントに有効化されており、10回の失敗でロックアウトエラー（**AADSTS50053**）を返します。この状態になると、**パスワードの正誤に関わらず同じエラーが返る**ため、攻撃ツールはこれを検知して停止する必要があります（実際にはAzure上のアカウント自体が完全にロックされるわけではありません）。
- **回避ツール**: IPアドレスをベースに判定されるため、**TrevorSpray**や**Spray365**を使用し、複数のSSHプロキシやAPIゲートウェイを介して**IPアドレスをローテーションさせる**ことが有効です。

### 5. NTLMv2攻撃とResponder

KerberosはSPNとホスト名に依存するため、IPアドレス（例: `\\192.168.10.10`）を使用したアクセスなどではNTLMv2へフォールバックします。

- **ハッシュの取得とクラッキング**: ネットワーク上の認証をSniffingし、**PCredz**を使用してpcapファイルからハッシュを抽出し（`NTLMv2.txt`などに出力）、**Hashcat（モード5600）**でオフラインクラックします。
- **Responderの使用**: NBT-NS、LLMNR、mDNSといったブロードキャスト/マルチキャストの名前解決プロトコルを悪用し、攻撃者のマシンを正規のサーバーと偽装してターゲットにNTLMv2のチャレンジ/レスポンス認証を行わせます。
    - **WPADの悪用**: ブラウザの「プロキシ設定を自動的に検出する」機能が有効な場合、Responderは`wpad`の解決要求に応答し、トラフィックを傍受したり、NTLM認証を強制したりできます。
- **SMBリレー攻撃**: Responderで傍受した認証要求を別のサーバーに中継（リレー）し、パスワードを知らなくてもそのサーバーへのアクセス権を得る攻撃です。
- **防衛策**: NBT-NS/LLMNRの無効化、SMB署名の有効化（リレー攻撃を防止）、プロキシ自動検出の無効化などが挙げられます。
- **ADIDNS（Active Directory Integrated DNS）**: Responderは通常ブロードキャストドメイン内に限定されますが、Powermadツール（`Enable-ADIDNSNode -Node *`）を使用してワイルドカードDNSレコードを作成することで、企業全体のどのサブネットからでもトラフィックをResponderへ誘導させることが可能になります。

### 6. エクスプロイトとMetasploit Framework

- **エクスプロイトの3大カテゴリ**:
    1. **サーバーサイド（サービスサイド）**: リッスンしているサービスへの攻撃。ファイアウォールで許可されている必要があります。
    2. **クライアントサイド**: ユーザーが攻撃者の用意したコンテンツ（Webやファイル）にアクセスすることで発火します。Googleの**「Rule of 2（2つの法則）」**によれば、「信頼できない入力」「安全でない言語（C/C++）」「サンドボックスの欠如」の3つのうち全てが揃うソフトウェアには必ず脆弱性が存在します。
    3. **ローカル権限昇格 (PrivEsc)**: すでに得ている制限付き権限（一般ユーザー）から、rootやSYSTEM権限へ昇格する攻撃（競合状態、カーネル攻撃、SUIDの悪用など）です。
- **Metasploitの構造**:
    - **exploits**: ターゲットの脆弱性を突くコード。OSごとに分類され、`windows/smb`（PsExecなど含む）は特に強力です。Rapid7による成功率のランク（ExcellentからManual）が存在します。
    - **payloads**:
        - **singles (単一/stageless)**: エクスプロイトと通信機能がすべて1つにパッケージ化されたもの。
        - **stagers**: ターゲットのメモリに展開され、後続のstageを呼び出すための小さなコード（`reverse_tcp`、`bind_tcp`など）。
        - **stages**: 実際の機能を提供するペイロード（`meterpreter`、`shell`、`vncinject`など）。
- **Meterpreterの主要コマンド**:
    - **プロセス管理**: `getpid`, `getuid`, `ps`, `migrate`（別のプロセスに移動。移動先は同等以下の権限である必要あり）。
    - **ファイル・コンソール操作**: `upload`, `download`, `edit`（viエディタ風）, `screenshot`, `keyscan_start/keyscan_dump`（キーストロークの取得。デスクトップにアクセスできるプロセスにmigrateする必要あり）。
    - **ネットワーク・ピボット**: `portfwd`（ターゲットをTCPリレー化する）。※Metasploit（msfconsole）側の`route`コマンドを使用すると、確立したMeterpreterセッションを介して、ターゲットの別サブネット全体へ後続のエクスプロイトをルーティング（ピボット）できます。
    - **モジュール追加**: `use priv` のように実行することで、セッション確立後でも機能を動的に拡張可能です。
