# # SEC560：Day 2 講義記録

## 1. ネットワークスキャニングの戦略

効率的なスキャンは「広さから深さへ」の段階的アプローチが基本です。

### スキャンワークフロー

1. **ホスト発見:** どのIPが生きているか（Nmap -Pn, -sL）。
    
2. **ポートスキャン:** どのサービスが開いているか。
    
3. **サービス・バージョン特定:** ソフトウェア名とバージョンの特定（Nmap -sV）。
    
4. **脆弱性スキャン:** 既知の弱点の検索（NSEスクリプト, Nessus等）。
    

### ツールの使い分け

- **Nmap (同期型):** 精度が高く、詳細な調査に向く。`-oA`で全形式保存が鉄則。
    
- **MassScan (非同期型):** 圧倒的に高速。数百万IPの広範囲スキャンに最適だが、ステートレスなため精度はNmapに譲る。

## 2. TCP/IPとポートスキャンの挙動

スキャン結果の解釈には、TCP 3ウェイハンドシェイクの理解が不可欠です。

|**応答パケット**|**意味**|**状態**|
|---|---|---|
|**SYN-ACK**|接続を受け入れた|**Open**|
|**RST-ACK**|接続を拒否した|**Closed** (ホストは生存)|
|**無応答 / ICMP Error**|遮断されている|**Filtered**|

## 3. 視覚的スキャンと脆弱性分析

多数のWebサイトを1つずつブラウザで開くのは非効率なため、自動化ツールを活用します。

- **GoWitness / Eyewitness:** NmapのXMLを読み込み、全Webサイトのスクリーンショットを自動生成。管理画面や古いログインページを一目で特定可能。
    
- **NSE (Nmap Scripting Engine):** `-sC`（デフォルト）や特定カテゴリ（`vuln`など）を指定して、軽量な脆弱性調査を並行実施。
    
- **OS特定:** `nmap -O`より、**TTL値**（Linux=64, Windows=128）やバナー情報の方が信頼性が高い。

## 4. 初期アクセス：パスワード攻撃

現代のペネトレーションテストにおいて、最も成功率の高い侵入手法の一つです。

### 攻撃手法の分類

- **パスワードスプレー:** 1つの「ありがちなパスワード（Spring2026!など）」を多数のユーザーに試す。ロックアウトを回避しやすい。
    
- **クレデンシャルスタッフィング:** 過去の漏洩データから得た「ID:PW」のペアを試す。
    

### 実務上の注意点

- **ロックアウト回避:** クライアントのパスワードポリシー（何回でロックするか）を事前に確認し、ジッター（時間差）を設けて実行する。
    
- **PWInspector:** ターゲットの文字数制限や複雑性ルールに合わないパスワードをリストから除外し、無駄な試行を減らす。

## 5. Azure (IntraID) への攻撃

クラウド環境（M365/Azure）特有の偵察・攻撃手法です。

- **ユーザー列挙:** `getCredentialType` エンドポイントを叩き、応答値（0=存在する, 1=存在しない）で有効なメールアドレスを特定。
    
- **MFAバイパス:** IMAP/POP3などの「基本認証（Basic Auth）」が有効な場合、MFAを回避してログインできる可能性がある。
    
- **スマートロックアウト対策:** Azureは「IPアドレス」をベースにロックをかけるため、**IPローテーション**（複数のプロキシやクラウドのIPv6範囲を利用）が有効な回避策となる。
