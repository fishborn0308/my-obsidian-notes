# # SEC560：Day 4講義記録

## 1. 次世代C2フレームワーク：Sliver

Metasploit（Meterpreter）の一歩先を行く、レッドチーム仕様のC2です。

- **マルチプレイヤーモード:** チーム全体で1つのインプラント（潜入ソフト）を共有・操作可能。
    
- **ガードレール機能:** 「特定のドメイン環境以外では動かない」「指定日時を過ぎたら自壊する」といった設定が可能で、意図しない拡散を防ぎます。
    
- **Armory（拡張機能）:** `armory install` で、メモリ内で動作する強力なツール群（例：Seatbelt, SharpUp）を即座にロードできます。

## 2. 状況認識（Post-Exploitation）

侵入直後、システムが「どのような環境か」を把握するフェーズです。

### Linuxの探索

- **権限の確認:** `cat /etc/group` でsudo権限を持つユーザーを特定。
    
- **設定ミス:** `find / -perm -2` で誰でも書き込み可能なファイルを探し、設定改ざんの足掛かりにします。
    
- **GTFO Bins:** `find` や `vi` など、特定のバイナリが持つ「シェル起動機能」を悪用してroot昇格を狙います。
    

### Windowsの探索とSeatbelt

- **自動化ツール:** `Seatbelt.exe` を使用し、インストールされているソフト、最近の実行履歴、ブラウザの保存情報などを一括収集します。
    
- **LoLBins:** `bitsadmin` や `certutil` など、OS標準の署名済みバイナリ（Living Off The Land）を用いて検知を回避しながらファイルを転送します。

## 3. Active Directory特権昇格の「三種の神器」

AD環境において、一般ユーザーからドメイン管理者（Domain Admin）を目指すための核心的な攻撃手法です。

### ① ケルベロスティング (Kerberoasting)

- **原理:** サービスアカウント（SPN設定済み）のパスワードハッシュが含まれる「サービスチケット」を要求し、それをオフラインで解読（Hashcat等）します。
    
- **狙い目:** ユーザーが手動で設定した「脆弱なパスワード」を持つサービスアカウントが標的です。
    

### ② BloodHound (攻撃パスの可視化)

- **役割:** AD内の膨大な「関係性（ユーザー、グループ、権限）」をグラフ理論で分析します。
    
- **価値:** 「ユーザーAはグループBに属し、グループBはコンピュータCの管理者権限を持ち、コンピュータCにはドメイン管理者がログイン中である」といった、人間では気づけない**最短の攻撃ルート**を自動で見つけ出します。
    

### ③ ADCS（証明書サービス）の悪用：ESC1 / ESC4 / ESC8

ADと統合された証明書発行機能（PKI）の設定ミスを突きます。

- **ESC1:** 証明書申請時に「他人の名前（SAN）」を勝手に名乗れる設定ミス。ドメイン管理者の証明書を偽造できます。
    
- **ESC4:** 低権限ユーザーが「証明書テンプレートの設定」を書き換えられるミス。一時的に脆弱な設定（ESC1状態）に変更して証明書を盗み、後で元に戻します。
    
- **ESC8 (NTLMリレー):** HTTP経由の証明書登録画面へ、盗んだ認証情報をリレーして不正に証明書を発行させます。

## 4. ペイロードの巧妙な配信手法

防御側を欺くための「コンテナ」の活用です。

- **ISO / VHDファイル:** 近年のWindowsではダブルクリックでマウントされますが、この中のファイルには「インターネットから来た」というマーク（MotW）が付かないため、セキュリティ警告を回避しやすくなります。
    
- **LNKファイル:** ショートカットに見せかけて、裏で複雑なPowerShellコマンドやC2ペイロードを実行させます。

### 【実習のアクションヒント】

1. **Sliverの安定性:** `sliver-server` を立ち上げたら、まず `operator` を作成してクライアントから接続する「マルチプレイヤー構成」に慣れてください。
    
2. **ADCSの証跡管理:** ESC4の実習でテンプレートを書き換えた後は、**必ず元の設定に復元する**手順（`CertiPy`の自動復元機能など）を確認してください。
    
3. **BloodHoundのデータ抽出:** `SharpHound.exe` を実行する際は、大量のトラフィック（LDAPクエリ）が発生するため、EDR等に検知される可能性があることを念頭に置いてください。
