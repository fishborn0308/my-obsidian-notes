# # SEC560：Day 5 講義記録

## 1. 横方向の移動（ラテラルムーブメント）

1つの足掛かりからネットワーク全体へ支配を広げるプロセスです。

### Windowsにおける戦略

- **認証情報の再利用:** `Pass-the-Hash (PTH)` を使い、平文パスワードを知らなくてもハッシュだけで別システムへログインします。
    
- **リモート実行ツール:**
    
    - **PSExec:** サービスを作成して実行。強力ですがEDRに検知されやすい。
        
    - **WMI / WinRM:** OS標準の管理機能を悪用。正規の管理活動に紛れ込みやすい。
        
- **永続化（Persistence）:** 再起動後もアクセスを維持するため、**スケジュールされたタスク**や**WMIイベントコンシューマー**（例：ログイン失敗をトリガーにシェルを起動）を設定します。
    

### Linuxにおける戦略

- **SSHキーの窃取:** ユーザーディレクトリから秘密鍵を盗み、パスワードなしで他サーバーへログインします。
    
- **SSHポートフォワーディング:**
    
    - **ローカル (-L):** 自分のPCから、ピボット（中継）ホスト経由で内部サーバーへ接続。
        
    - **ダイナミック (-D):** SOCKSプロキシを作成し、ブラウザやProxychainsで内部サイトを閲覧。

## 2. 高度なActive Directory支配（ゴールデン/シルバーチケット）

KRBTGTアカウントのハッシュを手に入れた攻撃者は、もはや「神」の権限を持ちます。

- **ゴールデンチケット:** ドメイン全体の偽造TGT。有効期限を10年に設定することも可能で、パスワードを変更されてもKRBTGTハッシュが変わらない限り有効です。
    
- **DC Sync:** 他のドメインコントローラー（DC）になりすまし、全ユーザーのハッシュをレプリケーション（複製）して一気に盗み出します。
    
- **DC Shadow:** 一時的に「偽のDC」として振る舞い、ログを残さずにADオブジェクト（グループ所属など）を書き換えます。

## 3. Azure (IntraID) とクラウドの攻略

オンプレミスからクラウドへ、あるいはクラウド内での権限昇格です。

- **Managed Identitiesの悪用:** 侵害したAzure VM上で**インスタンス・メタデータ・サービス (IMDS)** にクエリを投げ、そのVMに付与された認証トークンを盗み出します。これにより、VMがアクセス権を持つストレージやDBへ侵入できます。
    
- **コントロールプレーン経由の実行:** `az vm run-command` を使い、ネットワーク的に隔離されたVMに対しても、Azureの管理機能を介して直接スクリプトを実行できます。

## 4. プロフェッショナル・レポートの極意

ペネトレーションテストの価値は、ツールを動かすことではなく「報告書」で決まります。

- **エグゼクティブサマリー:** 経営層向け。技術用語を排除し、「ビジネスリスク」と「解決のための投資先」を1.5ページ以内にまとめます。
    
- **「二日酔い」を避ける:** 最終日にまとめて書くのではなく、実習や発見の**その瞬間に**スクリーンショットとメモを残してください。
    
- **根本原因の分析:** 「パッチが未適用」と書くだけでなく、「パッチ管理プロセスに不備がある」といった組織的な課題を指摘します。
